# Codex 桌面版 26.616.71553 macOS / Windows 对照

> 第三方跨平台变更日志 · 非官方

本批次的实质变化集中在 macOS arm64 包；Windows x64 MSIX 输入样本的 from 与 to 完全相同，只能确认该样本无内容变化。  
One-liner: macOS moved forward with app UI and bundled browser plugin updates, while the Windows sample is byte-identical across the compared pair.

## 关于这份报告

本报告只使用官方签名分发包逆向重建后的机器提取事实，逐组盘点版本与渠道、Bundle 文件树、内置插件、Computer Use 运行时、`cua_node` 依赖、`app.asar`、前端模块、可读字符串、平台 manifest、CSP、资源与关键文件 diff。

证据分两档：【实证】来自 plist、manifest、插件清单、服务入口、技能文档、依赖清单、签名信息、关键文件 diff，以及不属于“同名前端模块改动线索”的可读字符串；【信号】来自同名前端模块体积、模块名、样式类名与该区块内的新增/移除字符串。平台标记为 `[macOS+Windows]`、`[仅 macOS]`、`[仅 Windows]`。

对比范围分别是 macOS `26.616.51431 build 4212` → `26.616.71553 build 4265`，以及 Windows `26.616.6631.0` → `26.616.6631.0`。Windows 样本两端 SHA-256 相同，不能据此扩展为 Windows 未获得后续变化，只能说本输入样本无内容变化。

## ✨ 重点变化

### 【实证】[仅 macOS] 版本与签名状态更新

macOS arm64 从 `26.616.51431` 升至 `26.616.71553`，build 从 `4212` 升至 `4265`。`Info.plist` 中 `CFBundleShortVersionString`、`CFBundleVersion` 和 `ElectronAsarIntegrity/Resources/app.asar/hash` 均发生变化。

两个 macOS 全量包均通过公证验证，来源为 `Notarized Developer ID`，签名主体为 `Developer ID Application: OpenAI OpCo, LLC (2DC432GLL2)`。

Windows 对比样本的上一版和最新版均为 `26.616.6631.0`，SHA-256 均为 `b572c1820575f005f1492dd61276d1ee935e70f4f0c85f64c2d631438fde533e`，文件内容无变化。

### 【实证】[仅 macOS] 内置 browser 与 chrome 插件版本同步

macOS 包内两个 OpenAI bundled plugin 的 manifest 版本号同步更新：

- `plugins/openai-bundled/plugins/browser/.codex-plugin/plugin.json`: `26.616.51431` → `26.616.71553`
- `plugins/openai-bundled/plugins/chrome/.codex-plugin/plugin.json`: `26.616.51431` → `26.616.71553`

`browser` 插件 manifest 仍描述为 in-app browser，用于打开、检查、导航、测试、点击、输入或截图本地 Web 目标，例如 `localhost`、`127.0.0.1`、`::1`、`file://` URL 或当前内置浏览器标签页。`chrome` 插件 manifest 仍指向依赖用户现有 Chrome 状态的自动化，包括标签页、登录会话、cookie、扩展和 Codex Chrome Extension 设置。

Windows 样本没有对应内容变化。

### 【信号】[仅 macOS] 设置页扩展到权限、上下文用量、工作模式、语音与窗口默认行为

`general-settings` 前端模块体积从约 `0.0 MB` 增至 `0.1 MB`，增加 `120320 bytes`。新增字符串和 aria 标签指向多个设置面：

- disabled default permissions toggle
- composer context usage indicator setting toggle
- work mode settings options
- code across chats and diffs 的 base size
- hold-to-dictate、realtime voice、wake word
- hotkey window capture input
- Popout Window projectless default setting toggle
- dictation dictionary entry

这属于同名前端模块改动线索，只能说明设置 UI、可访问性标签与状态结构发生变化，不能断言所有开关已对用户可见或行为已启用。Windows 样本无对应变化。

### 【信号】[仅 macOS] Markdown 渲染与引用交互增强

`markdown` 模块增加 `72518 bytes`，新增字符串集中在文件引用、媒体 fallback、表格复制与“Add to chat”：

- file citation chip，包括带位置信息、无扩展名文件、无扩展名且带位置信息的情况
- markdown image/video 的无 alt 文本、加载中和加载失败 fallback
- `Copy table`
- `Add to chat`
- 将当前选中文本从 overlay 加入聊天的按钮标签
- `data-markdown-copy`、`data-markdown-image-preview-trigger`、`data-markdown-table`

这是 Markdown 内容交互与可访问性方向的前端信号。Windows 样本无对应变化。

### 【信号】[仅 macOS] 首页公告横幅新增远程连接、多智能体、速率限制与 Fast mode 入口线索

`home-announcements` 模块增加 `29452 bytes`。新增字符串显示首页 banner 相关状态扩展：

- Codex rate limit reset home banner
- Fast mode home banner
- backend-driven Codex app home banner
- multi-agent composer banner
- remote connections banner above the home composer
- `Add Connections`
- ambient suggestions connected apps home banner
- available Codex rate limit resets modal
- `has-seen-fast-mode-home-banner`
- `has-seen-multi-agent-composer-banner`

这些是首页公告、连接入口、用量重置、多智能体 composer 与 Fast mode 的 UI 信号，不构成实际功能发布范围的实证。Windows 样本无对应变化。

### 【信号】[仅 macOS] RPC 层增加 stream stub、disposed 状态与 promise export 复用错误路径

`rpc` 模块增加 `56900 bytes`。新增可读字符串包括 RPC stub 不能作为 constructor、disposed 后访问、ReadableStream/WritableStream stub 禁止访问属性或调用方法、无 RPC session 不能创建 pipes，以及 peer promise 复用 export ID 的系统错误。

这说明前端打包内进入了更丰富的 RPC 错误路径和 stream stub 处理信号。Windows 样本无对应变化。

### 【信号】[仅 macOS] 插件详情页分享与 MCP enablement 文案减少

`plugin-detail-page` 模块减少 `133332 bytes`。移除字符串包括 plugin share dialog 的用户/组搜索、移除 workspace share target、`Add plugin`、`Add people or groups`、workspace link sharing，以及 MCP server enablement toggle 的 accessible label。

这只是同名前端模块的移除线索，说明插件详情页相关分享、workspace link 与 MCP enablement UI 文案在该 bundle 中减少或重组；不能单独断言功能被移除。Windows 样本无对应变化。

### 【实证】[仅 macOS] 主程序二进制体积增加，Computer Use 运行时和 `cua_node` 依赖未变

macOS `Contents/Resources/codex` 从 `245.2 MB` 增至 `249.3 MB`，增加 `4.11 MB`。`app.asar` 维持约 `155.8 MB`，事实包显示 `+0.00 MB`。

Computer Use 运行时 `cua_node` 总量为 `377.4 MB` → `377.4 MB`，新增依赖包 `0` 个。Windows 的 `cua_node` 总量为 `298.9 MB` → `298.9 MB`，新增依赖包同样为 `0` 个。
