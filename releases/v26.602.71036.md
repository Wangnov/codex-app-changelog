# Codex 桌面版 26.602.71036(macOS / Windows 对照)

> 第三方跨平台变更日志 · 非官方

本批次显示 Codex 桌面版在 macOS 与 Windows 上同步推进到同一轮应用层变更，但平台版本号与可读证据密度不同。  
One-liner: this cross-platform batch looks like a coordinated app-layer refresh with no new files or new cua_node dependencies on either platform.

## 关于这份报告

本报告基于两份官方签名分发包的逆向 diff(差异比对):macOS 取 Sparkle(应用更新框架)全量包，Windows 取 MSIX(Windows 应用安装包)包，对各自上一版与最新版进行文件树、清单、app.asar(Electron 应用归档)和插件目录对照。

可信度分两档:【实证】表示来自 plist(属性列表)、AppxManifest(Windows 应用清单)、技能文档、类型声明、package.json(包配置文件)、依赖清单等可读证据;【信号】表示仅来自前端模块名、文件体积或构建产物变化，只能说明方向，不能还原源码意图。

平台归属标记含义: `[macOS+Windows]` 表示两平台都有同类变化; `[仅 macOS]` 表示目前只在 macOS 事实包中出现; `[仅 Windows]` 表示目前只在 Windows 事实包中出现。

对比范围:macOS 为 26.602.40724 build 3593 → 26.602.71036 build 3685; Windows 为 26.602.4764.0 → 26.602.9276.0。本批次发布日期按输入事实使用 2026-06-07。

## ✨ 重点变化

### 版本与构建号推进【实证】[macOS+Windows]

macOS 的 Info.plist 显示 `CFBundleShortVersionString` 从 `26.602.40724` 更新到 `26.602.71036`，`CFBundleVersion` 从 `3593` 更新到 `3685`。Windows 版本从 `26.602.4764.0` 更新到 `26.602.9276.0`。

平台差异在于版本命名:macOS 使用 short version 加 build 的组合，Windows 使用四段式版本号;两者对应同一批次，但数字不相同，不能混用。

### 应用层 app.asar 被重建【实证】[macOS+Windows]

两平台的 `app.asar` 都发生内容变化。macOS 的 `ElectronAsarIntegrity` hash 从 `0d60982859a8e93d9df7bd629e63cf70b2d83c482420bee78c3af36fb0bf5e67` 变为 `1eaacb8bd09c1647a501ac76f4718bc83badec23ad7160e1691ec161790790ab`，这是可读清单级证据。Windows 事实包也显示 `resources/app.asar` 参与本批次体积对照。

平台差异在于 macOS 提供了 asar 完整性 hash 的明确变更记录，Windows 事实包只给出文件级统计与体积项，未提供同等的清单 hash。

### 前端构建产物大量轮换，但不能据此判断功能【信号】[macOS+Windows]

macOS 的 app.asar 内部文件为 `2830 → 2830`，新增 `285`、移除 `285`、改动 `12`;Windows 为 `2778 → 2778`，新增 `285`、移除 `285`、改动 `11`。两份事实包都明确指出 `webview/assets` 下大量增减是 Vite(前端构建工具)构建 hash 重命名，属于噪音。

平台差异很小:两平台新增与移除数量相同，改动数 macOS 多 1 个;这更像同一轮前端构建产物刷新，而不是可直接命名的新功能。

### Browser 与 Chrome 插件随应用版本同步【实证】[仅 macOS]

macOS 的可读 diff 显示 `browser` 插件和 `chrome` 插件的 `.codex-plugin/plugin.json` 版本号从 `26.602.40724` 更新到 `26.602.71036`。这说明内置 Browser(内置浏览器自动化插件)与 Chrome(谷歌浏览器自动化插件)插件元数据跟随应用版本推进。

平台差异很重要:Windows 事实包列出了对应脚本文件参与体积对照，但没有给出 `plugin.json` 的可读 diff，因此只能确认 macOS 侧有这条实证，不能把同样的插件元数据变更写成 Windows 事实。

### 未新增 cua_node 依赖【实证】[macOS+Windows]

两平台的 `cua_node` 依赖变化均显示新增 `0` 个包。

平台差异:无。至少从依赖清单层面看，本批次没有引入新的 `cua_node` 包。

### 顶层文件树保持稳定【实证】[macOS+Windows]

macOS 顶层 bundle(应用包)文件数为 `2075 → 2075`，新增 `0`、移除 `0`、内容改动 `53`。Windows 文件数为 `8724 → 8724`，新增 `0`、移除 `0`、内容改动 `15`。

平台差异在于打包结构与文件数量:Windows 包含更多文件，macOS 本批次内容改动文件数更多;但两平台都没有顶层文件新增或移除。
