## 关于这份报告

这不是官方变更日志，而是一份**基于官方分发产物的逆向差异报告**。本次为历史回填：两个版本均已超出官方 Sparkle(应用自动更新框架)增量窗口，因此事实包采用两个官方全量包直接比较，而不是增量包重建。

对比流程来自机器提取事实：下载上一版与最新版官方全量包，基于 Apple(苹果)公证与 OpenAI Developer ID(开发者身份签名)代码签名作为信任根，再进行结构化 bundle(应用包)差异分析。事实包显示，两个版本的 `Codex.app` 均通过 `codesign / spctl` 相关公证验证，来源为 `Notarized Developer ID`，签名主体为 `Developer ID Application: OpenAI OpCo, LLC (2DC432GLL2)`。

本文使用两档证据等级：

- **【实证】**：来自人类可读的直接证据，例如 `Info.plist`、插件 `plugin.json`、配置 diff、签名与公证结果。
- **【信号】**：来自间接证据，例如前端模块名或二进制体积变化；这类证据只能说明“指向某方向，但无法从产物还原到源码级意图”。

对比范围：`26.601.20914` build `3497`，发布于 `2026-06-02`；到 `26.601.21317` build `3511`，发布于 `2026-06-03`。

## ✨ 重点变化

### 1. 主应用版本推进到 `26.601.21317` 【实证】

这是一个明确的版本更新：`CFBundleShortVersionString` 从 `26.601.20914` 变为 `26.601.21317`，`CFBundleVersion` 从 `3497` 变为 `3511`。

这意味着 macOS(苹果桌面系统) 层面能识别到新的应用版本与构建号；但仅凭这组键值，不能判断具体功能变化。

关键证据：

```diff
/CFBundleShortVersionString: '26.601.20914' → '26.601.21317'
/CFBundleVersion: '3497' → '3511'
```

### 2. 内置 Browser 与 Chrome 插件版本同步到新版 【实证】

内置 `browser` 与 `chrome` 插件的 `plugin.json` 中，插件版本号都从 `26.601.20914` 更新为 `26.601.21317`。

这说明这两个随应用打包的自动化插件已随主应用一起重新发布。事实包没有显示它们的描述、能力边界或配置发生变化，因此不能写成“新增浏览器能力”。

关键证据：

```diff

```

| 插件 | 证据文件 |
| --- | --- |

### 3. 主应用资源包内容发生小幅变化 【信号】

`Contents/Resources/app.asar` 从 `141.6 MB` 变为 `141.3 MB`，减少 `0.33 MB`；同时 `Info.plist` 中 `ElectronAsarIntegrity` 的 `app.asar` 哈希也发生变化。

这指向应用层资源内容确实重新构建或调整过，但无法从产物还原到源码级意图。事实包也明确提示，`webview/assets` 下大量增减属于 Vite(前端构建工具) hash 文件名重命名噪音，不能据此判断新增或删除功能。

关键证据：

```diff

'97e10d2004ddc2988ee1a145a5950be9e90e3a4c7c9957961e278490b47c4fd3'
```

`owl-electron-app.json` 中的 `runtimeArchiveSha` 发生变化，`runtimeName` 仍为 `owl`。

这说明打包时引用的 runtime archive(运行时归档) 指纹已经更新；但事实包没有提供 runtime 内部差异，因此不能进一步判断它带来了哪些行为变化。
