# Codex 桌面版 26.601.21317（macOS / Windows 对照）

> 第三方跨平台变更日志 · 非官方

Codex（OpenAI 的代码智能体桌面应用）本批次更像一次跨平台应用层打包刷新：macOS（苹果桌面操作系统）侧可确认更多运行时与插件元数据，Windows（微软桌面操作系统）侧目前只看到包级与 app.asar（Electron 应用归档）级变化。  
One-liner: A small cross-platform application-bundle refresh, with macOS exposing additional runtime and bundled-plugin version evidence.

## 关于这份报告

本文基于官方签名分发包逆向 diff（差异比较）：macOS 取 Sparkle（macOS 应用更新框架）相关全量包，Windows 取 MSIX（Windows 应用打包格式）包。对比范围为 macOS `26.601.20914` build `3497` → `26.601.21317` build `3511`，Windows `26.601.1994.0` → `26.601.2237.0`，批次日期按输入事实包归档为 `2026-05-30`。

证据等级分两档：【实证】表示来自 plist（macOS 应用信息清单）、插件元数据、依赖清单或 SHA-256（哈希摘要算法）等可读证据；【信号】表示只来自体积、文件数量或前端构建产物变化，能说明方向但不能还原源码意图。平台标记 `[macOS+Windows]` 表示两边都有同类证据，`[仅 macOS]` / `[仅 Windows]` 表示只在该平台事实包中可确认。`webview/assets` 与 `.vite/build` 的 hash（哈希）重命名按噪音处理，不作为功能增减。

## ✨ 重点变化

### 应用归档体积小幅下降【信号】[macOS+Windows]

两平台的主应用归档都减少了 `0.33 MB`：macOS `Contents/Resources/app.asar` 从 `141.6 MB` 到 `141.3 MB`，Windows `resources/app.asar` 从 `145.4 MB` 到 `145.1 MB`。这说明应用层打包内容发生变化，但单靠体积不能判断具体功能。

### 应用层内部文件净少 1 个【信号】[macOS+Windows]

macOS 的 app.asar 内部文件 `2804 → 2803`，新增 `428`、移除 `429`、改动 `13`；Windows 为 `2752 → 2751`，新增 `428`、移除 `429`、改动 `12`。两边模式高度一致，但事实包明确提示 `webview/assets` 下大量增减属于 Vite（前端构建工具）构建 hash 重命名，因此不能据此写成新前端功能。

### cua_node 依赖没有扩张【实证】[macOS+Windows]

两平台的 `cua_node`（随包原生依赖目录）新增包数量均为 `0`。这是一条跨平台一致的负向变化：至少从依赖清单看，本批次没有新增这一组原生依赖。

### macOS 包元数据与运行时指纹更新【实证】[仅 macOS]

macOS 的 `Info.plist` 显示版本从 `26.601.20914` 到 `26.601.21317`，build 从 `3497` 到 `3511`，`ElectronAsarIntegrity`（Electron 归档完整性记录）里的 app.asar hash 从 `8293044a58d98dd06f740b67b9cc68e26b0b74c476baa48caae1161cdd130356` 变为 `97e10d2004ddc2988ee1a145a5950be9e90e3a4c7c9957961e278490b47c4fd3`。

同平台的 `owl-electron-app.json` 还显示 `runtimeArchiveSha` 从 `9084289a644128f65ba44b97caa360ad87cc27291951f6bca141742ca4bb7035` 变为 `c6a2b52fed12c736be826f39d524cc764b3bd232ee0727c92e44626e662e06fe`。Windows 事实包没有给出同等级运行时清单差异，因此不能确认 Windows 是否已有对应变化。

### 内置 browser/chrome 插件版本跟随主版本【实证】[仅 macOS]

macOS 事实包中，`browser`（内置浏览器插件）和 `chrome`（Chrome 自动化插件）的 `.codex-plugin/plugin.json` 版本字段都从 `26.601.20914` 更新到 `26.601.21317`。可读 diff 只证明插件元数据版本跟随主应用版本，不证明插件能力本身发生变化。Windows 事实包未给出对应插件元数据 diff，因此这里按 `[仅 macOS]` 记录。

### Windows 侧未出现可读独有功能证据【实证】[仅 Windows]

本次提供的 Windows 事实包包含 SHA-256、外层文件树统计、app.asar 统计与 `cua_node` 新增包数量，但没有列出 AppxManifest（Windows 应用清单）、技能文档、类型声明、package.json（Node.js 包配置）或插件元数据层面的可读差异。因此本文不宣称任何 `[仅 Windows]` 的功能性变化。
