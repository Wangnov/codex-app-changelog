# Codex 桌面版 26.601.21317 macOS / Windows 对照

> 第三方跨平台变更日志 · 非官方

这一批次的共同主线是：插件入口和线程工具文案继续产品化，同时两端都换入新的 owl 运行时包。  
One-liner: this cross-platform build shifts plugin discovery toward work plugins, refreshes thread-tool wording, and updates the packaged owl runtime.

## 关于这份报告

本报告只使用输入事实包中的机器提取结果：macOS 对比 `26.601.20914 build 3497` → `26.601.21317 build 3511`，Windows 对比 `26.601.1994.0` → `26.601.2237.0`。两端版本号和发布时间样本不同，以下每条都按各自平台证据归属，不把一个平台的结论外推到另一个平台。

证据等级分两档：`【实证】` 来自 manifest、plist、package 配置、服务入口、技能说明、依赖清单、签名信息或事实包中不属于“同名前端模块改动线索”的可读字符串；`【信号】` 来自同名前端模块的模块名、体积变化、可读字符串差异和样式类名差异。平台归属使用 `[macOS+Windows]`、`[仅 macOS]`、`[仅 Windows]`。

本次证据盘点覆盖了版本与渠道、Bundle 文件树、内置插件清单、cua_node 依赖、app.asar 概览、前端模块真实增删、同名前端模块改动线索、主进程和前端新增可读字符串、Info.plist / AppxManifest、内容安全策略、新增组件与资源、关键文件改动。输入中没有给出 Computer Use 运行时总量、CSP 变化、AppxManifest 变化或新增内置插件明细，因此本文不补造这些结论。

## ✨ 重点变化

### 【信号】[macOS+Windows] 工作类插件公告取代单一 Sales 插件公告

两端 `app-main` 都新增了 `work-plugins-announcement` 相关文案，包括添加插件按钮、已添加插件状态、公告正文，以及 `Build deal materials faster`、`Create polished campaign visuals`、`Execute deals with confidence`、`Prepare sales work faster` 等短句。新增短描述明确点到 Creative Production、Investment Banking、Product Design，样式类名还出现了 `data-analytics@openai-curated-remote`、`product-design@openai-curated-remote`、`public-equity-investing@openai-curated-remote`。

对应地，两端都移除了 `Add Sales plugin`、`Meet your new AI sales assistant`、Sales 插件公告标题和主按钮 key，并把旧的 `has-seen-sales-plugin-announcement` 迁移为 `has-seen-work-plugins-announcement`。这说明前端公告面从单一 Sales 插件转向更宽的工作插件组合，但它仍属于同名前端模块线索，不能据此断言所有用户都已看到该公告。

### 【信号】[macOS+Windows] 调试面板显著扩展，覆盖 App Server、会话、浏览器标签和账户信息

两端 `debug-modal` 体积都从接近 0 MB 增至约 0.2 MB：macOS 增加 177,618 bytes，Windows 增加 175,508 bytes。新增可读字符串包括 `(command unavailable)`、`Account ID`、`Active browser tab`、`Active conversation`、`Allow host`、`Allow host starting`、`Already current`、`Ambient suggestion chats`、`App Actions`、`App Server`。Windows 还额外出现 `App session ID`。

这组变化指向调试面板覆盖范围扩大，特别是应用服务、当前对话、浏览器标签、账户与 host allow 流程。平台差异较小，Windows 多一个会话 ID 字符串。

### 【信号】[macOS+Windows] 线程动态工具文案去掉 “in the background”

两端 `app-server-dynamic-tools` 都把线程工具描述中的 `in the background` 去掉。变更覆盖归档/取消归档、列出最近线程、置顶/取消置顶、重命名线程、向既有线程发送 follow-up prompt。

这是一组明确进入前端 bundle 的文案变化，指向“后台线程操作”的表述被弱化为普通线程操作。由于证据来自同名前端模块差异，归类为信号。

### 【信号】[macOS+Windows] 创建聊天状态也去掉 “background chat”

两端 `split-items-into-render-groups` 都把 `Creating background chat`、`Created background chat`、对应进行中和完成标签，替换为 `Creating chat`、`Created chat`、创建 Codex chat 的进行中和完成标签。

这与动态工具文案一致：用户可见状态文案正在从 “background chat” 回到更中性的 “chat”。证据仍为前端模块差异。

### 【实证】[macOS+Windows] Browser 与 Chrome 内置插件版本同步到 26.601.21317

两端 `browser/.codex-plugin/plugin.json` 的 `version` 都从 `26.601.20914` 更新到 `26.601.21317`，插件描述仍指向 Codex 内置浏览器、本地地址、file URL、截图、点击、输入和测试等使用场景。

两端 `chrome/.codex-plugin/plugin.json` 也从 `26.601.20914` 更新到 `26.601.21317`，描述仍为面向远程 URL、依赖登录态或 profile 的页面、现有 Chrome 标签、cookies、扩展和 Codex Chrome Extension 设置。这是 manifest 级实证，但输入没有显示这些插件新增能力，只能确认版本元数据更新。

### 【实证】[仅 Windows] Computer Use 插件版本同步到 26.601.21317

Windows 的 `resources/plugins/openai-bundled/plugins/computer-use/.codex-plugin/plugin.json` 从 `26.601.20914` 更新到 `26.601.21317`，描述为从 Codex 控制 Windows 桌面应用。

macOS 输入的关键文件 diff 没有出现对应 Computer Use 插件 manifest 变化；不能据此判断 macOS 没有该插件，只能说本样本仅在 Windows 事实包中提供了这条插件版本实证。

### 【信号】[仅 macOS] 浏览器使用设置新增站点、下载、上传、密码管理与历史访问相关入口

macOS `browser-use-settings` 体积增加 62,967 bytes，新增可读字符串包括 `Access history without asking`、嵌入式浏览器设置面包屑、扩展设置、密码管理设置、站点设置、清除浏览数据选项显示/隐藏、不可用插件禁用 toggle，以及 `Add allowed domain`、`Add allowed download domain`、`Add allowed upload domain`、`Add blocked domain`。

Windows 输入没有列出对应模块变化。macOS 这组信号很集中，指向浏览器使用权限和站点规则配置被扩展，但仍不能据此断言设置页已经对所有用户开放。

### 【信号】[仅 macOS] @ mention 菜单新增 Computer Use、agents、MCP servers 与 Mac apps 分组线索

macOS `at-mention-list` 新增 `Computer use`、`Custom agents`、`Desktop apps`、`Live agents`、`MCP servers`、`Mac apps`、`No results`，以及当前 MCP server mention scope、文件搜索加载状态和空查询状态说明。

Windows 的同名模块方向相反：这些字符串被移除，体积减少 29,315 bytes。两端在 @ mention 菜单的这组线索出现明显分叉：macOS 像是在扩展入口，Windows 则像是在移除同一批入口文案。

### 【信号】[仅 Windows] 插件详情页移除添加、分享、启用和 App unavailable 相关线索

Windows `plugin-detail-page` 体积减少 112,310 bytes，移除了插件详情页 included resource 操作菜单、workspace share target 移除、MCP server enablement toggle、section navigation、`Add plugin`、`Adding plugin`、`Anyone in this workspace with the link`、`App unavailable` 等字符串，以及一组详情页样式类名。

macOS 输入没有列出同名模块变化。这表示 Windows 样本中的插件详情页 bundle 线索明显收缩，但不能仅凭体积和字符串判断最终交互被删除。

### 【信号】[macOS+Windows] 插件可用性新增管理员禁用状态

两端 `plugins-availability` 都新增 `Disabled by admin`、管理员禁用时的主按钮标签说明，以及 `settings.mcp.appConnectModal.disabledByAdminButton`。这是进入前端模块的可读字符串，说明插件连接/可用性路径新增了“被管理员禁用”的状态文案。

### 【信号】[macOS+Windows] 首页公告与 AppGen 分享对话框线索被收缩

两端 `home-announcements` 都移除了 Fast mode、后端驱动 Codex app 首页 banner、多 agent composer banner、remote connections banner 的 dismiss label，以及 `Add Connections` 和相关 banner body 文案。

两端 `appgen-share-dialog` 都移除了站点分享对话框中的 workspace 用户/组搜索、移除已选用户、保存中按钮、`Add people or groups`、workspace link、internet link、复制 live site URL、外部打开 live site 等字符串。由于这些都来自同名前端模块差异，应视为可见界面方向的信号，而不是功能删除实证。

### 【实证】[macOS+Windows] owl 运行时包更新，app.asar 同步缩小约 0.33 MB

macOS `Contents/Resources/owl-electron-app.json` 的 `runtimeArchiveSha` 从 `9084289a...` 更新为 `c6a2b52f...`。Windows `resources/owl-electron-app.json` 从 `8ee5ac24...` 更新为 `76a82b58...`，`.installed-owl-shell.json` 也更新了 archive URL、installedAt 和 archiveSha。

两端顶层 app.asar 都缩小 0.33 MB：macOS `141.6 MB` → `141.3 MB`，Windows `145.4 MB` → `145.1 MB`。这是运行时和包体层面的实证；输入未给出运行时内容级 diff，因此不展开功能判断。

### 【实证】[仅 macOS] Info.plist 版本和 app.asar 完整性 hash 更新

macOS `CFBundleShortVersionString` 从 `26.601.20914` 到 `26.601.21317`，`CFBundleVersion` 从 `3497` 到 `3511`，`ElectronAsarIntegrity/Resources/app.asar/hash` 也更新为新 hash。两版 notarization 验证均为 accepted，来源为 Notarized Developer ID，origin 为 OpenAI OpCo, LLC。

Windows 输入没有提供 AppxManifest 变化，因此本文不报告 Windows manifest 字段变化。
