# Codex 桌面版 26.513.20950 macOS / Windows 对照

> 第三方跨平台变更日志 · 非官方

本批次的有效变化全部来自 macOS arm64 样本；Windows x64 样本在本输入中前后包完全一致。

One-liner: macOS adds a reshaped Browser plugin, richer LaTeX tooling, spreadsheet file associations, and multiple UI/runtime signals, while the Windows sample is unchanged.

## 关于这份报告

本报告只使用两份事实包中的机器提取事实。macOS 对比范围是 26.506.31421 build 2620 → 26.513.20950 build 2816；Windows 对比范围是 26.506.3741.0 → 26.506.3741.0，SHA-256 也完全相同。

证据等级分两档：【实证】表示来自 manifest、plist、package.json、技能文档、服务入口、CSP、可读字符串或明确配置；【信号】表示来自前端模块名、压缩 bundle 线索、体积变化或同名模块字符串/样式差异。平台标记为 [macOS+Windows]、[仅 macOS]、[仅 Windows]。本输入样本没有可归入 [macOS+Windows] 的实际内容变化。

证据盘点已覆盖版本与渠道、Bundle 文件树、内置插件、Computer Use 相关运行时文件、cua_node 依赖、app.asar、前端模块增删、同名前端模块线索、主进程/前端新增字符串、Info.plist/AppxManifest、CSP、新增组件资源和关键文件 diff。Windows 事实包没有插件清单变化、AppxManifest 变化、CSP 变化、同名前端模块线索或关键文件 diff。

## ✨ 重点变化

### 【实证】[仅 macOS] 内置插件从旧命名迁移到新命名，并保留浏览器自动化能力面

macOS 内置插件清单从 `browser-use`, `chrome`, `computer-use`, `latex-tectonic` 变为 `browser`, `chrome`, `computer-use`, `latex`。这不是单纯体积变化：新增 `browser` 插件有 manifest、技能说明、默认提示和服务脚本支撑，版本为 `0.1.0-alpha2`，显示名为 `Browser`，短描述是用 Codex 控制内置浏览器。

`browser` 的说明明确覆盖打开、导航、检查、测试、点击、输入、截图，以及 localhost、127.0.0.1、::1、file:// 和当前内置浏览器页签。它还保留 `@browser`、`@browser-use`、`browser-use`、`Browser`、`in-app browser` 等别名，说明这是能力面迁移和命名整理，而不是浏览器能力删除。最大文件包括 `scripts/browser-client.mjs` 0.9 MB 以及 classic-level 多平台 native 预构建文件。

Windows 样本没有插件清单变化；不能据此说 Windows 已获得或未获得该插件改名，只能说本输入样本无变化。

### 【实证】[仅 macOS] LaTeX 插件升级为更完整的 TeX 工作流

macOS 新增 `latex` 插件，文件数 18，总大小 50.4 MB，版本 `0.2.0`，显示名 `LaTeX`。旧插件 `latex-tectonic` 被移除，新插件说明从“只编译 LaTeX/TeX 文档与 bundled Tectonic”扩展为：优先使用 bundled Tectonic，必要时回退到已检测到的 TeX Live 或 MacTeX，并可在没有可用系统安装时安装 Codex 管理的完整 TeX Live runtime。

技能文档显示三个明确入口：`latex-compile` 用 `scripts/compile_latex.py` 构建 TeX 项目；`latex-doctor` 用 `scripts/latex_doctor.py` 检测 Tectonic、TeX Live、MacTeX、latexmk、pdflatex、xelatex、lualatex、biber、kpsewhich；`texlive-runtime-installer` 默认只检测，只有用户确认后才运行 `--install-managed-full`。托管安装路径写明为 `~/.cache/codex-runtimes/codex-texlive/full`，且说明不使用 sudo、不写 `/Library/TeX`、不写 `/usr/local/texlive`、不修改 shell 启动文件。

Windows 样本无对应变化。

### 【实证】[仅 macOS] 电子表格文件关联新增 TSV/XLS/XLSM/XLSX 查看入口

macOS `Info.plist` 新增四类文档类型：TSV、XLS、XLSM、XLSX，角色均为 `Viewer`，处理优先级为 `Alternate`。其中 TSV 还声明 MIME `text/tab-separated-values` 与 UTI `public.tab-separated-values-text`；XLS/XLSM/XLSX 分别声明对应 Excel MIME。

这说明 macOS 包层面新增了电子表格类文件的打开/查看关联。它与前端 `runtime.worker`、`rate-limit-rows`、新增 `google-drive` 模块等信号可以互相呼应，但文件关联本身是 plist 级实证。

### 【实证】[仅 macOS] Electron 与本地依赖升级，构建脚本被整理

macOS `package.json` 显示应用版本从 `26.506.31421` 升到 `26.513.20950`，`codexBuildNumber` 从 `2620` 升到 `2816`。Electron 从 `41.2.0` 升到 `42.0.1`，`better-sqlite3` 从 `^12.8.0` 升到 `^12.9.0`。

开发脚本从直接串联 helper 构建改为 `scripts/prepare-dev.mjs`，新增 `forge:make:owl-shell` 和 `e2e:computer-use-native-pipe`。这说明 macOS 包内源码配置已经包含 Owl shell 打包路径和 Computer Use native pipe 的端到端测试入口。

### 【实证】[仅 macOS] Computer Use 增加锁屏守护相关组件，服务体积小幅增加

macOS 文件树显示 `SkyComputerUseService` 从 14.7 MB 增至 15.0 MB，新增 `CUALockScreenGuardian.app` 主二进制 14.7 MB 与其 `Assets.car` 3.0 MB，同时新增 `Codex Computer Use Installer.app` 的 `Assets.car` 3.0 MB。旧版中两个 `embedded.provisionprofile` 文件被移除。

这些是包内组件级事实，表明 Computer Use 运行时在 macOS 上新增锁屏守护/安装器资源。事实包没有提供 Windows Computer Use 运行时变化；cua_node 新增依赖为 0。

### 【实证】[仅 macOS] MCP app sandbox 增加 `navigate` 消息端口

macOS `.vite/build/sandbox-preload.js` 的端口名数组新增 `navigate`，原有 `notifyMcpAppsHostContext`、tool input/result、resource teardown、widget code、safe area、theme、widget data/view 等端口仍在。

这是 sandbox preload 层的明确通信能力变化，说明 MCP app sandbox 的宿主-访客消息通道新增了导航相关端口。注意这只能证明包内通信接口新增，不能单独推断完整 UI 是否对所有用户开放。

### 【实证】[仅 macOS] Webview CSP 未见策略放宽，只有资源入口 hash 更新

macOS `webview/index.html` 的入口脚本和 modulepreload 文件 hash 发生变化，`path-browserify` preload 被移除，`src` chunk hash 改名。CSP 内容仍为 `default-src 'none'`，图片允许 self/app/blob/data/https，child/frame 允许 web-sandbox 域，worker 允许 self/blob，script 允许 self、一个 sha256 和 `wasm-unsafe-eval`，connect 允许 self、`https://ab.chatgpt.com`、`https://cdn.openai.com`。

因此本批次在 macOS 输入样本中没有看到 CSP 放宽；主要是构建产物入口变化。

### 【实证】[仅 macOS] 远程控制、设备连接和 SSH 连接界面文案进入包内

新增前端模块包含 `device-connections`、`remote-control-connections-visibility`、`remote-projects`，同名模块 `remote-connections-settings` 新增大量字符串：从“Access and control other devices from this computer”到“Add SSH Connection”“Add device”，以及授权这台 Mac 或这台 Windows 设备控制其他设备的无障碍标题。

新增可读字符串还覆盖“Activar control remoto”“Afstandsbediening inschakelen/instellen/uitschakelen”“Accesso al dispositivo revocato”“Accesul dispozitivului a fost revocat”等多语言远程控制与设备撤销文案。这些字符串确凿进入 macOS 包内，但不等于所有用户都可见。

### 【实证】[仅 macOS] 用量、积分、自动充值和套餐比较界面增强

`usage-settings` 新增 “Add credits”“Auto-reload credits”“Auto-reload is on/off”“Automatic reload”“Automatically add credits when your balance runs low”等字符串，并新增 `https://developers.openai.com/codex/pricing`。`rate-limit-rows` 新增 “20x more usage than Plus”“5x more usage than Plus”“Choose Pro plan tier”“Compare personal and business plans”“Connect to Google Workspace”“Current plan”等字符串。

`app-main` 也新增 “Add credits”、从 core usage surface 添加积分或升级的按钮文案、free account footer 升级按钮。多语言 bundle 中出现 `GPT-5.5 Pro`、`GPT-5.5 Thinking`、购买积分、取消/升级订阅等文案。证据边界是：这些是包内字符串和 UI 模块变化，不能推断具体计费策略已经对所有账号生效。

### 【实证】[仅 macOS] 插件市场管理增加管理员禁用、来源和创建入口文案

`plugins-page` 新增 “Access is turned off by your admin”“Disabled by admin”“Choose another plugin marketplace”“Create a plugin for your team”“Created by me”“Curated by OpenAI”“Label for plugins shared with the current user”等字符串，并新增 `disabled-by-admin` 样式类名。

这表明插件页内已经包含管理员控制、市场切换、团队创建、个人本地插件和共享插件相关文案。旧版移除了 installed plugin card 的 more actions/share menu 文案，显示插件管理表面有重排信号。

### 【实证】[仅 macOS] Hooks 设置从事件级列表转向项目级安全审查

`hooks-settings` 新增 “Hooks can run outside of the sandbox so we ask you to review any recently installed or modified hooks”“No hooks found”“No projects found”“Projects with configured hooks will appear here”“Hook load issue count for a project row”等字符串。`user-message-attachments` 新增 hook runs 汇总、blocked/failed/ran count tooltip 相关 messageId。

这说明 hook 安全审查、项目级 hook 状态和消息气泡中的 hook 运行统计进入包内。旧字符串里事件级 “Before/After a tool executes”“Before/After Codex compacts the conversation”等被移除，属于设置页组织方式变化的强信号。

### 【实证】[仅 macOS] Avatar/pet 浮层获得等待请求、权限响应和补丁摘要文案

新增模块包括 `avatar-mascot-button`、`locked-computer-use-icon`、`use-avatar-options`。`avatar-overlay-page` 新增 “floating avatar notification waiting on a question answer”“Allow network”“Allow once”“Apply changes”“Ask a question”“reviewing a waiting request”等字符串，样式类名包括 `mcp-elicitation`、`permission-response`、`plan-start`。

多语言资源还新增 `avatarOverlay.compactPatchAdditions`、`compactPatchDeletions`、`compactPatchFileCount`、`waitingRequest.allowNetwork` 等 messageId。可确认的是这些交互文案进入 macOS 包内；是否显示为默认 pet 体验仍需运行时验证。

### 【信号】[仅 macOS] Composer 和对话线程围绕目标、站点权限、附件与任务状态继续调整

`composer` 体积增加约 81,827 bytes，新增“Allow Codex to interact with any website without asking?”、“Always allow any website”、“Always allow this site”、“Action required”、“All chats”等字符串，并移除“Allow Browser Use to interact…”和 goal editor 相关旧 aria 文案。这与 browser-use 到 Browser/Codex 的命名迁移相吻合。

`local-conversation-thread` 新增 background tasks、generated image carousel、completed turn diff、MCP app navigation controls、conversation resource rows 等字符串和类名。证据等级仍是【信号】，因为它来自同名前端模块压缩产物差异，但它指向对话页状态、导航和差异展示的持续调整。

### 【信号】[仅 macOS] 文档、PDF、Mermaid、ProseMirror 和表格 worker 进入或扩大前端面

真实新增模块包含 `mermaid-diagram`、`prosemirror`、`marked.esm`、`annotation-comment-editor-card`、`image-preview-shortcuts`、`pull-request-code-review-comments`。`pdf` 模块从接近 0 增到 0.4 MB，出现 PDF API、annotation、XFA、dark/forced-colors 和大量 aria/data 属性字符串。`runtime.worker` 从 1.5 MB 增至 2.6 MB，新增 “Add Sheet”、workbook help、worksheet scope、bond object 等字符串。

这些是文档/富文本/图表/表格能力增强的强信号，但因为主要来自前端模块和压缩 worker 字符串，不能直接写成完整功能已经开放。
