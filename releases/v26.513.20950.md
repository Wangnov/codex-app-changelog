# Codex 桌面版 26.513.20950(macOS / Windows 对照)

> 第三方跨平台变更日志 · 非官方

这一批次的核心结论很明确：macOS 包已经前进到 26.513.20950 build 2816，而 Windows MSIX(微软应用安装包格式) 对照包仍是 26.506.3741.0，且两次提取结果完全一致。

One-liner: macOS picked up the observable app, runtime, plugin, and document-handling changes; the compared Windows MSIX did not change.

## 关于这份报告

本报告基于官方签名分发产物的逆向 diff(差异对比)：macOS 使用 Sparkle(应用自动更新框架) 全量包对比，Windows 使用 MSIX 包对比。macOS 对比范围是 26.506.31421 build 2620 → 26.513.20950 build 2816；Windows 对比范围是 26.506.3741.0 → 26.506.3741.0。发布日期按本批次输入记录为 2026-05-08。

证据等级分两档：【实证】表示来自 plist(应用属性清单)、AppxManifest(Windows 应用清单)、CSP(内容安全策略)、技能文档、类型声明、package.json(项目配置文件)、依赖清单等可读证据；【信号】表示只来自前端模块名或体积变化，能指出方向，但不能还原源码意图。

平台归属标记含义是：[macOS+Windows] 表示两平台都有同类事实；[仅 macOS] 表示只在 macOS 包中观察到；[仅 Windows] 表示只在 Windows 包中观察到。本批次没有观察到 Windows 独有功能变化。

## ✨ 重点变化

### 应用版本与运行时升级【实证】[仅 macOS]

macOS 包从 26.506.31421 build 2620 更新到 26.513.20950 build 2816，package.json 同步更新版本号、build 号和 Sparkle 公钥，Info.plist(应用属性清单) 也同步更新 `CFBundleShortVersionString`、`CFBundleVersion` 和 app.asar(应用代码归档) 完整性 hash。

同一批次的 Windows 包仍为 26.506.3741.0，前后 SHA-256 完全相同，文件数、内容改动数和 app.asar 内部文件数均为 0 变化。因此这些版本与运行时变化没有在 Windows MSIX 中出现。

### Electron 与本地依赖更新【实证】[仅 macOS]

macOS 的 package.json 显示 Electron(桌面应用运行框架) 从 41.2.0 升到 42.0.1，better-sqlite3(SQLite 数据库原生绑定) 从 ^12.8.0 升到 ^12.9.0。Electron Framework 二进制体积从 177.9 MB 增至 181.5 MB，增加 3.64 MB。

Windows 对照包没有任何文件级变化，也没有可见依赖变化，因此不能把这次 Electron 或 better-sqlite3 更新归到 Windows。

### 开发与打包脚本整理【实证】[仅 macOS]

macOS 的 package.json 中，`dev` 和 `owl` 脚本改为通过 `scripts/prepare-dev.mjs` 准备开发环境；新增 `forge:make:owl-shell`；新增 `e2e:computer-use-native-pipe`。同时原有的 `computer-use:windows:bundle` 脚本在该 diff 中被移除。

这些是打包产物内可读配置变化，说明开发、打包和 native pipe(原生管道通信) 端到端测试相关流程有调整。Windows 包本身未变化。

### 电子表格文件类型支持扩展【实证】[仅 macOS]

macOS 的 Info.plist 新增 TSV(制表符分隔文本)、XLS、XLSM、XLSX 文件类型声明，角色均为 Viewer(查看器)，LSHandlerRank 为 Alternate(备用处理程序)。这意味着 macOS 系统层面可以把这些文件类型识别为 Codex 可查看的文档类型之一。

Windows 的 AppxManifest 未提供任何变化事实，Windows 包也没有文件级变化，因此本批次不能确认 Windows 获得同等文件关联更新。

### 浏览器插件目录从 browser-use 迁移到 browser【实证】[仅 macOS]

macOS 包中新增 `plugins/openai-bundled/plugins/browser/...` 下的 browser-client.mjs 与 classic-level 多平台预构建文件，同时移除了 `plugins/openai-bundled/plugins/browser-use/...` 下对应文件与技能文档。体积上，新增 browser-client.mjs 为 0.9 MB，移除的 browser-use browser-client.mjs 为 0.7 MB。

这更像是插件命名或目录结构迁移，而不是简单新增一套浏览器能力。Windows 对照包无变化，因此该迁移只在 macOS 包中观察到。

### LaTeX 插件目录更名【实证】[仅 macOS]

macOS 包新增 `plugins/openai-bundled/plugins/latex/bin/tectonic`，大小 50.3 MB；同时移除 `plugins/openai-bundled/plugins/latex-tectonic/bin/tectonic`，同为 50.3 MB。可见事实更支持“目录命名从 latex-tectonic 改为 latex”，而不是证明底层 tectonic 二进制能力发生变化。

Windows 对照包没有相应文件变化。

### Computer Use 辅助组件扩展【实证】[仅 macOS]

macOS 包新增 `CUALockScreenGuardian.app` 相关文件，其中主二进制为 14.7 MB，资源 Assets.car 为 3.0 MB；同时 `SkyComputerUseService` 从 14.7 MB 增至 15.0 MB，增加 0.27 MB。另有两个 embedded.provisionprofile 文件被移除。

这些文件名指向 Computer Use(计算机使用能力) 的锁屏守护或辅助安装相关组件，但事实包没有源码，不能进一步断言用户侧行为。Windows 包未变化。

### MCP 沙箱消息通道增加 navigate【实证】[仅 macOS]

macOS 的 `.vite/build/sandbox-preload.js` 中，MCP(Model Context Protocol，模型上下文协议) app sandbox(应用沙箱) 允许的端口名列表新增 `navigate`。这是可读代码层面的明确变化，说明沙箱消息桥接面增加了一个名为 `navigate` 的通道。

Windows app.asar 没有变化，因此该通道新增只在 macOS 包中观察到。

### 前端模块显示多条产品方向信号【信号】[仅 macOS]

macOS app.asar 内部文件从 1382 增至 1419，去除 webview/assets 与 .vite/build hash 噪音后，新增模块名包括 `codex-mobile-page`、`codex-mobile-setup-flow`、`device-connections`、`remote-projects`、`google-drive`、`plugin-uninstall-dialog`、`pull-request-code-review-comments`、`mermaid-diagram`、`inline-mentions`、`keyboard-shortcuts-search-input` 等。

这些模块名指向移动端连接、设备连接、远程项目、Google Drive(谷歌云端硬盘)、插件卸载、PR(拉取请求) 代码审查评论、Mermaid(文本图表语法)、内联提及、快捷键搜索等方向。但这只是模块级信号，不能证明这些能力已经对用户开放。Windows app.asar 为 1336 → 1336，新增 0、移除 0、改动 0。

### 新图标资源加入包内【实证】[仅 macOS]

macOS 新增 `Contents/Resources/icon.icns`，大小 0.9 MB。Windows 包无文件变化，因此没有对应观察。
