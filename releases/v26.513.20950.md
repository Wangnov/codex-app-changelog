# Codex 桌面版 `26.513.20950`(build 2816)

> 第三方变更日志 · 非官方 · 基于官方更新包逆向差异

这一版的主要变化集中在文件类型支持、浏览器插件命名与资源布局、Computer Use(让模型操作桌面 / 浏览器的能力) 辅助组件，以及 Electron(桌面应用运行框架) 运行时升级。  
English one-liner: this build looks like a capability-packaging and desktop-runtime update rather than a UI-only release.

## 关于这份报告

这不是官方变更日志，而是一份**基于官方分发产物的逆向差异报告**。本次是历史回填：两个版本均已超出官方 Sparkle(应用自动更新框架) 增量窗口，因此采用两个官方 macOS arm64 全量包直接比较，而不是增量包重建。

可信度方面，两个 App(应用程序) 均通过 Apple(苹果) 公证与 OpenAI Developer ID(开发者身份签名) 验证：`codesign / spctl` 结论均为 `accepted`，来源为 `Notarized Developer ID`，签名主体为 `Developer ID Application: OpenAI OpCo, LLC (2DC432GLL2)`。

本文使用两档证据等级：

- **【实证】**：来自人类可读的直接证据，例如 `Info.plist`、`package.json`、可读脚本 diff(差异文本)、文件路径与签名验证结论。
- **【信号】**：来自前端模块名或二进制体积变化，只能说明“指向某方向，但无法从产物还原到源码级意图”。

对比范围：`26.506.31421` build `2620`，发布于 `2026-05-09` → `26.513.20950` build `2816`，发布于 `2026-05-14`。

## ✨ 重点变化

### 1. 新增表格文件类型关联：TSV / XLS / XLSM / XLSX 【实证】

Codex 桌面版在 `Info.plist` 中新增了多种表格文件类型的文档声明，包括 `TSV`、`XLS`、`XLSM`、`XLSX`，角色均为 `Viewer`，`LSHandlerRank` 为 `Alternate`。

这意味着 macOS(苹果桌面操作系统) 层面现在知道 Codex 可以作为这些文件的备选查看器；用户可能会在“打开方式”或文件拖入相关流程中看到更完整的表格文件支持信号。

关键证据：

```diff
+ CFBundleTypeExtensions = tsv / xls / xlsm / xlsx
+ CFBundleTypeRole = Viewer
+ LSHandlerRank = Alternate
```

### 2. 浏览器插件资源从 `browser-use` 调整为 `browser` 【实证】

文件树中出现新的 `Contents/Resources/plugins/openai-bundled/plugins/browser/` 路径，同时移除了大量 `browser-use` 路径下的对应资源，包括 `browser-client.mjs` 与 `classic-level` 预编译模块。

这更像是内置浏览器插件的命名与封装路径调整，而不是简单新增一个全新浏览器能力。可感知点可能体现在插件名称、加载路径或内部调用方式上，但事实包没有给出用户界面文案。

关键证据：

```text
新增: Contents/Resources/plugins/openai-bundled/plugins/browser/scripts/browser-client.mjs (0.9 MB)
移除: Contents/Resources/plugins/openai-bundled/plugins/browser-use/scripts/browser-client.mjs (0.7 MB)
```

### 3. Computer Use 增加锁屏守护相关辅助组件 【实证】

Computer Use 插件目录下新增 `CUALockScreenGuardian.app`，其中主二进制大小为 `14.7 MB`，同时 `SkyComputerUseService` 从 `14.7 MB` 增至 `15.0 MB`。

这表明 Computer Use 的本地辅助程序包有所扩展，方向上与锁屏状态相关；但仅凭产物不能判断具体交互策略或触发条件。

关键证据：

```text
新增: CUALockScreenGuardian.app/Contents/MacOS/CUALockScreenGuardian (14.7 MB)
SkyComputerUseService: 14.7 MB → 15.0 MB (+0.27 MB)
```

### 4. MCP 应用沙箱新增 `navigate` 消息通道 【实证】

`sandbox-preload.js` 中，MCP(Machine-readable Context Protocol，模型上下文协议) 应用沙箱可转发的端口名列表新增了 `navigate`。

这说明沙箱内外通信能力增加了一个与导航相关的通道。可感知点可能是嵌入式应用或沙箱页面的跳转能力更完整，但事实包只证明通道存在，不证明具体产品入口已开放。

关键证据：

```diff
- [`notifyMcpAppsHostContext`, ...]
+ [`navigate`, `notifyMcpAppsHostContext`, ...]
```

### 5. Electron 升级到 `42.0.1`，本地数据库依赖升级到 `better-sqlite3 ^12.9.0` 【实证】

`package.json` 显示 Electron 从 `41.2.0` 升级到 `42.0.1`，`better-sqlite3` 从 `^12.8.0` 升级到 `^12.9.0`。

这属于底层运行时与本地存储依赖升级。用户一般不会直接看到版本号，但可能影响窗口、WebView(内嵌网页视图)、原生模块兼容性或本地数据访问稳定性；具体行为变化不能仅由依赖版本推出。

关键证据：

```diff
- "electron": "41.2.0"
+ "electron": "42.0.1"

- "better-sqlite3": "^12.8.0"
+ "better-sqlite3": "^12.9.0"
```

### 6. 开发与打包脚本被收敛到 `prepare-dev.mjs` 【实证】

`package.json` 中，`dev` 与 `owl` 脚本从多个独立构建步骤改为调用 `scripts/prepare-dev.mjs`，并新增 `forge:make:owl-shell` 与 `e2e:computer-use-native-pipe`。

这主要影响开发者与内部构建流程：开发准备动作被集中封装，Computer Use 原生管道也出现了端到端测试入口。对普通用户的直接影响有限。

关键证据：

```diff
+ "dev": "pnpm exec node ./scripts/prepare-dev.mjs --native-modules ..."
+ "e2e:computer-use-native-pipe": "node ./scripts/computer-use-native-pipe-e2e.mjs"
```

### 7. 前端模块名指向移动端连接、远程项目、插件管理与代码审阅入口扩展 【信号】

去除 Vite(前端构建工具) hash(内容指纹) 噪音后，真实新增模块中出现 `codex-mobile-page`、`codex-mobile-setup-flow`、`device-connections`、`remote-projects`、`plugin-uninstall-dialog`、`pull-request-code-review-comments`、`google-drive`、`mermaid-diagram` 等名称。

这些模块名指向移动端设置、设备连接、远程项目、插件卸载、Pull Request(代码合并请求) 评论、Google Drive(谷歌云端硬盘) 与 Mermaid(图表文本语法) 渲染等方向，但无法从产物还原到源码级意图，也不能证明这些入口已经对所有用户开放。

关键证据：

```text
新增模块: codex-mobile-page, codex-mobile-setup-flow, device-connections, remote-projects,
plugin-uninstall-dialog, pull-request-code-review-comments, google-drive, mermaid-diagram
```
