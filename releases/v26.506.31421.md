# Codex 桌面版 26.506.31421 macOS / Windows 对照

> 第三方跨平台变更日志 · 非官方

这是一份基于官方签名分发包逆向 diff 的跨平台对照报告，重点区分 macOS arm64 与 Windows x64 包内实际变化。  
One-liner: this batch is a small cross-platform binary/app bundle refresh, with macOS metadata changes and a Windows sandbox setup binary size change.

## 关于这份报告

本报告只使用输入事实包中的机器提取事实。macOS 样本对比 `26.506.31004 build 2604` → `26.506.31421 build 2620`；Windows 样本对比 `26.506.3484.0` → `26.506.3741.0`。

证据等级分两档：

- 【实证】：来自 plist、manifest、完整性哈希、文件树、asar 统计、依赖清单、签名/公证验证、明确文件路径与体积变化等事实。
- 【信号】：仅来自前端模块名、压缩 bundle 方向线索或体积变化的弱证据。本批次输入没有提供同名前端模块改动线索或新增可读字符串，因此不把 UI 或功能可见性写成事实。

平台归属标记为 `[macOS+Windows]`、`[仅 macOS]`、`[仅 Windows]`。两个平台来自同一套 Codex 代码的不同打包，但版本号与 build 不一致，本报告分别引用各自事实，不互相套用。

本次输入中未提供内置插件清单变化、Computer Use 运行时总量、前端模块真实增删明细、同名前端模块改动线索、新增可读字符串、内容安全策略变化、新增组件与资源、关键文件具体改动等细项，因此这些方向只在证据边界中说明，不扩展推断。

## ✨ 重点变化

### 【实证】[macOS+Windows] 核心 codex 可执行体小幅增大

两平台的核心 codex 组件都有体积增长。

macOS 中，`Contents/Resources/codex` 从 `190.8 MB` 增至 `191.5 MB`，增加 `+0.72 MB`。Windows 中，`resources/codex.exe` 从 `245.0 MB` 增至 `245.8 MB`，增加 `+0.85 MB`；同时 `resources/codex` 从 `217.9 MB` 增至 `218.7 MB`，增加 `+0.84 MB`。

这说明两个平台的核心运行组件都发生了内容更新。由于输入没有给出可读字符串、命令清单或具体代码差异，不能进一步断言新增了哪项用户可见能力。

### 【实证】[macOS+Windows] app.asar 内部结构保持稳定，仅有少量内容变化

macOS 的 `app.asar` 内部文件数为 `1382 → 1382`，其中新增 `1`、移除 `1`、改动 `8`。Windows 的 `app.asar` 内部文件数为 `1336 → 1336`，其中新增 `1`、移除 `1`、改动 `10`。

两个平台都显示应用层包内文件总数不变，但存在少量新增、移除和修改。事实包明确说明 `webview/assets` 下大量增减是 Vite 构建 hash 重命名噪音，不能据此判断新功能。

### 【实证】[仅 macOS] Info.plist 版本号、build 与 app.asar 完整性哈希更新

macOS 包的 `Info.plist` 显示：

- `CFBundleShortVersionString`: `26.506.31004` → `26.506.31421`
- `CFBundleVersion`: `2604` → `2620`
- `ElectronAsarIntegrity/Resources/app.asar/hash`: `f2d68d6d80d5a43a7e9c91653bbc4a30ac7e6e24444a8abc899e1ba49a1f74a5` → `d735f12f9b353a7b5d4f190919e8181b723073ae23555a0f45d496913daae917`

这是 macOS 包级元数据与 asar 完整性记录的明确变化。Windows 输入没有提供 AppxManifest 变化，因此不能写成跨平台 manifest 变化。

### 【实证】[仅 Windows] sandbox setup 可执行体出现小幅体积变化

Windows 包中 `resources/codex-windows-sandbox-setup.exe` 从 `8.1 MB` 到 `8.1 MB`，记录变化为 `+0.03 MB`。

这表明 Windows sandbox setup 二进制内容发生了小幅变化。输入没有提供该可执行体的字符串、manifest 或行为 diff，因此不能进一步说明 sandbox 逻辑变化。

### 【实证】[macOS+Windows] browser-use 与 chrome 插件脚本内容有改动，但没有新增依赖包

macOS 与 Windows 都列出以下脚本体积变化记录：

- `plugins/openai-bundled/plugins/browser-use/scripts/browser-client.mjs`
- `plugins/openai-bundled/plugins/chrome/scripts/browser-client.mjs`

两者在表中均显示约 `0.7 MB → 0.7 MB`，变化为 `+0.00 MB`。这属于内容改动被列入体积变化表，但四舍五入后体积差异不可见。

同时，两平台 `cua_node` 依赖变化均为“新增 0 个包”。因此，本批次没有证据显示 `cua_node` 新增依赖包；插件脚本有内容变化线索，但输入没有提供插件 manifest、服务入口、技能说明或默认提示变化，不能扩展为完整新增能力面。

### 【实证】[仅 macOS] 两个 macOS 全量包均通过公证验证

macOS 上一版与最新版 `Codex.app` 均显示：

- `accepted`
- `source=Notarized Developer ID`

### 【实证】[macOS+Windows] 本批次没有文件数量级扩张

macOS bundle 文件数 `1238 → 1238`，新增 `0`、移除 `0`、内容改动 `54`。Windows bundle 文件数 `815 → 815`，新增 `0`、移除 `0`、内容改动 `16`。
