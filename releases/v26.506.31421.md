# Codex 桌面版 26.506.31421(macOS / Windows 对照)

> 第三方跨平台变更日志 · 非官方

本批次没有可读发布说明，逆向 diff 显示两平台共同更新了核心二进制与应用层包，但可还原的功能语义有限。  
One-liner: this cross-platform batch looks like a compact core/app-layer refresh, with Windows carrying an additional sandbox setup delta.

## 关于这份报告

本报告基于官方签名分发包的逆向 diff（差异对比）：macOS 侧比较 Sparkle（macOS 应用更新框架）全量包，Windows 侧比较 MSIX（Windows 应用安装包）包。macOS 对比范围为 26.506.31004 build 2604 → 26.506.31421 build 2620；Windows 对比范围为 26.506.3484.0 → 26.506.3741.0。发布日期按本批次输入使用 2026-05-08。

证据分两档：【实证】表示来自 plist（macOS 配置清单）、package.json（包配置文件）、依赖清单、类型声明、技能文档、可读清单或明确文件级统计；【信号】表示只看到模块名、文件名或体积变化，能指出变化方向，但不能还原源码意图。

平台归属标记含义：[macOS+Windows] 表示两平台都出现；[仅 macOS] 表示只在 macOS 事实包中出现；[仅 Windows] 表示只在 Windows 事实包中出现。webview/assets（网页视图资源）与 .vite/build（Vite 构建目录）的 hash（哈希）重命名已按输入要求视为噪音，不作为功能增减依据。

macOS 两个全量包均通过公证验证，来源显示为 Notarized Developer ID（已公证开发者身份），签名主体为 OpenAI OpCo, LLC。Windows 事实包提供了上一版与最新版包的 SHA-256（文件校验摘要），但未给出同类可读签名验证结果。

## ✨ 重点变化

### 核心可执行文件小幅增长【信号】[macOS+Windows]

两平台的核心 Codex 二进制都有体积增长。macOS 的 `Contents/Resources/codex` 从 190.8 MB 增至 191.5 MB，增加 0.72 MB；Windows 的 `resources/codex.exe` 从 245.0 MB 增至 245.8 MB，增加 0.85 MB，同时 `resources/codex` 从 217.9 MB 增至 218.7 MB，增加 0.84 MB。

这说明两端都包含核心层更新，但事实包没有提供符号、源码或可读 changelog（变更日志），因此不能进一步断言具体功能。

### 应用层 app.asar 有小幅结构变化【实证】[macOS+Windows]

两平台的 app.asar（Electron 应用归档包）内部文件总数保持不变，但都出现“新增 1 / 移除 1 / 改动若干”的变化。macOS 为 1382 → 1382，新增 1、移除 1、改动 8；Windows 为 1336 → 1336，新增 1、移除 1、改动 10。

平台差异在于 Windows 的改动文件数略多，但两端都没有文件总数净增长。由于 webview/assets 与 .vite/build 的 hash 重命名被排除为构建噪音，这里只作为应用层发生小规模更新的事实，不推断具体 UI（用户界面）或功能变化。

### 浏览器相关插件客户端脚本有变化【信号】[macOS+Windows]

两平台都列出了 `openai-bundled`（OpenAI 随包插件集合）下的 Chrome（浏览器）与 browser-use（浏览器自动化使用）插件客户端脚本变化：`plugins/chrome/scripts/browser-client.mjs` 与 `plugins/browser-use/scripts/browser-client.mjs` 均出现在体积变化列表中，显示为 0.7 MB → 0.7 MB，变化四舍五入后为 +0.00 MB。

这是一条方向性信号：浏览器插件客户端代码被触及，但事实包只给出文件名和四舍五入后的体积，不能判断是修复、重构还是行为变化。

### cua_node 未新增依赖【实证】[macOS+Windows]

两平台的 cua_node（本地自动化相关 Node 依赖集合）依赖变化均显示“新增 0 个包”。

这意味着本批次没有观察到 cua_node 新包引入。注意这只覆盖“新增包”事实，不代表既有包内容没有变化，也不能推出运行行为没有变化。

### macOS 版本元数据与 asar 完整性哈希更新【实证】[仅 macOS]

macOS 的 Info.plist 显示版本字段更新：`CFBundleShortVersionString` 从 26.506.31004 更新到 26.506.31421，`CFBundleVersion` 从 2604 更新到 2620。`ElectronAsarIntegrity/Resources/app.asar/hash` 也从旧值更新为新值。

这是 macOS 打包元数据层面的明确变化。Windows 事实包没有提供对应 AppxManifest（Windows 应用清单）变化，因此不能写成双平台共同项。

### Windows sandbox 安装组件发生变化【信号】[仅 Windows]

Windows 侧 `resources/codex-windows-sandbox-setup.exe` 从 8.1 MB 到 8.1 MB，变化为 +0.03 MB。

这表明 Windows 专属 sandbox（沙箱）安装组件发生了内容或打包层变化。事实包没有给出可读安装逻辑、版本字段或配置差异，因此只能作为 Windows 独有信号，不能断言 sandbox 能力新增。
