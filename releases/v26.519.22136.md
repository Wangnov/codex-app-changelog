# Codex 桌面版 `26.519.22136`(build 3003)

> 第三方变更日志 · 非官方 · 基于官方更新包逆向差异

Codex（代码智能体桌面应用）这一版最明确的变化集中在 Chrome（Google 浏览器）自动化插件：安装、检测、标签页接管和清理语义都有可读证据。  
English one-liner（英文一句话）: Chrome automation now has clearer multi-profile diagnostics and a more self-contained host setup.

## 关于这份报告

这不是官方变更日志，而是一份**基于官方签名分发产物的第三方逆向差异报告**。事实包说明，本次是历史回填：两个版本均已超出官方 Sparkle（macOS 应用更新框架）增量窗口，因此改为下载两个官方全量包后直接比较，而不是使用 EdDSA（爱德华曲线数字签名算法）和 BinaryDelta（二进制增量补丁机制）重建。

可信度来自分发包本身的验证链：两版 `Codex.app` 均通过 Apple（苹果）公证验证，来源显示为 `Notarized Developer ID`，签名来源为 `Developer ID Application: OpenAI OpCo, LLC (2DC432GLL2)`。事实包同时说明信任根为 Apple 公证与 OpenAI（人工智能研究与产品公司）Developer ID（苹果开发者身份签名）代码签名，即 codesign（macOS 代码签名验证工具）/ spctl（macOS Gatekeeper 策略验证工具）验证路径。

证据分两档：**【实证】**表示来自 Info.plist（macOS 应用元信息文件）、技能文档、脚本 diff（差异文本）、plugin.json（插件配置文件）、依赖清单或签名结论等人类可读证据；**【信号】**表示只来自前端模块名或二进制体积变化，指向某方向，但无法从产物还原到源码级意图。

对比范围：`26.513.31313` build 2867（2026-05-16）→ `26.519.22136` build 3003（2026-05-21），平台为 macOS arm64（苹果芯片架构）。

## ✨ 重点变化

### 1. Chrome 插件版本号改为与桌面版对齐，并补充扩展宿主配置 【实证】

Chrome 插件的 `.codex-plugin/plugin.json` 版本从 `0.1.7` 改为 `26.519.22136`，与桌面应用短版本号一致。更重要的是，`installManifest.mjs` 不再只写 Native Messaging（浏览器原生消息通信）宿主清单，还会写入 `extension-host-config.json`，其中包含 `channel: "prod"`、`codexCliPath`、`nodePath`、`nodeReplPath`、`browserClientPath`、`proxyHost: "127.0.0.1"` 和 `proxyPort: 0` 等运行时配置。

这意味着 Chrome 扩展宿主的安装链路携带了更多明确上下文，后续排查“扩展已装但宿主不可用”这类问题时，产物层面已有更完整的路径和配置证据。对应新增文件也支持这一点：Chrome 插件目录下新增了 `app-server-runtime/codex`、`app-server-runtime/node` 和 `app-server-runtime/node_repl`。

关键证据：`plugin.json` 中 `"version": "0.1.7"` → `"version": "26.519.22136"`；`installManifest.mjs` 新增写入 `extension-host-config.json` 的逻辑。

### 2. Chrome 扩展安装检测从单一配置目录扩展为多配置目录扫描 【实证】

`check-extension-installed.js` 的行为发生了明确变化：旧版只检查选中的 Chrome profile（浏览器用户配置目录），新版会扫描每个可用的 `Default` 或 `Profile X` 目录，并在 JSON（结构化文本格式）输出中返回完整的 `profiles` 数组，同时给出 `selectedProfileDirectory`。

这对多 Chrome 配置目录用户更可感知：工具现在能区分“某个配置目录里装了扩展”和“Chrome 实际会启动的配置目录是否可用”。顶层 `installed`、`enabled` 和退出码仍反映选中的配置目录，因此自动化排障可以更准确地判断当前要使用的配置目录是否准备好。

关键证据：技能文档从“checks whether the selected Google Chrome profile”改为“checks every usable Google Chrome profile”，并明确 JSON 输出包含 `profiles` 和 `selectedProfileDirectory`。

### 3. Chrome 标签页接管与清理语义改为“控制后释放，尽量保留原位” 【实证】

Chrome 技能文档说明，`browser.user.claimTab(tab)` 现在是在不移动标签页到 agent（智能体）标签组的情况下获得控制权，并返回可控的 `Tab`。旧文档写的是会把选择的 Chrome 标签页移动到当前 agent 标签组。

标签页收尾语义也变了：`deliverable` 标签页不再写成移动到共享的 `✅ Codex` 标签组，而是“释放当前浏览器会话控制后保持打开”；`handoff` 标签页也强调释放控制并留在当前位置。这说明自动化会更少改变用户已有标签页分组，至少从技能文档定义看，交接动作更偏向“释放控制”而不是“重新归组”。

关键证据：技能文档中 `Claiming moves...` 改为 `Claiming gives... without moving...`；`deliverable` 从“move to shared ✅ Codex tab group”改为“left open after the current browser session releases them”。

### 4. Browser 与 Chrome 自动化技能文档同步调整了交互约束 【实证】

Browser（内置浏览器自动化能力）和 Chrome 两套技能文档都删除了“不要使用 `tab.playwright.waitForTimeout(...)`”这一条限制，同时把 `FinalizeTabsOptions.keep` 的注释从“要保持打开的标签页”改为“清理后显式保留的标签页处置”。

这只能证明文档层面的约束发生了变化，不能据此推断 Playwright（浏览器自动化库）的完整上游 API（应用程序接口）都可用。可感知点主要在开发者和自动化流程编写者：标签页清理和等待行为的说明更少绝对化。

关键证据：`plugins/chrome/skills/chrome/SKILL.md` 与 `plugins/browser/skills/browser/SKILL.md` 中同一条 `waitForTimeout` 禁令被移除。

### 5. 前端模块名显示主界面、插件、工作区与评审能力继续扩展 【信号】

去掉 Vite（前端构建工具）hash（构建指纹）噪音后，事实包列出的真实新增前端模块指向几个方向：应用外壳与窗口管理，如 `app-shell-tab-controller`、`app-window`、`thread-app-shell-chrome`；插件与 MCP（模型上下文协议）能力，如 `plugin-auth`、`plugin-config-edits`、`mcp-capability-view-page`、`use-plugin-install-flow`；Git（版本控制系统）与评审流程，如 `git-branch-switcher`、`diff-stats`、`parse-diff`、`review-mode-content`、`pull-request-open`；以及输入框、建议、队列消息和置顶线程相关模块。

这些模块名指向界面与工作流在继续拆分和扩展，但无法从产物还原到源码级意图，也不能证明这些能力已经对所有用户开放。

关键证据：事实包中“去掉 hash 噪音后”的新增模块列表包含上述模块名；同时明确说明“模块名未混淆，可反推方向，但出现≠对用户开放”。

### 6. 体积增长主要落在 app.asar、codex 二进制和 Chrome 运行时相关组件 【信号】

文件级统计显示，`Contents/Resources/app.asar` 从 147.7 MB 增至 159.0 MB，增加 11.33 MB；`Contents/Resources/codex` 从 190.9 MB 增至 192.1 MB，增加 1.13 MB。Chrome 插件还新增了较大的 `app-server-runtime/codex`、`app-server-runtime/node` 与 `app-server-runtime/node_repl` 文件。

这说明打包内容和运行时随本版变大，方向上与 Chrome 扩展宿主配置的增强相互呼应。但二进制体积变化本身只能作为信号，无法从产物还原到源码级意图。

关键证据：Bundle 文件树概览中的体积变化表与新增文件节选。
