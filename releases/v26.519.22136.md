# Codex 桌面版 26.519.22136 macOS / Windows 对照

> 第三方跨平台变更日志 · 非官方

本批次的可见变化集中在 macOS 26.513.31313 → 26.519.22136；Windows 输入样本是同一 26.519.2736.0 包的自比较，未提供内容变化。

One-liner: macOS gained a visibly larger app/runtime surface around Chrome automation, App Server, Appshot, plugins, and design-edit UI signals, while the Windows sample is content-identical.

## 关于这份报告

本报告只使用输入事实包，逐组核对版本与渠道、Bundle 文件树、插件清单、Computer Use 运行时、cua_node 依赖、app.asar、前端模块增删、同名模块线索、可读字符串、Info.plist / AppxManifest、CSP、新增组件与资源、关键文件 diff。

证据分两档：【实证】来自 plist、manifest、技能文档、依赖清单、服务入口、配置、明确 diff、以及事实包中非“同名前端模块改动线索”的新增可读字符串；【信号】来自同名前端模块的字符串/样式变化、模块名、体积方向和压缩 bundle 线索。

平台归属按输入样本标注：[macOS+Windows]、[仅 macOS]、[仅 Windows]。本次没有可确认的 [macOS+Windows] 内容变化；Windows 只能表述为“本输入样本无内容变化”。

## ✨ 重点变化

### Chrome 插件版本并入桌面版本号，且新增内置 App Server 运行时配置
【实证】[仅 macOS]

macOS 的 `chrome` 插件版本从 `0.1.7` 改为 `26.519.22136`。同一包内还新增了 `plugins/chrome/app-server-runtime/codex`、`node`、`node_repl` 和 `docs/capabilities/tab/pageAssets.md`，其中 `codex` 为 192.1 MB、`node` 为 118.4 MB、`node_repl` 为 15.1 MB。

`installManifest.mjs` 不再只写 native messaging host manifest；新版会生成 `extension-host-config.json`，包含 `schemaVersion: 1`、`channel: "prod"`、`codexCliPath`、`browserClientPath`、`nodePath`、`nodeReplPath`、`proxyHost: "127.0.0.1"`、`proxyPort` 等字段。这说明 Chrome 插件打包面从“浏览器扩展 host 安装”扩展到“随插件携带 App Server 运行时配置”。

Windows 样本前后 SHA-256 相同，文件数、app.asar 和内容改动均为 0，因此本输入无法确认 Windows 在本批次发生同类变化。

### Chrome 扩展安装检测改为扫描多个 Chrome Profile
【实证】[仅 macOS]

`check-extension-installed.js` 从“检查选中 Profile”改为“列出并检查所有可用 `Default` / `Profile X`”。JSON 输出新增 `profiles` 数组、`selectedProfileDirectory`、`userDataDirectory`，每个 Profile 都报告 `installed`、`registered`、`enabled`、`disabled`、`versions`、路径和禁用原因。

技能文档同步改写：顶层 `installed`、`enabled` 和退出码仍反映选中 Profile，便于判断实际将启动的 Chrome Profile 是否就绪。调试入口也从偏向单 Profile，改为默认扫描全部可用 Profile，并允许用 `CODEX_CHROME_USER_DATA_DIR` 或 `CODEX_CHROME_PREFERENCES_PATH` 限定范围。

### 浏览器 Tab 清理语义调整：保留页面不再强调移动到 Codex 分组
【实证】[仅 macOS]

Chrome 技能文档把“claim tab 会移动到当前 agent tab group”改为“当前 browser session 获得控制权但不移动到 agent tab group”。`finalize` 相关说明也从“deliverable tabs move to shared Codex tab group”改为“释放控制后保持打开”。

Browser 和 Chrome 两个技能文档都把 `FinalizeTabsOptions.keep` 的注释从“Tabs to keep open”改为“Explicit tab dispositions to preserve after cleanup”，并把 `FinalizeTabsKeep.tab` 解释为要保留的 tab object。这是浏览器自动化生命周期语义的明确文档变化。

### App 启动路径新增条件门控
【实证】[仅 macOS]

`.vite/build/bootstrap.js` 的启动流程在 `sparkleManager.initialize()` 和 `runMainAppStartup()` 之前新增了 `if(await t.g())` 条件。事实包无法还原 `t.g()` 的完整语义，但可以确认主应用启动被一个新增条件包住；这是 macOS app bundle 内启动路径的实证变化。

### 前端架构出现 App Shell、底部面板、线程侧栏和项目状态重组

去 hash 后新增模块包括 `app-shell-state`、`app-shell-tab-controller`、`thread-app-shell-chrome`、`thread-side-panel-tabs`、`thread-page-bottom-panel-state`、`thread-panel-state`、`summary-panel-row`、`pending-request-item-panel`、`review-navigation-model`、`review-mode-content`、`review-header-toolbar` 等。

### Appshot 捕获与附件进入 composer / 多语言文案
