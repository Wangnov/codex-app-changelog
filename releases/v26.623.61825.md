# Codex 桌面版 26.623.61825 macOS / Windows 对照

> 第三方跨平台变更日志 · 非官方

本批次的共同主线是内置插件版本随包更新，前端在 worktree 故障恢复、模型 reasoning 约束、项目/任务入口上继续调整；平台差异主要体现在 macOS 的模块重排更广，Windows 的可见信号更集中。  
One-liner: This batch updates bundled plugin metadata on both platforms while surfacing shared worktree/reasoning signals, with broader macOS UI churn and more concentrated Windows plugin/task/RPC changes.

## 关于这份报告

本报告只使用输入事实包中的机器提取结果。macOS 样本为 `26.623.42026` build `4514` 到 `26.623.61825` build `4548`；Windows 样本为 `26.623.5546.0` 到 `26.623.8305.0`，但其内置插件 manifest 中的插件版本字段显示 `26.623.42026` 到 `26.623.61825`。两个平台分别引用各自事实，不互相外推。

证据分两档：【实证】来自 plist、manifest、CSP、依赖清单、插件清单、服务入口、技能文档、类型声明、package.json，以及不属于“同名前端模块改动线索”的可读字符串；【信号】来自同名前端模块的模块名、体积变化、可读字符串和样式类名变化。平台标记使用 `[macOS+Windows]`、`[仅 macOS]`、`[仅 Windows]`。

## ✨ 重点变化

### 【实证】[macOS+Windows] 内置 Browser 与 Chrome 插件版本随包更新

macOS 与 Windows 都显示 `browser`、`chrome` 两个内置插件的 `.codex-plugin/plugin.json` 版本号从 `26.623.42026` 更新到 `26.623.61825`。两端的插件描述未在事实包中显示语义变化，Browser 仍指向 in-app browser、本地目标、localhost、file URL、截图、点击和输入等使用场景；Chrome 仍指向依赖用户现有 Chrome 状态的自动化场景。

平台差异是 Windows 还额外显示 `computer-use` 插件 manifest 版本号从 `26.623.42026` 更新到 `26.623.61825`，描述为通过 Computer Use 从 Codex 控制 Windows 桌面应用。

### 【实证】[仅 Windows] Computer Use 插件 manifest 进入本次可读 diff

Windows 的 `resources/plugins/openai-bundled/plugins/computer-use/.codex-plugin/plugin.json` 显示版本更新，插件描述为 “Control desktop apps on Windows from Codex through Computer Use.” 这是 Windows 侧独有的 manifest 证据。

macOS 事实包没有对应的 `computer-use` 插件 manifest diff；两端的 `cua_node` 运行时总量均无变化，macOS 为 `377.4 MB → 377.4 MB`，Windows 为 `298.9 MB → 298.9 MB`，且 `cua_node` 新增依赖均为 `0`。

### 【信号】[macOS+Windows] Worktree 初始化失败后的恢复路径更明确

两端 `worktree-init-v2-page` 同名模块都新增了相同的一组用户可读文案，包括 `Continue anyway`、`Edit environment`、`Fix this project`、`Fix worktree setup`，以及本地环境设置失败后继续启动线程、打开本地环境设置、启动修复线程、取消 worktree 创建并启动本地对话等按钮描述。

这是前端模块线索，不能断言功能已对所有用户开放；但两端一致出现，说明 worktree 初始化失败后的继续、修复和本地对话回退流程在 UI 层被加强。

### 【信号】[macOS+Windows] 模型 reasoning effort 的约束提示更具体

macOS `onboarding-page` 与 Windows `split-items-into-render-groups` 都新增了 `. Use a model and reasoning combination listed in the tool description, or omit thinking.`、`Optional reasoning effort override. Must be supported by the selected model.`、`This model supports no reasoning effort overrides.` 等字符串，并出现 `machine-readable`、`model/reasoning` 样式或状态类名。

这表明 composer 或工具参数说明中，对模型与 reasoning effort 组合的合法性提示更严格。Windows 还移除了较泛化的 `Optional reasoning effort override.` 和 `Timed out loading thread tool model guidance.`。

### 【信号】[仅 macOS] 权限请求自动审查文案进入前端 bundle

macOS 的 `remote-conversation-page~new-thread-panel-page~appgen-library-page` 相关同名模块新增多条 auto-review 文案，例如 reviewer agent 在 Codex 运行前审查请求、审查完成、审查停止、审查超时，以及自动审查网络访问请求、权限请求、MCP tool call、单文件编辑的 action summary。

Windows 事实包没有对应同名信号。该项只能说明文案进入 macOS 前端 bundle，不能断言所有用户已看到自动审查流程。

### 【信号】[仅 macOS] Skills 页面出现完整页面级文案

macOS `skills-page` 新增了创建新 skill、刷新 skills 列表、新安装 skill 需要刷新后才能在 composer 使用、过滤空状态、页面标题、已安装 skills、推荐 skills、搜索输入与 `Loading skills...` 等文案。

这比单个按钮更接近完整页面能力面信号，但事实来源仍是同名前端模块改动线索，因此标为【信号】。

### 【信号】[仅 Windows] 插件详情页与分享/启用入口增强

Windows `plugin-detail-page` 新增 workspace 分享目标移除、搜索用户和组、MCP server enablement toggle、section navigation、`Add people or groups`、workspace link 权限、`App unavailable`、app tools modal 和 skill preview modal 更多操作菜单、skill card actions menu 等文案与样式类名。

macOS 同名 `plugin-detail-page` 反而移除了相近的一组文案，说明此输入样本中两端并不同步：Windows 侧是插件详情页能力聚合增强信号，macOS 侧是相关代码从该同名 chunk 移走或被重排的信号，不能解读为 macOS 功能删除。

### 【信号】[仅 Windows] Header 出现最近任务、Cloud tasks 与 Codex for Chrome 入口信号

Windows `header` 模块新增 `Accessible label for opening the recent tasks menu`、`All tasks`、`App settings`、`Chrome computer use settings`、`Cloud tasks`、`Codex for Chrome`、`Clear search` 等文案。

macOS 同名 `header` 模块移除了相同方向的一部分字符串。结合 macOS 的大量新增/移除 chunk，这更像前端打包重排或入口迁移信号；在 Windows 样本中，它明确体现为 header chunk 新增可读文案。

### 【信号】[仅 macOS] 主页横幅与 ambient suggestions 文案增加

macOS `home-announcements` 新增了多个可关闭横幅的无障碍标签，包括 gift credits、rate limit reset、Fast mode、backend-driven app banner、multi-agent composer banner、remote connections banner；还新增 `Add Connections`、ambient suggestions connected apps banner、remote connections banner、multi-agent composer banner、禁用 ambient suggestions 的按钮文案等。

Windows 事实包没有对应 `home-announcements` 信号。

### 【信号】[仅 macOS] Appgen、发布、library、pricing 与项目相关 chunk 大规模重排

macOS 去 hash 噪音后新增大量模块名，集中包含 `appgen-page`、`appgen-settings-page`、`appgen-library-page`、`appgen-publication-terms-route`、`pricing-plan-page`、`plugin-detail-page`、`projects-index-page`、`pull-requests-page`、`worktree-init-v2-page`、`remote-conversation-page` 等组合。

这些是高密度前端路由和共享 chunk 信号，说明 macOS 包内 appgen、项目、PR、插件详情、发布条款和定价相关代码组织发生较大变化；但模块名本身不能证明用户可见功能新增。

### 【信号】[仅 macOS] 文件树组件出现更明确的错误路径与可访问性状态

macOS `remote-conversation-page~pull-requests-page~onboarding-page~hotkey` 相关模块新增 `Cannot create a directory that collides with an existing file:`、`Cannot move a directory into one of its descendants`、`Duplicate path:`、`FileTree requires a shadow root`、`FileTree requires paths or preparedInput` 等字符串，并新增 `aria-level`、`aria-posinset`、`aria-setsize`、`data-file-tree-search-input`、`data-file-tree-sticky-path` 等状态类名。

这是文件树路径存储、拖放、搜索、sticky path 和可访问性方向的前端信号。

### 【信号】[仅 macOS] 项目页嵌入终端与 Codex Cloud 信号增强

macOS `remote-conversation-page~projects-index-page` 体积从 `0.3 MB` 到 `0.6 MB`，新增 `Codex Cloud`、`Error forking local task`、终端 crash 后重新加载按钮、embedded terminal crash 描述、xterm 相关 `data-codex-terminal`、`data-codex-xterm`、`bottom-panel`、`bg-token-terminal-background` 等类名。

这些线索指向项目页里的底部面板、终端渲染和本地任务 fork 错误处理。

### 【信号】[仅 macOS] Windows sandbox 设置字符串出现在 macOS bundle

macOS 某 appgen/onboarding 相关模块新增 `Failed to load Windows sandbox mode`、`Failed to load Windows sandbox readiness`、`Failed to update Windows sandbox mode`、`windows.sandbox` 和 `windows-sandbox` 类名。

这是 macOS 包内进入了 Windows sandbox 设置或状态文案的事实信号来源，但由于它位于 macOS 前端 bundle，不能推断 macOS 本身提供 Windows sandbox 能力，也不能代替 Windows 平台结论。

### 【信号】[仅 Windows] RPC 运行层错误路径进入前端 chunk

Windows `rpc` 模块新增 RPC stub、ReadableStream/WritableStream stub、StubHook、promise export ID、pipes、deserialize RPC stubs 等错误字符串，并出现 `workers-module` 类名。

这说明 Windows 样本中 RPC/worker 运行层相关代码或错误路径被打入可读 chunk。macOS 事实包没有对应 `rpc` 同名模块线索。
