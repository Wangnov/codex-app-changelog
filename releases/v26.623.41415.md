# Codex 桌面版 26.623.41415 macOS / Windows 对照

> 第三方跨平台变更日志 · 非官方

这一批次看起来不是单点功能发布，而是围绕本地环境、Browser Use、Appgen、设置页和运行状态展示的一次跨平台前端与内置插件同步整理。  
One-liner: this batch refreshes local-worktree recovery, browser automation surfaces, Appgen/settings wiring, and platform-specific plugin packaging signals.

## 关于这份报告

本文基于两份官方签名分发包的逆向重建 diff：macOS arm64 为 `26.623.31921` build `4452` 到 `26.623.41415` build `4505`；Windows x64 为 `26.623.4041.0` 到 `26.623.5175.0`，同时其内置插件 manifest 版本从 `26.623.31921` 变为 `26.623.41415`。

证据等级分两档：【实证】来自 plist、manifest、CSP、插件配置、可读字符串、文件级 diff 和依赖清单；【信号】来自同名前端模块、体积、模块名、样式类名与压缩 bundle 方向线索。平台归属使用 `[macOS+Windows]`、`[仅 macOS]`、`[仅 Windows]` 标记。

## ✨ 重点变化

### 【实证】[macOS+Windows] Browser 与 Chrome 内置插件版本同步到 26.623.41415

两平台的 `browser` 与 `chrome` 插件 manifest 都把 `version` 从 `26.623.31921` 更新到 `26.623.41415`。插件描述本身没有显示语义变化：Browser 仍定位为 in-app browser / browser-use，本地 `localhost`、`127.0.0.1`、`file://` 与当前内置浏览器标签的打开、检查、点击、输入和截图；Chrome 仍定位为依赖用户现有 Chrome 状态的自动化。

平台差异是：Windows 还额外包含 `computer-use` 插件 manifest 版本更新；macOS 输入样本里没有对应 Computer Use 插件 manifest diff。

### 【实证】[仅 Windows] Computer Use 插件版本更新

Windows 的 `resources/plugins/openai-bundled/plugins/computer-use/.codex-plugin/plugin.json` 将 `computer-use` 版本从 `26.623.31921` 更新到 `26.623.41415`，描述为通过 Computer Use 控制 Windows 桌面应用。

这是一条 Windows 独有的插件级实证；macOS 样本只给出 Browser 与 Chrome 插件版本变化。

### 【信号】[macOS+Windows] worktree 初始化失败后的修复路径更明确

两平台的 `worktree-init-v2-page` 都新增了同一组用户可见文案：`Continue anyway`、`Edit environment`、`Fix this project`、`Fix worktree setup`，以及“本地环境设置失败后继续启动线程”“打开本地环境设置”“创建修复线程”“取消 worktree 设置并启动本地会话”等按钮说明。

这表明 worktree 初始化失败后的 UI 流程可能更偏向可恢复：用户可以继续、编辑环境、发起修复线程或转入本地会话。但因为这些来自同名前端模块改动线索，本文只把它定为【信号】，不推断实际逻辑已经完整开放。

### 【信号】[macOS+Windows] 分享对话框相关 Appgen 站点分享文案被移除

两平台的 `appgen-share-dialog` 都移除了 `Add people or groups`、`Anyone in this workspace with the link`、`Anyone on the Internet`、复制 live site URL、外部打开 live site 等文案和相关可访问性标签。

这是 Appgen 分享界面收缩或重排的跨平台信号。它说明旧模块中这批分享入口文案不再出现在同名 bundle 的字符串差异里，但不能单独断言功能被删除。

### 【信号】[macOS+Windows] 通用设置页拆分或迁移

两平台 `general-settings` 都明显变小，并移除了默认权限 toggle、work mode settings、代码字号基准、hold-to-dictate hotkey、hotkey window hotkey、dictation dictionary entry 等字符串。

这更像设置页内容被迁移、拆分或延迟加载，而不是简单删除。macOS 同时新增 `settings-page`、`projects-index-page~appearance-settings~general-settings`、`pets-settings~appearance-settings~general-settings` 等模块组合；Windows 输入中则没有同等规模的新增模块清单。

### 【信号】[仅 Windows] Browser Use 设置页扩展：历史、下载、站点与允许域名

Windows 的 `browser-use-settings` 从约 `0.0 MB` 增至 `0.1 MB`，新增 `Access history without asking`、embedded browser downloads/history/settings breadcrumbs/contact info/extension/password manager/site settings、数据清理选项展开/隐藏、不可用插件设置 toggle、`Add allowed domain` 等字符串。

这是一组很高可见度的 Windows 独有 Browser Use 设置扩展信号，指向内置浏览器历史、下载、站点设置、密码管理、扩展设置、允许域名与访问历史授权。

### 【信号】[仅 macOS] Browser Use 任意网站访问确认进入请求面板

macOS 的 `pending-request-item-panel` 新增 `Allow Codex to interact with any website without asking?`、`Allow any website`、确认/取消允许 Browser Use 访问任意网站、风险警告，以及外部浏览器连接访问任意网站的标题。

这些字符串说明 macOS 包内出现了“允许任意网站”的确认与风险提示路径。它是可读字符串进入包内的事实，但来源属于同名前端模块线索，因此这里按【信号】处理，不断言默认可见或默认开启。

### 【信号】[仅 macOS] 浏览器上下文与 Chrome 扩展侧栏提示进入会话相关 bundle

macOS 的 `worktree-init-v2-page~remote-conversation-page~new-thread-panel-page` 组合模块体积从 `0.3 MB` 增至 `1.5 MB`，新增了 “用户已在页面选择文本，必须调用……”、“Chrome extension side panel open”、“in-app browser open with……”、“in-app browser open”等上下文提示字符串。

这些文案指向对浏览器状态、选中文本、Chrome 扩展侧栏和内置浏览器打开状态的上下文感知，但仍属于前端模块线索，不能当作运行时行为已开放的源码事实。

### 【信号】[仅 macOS] 本地会话和环境状态展示被重新编入前端图谱

macOS 新增多个 `local-conversation-page` 相关模块，包括 `app-initial~app-main~local-conversation-page`、`remote-conversation-page~thread-app-shell-chrome~local-conversation-page`、`hotkey-window-detail-layout~local-conversation-page`、`worktree-init-v2-page~hotkey-window-worktree-init-page~local-conversation-page`。同时一个 Appgen/项目索引相关模块新增 `This conversation is running in Codex Cloud.`、`This conversation is running in a local git worktree.`、`This conversation is running locally.` 以及 cloud/local/worktree 环境图标 tooltip。

这些是 macOS 侧更清楚地区分 Codex Cloud、本地 git worktree 和本地运行会话的信号。

### 【信号】[仅 macOS] 外部 agent 导入与 provider 设置进入设置相关 bundle

macOS 的 `pets-settings~appearance-settings~general-settings` 新增大量 external agent onboarding/import 字符串，包括 provider toggle、provider list、import options、项目导入、最近聊天导入、tools and setup 导入、导入中/导入完成状态、`Apps found`、导入 tooltip 中的 agent count。

这表明 macOS 设置相关前端里出现了外部 agent 导入或迁移流程信号。因为它来自同名前端模块差异，不能断言该流程已对所有用户可用。

### 【信号】[仅 macOS] Appgen、项目索引、发布条款与 onboarding 模块重排更大

macOS app.asar 去掉 hash 噪音后新增大量模块组合，集中在 `appgen-page`、`appgen-settings-page`、`appgen-library-page`、`appgen-publication-terms-route`、`projects-index-page`、`new-thread-panel-page`、`onboarding-page`、`plugin-detail-page`、`pull-requests-page` 和 `worktree-init-v2-page`。

这是一组 macOS 独有的大规模前端路由与 chunk 重排信号。它比 Windows 样本更活跃，但由于多数证据是模块名和 preload 图谱，不应写成具体功能已发布。

### 【信号】[仅 macOS] 自动化页体积显著增长，并出现模型适用性、AGENTS.md 与 referral 可访问性文案

macOS `automations-page` 从 `0.9 MB` 增至 `1.3 MB`，新增 `(Does not apply to current model)`、`AGENTS.md already exists here. Skipping /init to avoid overwriting it.`、slash command dialog 可访问性标签，以及 rate-limit reset referral、referral invite tracking、successful rate-limit reset referral 的可访问性描述。

这些信号覆盖模型适用性提示、`/init` 避免覆盖 `AGENTS.md`、斜杠命令对话框和 referral 相关弹窗。它们来自同名前端模块线索，因此只说明进入该 bundle 的方向。

### 【信号】[仅 macOS] Debug modal 扩展到应用动作、活动上下文与 unpackaged 构建开关

macOS `debug-modal` 从约 `0.0 MB` 增至 `0.2 MB`，新增 `Account ID`、`Active browser tab`、`Active conversation`、`Ambient suggestion chats`、`App Actions`、`Allow host`、`Allow host starting`、`Already current`，以及 `Allow automatic worktree cleanup in this unpackaged build.`。

这是 macOS 调试窗口能力面扩展的信号，覆盖账号、浏览器标签、会话、环境建议、App Actions、host 允许状态和未打包构建 worktree 清理开关。

### 【信号】[macOS+Windows] 宠物相关状态仍在重排，但两端证据不同

Windows 的 `pet-install-state` 新增样式类 `pet-install` 与 `pet-install-preview`。macOS 新增 `app-initial~app-main~pets-settings`、`avatar-overlay-page~avatar-overlay-native-page~pets-settings`，同时设置相关模块新增多组 pet/avatar preload 变化。

两端都出现宠物相关前端重排，但证据形态不同：Windows 是样式状态类，macOS 是模块图谱与资源 preload。这里不能断言新宠物功能，只能说宠物设置与安装预览相关 bundle 有调整。

### 【信号】[仅 macOS] 代码高亮语法资源重排

macOS 多个语法模块出现明显增减：`typescript`、`php`、`c`、`haxe`、`java`、`stylus`、`gdshader` 新增大量 scope 字符串；`javascript`、`less`、`python`、`scss`、`postcss`、`katex` 则移除大量字符串或体积下降；`diff` 新增 `markup.changed.diff`、`meta.diff.header.git`、`meta.diff.range.unified` 等 diff scope。

这更像语法高亮资源或代码展示依赖重排。它对代码阅读体验可能有影响，但仍属于 bundle 级信号。

### 【实证】[仅 macOS] CSP 未见策略语义变化，仅入口资源 hash 与 preload 图谱变化

macOS `webview/index.html` 的 CSP 内容保持为 `default-src 'none'`，并继续允许图片 `self/app/blob/data/https`，frame/child 指向 `web-sandbox.oaiusercontent.com`，worker 为 `self/blob`，script 为 `self` 加固定 sha256 与 `wasm-unsafe-eval`，connect 包含 OpenAI/Mapbox 与 ChatGPT websocket 端点。diff 展示的是入口脚本和 modulepreload 资源变化。

Windows 输入没有提供 AppxManifest 或 CSP 差异，因此不能做对应结论。
