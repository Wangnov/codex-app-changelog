# Codex 桌面版 26.601.20914(macOS / Windows 对照)

> 第三方跨平台变更日志 · 非官方

本批次显示 Codex(代码智能体产品) 桌面版在 macOS(苹果桌面操作系统) 与 Windows(微软桌面操作系统) 上同步进入 26.601 系列，但两端打包内容并不完全同速：应用层变化高度一致，平台外壳与内置组件各有侧重。  
One-liner: Codex 26.601 refreshes the shared app layer, browser/control policies, and packaged runtime, with macOS adding installer migration logic and Windows exposing more DevTools/Sites payload.

## 关于这份报告

这份报告基于官方签名分发包的逆向 diff(差异比对):macOS 取 Sparkle(应用更新框架) 全量包，Windows 取 MSIX(微软应用安装包) 包。macOS 对比 26.527.60818(build 3437) → 26.601.20914(build 3497)，Windows 对比 26.527.7698.0 → 26.601.1994.0，发布日期字段按输入事实包统一写为 2026-05-30。

证据等级分两档:【实证】表示来自 plist(属性列表)、AppxManifest(Windows 应用清单)、技能文档、package.json(项目配置)、依赖清单或可读 diff；【信号】表示只看到模块名、文件名或体积变化，能提示方向，但不能还原源码意图或确认用户可见功能。

平台归属标记含义是:[macOS+Windows] 两端都有证据；[仅 macOS] 只在 macOS 包中出现；[仅 Windows] 只在 Windows 包中出现。本文刻意忽略 webview/assets 与 .vite/build 的 hash(内容指纹) 重命名噪音。

## ✨ 重点变化

### 【实证】[macOS+Windows] 版本与运行时底座进入 26.601 / Chromium 149 批次

macOS 从 26.527.60818(build 3437) 升到 26.601.20914(build 3497)，Info.plist 显示 Chromium(浏览器内核项目) 从 148.0.7778.179 / 7778.179 升到 149.0.7827.54 / 7827.54。Windows 从 26.527.7698.0 升到 26.601.1994.0，文件层也出现 148.0.7778.179.manifest 移除、chrome.dll 与 resources.pak 体积变化，说明 Windows 包同样经历了底层浏览器运行时更新。

平台差异是 macOS 给出了清晰的 Info.plist 字段与 SCMRevision(源码管理修订标识) 变化；Windows 事实包主要体现为文件树与二进制/资源体积变化，没有给出同等清单字段。

### 【实证】[macOS+Windows] 应用层 app.asar 小幅增大，内部文件增减形态高度一致

两端 resources/app.asar 均增加约 +1.32 MB。macOS 应用层内部文件 2774 → 2804，新增 1001、移除 971、改动 77；Windows 为 2722 → 2752，新增 1001、移除 971、改动 76。两端都明确标注 webview/assets 下大量增减是 Vite(前端构建工具) 构建 hash 噪音，不应用来判断功能增减。

平台差异主要在基线文件数与改动数:macOS 比 Windows 多 52 个 app.asar 内部文件，内容改动多 1 个，但新增/移除数量一致，说明共享应用层更新节奏基本同步。

### 【实证】[macOS+Windows] 浏览器与 Chrome 控制策略放宽 CAPTCHA 处理，但仍要求确认

macOS 的 in-app browser(应用内浏览器) 技能文档和 Chrome(谷歌浏览器) 控制技能文档都把“不得解 CAPTCHA”改为“每次看到 CAPTCHA(人机验证) 时询问用户是否要代解，确认后仅解该次 CAPTCHA”。同时仍保留不得绕过 paywall(付费墙)、浏览器/网页安全拦截、年龄验证、最终密码修改提交等限制。

Windows 事实包未列出同一 diff 文本，但 Windows app.asar 的增删形态与 macOS 高度一致，且 Windows 新增文件中出现 `resources/plugins/openai-bundled/plugins/browser/.codex-plugin/control-in-app-browser-multi-tab.md`，表明浏览器控制插件也在更新。严格说，CAPTCHA 文案变更的可读实证来自 macOS；跨平台归属来自两端共享应用层与插件文件变化的合并判断。

### 【实证】[macOS+Windows] Sites 插件进入打包/市场入口，但 Windows 文件载荷更完整可见

macOS 的 marketplace.json(插件市场索引) 新增 `sites` 插件，source 指向 `./plugins/sites`，策略为 installation `AVAILABLE`、authentication `ON_INSTALL`，分类为 `Productivity`。Windows 的新增文件直接出现 `resources/plugins/openai-bundled/plugins/sites/skills/sites-building/templates/auth-siwc-enabled/package-lock.json` 与 `vinext-starter/package-lock.json`。

平台差异很明确:macOS 可读证据证明 Sites(站点相关插件) 已进入插件市场入口；Windows 可见的是 Sites 插件技能模板文件已经随包分发。Windows 事实包没有提供 marketplace.json diff，因此不能断言 Windows 的市场入口字段完全相同。

### 【实证】[仅 macOS] 新增从 DMG 挂载卷迁移到 Applications 的安装流程

macOS 的 bootstrap.js 可读 diff 新增 `pending-source-dmg-cleanup.json`、检测当前 app 是否位于 `/Volumes/`、定位 source DMG(磁盘镜像)、复制 app bundle(应用包) 到 `/Applications`、打开已安装应用、卸载 DMG 并移入废纸篓等逻辑。还新增一个安装进度窗口，显示 Installing、Moving to Applications、Opening、failed/openFailed 等状态。

Windows 包中没有对应 DMG / Applications 概念，也没有可比证据。这是本批次最明确的 macOS 独有平台外壳变化。

### 【实证】[仅 macOS] 更新确认、关于窗口与构建信息文案扩展到大量本地化文件

macOS 大量 native-menu-locales(原生菜单本地化资源) 增加 `appHeader.installUpdate.*`、`codex.aboutDialog.*` 等键，包括“现在更新 Codex？”、“更新会中断本机活跃本地会话”、“关于 {appName}”、“版本 {version}”、“构建信息”等。中文简体文件也新增了对应文本。

Windows 事实包没有列出这些本地化 diff，因此本文只标为 macOS 实证。考虑 app.asar 更新高度一致，Windows 可能也包含类似应用层文案，但输入事实不足，不能写成 Windows 已确认。

### 【信号】[macOS+Windows] 设置、计费、计划管理、线程虚拟化与 PR 修复入口出现模块级变化

macOS 去 hash 噪音后的新增模块包括 `agent-settings.search`、`computer-use-settings.search`、`mcp-settings.search`、`personalization-settings.search`、`plan-management-state`、`plan-pricing`、`upgrade-plan-dialog`、`pull-request-fix-button`、`thread-virtualizer`、`thread-resource-card` 等；移除模块包括 `plugin-auth`、`pull-request-board-model`、`virtualized-turns` 等。Windows 没有展开模块名列表，但 app.asar 新增/移除数量与 macOS 完全一致。

这些是方向信号，不等价于功能已开放。平台差异是 macOS 提供模块名，可读性更强；Windows 只提供同形态 app.asar 统计。

### 【信号】[macOS+Windows] 浏览器侧栏/隐藏 WebView/多标签方向增强

macOS 新增 `browser-sidebar-webview`、`browser-sidebar-retained-webview`、`browser-sidebar-hidden-background-webview-host`、`browser-sidebar-hidden-browser-use-webview-host` 等模块；Windows 新增 `control-in-app-browser-multi-tab.md` 文件。两者共同指向浏览器侧栏、WebView(嵌入式网页视图) 保留/隐藏宿主、多标签浏览控制方向的更新。

这里仍属【信号】:事实包没有源码语义与交互路径，不能确认用户界面实际开放哪些能力。

### 【信号】[仅 Windows] DevTools / Lighthouse 相关载荷显著变化

Windows 的 `gen/third_party/devtools-frontend/.../lighthouse-dt-bundle.js` 从 2.1 MB 增到 6.3 MB，增加 +4.23 MB，是 Windows 包体积变化最大的组件；同时新增/改动涉及 GreenDevAgent、WebMCPView、Puppeteer、AI assistance、HeapSnapshot 等 DevTools(开发者工具) 前端路径。macOS 事实包没有同类文件路径级证据。

这不说明 Codex 用户界面新增了对应 DevTools 功能，只说明 Windows MSIX 中的 Chromium/DevTools 资源有明显更新。

### 【信号】[仅 macOS] 开发命令 owl 增加 mock keychain 参数

macOS package.json 中 `owl` 脚本新增 `--use-mock-keychain`。这是开发/测试命令层面的变化，未显示为用户功能。Windows 事实包没有 package.json diff，因此仅标 macOS。
