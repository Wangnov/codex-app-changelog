# Codex 桌面版 26.601.20914 macOS / Windows 对照

> 第三方跨平台变更日志 · 非官方

这一批次的主轴是：Sites 能力面进入内置插件集合，同时桌面更新、关于窗口、设置搜索、插件安装、浏览器侧栏、PR 处理、跨主机/工作区会话等界面与运行时线索同步变化。  
One-liner: Sites becomes a bundled capability while the desktop shell, plugin flows, browser tooling, and task/worktree surfaces move forward across both platforms.

## 关于这份报告

本报告只使用输入事实包中的机器提取差异。macOS 样本对比 26.527.60818 build 3437 → 26.601.20914 build 3497；Windows 样本对比 26.527.7698.0 → 26.601.1994.0。两者来自同一套 Codex 代码的不同打包，但版本号、包体、文件数与平台组件分别引用，不互相代替。

证据分两档：  
【实证】来自插件清单、技能文档、package.json、Info.plist、可读字符串、资源清单、依赖清单与具体配置/文档 diff。  
【信号】来自同名前端模块变化、模块增删、样式类名、体积方向与压缩 bundle 线索；这些说明方向，不等于完整源码语义，也不保证已对所有用户开放。

平台标记为：[macOS+Windows]、[仅 macOS]、[仅 Windows]。webview/assets 与 .vite/build 的 hash 改名已按事实包要求视为构建噪音。

## ✨ 重点变化

### 【实证】[macOS+Windows] Sites 作为内置插件进入分发包

两端内置插件列表都从 `browser`, `chrome`, `computer-use`, `latex` 扩展为 `browser`, `chrome`, `computer-use`, `latex`, `sites`。新增插件显示名为 Sites，版本 `0.1.10`，分类为 Productivity，关键词包含 `deploy`, `sites`, `games`。

Sites 的能力描述非常明确：Create, save, deploy, and inspect sites hosted by OpenAI。默认提示包括披萨餐厅落地页、带排行榜的每日数独、图像生成网站。macOS 插件目录 60 个文件、0.8 MB；Windows 为 60 个文件、0.9 MB。

技能文档显示这不是单纯包体变化，而是完整能力面：`sites-building` 面向创建或修改站点，要求产物兼容 Cloudflare Workers，优先使用 vinext 模板，并在 `.openai/hosting.json` 中保存托管元数据；`sites-hosting` 面向创建/复用托管配置、保存部署版本、部署生产环境、检查部署状态与管理访问控制。模板中还包含 `vinext-starter` 与 `auth-siwc-enabled`，最大文件为两个 `package-lock.json`，各 0.4 MB。

### 【实证】[macOS+Windows] app.asar 规模同步增长，前端真实模块增删高度一致

macOS `Contents/Resources/app.asar` 从 140.3 MB 增至 141.6 MB，+1.32 MB；Windows `resources/app.asar` 从 144.1 MB 增至 145.4 MB，+1.32 MB。去掉 hash 噪音后，两端新增模块列表一致，内部文件规模也接近：macOS 2774 → 2804，Windows 2722 → 2752，均为新增 1001、移除 971。

新增模块覆盖设置搜索、Sites 颜色、服务层级、计划与升级、线程资源卡、虚拟化渲染、工作区/项目近期聊天信号、浏览器侧栏 webview、插件连接流、PR 修复按钮、VS Code API 等。移除模块包括旧的 app connect modal、plugin-auth、share、sidebar-thread-section、virtualized-turns、pull-request-board-model、local-remote-selection 等。作为【信号】，这表明前端壳层、设置、插件、浏览器、PR 与线程渲染相关区域在重排，但不能据此还原源码实现。

### 【实证】[macOS+Windows] 桌面更新确认与关于窗口进入多语言资源

两端主进程/前端新增字符串都包含 `appHeader.installUpdate.confirmTitle`、`appHeader.installUpdate.confirmSubtitle`、`codex.aboutDialog.*` 与 `Build information`。中文资源明确新增“现在更新 Codex？”、“Codex 将退出以安装更新，这将中断此设备上正在进行的本地会话”、“关于 {appName}”、“版本 {version}”、“版本 {version} • 发布于 {releaseDate}”与“构建信息”。

同名 `app-main` 模块也新增 forced desktop app update warning popup、restart deadline、`Codex needs to restart to finish installing an update` 等字符串。平台差异主要在打包层：macOS 同时有 Info.plist 版本、Chromium 与 app.asar 完整性 hash 更新；Windows 事实包未给出 AppxManifest 具体 diff，但 package.json 与本地化资源同样进入新版本内容。

### 【信号】[macOS+Windows] 设置页改为可搜索、分组导航更清晰

新增模块中出现 `agent-settings.search`、`computer-use-settings.search`、`general-settings.search`、`git-settings.search`、`mcp-settings.search`、`personalization-settings.search`。同名 `settings-page` 模块新增“settings search input”、“clear settings search”、“no matching destinations”以及 app-level、archived、coding、host-scoped、integration settings 等导航分组文案。

这属于界面与导航重构信号：搜索入口、空状态、分组标题和可折叠导航进入前端 bundle，但事实包不证明每个设置项已经对所有用户开放。

### 【信号】[macOS+Windows] 插件安装、连接器与权限说明更面向付费计划和风险提示

`use-plugin-install-flow` 两端新增“Badge label shown on Google plugins that need a paid plan”、Gmail drafting、Gmail task extraction、Google Calendar editing/planning/scheduling 等升级安装弹窗说明，并新增“Codex can read the thread, understand the ask, and draft a useful response”。

`plugins-availability` 新增“Approved by your admin”、“Advanced settings (opens ChatGPT.com)”、“Connectors may introduce risk”、“Connectors are designed to respect your privacy, but sites may attempt to steal your data.”、“Could not load this app”等文案。Windows 额外列出 `plugin-detail-page` 改动：`Add plugin` 替代 `Add to Codex`，新增 MCP servers、included skill details、打开 Hooks settings、插件共享目标与成员计数相关文案。macOS 事实包未列出同名 `plugin-detail-page` 线索，因此该细节只记为 Windows 样本中的信号。

### 【实证】[仅 macOS] 浏览器控制技能的 CAPTCHA 策略被改写为“确认后可解”

macOS 的 `control-chrome/SKILL.md` 与 `control-in-app-browser/SKILL.md` 均从“Do not solve CAPTCHAs”改为“每次看到 CAPTCHA，询问用户是否希望你解决；确认后只解决该 CAPTCHA”。同时仍保留不得绕过付费墙、安全插页、年龄验证或最终密码修改步骤的限制。

Windows 事实包没有列出这两个技能文件的同样 diff，所以本报告只标为 macOS 实证。即使两端代码来源接近，也不能把 macOS 文件级结论套到 Windows。

### 【实证】[macOS+Windows] 多标签浏览器后端、浏览器识别与文件解析字符串进入包内

两端新增可读字符串包含 `browser.tabs.finalize is only available with Chrome or multi-tab IAB.`、`browser.user.claimTab is only available with the Chrome or multi-tab IAB backend.`，相比移除项中仅限 Chrome backend 的表述，说明多标签 IAB 路径进入包内。

同一批字符串还新增 Brave Browser、Firefox Developer Edition、Google Chrome Beta/Canary、Microsoft Edge/Beta、Safari、Opera、Vivaldi、Chromium、Kagi 等浏览器名或 bundle id，以及 `List Chrome tabs`、`Codex browser tab was closed by Browser Use`。文件解析方面，新增 ZIP、PNG、TIFF、ASF、EBML/WebM 相关错误与结构字符串，如 `Encrypted ZIP`、`Unsupported ZIP compression method:`、`PNG chunk payload`、`TIFF IFD offset`、`EBML DocType`。这些是包内实证，但不能单独断言完整功能入口已公开。

### 【信号】[macOS+Windows] Browser Use 网站权限与原生 Setup Codex 流程出现新交互

`pending-request-item-panel` 两端新增“Allow Codex to interact with any website without asking?”、“Allow any website”、“cancel/confirm allowing Browser Use on any website”，同时新增 MCP form elicitation、Setup Codex role picker、context source 已连接状态、必填字段提示等文案。

`app-server-manager-signals` 新增“Advance the native Setup Codex flow through role, task, context, plugin, and completion steps.”，并加入 Browser Use URL 无 host、URL 无效、Chrome tab URL blocked、非文本文档保存限制等错误路径。这里的权限、设置向导与浏览器安全拦截是高可见信号，但仍属于压缩前端模块线索。

### 【信号】[macOS+Windows] PR 页面与 Git/CI 工作流交互增强

两端 `pull-requests-page` 新增 PR 活动回复、详情评论、打开 GitHub、auto-merge toggle、刷新 CI checks、actions、reviewers、description 等可访问标签。新增模块中还有 `pull-request-fix-button` 与 `parse-diff`。

macOS 的 `thread-side-panel-tabs` 还新增一组 CI 修复流程提示字符串，包括先检查 CI 系统、凭据、权限、token、MCP 或工具访问，缺失时明确说明，具备条件则获取外部日志、诊断并做最小修复。Windows 事实包没有列出该模块，因此这组 CI 指令只作为 macOS 样本信号。

### 【信号】[macOS+Windows] 工作区、worktree、跨主机 handoff 与后台 chat 文案重排

`split-items-into-render-groups` 两端把 thread 文案改为 chat 文案：`Created background chat`、`Created worktree chat`、`Creating background chat`、`Creating worktree chat`，并新增 `worktree-backed` 样式类。

`local-remote-dropdown` 两端新增 branch 已存在、输入 worktree 分支名、handoff 前检查队列消息、检查当前 host 是否匹配项目、移动本地/远程 conversation 到 worktree 或 checkout 等文案。这说明本地、远程、worktree 与 handoff 的交互面在扩展，但仍是前端信号。

### 【信号】[macOS+Windows] 线程侧栏与底部面板向多标签工作台靠拢

`thread-app-shell-chrome` 两端新增打开 side chat、browser tab、review tab、terminal、timeline、workspace file browser 的动作标签，同时新增 Bottom panel、关闭底部面板、从底部或侧边面板打开另一个 tab。macOS 的 `thread-side-panel-tabs` 体积从 0.0 MB 到 0.3 MB，新增大量可访问标签与工具流文案；Windows 事实包则列出 `header` 模块新增 recent tasks menu、All tasks、Cloud tasks、环境过滤与 Chrome extension thread 打开入口。

这两组都指向任务、面板、标签页和侧栏调度增强，但平台样本中可见模块不同。

### 【信号】[macOS+Windows] Sites 列表从卡片/网格向表格行交互调整

`appgen-page` 两端新增“Sites list”的站点列头、打开 live site、打开 settings、打开 sharing settings、Open in browser、外部浏览器 hover 副标题等字符串；移除项中则出现 grid、card actions dropdown、relative today/yesterday、site actions 等旧文案。样式类从卡片 hover/opacity/grid-cols-1/lg:grid-cols-3 转向 `data-appgen-row`、`grid-cols-subgrid`、`h-[50px]`、`border-b` 等行式布局信号。

结合 Sites 插件加入，这是一个较强的 Sites 管理界面改版信号，但仍不能证明用户一定能看到完整 Sites 列表。

### 【信号】[macOS+Windows] 通用设置新增语音、字号、权限与工作模式相关入口

### 【实证】[仅 Windows] Chromium/DevTools 侧体积变化明显，新增 GreenDev 与 WebMCP 相关文件
