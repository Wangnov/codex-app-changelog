# Codex 桌面版 26.715.61943 macOS / Windows 对照

> 第三方跨平台变更日志 · 非官方

本批次的共同主线是线程工作区面板重构；Windows 包还出现更密集的浏览器操作、Computer Use、Office 实时控制和文件渲染信号，两平台在设置、站点分享和首页公告模块上则明显错位。

Shared thread-workspace changes are joined by Windows-heavy browser, Computer Use, Office control, and file-rendering signals, while settings, sharing, and home announcements move in opposite packaging directions.

## 关于这份报告

本报告比较两组官方分发产物：macOS arm64 从 26.715.52143（build 5591）到 26.715.61943（build 5628），Windows x64 MSIX 从 26.715.7063.0 到 26.715.8383.0。两组版本窗口并不一致，因此所有结论均按平台独立引用，不能将一侧的变化直接视为另一侧同期变化。

macOS 为历史全量包对比，两个样本均通过 Apple 公证验证，签发主体为 OpenAI OpCo, LLC。Windows 事实包提供了两个全量包的 SHA-256，但未附可单独引用的签名验证输出。

证据分为两档：

- 【实证】来自 plist、插件 manifest、依赖清单、服务入口、签名验证及不属于同名前端模块线索的包内可读事实。
- 【信号】来自同名前端模块中的字符串、样式类名及体积变化，只能说明相关 UI、状态或交互代码发生变化，不能证明功能已经开放。

平台标记为 `[macOS+Windows]`、`[仅 macOS]` 和 `[仅 Windows]`。

证据盘点覆盖了版本与渠道、Bundle 文件树、Computer Use 运行时、`cua_node` 依赖、`app.asar`、同名前端模块、Info.plist 和关键文件 diff。输入没有提供独立的新增插件清单、主进程新增字符串、CSP diff、Windows AppxManifest diff、新增组件资源清单或去除 hash 噪音后的真实前端文件增删表，因此不对这些项目作额外推断。

## ✨ 重点变化

### 线程侧栏与底部面板形成共同变化主线

【信号】[macOS+Windows]

两平台的 `thread-app-shell-chrome` 都新增了同一组面板文案和响应式条件，包括：

- `Bottom panel`，以及关闭线程底部面板、从底部或侧边面板打开另一标签页的按钮标签。
- 从任务侧栏打开浏览器、Review、终端和工作区文件浏览器。
- 从聊天侧栏打开 side chat，以及从侧栏新标签页打开 Detail。
- 当前聊天没有可用侧栏动作时的空状态。
- `(max-width: 920px)` 响应式条件及一组面板布局、背景、边框和焦点样式。

这是本批次最明确的跨平台共同 UI 信号。Windows 的相关变化随后延伸到完整浏览器标签与任务操作，而 macOS 输入只支持共同壳层这一层结论。

### Windows 线程面板出现完整浏览器控制与任务操作信号

【信号】[仅 Windows]

`thread-side-panel-tabs` 增长约 343,826 bytes，新增字符串集中指向浏览器标签和任务侧栏：

- 浏览器标签的音频播放、摄像头或麦克风占用状态。
- 下载操作菜单、下载行及在系统文件浏览器中显示文件。
- 地址栏建议。
- 设备工具栏的宽度、高度、缩放和关闭操作。
- 取消正在执行的 Git 操作、接受编辑并暂存，以及分支已有 Pull Request 的状态。

这些字符串证明相关文案和状态已进入 Windows 前端模块，但不能据此断言所有用户都能看到完整浏览器设备模拟或 Git 工作流。

### Windows 的 @ 提及入口连接 Computer Use、实时 Office 控制与代理类型

【信号】[仅 Windows]

`at-mention-list` 增长约 170,819 bytes，新增了相互印证的一组入口文案与状态类名：

- `Computer use` 及对应功能标签。
- Microsoft Excel 的实时工作簿控制。
- Microsoft PowerPoint 的实时演示控制。
- `Custom agents`、`Live agents`、`Desktop apps`。
- `ChatGPT conversation(s)`、`Files and chats`。
- `microsoft-excel-plugin`、`microsoft-powerpoint-plugin`、`mcp-capability-mentions` 和 `mcp-servers` 类名。

这是 Windows 包中新的入口与路由方向信号，不等同于确认这些能力已面向全部账户启用。

### Windows Markdown 渲染加入文件引用和图片加载状态信号

【信号】[仅 Windows]

`markdown` 模块从约 0.1 MB 增至 0.4 MB，新增文案覆盖：

- 带位置信息的文件引用 chip。
- Markdown 图片缺少替代文本时的按钮标签。
- 图片字节加载期间的占位状态。
- 图片加载失败时的回退状态。
- 已有一条后续消息等待确认的状态。

模块还加入大量可访问性状态类名，并移除一批 Markdown/TextMate 风格的语法作用域字符串。事实支持渲染和交互路径重组，不足以确认具体编辑器实现被替换。

### 浏览器与 Chrome 插件元数据在两平台同步更新

【实证】[macOS+Windows]

两平台内置 `browser` 和 `chrome` 插件的 manifest 版本均从 `26.715.52143` 更新为 `26.715.61943`。

`browser` manifest 继续定义 `@browser`、`@browser-use` 等别名，并将本地地址、`file://`、当前内置浏览器标签的打开、检查、导航、测试、点击、输入和截图归入内置浏览器使用范围。`chrome` manifest 继续面向依赖现有 Chrome 标签页、登录会话、Cookie、扩展和 ChatGPT Chrome Extension 配置的自动化任务。

本次可确认的具体 diff 是版本字段同步，描述文本本身没有变化，因此不能将这些能力描述成这一批才新增。

### Windows 的 Computer Use 插件同步内部版本

【实证】[仅 Windows]

Windows 内置 `computer-use` 插件的 manifest 版本从 `26.715.52143` 更新到 `26.715.61943`。其描述仍为通过 Computer Use 从 ChatGPT 控制 Windows 桌面应用。

这是既有插件的版本元数据变化，不是插件新增证据。Windows 的 `cua_node` 总量保持 287.5 MB，依赖包新增数为 0。

### 插件、技能和 MCP 设置在 macOS 出现合并入口信号

【信号】[仅 macOS]

`plugins-settings` 新增：

- `Manage plugins, skills, and MCPs`
- “合并后的插件设置区域”副标题
- `plugins-settings` 类名

这指向插件、技能与 MCP 管理界面的合并组织方式。输入没有提供新插件目录或 manifest 新增记录，因而不能推断本批次增加了新的内置插件。

### 首页公告在两平台呈相反打包方向

【信号】[macOS+Windows]

macOS 的 `home-announcements` 增长约 28,328 bytes，加入：

- 多代理 composer banner 的正文和关闭标签。
- `Delegate work to subagents that work in parallel. Note: may increase token usage.`
- Fast mode、Codex gift credits、rate limit reset 和后端驱动首页 banner 的关闭标签。
- 礼品额度购买流程、可用 rate-limit reset 弹窗入口。
- `has-seen-multi-agent-composer-banner` 等状态类名。

Windows 的同名模块则缩小约 26,379 bytes，并移除上述多代理、Fast mode、gift credits、rate-limit reset 和后端 banner 字符串及状态类名。

这说明两个版本窗口中的前端打包方向相反；不能由此断言某项公告在某个平台永久上线或下线。

### 通用设置模块在两平台反向移动

【信号】[macOS+Windows]

Windows 的 `general-settings` 增长约 92,351 bytes，新增的设置文案包括：

- 默认权限与 full access 可见性。
- composer context usage 指示器。
- 聊天、diff 代码字号和 ChatGPT UI 基础字号。
- 快捷键捕获。

- diff 颜色标记选项。

Windows 包内出现带有 “macOS Dock icon” 的字符串，只能证明这些标签进入该共享前端构建，不能视为 Windows 提供 macOS 专属设置。

### 站点发布与分享界面在 Windows 加入、macOS 移出相关信号

【信号】[macOS+Windows]

Windows 的 `appgen-share-dialog` 增长约 23,410 bytes，新增：

- 添加人员或群组、搜索与移除工作区用户。
- “工作区内持链接者”和“Internet 上任何人”可见性。
- 发布站点到公网。

这是两个不同版本窗口内的反向模块信号，不足以证明站点发布能力只属于 Windows。

### Windows 临时聊天 UI 路径被缩减

【信号】[仅 Windows]

`chatgpt-temporary-chat-ui` 缩小约 13,539 bytes，并移除：

- 开启和关闭 temporary chat 的按钮与可访问标签。

这说明该独立 UI 模块中的临时聊天文案被移除；不能仅据此确认临时聊天能力本身已被删除。

### macOS 工作簿相关代码发生明显重新分块

【信号】[仅 macOS]

`workbook-from-markdown` 增长约 108,179 bytes，新增多组文档关闭、序列化、插件列表、查找替换、路径和异步处理错误字符串。

与此同时，`workbook` 从约 2.6 MB 缩至接近 0 MB，并移除命名范围、主题色和大量 `data-granola-*` 样式或节点类名。两者结合更像工作簿代码重新分块或依赖搬移，但没有源码或新增文件清单支持具体架构结论。

### macOS 加入插件事件总线连接与排队信号

【信号】[仅 macOS]

`esm` 模块新增了插件事件订阅、客户端总线和内部事件目标相关文案，包括：

- 连接 event bus。
- 总线不可用时排队，连接后推送。
- 发出排队事件。
- event bus client 禁用时不发送或不注册。
- 达到最大重试次数。
- shim event 分发失败。

同时移除了 `Failed to deliver`、`Not ready`、`Promise timed out` 等旧错误文案。这指向插件事件传递与重连路径调整。

### 调试弹窗在两平台共同大幅缩减

【信号】[macOS+Windows]

`debug-modal` 在 macOS 缩小约 230,423 bytes，在 Windows 缩小约 226,002 bytes。两侧共同移除 `Account ID`、`Active browser tab`、`Active conversation`、`Activity grouping version`、进程与类型筛选，以及自动清理 worktree 等调试字符串和大量弹窗样式。

这支持调试界面被拆分、裁剪或移位的方向判断，但不能确认这些诊断能力从应用内部彻底删除。
