# Codex 桌面版 26.513.31313 macOS / Windows 对照

> 第三方跨平台变更日志 · 非官方

这批输入显示：Windows 包发生了明显的内置插件重命名与 LaTeX/Browser 能力面重组；macOS 包则更像同一代码线上的轻量落点，主要体现在 Computer Use 运行时与若干前端交互信号。

One-liner: Windows gets the heavier plugin/runtime reshuffle, while macOS shows a narrower Computer Use and UI-signal update.

## 关于这份报告

本报告只基于输入中的官方签名分发包逆向 diff 事实：macOS arm64 为 `26.513.20950` build `2816` 到 `26.513.31313` build `2867`；Windows x64 MSIX 为 `26.506.3741.0` 到 `26.519.2736.0`。

证据分两档：

- 【实证】：来自 plist、manifest、package.json、插件清单、技能文档、服务入口、依赖清单、明确配置 diff、以及事实包中新增可读字符串等。
- 【信号】：来自前端模块名、压缩 bundle 体积、同名模块可读字符串/样式类名变化；说明功能方向进入包内或界面结构有调整，但不等于所有用户已可见。

平台标记为 `[macOS+Windows]`、`[仅 macOS]`、`[仅 Windows]`。两平台版本和发布时间不对齐，因此每条结论只引用对应平台事实，不把一个平台的变化外推到另一个平台。

## ✨ 重点变化

### 【实证】[仅 Windows] 内置插件从 `browser-use` / `latex-tectonic` 重组为 `browser` / `latex`

Windows 内置插件清单从上一版的 `browser-use`、`chrome`、`latex-tectonic`，变为最新版的 `browser`、`chrome`、`latex`。这不是单纯文件名变化：事实包给出了新增插件 manifest、技能说明、默认提示和最大文件。

`browser` 插件版本为 `26.519.22136`，文件数 `341`，总大小 `7.6 MB`。它的说明明确覆盖 Codex 内置浏览器的打开、导航、检查、点击、输入、截图，以及对 `localhost`、`127.0.0.1`、`::1`、`file://` 等本地目标的测试。默认提示包含“Test my checkout flow on localhost”。

`latex` 插件版本为 `0.2.0`，文件数 `18`，总大小 `49.7 MB`。它不只是把 `latex-tectonic` 改名：新说明覆盖 Tectonic 优先、TeX Live 或 MacTeX 回退、`latex-doctor` 检测、`latex-compile` 编译、以及在无可用系统安装时安装 Codex 托管 TeX Live 的流程。最大文件为 `bin/tectonic.exe`，大小 `49.6 MB`。

### 【实证】[仅 Windows] Browser 插件运行时被内置进包内，体积变化显著

Windows 新增文件中出现 `resources/plugins/openai-bundled/plugins/chrome/app-server-runtime/codex`，大小 `247.7 MB`，以及同目录 `node`，大小 `91.4 MB`，`node_repl` 为 `13.5 MB`。同时 `resources/app.asar` 从 `136.7 MB` 增至 `159.0 MB`，增加 `22.25 MB`；`Codex.exe` 从 `222.7 MB` 增至 `226.6 MB`，增加 `3.92 MB`。

这说明 Windows 包在本输入样本中把更多浏览器/应用服务运行时内容纳入分发产物。macOS 本批次没有对应的新增插件清单变化。

### 【实证】[仅 Windows] Chrome 扩展检测开始按多 Profile 汇总

Windows 的 `check-extension-installed.js` 明确改变了 Chrome 扩展检测逻辑：旧版围绕单个 Preferences 路径解析，新版新增 `listChromeProfileDirectories`、`getChromeExtensionInstallStatusForProfile`，会枚举 `Default` 与 `Profile N`，并输出每个 profile 的安装、注册、启用、禁用原因和版本信息。

新版还把 unpacked extension 的 `preferences.path` 作为安装判断的一部分。命令行输出从“Checked Chrome profile”转向“Checked Chrome user data directory”“Selected Chrome profile”“Installed in any profile”“Enabled in any profile”。

### 【实证】[仅 Windows] LaTeX 从单引擎打包走向完整工作流

Windows 旧插件 `latex-tectonic` 描述为用 bundled Tectonic 编译 LaTeX 和 TeX；新插件 `latex` 描述为先用 bundled Tectonic 处理简单项目，再回退到 TeX Live 或 MacTeX，并可检测本机 LaTeX 准备情况。

技能文档给出三条能力线：

- `latex-compile`：运行 `python3 scripts/compile_latex.py /absolute/path/to/main.tex`，支持 `--compiler tectonic`、`--compiler texlive`、`--engine xelatex`、`--json` 等。
- `latex-doctor`：检测 Tectonic、TeX Live、MacTeX、`latexmk`、`pdflatex`、`xelatex`、`lualatex`、`biber`、`kpsewhich`。
- `texlive-runtime-installer`：默认只检测；只有用户确认后才运行 `--install-managed-full`，安装路径为 `~/.cache/codex-runtimes/codex-texlive/full`，并声明不使用 `sudo`、不写 `/Library/TeX`、不写 `/usr/local/texlive`、不改 shell 启动文件。

### 【实证】[仅 macOS] Computer Use 插件小版本升级，运行时二进制小幅增大

macOS 的 `computer-use` 插件 manifest 从 `1.0.791` 升至 `1.0.793`。同一平台文件树没有新增或移除文件，文件数保持 `1331 → 1331`，内容改动 `96` 个。

体积变化集中在 Computer Use 运行时：`CUALockScreenGuardian` 从 `14.7 MB` 到 `14.9 MB`，增加 `0.17 MB`；`SkyComputerUseService` 从 `15.0 MB` 到 `15.1 MB`，增加 `0.14 MB`；`SkyComputerUseClient` 从 `10.1 MB` 到 `10.2 MB`，增加 `0.08 MB`。

### 【信号】[macOS+Windows] 远程连接与设备控制界面继续演进，但两平台落点不同

macOS 的 `remote-connections-settings` 新增了加载“可控制本机的设备”失败、需要重新 ChatGPT 认证、重试加载设备等文案，包括 `settings.remoteConnections.remoteControlClients.loadError.authRequired` 和 `Retry to load devices that can control this computer`。

Windows 同名模块变化更大，新增“Access and control other devices from this computer”“Add SSH Connection”“Add device”“Add device to control this Mac remotely”“Add device to control this PC remotely”等文案，并增加远程连接视图选择器、SSH 错误详情 tooltip、刷新已登录设备和已发现 SSH 连接等可访问性标签。

结论边界：这些是进入前端 bundle 的 UI/状态信号；不能据此断言对应入口对所有用户开放。

### 【信号】[macOS+Windows] 插件页、插件详情和 marketplace 管理文案明显扩展

macOS 的 `plugin-detail-page` 新增 `Add to Codex`、`Adding to Codex`、`App enabled`、`App disabled`、`App unavailable`、`Add people`、`Anyone at workspace with the link`，还出现插件分享对话框与 MCP server enablement toggle 的可访问性文案。

Windows 的 `plugins-page` 新增 `Access is turned off by your admin`、`Built-in marketplaces are upgraded by Codex`、`Built-in marketplaces cannot be removed`、`Choose another plugin marketplace`、`Create a plugin for your team`、`Created by me`、`Curated by OpenAI`、`Disabled by admin`、`Have a workflow in mind?`。

这是一组跨平台插件管理能力的高可见信号，但 macOS 证据来自插件详情页，Windows 证据来自插件页和新增插件清单，不能合并成完全相同的功能宣告。

### 【信号】[macOS+Windows] 本地对话线程增加摘要、评审、差异操作和附件相关入口

macOS 的 `local-conversation-page` 新增 `Toggle summary`、`Toggle pinned summary`；`local-conversation-thread` 新增 `Open code review comments`、`localConversation.reviewComments.open`，并出现 `model-authored`、`group/review-comments` 等类名。

Windows 的 `local-conversation-thread` 新增打开 Google Drive conversation resource card、打开 active heartbeat automation、打开 completed turn diff card 的 review view、pull request summary row 披露箭头等文案。`diff-unified` 新增 `Stage all`、`Stage file`、`Revert all`、`Revert file`，以及按 file/hunk/section stage、unstage、revert 的 tooltip。

这些变化共同指向线程内摘要、评审、差异处理、外部资源卡片和自动化状态的界面重组。

### 【信号】[仅 Windows] Codex Mobile、Appshot、浮动头像和项目生成入口进入更多界面

Windows 新增模块包含 `codex-mobile-page`、`codex-mobile-setup-dialog`、`codex-mobile-setup-flow`、`appshot-logo`、`appshot-window`、`appshots-settings`、`avatar-mascot-button`、`avatar-overlay-analytics`。同名模块中还出现 `Appshot shortcut failed/found target/had no target/received/started appshot`，以及 `Appshot attachment`、`Alt text for appshot attachment in local conversation`。

`welcome-page` 新增 teen onboarding 示例提示，包括 `Build a game you can play in your browser.`、`Dodging Game`、`Write my own prompt`。这些是 Windows 前端中较高可见度的产品入口信号，macOS 本输入没有同等新增模块证据。

### 【信号】[仅 Windows] 使用量、额度和自动充值设置进入更完整状态

Windows `usage-settings` 新增 `Add credits`、`Auto-reload credits`、`Auto-reload is off`、`Auto-reload is on`、`Automatic reload`、`Automatically add credits when your balance runs low`、保存自动充值设置和提交加 credits 流程的按钮文案，并出现 `https://developers.openai.com/codex/pricing`。

`app-main` 与 `composer` 也新增 `Add credits`、`Add credits to continue using Codex now.`、`Button label for upgrading from a core usage surface` 等文案。证据等级仍为信号，因为这些来自前端 bundle 文案和模块变化。

### 【信号】[仅 Windows] Hooks 设置从事件列表转向项目级信任审查

Windows `hooks-settings` 新增 `Hooks can run outside of the sandbox so we ask you to review any recently installed or modified hooks`、`Banner shown when hooks require trust review`、`Projects with configured hooks will appear here`、`No hooks found`、`No projects found`、项目行 hook load issue count 等文案。

同时移除了 `Before a tool executes`、`After a tool executes`、`Before Codex compacts the conversation`、`After Codex compacts the conversation` 等更偏事件枚举的文案。整体信号是 Hooks 页面在强调项目级发现、加载问题和信任审查。

### 【实证】[仅 Windows] AppxManifest 新增 Office 表格类文件关联

Windows AppxManifest 新增文件类型关联：`.tsv`、`.xls`、`.xlsm`、`.xlsx`。这属于 manifest 级事实，说明 Windows 包声明了新的文件类型处理范围。

### 【实证】[仅 Windows] Electron 和关键依赖版本升级

Windows `package.json` 显示应用层版本从 `26.506.31421` 到 `26.519.22136`，`codexBuildNumber` 从 `2620` 到 `3003`。`electron` 从 `41.2.0` 升到 `42.1.0`，`better-sqlite3` 从 `^12.8.0` 升到 `^12.9.0`。

脚本层面，`dev` 和 `owl` 统一改为调用 `scripts/prepare-dev.mjs`，新增 `forge:make:owl-shell` 和 `e2e:computer-use-native-pipe`。

### 【实证】[仅 Windows] Web sandbox preload 新增 `navigate` 端口

Windows `.vite/build/sandbox-preload.js` 的 MCP apps sandbox guest message 端口列表新增 `navigate`。原列表从 `notifyMcpAppsHostContext` 等开始，新版把 `navigate` 加入同一个 portNames 集合。证据来自可读 preload diff，但具体导航语义仍不能脱离源码进一步展开。

### 【信号】[macOS+Windows] 代码高亮/语法语法包在多个语言上变化明显

macOS 同名前端模块中，`cpp`、`typescript`、`tsx`、`php`、`less`、`csharp`、`fortran-free-form`、`haxe`、`scss`、`python`、`ruby`、`tex`、`regexp`、`gdshader`、`sql`、`json` 等模块新增大量语法 scope 字符串和类名。

Windows 的对应证据更分散，但 `pdf`、`general-settings`、`skills-page` 等模块出现与 macOS 相反方向的体积迁移：例如 macOS `pdf` 体积从 `0.4 MB` 降到 `0.0 MB`，Windows `pdf` 从 `0.0 MB` 增到 `0.4 MB`；macOS `general-settings` 移除大量设置文案，Windows `general-settings` 新增同类文案。这更像跨平台打包切分或版本窗口错位造成的前端模块重排，不能直接解释为功能删除/新增。
