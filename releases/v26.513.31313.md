# Codex 桌面版 26.513.31313(macOS / Windows 对照)

> 第三方跨平台变更日志 · 非官方

这一批次主要是 OpenAI（人工智能公司）Codex（OpenAI 的代码代理桌面应用）桌面版在 macOS（苹果桌面操作系统）与 Windows（微软桌面操作系统）两个分发包里的应用层、浏览器相关插件与底层运行组件同步变化，同时 Windows 包含更多目录级重组信号。

One-liner: this cross-platform batch updates app-layer and browser-automation-related components, with Windows showing larger packaging and plugin layout changes.

## 关于这份报告

本报告基于官方签名分发包的逆向 diff（差异比较）：macOS 侧比较 Sparkle（macOS 应用更新框架）全量包，Windows 侧比较 MSIX（Windows 应用安装包格式）相关分发包。macOS 对比范围为 26.513.20950(build 2816) → 26.513.31313(build 2867)；Windows 对比范围为 26.506.3741.0 → 26.519.2736.0。发布日期按本批次事实包给定为 2026-05-16。

证据分两档：【实证】来自 plist（macOS 属性列表）、插件配置、可读文件差异、依赖清单、文件树和包元数据等可直接核对的内容；【信号】来自前端模块名、文件体积、目录迁移或二进制体积变化，只能说明方向，不能还原源码意图。

平台归属标记含义：`[macOS+Windows]` 表示两平台事实包都出现同主题变化；`[仅 macOS]` 表示只在 macOS 包中可见；`[仅 Windows]` 表示只在 Windows 包中可见。

## ✨ 重点变化

### 应用层资源整体更新【实证】[macOS+Windows]

两平台的 app.asar（Electron 应用资源归档）都有变化。macOS 的 `app.asar` 从 147.7 MB 到 147.7 MB，变化约 +0.01 MB；内部文件从 1419 到 1421，新增 972、移除 970、改动 11。Windows 的 `resources/app.asar` 从 136.7 MB 增至 159.0 MB，增加 +22.25 MB；内部文件从 1336 到 1530，新增 1188、移除 994、改动 15。

平台差异很明显：macOS 的应用层体积几乎不变，Windows 则出现大幅增长。两边都存在 WebView（内嵌网页视图）/assets 与 Vite（前端构建工具）构建 hash（哈希标识）重命名噪音，因此不能把这些资产文件名变化直接解读成新功能。

### 浏览器自动化相关插件继续调整【实证】[macOS+Windows]

macOS 侧可见 `browser-client.mjs` 文件有小幅变化，出现在 `chrome` 与 `browser` 插件路径下；Windows 侧 `resources/plugins/openai-bundled/plugins/chrome/scripts/browser-client.mjs` 从 0.7 MB 增至 0.9 MB，且新增了 `resources/plugins/openai-bundled/plugins/browser/scripts/browser-client.mjs`。

平台差异在于 Windows 的目录级变化更强：旧的 `browser-use` 插件路径下大量脚本、依赖和技能文档被移除，同时新增的路径集中在 `browser` 插件下。这更像一次插件命名或布局迁移；但事实包没有源码语义，不能进一步断言用户可见能力变化。

### Windows 出现 Chrome 插件运行时与宿主扩展【实证】[仅 Windows]

Windows 新增了 `resources/plugins/openai-bundled/plugins/chrome/app-server-runtime/codex`、`node`、`node_repl` 等大型文件，其中 `codex` 为 247.7 MB，`node` 为 91.4 MB，`node_repl` 为 13.5 MB。同时 `chrome/extension-host/windows/x64/extension-host.exe` 从 0.6 MB 增至 0.8 MB，增加 +0.23 MB。

macOS 事实包没有对应的新增大文件列表。这里能确认的是 Windows 分发包中 Chrome（谷歌浏览器）插件相关运行时和扩展宿主内容发生了显著打包变化；是否代表功能入口开放，事实包不足以判断。

### LaTeX 组件路径重命名【实证】[仅 Windows]

Windows 包中新增 `resources/plugins/openai-bundled/plugins/latex/bin/tectonic.exe`，大小 49.6 MB；同时移除 `resources/plugins/openai-bundled/plugins/latex-tectonic/bin/tectonic.exe`，大小同为 49.6 MB。

这表明 Windows 侧 LaTeX（排版系统）相关插件路径从 `latex-tectonic` 迁到 `latex`。macOS 事实包没有出现对应变化。

### Computer Use 插件在 macOS 小版本更新【实证】[仅 macOS]

macOS 包中 `computer-use` 插件配置的版本号从 `1.0.791` 升至 `1.0.793`。同一批 macOS 体积变化最大的组件也集中在 Computer Use（计算机操作能力）相关二进制：`CUALockScreenGuardian` +0.17 MB，`SkyComputerUseService` +0.14 MB，`SkyComputerUseClient` +0.08 MB。

Windows 事实包没有给出 `computer-use` 插件版本号变化。因此只能确认 macOS 侧该插件小版本更新，不能推断 Windows 是否同步了同一内部版本。

### 前端状态与图标模块有小幅变化【信号】[仅 macOS]

macOS 去除 hash 噪音后，前端模块新增 `persisted-signal`、`pin`、`pin-filled`，移除 `settings-route-state`。模块名没有混淆，因此可以作为方向信号：可能涉及持久化状态、置顶/固定图标，以及设置路由状态相关整理。

Windows 事实包没有列出对应的去噪后前端模块增删清单。因此这条只标为 macOS 信号，不应视为跨平台功能事实。

### 核心可执行文件与运行组件体积更新【信号】[macOS+Windows]

macOS 侧 `node_repl` 从 15.0 MB 到 15.1 MB，增加 +0.01 MB；`better_sqlite3.node` 从 1.9 MB 到 2.0 MB，增加 +0.03 MB。Windows 侧 `Codex.exe` 从 222.7 MB 到 226.6 MB，增加 +3.92 MB；`resources/codex.exe` 从 245.8 MB 到 247.7 MB，增加 +1.90 MB；`resources/codex` 从 218.7 MB 到 219.5 MB，增加 +0.79 MB；`resources/codex-command-runner.exe` 从 0.8 MB 到 0.9 MB，增加 +0.10 MB；`resources/node_repl.exe` 从 13.5 MB 到 13.5 MB，增加 +0.04 MB。

这些都是二进制体积信号，只能说明打包内容或编译产物发生变化，不能单独推导具体功能。
