# Codex 桌面版 `26.217.1959` build 669

> 第三方变更日志 · 非官方 · 基于官方更新包逆向差异

本版从产物证据看，核心变化集中在会话迁移、worktree 分支管理、模型容量提示、权限与实验功能入口。  
English one-liner: this build points to richer local/worktree thread handoff, Git-state safeguards, and model-capacity UI signals.

## 关于这份报告

这不是官方变更日志，而是一份**基于官方分发产物的逆向差异报告**。本次对比范围为 macOS arm64 上的 `26.212.1823` build 661 到 `26.217.1959` build 669，两个版本发布日期分别为 2026-02-13 与 2026-02-18。

事实来源包括 Info.plist、package.json diff、bundle 文件树统计、主进程/前端 bundle 新增字符串、同名前端模块可读字符串差异，以及签名、公证、SHA-256 验证结果。由于这是历史回填，官方 appcast 未对这一对版本提供增量包，本报告改为基于两个官方全量包直接比较。

证据等级分两档：**【实证】**表示产物中存在人类可读的直接证据，例如配置 diff、可读字符串、签名或校验结果；**【信号】**表示只有前端模块名、同名 bundle 字符串差异或体积变化等间接线索，指向某方向，但无法从产物还原到源码级意图。

## ✨ 重点变化

### 1. 会话迁移到本地或 worktree 的能力明显扩展 【实证】

新版主进程 bundle 中新增了多条与 thread 迁移、worktree、分支和 stash 相关的可读错误路径与状态文案，指向“把当前会话移动到本地或新的 worktree”这一工作流正在被补强。可感知点可能包括迁移失败提示、默认分支限制、目标 worktree 打开、未提交改动处理等。

关键证据包括：`Failed to move thread to local`、`Failed to move thread to worktree`、`Move to worktree is not available on the default branch`、`Branch is already checked out in another worktree`、`You have uncommitted changes. Commit or stash them first.`、`Stashed Codex worktree changes`。

### 2. 前端出现 fork slash command 与本地/worktree 分叉入口 【信号】

同名前端模块和多语言资源新增了 `composer.forkSlashCommand.*`、`move-thread-to-local`、`move-thread-to-worktree`、`move-to-local`、`move-to-worktree` 等字符串和状态类名。这些线索指向 UI 层正在暴露“fork 当前对话到本地或新 worktree”的入口，但无法从产物还原到源码级意图。

关键证据包括：`composer.forkSlashCommand.title`、`composer.forkSlashCommand.option.local.title`、`composer.forkSlashCommand.option.worktree.title`、`Button label for moving a local conversation to a new worktree`。

### 3. 模型容量与限流提示进入界面文案 【信号】

多个语言 bundle 新增了 `codex.modelCapacityBanner.*`、`composer.mode.rateLimit.modelSectionLabel`、`composer.statusPlain.modelRateLimitSectionLabel` 与 `composer.modelChangeDuringConversationWarning.toast`。这些同名前端模块线索指向模型容量、限流和会话中切换模型的提示正在进入 UI，但无法从产物还原到源码级意图。

关键证据包括：`codex.modelCapacityBanner.title`、`codex.modelCapacityBanner.headline.noReset`、`codex.modelCapacityBanner.headline.withReset`、`composer.modelChangeDuringConversationWarning.toast`。

### 4. 权限、实验功能、计划下载和同步提示的设置面扩展 【信号】

多语言资源新增了权限下拉、实验功能、计划下载、同步提示和 follow-up toast 相关文案，指向设置与会话辅助操作在 UI 层继续扩展。由于这些来自同名前端模块改动线索，只能判断方向，不能断言完整功能行为。

关键证据包括：`settings.general.experimentalFeatures`、`settings.general.experimentalFeatures.empty`、`composer.permissionsDropdown.default.label`、`composer.permissionsDropdown.trigger.tooltip`、`localConversation.planSummary.download`、`localConversation.sync.buttonLabel.tooltip`、`pendingRequestStory.toastFollowupPrompt`。

### 5. 后台终端详情面板和上下文自动压缩有新的可访问性与状态提示 【信号】

主前端模块新增了后台终端完成、展开/折叠详情面板、自动压缩上下文等字符串。这些线索指向后台终端状态和上下文压缩状态在 UI 上更可见，但仍属于前端 bundle 信号，无法还原源码级意图。

关键证据包括：`Background terminal finished`、`Aria label for button that expands background terminals details panel.`、`Aria label for button that collapses background terminals details panel.`、`Automatically compacting context`。

### 6. Xcode 打开路径与通知处理出现新增运行路径 【实证】

主进程 bundle 新增了与 Xcode 打开路径和系统通知相关的可读字符串，说明新版产物中存在这些运行路径。可感知点可能是从 Codex 打开目标路径到 Xcode 的 fallback 行为，以及通知点击或 action 处理。

关键证据包括：`Opening path in Xcode via xed`、`Opening path in Xcode via LaunchServices fallback`、`show notification`、`notification click open`、`notification action`。

### 7. 开发脚本新增 ESLint 缓存模式 【实证】

`package.json` 新增了 `lint:cached` 脚本，使用 ESLint cache 并把缓存放到 `node_modules/.cache/eslintcache`。这属于开发流程变化，对最终用户不可见，但对参与构建或维护该应用的人可感知。

关键 diff：

```diff
+    "lint:cached": "eslint src --max-warnings 0 --cache --cache-strategy content --cache-location node_modules/.cache/eslintcache",
```

### 8. 版本号与构建号更新 【实证】

Info.plist 与 package.json 均显示版本从 `26.212.1823` 升至 `26.217.1959`，build 从 `661` 升至 `669`。`ElectronAsarIntegrity` 中的 `app.asar` hash 也随内容变化更新。

关键证据：`CFBundleShortVersionString: 26.212.1823 → 26.217.1959`，`CFBundleVersion: 661 → 669`。
