# Codex 桌面版 26.707.51957 macOS / Windows 对照

> 第三方跨平台变更日志 · 非官方

本批次的共同主线是首页工作模式、Quick Chat、浏览器数据管理和模型选择能力继续扩展；Windows 与 macOS 的界面构建节奏并不完全一致。

The shared direction is an expansion of work modes, Quick Chat, browser data controls, and model selection, with platform-specific UI and packaging changes.

## 关于这份报告

本报告比较两个官方全量分发包：macOS arm64 从 `26.707.31428` build 5059 到 `26.707.51957` build 5175；Windows x64 MSIX 从 `26.707.3748.0` 到 `26.707.6957.0`。两平台版本号和发布时间可能错位，因此以下结论均按各自事实包独立归属。

macOS 两个样本均通过 Developer ID 公证验证，签发主体为 OpenAI OpCo, LLC。旧包 SHA-256 为 `ffdc351a507105d55d7464e3340302c758b8a54b926711c4b15bf373ccb47d64`，新包为 `cb9655bb0adfc06287ef369b6d07ed324430d0668e1a9fa473c4e9d26f934f36`。Windows 两个样本的 SHA-256 分别为 `181faac13711ed38213c38644be2fc176f78f65a057cd167dc044935d3e81ab9` 和 `62892fd46d0035f6263a369b58c8c403523d5b15c0f4714712ab782e8aa15dd7`。

证据分为两档：

- 【实证】来自 plist、manifest、插件清单、依赖清单、人类可读配置，以及不属于同名前端模块线索的包内字符串。字符串进入包内不代表功能已经向所有用户开放。
- 【信号】来自前端模块名、同名压缩模块中的字符串和样式集合、模块体积及构建方向，不能还原为源码级实现事实。

平台标记为 `[macOS+Windows]`、`[仅 macOS]`、`[仅 Windows]`。

## ✨ 重点变化

### 工作模式与首页定位重写 【信号】[macOS+Windows]

两端的前端构建都出现 `ChatGPT Work`、`Build, debug, and ship`、`Create, learn, and explore` 以及按“有无项目”区分的首页标题。旧有的 `For developers`、`For getting work done` 和统一的 `What should we get done?` 文案被移除或替换。

Windows 的证据集中在 `app-main` 同名模块；macOS 的对应线索分散在主页面模块和多语言资源中。它们共同指向 Codex 与 ChatGPT Work 的定位、侧栏模式选择器和项目上下文首页正在重排，但不能据此确认所有账户都已看到新版入口。

### Quick Chat 更明确地衔接当前任务 【信号】[macOS+Windows]

多个两端共有的语言模块加入 `quickChat.addToTask`、`quickChat.addToTaskTooltip`，并移除 `quickChat.addToComposer`。其他语言文案也出现“将此聊天加入当前任务”。

这表明 Quick Chat 的后续动作正在从简单回填 composer 转向加入任务。macOS 还出现打开 Quick Chat 历史、返回当前聊天、开始新会话，以及调整聊天宽度和高度的无障碍文案；这些额外交互线索未在 Windows 样本中出现。

### Pull Request 合并拦截状态更细 【信号】[macOS+Windows]

两端多个语言模块同步加入 Pull Request 无法合并时的状态键，包括：

- `blocked`
- `draft`
- `failingChecks`
- `pending`
- `pendingChecks`
- `unknown`

同时出现“先修复失败检查再合并”“GitHub 仍在检查是否可以合并”“另一项 Pull Request 更新正在进行”等文案。证据支持合并按钮禁用原因和状态反馈正在细化，但不支持推断实际合并逻辑如何实现。

### 内嵌浏览器的数据、下载与站点权限设置扩展 【信号】[macOS+Windows]

两端 `browser-use-settings` 模块都从接近空壳增长到约 0.1 MB，并加入以下设置与导航文案：

- 无需询问即可访问历史记录
- 浏览器下载与下载历史
- 浏览历史
- 联系信息
- 扩展
- 密码管理器
- 站点设置
- 展开或隐藏单项数据清理选项

Windows 还出现 `Add site permission` 和“插件不可用时的禁用开关”无障碍标签。macOS 则额外出现设置页前进、后退按钮标签。由于这些都来自同名前端模块差异，只能视为设置界面和权限模型的方向信号。

### Ultra 模型选择入口与 DeviceCheck 错误路径进入两端包体 【实证】[macOS+Windows]

两端主进程或前端 bundle 都新增了：

- `Whether Ultra appears in the model picker slider`
- `DeviceCheck registration returned no cookie`

第一条确认 Ultra 是否出现在模型选择滑杆中的控制文案已经进入两端包体；第二条确认 DeviceCheck 注册未返回 cookie 的错误路径已经存在。两条证据均不代表 Ultra 已对所有用户开放，也不能从字符串推导具体注册恢复逻辑。

### Windows 首页出现多代理并行委派横幅 【信号】[仅 Windows]

Windows 的 `home-announcements` 模块新增多代理 composer 横幅及其状态键，包含：

- `Delegate work to subagents that work in parallel. Note: may increase token usage.`
- `has-seen-multi-agent-composer-banner`
- 多代理横幅关闭标签
- Fast mode、额度重置、赠送额度和后端驱动首页横幅的关闭标签

这说明 Windows 首页横幅系统已包含并行委派和多类账户提示的前端资源。macOS 事实包出现新的 `home-announcements` 模块组合，但没有同等明确的多代理横幅字符串，因此本条只归属于 Windows。

### macOS 的计划摘要与文件 diff 交互继续细化 【信号】[仅 macOS]

macOS 的 onboarding 同名模块新增了计划侧栏和 diff 相关文案：

- 关闭计划侧栏
- 折叠或展开计划摘要
- 在新窗口打开计划摘要
- 切换文件 diff 区域
- 在编辑器中打开大 diff 文件
- 从 diff 文件头打开当前文件
- `Binary file not shown`

配套类名还出现 `hunk-actions`、diff header 容器、hover/focus 显示状态。这些线索共同指向计划摘要、文件级 diff 和 hunk 操作的布局及可访问性调整。

### macOS 新增项目选择、文件输入与可调整聊天窗口线索 【信号】[仅 macOS]

macOS 主页面模块新增：

- ChatGPT composer 文件输入标签
- 首页有项目和无项目状态下的项目选择器标签
- `Add local project`
- Quick Chat 历史入口
- 调整聊天宽度与高度
- ChatGPT 项目选择和项目创建对话框标签

同时新增的模块组合覆盖 `select-workspace-page`、`projects-index-page`、本地和远程会话页面。这是一组相互印证的项目上下文与窗口交互信号，但事实包未给出对应源码或设置 manifest。

### macOS 的自动化调度与远程项目状态面扩大 【信号】[仅 macOS]

macOS 的项目、本地环境同名模块从接近 0 MB 增至 0.4 MB，并出现两个明确错误文案：

- 自动化在应用中不存在，可能已被用户手动删除，无法更新
- 自动化计划没有未来运行时间，需要选择当前时间之后仍会发生的计划

同一模块还出现 `active-remote-project-id`、`active-workspace-roots`、`host-id-remote-control-allowed`、`hook/started`、`hook/completed`、`hotkey-window-projectless-default-enabled` 和 `mac-menu-bar-enabled`。这些是自动化校验、远程控制、workspace roots、hook 状态和菜单栏设置扩展的组合信号。

### macOS 前端模块进行大范围重新分块 【信号】[仅 macOS]

去除 Vite hash 改名后，macOS 新模块组合覆盖：

- `codex-micro-settings~codex-micro-bridge`
- `cloud-environments-settings`
- `remote-connections-settings`
- 本地与远程 conversation 页面
- `automations-page`
- skills、plugins 与 settings
- AppGen library、settings 与 publication
- Pull Request route 与 code review
- notebook preview 和 artifact tab
- `home-ambient-suggestions-content`

同时移除了独立 `codex-micro-settings`、`codex-micro-joystick-hud`、`debug-settings` 及多组旧的 onboarding、Quick Chat、AppGen 和 Pull Request 分块。这更像是前端依赖图和加载边界的重组，其中部分模块名具有明确产品方向，但不能当作新增源码能力或用户可见功能。

### macOS 内置 Computer Use 与 Chrome 插件版本前移 【实证】[仅 macOS]

`computer-use` 插件 manifest 从 `1.0.1000366` 更新到 `1.0.1000387`；`chrome` 插件 manifest 从 `26.707.31428` 更新到 `26.707.51957`。

Computer Use 的描述仍明确面向通过 ChatGPT 控制 macOS 桌面应用，并优先使用专用 connector、API 或 CLI。Chrome 插件描述仍面向依赖现有标签页、登录会话、cookie、扩展和 ChatGPT Chrome Extension 设置的自动化。

事实包没有给出插件清单增减、技能说明或服务入口变化，因此这里只能确认两个既有插件的 manifest 版本更新，不能宣称新增插件或新增完整能力面。

### Computer Use 运行时总量和依赖数量保持稳定 【实证】[macOS+Windows]

macOS `cua_node` 总量保持 `365.7 MB`，Windows 保持 `287.2 MB`，两端变化均为 `+0 MB`、`+0%`；两端新增依赖包数量均为 0。

macOS 的两套 Computer Use 服务和客户端二进制各有约 `+0.12 MB` 至 `+0.13 MB` 的变化，但总运行时口径不变。Windows 事实包没有列出对应 Computer Use 二进制级变化。

### Chromium 基座从 150.0.7871.101 前移到 150.0.7871.115 【实证】[macOS+Windows]

macOS Info.plist 将 `ChromiumBaseVersion` 从 `150.0.7871.101` 更新到 `150.0.7871.115`，并整体替换同版本号的 Codex Framework 目录。

Windows 新增 `150.0.7871.115.manifest`、移除 `150.0.7871.101.manifest`。两端均显示相同的 Chromium patch 基座迁移，但平台二进制体积走势不同：macOS 新 Framework 主二进制为 254.3 MB，替换的旧文件为 255.4 MB；Windows `chrome.dll` 从 312.4 MB 降至 311.9 MB。

### Windows 随包 DevTools 更新到版本 150 【实证】[仅 Windows]

Windows 的 DevTools 发布说明从 149 更新到 150，包内说明列出：

- DevTools for agents 1.4 及以后版本的实验性内存调试
- 结构化内容输出的实验性 TOON 支持
- 改进的 skill discovery
- AI assistance walkthrough 的性能网络轨、时间线事件、网络请求、语法高亮源码和文件列表组件
- 在 Styles 面板直接编辑 `@container` 和 `@function`
- Chrome 148–150 highlights 视频入口

这是随 Chromium 打包的 DevTools 内容变化，不应等同于 Codex 自身新增同名能力。

### 原生退出菜单的多语言措辞微调 【实证】[macOS+Windows]

两端的泰语、古吉拉特语、哈萨克语和立陶宛语原生菜单文件发生相同方向的文本调整，主要涉及退出确认标题、退出按钮和“此设备”措辞。它们属于本地化修订，不改变退出时中断本地任务及停止计划任务的既有语义。
