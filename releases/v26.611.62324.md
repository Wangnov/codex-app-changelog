# Codex 桌面版 26.611.62324 macOS / Windows 对照

> 第三方跨平台变更日志 · 非官方

这一批次更像是同一代码线在不同平台上的打包对齐：共同变化集中在内置插件版本和 app-server 临时状态处理，前端模块信号在 macOS 与 Windows 上方向并不完全一致。  
One-liner: shared app-server reconciliation strings and bundled plugin version bumps, with macOS showing added UI surface signals while Windows shows several counterpart modules shrinking or removing strings.

## 关于这份报告

本报告只基于两份官方签名分发包逆向重建后的机器差异事实：macOS arm64 为 `26.611.61753` build `4008` → `26.611.62324` build `4028`，Windows x64 为 `26.611.8273.0` → `26.611.8604.0`。两端版本号与发布时间可能错位，因此下文分别引用各自事实，不把一个平台的结论套到另一个平台。

证据等级分两档：【实证】来自 plist、manifest、插件配置、依赖清单、可读字符串、签名验证、package 或明确文件 diff；【信号】来自同名前端模块名、压缩 bundle 字符串差、样式类名与体积方向，能说明界面和交互方向，但不等价于完整源码事实。

平台归属使用 `[macOS+Windows]`、`[仅 macOS]`、`[仅 Windows]`。webview/assets 和 `.vite/build` 的 hash 改名按输入说明视为构建噪音，未作为功能增减。

## ✨ 重点变化

### 【实证】[macOS+Windows] app-server 临时状态协调进入两端包内

两端新版主进程/前端 bundle 都新增了可读字符串：

- `Failed to reconcile temporary app-server state`
- `Reconciled temporary app-server state`

这说明“临时 app-server 状态协调”的成功与失败路径已经进入 macOS 与 Windows 包内。证据只能确认字符串存在于分发产物中，不能进一步断言触发条件、默认启用范围或所有用户可见性。

### 【实证】[macOS+Windows] Browser 与 Chrome 内置插件版本同步更新

macOS 与 Windows 的 `browser`、`chrome` 内置插件 manifest 都显示版本从 `26.611.61753` 更新到 `26.611.62324`。

Browser 插件说明仍明确指向内置浏览器、本地目标打开、截图、点击、输入，以及在显式 `@browser` / `@browser-use` 请求时优先使用 Codex 内置浏览器。Chrome 插件说明仍定位于依赖用户现有 Chrome 状态的自动化，包括标签页、登录态、cookie、扩展与 Codex Chrome Extension 设置。

平台差异是：macOS 应用自身版本也升到 `26.611.62324`，Windows 应用版本则是 `26.611.8273.0` → `26.611.8604.0`，但其内置插件 manifest 版本仍对齐到 `26.611.62324`。

### 【实证】[仅 Windows] Computer Use 插件版本更新，运行时本体未见增量

Windows 的 `computer-use` 插件 manifest 显示版本从 `26.611.61753` 更新到 `26.611.62324`，描述为通过 Computer Use 从 Codex 控制 Windows 桌面应用。

但 Windows 的 Computer Use 运行时总量为 `285.6 MB → 285.6 MB`，`cua_node` 新增依赖为 `0` 个。因此本样本支持“插件 manifest 版本更新”，不支持“运行时依赖或运行时体积新增”。

### 【实证】[仅 Windows] owl shell 安装时间戳更新，运行时标识保持同一缓存键

Windows `.installed-owl-shell.json` 中 `installedAt` 从 `2026-06-17T02:33:52.524Z` 变为 `2026-06-17T14:21:58.283Z`。同一 diff 片段显示 `cacheKey` 仍指向 `owl-10cade0011e3f9513741d5e3100e27194ebb4b41-codex-win32-x64.zip`，`archiveSha` 为 `09944e70901184dbd560e6349ba46b80b426b60cf6a11ebd8122aab9d0b46737`。

这更像 Windows 打包/安装元数据更新，而不是事实包能证明的 owl 运行时内容升级。

### 【信号】[仅 macOS] Markdown 渲染与引用交互增强

macOS 的 `markdown` 模块体积增加 `11343 bytes`，新增多组无 alt 文本时的图片、视频、占位、失败态 accessible label，以及文件引用 chip 的 accessible label。还出现 `Add to chat`、选中文本 overlay 加入聊天的按钮说明、无扩展名代码/文档引用 fallback 标签。

对应样式类名包括 `data-markdown-copy`、`data-markdown-image-preview-trigger`、`data-selected-text-overlay-target`、`cursor-zoom-in`、`data-hex-color-swatch`。这些共同指向 Markdown 内容里的媒体预览、文件引用可访问性、复制/选区加入聊天、色块或代码呈现相关交互增强。它们是前端同名模块改动线索，因此仍标为【信号】。

### 【信号】[macOS+Windows] 内置浏览器设置继续围绕权限、历史、下载域与数据清理调整，但两端细节不同

两端 `browser-use-settings` 都新增或保留了同主题信号：`Access history without asking`、嵌入式浏览器设置面包屑、站点设置、扩展设置、密码管理器、联系方式设置、数据清理选项展开/收起、不可用插件禁用 toggle 的 accessible label。

差异在于 Windows 新增字符串里还出现 `Add allowed domain` 和 `Add allowed download domain`；macOS 则额外出现 embedded browser settings 的 back/forward button accessible label。两端都显示浏览器设置页在权限、历史访问、站点/扩展/密码、数据清理与下载域控制方面有界面层调整信号。

### 【信号】[仅 macOS] 侧栏、浏览器标签与 CI 诊断提示进入前端模块

macOS `thread-side-panel-tabs` 从约 `0.0 MB` 增至 `0.3 MB`，新增字符串包括 CI 处理步骤：“先使用”“检查技能或工具能否访问 CI 系统”“检查凭据、权限、token、MCP 或 tool access”“抓取外部日志、诊断问题、做最小安全修复”。同一模块还出现 `A pull request already exists for this branch`、`Accept edits to stage them`，以及浏览器标签播放音频、使用摄像头或麦克风、设备工具栏宽高输入的 accessible label。

这组信号把 side panel、CI 故障处理、PR 状态、暂存编辑、浏览器设备工具栏与媒体权限状态放在同一个模块变化面上。Windows 对应模块在本样本中反向减少 `308471 bytes`，并移除了同组 CI 与 PR/暂存字符串，因此不能说这是 Windows 本批次新增能力。

### 【信号】[仅 macOS] Debug modal 暴露更多会话、账户、浏览器与 host 状态字段

macOS `debug-modal` 增加 `180474 bytes`，新增字符串包括 `(command unavailable)`、`Account ID`、`Active browser tab`、`Active conversation`、`All types`、`Allow automatic worktree cleanup in this unpackaged build.`、`Allow host`、`Allow host starting`、`Already current`、`Ambient suggestion chats`、`App Server`、`App session ID`。

这显示调试界面在账户、会话、浏览器、app server、host 权限或 unpackaged build 清理选项上有新增展示/控制信号。Windows 对应模块本批次减少 `178364 bytes` 并移除了其中一部分字符串，因此只归为 macOS 新增信号。

### 【信号】[仅 macOS] 插件详情页出现分享、workspace 可见性与 MCP server 开关信号

macOS `plugin-detail-page` 增加 `123194 bytes`，新增字符串包括插件分享对话框中搜索 workspace 用户和组、移除已选分享目标、`Add people or groups`、`Anyone at workspace with the link`、`Anyone in this workspace with the link`、`App unavailable`，以及 `MCP server enablement toggle`、section navigation、skill preview modal actions、skill card actions。

这些信号说明插件详情页正在扩展分享、workspace 链接可见性、MCP server 启用开关、技能预览与操作菜单。事实包没有 Windows 对应新增模块证据，因此归为仅 macOS。

### 【信号】[仅 Windows] 多个前端模块在本样本中移除高可见文案

Windows 的多个同名前端模块显示体积减少并移除可读字符串：

- `general-settings` 减少 `105109 bytes`，移除默认权限 toggle、composer context usage indicator、work mode settings、UI/code 字号调整、hold-to-dictate hotkey capture 等设置文案。
- `home-announcements` 减少 `29349 bytes`，移除 Codex rate limit reset、Fast mode、backend-driven banner、multi-agent composer banner、remote connections banner、`Add Connections` 等首页横幅文案。
- `appgen-share-dialog` 减少 `19081 bytes`，移除 live site 分享、复制 URL、外部打开、workspace 用户/组搜索、互联网可见性等分享文案。
- `thread-side-panel-tabs` 与 `debug-modal` 也分别移除了 CI/PR/暂存与调试状态相关字符串。

这不是“Windows 不支持这些能力”的结论，只能说明本输入样本里 Windows 新版相对于旧版移除了这些模块内可读字符串或压缩 bundle 片段。

## 🔎 低层信号索引

### 可访问性与状态标签

【信号】[macOS+Windows] 两端都出现大量 `aria-label`、`aria-expanded`、`aria-hidden`、`aria-controls`、`aria-labelledby` 等类名或属性线索。它们支持“相关界面区域在可访问性与状态表达上有调整”的判断，但不能单独还原具体组件行为。

### 浏览器权限与设备状态

【信号】[仅 macOS] `thread-side-panel-tabs` 新增浏览器标签播放音频、使用摄像头或麦克风、设备工具栏宽高输入的 accessible label。结合 `browser-use-settings` 的站点/扩展/密码/历史/数据清理信号，可读为浏览器内嵌体验继续补齐状态与设置面。

### 文件引用、媒体 fallback 与选区操作

【信号】[仅 macOS] `markdown` 新增图片/视频无 alt 文本 fallback、加载占位、失败态、文件引用 chip、无扩展名文件类型 fallback，以及 `Add to chat`。这类信号边界是 Markdown 前端呈现和交互，不证明后端文件处理协议变化。

### 分享与 workspace 可见性

【信号】[macOS+Windows] macOS 在 `plugin-detail-page` 新增插件分享与 workspace 链接文案；Windows 的 `appgen-share-dialog` 则移除站点分享、互联网可见性、live site URL 操作等文案。两者都触及分享和 workspace 可见性，但对象不同、方向相反，不能合并成同一功能结论。

### 设置项瘦身或迁移线索

【信号】[仅 Windows] `general-settings`、`home-announcements`、`debug-modal`、`thread-side-panel-tabs` 多个模块在 Windows 样本里缩小并移除字符串。证据边界是 bundle 差异，不足以判断这些设置消失、迁移到其他模块，或仅是打包路径变化。
