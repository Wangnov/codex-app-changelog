# Codex 桌面版 `26.415.30602` build 1773

> 第三方变更日志 · 非官方 · 基于官方更新包逆向差异

这一版从 `26.415.21839` build 1763 推进到 `26.415.30602` build 1773，产物层面没有新增或移除 bundle 文件，也没有新增 cua_node 依赖。  
English one-liner: a small build bump with unchanged bundle file counts and only minimal app.asar-level signals.

## 关于这份报告

这不是官方变更日志，而是一份**基于官方分发产物的逆向差异报告**。本次为历史回填：官方 appcast 未对这一对版本提供增量包，因此比较方式改为下载两个官方全量包后直接做结构化 bundle 差异。

对比范围为 macOS arm64 的 `26.415.21839` build 1763 与 `26.415.30602` build 1773，二者发布日期均为 `2026-04-17`。

本文使用两档证据等级：

- **【实证】**：来自 Info.plist、签名/公证验证、文件级统计、依赖清单、hash 等可直接读取的结构化事实。
- **【信号】**：只来自前端同名模块改动线索或体积变化，指向某方向，但无法从产物还原到源码级意图。

这是一次明确的应用版本推进：`CFBundleShortVersionString` 从 `26.415.21839` 变为 `26.415.30602`，`CFBundleVersion` 从 `1763` 变为 `1773`。

对用户和开发者而言，这是可用于定位问题、复现实验和核对安装产物的最直接版本边界。

```diff

```

### 4. 没有新增 cua_node 依赖 【实证】

事实包显示 cua_node 依赖新增数量为 `0`。因此不能把本版解读为通过新增 native 或 Node 依赖来扩展能力面。

对开发者而言，这是一条重要边界：本次差异没有证据表明新增了 PDF、OCR、图像处理、浏览器自动化等依赖级能力。
