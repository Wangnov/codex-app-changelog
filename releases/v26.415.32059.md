## ✨ 重点变化

### 1. 设置页出现更丰富的 Agent、浏览器与跨客户端控制项 【信号】

`general-settings` 前端模块体积从约 `0.0 MB` 增至 `0.1 MB`,增加 `93277 bytes`,并出现多组新的设置文案。它们指向设置页新增或调整了 Agent 环境、内置浏览器控制、跨已登录客户端连接、实验功能切换等入口,但无法从产物还原到源码级意图。

可感知点是:用户可能会在设置中看到更多与运行环境、浏览器任务、远程连接或 beta 功能相关的控制项。

关键线索包括:

```text
Agent environment
Allow Codex to control the in-app browser for browser tasks. Restart Codex after changing this setting.
Allows other signed-in Codex clients to connect to this computer
Aria label for toggling a beta experimental feature
Aria label for toggling the Telepathy experimental feature
```

### 2. 设置页补充了 UI 字号、代码字号与计划用量提示 【信号】

同一 `general-settings` 模块新增了与界面字号、代码字号、速度/计划用量相关的文案。该信号指向显示密度或可读性相关设置有所扩展,同时有更明确的计划消耗提示,但无法从产物还原到源码级意图。

可感知点是:开发者可能会看到更细的 UI 和代码显示调节项,以及类似 `1.5x speed, 2x plan usage` 的用量提示。

关键线索包括:

```text
Adjust the base size used for code across chats and diffs
Adjust the base size used for the Codex UI
1.5x speed, 2x plan usage
```

### 3. 设置页可访问性与主题控制线索增强 【信号】

`general-settings` 中新增了多组 `Aria label` 文案和 ARIA 属性类名,还出现了与对比度滑块、macOS 菜单栏开关、半透明侧边栏开关相关的可读字符串。它们指向设置界面的可访问性标注和主题控制更细,但仍属于前端模块信号。

可感知点是:辅助技术、键盘操作或视觉主题相关体验可能有所调整。

关键线索包括:

```text
Aria label for hotkey window hotkey capture input
Aria label for the contrast slider in chrome theme settings
Aria label for the macOS menu bar setting toggle
Aria label for the translucent sidebar toggle in chrome theme settings
aria-label
aria-valuenow
aria-valuetext
```

### 4. 项目外线程的起始目录描述发生变化 【信号】

`app-server-manager-signals` 模块中的文案从 `This projectless thread starts in the user` 变为 `This projectless thread starts in a generated directory under the user`。这指向项目外线程的工作目录描述更精确,但无法确认底层目录策略是否实际改变。

可感知点是:当用户启动不绑定项目的线程时,界面说明可能会更明确地提示其起始位置位于用户目录下的生成目录中。

关键线索:

```diff
- This projectless thread starts in the user
+ This projectless thread starts in a generated directory under the user
```
