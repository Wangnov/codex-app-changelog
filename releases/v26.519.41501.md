# Codex 桌面版 26.519.41501 macOS / Windows 对照

> 第三方跨平台变更日志 · 非官方

这次对照显示，macOS 26.519.41501 与 Windows 26.519.5221.0 来自相近代码批次，但两平台打包节奏和前端模块状态并不完全一致。

One-liner: this batch adds shared UI signals around context usage, Markdown media/citations, and Computer Use approval persistence, while Windows carries additional side-panel, CI/PR, browser-device, debug, and settings churn.

## 关于这份报告

本报告只使用输入事实包中的机器提取结果。macOS 样本对比 26.519.31651 build 3017 → 26.519.41501 build 3044；Windows 样本对比 26.519.3891.0 → 26.519.5221.0。两平台版本号与构建号分别引用，不互相推导。

证据分两档：  
【实证】来自 Info.plist、插件 manifest、签名/公证结果、SHA-256、文件级统计、package 或依赖清单、服务入口、技能文档、配置差异等。  
【信号】来自同名前端模块的可读字符串、样式类名、模块名和体积方向，说明相关 UI、状态或交互进入 bundle 差异，但不等同于源码事实，也不保证所有用户可见。

平台归属按 `[macOS+Windows]`、`[仅 macOS]`、`[仅 Windows]` 标注。输入中没有提供内容安全策略变化、Computer Use 运行时总量、AppxManifest 变化、新增内置插件明细或新增文件体积明细；因此这些项不扩展推断。

## ✨ 重点变化

### 【实证】[macOS+Windows] 浏览器相关内置插件版本同步到 26.519.41501

两平台的 `browser` 与 `chrome` 内置插件 manifest 都只显示版本字段更新：`26.519.31651` → `26.519.41501`。

macOS 路径为 `Contents/Resources/plugins/openai-bundled/plugins/.../.codex-plugin/plugin.json`；Windows 路径为 `resources/plugins/openai-bundled/plugins/.../.codex-plugin/plugin.json`。插件描述文本未显示功能性改写。

平台差异：macOS 应用自身也升到 `26.519.41501` build `3044`；Windows 包版本是 `26.519.5221.0`，但这两个插件 manifest 内的版本仍同步为 `26.519.41501`。

### 【信号】[macOS+Windows] 新增 `computer-use-plugin-icon` 前端模块

两平台在去掉 Vite hash 噪音后，真实新增前端模块都只有 `computer-use-plugin-icon`。

这说明 Computer Use 插件视觉入口或插件标识相关代码进入两平台前端 bundle。事实包没有给出对应 manifest、技能文档或服务入口新增，因此这里不能写成完整新增能力，只能作为前端信号。

平台差异：两平台模块名一致，macOS app.asar 内部文件为 `1576 → 1577`，Windows 为 `1530 → 1531`。

### 【信号】[macOS+Windows] Computer Use 应用控制审批加入“始终允许”持久选项

`pending-request-item-panel` 在两平台都新增字符串：`Checkbox label for always allowing Computer Use to control an app` 与 `composer.computerUseAppApproval.persist.always`。

这指向 Computer Use 控制某个应用时，审批面板可能出现“始终允许”类持久授权选项。由于证据来自同名前端模块改动线索，应视为 UI/状态信号，而非确认所有用户已经可见。

平台差异：两平台该模块体积均增加 `+3548 bytes`，方向一致。

### 【信号】[macOS+Windows] 线程状态区新增上下文用量与速度信息

`local-conversation-thread` 在两平台都新增上下文用量、token 总量、百分比、不可用状态、tokens per second、compact 操作入口与任务进行中禁用提示相关字符串。

可读字符串包括 `Context usage unavailable`、`Tokens per second in the thread summary status section`、`Menu item that compacts the current thread context from the status section`、`Compact is disabled while a task is in progress`、`codex.localConversation.status.compactOption`。

平台差异：两平台模块体积均增加 `+3928 bytes`，样式类名也同样新增 `hover:before:bg-token-list-hover-background`、`mx-4`、`w-45`。

### 【信号】[macOS+Windows] Composer 的状态入口从复制 session id 转向上下文用量切换

`composer` 在两平台都新增 `Toggle context usage`，同时移除 `Copy session id`、`Copied session id`、`Copy ID`、`Copied ID` 及上下文窗口元数据描述字符串。

这表明 composer 附近的状态 UI 可能从 session id 复制入口，调整为上下文用量展示/切换入口。由于证据来自同名前端模块字符串变化，只能标为信号。

平台差异：两平台模块体积均减少 `-3522 bytes`，移除的布局类名集合一致。

### 【信号】[macOS+Windows] Markdown 渲染增强媒体、引用 chip 与选中文本加入聊天的可访问性线索

`markdown` 模块在两平台都显著增大 `+60010 bytes`，新增字符串覆盖图片、视频、文件引用 chip、无扩展名文件引用、行号信息、图片加载占位、失败 fallback，以及 `Add to chat`。

这组信号指向 Markdown 内容渲染对媒体、文件引用和选中文本浮层的可访问性增强。`data-markdown-image-grid`、`data-selected-text-overlay-target`、`cursor-zoom-in` 等类名也支持图片网格、选中文本浮层或缩放交互方向。

平台差异：两平台新增字符串与样式类名一致。

### 【信号】[macOS+Windows] 首页公告与连接类 banner 有移除或收敛迹象

`home-announcements` 在两平台都移除 Fast mode、backend-driven banner、multi-agent composer banner、remote connections banner、`Add Connections` 以及 connected apps home banner 相关字符串。

这说明首页公告区域的若干 banner 文案在本批次 bundle 中被移除或迁移。证据不足以说明功能本身被删除，只能说明这些前端文案不再出现在该同名模块差异中。

平台差异：两平台该模块体积均减少 `-29765 bytes`。

### 【信号】[仅 macOS] worktree 初始化 v2 页面移除失败、取消、重试和环境编辑文案

macOS 的 `worktree-init-v2-page` 移除 `Creating a worktree and running setup.`、`Creating a worktree to fork this conversation.`、`Edit environment`、`Retry button`、`Cancel button for worktree creation`、`Error creating local task from worktree` 等字符串。

这表明 macOS 样本中 worktree 初始化页的流程文案或错误处理 UI 有调整。Windows 事实包没有列出该模块变化，不能推断 Windows 同步发生。

### 【信号】[仅 macOS] browser-use 设置页移除域名允许/阻止与历史访问相关文案

macOS 的 `browser-use-settings` 移除 `Access history without asking`、`Add allowed domain`、`Add allowed download domain`、`Add allowed upload domain`、`Add blocked domain` 等字符串。

这指向 browser-use 设置中的历史访问、上传/下载域名允许列表、阻止域名配置 UI 出现收敛或迁移。Windows 事实包没有列出该模块变化。

### 【信号】[仅 Windows] 线程侧栏大幅加入 CI/PR、编辑接受、浏览器设备工具栏与事件状态线索

Windows 的 `thread-side-panel-tabs` 从约 `0.0 MB` 增至 `0.5 MB`，新增大量字符串：CI 日志诊断流程、权限/凭据/MCP 或工具访问检查、`A pull request already exists for this branch.`、`Accept edits to stage them`、`Accepted event`、`Accepting event`，以及浏览器设备工具栏宽高输入的可访问标签。

这是一组高可见信号，覆盖 CI 修复流程、PR 状态、编辑 staging、浏览器设备尺寸控制和事件接受状态。macOS 同名模块在本输入中反向显示大量移除，不能把 Windows 方向套到 macOS。

### 【信号】[仅 Windows] debug modal 大幅瘦身，移除 app/session、ambient suggestion 与 unpackaged build 文案

Windows 的 `debug-modal` 从约 `0.2 MB` 降至 `0.0 MB`，移除 `(command unavailable)`、`Account ID`、`Allow automatic worktree cleanup in this unpackaged build.`、`Already current`、`Ambient suggestion chats`、`App Actions`、`App Server`、`App session ID`。

这说明 Windows 调试弹窗中的账号、会话、应用服务、ambient suggestion、未打包构建清理等文案在该模块中消失或迁移。macOS 事实包没有列出该模块变化。

### 【信号】[仅 Windows] general settings 移除字体尺寸、工作模式、默认权限和听写热键相关文案

Windows 的 `general-settings` 从约 `0.1 MB` 降至 `0.0 MB`，移除 `Adjust the base size used for code across chats and diffs`、`Adjust the base size used for the Codex UI`、`Accessible label for work mode settings options`、`Accessible label for disabled default permissions toggle`、`Agent environment`、`Aria label for hold-to-dictate hotkey capture input`。

这表明 Windows 设置页在字体尺寸、工作模式、默认权限、代理环境与听写热键捕获方面出现模块级重排或迁移信号。不能据此断言设置项被正式移除。
