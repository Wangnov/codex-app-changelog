# Codex 桌面版 26.519.41501(macOS / Windows 对照)

> 第三方跨平台变更日志 · 非官方

这批 Codex(OpenAI 的代码智能体产品)桌面版更新更像一次跨平台维护批次: 两端都有应用层包变化,但可读证据在 macOS(苹果桌面操作系统)侧更明确,Windows(微软桌面操作系统)侧更多体现为二进制体积与打包结构信号。

One-liner: A cross-platform maintenance batch with shared app-layer churn, confirmed bundled plugin version bumps on macOS, and stronger runtime/package-size movement on Windows.

## 关于这份报告

本报告基于两份官方签名分发包的逆向 diff(差异比对): macOS 侧取 Sparkle(macOS 应用更新框架)相关官方全量包重建比较,Windows 侧取 MSIX(Windows 应用安装包格式)对应全量包重建比较。macOS 对比范围是 26.519.31651 build 3017 → 26.519.41501 build 3044; Windows 对比范围是 26.519.3891.0 → 26.519.11010.0。本批次发布日期按输入事实包记为 2026-05-27。

证据分两档:【实证】表示来自 plist(属性列表)、AppxManifest(Windows 应用清单)、技能文档、类型声明、package.json(包配置文件)、依赖清单或其他可读配置/文档;【信号】表示仅来自前端模块名、文件树或体积变化,能提示方向,但不能还原源码意图。

平台归属标记含义: `[macOS+Windows]` 表示两端事实包都显示相关变化; `[仅 macOS]` 表示目前只在 macOS 事实包看到; `[仅 Windows]` 表示目前只在 Windows 事实包看到。本文刻意忽略 webview/assets 与 .vite/build 的 hash(哈希)重命名噪音。

## ✨ 重点变化

### 应用层包都有变化,但没有足够证据指向明确新功能【信号】[macOS+Windows]

两端的 app.asar(Electron 应用源码归档)都发生变化。macOS 内部文件从 1576 到 1577,新增 252、移除 251、改动 9;Windows 内部文件从 1530 到 1531,新增 521、移除 520、改动 12。

平台差异在于,macOS 事实包额外列出了一个去噪后的新增前端模块 `computer-use-plugin-icon`;Windows 事实包没有给出去噪后的新增模块名。因此,可以说两端都有应用层重打包/更新信号,但只有 macOS 侧出现“computer use 插件图标”方向的模块名信号。

### Browser 与 Chrome 内置插件版本在 macOS 侧同步到新版【实证】[仅 macOS]

macOS 可读 diff 显示,openai-bundled(内置 OpenAI 插件集合)里的 `browser` 与 `chrome` 两个插件,其 `plugin.json` 版本号均从 `26.519.31651` 更新为 `26.519.41501`。

Windows 事实包列出了 Browser / Chrome 相关 `browser-client.mjs` 文件体积微变,但没有提供对应 `plugin.json` 可读 diff。因此这里不能把“插件版本号同步”外推到 Windows,只能标为 macOS 实证。

### macOS 新增 `computer-use-plugin-icon` 前端模块信号【信号】[仅 macOS]

macOS 去除 Vite(前端构建工具)hash 噪音后,新增模块名为 `computer-use-plugin-icon`。这说明前端包里出现了与 computer use(计算机使用能力)插件图标相关的模块名。

但模块出现不等于功能对用户开放,也不能证明交互入口、权限模型或后端能力有变化。Windows 事实包没有给出同名模块信号。

### Windows 运行组件体积变化更明显【信号】[仅 Windows]

Windows 文件级统计显示多个运行组件体积变化较大: `resources/codex.exe` 从 247.7 MB 降到 239.6 MB,减少 8.11 MB;`resources/plugins/openai-bundled/plugins/chrome/app-server-runtime/codex` 同样减少 8.11 MB;`resources/codex-windows-sandbox-setup.exe` 从 8.3 MB 降到 6.1 MB,减少 2.21 MB;同时 `resources/codex` 从 219.5 MB 增至 221.7 MB,增加 2.17 MB。

这些是强烈的打包/运行时变化信号,但事实包没有可读源码或配置 diff 解释具体原因,所以不能进一步判断为性能、功能或依赖裁剪。

### Windows 移除了 `bwrap` 资源文件【信号】[仅 Windows]

Windows 文件树从 832 到 831,新增 0、移除 1、内容改动 19;被移除的文件是 `resources/codex-resources/bwrap`,大小 0.5 MB。

事实包只证明该资源文件在 Windows 新包中消失,不能证明相关沙箱路径、运行策略或平台支持发生了怎样的行为变化。

### macOS 原生组件小幅变化,且无新增 cua_node 包【信号】[仅 macOS]

macOS 包内多个 native(原生)组件出现小幅体积变化,例如 `bare-modifier-monitor` 增加 0.02 MB,`sparkle.node`、`sky.node`、`remote-control-device-key.node`、`browser-use-peer-authorization.node` 均有约 0.01 MB 级别变化。`better_sqlite3.node` 从 2.0 MB 降到 1.9 MB,减少 0.03 MB。

macOS 的 cua_node 依赖变化显示新增 0 个包。Windows 同样显示 cua_node 新增 0 个包,但没有列出同样的 native 组件清单,因此原生组件细节只归为 macOS 信号。
