# Codex 桌面版 26.611.61049 macOS / Windows 对照

> 第三方跨平台变更日志 · 非官方

本批次的实质变化集中在 macOS 26.609.71450 → 26.611.61049；Windows 输入样本是 26.611.7849.0 → 26.611.7849.0，文件与内容完全一致。

One-liner: macOS shows a broad capability and UI/runtime update, while this Windows sample is unchanged.

## 关于这份报告

本报告只使用两份机器提取事实包：macOS arm64 来自官方全量包逆向 diff，Windows x64 MSIX 样本来自同版本号包对比。macOS 样本有版本、文件树、app.asar、前端模块、可读字符串、Info.plist、技能文档、API 文档、package.json 与关键源码片段；Windows 样本的 from/to 完全相同，因此只能说明“本输入样本在 Windows 平台无内容变化”。

证据等级分两档：  
【实证】表示来自 plist、manifest、CSP、技能文档、API 文档、package.json、依赖清单、插件文档、服务入口、可读字符串或关键文件 diff。  
【信号】表示仅来自前端模块名、同名模块体积/字符串/样式线索或压缩 bundle 方向线索，说明进入包内的方向，但不等同于功能已对所有用户开放。

平台标记使用 `[macOS+Windows]`、`[仅 macOS]`、`[仅 Windows]`。由于 Windows 样本无内容变化，本报告没有可归为 `[macOS+Windows]` 的新增变化。

## ✨ 重点变化

### 【实证】[仅 macOS] macOS 版本、Chromium 基线与签名产物更新

macOS 从 `26.609.71450` build `3965` 升到 `26.611.61049` build `3996`。Info.plist 同步显示 `CFBundleShortVersionString`、`CFBundleVersion`、`ElectronAsarIntegrity/Resources/app.asar/hash` 与 `SCMRevision` 更新。

Chromium 基线从 `149.0.7827.54` / `7827.54` 变为 `149.0.7827.115` / `7827.115`。文件树中也出现新的 `Codex Framework.framework/Versions/149.0.7827.115/...`，并移除对应的 `149.0.7827.54/...` 文件。

Windows 输入样本的上一版与最新版均为 `26.611.7849.0`，SHA-256 相同，文件数、app.asar 与 cua_node 统计均无变化。

### 【实证】[仅 macOS] Browser / Chrome 控制 API 增加 JavaScript 对话框能力，并调整浏览器能力描述

`browser/docs/api.md`、`chrome/docs/api.md` 与 `browser/.codex-plugin/api-multi-tab.md` 都新增 `Tab.getJsDialog(): Promise<undefined | Dialog>`，并声明 `AlertDialog`、`BeforeUnloadDialog`、`ConfirmDialog`、`PromptDialog` 四类对话框能力。`PromptDialog.accept(text)`、`ConfirmDialog.accept()` / `dismiss()` 等接口进入包内。

同一组 API 文档还把 `Tab.clipboard` 的说明从“this tab”改成“the browser session's clipboard”，并把 `Browsers.list()` 返回类型展开为包含 browser/tab capabilities、metadata、name、type 的结构。这是浏览器自动化能力面的明确文档变化。

### 【实证】[仅 macOS] 浏览器技能新增登录受阻时的行为约束

`control-chrome/SKILL.md`、`control-in-app-browser/SKILL.md` 与 `control-in-app-browser-multi-tab.md` 都新增同一条规则：当用户明确要求在浏览器里导航到某页，但认证或登录阻挡任务时，不应切换到 web search、搜索引擎、其他站点或其他来源绕过登录；若环境提供安全浏览器认证则使用该流程，否则要求用户登录后继续。

这不是 UI 信号，而是内置技能说明直接变化，影响浏览器控制任务的默认执行边界。

### 【实证】[仅 macOS] Computer Use 的 macOS 操作被包上策略层

`@oai/sky` 从 `0.4.10` 升到 `0.4.13`。`package.json` 同时移除了 `objc-js` optionalDependencies。`press_key.js`、`type_text.js`、`drag.js`、`set_value.js`、`scroll.js`、`click.js`、`perform_secondary_action.js`、`get_app_state.js`、`select_text.js` 都从直接调用 client 改为通过 `withComputerUsePolicy(...)` 执行。

`list_apps.js` 新增 `setComputerUseResponseMeta(null)`。`client.d.ts` 从 `apple-event` 改为引用 `native-pipe` 的 `CodexMetadata`，去掉 `serviceAppPath`、`serviceBundleIdentifier`、`serviceProcessIdentifier` 请求参数，并新增 transport 相关私有字段。包内还出现新增字符串 `computer-use native pipe helper paths unavailable`，旧字符串里移除了 `computer-use native pipe transport module missing`。

这组变化说明 macOS Computer Use 运行时在输入动作、状态读取与传输层上都进入了新的策略/transport 路径。Windows 样本 cua_node 仍为 `285.6 MB → 285.6 MB`，无依赖或文件变化可写。

### 【实证】[仅 macOS] Computer Use 安全边界新增“非 OpenAI URL 禁止附着桌面 surface”

新版 bundle 新增可读字符串 `Refusing to attach desktop surface to non-OpenAI URL`。同批还出现 `Failed to position native pet window`、`Failed to start realtime in the avatar overlay`、`Process stdin is unavailable` 等运行时错误路径。

这些字符串确凿进入包内，可作为安全拦截、浮窗定位、实时语音启动和进程 stdin 错误路径进入产物的证据；但不能仅凭字符串断言用户界面一定可见。

### 【信号】[仅 macOS] 远程控制增加手动配对设备入口，删除逻辑从“离线”转向“先禁用环境”

`remote-connections-settings` 新增 `Add a device`、`Pairing code`、`Enter pairing code generated by the device you want to control`、`Failed to add device`、`Disable this environment before deleting it` 等字符串；旧字符串 `Only offline environments can be deleted` 与 `settings.remoteControlConnections.delete.offlineOnly` 被移除。

主进程新增字符串也有 `Remote-control client has been revoked`。这些共同指向远程控制的设备配对、撤销与删除前置条件变化。证据来自同名前端模块与 bundle 字符串，按规则以【信号】呈现；其中新增 bundle 字符串本身是进入包内的【实证】事实。

### 【信号】[仅 macOS] 设置页扩展：云环境、默认权限、字典听写、字体大小、弹出窗口、环境变量与 secrets

新增前端模块包括 `cloud-environments-settings-page`、`cloud-preferences`、`cloud-preferences-settings`、`code-review-settings`、`settings-surface`。`settings-page` 新增 `Cloud environment`、`Create cloud environment`、`Create environment`、`Controls internet access after setup completes`、`Add variable`、`Add secret`、`Add editor`、`Additional editors`、`All settings`、`Automatic setup`。

`general-settings` 新增 `Accessible label for disabled default permissions toggle`、`composer context usage indicator setting toggle`、`work mode settings options`、`hold-to-dictate hotkey capture input`、`one dictation dictionary entry`、`Popout Window projectless default setting toggle`、`ambient suggestions toggle`，还出现 `Adjust the base size used for code across chats and diffs` 与 `Adjust the base size used for the Codex UI`。

这些是设置面扩大的强信号，但来自前端模块与同名模块线索，不能写成所有设置已对所有用户开放。

### 【信号】[仅 macOS] 语音、听写与浮动 avatar overlay 被明显重做

新增模块包括 `avatar-overlay-realtime-voice-button`、`global-dictation-orb`、`use-dictation`、`use-dictation-hotkey`、`use-avatar-overlay-selection`。`avatar-overlay-native-frame` 新增 `Dictation is not available on this device`、`composer.dictation.unsupported`、`Follow up`、`Accessible label for replying to a floating avatar notification`、`Accessible label for stopping a running task from the floating avatar notification tray`。

主进程新增 `Preferred microphone input device for dictation` 与 `Failed to start realtime in the avatar overlay`。多语言 bundle 也出现 `avatarOverlay.startingRealtimeVoice`、`avatarOverlay.stopRealtimeVoice`、`avatarOverlay.notificationFollowUpError`。这组信号指向听写热键、麦克风选择、实时语音按钮与浮窗通知后续输入。

### 【信号】[仅 macOS] 内置浏览器侧栏增加复制链接、打开方式与历史导航控制

新版主进程/前端字符串新增 `Copy link`、`Open in browser`、`Open in external browser`、`Preferred target for opening links`、`Preferred target for opening local URLs`、`browserSidebar.contextMenu.copyLink`、`browserSidebar.contextMenu.openInBrowser`。本地菜单多语言文件大批新增 `browserSidebar.contextMenu.copyLink` 的翻译，例如中文为“复制链接地址”。

新增模块 `back-forward-navigation-buttons`、`page-search-input`、`google-chrome`、`browser-profile-import-dialog`、`browser-profile-import-query`、`browser-use-settings-visibility` 也支持浏览器侧栏、浏览器配置导入与可见性设置方向。这里既有可读字符串实证，也有前端模块信号；整体按 UI 方向信号处理。

### 【信号】[仅 macOS] 命令菜单拆分关闭 Tab / Window，并增加反馈 slash command、计划模式切换与推理力度调节

主进程新增 `Close Tab`、`codex.commandDescription.closeTab`、`codex.commandMenuTitle.closeTab`、`codex.commandDescription.closeWindow`、`codex.commandMenuTitle.closeWindow`，同时旧字符串 `closeTabOrWindow` 被移除。`app-main` 新增 `Description for the /feedback slash command`、`Send feedback about this chat`、`composer.feedbackSlashCommand.title`、`composer.feedbackSlashCommand.description`、`composer.increaseReasoningEffort`、`composer.decreaseReasoningEffort`，主进程还新增 `composer.togglePlanMode`。

这说明命令体系从合并关闭动作变为更细分的 Tab/Window 动作，并加入反馈、计划模式和推理力度相关文案。

### 【信号】[仅 macOS] @ mention 面板扩展到 Computer Use、自定义 agents、桌面 app、MCP servers 与 live agents

`at-mention-list` 新增 `Computer use`、`Custom agents`、`Desktop apps`、`Live agents`、`MCP servers`、`Mac apps`、`No results`、`Section header for the current MCP server mention scope`，样式类名出现 `mcp-capability`、`mcp-capability-mentions`、`mcp-servers`。

这是输入 mention 面板支持更多对象类型的强信号，但仍来自前端模块线索，不能断言所有 mention 类型都已可用。

### 【信号】[仅 macOS] Sites 工作流文案进入会话线程，技能文档收紧认证脚手架策略

`local-conversation-thread` 新增 `Creating Sites project`、`Created Sites project`、`Getting Sites environment`、`Getting Sites project`、`Getting Sites version`、`Deploying Sites version`、`Deployed Sites version`、`Got Sites deployment`。多语言 bundle 也出现 Sites 项目、版本、部署、访问组相关翻译。

`sites-building/SKILL.md` 的认证建议发生实质变化：原先建议公共或外部认证站点添加 SIWC/OAuth，且可使用 `templates/auth-siwc-enabled`；新版改为先确认当前平台 auth path，不再从 starter 脚手架添加 app-owned public sign-in 或 external OAuth。模板 `auth-siwc-enabled/package-lock.json` 也在 bundle 文件树中作为移除大文件出现。

这是 Sites 能力面与认证边界的组合变化：UI 文案是【信号】，技能文档是【实证】。

### 【信号】[仅 macOS] App generation / artifacts / image generation 相关线索增多

新增模块包括 `appgen-access`、`artifact-analytics`、`appshot-icon-cache`、`patch-diff-stats`。多语言 bundle 出现 `appgenConversation.createPrompt`、`appgenConversation.editPrompt`、`appgenPage.edit`、`appgenPage.share.tooltip`、`artifactAnnotationComment.placeholder`。`local-conversation-thread` 新增 `Generating image...` 和 `Aria label for the image generation loading placeholder`。

### 【实证】[仅 macOS] Work Louder 依赖升级，但调试字符串减少

应用 `package.json` 中 `@worklouder/device-kit-oai` 从 `0.1.6` 升到 `0.1.10`，`@worklouder/wl-device-kit` 从 `0.1.10` 升到 `0.1.18`。同时主进程移除 `Work Louder debug`、`Work Louder error`、`Work Louder info`、`Work Louder warning` 字符串。
