# Codex 桌面版 26.611.61049(macOS / Windows 对照)

> 第三方跨平台变更日志 · 非官方

本批次的核心结论很直接：macOS 包从 26.609.71450 build 3965 升到 26.611.61049 build 3996，Windows x64 MSIX 对照包仍是 26.611.7849.0，文件级差异为 0。

The observable update lands on macOS only in this fact set; the Windows MSIX comparison is byte-for-byte unchanged.

## 关于这份报告

这份报告基于两份官方签名分发包的逆向重建 diff：macOS 来自 Sparkle 分发包，Windows 来自 MSIX 对照包。macOS 使用两个官方全量包比较，并验证为 Notarized Developer ID，签名来源为 OpenAI OpCo, LLC。Windows 事实包给出的前后全量包 SHA-256 完全相同，文件树也完全相同，因此本轮 Windows 只能判定为“无可观测包内容变化”。

证据分两档：【实证】表示来自 plist、AppxManifest、CSP、技能文档、类型声明、package.json、依赖清单等可读材料；【信号】表示仅来自前端模块名、bundle 字符串或体积变化，能指示方向但不能还原源码意图。

平台归属含义为：[macOS+Windows] 表示两边事实包都能看到同类变化；[仅 macOS] 表示只在 macOS diff 中出现；[仅 Windows] 表示只在 Windows diff 中出现。本批次没有可确认的 [macOS+Windows] 功能变化。

对比范围：macOS 为 26.609.71450 build 3965 → 26.611.61049 build 3996；Windows 为 26.611.7849.0 → 26.611.7849.0。

## ✨ 重点变化

### Computer Use 运行时更新与 macOS 传输层改造【实证】[仅 macOS]

macOS 内置 `@oai/sky` 从 0.4.10 升到 0.4.13，`package.json` 同时移除了 `objc-js` optionalDependency。`cua_node` 总体积基本不变，为 365.8 MB → 365.9 MB。

更关键的是 macOS Computer Use 客户端从直接依赖 Apple Event/service app 路径、bundle identifier、pid 的请求方式，改为 `MacNativePipeTransport`，并新增按 `apiVersion` 缓存 transport 的逻辑。类型声明也删除了 `serviceAppPath`、`serviceBundleIdentifier`、`serviceProcessIdentifier` 这些请求选项。可读代码显示 `click`、`drag`、`press_key`、`scroll`、`set_value`、`type_text`、`select_text`、`perform_secondary_action`、`get_app_state` 等操作都包上了 `withComputerUsePolicy`。

Windows 对照包没有任何文件改动，因此这组 Computer Use 更新在本事实集中不能算 Windows 已跟进。

### 浏览器插件 API 增加 JavaScript 弹窗处理【实证】[仅 macOS]

macOS 的 browser/chrome 插件文档新增 `Tab.getJsDialog()`，并定义了 `AlertDialog`、`BeforeUnloadDialog`、`ConfirmDialog`、`PromptDialog` 与统一的 `Dialog` 类型。`Tab.clipboard` 说明也从“this tab”调整为“browser session's clipboard”，浏览器列表返回类型则内联展开了 capabilities、metadata、name、type 等结构。

这属于插件 API 文档级别的明确变化。Windows 本轮对照无变化，不能确认 Windows MSIX 已包含这些文档更新。

### 浏览器认证阻塞时不再绕路搜索【实证】[仅 macOS]

macOS 的 in-app browser 与 Chrome 控制技能文档新增规则：当用户明确要求在浏览器中导航，且认证或登录阻塞任务时，不应切换到 web search、搜索引擎、其他站点或其他来源绕过登录；若环境提供 secure browser authentication，应使用该流程，否则停止并请用户登录。

这是一条行为策略变化，影响浏览器自动化遇到登录墙时的处理边界。Windows 对照无文件变化。

### 站点技能弱化自带 SIWC/OAuth 模板路径【实证】[仅 macOS]

`sites-building` 技能文档删除了“公共或外部认证站点直接使用 auth-siwc-enabled starter”的建议，改为要求先确认当前平台认证路径，不再从 starter 脚手架添加 app-owned public sign-in 或 external OAuth。新站点 starter 选择也从“多数站点用 vinext，需公共登录时用 auth-siwc-enabled”收敛为使用 `vinext-starter` 并保留结构。

这说明站点构建指引把认证决策从模板默认值转向平台路径确认。Windows 对照无变化。

### 内置浏览器右键菜单出现复制链接入口【实证】[仅 macOS]

macOS 新增可读字符串和多语言菜单项 `browserSidebar.contextMenu.copyLink`，英文为 `Copy link`，大量 native-menu locale 文件加入对应翻译。主进程/前端字符串还出现 `Open in browser`、`Open in external browser`、`Close Tab`、`Copy link` 等。

这能实证说明“复制链接”菜单文案进入包内；是否对所有用户开放仍取决于运行时开关或 UI 路径，事实包无法确认。Windows 对照无变化。

### 前端功能方向：云环境、代码审查、语音、浏览器导入与设置页【信号】[仅 macOS]

去掉 Vite hash 噪音后，macOS 新增模块名包括 `cloud-environments-settings-page`、`cloud-preferences`、`cloud-preferences-settings`、`code-review-settings`、`browser-profile-import-dialog`、`browser-profile-import-query`、`global-dictation-orb`、`use-dictation`、`use-dictation-hotkey`、`avatar-overlay-realtime-voice-button`、`page-search-input`、`back-forward-navigation-buttons`、`patch-diff-stats`、`thread-right-panel-state` 等。

这些模块名显示设置、云环境、代码审查、浏览器资料导入、语音/听写、浏览器导航和 diff 统计方向都有动作，但仅是 bundle 级信号，不构成“功能已上线”的源码事实。Windows 对照无变化。

### Windows MSIX 本轮无可观测变化【实证】[仅 Windows]

Windows x64 对照包前后版本均为 26.611.7849.0，SHA-256 均为 `d19ebe0d2620d7c58f6681c3ed3ef23e12f6396354874db7721086bbd75f6fec`。文件数 11943 → 11943，新增 0、移除 0、内容改动 0；`app.asar` 内部文件 3024 → 3024，新增 0、移除 0、改动 0；`cua_node` 总量 285.6 MB → 285.6 MB。

因此，本文所有 macOS 可见的新行为和文档变化，都不能据此归入 [macOS+Windows]。
