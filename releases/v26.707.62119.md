# Codex 桌面版 26.707.62119 macOS / Windows 对照

> 第三方跨平台变更日志 · 非官方

本批次两平台共同更新了 Browser 与 Chrome 内置插件的版本标记；更广泛的界面与交互变化集中出现在 macOS 样本，Windows 样本则额外包含 Computer Use 插件更新。

Browser and Chrome plugin versions moved forward on both platforms, while broader UI signals are concentrated in the macOS sample and Windows additionally updates its Computer Use plugin.

## 关于这份报告

本报告比较两个分别取得的官方签名分发包样本：

- macOS arm64：`26.707.61608` build 5200 → `26.707.62119` build 5211。
- Windows x64 MSIX：`26.707.8168.0` → `26.707.8479.0`。

两平台的外层版本号并不对齐，因此这里只合并两份事实包中能够独立验证的共同变化，不把一个平台的前端或运行时结论套用到另一个平台。Windows 事实包的“平台/架构”字段误写为 macOS arm64；本报告依照输入定义，将该样本作为 Windows x64 MSIX 处理。

证据分为两档：

- 【实证】来自 plist、插件 manifest、服务或依赖清单、签名验证、可读配置，以及不属于“同名前端模块改动线索”的包内字符串。
- 【信号】来自前端模块名、压缩 bundle 体积、同名模块内的字符串或样式集合变化。它们能说明代码方向进入构建，但不能证明功能已向所有用户开放。

平台标记 `[macOS+Windows]`、`[仅 macOS]`、`[仅 Windows]` 表示证据实际来自哪份样本。Vite hash 文件改名已作为构建噪音排除。

事实包未给出单独的“新增内置插件明细”“主进程新增可读字符串”“新增组件与资源”或完整插件清单变化。两平台 Bundle 文件树均无新增或移除文件；因此本报告不会把现有插件的版本改动描述成新插件安装。

## ✨ 重点变化

### Browser 与 Chrome 内置插件同步刷新【实证】[macOS+Windows]

两平台的 `browser` 与 `chrome` 插件 manifest 都把内部版本从 `26.707.61608` 更新为 `26.707.62119`。

Browser 插件说明仍明确面向 ChatGPT 内置浏览器及 `@browser`、`@browser-use` 等别名，覆盖打开、检查、导航、测试、点击、输入和截图本地 Web 目标；Chrome 插件仍面向依赖现有标签页、登录会话、Cookie、扩展与 ChatGPT Chrome Extension 状态的自动化。此次具体 diff 只改变版本字段，不能据此宣称这些能力在本批次首次加入。

平台差异在于外层包版本：macOS 应用版本与插件版本同为 `26.707.62119`，Windows MSIX 外层版本为 `26.707.8479.0`，但插件内部版本同样更新到 `26.707.62119`。

### Windows Computer Use 插件同步到新内部版本【实证】[仅 Windows]

Windows 的 `computer-use` 插件 manifest 从 `26.707.61608` 更新到 `26.707.62119`，其说明仍为通过 Computer Use 从 ChatGPT 控制 Windows 桌面应用。

该 diff 只确认既有插件的版本推进，没有插件新增文件，也没有 Computer Use 依赖扩张。Windows 的 `cua_node` 内容总量保持 `287.2 MB`，新增依赖包为 0。macOS 样本没有对应 manifest 改动，不能把这条更新标为跨平台共同变化。

### 自动审查与权限请求状态进入前端改动面【信号】[仅 macOS]

macOS 的同名 onboarding 前端模块新增了多组自动审查文案，包括：

- reviewer agent 正在审查请求；
- 审查完成、提前停止或超时；
- 自动审查网络访问请求；
- 带理由或不带理由的权限请求；
- MCP 工具调用；
- 单文件编辑。

这些字符串全部来自“同名前端模块改动线索”，因此只能确认相关状态和动作摘要进入了本次前端构建方向，不能证明自动审查已默认启用。

同一批次中，Quick Chat/ChatGPT conversation 模块移除了 `Allow once`、`Allow this conversation`、`Always allow`、审批范围选项和拒绝按钮等字符串，并移除了 `data-codex-approval-surface` 样式标记。这表明审批界面可能经历重组，但不能据此认定相应审批能力被删除。

Windows 事实包没有前端模块或字符串级证据。

### Quick Chat 获得更完整的窗口与历史交互线索【信号】[仅 macOS]

macOS 主页面模块新增了以下可访问性与动作字符串：

- 从当前聊天标题打开 Quick Chat 历史；
- 从历史返回当前聊天；
- 开始新的 Quick Chat 对话；
- 调整聊天窗口高度和宽度。

对应样式集合新增拖动、对角缩放、Quick Chat 滚动容器与滚动占位元素，前端真实模块增删中也多次出现 `quick-chat-window-page`、`hotkey-window-thread-page`、`hotkey-window-home-page` 和 `composer-utility-bar` 的重新组合。

这些证据共同指向 Quick Chat 窗口的尺寸调整、历史导航和布局重构，但仍属于前端信号。

### 线程工作区出现底部面板与侧边任务标签入口【信号】[仅 macOS]

`thread-app-shell-chrome` 同名模块新增了底部面板、关闭底部面板、在底部或侧边面板打开其他标签，以及从任务侧栏打开以下内容的动作文案：

- Detail；
- side task；
- browser；
- review；
- terminal；
- workspace file browser。

同时出现响应式宽度条件、侧栏空状态和多组面板布局样式。前端模块增删还涉及 `thread-app-shell-chrome`、本地/远程 conversation 页面、项目索引页和热键窗口。

这是一组较强的任务工作区布局信号，指向浏览器、审查、终端、文件浏览器与任务详情在侧边或底部面板中的统一组织；事实包不能证明所有入口已经可见。

### 内置浏览器设置覆盖历史、下载、密码与站点数据【信号】[仅 macOS]

`browser-use-settings` 模块新增了以下字符串：

- 无需询问即可访问历史记录；
- 浏览器下载与下载历史；

- 展开或隐藏单项浏览数据清理选项；

这些文案和配套状态类表明内置浏览器的数据管理与设置界面正在扩展。由于它们来自同名前端模块差异，不能据此确认默认权限策略，也不能断言所有设置已开放。

值得注意的是，前端真实增删中旧的 `settings-page~browser-use-settings~computer-use-settings` 组合模块被移除；这更可能反映代码拆分或重组，不能单独解释为 Browser Use 或 Computer Use 设置被删除。

键盘快捷键/线程壳层共享模块新增了：

首页公告模块还出现礼品额度、限额重置、Fast mode 与后端驱动公告的关闭按钮，以及打开礼品额度购买流程和可用限额重置弹窗的动作。

这些是用量和付费相关界面的高可见信号，但事实包不包含价格、资格、地区、账户层级或正式开放状态。

### 多智能体并行委派入口进入首页公告【信号】[仅 macOS]

`home-announcements` 新增了 multi-agent composer banner 相关字符串，其中包含“把工作委派给并行工作的子智能体，可能增加 token 用量”的正文，并新增 `multi-agent` 与已读状态类名。

这确认多智能体入口相关文案进入了 macOS 前端改动面，但不证明该入口已对所有用户出现，也不提供调度数量或权限策略。

### Appgen 站点分享与公开发布流程扩展【信号】[仅 macOS]

`appgen-share-dialog` 新增了：

这表明 Appgen 站点分享正在覆盖成员选择、链接范围与公开发布流程。另一方面，`appgen-settings-page` 移除了自定义域名、环境变量或 secret、站点标题、slug 和删除确认等字符串，说明设置界面也在重组。两组证据均为前端信号，不能解释为功能正式新增或删除。

### 远程 Codex 连接增加可诊断错误路径【信号】[仅 macOS]

本地 conversation 相关模块新增了多种远程连接错误文案：

- 远程机器未安装 Codex CLI；
- 远程 Codex 版本过旧；

这些字符串表明远程环境连接错误正在获得更细粒度的诊断。它们仍来自同名前端模块，不能证明远程连接协议或安装流程本身发生了何种底层改变。

### Pull Request、diff 与冲突查看能力重组【信号】[仅 macOS]

Pull Request/code review 相关共享模块从接近空体积增长约 `815,704` 字节，并新增：

- `(Current Change)` 与 `(Incoming Change)`；
- 文件与目录冲突错误；
- diff annotation、additions、change type 等状态类；
- Ayu Dark、Catppuccin Latte 等主题字符串。

onboarding 共享模块还新增了打开当前 diff 文件的按钮说明、diff header、file diff、hunk actions 与 managed worktree 状态类。前端真实模块增删中也出现多组 `pull-request-route`、`pull-request-code-review` 和 artifact tab 重新组合。

这些证据指向差异查看、合并冲突或审查界面的显著打包调整，但不能仅凭压缩 bundle 判断具体交互已经上线。

### ChatGPT 项目编辑权限提示更明确【信号】[仅 macOS]

远程 conversation 页面新增了“用户无法编辑 ChatGPT 项目”的 tooltip 和 `deletePermissionRequired` 相关键，同时移除了管理员禁用项目操作、无法删除项目、`All tasks`、`Codex Cloud` 及多组侧栏分区文案。

这显示项目权限提示与侧栏组织正在调整。字符串的增加与移除不等同于权限模型本身发生改变。

### 热键冲突处理增加取消与警告状态【信号】[仅 macOS]

`hotkey-setter` 新增取消热键捕获的按钮文案，以及正在捕获的热键已被其他动作占用时的警告，并加入 `settings.hotkeySetter.cancel` 和 `settings.hotkeySetter.conflict` 键。

这是一项直接面向设置交互的信号，但没有 Windows 对应证据。

### 表格处理 bundle 明显增长【信号】[仅 macOS]

`spreadsheet` 模块增加约 `190,692` 字节，并新增 `oaiproto.coworker.xlsx` 字符串；与此同时，`document` 模块缩小约 `26,737` 字节并移除 `oaiproto.coworker.docx`。

这表明 Office 文件处理代码的打包分布发生变化，尤其偏向表格路径。仅凭 bundle 变化不能认定 XLSX 能力首次加入或 DOCX 能力被删除。

### macOS 应用版本与完整性记录更新【实证】[仅 macOS]

`Info.plist` 记录：

- `CFBundleShortVersionString`：`26.707.61608` → `26.707.62119`；
- `CFBundleVersion`：`5200` → `5211`；
- `ElectronAsarIntegrity/Resources/app.asar/hash` 更新为新的完整性散列。

前后两个 macOS 全量包均通过公证验证，结果为 `accepted`、`source=Notarized Developer ID`，签发主体为 OpenAI OpCo, LLC。

Windows 事实包没有提供 AppxManifest 的字段级变化或签名验证输出，因此不补写对应结论。

### CSP 没有出现可归因的策略变化【实证】[仅 macOS]

macOS `webview/index.html` 的具体 diff 显示入口脚本和 module preload 集合重排，而展示出的 Content Security Policy 保持相同：仍限制默认源，允许指定的 Web Sandbox frame/child、OpenAI 与 Mapbox 连接目标、ChatGPT WebSocket，以及本地脚本、样式、字体和媒体来源。

Windows 事实包没有 CSP 差异区块，不能推断其策略是否相同。
