这不是官方变更日志，而是一份**基于官方分发产物的逆向差异报告**。本次为历史回填：两个版本均已超出官方 Sparkle(应用自动更新框架) 增量窗口，因此采用两个官方全量包直接比较，而不是增量包重建。

应用的公开版本号与内部 build(构建编号) 都发生了更新：`CFBundleShortVersionString` 从 `26.406.31014` 变为 `26.406.40811`，`CFBundleVersion` 从 `1395` 变为 `1457`。

这意味着这是一个明确发布的新桌面版构建，而不是仅重签名或重新打包。

证据：

```diff

CFBundleVersion: 1395 -> 1457
```

### 2. 应用层内容有少量更新，但没有观察到新增 bundle 组件 【实证】

证据：

```text

```

事实包显示 `cua_node` 依赖新增数量为 `0`。这说明本次差异里没有观察到新的底层 Node(运行时环境) 依赖包加入该依赖集合。
