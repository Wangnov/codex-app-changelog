# Codex 桌面版 `26.323.20928` build 1173

> 第三方变更日志 · 非官方 · 基于官方更新包逆向差异

本版从 `26.318.11754` build 1100 更新到 `26.323.20928` build 1173，最明显的可见变化集中在启动体验、设置页结构、插件入口和代码展示能力。  
English one-liner: this build adds a native startup loader and shows early restructuring around plugins, settings, remote connections, and syntax rendering.

## 关于这份报告

这不是官方变更日志，而是一份**基于官方分发产物的逆向差异报告**。本次为历史回填：官方 appcast 未对这一对版本提供 Sparkle 增量包，因此事实包改为下载两个官方全量包并直接比较 bundle 内容。

对比范围是 macOS arm64：上一版 `26.318.11754` build 1100，发布于 `2026-03-19`；最新版 `26.323.20928` build 1173，发布于 `2026-03-24`。

证据等级分两档：**【实证】**表示差异来自 Info.plist、可读配置、HTML/CSS、主进程 bundle 字符串、签名或验证结果等人类可读证据；**【信号】**表示只来自前端模块名、同名前端模块字符串差或体积变化，指向某方向，但无法从产物还原到源码级意图。

## ✨ 重点变化

### 1. 启动阶段新增内联 Codex logo 加载动画 【实证】

`webview/index.html` 在 React 主 bundle 加载前新增了一整段内联启动样式和 DOM：`startup-loader`、`startup-loader__logo`、`startup-codex-logo-fade-in`、`startup-codex-logo-shimmer`，并通过 SVG mask 绘制 Codex logo。

这意味着应用白屏阶段可能更早出现一个轻量启动占位，而不必等主前端 bundle 完成加载。证据来自 `webview/index.html` 的可读 diff：新增 `color-scheme: light dark`、`prefers-reduced-motion: reduce` 和 `startup-codex-logo-shimmer 2200ms`。

### 2. 插件设置入口被重新拆分成独立前端模块 【信号】

去噪后的真实新增模块包含 `plugins-page`、`plugins-settings`、`settings-content-layout`、`settings-sections`，`settings-surface` 中还新增了 `Title for plugins settings section` 与 `plugins-settings`。

这指向插件相关设置正在从旧页面结构中拆出来，但这些线索来自前端模块名和同名模块改动，无法从产物还原到源码级意图。相关旁证是 `skills-page` 移除了 `All plugins`、`App enabled`、`App disabled`、`Accessible label for the plugin enablement toggle` 等字符串。

### 3. 远程连接与远程项目路径的交互继续调整 【实证】

主进程/前端 bundle 新增可读字符串包括 `Remote command exec connection is unavailable` 和 `resolveRemoteSshCodexHome timed out`；`remote-connections-settings` 新增 `Add new connection` 与 `Button label to manually add a remote connection`。

这说明远程命令执行连接、远程 Codex home 解析超时、手动新增远程连接等路径在本版有明确可读痕迹。与此同时，`index` 模块中出现 `Accessible label for the disabled input showing the current remote folder path`、`Add your remote project from this menu`，指向远程项目加入流程的 UI 文案调整。

### 4. 设置页新增 Enter 行为开关相关文案 【信号】

`general-settings` 新增 `settings.general.enterBehavior.label`、`settings.general.enterBehavior.description`、`Label for the enter key behavior toggle` 和 `Description for the enter key behavior toggle`。

这指向通用设置里出现 Enter 键行为配置，但该证据来自同名前端模块改动线索，无法从产物还原到源码级意图或确认具体可配置项。

### 5. 合并冲突展示与操作状态有新增痕迹 【信号】

`general-settings` 新增 `(Current Change)`、`(Incoming Change)`、`merge conflict actions`，以及 `data-merge-conflict-action`、`data-merge-conflict-conflict-index`、`merge-conflict-action` 等状态类名。

这指向合并冲突展示或操作控件的 UI 能力增强，但线索仍停留在前端 bundle 的字符串与类名层面，无法确认完整交互行为。

### 6. Diff 与多语言语法高亮规则大幅调整 【信号】

`diff` 模块新增 `markup.changed.diff`、`markup.deleted.diff`、`markup.inserted.diff`、`meta.diff.header.git`、`meta.diff.range.unified` 等规则；`jsx`、`php`、`less`、`python`、`java`、`shellscript`、`ruby` 模块体积上升并新增大量语法 scope 字符串。

这指向代码展示、diff 渲染和语言高亮能力被重新打包或扩展，但无法从产物确认源码级实现。与此同时，`cpp`、`fortran-free-form`、`stylus`、`scss`、`vue`、`postcss`、`katex`、`tex` 等模块出现体积下降或字符串移除，说明语法资源也存在重新裁剪。

### 7. 主进程新增窗口与 Git 启动错误文案 【实证】

新版主进程 bundle 新增可读字符串 `New Window` 与 `Failed to start git process`。

这表明本版至少引入了新的窗口入口文案，以及 Git 进程启动失败的错误路径文案。由于证据是压缩 bundle 中的字符串字面量，只能确认这些文案存在，不能进一步确认触发条件。

### 8. 蓝牙权限说明从 Always 切换为 Peripheral 【实证】

`Info.plist` 移除了 `/NSBluetoothAlwaysUsageDescription`，新增 `/NSBluetoothPeripheralUsageDescription = 'This app needs access to Bluetooth'`。

这是一项 macOS 权限声明层面的变化，用户可能在系统权限提示或隐私设置中看到不同的蓝牙权限类别。版本号也同步从 `26.318.11754` / `1100` 更新到 `26.323.20928` / `1173`。
