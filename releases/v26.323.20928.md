# Codex 桌面版 `26.323.20928`(build 1173)

> 第三方变更日志 · 非官方 · 基于官方更新包逆向差异

这一版最可见的变化是 Webview(内嵌网页视图) 启动阶段加入 Codex 标志加载动画；同时,前端模块名出现插件页面与插件设置方向的信号。  
One-liner: startup polish, plugin-settings signals, and a larger native `codex` payload.

## 关于这份报告

这不是官方变更日志,而是一份**基于官方分发产物的逆向差异报告**。本次是历史回填:两个版本均超出官方 Sparkle(应用更新框架) 增量窗口,因此采用两个官方全量包直接比较,再做结构化 bundle(应用包) 差异分析。

可信度方面,两个制品均来自官方下载地址,并记录了 SHA-256(文件校验哈希)。公证验证结果均返回 `CSSMERR_TP_CERT_REVOKED`,事实包说明这是部分早期版本签名证书后来被吊销导致,影响该版本能否被系统安全安装,但不影响对 bundle 内容做差异分析。

本文使用两档证据等级:

- **【信号】**: 来自前端模块名或二进制体积变化等间接证据；这类证据只指向某方向,但无法从产物还原到源码级意图。

## ✨ 重点变化

### 1. 启动阶段新增 Codex 标志加载动画 【实证】

`webview/index.html` 新增了一整段内联样式,定义了 `.startup-loader`、`.startup-loader__logo`、淡入动画与 shimmer(扫光) 动画,并内嵌 Codex 标志的 SVG(矢量图) mask。

这意味着应用 Webview 在主脚本加载前,会先显示一个更完整的启动加载状态；同时它尊重 `prefers-reduced-motion: reduce`,对减少动态效果的系统偏好做了处理。

关键证据:

```diff
+      .startup-loader {
+        display: flex;
+        width: 100%;
+        height: 100%;
+        align-items: center;
+        justify-content: center;
+      }
+      .startup-loader__logo {
+        width: 56px;
+        height: 56px;
+        animation: startup-codex-logo-fade-in 180ms ease-out 60ms forwards;
+      }
+      .startup-loader__overlay {
+        animation: startup-codex-logo-shimmer 2200ms cubic-bezier(0.4, 0, 0.2, 1) infinite;
+      }
```

### 2. 插件设置相关界面开始出现在前端构建中 【信号】

去掉 Vite(前端构建工具) hash(内容指纹) 噪音后,真实新增模块包含 `plugins-page`、`plugins-settings`、`settings-content-layout`、`settings-sections`。

这指向“插件页面 / 插件设置 / 设置页结构”方向的界面工作,但无法从产物还原到源码级意图,也不能证明这些入口已经对用户开放。

关键证据:

```text
新增模块: plugins-page, plugins-settings, settings-content-layout, settings-sections
```

### 3. 设置与操作控件模块有一组前端补充 【信号】

新增模块还包括 `control-group`、`regenerate`、`pencil`、`arrow-left`、`invalidate-queries-and-broadcast`、`data-url-from-svg`。

这些名称指向控件分组、重新生成、编辑、返回、查询失效并广播、SVG 转 data URL(内联资源地址) 等方向,但仍属于前端模块名信号,无法从产物还原到源码级意图。

关键证据:

```text
新增模块: arrow-left, control-group, data-url-from-svg, invalidate-queries-and-broadcast, pencil, regenerate
```

### 4. 蓝牙权限描述从 Always 切换为 Peripheral 【实证】

`Info.plist` 移除了 `NSBluetoothAlwaysUsageDescription`,新增 `NSBluetoothPeripheralUsageDescription`,两者文案均为 `This app needs access to Bluetooth`。

这表示 macOS(苹果桌面操作系统) 权限声明键发生了调整；事实包没有给出对应功能入口,因此只能确认权限描述键变化,不能进一步推断蓝牙功能变化。

关键证据:

```text
新增 /NSBluetoothPeripheralUsageDescription = 'This app needs access to Bluetooth'
移除 /NSBluetoothAlwaysUsageDescription = 'This app needs access to Bluetooth'
```

### 5. 底层 `codex` 组件体积明显增长 【信号】

`Contents/Resources/codex` 从 `117.0 MB` 增至 `146.5 MB`,变化为 `+29.48 MB`,是本次体积变化最大的组件。

这说明底层 `codex` 载荷有明显增加,但仅凭二进制体积无法判断具体新增了什么能力或代码路径；它指向底层能力或打包内容变化,但无法从产物还原到源码级意图。

关键证据:

```text
Contents/Resources/codex: 117.0 MB → 146.5 MB (+29.48 MB)
```
