# Codex 桌面版 `26.313.41514` build 1043

> 第三方变更日志 · 非官方 · 基于官方更新包逆向差异

这是一次体积和文件结构几乎不变的维护版本，变化集中在 macOS 权限声明与任务启动错误路径的前端提示。  
One-liner: a small maintenance release focused on permission metadata and task-start error messaging.

## 关于这份报告

这不是官方变更日志，而是一份**基于官方分发产物的逆向差异报告**。本次为历史回填：官方 appcast 未对 `26.313.41036` → `26.313.41514` 提供增量包，因此改为下载两个官方全量包直接比较。

对比范围为 macOS arm64，上一版 `26.313.41036` build `1032`，发布于 `2026-03-16`；最新版 `26.313.41514` build `1043`，发布于 `2026-03-17`。

证据等级分两档：

- **【实证】**：来自 Info.plist、可读字符串、依赖清单、验证输出等人类可读的直接证据。
- **【信号】**：仅来自同名前端模块改动或体积变化，指向某方向，但无法从产物还原到源码级意图。

## ✨ 重点变化

### 1. 蓝牙权限声明从 Always 切换为 Peripheral 【实证】

Info.plist 中移除了 `NSBluetoothAlwaysUsageDescription`，新增了 `NSBluetoothPeripheralUsageDescription`，两者文案均为 `This app needs access to Bluetooth`。

这意味着应用的 macOS 蓝牙权限元数据发生了调整；从产物层面只能确认声明键名变化，不能进一步判断具体蓝牙功能是否变化。

关键证据：

```diff
- /NSBluetoothAlwaysUsageDescription = "This app needs access to Bluetooth"
+ /NSBluetoothPeripheralUsageDescription = "This app needs access to Bluetooth"
```

### 2. 任务启动失败提示被细分，尤其是云端需求加载失败路径 【信号】

同名 `index` 前端模块中，新增了与启动 thread、云端任务、云端需求加载、任务恢复失败相关的可读字符串，例如 `Error starting thread. Please log out and sign in again`、`composer.localTaskError.cloudRequirements.v2`、`localTaskRow.resumeError.v2`。

这些线索指向任务创建 / 启动 / 恢复失败时的错误呈现被重新梳理，尤其把“cloud requirements could not be loaded”作为更明确的失败原因暴露出来。但该结论仅来自同名前端模块字符串变化，无法从产物还原到源码级意图。

### 3. 错误文案命名从 conversation/task 过渡到 thread/task v2 【信号】

旧字符串包括 `Error starting conversation`、`Error creating task`、`Failed to resume task`、`composer.cloudTaskCreationError`；新字符串转向 `Error starting thread...`、`composer.cloudTaskCreationError.v2`、`composer.cloudTaskError.v2`、`composer.localTaskError.v2`。

这指向相关错误状态和国际化 key 进入了 v2 版本，并且 UI 文案中更强调 thread 这一概念。但这是前端产物的间接信号，不能据此断言底层模型或任务系统协议发生了变化。

### 4. Bundle 文件结构基本稳定 【实证】

外层 bundle 文件数保持 `415 → 415`，新增 `0`、移除 `0`、内容改动 `40`。主要组件体积也没有显著变化：`Contents/Resources/app.asar` 仍为 `41.7 MB`，显示为 `-0.00 MB`。

这说明本版不是一次大规模能力扩展或资源重组。`app.asar` 内部虽然有 `新增 13 / 移除 13 / 改动 8`，但事实包明确指出 `webview/assets` 下大量增减属于 Vite 构建 hash 重命名噪音，因此不应解读为新增或删除功能。
