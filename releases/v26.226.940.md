# Codex 桌面版 `26.226.940` build 720

> 第三方变更日志 · 非官方 · 基于官方更新包逆向差异

本版从产物层面看，是一次围绕工作区发现、远程连接、worktree 工作流、Apps/Skills 入口和模型限制文案的桌面端迭代。  
English one-liner: this build points to workspace discovery, editor integration, worktree handling, Apps/Skills management, and model-limit UI changes.

## 关于这份报告

这不是官方变更日志，而是一份**基于官方分发产物的逆向差异报告**。本次对比范围是 macOS arm64 的 `26.224.1209` build `697` 与 `26.226.940` build `720`。

事实来源包括 bundle 文件树、`Info.plist`、`package.json` diff、主进程/前端 bundle 中新增的可读字符串，以及去除 Vite hash 噪音后的前端模块变化。`webview/assets` 和 `.vite/build` 下 hash 文件名的增减不被当作功能增删。

证据等级分为两档：**【实证】**表示产物中存在人类可读的直接证据，例如 plist 键值、配置 diff、依赖声明、可读字符串；**【信号】**表示只有模块名、同名前端模块字符串差异或体积变化等间接证据，指向某方向，但无法从产物还原到源码级意图。

### 8. 新增 Bluetooth 权限声明 【实证】

`Info.plist` 新增了 `NSBluetoothAlwaysUsageDescription`。  
这表示 macOS 权限声明层面新增了 Bluetooth 访问说明；产物中只显示权限文案本身，无法判断实际调用 Bluetooth API 的路径。

证据：

```text

```

### 9. 开发脚本和依赖清理：dev 引入 cross-env，lint 收敛到 oxlint 【实证】

`package.json` 中 `dev` 脚本改为通过 `cross-env` 设置 `NODE_ENV`，新增 `cross-env` 依赖；lint 脚本从 oxlint + eslint 组合收敛为 `pnpm exec oxlint`，并移除了 `eslint`、`eslint-plugin-import`、`oxfmt` 依赖声明。  
这主要影响项目内部开发与构建维护，不是普通用户直接可见变化。
