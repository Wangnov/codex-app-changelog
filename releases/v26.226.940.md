# Codex 桌面版 `26.226.940`(build 720)

> 第三方变更日志 · 非官方 · 基于官方更新包逆向差异

这一版 Codex（OpenAI 的桌面编程代理应用）在可见能力层面变化不大，最明确的变化是新增 Bluetooth（蓝牙）访问说明，同时应用包和内置命令行组件都有小幅增大。  
English one-liner: a small desktop update with Bluetooth permission metadata and build-tooling cleanup signals.

## 关于这份报告

这不是官方变更日志，而是一份**基于官方分发产物的逆向差异报告**。本次为历史回填：两个版本均已超出官方 Sparkle（macOS 应用更新框架）增量窗口，因此事实包采用两个官方全量包直接比较，而不是增量包重建。

对比范围为 `26.224.1209` build `697`，发布于 `2026-02-24`，到 `26.226.940` build `720`，发布于 `2026-02-26`。平台为 macOS（苹果桌面操作系统）arm64（Apple Silicon 架构）。

可信度方面，两个全量包均来自官方下载地址，并给出了 SHA-256（文件指纹）校验值。公证验证结果均显示 `CSSMERR_TP_CERT_REVOKED`，事实包说明这是部分早期版本签名证书后来被吊销导致，会影响系统安全安装判断，但不影响对 bundle（应用包）内容做差异分析。

本文使用两档证据等级：

- **【实证】**：有人类可读的直接证据，例如 Info.plist（应用元数据文件）、package.json（项目配置文件）diff、签名或验证输出。
- **【信号】**：只有间接证据，例如前端模块名或二进制体积变化；这类证据只指向某方向，但无法从产物还原到源码级意图。

## ✨ 重点变化

### 1. 新增蓝牙访问用途声明 【实证】

最新版 `Info.plist` 新增了 `NSBluetoothAlwaysUsageDescription`，值为：

```text
This app needs access to Bluetooth
```

这意味着系统层面已经为 Codex 声明了蓝牙访问说明。它并不等于已经能确认某个面向用户的蓝牙功能上线；事实包只证明应用元数据里新增了这项权限用途文案。

### 2. 应用版本与构建号推进 【实证】

`package.json`、`Info.plist` 中的版本信息同步从 `26.224.1209` 推进到 `26.226.940`，构建号从 `697` 推进到 `720`。

关键证据：

```diff
-  "version": "26.224.1209",
+  "version": "26.226.940",
```

```diff
-  "codexBuildNumber": "697",
+  "codexBuildNumber": "720",
```

这是一个明确的小版本更新，但事实包没有提供官方 release notes（发布说明），因此不能进一步断言官方发布目标。

### 3. 开发侧 lint 工具链被收敛到 oxlint 【实证】

`package.json` 中，原先组合执行 `oxlint` 与 `eslint` 的 lint（代码检查）脚本被简化为直接执行 `oxlint`，并移除了 `eslint`、`eslint-plugin-import`、`oxfmt` 相关依赖条目。

关键证据：

```diff
-    "lint": "pnpm run lint:oxlint && pnpm run lint:eslint",
+    "lint": "pnpm exec oxlint --tsconfig ./tsconfig.json --max-warnings 0 --type-aware --type-check",
```

这更像是构建和开发流程的内部整理。对普通用户是否有可感知变化，事实包没有直接证据；对开发者而言，它说明此版本的应用源码配置更偏向单一 lint 工具链。

### 4. 开发命令补入 cross-env 【实证】

`dev` 脚本从直接设置 `NODE_ENV=development` 改为通过 `cross-env`（跨平台环境变量设置工具）设置环境变量，同时新增依赖 `cross-env` `^7.0.3`。

关键证据：

```diff
-    "dev": "pnpm --dir .. run generate:third-party-notices && NODE_ENV=development electron-forge start",
+    "dev": "pnpm --dir .. run generate:third-party-notices && cross-env NODE_ENV=development electron-forge start",
```

这指向开发环境启动脚本的跨平台兼容性调整，但事实包没有显示它改变了最终用户功能。

### 5. 出现 usage-settings 与 useForm 前端模块名 【信号】

去掉 Vite（前端构建工具）hash（构建指纹）噪音后，事实包中可见真实新增前端模块名：

```text
usage-settings
useForm
```

这指向“用量设置”或“表单处理”相关方向，但无法从产物还原到源码级意图，也不能证明对应入口已经对用户开放。

### 6. 应用主体和内置 codex 组件小幅增大 【信号】

体积变化最大的组件是：

- `Contents/Resources/codex`：`75.0 MB` → `75.8 MB`，增加 `0.80 MB`
- `Contents/Resources/app.asar`：`34.7 MB` → `35.5 MB`，增加 `0.77 MB`

这说明内置命令行组件与 Electron（桌面应用框架）应用层包体都有小幅增长，但体积变化只能作为方向信号，无法从产物还原到具体源码级功能意图。
