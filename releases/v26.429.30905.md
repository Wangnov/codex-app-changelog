# Codex 桌面版 `26.429.30905` build 2345

> 第三方变更日志 · 非官方 · 基于官方更新包逆向差异

本版从 `26.429.20946` 升至 `26.429.30905`，可见变化集中在 Amazon Bedrock 认证入口信号、插件与技能页面前端模块调整，以及 `app.asar` 体积下降。  
One-liner: this build points to Amazon Bedrock auth UI work and trims part of the plugin / skills frontend surface.

## 关于这份报告

这不是官方变更日志，而是一份**基于官方分发产物的逆向差异报告**。本次为历史回填：官方 appcast 未对 `26.429.20946` → `26.429.30905` 提供增量包，因此采用两个官方全量包直接比较。

证据分为两档：**【实证】**表示来自 Info.plist、可读字符串、签名、公证、依赖清单或文件级统计等可直接核验的产物事实；**【信号】**表示仅来自前端模块名、同名前端模块字符串差异或体积变化，指向某方向，但无法从产物还原到源码级意图。

对比范围为 macOS arm64，上一版 `26.429.20946` build `2312`，发布于 `2026-04-30`；最新版 `26.429.30905` build `2345`，发布于 `2026-05-01`。

## ✨ 重点变化

### 1. 个人菜单出现 Amazon Bedrock 登录相关入口 【信号】

`composer` 前端模块新增了 Amazon Bedrock 认证相关可读字符串，指向个人资料下拉菜单或认证状态展示正在接入 Bedrock 登录状态。

这对用户或开发者的可感知点可能是：应用内模型或账户相关区域开始暴露 Amazon Bedrock 身份状态。但该证据来自同名前端模块改动线索，只能说明 UI 与状态方向，无法从产物还原到源码级意图。

关键证据：

```text

Label indicating the user is authenticated with Amazon Bedrock
Logged in with Amazon Bedrock
codex.profileDropdown.amazonBedrockAuth
```

### 2. 插件详情页的添加与启用交互表面被收缩 【信号】

`plugin-detail-page` 模块体积从 `0.1 MB` 降至 `0.0 MB`，减少 `84639 bytes`，并移除了多条与插件添加、MCP server 启用开关、技能预览菜单和页面导航相关的可读字符串。

这指向插件详情页的 UI、状态或交互路径发生调整，尤其是 `Add to Codex`、`Adding to Codex`、`App unavailable` 这类文案的移除。但这些线索来自同名前端模块差异，只能作为前端行为方向的信号。

关键证据：

```text

App unavailable
Accessible label for the MCP server enablement toggle on the plugin detail page
Accessible label for the section navigation on the plugin detail page
```

### 3. 技能页面的安装列表与刷新入口相关文案被移除 【信号】

`skills-page` 模块减少 `11676 bytes`，移除了创建技能、刷新技能列表、已安装技能分区、空状态和过滤后空状态相关文案。

这指向技能页面的信息架构或入口状态发生收缩或重排，但无法仅凭产物确认是功能删除、路径迁移还是构建层面的前端拆分。


### 4. 主进程 bundle 新增解析错误与 shell 执行相关字符串 【实证】

新版主进程 bundle 中新增了 `Parse Error` 与 `exec /bin/sh -c ` 两个可读字符串。字符串确凿存在于产物中，因此可作为新增错误路径或命令执行路径方向的实证线索。

这意味着底层流程里可能新增或改动了某类解析失败处理，以及经 `/bin/sh -c` 执行命令的路径。但由于证据来自压缩后的主进程 bundle 字符串，不能还原完整源码语义。

关键证据：
