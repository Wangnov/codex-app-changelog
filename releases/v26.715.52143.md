# Codex 桌面版 26.715.52143 macOS / Windows 对照

> 第三方跨平台变更日志 · 非官方

本批次最明确的共同变化，是两平台都加入了原生 HID 拓扑监视组件及对应错误路径；前端方面，Windows 出现更集中的技能、多智能体和内部调试界面扩展信号，macOS 则出现站点发布与共享流程变化信号。

Both packages add a native HID-topology watcher, while Windows shows stronger signals around skills, multi-agent workflows, and debugging, and macOS shows a distinct site-sharing and publishing direction.

## 关于这份报告

本报告比较官方签名分发包逆向重建后的机器差异。由于这是历史回填，macOS 采用两个官方全量包直接比较；两个 macOS 包均通过公证验证，来源为 OpenAI OpCo, LLC。Windows 部分比较两个官方 MSIX 全量包。

对比范围分别为：

- 【实证】[仅 macOS] 26.715.31925 build 5551 → 26.715.52143 build 5591。
- 【实证】[仅 Windows] MSIX 26.715.4045.0 → 26.715.7063.0；包内应用版本字段同时从 26.715.31925 更新到 26.715.52143，构建号从 5551 更新到 5591。

证据分为两档：

- 【实证】来自 plist、manifest、package.json、插件 manifest、依赖清单、原生文件以及包内新增可读字符串。字符串存在于分发包是确定事实，但不等于对应界面已向所有用户开放。
- 【信号】来自同名前端压缩模块的体积、字符串和样式集合变化，只说明 UI、状态、布局或代码拆分方向，不能还原源码，也不能单独证明功能上线或下线。

平台标记分别为 `[macOS+Windows]`、`[仅 macOS]` 和 `[仅 Windows]`。

事实包没有给出完整的内置插件清单增减、去除 hash 噪音后的前端真实文件增删、CSP 差异或 Windows AppxManifest 差异，因此本报告不对这些项目是否保持不变作推断。

## ✨ 重点变化

### 原生 HID 拓扑监视进入两平台包体

【实证】[macOS+Windows] 两个平台都新增了 `hid-topology-watcher.node` 原生组件。macOS 文件为 0.1 MB，Windows 文件为 1.1 MB。

【实证】[macOS+Windows] 新版主进程或前端 bundle 同时加入三条错误路径：

- `Failed to reconcile HID topology`
- `Failed to watch HID topology`
- `HID topology watcher addon not found`

这些证据确认包内新增了 HID 拓扑监视、协调和原生组件缺失处理路径，但不能仅据此判断它服务于哪一种具体设备或用户流程。

【实证】[macOS+Windows] Computer Use 运行时总量没有变化：macOS 保持 365.7 MB，Windows 保持 287.5 MB；两平台的 `cua_node` 依赖均未新增包。因此，这次 HID 变化表现为应用原生资源的独立新增，而不是 Computer Use 运行时依赖扩张。

### 聊天导航、历史与环境筛选共同重构

【信号】[macOS+Windows] 两平台的 `header` 模块都增加约 40 KB，并出现同一组文案与无障碍标签，包括 `All chats`、`Chat history`、`Cloud chats`、`All environments filter option`、`App settings`、`Clear search`、返回按钮标签，以及 Chrome 扩展侧栏的更多操作按钮。

这组共同信号指向聊天历史入口、云端聊天、环境筛选、最近任务搜索和 Chrome 侧栏操作正在被集中到新的头部导航结构中。由于证据来自同名压缩模块差异，它不能证明所有入口已对所有用户可见。

### Windows 技能页面获得创建、搜索与刷新流程信号

【信号】[仅 Windows] `skills-page` 增加 15,629 字节，新增文案覆盖：

- `New skill` 与创建新技能按钮；
- 技能列表搜索；
- 已安装技能与推荐技能分区；
- 加载状态和筛选后的空状态；
- 重新加载技能列表；
- 新安装技能需要刷新后才能在 composer 中使用的提示。

这是完整度较高的技能管理界面信号，涵盖发现、创建、安装后刷新和筛选状态，但事实包没有提供新增技能插件、技能文档或服务入口，不能据此确认新的技能运行时能力。

### Windows 出现多智能体首页引导

【信号】[仅 Windows] `home-announcements` 增加 26,379 字节，并新增多智能体 composer 横幅文案、关闭横幅的无障碍标签以及：

`Delegate work to subagents that work in parallel. Note: may increase token usage.`

同一模块还新增 Fast mode、额度重置、赠送额度和后端驱动首页横幅相关文案，以及 `Codex App Home Beacon Debug`。这确认前端模块正在承载多智能体引导与更多首页公告状态，但不证明横幅已普遍展示。

【信号】[仅 macOS] macOS 的同名模块方向相反，减少 28,328 字节，并移除了多智能体、Fast mode、额度重置、赠送额度和后端驱动横幅相关字符串。这里的“移除”只表示这些字符串离开了该同名模块，不能据此断言对应能力从 macOS 产品中下线。

### macOS 出现站点发布与访问控制界面信号

【信号】[仅 macOS] `appgen-share-dialog` 增加 24,643 字节，新增文案覆盖：

- 添加人员或群组；
- 搜索工作区用户和群组；
- `Anyone in this workspace with the link`；
- `Anyone on the Internet`；
- `Can visit`；
- 保存中的状态标签；
- 复制线上站点 URL；
- 在外部打开线上站点；
- 将站点发布到公共互联网；
- `Copied to clipboard` 与 `Copy link`。

这些字符串与滚动列表、保存操作和无障碍状态样式共同构成较完整的站点共享、访问范围选择和公开发布界面信号。它们仍属于同名前端模块差异，不能证明公开发布已默认开放。

### Windows 调试界面显著扩张

【信号】[仅 Windows] `debug-modal` 从接近零增长到约 0.2 MB，增加 226,002 字节。新增字符串包括 `Account ID`、`Active browser tab`、`Active conversation`、`Activity grouping version`、`All processes`、`Ambient suggestion tasks`、`App Actions`、`Allow host`、`Allow host starting` 和 `(command unavailable)`。

大量对话框、开关、排序、状态色及模糊背景样式也进入该模块。这表明 Windows 包内的调试或诊断界面覆盖了更多账户、浏览器标签、会话、进程、主机许可和应用动作状态；它不代表这些项目是普通用户设置。

### Windows 权限与通用设置模块发生大幅拆分

【信号】[仅 Windows] `general-settings` 减少 92,351 字节，离开该模块的字符串涉及默认权限、Full Access 可见性、composer 上下文用量、聊天和 diff 的代码字号、ChatGPT UI 基础字号、快捷键捕获，以及 ChatGPT macOS Dock 图标选择。

【信号】[仅 Windows] `permissions-mode-dropdown` 减少 25,480 字节，移除字符串包括 `Approve for me`、`Ask for approval`、`Change permissions`、`Custom (config.toml)`，以及访问网站、发送数据、使用插件和编辑外部文件的权限说明。

这些是设置与权限界面重组的高可见信号，但由于只观察到字符串离开原模块，不能判定设置被删除；它们可能被迁移、重新分块或改写。

### Windows 的任务引导、@ 提及与结账界面出现拆分信号

【信号】[仅 Windows] `tasks` 减少 306,482 字节，离开该模块的内容包括连接应用、安装应用、准备连接应用任务、执行 onboarding 任务，以及生成 CSV 图表预览时的动态状态文案。

【信号】[仅 Windows] `at-mention-list` 减少 170,819 字节，移除字符串涉及 ChatGPT conversations、Computer use、Custom agents、Desktop apps、Files and chats，以及 Microsoft Excel、PowerPoint 和 MCP 能力提及入口。

【信号】[仅 Windows] `checkout-webview-page` 减少 13,227 字节，`Back to ChatGPT`、嵌入式结账工具栏返回按钮和 `https://pay.openai.com` 等线索离开该模块。

这些变化共同指向 Windows 前端对任务引导、能力提及入口和嵌入式结账页面进行了较大的重新分块；事实不足以判定相关入口被移除。

### macOS 的插件详情、工作簿与动画资源模块收缩

【信号】[仅 macOS] `plugin-detail-page` 减少 158,328 字节，插件共享对象搜索、链接访问范围、MCP server 启用开关、详情页分区导航和 `App unavailable` 等字符串离开该模块。

【信号】[仅 macOS] `workbook-from-markdown` 减少 108,179 字节，多条文档关闭、序列化、插件列表及查找替换错误字符串被移除。

【信号】[仅 macOS] `plugins-settings` 移除了 `Manage plugins, skills, and MCPs` 及组合插件设置区副标题；这只能证明原模块文案发生变化，不能证明插件、技能或 MCP 管理入口被取消。

【信号】[仅 macOS] `animated-icon` 增加 11,729 字节，并出现多档图标尺寸、`web-search` 和 dotLottie WASM CDN 地址，显示动画图标加载与尺寸处理有所扩展。

### 内置浏览器与 Chrome 插件版本同步

【实证】[macOS+Windows] `browser` 与 `chrome` 内置插件 manifest 的版本都从 26.715.31925 更新到 26.715.52143。给出的 diff 中只有版本字段变化，插件描述未发生变化，因此没有证据支持这两个插件获得了新的命令、权限或服务入口。

【实证】[仅 Windows] `computer-use` 插件 manifest 同样从 26.715.31925 更新到 26.715.52143，描述仍为通过 Computer Use 在 Windows 上控制桌面应用。事实包未显示该插件的其他 manifest 变化。

### 开发构建脚本与平台版本元数据更新

【实证】[macOS+Windows] 两平台 `package.json` 中的应用版本从 26.715.31925 更新到 26.715.52143，`codexBuildNumber` 从 5551 更新到 5591。`prepare:owl` 同时改为调用 `prepare-dev.mjs --owl`，表明 Owl 开发准备流程新增了专用参数。

【实证】[仅 macOS] `CFBundleShortVersionString` 更新为 26.715.52143，`CFBundleVersion` 更新为 5591，Electron ASAR 完整性 hash 随 `app.asar` 内容变化而更新。

【实证】[仅 Windows] MSIX 外层版本为 26.715.7063.0，而包内 `package.json` 版本为 26.715.52143、构建号为 5591，显示这一批次的 Windows 分发版本号与共享应用代码版本号并不相同。

## 🔎 低层信号索引

### 事件与插件错误处理

【信号】[仅 macOS] `esm` 模块新增 `Callback Error`、`Context Cancel`、`Failed to deliver`、destination 加载或卸载失败、`Not ready`、Promise 超时和 `plugin Error`；旧的 event bus 连接、排队和 shim dispatch 文案离开模块。它提示内部事件或插件错误路径发生重组，不足以确定具体运行时行为。

### 插件详情与分享模块迁移

【信号】[仅 macOS] 插件详情页中的分享与 MCP 开关线索减少，而站点分享对话框明显增长。两者可以作为界面职责重新划分的弱关联信号，但不能证明插件分享被站点分享取代。

### Windows 前端模块集中重分块

【信号】[仅 Windows] `debug-modal`、`skills-page` 和 `home-announcements` 增长，与 `tasks`、`general-settings`、`permissions-mode-dropdown`、`at-mention-list` 和结账页面收缩同时发生。这是构建边界和页面职责调整的信号，不能把字节从一个模块的消失等同于产品能力删除。

### 短通用模块与构建噪音

【信号】[仅 macOS] `dist` 仅移除了少量 `dil.action`、`dil.error`、`dil.load` 等字符串，缺乏足够上下文，不提升为功能变化。

【信号】[macOS+Windows] `webview/assets` 下的大量新增和移除来自 Vite hash 重命名。它们不计作真实功能文件增减。
