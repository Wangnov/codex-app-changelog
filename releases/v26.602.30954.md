# Codex 桌面版 26.602.30954(macOS / Windows 对照)

> 第三方跨平台变更日志 · 非官方

本批次的主线是 OpenAI Codex(OpenAI 编程代理桌面应用)在浏览器控制、Chrome(浏览器)集成、Sites(站点构建与托管功能)发布链路上的技能文档和打包结构同步更新。  
One-liner: this release mostly tightens browser automation guidance and Sites deployment packaging across macOS and Windows, with platform-specific packaging differences around Computer Use.

## 关于这份报告

本文基于官方签名分发包的逆向 diff(差异比较)结果：macOS(苹果桌面操作系统)侧比较 Sparkle(macOS 应用更新框架)全量包，Windows(微软桌面操作系统)侧比较 MSIX(Windows 应用安装包格式)分发包。macOS 对比 26.601.21317 build 3511 → 26.602.30954 build 3575；Windows 对比 26.601.2237.0 → 26.602.3474.0。本批次发布日期按输入事实包标记为 2026-05-30。

证据分两档：【实证】表示来自 plist(macOS 属性列表)、plugin.json、package.json、技能文档、模板配置、依赖清单等可读文件；【信号】表示只来自前端模块名或体积变化，可说明方向，但不能当成源码意图。平台归属标记含义为：[macOS+Windows] 两端都有证据，[仅 macOS] 只在 macOS 包中看到，[仅 Windows] 只在 Windows 包中看到。

## ✨ 重点变化

### 浏览器与 Chrome 自动化文档重组【实证】[macOS+Windows]

两端都把 Browser(内置浏览器控制能力)与 Chrome 技能文档大幅压缩，并把大量运行细节迁移到可按需读取的文档入口，例如 `api.md`、`playwright.md`、`confirmations.md`、`api-troubleshooting.md`；Chrome 侧还新增 `chrome-troubleshooting.md`，Windows 事实包还列出 `file-management.md`。旧的 `browser/docs/capabilities/tab/cdp.md` 在两端都被移除。

平台差异在于：macOS 给出了具体 diff，可见 `control-in-app-browser` 从 0.1 MB 降到 0.0 MB，Chrome 技能描述改为优先使用专用 connector/API/CLI(连接器/应用程序接口/命令行工具)，认证失效时要求用户重新认证或明确批准 Chrome 兜底；Windows 文件级统计与新增/移除清单显示同样方向，但输入事实包没有展开 Windows 的技能文档 diff。

### Browser 运行入口更强调 Node REPL 与完整运行时文档【实证】[macOS+Windows]

macOS diff 明确显示 Browser 与 Chrome 技能现在要求通过 Node REPL(Node.js 交互式执行环境)的 `js` 工具加载 `browser-client.mjs`，再调用 `agent.browsers.get("iab")` 或 `agent.browsers.get("extension")`。文档还要求在交互前读取完整 `browser.documentation()`，并在缺少 `js` 工具时通过工具发现查找。

Windows 包同样新增 Browser/Chrome 的 API 与 Playwright(浏览器自动化接口)文档，并且相同三个技能文档体积下降，因此可确认这类文档拆分与运行入口整理是跨平台变化；但 Windows 侧没有提供逐行 diff，不能进一步还原具体措辞。

### Sites 从“创建与部署”扩展为更通用的网站构建/托管工作流【实证】[macOS+Windows]

两端都更新了 Sites 插件元数据：`sites` 插件从 0.1.10 升到 0.1.12，描述改为“Build and deploy websites with Sites”，关键词新增 build、website、landing page、portfolio、dashboard、KPI dashboard、portal、tracker、hub、internal tool，品牌色从 `#10A37F` 改为 `#0C79D8`，默认提示也从披萨餐厅落地页、数独游戏、图片生成网站，调整为新员工门户、数独排行榜、客服趋势仪表盘。

平台差异不大：macOS 给出完整 plugin.json diff，Windows 体积表显示相同文件变化，并在新增文件中列出 Sites 模板的 `sites-vite-plugin.ts`。

### Sites 构建产物改由 Vite 插件负责打包元数据【实证】[macOS+Windows]

两端的 Sites 模板从 `package.json` 的 build 脚本中移除手工复制 `_appgen_meta` 的逻辑，build 脚本简化为 `vinext build`。模板 `vite.config.ts` 新增 `sites()` Vite(前端构建工具)插件；托管技能文档则把部署元数据路径从 `dist/_appgen_meta/appgarden.json` 与 `dist/_appgen_meta/drizzle/` 改成 `dist/.openai/hosting.json` 与 `dist/.openai/drizzle/`。

这说明 Sites 的构建/部署合同更集中在模板插件与 `.openai` 目录结构上。Windows 侧新增清单明确出现两个模板的 `build/sites-vite-plugin.ts`；macOS 侧 diff 明确展示 `vite.config.ts` 引入 `sites()`，但新增文件节选没有列出该插件文件名。

### Computer Use 组件版本与原生包变化不同步【实证】[macOS+Windows]

macOS 的 `computer-use` 插件版本从 1.0.799 升到 1.0.809，并且原生组件体积增量明显：`SkyComputerUseService` +0.74 MB，`CUALockScreenGuardian` +0.71 MB，`SkyComputerUseClient` +0.18 MB。macOS 还新增 `SkysightSummarizer.md` 与多个签名/配置相关文件。

Windows 也有 `computer-use` 相关文件变化，但呈现方式不同：`@oai/sky/bin/windows/codex-computer-use.exe` 只增加 +0.01 MB，同时移除了 `tslib` 多个文件和 `objc-js@1.5.0` 下的两个文件。也就是说，两端都动到了 Computer Use(桌面控制能力)包，但 macOS 更像原生应用组件升级，Windows 更明显体现为依赖裁剪与小幅二进制变化。

### 应用层新增若干前端功能方向信号【信号】[仅 macOS]

macOS 的 `app.asar` 内部模块名新增了 `pricing-plan-page`、`rate-limit-reset-modal`、`github-details`、`share`、`comment`、`composer-controller`、`local-remote-selection`、`selectable-remote-connections-signal`、`project-context-signal`、`bundled-plugin-auto-install-preference` 等；移除了 `device-connections`、`file-preview-page`、`mcp-capability-file-viewer-frame`、`use-automation-history-items` 等。

这些只能说明前端包里出现了价格/套餐页、限额重置弹窗、GitHub(代码托管平台)详情、分享、评论、远程连接选择、项目上下文、捆绑插件自动安装偏好等方向信号；事实包明确提醒“出现 ≠ 对用户开放”。Windows 事实包没有列出模块名，所以不能确认这些模块名在 Windows 包中是否同样存在。

### 开发脚本与 lint 入口调整【实证】[仅 macOS]

macOS 的 `package.json` 把 `owl` 脚本拆成 `prepare:owl` 与 `owl` 两段，并把 `lint` / `lint:fix` 从直接调用 `oxlint` 改成 `node ../scripts/run-package-oxlint.mjs --include-top-level-ts-scripts`。这属于开发/构建链路变化，不直接等同于用户可见功能。

Windows 事实包没有提供对应 `package.json` diff，因此本文不把它标成跨平台变化。

### 本地化文案小修【实证】[仅 macOS]

macOS 包内 `native-menu-locales/mn.json` 与 `native-menu-locales/bg-BG.json` 有关于 About(关于)对话框、build 信息等文案的调整。Windows 事实包未给出对应本地化 diff。
