# Codex 桌面版 `26.318.11754` build 1100

> 第三方变更日志 · 非官方 · 基于官方更新包逆向差异

这一版从产物层面看，是一次偏 UI 与工作流的小步更新：自动审批 review、插件/技能发现、问题通知和代码/公式渲染资源都有可见变化。  
One-liner: a compact desktop update around approval review, plugin discovery, notifications, and editor rendering assets.

## 关于这份报告

这不是官方变更日志，而是一份**基于官方分发产物的逆向差异报告**。本次对比范围为 macOS arm64 的 `26.317.21539` build 1088 到 `26.318.11754` build 1100，两个版本发布日期均为 `2026-03-19`。

本版属于历史回填：事实包显示官方 appcast 未对这一对版本提供增量包，因此采用两个官方全量包直接比较，并对 bundle 文件树、`app.asar`、`Info.plist`、前端 bundle 字符串和关键可读文件做结构化差异分析。

证据等级分两档：

- **【实证】**：来自 Info.plist、CSP、关键文件 diff、依赖清单、签名/验证结果、主进程/前端 bundle 新增可读字符串等人类可读证据。
- **【信号】**：来自同名前端模块、模块名或体积变化等间接证据，只能说明指向某方向，但无法从产物还原到源码级意图。

## ✨ 重点变化

### 1. 自动审批前增加 reviewer 流程相关文案 【信号】

前端主 `index` 模块新增了多条与 automatic approval review 相关的 UI 文案，包括 `Automatic approval review`、`A carefully prompted reviewer agent is reviewing this request before Codex runs it.`、`reviewed this request` 与 `stopped reviewing this request before Codex ran it.`。

这指向一种在 Codex 执行前插入 reviewer agent 的审批/风险检查体验；不过这些线索来自同名前端模块改动，只能判断 UI 与状态方向，无法从产物还原到完整源码级意图。相关状态类名还出现了 `high-risk`、`medium-risk`、`bg-token-danger/15`。

### 2. 插件与技能发现页继续成形 【信号】

`skills-page` 模块新增了 `All plugins`、`New plugin`、`Hide extra results`、`skills.appsPage.createPlugin`、`skills.pluginsPage.section.discover.showExtraSearchResults` 等字符串，并出现 Adobe Acrobat、Data Science、Google Calendar、Google Docs、Google Drive、Google Sheets、Google Slides 等可见项。

这指向插件/技能发现页在搜索、筛选、创建入口和应用分类上的继续完善；但由于证据来自同名前端模块改动，只能标为 UI 与交互方向信号。

### 3. 新增问题通知设置 【信号】

`general-settings` 模块新增 `Enable question notifications`、`Show alerts when input is needed to continue`、`notifications.questions.label` 与 `notifications.questions.description`。

这指向设置页新增或准备新增“需要用户输入时提醒”的开关。证据来自同名前端模块改动，因此只能说明设置 UI 与通知文案方向，不能确认完整通知实现路径。

### 4. 对话 fork 与会话错误路径有新增提示 【信号】

`index` 模块新增了 `Aria label for the button that forks a conversation from this user message`，`app-server-manager-hooks` 模块新增 `Cannot fork a conversation while it is in progress.`、`Forked conversation state not found.`、`Source conversation not found.`、`Target turn not found.` 与 `Target turn must not be after the latest turn.`。

这些线索指向从用户消息派生对话的交互，以及对 fork 边界条件的错误处理。由于来自同名前端模块改动，只能判断功能方向和用户可见错误状态，不能还原后端或状态机实现。

### 5. 主进程新增 GitHub PR 与子进程快照相关错误文案 【实证】

新版主进程/前端 bundle 新增了可读字符串 `Failed to list pull requests`、`Failed to collect child process snapshot` 与 `fsUtils.readdir requires AppServerClient`。

这些字符串确凿存在于新版 bundle，说明产物中新增了与拉取 pull requests、采集 child process snapshot、以及 AppServerClient 文件系统读取约束相关的错误路径。不过事实包只提供压缩 bundle 字符串，不能进一步还原完整调用链。

### 6. 代码高亮与数学渲染资源发生明显调整 【信号】

多个语言和渲染模块出现体积或 token 集变化：`katex` 增加 `263700 bytes`，`fortran-free-form` 增加 `87114 bytes`，`c` 增加 `72082 bytes`，`css` 增加 `48992 bytes`，`go` 增加 `46744 bytes`，同时 `jsx`、`tsx`、`javascript`、`php`、`markdown` 等模块减少了较大体积。

这指向编辑器/渲染层的语法高亮、MathML/KaTeX 或语法资源调整，但所有线索均来自同名前端模块与体积变化，无法从产物确认源码级重构意图。

### 7. Bluetooth 权限描述从旧键迁移到新键 【实证】

`Info.plist` 新增 `/NSBluetoothAlwaysUsageDescription = 'This app needs access to Bluetooth'`，同时移除 `/NSBluetoothPeripheralUsageDescription`，旧值同样为 `This app needs access to Bluetooth`。

这意味着 macOS 权限声明仍保留 Bluetooth 用途文案，但 plist 键从旧项迁移到 `NSBluetoothAlwaysUsageDescription`。这是直接可读的 Info.plist 变化。
