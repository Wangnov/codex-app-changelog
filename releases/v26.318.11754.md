# Codex 桌面版 `26.318.11754`(build 1100)

> 第三方变更日志 · 非官方 · 基于官方更新包逆向差异

这次 `26.317.21539` 到 `26.318.11754` 的变化非常克制：版本号和少量 bundle(应用包)内容更新之外，唯一明确可读的权限变化是蓝牙用途说明从旧键迁移到新键。

English one-liner: a small macOS build update with a Bluetooth privacy-key migration and no confirmed new app-level capability.

## 关于这份报告

这不是官方变更日志，而是一份**基于官方分发产物的逆向差异报告**。本次是历史回填：两个版本均超出官方 Sparkle(应用自动更新框架)增量窗口，因此事实包采用两个官方全量包直接比较，而不是增量包重建。

对比范围是 macOS arm64 版本，从 `26.317.21539`(build 1088，2026-03-19)到 `26.318.11754`(build 1100，2026-03-19)。两个包都给出了 SHA-256(哈希摘要)和系统公证验证结果；事实包显示两个版本的公证验证均返回 `CSSMERR_TP_CERT_REVOKED`。这说明相关早期签名证书后来被吊销，会影响系统安全安装判断，但不影响对 bundle 内容做差异分析。

本文使用两档证据等级：

- **【实证】**：来自 Info.plist(应用配置文件)、CSP(内容安全策略)、可读配置或明确 diff(差异)的直接证据。
- **【信号】**：来自前端模块名、二进制体积等间接证据，只能说明“指向某方向，但无法从产物还原到源码级意图”。

## ✨ 重点变化

### 1. 蓝牙权限描述迁移到新的 macOS 隐私键 【实证】

`Info.plist` 新增了 `NSBluetoothAlwaysUsageDescription`，值为 `This app needs access to Bluetooth`；同时移除了旧的 `NSBluetoothPeripheralUsageDescription`，旧值同样是 `This app needs access to Bluetooth`。

这意味着应用对蓝牙权限的系统描述键发生了迁移。可感知点主要在 macOS(苹果桌面操作系统)权限提示或系统隐私权限元数据层面；事实包没有提供进一步证据说明蓝牙相关功能本身发生了变化。

关键证据：

```diff
+ /NSBluetoothAlwaysUsageDescription = 'This app needs access to Bluetooth'
- /NSBluetoothPeripheralUsageDescription = 'This app needs access to Bluetooth'
```

### 2. 应用版本推进到 build 1100 【实证】

应用短版本号从 `26.317.21539` 更新到 `26.318.11754`，build 从 `1088` 更新到 `1100`。

这是明确的版本推进，但从当前事实包看，除权限键迁移和构建产物内容变化外，没有足够直接证据证明新增了用户可见功能。

关键证据：

```diff
CFBundleShortVersionString: 26.317.21539 → 26.318.11754
CFBundleVersion: 1088 → 1100
```

### 3. 前端入口与预加载资源发生构建更新，但不能据此判断新增功能 【信号】

`webview/index.html` 中大量 `assets/*.js` 与 CSS(层叠样式表)文件名发生变化，例如主入口从 `index-c0_FEK8V.js` 变为 `index-Dc10NZPO.js`。这些文件名带有构建 hash(内容指纹)，事实包也明确提示 `webview/assets` 下大量增减属于 Vite(前端构建工具)构建 hash 重命名噪音。

这指向前端包经历了一次重新构建，但无法从产物还原到源码级意图，也不能把这些 hash 文件名变化写成新增或删除功能。当前事实包中没有列出“前端真实新增模块”，因此这里不归纳为具体功能变化。

关键证据：

```diff
- ./assets/index-c0_FEK8V.js
+ ./assets/index-Dc10NZPO.js
```

### 4. 内容安全策略保持相同的外连范围 【实证】

`webview/index.html` 中 CSP 的 `connect-src` 仍然只放行 `self`、`https://ab.chatgpt.com` 和 `https://cdn.openai.com`。事实包没有显示新增外连域名。

这意味着从可读的页面安全策略看，本版没有证据表明新增了新的网络连接目标。

关键证据：

```html
connect-src 'self' https://ab.chatgpt.com https://cdn.openai.com
```
