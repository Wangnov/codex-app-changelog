本报告基于两份官方签名分发产物的逆向 diff：macOS 取 Sparkle 分发包，Windows 取 MSIX 分发包；两者发布时间略有错位，因此按“同一批次跨平台对照”处理，而不是强行视为同一构建号。

证据等级分两档：

- 【实证】：来自 plist、AppxManifest、包文件树、app.asar 清单、依赖清单、签名/公证校验等可读证据。
- 【信号】：仅来自前端模块名、体积变化或打包产物变化，只能说明方向，不能还原源码意图。

平台归属标记含义：

- `[macOS+Windows]`：两平台都观察到。
- `[仅 macOS]`：只在 macOS 包中观察到。

## ✨ 重点变化

### 【实证】[仅 macOS] macOS 本批次为零差异回填

macOS 两个全量包的下载地址相同，SHA-256 也相同：

`caa5b3ca8e22ca37cc47e594644627f1590f5edf2c7fa01d800b9ab31379090c`

文件树统计显示 8416 → 8416，新增 0、移除 0、内容改动 0。应用层 app.asar 内部文件同样是 3115 → 3115，新增 0、移除 0、改动 0。

平台差异上，macOS 没有观察到任何包内容变化；Windows 则有少量内容改动。

### 【信号】[仅 Windows] Windows 应用层 app.asar 有轻量刷新

Windows 包整体文件数保持 9216 → 9216，但内容改动 15。应用层 app.asar 内部文件保持 3078 → 3078，同时出现新增 66、移除 66、改动 12。

由于事实包提示 `webview/assets` 下大量增减属于 Vite 构建 hash 重命名噪音，不能据此判断新功能。因此这里仅能确认 Windows 应用层产物发生过轻量替换或重建，不能进一步断言具体功能变化。

平台差异上，这一变化只出现在 Windows；macOS 对应批次没有 app.asar 内容变化。

### 【实证】[macOS+Windows] Computer Use 运行时体积未变化

Computer Use 运行时，即事实包中的 `cua_node` 运行时，两平台都没有观察到总体积变化：

- macOS：365.9 MB → 365.9 MB，+0 MB，+0%。
- Windows：285.6 MB → 285.6 MB，+0 MB，+0%。

平台差异上，两平台的基准体积不同，但各自前后都没有增长或收缩。

### 【实证】[macOS+Windows] cua_node 依赖未新增

两平台的 `cua_node` 依赖变化均为新增 0 个包。

这意味着本批次没有可读证据显示 Computer Use 运行时依赖栈扩张；Windows 的应用层变化也没有伴随 `cua_node` 依赖新增。

### 【实证】[仅 Windows] Windows app.asar 体积几乎不变

Windows `resources/app.asar` 从 159.6 MB 到 159.6 MB，记录变化为 -0.00 MB。

这与“内部有少量文件替换，但整体体积基本持平”的观察一致。macOS 没有对应体积变化记录，因为 macOS 本批次内容改动为 0。
