# Codex 桌面版 26.623.141536 macOS / Windows 对照

> 第三方跨平台变更日志 · 非官方

这一批次显示，macOS 包内前端面更活跃，Windows 包内可见变化更集中。  
One-liner: macOS carries broad UI and workflow signals, while Windows mainly mirrors plugin version bumps and the thread panel update.

## 关于这份报告

本报告只使用输入事实包中的机器提取差异。macOS 对比范围是 `26.623.101652` build `4674` 到 `26.623.141536` build `4753`；Windows 对比范围是 `26.623.13972.0` 到 `26.623.19656.0`，其插件 manifest 中版本从 `26.623.101652` 更新到 `26.623.141536`。

证据等级分两档：【实证】来自 plist、manifest、CSP、依赖清单、插件清单、服务入口、技能文档、关键文件 diff，以及非“同名前端模块改动线索”的可读字符串；【信号】来自前端模块名、体积变化、压缩 bundle 的可读字符串和样式类名变化。平台归属用 `[macOS+Windows]`、`[仅 macOS]`、`[仅 Windows]` 标明。

## ✨ 重点变化

### 【实证】[macOS+Windows] 内置 browser 与 chrome 插件版本随包更新

两端都更新了 `browser` 和 `chrome` 内置插件 manifest 的 `version` 字段：从 `26.623.101652` 到 `26.623.141536`。描述文本未显示能力变化。

`browser` 插件描述继续覆盖 in-app browser、`@browser` / `@browser-use`、localhost / file URL、点击、输入、截图和前端变更后的本地目标打开等场景。`chrome` 插件描述继续强调使用用户现有 Chrome 状态，包括标签页、登录会话、cookies、扩展和 Codex Chrome Extension setup。

### 【实证】[仅 Windows] computer-use 插件 manifest 版本更新

Windows 包内 `resources/plugins/openai-bundled/plugins/computer-use/.codex-plugin/plugin.json` 的 `version` 从 `26.623.101652` 更新到 `26.623.141536`。描述仍是通过 Computer Use 从 Codex 控制 Windows 桌面应用。

macOS 事实包未给出 computer-use 插件 manifest 改动；macOS 的 `cua_node` 总量也保持 `377.4 MB → 377.4 MB`，Windows 为 `298.9 MB → 298.9 MB`。

### 【信号】[macOS+Windows] 线程页出现侧栏/底部面板入口变化

两端 `thread-app-shell-chrome` 都新增了同一组可读字符串，包括从 thread side panel 打开 side chat、browser tab、review tab、workspace file browser，以及从 panel New tab page 打开 terminal、History；还新增了 `Bottom panel`、关闭线程底部面板、从底部面板或侧面板再开一个 tab 的按钮文案。

这属于同名前端模块改动线索，不能断言功能已经对所有用户开放；但两端同时出现，说明线程 shell 的面板导航、tab 扩展和底部面板布局是本批次共同方向。

### 【信号】[仅 macOS] 新线程、项目、远程会话和 appgen 路由重组明显

macOS 的前端真实模块增删显示大量新组合模块进入包内，集中在 `new-thread-panel-page`、`remote-conversation-page`、`projects-index-page`、`worktree-init-v2-page`、`pull-requests-page`、`appgen-page`、`appgen-library-page`、`appgen-publication-terms-route`、`appgen-settings-page`、`plugin-detail-page`。

这不是单纯 hash 改名：事实包已剔除 hash 噪音后列出真实模块增删。可见方向是新线程面板、远程会话、项目索引、PR、worktree 初始化和 appgen 发布/库/设置的 chunk 边界发生重排。Windows 样本未给出对应真实新增模块清单。

### 【信号】[仅 macOS] 设置页扩大到热键、默认权限、工作模式、语音和外观细节

macOS `general-settings` 体积从约 `0.0 MB` 到 `0.1 MB`，新增字符串包括 disabled default permissions toggle、work mode settings options、code base size for chats and diffs、hold-to-dictate hotkey、hotkey window hotkey、dictation dictionary entry、Popout Window projectless default、ambient suggestions、color diff marker、chrome theme contrast slider。

这些是设置项进入前端 bundle 的信号，不等于设置已全部可见。Windows 事实包未显示对应设置模块变化。

### 【信号】[仅 macOS] Browser Use 任意网站权限确认文案进入权限面板

macOS `pending-request-item-panel` 新增 `Allow Codex to interact with any website without asking?`、`Allow any website`、确认/取消按钮、外部浏览器连接标题、风险警告和 `composer.browserWebsiteAccess.confirm.*` 键名。

这是高可见权限和安全拦截方向信号：Browser Use 的任意网站访问确认、风险提示和外部浏览器连接路径进入包内。Windows 样本未显示对应文案变化。

### 【信号】[仅 macOS] 自动化页转向远程项目、固定线程和归档任务

macOS `automations-page` 新增字符串包括 remote connection group/status loading、remote connection color picker、project type dialog、`Add your remote project here`、pin/unpin thread、`Archive all chats`、`Archive and remove`、`Archive chat and remove scheduled task?`。

同一模块移除了多条 pet 描述和 external agent import/onboarding 文案。证据边界是前端信号：可说明自动化页相关 UI 文案进入或退出包内，不能断言实际入口状态。

### 【信号】[仅 macOS] Onboarding 模块大幅增重并引入编辑器/PDF/可访问性相关线索

macOS `app-initial~app-main~onboarding-page` 从 `3.5 MB` 增至 `4.9 MB`，增加约 `1,393,567 bytes`。新增字符串包括 ProseMirror 相关提示、`style/prosemirror.css` 建议、`xfaNonInteractive input, .xfaNonInteractive textarea`、系统字体、强制颜色和深色偏好媒体查询；移除字符串包括 image attachment upload progress、reasoning effort slider、remove image attachment button、conversational onboarding task already starting。

这表明 onboarding chunk 内容重组且吸收了编辑器、PDF/XFA 或可访问性相关代码信号。它仍是 bundle 线索，不能当作源码级功能说明。

### 【信号】[仅 macOS] 代码高亮/语法资源发生再分布

macOS 多个语言 bundle 增加语法 scope：`typescript` +`181,117 bytes`、`php` +`111,110 bytes`、`c` +`72,155 bytes`、`haxe` +`35,206 bytes`、`java` +`27,256 bytes`、`stylus` +`31,111 bytes`、`gdshader` +`6,368 bytes`、`diff` +`2,611 bytes`。同时 `javascript`、`less`、`python`、`scss`、`postcss`、`go` 等模块减少并移除一批 scope 字符串。

这更像语法高亮资源拆分或升级的信号，而不是语言支持简单增减。Windows 样本未列出这些模块改动。

### 【信号】[仅 macOS] 数学渲染、YAML 和事件/插件错误路径有包内变化

macOS `katex` 增至约 `0.3 MB`，新增宏定义、控制序列、MathML、颜色字符串等错误和渲染相关文本。`chunk-XPW4576I` 新增 YAML/TAG directive 错误文本。`esm` 新增 `Callback Error`、`Context Cancel`、`Failed to deliver`、`Failed to load destination`、`plugin Error`，并移除 event bus 调试路径文本。

这些是运行时错误路径和解析器相关信号。由于来自同名前端模块线索，只说明包内文本变化。

### 【实证】[仅 macOS] CSP 本批次未显示策略扩展，仅入口资源 hash/预加载变更

macOS `webview/index.html` 的 Content-Security-Policy 仍包含 `default-src 'none'`，并保留 image、child/frame、worker、script、style、font、media、connect 的既有白名单形式。diff 中可见变化是入口脚本 hash 文件名和 modulepreload 资源调整，而非 CSP 规则新增。

Windows 事实包未提供 AppxManifest 或 CSP 差异。
