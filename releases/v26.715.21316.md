# Codex 桌面版 26.715.21316 macOS / Windows 对照

> 第三方跨平台变更日志 · 非官方

这一批次的共同主线，是把本地任务体验进一步统一为 Chat，并同步扩展项目、PR、自动化、浏览器、远程连接与应用内可视化的能力面。

Across both packages, this batch shifts local work toward a chat-centered model while expanding projects, pull requests, automations, browser workflows, remote connections, and inline visualization.

## 关于这份报告

本报告比较两个官方全量分发包区间：

- macOS arm64：`26.707.91948` build 5440 → `26.715.21316` build 5484。
- Windows x64 MSIX：`26.707.12708.0` → `26.715.2236.0`。

macOS 样本的两个全量包均通过公证验证，来源为 OpenAI OpCo, LLC。Windows 与 macOS 的版本号、文件统计和体积均分别引用，不能相互替代。

证据分为两档：

- 【实证】来自 plist、AppxManifest、技能文档、API 文档、配置、依赖文件、服务入口、签名结果及新增 bundle 可读字符串。字符串进入包内是确定事实，但不等于已向所有用户开放。
- 【信号】来自真实前端模块名、同名模块字符串差、样式类名差或压缩模块体积方向，只说明 UI、状态或交互正在变化。

平台标记为 `[macOS+Windows]`、`[仅 macOS]` 和 `[仅 Windows]`。Vite、webview、`.vite/build` 中单纯的 hash 改名已排除。输入没有列出新增内置插件清单；可见插件改动集中在既有 Browser、Chrome、Computer Use、Sites、Visualize 与 Record and Replay 内容。

## ✨ 重点变化

### 从 Task 向 Chat 统一命名与状态模型

【实证】[macOS+Windows]

两端主进程和前端 bundle 同步加入 `New Chat`、`New Chat in ChatGPT Sidebar`、`New standalone chat`、`Rename chat`、`Attach image to chat`、`Active local chats on this machine will be interrupted` 和 `Archiving inactive chats is only supported locally`；对应的 `New Task`、`Rename task`、`Attach image to task` 等字符串被移除。

`/review` 的提示也从“在当前 task 内执行或另开 review task”改成“在当前 chat 内执行或另开 review chat”。这是包内文案、命令路径和错误路径共同发生的迁移，不只是模块改名。

【信号】[macOS+Windows]

项目页、对话线程、归档、定时任务和 Codex Micro 的同名模块普遍把 task 改成 chat。项目搜索开始同时描述 ChatGPT chats 与本地 Codex tasks；过期 side task 改称 side chat；侧栏归档、固定和恢复状态也朝 chat 模型收敛。

### 项目页与 ChatGPT Project 的结合加深

【信号】[macOS+Windows]

两端 `projects-index-page` 出现相同的新线索：展开或折叠 ChatGPT project、加载更多 project chats、固定 project、打开 chat，以及从 ChatGPT project 发起本地 Codex task。搜索结果开始区分或合并匹配的 tasks 与 chats，并新增 `partial-error`、`aria-live` 等状态和无障碍线索。

首页建议模块同时加入“创建文件或网站”“新建文档、演示文稿、电子表格及模板”“新建网站”，并出现 `Google Calendar`、`Google Drive` 文案。这些是同名前端模块信号，不能据此断言连接器或入口已对所有用户启用。

【信号】[仅 Windows]

Windows 新增 `chatgpt-local-project`、`local-projects`、`composer-project-picker`、`composer-project-selector`、`project-default-icon`、`project-display-icon`、`select-project` 和侧栏 project group 模块。侧栏同名模块还加入 project 文件上传、`Add source`、`Advanced settings`、`Archive all chats` 等线索。

【信号】[仅 macOS]

macOS 的 AppGen 设置模块出现自定义域名、DNS 记录、站点标题、站点删除确认、环境变量和 secret 输入等字符串；这些仅能确认相关设置 UI 正在变化。

### PR 工作流从自动修复转向准备合并

【信号】[macOS+Windows]

`local-conversation-page` 两端都移除 `Auto-fix PR`，加入 `Prepare to merge`、`Preparation paused`、`Ready to merge`、自动满足要求后合并，以及暂停、恢复和执行合并的菜单文案。相关说明明确提到处理由该 PR 引起的合并冲突和失败检查。

`local-conversation-thread` 同时加入为已附加 PR 评论提供可选指导、保存或移除评论、从旧消息继续到当前工作区或新 worktree，以及让 Codex 检查项目并准备环境的线索。

【信号】[仅 macOS]

macOS `pull-request-route` 新增 PR 详情评论和回复编辑器、`Leave a comment`、`Post comment`、`Post reply` 等线索。

【信号】[仅 Windows]

Windows 的 thread side panel 出现“该分支已有 PR”、接受编辑以暂存、取消进行中的 Git 操作，以及更完整的 review、terminal、browser、workspace file browser 标签入口线索。

### 插件定时任务模板与自动化详情重构

【信号】[macOS+Windows]

两端 `automations-page` 都加入：

- 从插件打开 scheduled-task 模板；
- 打开提供该任务的插件；
- 将任务重置为插件模板；
- `Already using plugin defaults`；
- 关闭 scheduled-task 详情面板；
- 打开 heartbeat automation 对应 chat；
- `Create with Codex`。

旧的“通过与 ChatGPT 对话个性化模板”文案被移除。新增模块 `use-plugin-scheduled-tasks` 与上述线索互相印证，但仍不能证明所有插件模板已公开。

【信号】[仅 Windows]

Windows 还出现 `heartbeat-automation-eligibility` 的 chat 归档联动：归档带 heartbeat 的 chat 时，同时移除 scheduled task 并停止未来运行。

### 分块 IPC 与流式消息解析进入桌面桥接层

【实证】[macOS+Windows]

两端 `.vite/build/preload.js` 都加入 `codex-host-chunked-message-v1` 协议、transfer ID、序号、start/chunk/end 消息、ACK，以及数组、对象、键、值和分块字符串的重组器。

包内同时加入不匹配容器结束、多个根值、消息未完成、对象值缺少键、传输缺少结束部分等错误路径。该变化表明 renderer 与主进程之间已具备结构化大消息的分块传输与校验路径。

### Browser/Chrome 路由改为逐次优先发现专用工具

【实证】[macOS+Windows]

Browser 与 Chrome 的统一技能文档现在要求：对链接资源的每一次语义操作，都先查询可用及延迟加载的 connector、API 或 CLI；此前使用过浏览器，不会使后续操作自动保持 browser-first。只有专用工具不可用、无法访问资源、能力不足或仍需 UI 操作时，才使用 Browser。

插件 mention 现在明确选择浏览器：Browser mention 指向应用内浏览器，Chrome、chrome-internal 和 chrome-dev mention 指向 Chrome。若运行时选中的浏览器实际缺少认证，可在要求用户登录前尝试另一可用浏览器。

Browser 与 Chrome API 文档还移除了 file chooser 及其等待事件在 `iab` 中“默认不支持”的标记。这里只能确认限制标记从 API 文档中删除。

### Sites 发布确认与首屏文案调整

【实证】[macOS+Windows]

Sites Hosting 技能对共享或公开部署的处理更明确：必须调用 `request_user_input`，展示解析后的访问级别，例如 `Publish publicly` 或 `Publish to existing shared access`，同时提供 `Not now`；只有收到批准后才能调用 `deploy_site_version`。

Sites Building 模板测试把 `Codex is working` 改为 `Building your site`，首版提示改为准备完成后自动显示，并明确断言预览页不应包含 `Codex` 字样。

### Visualize 新增标准表格组件体系

【实证】[macOS+Windows]

Visualize 技能和 CSS 同步新增：

- `.table`：无外框语义表格、横向分隔、单元格换行；
- `.table-responsive`：窄屏无法容纳列时提供受控横向滚动；
- `.table-sm`：更紧凑的行距；
- `.text-end`、`.text-center`、`.text-nowrap`：表格对齐与不换行工具。

技能仍要求优先重排和换行，仅在表格列确实无法容纳时使用 responsive wrapper。

### Windows 终端退出与 ConPTY 进程清理调整

【实证】[macOS+Windows]

两个平台包内携带的 `node-pty` Windows 实现都发生相同变化：`WindowsTerminal.kill()` 会等待终端 ready 后再执行，并返回 agent kill 结果；`WindowsPtyAgent` 支持注入 `getConsoleProcessList`，将 ConPTY 终止和控制台子进程清理拆分为独立步骤，覆盖函数失败时回退到 inner PID。

这是 Windows 终端代码路径的实质修改；其同时存在于 macOS 包，是共享应用依赖被一并打包的结果。

### 本地环境继承与脚本显示方式调整

【信号】[macOS+Windows]

`local-environments-settings-page` 在两端都加入继承自父目录的本地环境、显示或隐藏继承项、来源目录与文件、编辑入口、平台脚本为空、项目加载失败和 `Environment needs attention` 等文案。

旧的 cleanup script 汇总、平台 override 和环境详情文案被移除。样式变化加入等宽字体、完整换行、较高的命令显示区域及屏幕阅读器文本。

### 远程连接从 SSH 专用描述抽象为通用连接

【信号】[macOS+Windows]

远程连接设置将多处 `SSH connection` 文案改成更通用的 `connection` 或 `remote host connection`，并加入 `Add a remote project`、可控制连接的设备列表，以及允许其他已授权设备远程控制的确认界面。

SSH 行仍作为打开远程项目对话框的入口之一，因此这是描述和能力边界的泛化信号，不代表 SSH 被移除。

### Codex Micro 增加旋钮模式与灯光控制

### Slack 链接、线程目录与下载错误路径进入包内

线程目录新增 host ID、CWD prefix、included/excluded thread ID、分页上限、游标所属 host、筛选与排序不匹配等校验字符串；文件路径则新增 `File download returned no response body`、`Temporary file is too large` 和原生下载历史读取失败。

### Computer Use 文档与安全规则分平台演进

【实证】[仅 macOS]

macOS Computer Use REPL 类型新增 `target: "mac"`。文档要求忽略上次 accessibility 文本时，下次检查必须获取完整树；AX 文本不可用或异常时改用截图、坐标和按键，并明确 `press_key`、`type_text` 不能触发全局快捷键。

确认政策被重写为 Hand-off、Action-time confirmation、Pre-approval 和 Not required 四档，细化凭据、永久访问、法律协议、软件来源、敏感信息、高影响沟通、普通交易和可恢复删除等边界。

【实证】[仅 Windows]

Windows Computer Use 的 `SKILL.md` 与 guidance 被大幅重写：入口固定为插件内 `scripts/computer-use-client.mjs`，要求完整读取技能文档，初始化时从 `list_apps()` 返回值严格选择唯一窗口，禁止猜测 app/window 字段，并采用“观察一轮、执行一个动作、立即刷新”的两单元循环。元素索引、截图 ID 和坐标只对生成它们的那次观察有效。

### Skysight 记忆输出加入结构化元数据

【实证】[仅 macOS]

Computer Use 与 Record and Replay 内的 Skysight 总结器要求在 Markdown 正文前输出固定 YAML frontmatter：

- `title`
- 单行两到三句的 `description`
- 去重后的 bundle identifier 数组 `applications`

10 分钟摘要使用事件记录中的精确 identifier，6 小时摘要从子摘要聚合，不得猜测。后续记忆整合必须忽略资源 frontmatter，只把 Markdown 正文作为证据。

同一修改放宽了此前对 URL、域名和 Slack 内容的绝对排除规则，但仍要求网页内容采用概括描述、不得输出 Markdown 链接，并继续禁止敏感信息和特权材料。

### 平台运行时与打包权限差异

【实证】[macOS+Windows]

Chromium 基线从 `150.0.7871.115` 更新到 `150.0.7871.124`：macOS 由 Info.plist、Framework 目录和 SCM revision 变化确认；Windows 由版本 manifest 替换及 `chrome.dll` 变化确认。

两端 `webview/index.html` 的 CSP 内容保持不变。允许范围仍包括自身资源、blob/data、Web Sandbox frame、Mapbox/OpenAI 连接与 ChatGPT WebSocket；本批次没有 CSP 扩权或收紧证据。

【实证】[仅 Windows]

Windows 串口 native binding 从 `serialport.node` 替换为 `bindings.node`；嵌套 `HID.node` 从 0.6 MB 增至 1.9 MB。输入未给出相应用户界面变化。
