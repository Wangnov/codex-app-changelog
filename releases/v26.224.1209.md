## 关于这份报告

这不是官方变更日志，而是一份**基于官方分发产物的逆向差异报告**。本次是历史回填：`26.217.1959` 与 `26.224.1209` 均已超出官方 Sparkle(自动更新框架) 增量窗口，因此事实包采用两个官方全量包直接比较，而不是增量包重建。

可信度方面，两个全量包都来自官方 `persistent.oaistatic.com` 下载地址，并给出了 SHA-256(文件指纹)；不过两版公证验证均返回 `CSSMERR_TP_CERT_REVOKED`。事实包说明这是部分早期版本签名证书后来被吊销导致，影响系统安全安装判断，但不影响对 bundle(应用包) 内容做差异分析。

本文使用两档证据等级：

- **【实证】**：来自人类可读的直接证据，例如 `Info.plist` 键值、`package.json` 差异、依赖清单或签名 / 公证验证结果。
- **【信号】**：来自间接证据，例如前端模块名或二进制体积变化；这类证据只能说明“指向某方向，但无法从产物还原到源码级意图”。

对比范围：`26.217.1959`(build 669，2026-02-18) → `26.224.1209`(build 697，2026-02-24)，平台为 macOS arm64(苹果芯片架构)。

应用版本从 `26.217.1959` 升至 `26.224.1209`，构建号从 `669` 升至 `697`。这意味着这是一个正式产物层面的新桌面版本，而不是单纯文件重打包。

### 2. 出现远程连接设置相关前端模块 【信号】

去掉 `webview/assets` 与 `.vite/build` 中 hash(构建指纹) 文件名噪音后，事实包识别到一个真实新增前端模块：`remote-connections-settings`。

这指向 Codex 桌面版可能在补充“远程连接设置”相关界面或配置入口，但仅凭模块名无法确认它是否已对用户开放，也无法还原源码级意图。

关键证据：

```text
新增模块: remote-connections-settings
```

### 3. 开发工具链从 Prettier 转向 oxfmt，并引入 oxlint 分层检查 【实证】

`package.json` 显示格式化命令从 Prettier(代码格式化工具) 改为 `oxfmt`，lint(代码检查) 命令也拆成 `oxlint` 与 ESLint(代码检查工具) 两层。

这主要是开发者与构建维护层面的变化，用户通常不会直接感知；但它说明桌面应用项目的代码质量检查流程发生了调整。

关键证据：

```diff
- "format": "prettier --check ."
- "format:fix": "prettier --log-level warn --write ."
+ "format": "oxfmt --check"
+ "format:fix": "oxfmt --write"

+ "lint:oxlint": "node ../scripts/run-tool-with-default-target.mjs oxlint src --tsconfig ./tsconfig.json --max-warnings 0 --type-aware --"
```

依赖层面也对应变化：

```diff
- "prettier": "^3.2.5"
- "prettier-plugin-organize-imports": "^4.3.0"
+ "oxfmt": "^0.34.0"
```

### 4. 加入 Playwright 相关开发入口 【实证】

本版新增 `playwright:agent:repl` 脚本，并在开发依赖中加入 `playwright`。这指向项目内部新增或强化了基于 Playwright(浏览器自动化工具) 的 Electron(桌面应用框架) 调试 / 自动化入口。

这属于开发与测试能力变化，不等同于用户侧新增浏览器自动化功能。

关键证据：

```diff
+ "playwright:agent:repl": "node ./scripts/playwright-electron-agent-cdp.mjs"
+ "playwright": "^1.58.2"
```

### 5. WebSocket 原生加速依赖被整理到运行依赖中 【实证】

`bufferutil` 与 `utf-8-validate` 出现在运行依赖区域；它们通常与 `ws` 的 WebSocket(网页长连接通信协议) 性能或校验路径相关。事实包显示这两个包不是本版新增包，而是依赖位置 / 顺序发生整理。

可感知层面不能据此断言连接性能提升，只能确认依赖清单结构发生变化。

关键证据：

```diff
+ "bufferutil": "^4.0.1"
+ "utf-8-validate": "^6.0.0"
```

同时事实包明确：

