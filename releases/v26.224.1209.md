# Codex 桌面版 `26.224.1209` build 697

> 第三方变更日志 · 非官方 · 基于官方更新包逆向差异

这一版不是大规模文件树变更，而是在现有应用中补入远程连接设置、会话分叉 / worktree、开源许可展示和主进程 RPC 相关能力线索。  
One-liner: a compact desktop update with clearer settings surfaces, stronger worktree-oriented flows, and new worker RPC diagnostics.

## 关于这份报告

这不是官方变更日志，而是一份**基于官方分发产物的逆向差异报告**。本次为历史回填：官方 appcast 未对 `26.217.1959` → `26.224.1209` 提供增量包，因此对两个官方全量包做直接比较。

对比范围为 macOS arm64，上一版 `26.217.1959` build `669`，发布于 `2026-02-18`；最新版 `26.224.1209` build `697`，发布于 `2026-02-24`。

证据等级分两档：

- **【实证】**：来自可读配置、`package.json` diff、Info.plist、主进程 bundle 新增可读字符串、可验证签名 / 公证输出等直接证据。
- **【信号】**：来自前端模块名、同名前端模块字符串 / 样式变化、体积变化等间接证据。它们指向某方向，但无法从产物还原到源码级意图。

## ✨ 重点变化

### 1. 新增远程连接设置模块 【信号】

前端去噪后出现一个真实新增模块：`remote-connections-settings`。这指向设置页中新增或拆分了远程连接相关入口，但仅凭模块名无法确认功能是否已完整对用户开放。

可感知点是：设置体系可能开始承载远程连接配置，后续版本中的连接管理、权限或目标环境选择可能会围绕这一入口展开。

证据：去掉 `webview/assets` hash 噪音后，前端真实新增模块为 `remote-connections-settings`。

### 2. 会话分叉、Git worktree 与项目执行环境文案扩展 【信号】

多个本地化前端模块新增了围绕 fork、local、worktree、branch、project name、execution environment 的文案。它们共同指向 Codex 在会话继续方式上引入或强化了“本地继续”和“新 worktree 继续”的选择。

可感知点是：开发者可能会看到更明确的会话分叉、创建分支、创建永久 worktree、把 worktree 保存为项目等流程提示；但这些证据来自同名前端模块改动，只能视作 UI / 状态方向信号。

关键线索包括：`composer.forkSlashCommand.title`、`composer.forkSlashCommand.option.local.title`、`composer.forkSlashCommand.option.worktree.title`、`Create branch` 的多语言对应文案，以及 `Apply requires a Git repository`。

### 3. 设置页加入开源许可与第三方 notices 入口 【信号】

`agent-settings` 前端模块新增了 `Open source licenses`、`Third-party notices for bundled dependencies`、`settings.openSourceLicenses.view` 等字符串，同时移除了部分直接引导打开 `config.toml` 的文案。

可感知点是：设置页可能新增了查看开源许可和第三方 notices 的入口，并对配置警告的展示方式做了调整。由于证据来自同名前端模块改动，这仍只能标为设置 UI 方向信号。

关键线索：`Button label to open the open source licenses page`、`Label for the open source licenses row`、`settings.agent.configuration.notice.fileContext`。

### 4. 构建与代码质量工具链从 Prettier 转向 oxfmt，并引入 oxlint 分层 lint 【实证】

`package.json` 显示格式化命令由 `prettier --check .` / `prettier --write .` 切换为 `oxfmt --check` / `oxfmt --write`，lint 脚本也拆成 `lint:oxlint` 与 `lint:eslint` 两段执行。

这对最终用户不可直接感知，但对开发流程有明确影响：桌面端项目的格式化和静态检查链路变得更偏向 `oxfmt` / `oxlint`，同时仍保留 ESLint。

关键 diff：

```diff
- "format": "prettier --check .",
- "format:fix": "prettier --log-level warn --write .",
+ "format": "oxfmt --check",
+ "format:fix": "oxfmt --write",
+ "lint": "pnpm run lint:oxlint && pnpm run lint:eslint",
```

### 5. 加入 Playwright 相关开发入口 【实证】

`package.json` 新增 devDependency `playwright`，并新增脚本 `playwright:agent:repl`，入口为 `node ./scripts/playwright-electron-agent-cdp.mjs`。

这指向桌面端内部开发 / 调试链路增加了面向 Electron agent 的 Playwright CDP REPL 能力，但事实包没有给出用户界面或正式功能入口。

关键 diff：

```diff
+ "playwright:agent:repl": "node ./scripts/playwright-electron-agent-cdp.mjs",
+ "playwright": "^1.58.2",
```

### 6. 主进程 worker RPC 增加错误路径与诊断字符串 【实证】

新版主进程 / 前端 bundle 中新增了 worker RPC 相关可读字符串，包括 `Mismatched main RPC response method`、`No main RPC handler configured for worker `、`Received main RPC request for wrong worker`、`Worker main RPC client disposed`。

这意味着产物中确实出现了新的主进程 RPC 错误处理或诊断路径。由于仍是压缩 bundle 字符串，不能还原完整源码逻辑，但新增字符串本身是直接证据。

### 7. WebSocket 原生加速依赖进入运行依赖 【实证】

`package.json` 的 dependencies 中出现 `bufferutil` 与 `utf-8-validate`，并与 `ws` 同处运行依赖区。它们通常服务于 WebSocket 数据处理路径，但事实包只证明依赖被纳入，并不证明具体性能收益。

可感知点可能体现在连接类路径的底层处理能力上；是否对用户体验有直接影响，差异包无法确认。

关键 diff：

```diff
+ "bufferutil": "^4.0.1",
+ "utf-8-validate": "^6.0.0",
```
