## ✨ 重点变化

### 1. 蓝牙权限描述键从 Always 切到 Peripheral 【实证】

`Info.plist` 中移除了 `/NSBluetoothAlwaysUsageDescription`，并新增 `/NSBluetoothPeripheralUsageDescription`，两者文案均为 `This app needs access to Bluetooth`。

这意味着应用声明蓝牙访问用途的 macOS 权限键发生了调整。可感知点可能体现在系统权限提示或权限审查路径上，但事实包没有提供运行时行为，因此不能进一步推断蓝牙功能本身发生变化。

关键证据：

```diff
- /NSBluetoothAlwaysUsageDescription = 'This app needs access to Bluetooth'
+ /NSBluetoothPeripheralUsageDescription = 'This app needs access to Bluetooth'
```

### 2. 版本号与构建号更新 【实证】

应用元数据从 `26.324.21329` / build `1214` 更新到 `26.324.21641` / build `1228`。

这是本次差异中最明确的发布层变化：同一平台、相邻日期发布，build 数字上升 14。

关键证据：

```diff

```

### 3. app.asar 内容发生变化，但未显示新增依赖或文件树扩张 【实证】

`ElectronAsarIntegrity` 中 `Resources/app.asar` 的哈希发生变化，说明应用层打包内容确实改变；但 bundle 文件总数仍为 `415 → 415`，cua_node 新增依赖为 `0` 个。

这更像是一次小范围应用层更新，而不是能力大扩展。事实包没有给出可读的功能 diff，因此不能把 `app.asar` 内部 hash 文件名变化解读为新增功能。

关键证据：


### 4. 没有新增 cua_node 依赖 【实证】

事实包显示 `cua_node` 新增包数量为 `0`。这意味着从依赖清单层面看，本版没有引入新的本地 Node(服务器端 JavaScript 运行环境) 能力包。
