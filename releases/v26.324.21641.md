# Codex 桌面版 `26.324.21641` build 1228

> 第三方变更日志 · 非官方 · 基于官方更新包逆向差异

这一版从可读产物看是一个很小的 macOS 维护更新:最明确的变化是蓝牙权限声明键从 Always 调整为 Peripheral。  
English one-liner: a small macOS maintenance build with a confirmed Bluetooth permission declaration change and no new `cua_node` packages.

## 关于这份报告

这不是官方变更日志,而是一份**基于官方分发产物的逆向差异报告**。本次为历史回填:事实包显示官方 appcast 未对 `26.324.21329` → `26.324.21641` 提供增量包,因此对两个官方全量包做直接比较。

对比范围是 macOS arm64 的 `26.324.21329` build `1214` 与 `26.324.21641` build `1228`。证据分为两档:

- **【实证】**: 来自 Info.plist 键值、依赖清单、签名/验证结果、可复现下载信息等人类可读直接证据。
- **【信号】**: 来自同名前端模块改动线索或体积变化,只能指向某方向,但无法从产物还原到源码级意图。

`webview/assets`、`.vite/build` 下带 hash 的文件名变化视为构建产物重命名噪音,不作为功能新增或删除依据。

## ✨ 重点变化

### 1. macOS 蓝牙权限声明从 Always 调整为 Peripheral 【实证】

新版 `Info.plist` 新增了 `NSBluetoothPeripheralUsageDescription`,同时移除了旧的 `NSBluetoothAlwaysUsageDescription`,两者文案均为 `This app needs access to Bluetooth`。

这意味着应用对 macOS 蓝牙权限的声明粒度发生了变化。用户可感知点可能体现在系统权限说明或权限请求路径上,但事实包没有提供运行时行为,因此不推断具体交互变化。

关键证据:

```diff
+ /NSBluetoothPeripheralUsageDescription = "This app needs access to Bluetooth"
- /NSBluetoothAlwaysUsageDescription = "This app needs access to Bluetooth"
```

### 2. 版本号、build 与 app.asar 完整性哈希更新 【实证】

`CFBundleShortVersionString` 从 `26.324.21329` 更新到 `26.324.21641`,`CFBundleVersion` 从 `1214` 更新到 `1228`。`ElectronAsarIntegrity` 中 `Resources/app.asar/hash` 也同步变化。

这说明新版确实包含应用层内容变更,但哈希本身只能证明内容不同,不能说明具体功能意图。

关键证据:

```diff
/CFBundleShortVersionString: "26.324.21329" → "26.324.21641"
/CFBundleVersion: "1214" → "1228"
/ElectronAsarIntegrity/Resources/app.asar/hash:
  1f5295b6f6f4570f8bad796cb98e29648c731d08a036ac73dec1f1a1b5c28b75
  → efec82927b30237ee372d382d8b7cec7b3249411f473ae1c6952a630cf0c7218
```

### 3. 前端主入口仅出现极小字符串替换线索 【信号】

同名 `index` 前端模块发生了内容变化,体积从 `4.1 MB` 到 `4.1 MB`,减少 `149 bytes`。可读字符串差异仅体现为 `bd` 到 `xd` 一类内部符号变化。

这指向前端构建产物中存在轻微逻辑或压缩后符号变化,但无法从产物还原到源码级意图,也不能据此判断有用户可见功能新增。

关键证据:

```text
webview/assets/index-CT1EpLpN.js → webview/assets/index-CsW2954H.js
4.1 MB → 4.1 MB (-149 bytes)
新增字符串: :for(i in xd(, :for(s in xd(
移除字符串: :for(i in bd(, :for(s in bd(
```

### 4. 未见新增 cua_node 依赖或 bundle 文件增减 【实证】

事实包显示 `cua_node` 新增依赖为 `0` 个;bundle 文件数保持 `415 → 415`,新增 `0`、移除 `0`、内容改动 `40`。

这意味着本版没有表现出新增本地能力包或大规模 bundle 结构调整的直接证据。应用层内部存在 hash 文件名变动,但事实包已标注这类变化主要受 Vite 构建 hash 重命名影响,不应解读为真实功能增删。
