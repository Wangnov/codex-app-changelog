# Codex 桌面版 26.527.60818 macOS / Windows 对照

> 第三方跨平台变更日志 · 非官方

这一批次看起来是一次跨平台代码同源、打包版本错位的增量：共同变化集中在插件版本、首页入口与分享体验，Windows 包里还露出更多本地自动化、浏览器权限和环境修复相关界面线索。  
One-liner: shared plugin/version and home/share UI updates, with Windows-only signals around Computer Use, browser controls, debug state, and worktree recovery.

## 关于这份报告

本报告只基于两份官方签名分发包逆向重建后的机器 diff。macOS 对比范围是 `26.527.31326 build 3390` → `26.527.60818 build 3437`；Windows 对比范围是 `26.527.3686.0` → `26.527.7698.0`。两平台版本号与 build 不互相套用。

证据等级分两档：【实证】来自 plist、manifest、插件配置、可读字符串、文件统计、依赖清单或明确 diff；【信号】来自同名前端模块、体积变化、样式类名和压缩 bundle 线索。平台归属用 `[macOS+Windows]`、`[仅 macOS]`、`[仅 Windows]` 标注。

已按输入区块盘点：版本与渠道、Bundle 文件树、内置插件变化、cua_node 依赖、app.asar、同名前端模块、可读字符串、plist/manifest、关键文件 diff。事实包没有提供独立的 Computer Use 运行时总量、内容安全策略变化、新增组件与资源明细；因此这些项不扩展推断。

## ✨ 重点变化

### 【实证】[macOS+Windows] 内置 Browser 与 Chrome 插件版本同步到 26.527.60818

两平台的 `browser` 与 `chrome` 插件 manifest 都把版本从 `26.527.31326` 更新到 `26.527.60818`。Browser 插件说明仍明确覆盖 in-app browser、本地 URL、文件 URL、截图、点击、输入和 localhost 打开等场景；Chrome 插件说明仍面向远程 URL、登录态页面、现有标签页、cookie、扩展和 Codex Chrome Extension 设置。

平台差异：macOS 事实包只显示 Browser 与 Chrome 两个插件 manifest 版本变化；Windows 还额外显示 Computer Use 插件版本变化。

### 【实证】[仅 Windows] Computer Use 插件 manifest 跟随版本更新

Windows 包中的 `resources/plugins/openai-bundled/plugins/computer-use/.codex-plugin/plugin.json` 从 `26.527.31326` 更新到 `26.527.60818`，描述为通过 Computer Use 从 Codex 控制 Windows 桌面应用。

平台差异：macOS 事实包没有对应 Computer Use 插件 manifest 改动条目；不能据此断言 macOS 没有能力变化，只能说本输入样本未给出对应实证。

### 【信号】[macOS+Windows] 首页公告新增 Fast mode、远程连接、连接应用与多 agent 作业入口文案

两平台 `home-announcements` 模块同名改动一致，新增可读字符串包括 Fast mode 首页 banner、backend-driven Codex app home banner、multi-agent composer banner、remote connections banner、`Add Connections`，以及 “Codex suggests what to do next by searching project files and connected apps” 和 “Delegate work to subagents that work in parallel. Note: may increase token usage.”

这说明首页层面进入包内的提示与入口信号覆盖：快速模式、远程连接、连接应用的 ambient suggestions，以及并行 subagents。证据边界是前端模块信号，不能断言所有用户已经可见或默认开启。

### 【信号】[macOS+Windows] App/site 分享体验加入更完整的访问范围与复制链接文案

两平台 `appgen-share-dialog` 模块都新增了站点分享相关字符串：添加 people/groups、workspace link、internet access、`Can visit`、复制 live site URL、外部打开 live site、保存中状态和自定义访问策略。  

平台差异：macOS 和 Windows 都出现同类信号，Windows 旧文件名不同但新增字符串集合一致。它指向 appgen/live site 分享和权限 UI 的扩展，但仍属于前端模块信号。

### 【信号】[仅 macOS] 插件详情页新增安装、启用、共享与可访问性线索

macOS `plugin-detail-page` 模块体积增加约 `130713 bytes`，新增字符串包括 `Add to Codex`、`Adding to Codex`、`App enabled`、`App disabled`、`App unavailable`、MCP server enablement toggle、plugin share dialog、workspace link、更多操作菜单和 section navigation 的可访问标签。

平台差异：Windows 事实包没有同名 `plugin-detail-page` 改动线索。该条只能作为 macOS 输入样本中的前端信号，不能升级为插件系统实证。

### 【信号】[仅 Windows] 浏览器使用设置加入历史访问、域名 allow/block 与上传下载域控制文案

Windows `browser-use-settings` 模块增加约 `59354 bytes`，新增字符串包括 `Access history without asking`、embedded extension/password manager/site settings、显示/隐藏浏览器数据清理选项、不可用插件的 disabled toggle，以及 allowed/blocked domain、download domain、upload domain 的添加入口。

这是一组高可见浏览器权限和站点控制信号。平台差异：macOS 事实包没有对应模块线索。

### 【信号】[仅 Windows] 调试面板扩展到账号、会话、浏览器标签、App Actions 与自动 worktree cleanup

Windows `debug-modal` 模块增加约 `171302 bytes`，新增字符串包括 `(command unavailable)`、`Account ID`、`Active browser tab`、`Active conversation`、`Ambient suggestion chats`、`App Actions`、`App Server`、`App session ID`，以及 `Allow automatic worktree cleanup in this unpackaged build.`

这指向调试界面覆盖运行状态、会话状态、浏览器状态、App 服务状态和开发/未打包构建的 worktree 清理开关。证据边界仍是同名前端模块信号。

### 【信号】[仅 Windows] @ mention 列表新增 Computer Use、桌面应用、agents 与 app capability 分组

Windows `at-mention-list` 模块新增 `Computer use`、`Desktop apps`、`Mac apps`、`Custom agents`、`Live agents`、Computer Use 标签说明，以及 app capability mention server/scope 的分组标题。  

这说明输入提及菜单中出现能力、应用和 agent 分组的界面信号。由于它同时提到 `Mac apps`，不能从 Windows 包反推出 macOS UI 已更新；只能记录为 Windows 包内字符串信号。

### 【信号】[仅 Windows] worktree 初始化失败后的继续、修复与环境设置流程更明确

Windows `worktree-init-v2-page` 模块新增 `Continue anyway`、`Edit environment`、`Fix this project`、取消 worktree 创建、设置失败后继续启动线程、打开本地环境设置、启动修复线程、创建 worktree 并运行 setup、fork conversation 等文案。

这是一组本地环境初始化失败后的恢复路径信号。平台差异：macOS 事实包没有对应模块改动线索。

### 【信号】[macOS+Windows] Composer 相关样式状态有小幅收敛

两平台 `composer` 模块均移除 `data-tooltip-overflow-target`、`h-token-button-composer`、`model-provider` 样式/状态类名，体积变化均为 `-1442 bytes`。  

这只能说明 composer bundle 的内部状态或样式结构有轻微调整，不能据此推断功能删除。

### 【信号】[仅 macOS] thread side panel 与 markdown 模块出现较大删减线索

macOS `thread-side-panel-tabs` 体积减少约 `291755 bytes`，移除字符串涉及 CI 诊断提示、外部日志、权限/凭据检查、已有 PR、接受编辑并暂存等；`markdown` 体积减少约 `63388 bytes`，移除字符串涉及 KaTeX annotation、文件引用 chip、图片/视频 alt 缺失、加载失败与预览触发器等。

这些是 macOS 前端 bundle 中的删减信号。由于事实包没有源码级配置或 manifest 佐证，不能写成相关功能被删除。
