# Codex 桌面版 `26.527.60818`(build 3437)

> 第三方变更日志 · 非官方 · 基于官方更新包逆向差异

这次从 `26.527.31326` 到 `26.527.60818` 的变化非常收敛：可读证据主要集中在版本号与内置插件版本同步，底层 `codex` 组件体积增加。  
English one-liner: a narrow build bump with bundled plugin version alignment and a slightly larger core binary.

## 关于这份报告

这不是官方变更日志，而是一份**基于官方分发产物的逆向差异报告**。本次为历史回填：两个版本均超出官方 Sparkle(应用自动更新框架) 增量窗口，因此事实包采用两个官方全量包直接比较，而不是增量包重建。

可信度方面，两个对比产物均来自官方下载地址，并通过 Apple(苹果) 公证与 OpenAI Developer ID(开发者身份签名) 验证。上一版与最新版的 `spctl`(macOS 安全评估工具) 结果均为 `accepted`，来源均为 `Notarized Developer ID`，签名来源均为 `Developer ID Application: OpenAI OpCo, LLC (2DC432GLL2)`。

本文使用两档证据等级：

- **【实证】**：来自人类可读的直接证据，例如 `Info.plist`、插件 `plugin.json`、签名与公证输出。
- **【信号】**：来自间接证据，例如二进制体积变化。它只指向某方向，但无法从产物还原到源码级意图。

对比范围：`26.527.31326` build `3390`，发布于 `2026-05-30`；对比到 `26.527.60818` build `3437`，发布于 `2026-06-01`。平台为 macOS arm64(苹果芯片架构)。

## ✨ 重点变化

### 1. 桌面应用版本推进到 `26.527.60818` 【实证】

应用的 `Info.plist`(macOS 应用元信息文件) 中，短版本号从 `26.527.31326` 更新到 `26.527.60818`，build 从 `3390` 更新到 `3437`。

这意味着这是一个正式签名并公证的新桌面版本，而不是只改动内部资源的重打包。

关键证据：

```diff
CFBundleShortVersionString: 26.527.31326 → 26.527.60818
CFBundleVersion: 3390 → 3437
```

### 2. 内置 Browser 插件版本与主应用同步 【实证】

`browser` 插件的 `plugin.json` 中，插件版本从 `26.527.31326` 更新为 `26.527.60818`。该插件描述仍指向 Codex 内置浏览器能力，包括打开、检查、导航、测试、点击、输入和截图本地 Web(网页) 目标，例如 `localhost`、`127.0.0.1`、`::1` 与 `file://`。

可感知点是：本版确认继续携带并同步维护内置 Browser(内置浏览器) 插件；但事实包只显示版本同步，没有显示插件能力文本发生变化。

关键证据：

```diff
"name": "browser",
"version": "26.527.31326" → "26.527.60818"
```

### 3. 内置 Chrome 插件版本与主应用同步 【实证】

`chrome` 插件的 `plugin.json` 中，插件版本同样从 `26.527.31326` 更新为 `26.527.60818`。其描述仍指向 Chrome(谷歌浏览器) 自动化能力，覆盖远程 URL、需要登录或用户资料的页面、已有标签页、cookie(浏览器会话数据)、扩展以及 Codex Chrome Extension(浏览器扩展) 设置。

这说明本版仍将用户已有 Chrome 状态相关的自动化能力作为内置插件分发；但从可读 diff 看，变化只体现在版本号。

关键证据：

```diff
"name": "chrome",
"version": "26.527.31326" → "26.527.60818"
```

### 4. 底层 `codex` 组件体积增加约 `2.81 MB` 【信号】

`Contents/Resources/codex` 从 `198.6 MB` 增至 `201.4 MB`，变化为 `+2.81 MB`。

这是一个间接信号，指向底层可执行组件或其打包内容有所变化，但无法从产物还原到源码级意图；事实包没有给出可读 diff，因此不能进一步断言新增了具体功能。

关键证据：

```text
Contents/Resources/codex: 198.6 MB → 201.4 MB (+2.81 MB)
```

### 5. 没有文件树级新增 / 移除，也没有新增 `cua_node` 依赖 【实证】

Bundle(应用包) 文件数保持 `1982 → 1982`，文件树层面新增 `0`、移除 `0`、内容改动 `51`。`cua_node` 依赖新增数量为 `0`。

这意味着本版更像是一次局部更新，而不是大规模新增组件或依赖扩张。`app.asar` 内部虽然有 `216` 个新增与 `216` 个移除，但事实包明确说明 `webview/assets` 下大量变动是 Vite(前端构建工具) hash(内容指纹) 重命名噪音，不能据此判断新功能。
