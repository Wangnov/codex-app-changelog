# Codex 桌面版 26.527.60818(macOS / Windows 对照)

> 第三方跨平台变更日志 · 非官方

本批次显示 OpenAI (人工智能公司) Codex (代码智能体桌面应用) 桌面版在 macOS (苹果桌面操作系统) arm64 (苹果芯片架构) 与 Windows (微软桌面操作系统) x64 (64 位 Windows 架构) 上同步发生了底层二进制体积增长和应用层资源重建，但可读配置证据主要出现在 macOS 包内。  
One-liner: this cross-platform batch looks like a binary/runtime refresh, with macOS exposing plugin version bumps that the Windows fact package does not show.

## 关于这份报告

本文基于两份官方签名分发产物的逆向 diff (差异比较):macOS 取 Sparkle (macOS 应用更新框架) 全量包，Windows 取 MSIX (Windows 应用安装包格式) 全量包，各自与上一版比较。macOS 对比范围是 26.527.31326 build 3390 → 26.527.60818 build 3437；Windows 对比范围是 26.527.3686.0 → 26.527.7698.0。本批次发布日期按输入事实包统一记录为 2026-05-30。

可信度分两档:【实证】表示来自 plist (苹果属性列表配置)、AppxManifest (Windows 应用清单)、技能文档、类型声明、package.json (Node.js 包配置文件)、依赖清单等可读证据；【信号】表示只来自前端模块名、体积变化或构建产物变化，只能说明方向，不能还原源码意图。

平台归属标记含义如下:[macOS+Windows] 表示两平台事实包都出现了相同主题证据；[仅 macOS] 表示仅 macOS 事实包可见；[仅 Windows] 表示仅 Windows 事实包可见。webview/assets 与 .vite/build 的 hash (哈希) 重命名已按事实包要求视为噪音，不作为功能增减依据。

## ✨ 重点变化

### Codex 主二进制继续增大【信号】[macOS+Windows]

两平台的核心 Codex 可执行组件都有明显体积增长。macOS 的 `Contents/Resources/codex` 从 198.6 MB 增至 201.4 MB，增加 2.81 MB；Windows 的 `resources/codex.exe` 从 253.5 MB 增至 258.0 MB，增加 4.51 MB，同时 `resources/codex` 从 223.2 MB 增至 226.7 MB，增加 3.48 MB。

这属于跨平台共同信号，说明本批次核心二进制内容发生更新。但事实包没有给出可读源码或符号级解释，因此不能进一步断言具体功能、模型逻辑或运行时行为变化。

### 应用层 app.asar 被重建，但功能含义不明确【信号】[macOS+Windows]

两平台的 app.asar (Electron 应用归档格式) 内部文件数量都保持不变，同时出现大量新增/移除配对。macOS 内部文件为 2774 → 2774，新增 216、移除 216、改动 12；Windows 内部文件为 2722 → 2722，新增 216、移除 216、改动 11。

事实包明确提示 webview/assets 下大量增减属于 Vite (前端构建工具) 构建 hash 重命名噪音。因此这里更适合解读为应用层构建产物重打包/重建的信号，而不是可确认的新功能列表。

### macOS 内置 Browser/Chrome 插件版本号同步到新版【实证】[仅 macOS]

macOS 包内的 `openai-bundled` 插件配置出现可读 diff。Browser (内置浏览器自动化插件) 的 `plugin.json` 版本从 `26.527.31326` 更新为 `26.527.60818`；Chrome (谷歌浏览器) 自动化插件的 `plugin.json` 版本也从 `26.527.31326` 更新为 `26.527.60818`。

这是本批次 macOS 侧最明确的应用层实证变化。Windows 事实包没有给出对应插件配置文件 diff，因此只能说 macOS 已有可读证据显示插件版本同步；不能据此断言 Windows 包内同一插件配置也完成了同样更新。

### 平台清单/版本元数据更新【实证】[仅 macOS]

macOS 的 `Info.plist` 显示 `CFBundleShortVersionString` 从 `26.527.31326` 更新为 `26.527.60818`，`CFBundleVersion` 从 `3390` 更新为 `3437`，并且 `ElectronAsarIntegrity/Resources/app.asar/hash` 从 `7b24b4099e5ac419d3b02d8928ab1422f8d1b650cb51a1af754e8bc2f4ff6e85` 变为 `63aa85903e8cd29e881fc2d098bc169fd9e7dce32532dee952b45e776338c0dc`。

Windows 事实包给出了版本对比 26.527.3686.0 → 26.527.7698.0，但没有列出 AppxManifest 或等价清单字段 diff。因此这一条按可读清单证据归为仅 macOS 实证；Windows 版本变化只在原始版本信息中记录。

### 依赖层没有新增 cua_node 包【实证】[macOS+Windows]

两平台事实包的 `cua_node` 依赖变化均显示新增 0 个包。也就是说，在这两份包的机器提取结果中，没有看到 `cua_node` 依赖新增。

这是一条跨平台一致的负向实证:本批次的可见变化不表现为新增 `cua_node` 包。

### Windows 辅助可执行文件有轻微体积变化【信号】[仅 Windows]

Windows 包中除核心 `codex.exe` 和 `codex` 外，`resources/codex-command-runner.exe` 从 1.2 MB 到 1.2 MB，变化 +0.01 MB；`resources/codex-windows-sandbox-setup.exe` 从 8.4 MB 到 8.4 MB，变化 +0.00 MB。

这仅说明 Windows 包内相关辅助可执行文件内容发生了机器可见变化。事实包没有提供可读 diff，因此不能确认命令运行器或 Windows 沙箱设置行为有何具体改变。
