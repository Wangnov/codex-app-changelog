# Codex 桌面版 `26.311.21342` build 993

> 第三方变更日志 · 非官方 · 基于官方更新包逆向差异

这一版从产物层面看，不是单点功能更新，而是围绕远程连接、设置体系、自动化和代码阅读体验的一次密集铺垫。  
One-liner: this build broadens remote/Git workflow paths and reshapes several UI surfaces around settings, Apps, automations, and code rendering.

## 关于这份报告

这不是官方变更日志，而是一份**基于官方分发产物的逆向差异报告**。本次对比范围是 macOS arm64 的 `26.309.31024` build 962 到 `26.311.21342` build 993。

本版属于历史回填：官方 appcast 未对这一对版本提供 Sparkle 增量包，因此事实包改为下载两个官方全量包直接比较。报告只使用事实包中可复现的 bundle 差异、配置 diff、依赖变化、可读字符串和体积统计。

证据等级分两档：**【实证】**表示存在人类可读的直接证据，例如 `package.json`、`Info.plist`、preload bridge、CSP、类型/配置文本或新增字面量；**【信号】**表示只来自前端模块名、同名前端模块字符串差异或体积变化，指向某方向，但无法从产物还原到源码级意图。

## ✨ 重点变化

### 1. 远程连接、SSH 与 Git/GitHub 工作流错误路径扩展 【实证】

应用层新增了 `ssh-config` 依赖，主进程/前端 bundle 里也出现多条远程 SSH、worktree、upstream branch、GitHub CLI 和 PR 生成相关字面量。

这意味着远程连接失败、同步分支状态、GitHub CLI 响应异常、PR 标题/正文生成等路径在桌面端产物中有了更明确的处理痕迹。关键证据包括：`package.json` 新增 `ssh-config` `^5.0.3`，以及 `Failed to connect remote SSH connection.`、`No synced branch config found for the current worktree.`、`Failed to parse GitHub CLI response`、`Unexpected GitHub CLI response`、`You are a helpful assistant. Generate a pull request title and body.`。

### 2. 自动化运行和周期日程解析路径出现 【实证】

新版 bundle 新增了自动化运行、模型读取失败和周期规则解析相关字面量。

可感知点可能是自动化任务失败和日程配置错误时有更明确的内部错误路径。直接证据包括 `Failed to load models for automation run`、`Invalid UNTIL value: `、`Invalid frequency: `、`Invalid weekday string: `、`Unsupported RFC prop `、`unsupported EXRULE parm: `、`unsupported RRULE parm: `。

### 3. 系统主题 variant 进入 Electron preload bridge 【实证】

`preload.js` 新增了系统主题 variant 的同步读取和订阅接口：`codex_desktop:get-system-theme-variant`、`codex_desktop:system-theme-variant-updated`、`getSystemThemeVariant`、`subscribeToSystemThemeVariant`。

这表示渲染层现在可以读取并监听系统主题 variant，给运行时主题切换和外观设置提供了更直接的桥接能力。

### 4. 设置里的外观、连接和 Apps 信息架构在重排 【信号】

新增模块名 `appearance-settings`、`remote-connections-settings`、`settings-row`、`app-server-connection-state`、`use-is-apps-enabled`，同名前端模块里还出现 `settings.nav.appearance`、`settings.nav.connections`、`All Apps`、`All skills`、`App enabled`、`App disabled`、`Choose Plugins, Skills, Apps, or a plugin marketplace` 等字符串。

这些线索指向设置页正在拆出外观和连接分区，同时 Skills/Plugins/Apps 入口也在整合；但这些都来自前端模块和同名模块改动线索，只能说明 UI、状态或交互方向，无法从产物还原到源码级意图或确认完整开放范围。

### 5. Markdown/内容渲染新增图像预览、Mermaid 和可访问性线索 【信号】

`markdown` 同名前端模块体积从 0.0 MB 增至 0.5 MB，新增字符串包括 `ARIA label for rendered mermaid diagrams`、`Accessible label for the image preview dialog`、`Aria label for closing the image preview dialog`。`webview/index.html` 的预加载清单中也出现 `math`、`katex`、`markdown` 等资源。

这些信号指向 Markdown 渲染层在图像预览、Mermaid、数学内容或可访问性方面有扩展，但无法从产物还原到源码级意图。

### 6. Diff 与多语言语法高亮包有明显调整 【信号】

`diff` 模块新增 `markup.changed.diff`、`markup.deleted.diff`、`markup.inserted.diff`、`meta.diff.header.git` 等语法 token；同时 `javascript`、`csharp`、`fortran-free-form`、`haxe`、`scss`、`ruby`、`hlsl`、`graphql`、`css` 等模块出现大量新增语法字符串，`tsx`、`less`、`python`、`wasm` 等模块体积或字符串则有收缩。

这指向 diff 视图和代码高亮语法包在重分包或升级，但无法从产物还原到源码级意图。

### 7. 本地开发脚本去掉 `dev` 前的 SQLite rebuild 【实证】

`package.json` 中 `dev` 脚本从 `pnpm run rebuild:sqlite && ... electron-forge start` 改为直接生成 third-party notices 后启动 `electron-forge start`。

对开发者可感知的变化是，本地开发启动路径不再在 `dev` 前置执行 `rebuild:sqlite`；但 `forge:package` 和 `forge:make` 仍保留 `pnpm run rebuild:sqlite && pnpm run rebuild:forge-natives`。
