# Codex 桌面版 `26.311.21342`(build 993)

> 第三方变更日志 · 非官方 · 基于官方全量包逆向差异

Codex(编码智能体桌面应用) 26.311.21342 是一次小幅版本推进：bundle(macOS 应用包)文件数不变，但应用层入口、预加载桥接和若干设置相关模块发生了变化。

## 关于这份报告

这不是官方变更日志，而是一份基于官方分发产物的逆向差异报告。本版为历史回填：两个版本均已超出官方 Sparkle(应用更新分发框架)增量窗口，因此事实包采用两个官方全量包直接比较，而不是 EdDSA(一种数字签名算法)验签增量包后再用 BinaryDelta(二进制增量重建格式)重建。

对比范围是 macOS(苹果桌面操作系统) arm64(苹果芯片架构)：从 `26.309.31024` build 962，发布于 `2026-03-12`，到 `26.311.21342` build 993，发布于 `2026-03-13`。

可信度方面，事实包给出了两个官方全量包的 SHA-256(哈希校验值)，并给出了公证验证结果。两个版本的验证都返回 `CSSMERR_TP_CERT_REVOKED`；事实包说明这会影响该版本能否被系统安全安装，但不影响对 bundle 内容做差异分析。

证据等级分两档：

- **【实证】**：来自人类可读的直接证据，例如 Info.plist(macOS 应用元数据文件)、package.json(依赖与脚本清单)、预加载脚本差异。
- **【信号】**：来自前端模块名或体积变化，只能指向某方向，但无法从产物还原到源码级意图，也不代表功能已经对用户开放。

## ✨ 重点变化

### 1. 外观设置开始接入系统主题变体 【实证】

`preload.js` 新增了系统主题变体相关桥接：`codex_desktop:get-system-theme-variant`、`codex_desktop:system-theme-variant-updated`，并向前端暴露 `getSystemThemeVariant` 与 `subscribeToSystemThemeVariant`。

这意味着前端现在可以读取并订阅系统主题变体变化。可感知方向是外观设置与系统主题联动更完整，但具体界面如何呈现仍需以运行时为准。

关键证据：

```diff
+ getSystemThemeVariant
+ subscribeToSystemThemeVariant
+ codex_desktop:get-system-theme-variant
+ codex_desktop:system-theme-variant-updated
```

### 2. 主题资源和外观设置模块明显增多 【信号】

去除 Vite(前端构建工具) hash 重命名噪音后，真实新增模块包含 `appearance-settings`、`use-resolved-theme-variant`，以及多组主题名模块：`absolutely-dark`、`absolutely-light`、`linear-dark`、`linear-light`、`lobster-dark`、`matrix-dark`、`notion-dark`、`notion-light`、`proof-light`、`sentry-dark`、`temple-dark`。

这些模块名指向外观设置和主题变体扩展，但只能作为方向性信号，无法从产物还原到源码级意图，也不能证明所有主题已对用户开放。

### 3. 应用层新增 `ssh-config` 依赖 【实证】

`package.json` 新增依赖 `ssh-config`，版本范围为 `^5.0.3`。

这至少说明应用层开始打包 SSH(安全远程登录协议)配置相关库。结合远端连接相关模块名，它指向远端环境配置方向，但依赖本身不等于功能已经开放。

关键证据：

```diff
+    "ssh-config": "^5.0.3",
```

### 4. 远端连接与应用服务器状态出现新前端模块 【信号】

新增模块包括 `remote-connections-settings`、`app-server-connection-state`、`onboarding-login-content`、`use-is-apps-enabled`，入口预加载列表中也出现了 `app-connect-modal` 与 `app-server-manager-hooks`。

这些名称指向远端连接设置、应用服务器连接状态和登录引导相关界面正在被整理，但这仍是模块名信号，无法从产物还原到源码级意图，也不能证明功能入口已稳定开放。

### 5. 差异查看、编辑入口与 VS Code 接口出现模块线索 【信号】

新增模块包含 `diff-view-mode`、`pencil`、`arrow-left`、`vscode-api`。其中 `diff-view-mode` 指向差异查看模式，`pencil` 指向编辑动作，`vscode-api` 指向 VS Code(微软代码编辑器)接口层。

这些变化可能与代码查看、编辑或外部编辑器集成相关，但证据仅来自模块名，不能写成确定功能。

### 6. 本地开发启动脚本调整 【实证】

`package.json` 中 `dev` 脚本移除了前置的 `pnpm run rebuild:sqlite &&`，而 `forge:package` 与 `forge:make` 仍保留 `rebuild:sqlite`。

对开发者可感知点是：运行 `dev` 脚本时，不再先执行这一条重建命令。

关键证据：

```diff
- "dev": "pnpm run rebuild:sqlite && pnpm --dir .. run generate:third-party-notices && cross-env NODE_ENV=development electron-forge start",
+ "dev": "pnpm --dir .. run generate:third-party-notices && cross-env NODE_ENV=development electron-forge start",
```
