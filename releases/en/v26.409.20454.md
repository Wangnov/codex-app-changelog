# Codex Desktop `26.409.20454`(build 1462)

> Third-party changelog · Unofficial · Based on reverse-engineered diffs of official update packages

This release looks more like a runtime foundation upgrade and permission adjustment: Electron was upgraded, the app size increased somewhat, permissions shifted from Bluetooth to audio capture, and frontend module names reveal signs of changes around experimental features, projectless sessions, and state management.  
One-liner: runtime upgrade, permission reshaping, and early UI signals around experimental/projectless flows.

## About this report

This is not an official changelog, but a **reverse-engineered diff report based on official distribution artifacts**. This update is a historical backfill: both versions are outside the official Sparkle delta window, so the comparison was not reconstructed from delta packages. Instead, both official full packages were downloaded and compared directly, then used to generate a structured diff.

In terms of confidence, both Codex.app bundles passed system notarization verification: the `spctl` results for both the previous and latest versions were `accepted`, the source was `Notarized Developer ID`, and the signing identity was `Developer ID Application: OpenAI OpCo, LLC (2DC432GLL2)`.

This report uses two evidence levels:

- **[Confirmed]**: Direct human-readable evidence, such as `Info.plist`, `package.json`, signing results, and notarization results.
- **[Signal]**: Frontend module names or binary size changes. These can only indicate a possible direction, but cannot reconstruct source-level intent from the artifact.

Comparison scope: from `26.406.40811` build `1457`, released on `2026-04-11`; to `26.409.20454` build `1462`, released on `2026-04-11`. Platform: `macOS arm64`.

## ✨ Highlights

### 1. Desktop runtime foundation upgraded to Electron 41.2.0 [Confirmed]

`package.json` shows that the app’s Electron version was upgraded from `40.0.0` to `41.2.0`.

This means the desktop shell, WebView, system integration, and Chromium-related capabilities have moved onto a newer runtime foundation. However, the artifact does not provide specific user-facing feature changes, so only the dependency version upgrade itself can be confirmed here.

Key evidence:

```diff
-    "electron": "40.0.0",
+    "electron": "41.2.0",
```

### 2. Permission declaration shifted from Bluetooth to audio capture [Confirmed]

`Info.plist` added `NSAudioCaptureUsageDescription` with the value `This app needs access to audio capture`; at the same time, the old `NSBluetoothPeripheralUsageDescription` was removed, whose previous value was `This app needs access to Bluetooth`.

This indicates that the new app version now declares audio-capture permission and no longer declares Bluetooth peripheral permission. At the user-visible level, this may affect the type of macOS system permission prompt shown. However, the artifact does not show the corresponding feature entry point, so it is not possible to determine which specific feature audio capture will be used for.

Key evidence:

```text
新增 /NSAudioCaptureUsageDescription = 'This app needs access to audio capture'
移除 /NSBluetoothPeripheralUsageDescription = 'This app needs access to Bluetooth'
```
