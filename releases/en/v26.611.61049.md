# Codex Desktop 26.611.61049(macOS / Windows Comparison)

> Third-party cross-platform changelog · Unofficial

The core finding is straightforward: macOS moved from 26.609.71450 build 3965 to 26.611.61049 build 3996, while the Windows x64 MSIX comparison remains 26.611.7849.0 with zero file-level delta.

The observable update lands on macOS only in this fact set; the Windows MSIX comparison is byte-for-byte unchanged.

This report is based on reverse-reconstructed diffs of two official signed distribution packages: Sparkle packages for macOS and MSIX packages for Windows. ## ✨ Highlights

### Computer Use Runtime Update And macOS Transport Rewrite [Confirmed] [macOS only]

The bundled macOS `@oai/sky` package moved from 0.4.10 to 0.4.13, and its `package.json` removed the `objc-js` optionalDependency. Total `cua_node` size was essentially flat: 365.8 MB → 365.9 MB.

More importantly, the macOS Computer Use client changed from a direct Apple Event/service-app path, bundle identifier, and pid request model to `MacNativePipeTransport`, with transport caching by `apiVersion`. The type declarations also removed `serviceAppPath`, `serviceBundleIdentifier`, and `serviceProcessIdentifier` request options. Readable code shows `click`, `drag`, `press_key`, `scroll`, `set_value`, `type_text`, `select_text`, `perform_secondary_action`, and `get_app_state` now pass through `withComputerUsePolicy`.

The Windows comparison has no file changes, so this fact set cannot count the Computer Use update as having landed on Windows.

### Browser Plugin API Gains JavaScript Dialog Handling [Confirmed] [macOS only]

The macOS browser/chrome plugin docs add `Tab.getJsDialog()` and define `AlertDialog`, `BeforeUnloadDialog`, `ConfirmDialog`, `PromptDialog`, plus the union `Dialog` type. The `Tab.clipboard` description also changes from clipboard content “in this tab” to the browser session’s clipboard, while the browser-list return type is expanded inline with capabilities, metadata, name, and type fields.

This is a clear plugin API documentation change. The Windows comparison has no changes, so the Windows MSIX package cannot be confirmed to include it here.

### Browser Automation Stops At Login Blocks Instead Of Routing Around Them [Confirmed] [macOS only]

The macOS in-app browser and Chrome control skill docs add a new rule: when the user explicitly asks to navigate in the browser and authentication or sign-in blocks the requested task, Codex should not switch to web search, a search engine, another site, or another source to work around the login. If secure browser authentication is advertised, that flow should be used; otherwise, Codex should stop and ask the user to log in.

This is a behavior-policy change for browser automation at login walls. The Windows comparison has no file changes.

### Site-Building Guidance Backs Away From Built-In SIWC/OAuth Starters [Confirmed] [macOS only]

The `sites-building` skill doc removes guidance that public or externally authenticated sites should use the `auth-siwc-enabled` starter. It now says to confirm the current platform auth path before adding an app-owned auth stack, and not to scaffold app-owned public sign-in or external OAuth from a starter. New-site starter selection also narrows toward `vinext-starter` while preserving the selected layout.

This shifts site-auth guidance away from template-default authentication and toward platform-path confirmation. The Windows comparison has no changes.

### Built-In Browser Context Menu Adds Copy Link Text [Confirmed] [macOS only]

The macOS package adds readable strings and locale entries for `browserSidebar.contextMenu.copyLink`, with the English text `Copy link`, across many native-menu locale files. Main/frontend strings also include `Open in browser`, `Open in external browser`, `Close Tab`, and `Copy link`.

This confirms that copy-link menu text is present in the package. Whether it is exposed to every user still depends on runtime UI paths or flags, which the fact package does not establish. The Windows comparison has no changes.

### Frontend Direction: Cloud Environments, Code Review, Voice, Browser Import, Settings [Signal] [macOS only]

After excluding Vite hash churn, new macOS module names include `cloud-environments-settings-page`, `cloud-preferences`, `cloud-preferences-settings`, `code-review-settings`, `browser-profile-import-dialog`, `browser-profile-import-query`, `global-dictation-orb`, `use-dictation`, `use-dictation-hotkey`, `avatar-overlay-realtime-voice-button`, `page-search-input`, `back-forward-navigation-buttons`, `patch-diff-stats`, and `thread-right-panel-state`.

These names point toward work around settings, cloud environments, code review, browser profile import, voice/dictation, browser navigation, and diff stats, but they are bundle-level signals only. They do not prove user-visible launch. The Windows comparison has no changes.

### Windows MSIX Has No Observable Change In This Round [Confirmed] [Windows only]

The Windows x64 comparison reports 26.611.7849.0 on both sides, with SHA-256 `d19ebe0d2620d7c58f6681c3ed3ef23e12f6396354874db7721086bbd75f6fec` for both packages. File count is 11943 → 11943, with 0 added, 0 removed, and 0 changed files. `app.asar` internals are 3024 → 3024, with 0 added, 0 removed, and 0 changed files. `cua_node` stays 285.6 MB → 285.6 MB.

Therefore none of the macOS-visible behavior or documentation changes in this report can be tagged [macOS+Windows].
