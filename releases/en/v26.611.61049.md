## ✨ Highlights

### [Confirmed] [macOS only] macOS Version, Chromium Baseline, and Signed Package Update

macOS moves from `26.609.71450` build `3965` to `26.611.61049` build `3996`. Info.plist also shows updates to `CFBundleShortVersionString`, `CFBundleVersion`, `ElectronAsarIntegrity/Resources/app.asar/hash`, and `SCMRevision`.

The Chromium baseline changes from `149.0.7827.54` / `7827.54` to `149.0.7827.115` / `7827.115`. The file tree also adds new `Codex Framework.framework/Versions/149.0.7827.115/...` files and removes the corresponding `149.0.7827.54/...` files.

The Windows input sample has `26.611.7849.0` as both previous and latest version, the same SHA-256, and no changes in file count, app.asar, or cua_node statistics.

### [Confirmed] [macOS only] Browser / Chrome Control APIs Add JavaScript Dialog Handling and Refine Capability Metadata

`browser/docs/api.md`, `chrome/docs/api.md`, and `browser/.codex-plugin/api-multi-tab.md` all add `Tab.getJsDialog(): Promise<undefined | Dialog>`, plus `AlertDialog`, `BeforeUnloadDialog`, `ConfirmDialog`, and `PromptDialog`. Methods such as `PromptDialog.accept(text)` and `ConfirmDialog.accept()` / `dismiss()` are now packaged.

The same API docs also change `Tab.clipboard` from “this tab” to “the browser session's clipboard”, and expand `Browsers.list()` to return browser/tab capabilities, metadata, name, and type. This is explicit documentation evidence for browser automation surface changes.

### [Confirmed] [macOS only] Browser Skills Add a Login-Block Handling Rule

`control-chrome/SKILL.md`, `control-in-app-browser/SKILL.md`, and `control-in-app-browser-multi-tab.md` all add the same rule: when the user explicitly asks the browser to navigate to a page and authentication or sign-in blocks the task, the agent should not switch to web search, a search engine, another site, or another source to work around the login. If secure browser authentication is available, use it; otherwise ask the user to log in before continuing.

This is not just a UI signal; it is a direct change to packaged skill instructions and affects the default boundary for browser-control tasks.

### [Confirmed] [macOS only] macOS Computer Use Actions Are Wrapped in a Policy Layer

`@oai/sky` moves from `0.4.10` to `0.4.13`. Its `package.json` also removes the `objc-js` optional dependency. `press_key.js`, `type_text.js`, `drag.js`, `set_value.js`, `scroll.js`, `click.js`, `perform_secondary_action.js`, `get_app_state.js`, and `select_text.js` change from direct client calls to execution through `withComputerUsePolicy(...)`.

`list_apps.js` adds `setComputerUseResponseMeta(null)`. `client.d.ts` changes the `CodexMetadata` import from `apple-event` to `native-pipe`, removes `serviceAppPath`, `serviceBundleIdentifier`, and `serviceProcessIdentifier` request options, and adds private transport-related fields. The package also adds the string `computer-use native pipe helper paths unavailable`, while removing `computer-use native pipe transport module missing`.

Together, these changes show a new policy/transport path for macOS Computer Use input actions, state reads, and transport handling. The Windows sample’s cua_node remains `285.6 MB → 285.6 MB`, with no dependency or file changes to report.

### [Confirmed] [macOS only] Computer Use Adds a Safety Boundary for Desktop Surface Attachment

These strings prove that safety interception, floating-window positioning, realtime voice startup, and process stdin error paths are packaged. They do not prove that every path is user-visible.
