## About This Report

This report is based on machine-extracted diffs from reconstructed official signed distribution packages: macOS arm64 from `26.623.70822` build `4559` to `26.623.81905` build `4598`, and Windows x64 from `26.623.9142.0` to `26.623.11225.0`. The release date follows the input batch: `2026-06-27`.

Evidence uses two levels: [Confirmed] comes from manifests, plist, CSP, plugin config, service entries, skill docs, dependency lists, explicit readable strings, or key file diffs; [Signal] comes from same-name frontend module changes, module names, size movement, and CSS/state class changes. Platform scope is marked as `[macOS+Windows]`, `[macOS only]`, or `[Windows only]`. webview/assets and `.vite/build` hash renames are not treated as feature additions or removals.

## ✨ Highlights

### [Confirmed] [macOS+Windows] browser and chrome bundled plugins synced to 26.623.81905

Both platforms update the `.codex-plugin/plugin.json` version for the `browser` and `chrome` openai-bundled plugins from `26.623.70822` to `26.623.81905`. The plugin description text in the provided diffs does not show functional copy changes.

The browser plugin still points to the Codex in-app browser for opening, inspecting, navigating, testing, clicking, typing, or screenshotting local web targets such as localhost, 127.0.0.1, `file://`, and the current in-app browser tab. The chrome plugin still describes automation that depends on the user's existing Chrome state, including tabs, logged-in sessions, cookies, extensions, and Codex Chrome Extension setup.

Platform difference: macOS shows only browser/chrome plugin version changes; Windows also shows a computer-use plugin version change.

In the Windows package, `resources/plugins/openai-bundled/plugins/computer-use/.codex-plugin/plugin.json` changes its version from `26.623.70822` to `26.623.81905`. Its description remains: controlling Windows desktop apps from Codex through Computer Use.

The macOS fact package does not include a matching computer-use plugin manifest diff, so this change cannot be generalized to macOS.
