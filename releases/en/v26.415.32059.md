## About This Report

This is not an official changelog. It is a **reverse-diff report based on official distribution artifacts**. For this historical backfill, the fact package states that the official appcast did not provide a delta package for `26.415.30602` → `26.415.32059`, so both official full packages were downloaded and compared directly.

Evidence is classified into two levels:

- **[Confirmed]**: Direct human-readable evidence such as Info.plist values, dependency lists, signing/notarization results, readable configuration, or other readable strings outside same-name frontend module hints.
- **[Signal]**: Indirect evidence from same-name frontend module change hints or binary size changes. These point in a direction but cannot recover source-level intent from the artifact.

The comparison covers macOS arm64. The previous version is `26.415.30602` build `1773`, released on `2026-04-17`; the latest version is `26.415.32059` build `1789`, released on `2026-04-18`.

The `general-settings` frontend module grew from about `0.0 MB` to `0.1 MB`, increasing by `93277 bytes`, and gained several new settings strings. They point to added or adjusted Settings entries for Agent environment, in-app browser control, connections from other signed-in clients, and experimental feature toggles, but the artifact cannot recover source-level intent.

User-visible impact: users may see more controls related to runtime environment, browser tasks, remote connectivity, or beta features in Settings.

### 2. Settings adds UI size, code size, and plan-usage hints [Signal]

The same `general-settings` module added copy related to UI base size, code base size, and speed/plan usage. This signal points to expanded display-density or readability settings, plus more explicit usage messaging, but it cannot be treated as source-level fact.

User-visible impact: developers may see finer controls for UI and code display, along with usage text such as `1.5x speed, 2x plan usage`.

Key signals include:

```text

```

### 3. Settings accessibility and theme-control signals are stronger [Signal]

`general-settings` gained several `Aria label` strings and ARIA-related class names, plus readable strings for a contrast slider, macOS menu bar toggle, and translucent sidebar toggle. These point to more detailed accessibility labeling and theme controls in Settings, but remain frontend-module signals.

User-visible impact: assistive technology, keyboard operation, or visual-theme behavior may have changed.

Key signals include:

```text

Aria label for the translucent sidebar toggle in chrome theme settings
aria-label
aria-valuenow
aria-valuetext
```

### 4. Projectless thread startup directory wording changed [Signal]

In `app-server-manager-signals`, the text changed from `This projectless thread starts in the user` to `This projectless thread starts in a generated directory under the user`. This points to more precise wording for projectless thread working directories, but does not confirm whether the underlying directory policy changed.

User-visible impact: when starting a thread without a project, the UI may more clearly state that it starts in a generated directory under the user directory.

Key signal:

```diff
- This projectless thread starts in the user
+ This projectless thread starts in a generated directory under the user
```
