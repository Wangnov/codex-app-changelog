## About This Report

This report is based on reverse diffs of officially signed distribution packages: the macOS side uses full Sparkle packages, and the Windows side uses MSIX packages. Because the two platforms were released with slight timing differences, this report groups them as one batch for comparison.

Evidence is split into two levels:

- [Confirmed]: readable evidence such as plist files, plugin configuration, package metadata, or dependency lists.
- [Signal]: evidence based only on frontend module names, file counts, or size changes; it can indicate direction but cannot reconstruct source intent.

Platform markers mean:

- `[macOS+Windows]`: both platform fact packs show the relevant change.
- `[macOS only]`: only the macOS fact pack provides evidence.
- `[Windows only]`: only the Windows fact pack provides evidence.

Comparison scope:

- macOS arm64: `26.527.30818` build `3370` → `26.527.31326` build `3390`.
- Windows x64: `26.527.3378.0` → `26.527.3686.0`.
- released: `2026-05-27`.

## ✨ Key Changes

### Version and Build Advancement

[Confirmed] [macOS+Windows]

macOS moved from `26.527.30818` to `26.527.31326`, with `CFBundleVersion` moving from `3370` to `3390`. Windows moved from `26.527.3378.0` to `26.527.3686.0`.

The platform difference is in versioning format: macOS uses a short version plus a build number, while Windows uses a full package version. They belong to the same batch, but the Windows version should not be directly converted into the macOS build number.

### Application-Layer app.asar Repackaging, Without Confirmed Feature Additions

[Signal] [macOS+Windows]

Both platforms show internal changes inside `app.asar`. macOS internal file count stayed at `2774 → 2774`, with `289` added, `289` removed, and `12` changed. Windows internal file count stayed at `2722 → 2722`, with `289` added, `289` removed, and `11` changed.

The platform difference is small: both sides have the same added and removed counts, while macOS has one additional changed file. The fact packs explicitly flag large `webview/assets` and `.vite/build` churn as Vite hash-renaming noise, so it should not be treated as evidence of new functionality.

### macOS Bundled Plugin Metadata Version Sync

[Confirmed] [macOS only]

In the macOS package, the `browser` and `chrome` plugin configuration files under `openai-bundled` changed their `version` fields from `26.527.30818` to `26.527.31326`. This shows that the macOS distribution synced the Browser and Chrome bundled plugin metadata to the application version.

The platform difference is explicit: the Windows fact pack does not include the corresponding plugin configuration diff, so we cannot claim Windows synced the same plugin metadata. We can only say macOS has readable evidence, while Windows lacks evidence at the same level in this batch.

### No New cua_node Packages

[Confirmed] [macOS+Windows]

Both platform fact packs report `0` newly added `cua_node` packages.

Platform difference: none. At least in the dependency layer listed by the fact packs, this batch does not introduce new `cua_node` packages.

### Stable Outer File Tree, With Changes Concentrated in Content Updates

[Confirmed] [macOS+Windows]

macOS outer file count stayed at `1982 → 1982`, with `0` added, `0` removed, and `51` changed. Windows outer file count stayed at `8648 → 8648`, with `0` added, `0` removed, and `13` changed.

The platform difference is that macOS has a notably higher changed-file count than Windows, but neither platform added or removed outer files. Combined with the `+0.00 MB` `app.asar` size delta, this looks more like content replacement and rebuild churn than the introduction of a large new component.
