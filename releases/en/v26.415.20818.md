# Codex Desktop `26.415.20818` build 1727

> Third-party changelog · Unofficial · Based on reverse diffs of official update artifacts

The clearest change in this build is the arrival of bundled Computer Use / browser-use components, while frontend modules and readable strings point to broader work on document previews, remote connections, automations, plugin management, and update states.

## About This Report

This is not an official changelog. It is a third-party reverse diff report based on official distribution artifacts. The comparison covers macOS arm64 from `26.409.20454` build `1462` to `26.415.20818` build `1727`.

This pair was backfilled historically: the official appcast did not provide a Sparkle delta for these versions, so the report compares two official full ZIP packages directly. Both App bundles passed notarization validation as `accepted`, with source `Notarized Developer ID` and origin `Developer ID Application: OpenAI OpCo, LLC (2DC432GLL2)`.

Evidence levels:

- **[Confirmed]**: direct human-readable evidence, such as `package.json` diffs, Info.plist values, service entries, plugin files, readable strings, or signing results.
- **[Signal]**: indirect evidence from frontend module names, same-name frontend module string diffs, or size changes. These point in a direction, but cannot reconstruct source-level intent or prove that a feature is enabled for all users.

Hash-named files under `webview/assets` and `.vite/build` are mostly build output renames. This report only uses the fact package’s de-noised module list and key diffs, and does not treat bulk hash churn as feature additions or removals.

## ✨ Highlights

### 1. Bundled Computer Use / browser-use components appear [Confirmed]

The new bundle adds `Codex Computer Use.app` under `Contents/Resources/plugins/openai-bundled/plugins/computer-use/`, including the `SkyComputerUseService` and `SkyComputerUseClient` executables. It also adds `browser-client.mjs` and skill documentation under the `browser-use` and `chrome` plugin directories.

This means the desktop package is gaining local runtime pieces for browser tasks and computer control. The artifact alone does not prove that the capability is enabled by default for all users.

### 7. Startup now includes an Intel-on-Apple-Silicon warning path [Confirmed]

`.vite/build/bootstrap.js` adds detection for `darwin`, `x64`, and `sysctl.proc_translated`, with warning text for running an Intel build on an Apple Silicon Mac.

Key added strings:

```text
{appName} is running the Intel build on an Apple Silicon Mac
This build works through Rosetta, but the Apple Silicon build launches faster and performs better.
Quit
Continue Anyway
```

The compared artifact is the macOS arm64 package, so this path would not trigger for the arm64 runtime. It appears to be a cross-architecture guard in shared startup code.
