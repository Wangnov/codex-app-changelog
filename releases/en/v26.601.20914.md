# Codex Desktop 26.601.20914 macOS / Windows Comparison

> Third-party cross-platform changelog · Unofficial

The main story in this batch: Sites lands as a bundled capability, while desktop update UI, About dialogs, settings search, plugin install flows, browser sidebars, PR handling, and cross-host/worktree chat surfaces all move forward.  
One-liner: Sites becomes a bundled capability while the desktop shell, plugin flows, browser tooling, and task/worktree surfaces move forward across both platforms.

## About This Report

This report only uses the machine-extracted facts in the supplied packages. The macOS sample compares 26.527.60818 build 3437 → 26.601.20914 build 3497. The Windows sample compares 26.527.7698.0 → 26.601.1994.0. They come from the same Codex codebase packaged for different platforms, but version numbers, package sizes, file counts, and platform components are cited separately.

Evidence levels:  
[Confirmed] means the fact comes from plugin manifests, skill docs, package.json, Info.plist, readable strings, resource manifests, dependency lists, or concrete config/docs diffs.  
[Signal] means the evidence comes from changed same-name frontend modules, module additions/removals, style classes, size direction, or compressed bundle clues. These indicate direction, not full source semantics, and do not prove broad user availability.

Platform markers are: [macOS+Windows], [macOS only], [Windows only]. Hash-only churn in webview/assets and .vite/build is treated as build noise.

## ✨ Highlights

### [Confirmed] [macOS+Windows] Sites Enters the Bundle as a Built-in Plugin

On both platforms, the bundled plugin list expands from `browser`, `chrome`, `computer-use`, `latex` to `browser`, `chrome`, `computer-use`, `latex`, `sites`. The new plugin display name is Sites, version `0.1.10`, category Productivity, with keywords `deploy`, `sites`, and `games`.

The capability description is explicit: Create, save, deploy, and inspect sites hosted by OpenAI. Default prompts include a pizza restaurant landing page, a daily sudoku game with a leaderboard, and an image generation website. The macOS plugin directory has 60 files totaling 0.8 MB; Windows has 60 files totaling 0.9 MB.

The skill docs show this is a complete capability surface, not just package bulk. `sites-building` covers creating or modifying sites, requires Cloudflare Workers-compatible output, prefers vinext templates, and stores hosting metadata in `.openai/hosting.json`. `sites-hosting` covers creating/reusing hosting config, saving deployable versions, production deploys, deployment status inspection, and access controls. Templates include `vinext-starter` and `auth-siwc-enabled`; the largest files are the two `package-lock.json` files at 0.4 MB each.

### [Confirmed] [macOS+Windows] app.asar Grows in Lockstep, with Matching Real Frontend Module Churn

macOS `Contents/Resources/app.asar` grows from 140.3 MB to 141.6 MB, +1.32 MB. Windows `resources/app.asar` grows from 144.1 MB to 145.4 MB, +1.32 MB. After removing hash noise, the added frontend module lists match closely. Internal file counts are also near-aligned: macOS 2774 → 2804, Windows 2722 → 2752, each with 1001 added and 971 removed files.

New modules cover settings search, Sites color, service tier, plan and upgrade UI, thread resource cards, virtualized rendering, recent project chat signals, browser sidebar webviews, plugin connect flow, a PR fix button, and VS Code API support. Removed modules include the older app connect modal, plugin-auth, share, sidebar-thread-section, virtualized-turns, pull-request-board-model, and local-remote-selection. As [Signal], this points to shell, settings, plugin, browser, PR, and thread rendering refactors, but does not reconstruct source behavior.

### [Confirmed] [macOS+Windows] Desktop Update Confirmation and About Dialog Strings Enter Localized Resources

Both platforms add main/frontend strings including `appHeader.installUpdate.confirmTitle`, `appHeader.installUpdate.confirmSubtitle`, `codex.aboutDialog.*`, and `Build information`. Chinese resources explicitly add “Update Codex now?”, a subtitle saying Codex will quit to install the update and interrupt active local sessions, About dialog title, version line, release-date version line, and build information.

The same-name `app-main` module also adds forced desktop app update warning popup strings, restart deadline text, and `Codex needs to restart to finish installing an update`. Platform packaging differs: macOS also includes Info.plist version, Chromium, and app.asar integrity hash changes. The Windows fact package does not include a concrete AppxManifest diff, but package.json and localized resources contain the new version content.

### [Signal] [macOS+Windows] Settings Move Toward Searchable, Grouped Navigation

New modules include `agent-settings.search`, `computer-use-settings.search`, `general-settings.search`, `git-settings.search`, `mcp-settings.search`, and `personalization-settings.search`. The same-name `settings-page` module adds strings for settings search input, clear search, no matching destinations, and app-level, archived, coding, host-scoped, and integration settings groups.

This is a UI/navigation refactor signal: search entry points, empty states, group headings, and collapsible navigation are in the frontend bundle, but the fact package does not prove every destination is exposed to all users.

### [Signal] [macOS+Windows] Plugin Install, Connectors, and Permission Copy Shift Toward Paid Plans and Risk Disclosure

`use-plugin-install-flow` adds strings for Google plugins needing a paid plan, Gmail drafting, Gmail task extraction, Google Calendar editing/planning/scheduling, and “Codex can read the thread, understand the ask, and draft a useful response.”

`plugins-availability` adds “Approved by your admin,” “Advanced settings (opens ChatGPT.com),” “Connectors may introduce risk,” “Connectors are designed to respect your privacy, but sites may attempt to steal your data,” and “Could not load this app.” Windows also lists `plugin-detail-page` changes: `Add plugin` replaces `Add to Codex`; new text covers MCP servers, included skill details, Hooks settings, plugin share targets, and member counts. The macOS package does not list the same `plugin-detail-page` clues, so that detail remains a Windows-sample signal.

### [Confirmed] [macOS only] Browser Control Skill Docs Now Allow CAPTCHA Solving After User Confirmation

On macOS, both `control-chrome/SKILL.md` and `control-in-app-browser/SKILL.md` change from “Do not solve CAPTCHAs” to a policy that asks the user for each CAPTCHA and solves only that CAPTCHA after confirmation. The prohibition remains for paywall bypass, browser/web safety interstitial bypass, age verification, and final password-change submission.

The Windows fact package does not list the same two skill file diffs, so this report marks it macOS-only. Even if the underlying codebase is close, the macOS file-level conclusion cannot be applied to Windows without Windows evidence.
