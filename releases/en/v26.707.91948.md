# Codex Desktop 26.707.91948: macOS / Windows Comparison

> Third-party cross-platform changelog · Unofficial

The shared theme is a rework of browser capabilities, recent-task navigation, and Full Access interactions, but the macOS and Windows frontend capability surfaces are notably out of sync.

Browser automation, recent-task navigation, and Full Access controls moved forward on both platforms, while macOS emphasized browser settings and rich artifacts and Windows emphasized Computer Use, Office control, and task-side-panel actions.

## About This Report

This report compares two sets of official signed distribution artifacts: macOS arm64 from 26.707.72221 (build 5307) to 26.707.91948 (build 5440), and Windows x64 MSIX from 26.707.9981.0 to 26.707.12708.0. The two version sequences are cited independently; neither platform is used to infer the other platform’s release timing or version mapping.

This is a historical backfill based on direct comparison of two full official packages. Both macOS samples passed notarization validation with an `accepted` result and OpenAI OpCo, LLC as the signing origin. The input does not provide a Windows signature-validation result.

Evidence uses two levels:

- [Confirmed] covers plist and manifest changes, plugin descriptions, dependency inventories, file statistics, and explicit differences outside the “changed same-name frontend module clues” section.
- [Signal] covers frontend module names, strings or styles extracted from changed same-name minified modules, and size direction. It shows that related code or copy entered, left, or moved within the package, but does not prove availability to every user.

Platform labels are `[macOS+Windows]`, `[macOS only]`, and `[Windows only]`. Vite hash renames are excluded and are not treated as functional additions or removals.

The fact packages do not include complete sections for bundled-plugin inventory changes, newly readable main-process strings, or newly added components and resources. The Windows package also contains no AppxManifest or CSP diff. This report therefore limits itself to the manifest changes, frontend clues, and file-level statistics actually supplied.

## ✨ Highlights

### Browser and Chrome Plugin Version Alignment [Confirmed] [macOS+Windows]

The bundled `browser` and `chrome` plugin manifests on both platforms move from version 26.707.72221 to 26.707.91948. The outer Windows MSIX version is 26.707.12708.0, so the internal plugin version must not be presented as the Windows package version.

The `browser` manifest continues to cover the in-app browser, local addresses, the current browser tab, and actions such as opening, inspecting, navigating, testing, clicking, typing, and taking screenshots. The `chrome` manifest targets automation that depends on existing Chrome tabs, authenticated sessions, cookies, extensions, and ChatGPT Chrome Extension setup, with Chrome positioned as a user-approved fallback when a purpose-built connector or API is unavailable.

The packages do not show these plugins as newly added in this batch; only their manifest version alignment is confirmed.

### Recent Tasks, Cloud Tasks, and Chrome Navigation [Signal] [macOS+Windows]

The `header` module on both platforms gains the same set of strings for the recent-tasks menu, unread task updates, task search, `All tasks`, `Cloud tasks`, environment filtering, app settings, `ChatGPT for Chrome`, and `Chrome computer use settings`.

This suggests that header navigation is being prepared to accommodate recent-task state, cloud-task filtering, and Chrome-related entry points. Because the evidence comes from changed same-name frontend modules, it does not prove that these controls are visible to every account.

### Full Access and Ultra Reasoning Conflict Handling [Signal] [macOS+Windows]

Both platforms contain new confirmation copy around Full Access:

- Full Access can access websites, send data, and use enabled plugins.
- When Ultra reasoning is selected, users may keep Full Access or switch permission mode.
- Ultra reasoning may be unavailable when no supported permission mode exists.
- The permission surface explicitly mentions the internet and connected apps.

On Windows, `composer` also contains `Full access is on` and describes running commands, installing software, and changing system settings. `general-settings` adds a confirmation title shown before Full Access is exposed in the composer. The corresponding macOS clues appear in shared new-thread, AppGen, and hotkey-window modules, with additional reasons for Codex Cloud being unavailable when the user is not signed in through ChatGPT or the workspace has not enabled Codex.

Because all of these strings come from changed same-name frontend modules, the overall finding remains a [Signal], not confirmed evidence of a changed default permission mode.

### Computer Use and Live Office Control in the @-Mention Surface [Signal] [Windows only]

The Windows `at-mention-list` adds `Computer use`, `Desktop apps`, `Custom agents`, `Live agents`, `Files and tasks`, ChatGPT conversations, and copy for live Excel workbook and PowerPoint presentation control.

Its style and state identifiers also include `mcp-capability`, `mcp-capability-mentions`, `microsoft-excel-plugin`, and `microsoft-powerpoint-plugin`. Together, these clues point to an @-mention surface organizing Computer Use, MCP capabilities, and live Microsoft Office controls, but they do not prove that every entry point is enabled.

### Windows Computer Use Plugin Updated with the Batch [Confirmed] [Windows only]

The bundled Windows `computer-use` plugin manifest moves from version 26.707.72221 to 26.707.91948. Its description explicitly identifies the capability as controlling Windows desktop applications from ChatGPT through Computer Use.

No new `cua_node` dependencies were added, and the Computer Use runtime remains 287.5 MB. This is therefore a plugin-version and application-integration update, not an expansion of the runtime dependency set.

### Expanded Browser Settings, History, and Downloads [Signal] [macOS only]

The macOS `browser-use-settings` module gains copy for:

- Accessing history without asking;
- Embedded browser downloads and history;
- Contact information, extensions, password manager, and site settings;
- Expanding and collapsing individual browser-data clearing options;
- Opening download history.

The `onboarding-page` module also gains horizontal, vertical, reverse-direction, alignment, and distribution options for a browser tweak editor, together with `selected-browser-element`, `selected-browser-region`, and `selected-browser-text` states. These are signals of more detailed browser settings and page-adjustment tooling.

### Browser Tab Actions and Task Continuation in the Side Panel [Signal] [Windows only]

The Windows `thread-side-panel-tabs` module adds copy for creating a tab, opening a tab immediately to the right, reloading, duplicating a browser tab, and continuing from the current task and its tabs into a new task.

Continuation can target the same worktree, a new worktree, or a new local task. The module also contains titles for an empty browser tab and the review tab in the task side panel, indicating an expanding interaction surface between browser tabs and task context.

### Multi-Agent Composer Banner Enters the Package [Signal] [macOS only]

The macOS `home-announcements` module adds a multi-agent composer banner, dismissal state, and the following body copy:

`Delegate work to subagents that work in parallel. Note: may increase token usage.`

The same module also gains dismissal labels for Fast mode, gift credits, rate-limit resets, and a backend-driven home banner. Because these strings come from a changed same-name frontend module, they only confirm that related copy and state logic entered the frontend package, not that the banners are shown to every user.

### Home Projects, Attachments, Dictation, and Personalization [Signal] [macOS only]

The macOS main-page modules add:

- `Add local project` and accessible labels for the home project selector;
- `Add photos` and adding photos to the active composer;
- A `hold-to-dictate` state;
- `Adjust tone and response style`;
- Several work-style or character descriptions;
- An accessible Codex Micro battery-charging status.

Codex Micro onboarding, Micro settings and keyboard shortcuts, personalization settings, pet settings, avatar overlays, quick chat, and local-conversation modules are also added or recomposed. Together, they indicate continued work on home input, voice, personalization, and lightweight overlay experiences, but module topology does not prove feature availability.

### Image, PDF, and Annotation Interaction Refinements [Signal] [macOS only]

A shared macOS onboarding module adds accessible copy for screenshot thumbnails attached to user comments, previews of selected document or PDF content, and removing a response annotation from the composer.

These strings show that image, document, PDF, and annotation paths entered the frontend build, but the package provides no confirmed file-size limits, upload constraints, or general-availability status.

### Rich Documents, Workbooks, Presentations, and Math Rendering [Signal] [macOS only]

Changed macOS modules show:

- `workbook` growing from near 0 MB to 2.6 MB, with workbook ranges, theme colors, and numerous layout data attributes;
- `workbook-from-markdown` gaining serialization, path, plugin-list, and find-and-replace errors;
- `presentation` growing to 0.3 MB and containing `oaiproto.coworker.pptx` plus protocol parsing errors;
- `katex` growing to 0.3 MB with markup-language, macro-definition, control-sequence, and MathML content;
- New `artifact-tab-content` module combinations and recomposed notebook-preview, pull-request, and code-review chunks.

This is a relatively strong rich-document and artifact-rendering signal, but the fact package includes no corresponding service entry point, command, or user-facing release note.

### Broader Syntax Coverage for Code and Diffs [Signal] [macOS only]

The macOS `diff` module gains scopes for changed, deleted, and inserted lines, Git headers, ranges, and file paths. PHP, Stylus, Shell, Go, TeX, R, XML, GraphQL, SDBL, GLSL, and WAT/WebAssembly modules also gain numerous syntax scopes and token names.

These changes point to updated syntax-highlighting or code-display resources. The WASM module itself shrinks from 0.6 MB to near 0 MB, so the set should not be described as uniform expansion across every language module.

### Conversational Onboarding Execution Constraints [Signal] [macOS only]

A shared macOS onboarding/debug module gains task instructions to execute the selected task immediately, avoid follow-up questions, avoid interactive onboarding tools, provide concise plain-language progress, and stop after one concise final result.

The same module contains states for onboarding already starting, plugins not becoming ready, hidden tasks, and cancelled starts. These are signals that conversational onboarding flow and error paths entered the package, not proof of a stable public interface.

### Local, Remote, and Quick-Chat Chunk Reorganization [Signal] [macOS only]

After hash noise is removed, macOS adds a standalone `local-conversation-page`, a combined `remote-conversation-page~local-conversation-page`, `projects-index-page~remote-conversation-page`, `quick-chat-window-page`, and combinations with ChatGPT conversations.

Remote-connections, usage, code-review, skills, plugins, and personalization settings also appear in new shared chunk combinations. Because many older combinations were removed at the same time, this is best interpreted as frontend code-splitting and route-dependency reorganization, not confirmed evidence of entirely new local or remote conversation products.

### Plugin Sharing, AppGen Settings, and Task Archive Copy Contracted [Signal] [macOS only]

Several changed same-name modules lose readable strings:

- `plugin-detail-page` loses workspace-share search, link sharing, MCP server enablement, and page-navigation copy;
- `appgen-settings-page` loses custom-domain, site-title, site-slug, environment-variable, and secret-input copy;
- A project/thread module loses pin, unpin, archive-confirmation, and scheduled-task archive strings;
- A skills/plugins settings module loses search-field clearing copy.

This only proves that the strings no longer reside in those same-name modules. They may have been removed, moved, or split into other chunks, so the corresponding features cannot be declared discontinued.

### Windows RPC and Warning-Presentation Foundations [Signal] [Windows only]

The Windows `rpc` module gains errors covering RPC stub lifecycles, ReadableStream and WritableStream stubs, sessions, reused promise export IDs, and pipe creation, along with a `workers-module` state identifier.

The frontend also adds a standalone `triangle-warning` module, while composer gains validation-error background and text styles. Together, these form a weak-to-moderate signal of changes to remote-call error handling and warning presentation.

## 🔎 Low-Level Signal Index

### macOS Interface and State Index [Signal] [macOS only]

- `select-workspace-page~projects-index-page` grows to roughly 0.4 MB and gains dropdown, ARIA, backdrop-blur, and date-formatting errors, indicating a reorganization of workspace and project-selection foundations.
- A pets and appearance settings chunk gains switch, disabled-state, focus-ring, and transition styles; avatar, pet, and Codex Micro modules are also recomposed.
- `middleware` gains page, screen, device, IP, referrer, timezone, email, and user-agent context field names. This shows analytics-middleware fields in the package but does not establish a particular collection policy.
- `appgen-settings-page`, `plugin-detail-page`, and debug-related modules shrink substantially; this index records only a migration or trimming direction.

### macOS Syntax and Rendering Index [Signal] [macOS only]

PHP, Stylus, Shell, Go, TeX, R, XML, GraphQL, SDBL, GLSL, and WAT modules gain syntax tokens, while KaTeX, workbook, presentation, and diff modules gain parsing or display resources. The evidence boundary is limited to changed minified frontend assets and does not establish new editor, compiler, or language-service capabilities.

### Windows Interface and Runtime Index [Signal] [Windows only]

- `pet-install-state` gains `pet-install` and `pet-install-preview` state classes, indicating a style change around pet installation previews.
- `local-conversation-turn` gains a `text/html` clue that may relate to content-type handling, but the evidence does not support a narrower conclusion.
- `debug-modal` shrinks substantially and loses account, active-browser-tab, active-conversation, worktree-cleanup, and host-allowance strings; only trimming or migration of debug content is supported.
- `esm` loses event-bus connection, queue, and dispatch logs; this does not prove that the event bus itself was removed.
- `browser-sidebar-tweaks` contains only a very small style change and remains low-value build evidence.
