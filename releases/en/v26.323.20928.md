> Third-party changelog · Unofficial · Based on reverse-engineered diffs of official update packages

The most visible change in this release is the addition of a Codex logo loading animation during the Webview startup phase. At the same time, frontend module names show signals pointing toward plugin pages and plugin settings.

## About This Report

This is not an official changelog, but a **reverse-engineered diff report based on officially distributed artifacts**. This is a historical backfill: both versions fall outside the official Sparkle delta update window, so the analysis directly compares two official full packages and then performs a structured bundle diff.

In terms of confidence, both artifacts came from official download URLs, and their SHA-256 hashes were recorded. Notarization verification returned `CSSMERR_TP_CERT_REVOKED` for both. The fact package indicates that this was caused by signing certificates for some early versions being revoked later. This affects whether the version can be safely installed by the system, but does not affect diff analysis of the bundle contents.

This report uses two evidence levels:

- **[Signal]**: Indirect evidence from frontend module names, binary size changes, and similar sources. This type of evidence points in a direction, but cannot reconstruct source-level intent from the artifact.

## ✨ Key Changes

### 1. New Codex Logo Loading Animation During Startup [Confirmed]

`webview/index.html` adds a full block of inline styles defining `.startup-loader`, `.startup-loader__logo`, fade-in animation, shimmer animation, and an embedded SVG mask for the Codex logo.

This means that before the main script is loaded, the app Webview now shows a more complete startup loading state. It also respects `prefers-reduced-motion: reduce`, handling the system preference for reduced motion.

Key evidence:

```diff

```
