# Codex Desktop 26.519.41501(macOS / Windows comparison)

> Third-party cross-platform changelog · Unofficial

This Codex desktop batch looks more like a cross-platform maintenance release: both platforms show app-layer package changes, but the readable evidence is clearer on macOS, while Windows shows stronger binary and package-structure signals.

One-liner: A cross-platform maintenance batch with shared app-layer churn, confirmed bundled plugin version bumps on macOS, and stronger runtime/package-size movement on Windows.

## About This Report

This report is based on reverse-engineered diffs of two officially signed distribution packages: the macOS side compares reconstructed full packages associated with Sparkle, and the Windows side compares reconstructed MSIX full packages. The macOS comparison is 26.519.31651 build 3017 → 26.519.41501 build 3044; the Windows comparison is 26.519.3891.0 → 26.519.11010.0. The release date for this batch is recorded from the input facts as 2026-05-27.

Evidence has two levels: [Confirmed] means the change comes from readable evidence such as plist, AppxManifest, skill documentation, type declarations, package.json, dependency lists, or other readable configuration/documentation. [Signal] means the evidence comes only from frontend module names, file trees, or size changes; it can point to a direction, but cannot reconstruct source intent.

Platform tags mean: `[macOS+Windows]` indicates both fact packs show the relevant change; `[macOS only]` indicates it currently appears only in the macOS fact pack; `[Windows only]` indicates it currently appears only in the Windows fact pack. This report deliberately ignores webview/assets and .vite/build hash-renaming noise.

## ✨ Key Changes

### The app-layer package changed on both platforms, but there is not enough evidence to identify a concrete new feature [Signal] [macOS+Windows]

Both platforms changed app.asar, the Electron application source archive. On macOS, internal files moved from 1576 to 1577, with 252 added, 251 removed, and 9 changed. On Windows, internal files moved from 1530 to 1531, with 521 added, 520 removed, and 12 changed.

The platform difference is that the macOS fact pack additionally lists one de-noised new frontend module, `computer-use-plugin-icon`; the Windows fact pack does not provide a de-noised new module name. So both platforms show app-layer repackaging/update signals, but only macOS exposes a module-name signal pointing toward a computer-use plugin icon.

### Browser and Chrome bundled plugin versions are confirmed to bump on macOS [Confirmed] [macOS only]

The macOS readable diff shows that the `browser` and `chrome` plugins inside the openai-bundled plugin set updated their `plugin.json` versions from `26.519.31651` to `26.519.41501`.

The Windows fact pack lists small size changes in Browser / Chrome related `browser-client.mjs` files, but does not provide a corresponding readable `plugin.json` diff. Therefore the plugin version bump cannot be extrapolated to Windows and is marked as macOS-only confirmed evidence.

### macOS adds a `computer-use-plugin-icon` frontend module signal [Signal] [macOS only]

After removing Vite hash noise, the macOS fact pack lists a new module named `computer-use-plugin-icon`. This indicates that the frontend package now contains a module name related to a computer-use plugin icon.

However, the presence of a module does not mean the feature is user-facing, nor does it prove any change to entry points, permissions, or backend capability. The Windows fact pack does not include the same module-name signal.

### Windows runtime components show larger size movement [Signal] [Windows only]

Windows file-level statistics show several larger runtime component size changes: `resources/codex.exe` decreased from 247.7 MB to 239.6 MB, a change of -8.11 MB; `resources/plugins/openai-bundled/plugins/chrome/app-server-runtime/codex` also decreased by 8.11 MB; `resources/codex-windows-sandbox-setup.exe` decreased from 8.3 MB to 6.1 MB, a change of -2.21 MB; meanwhile `resources/codex` increased from 219.5 MB to 221.7 MB, a change of +2.17 MB.

These are strong packaging/runtime-change signals, but the fact pack does not provide readable source or configuration diffs explaining the cause, so they should not be interpreted as performance, feature, or dependency-pruning claims.

### Windows removes the `bwrap` resource file [Signal] [Windows only]

The Windows file tree moved from 832 to 831 files, with 0 added, 1 removed, and 19 content changes. The removed file is `resources/codex-resources/bwrap`, sized 0.5 MB.

The fact pack only proves that this resource file disappeared from the new Windows package. It does not prove how sandbox paths, runtime policy, or platform support behavior changed.

### macOS native components changed slightly, with no new cua_node packages [Signal] [macOS only]

Several native components in the macOS package show small size changes, including `bare-modifier-monitor` at +0.02 MB, and `sparkle.node`, `sky.node`, `remote-control-device-key.node`, and `browser-use-peer-authorization.node` at roughly +0.01 MB each. `better_sqlite3.node` decreased from 2.0 MB to 1.9 MB, a change of -0.03 MB.

The macOS cua_node dependency change list shows 0 new packages. Windows also shows 0 new cua_node packages, but it does not list the same native component details, so the native-component discussion is treated as a macOS-only signal.
