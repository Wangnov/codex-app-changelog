## About This Report

This is not an official changelog. It is a **reverse-diff report based on official distribution artifacts**. The comparison covers macOS arm64 from `26.212.1823` build 661 to `26.217.1959` build 669, released on 2026-02-13 and 2026-02-18 respectively.

The evidence comes from Info.plist, the package.json diff, bundle file-tree statistics, newly added readable strings in main-process and frontend bundles, same-name frontend module string diffs, and SHA-256, signing, and notarization verification results. Because this is a historical backfill, the official appcast did not provide a delta package for this version pair, so the comparison was made directly from the two official full packages.

Evidence is split into two levels: **[Confirmed]** means there is direct human-readable evidence in the artifact, such as a configuration diff, readable string, signature, or checksum result. **[Signal]** means the evidence is indirect, such as frontend module names, same-name bundle string diffs, or binary size changes; it points in a direction, but cannot reconstruct source-level intent.

## ✨ Highlights

### 1. Thread handoff to local or worktree flows expanded [Confirmed]

The new main-process bundle contains multiple readable error paths and state messages related to thread migration, worktrees, branches, and stash handling. These point to a strengthened workflow for moving a current thread to local state or a new worktree. User-visible effects may include migration failure messages, default-branch constraints, target worktree opening, and handling of uncommitted changes.

Key evidence includes: `Failed to move thread to local`, `Failed to move thread to worktree`, `Move to worktree is not available on the default branch`, `Branch is already checked out in another worktree`, `You have uncommitted changes. Commit or stash them first.`, `Stashed Codex worktree changes`.

### 2. Frontend signals for fork slash command and local/worktree fork entry points [Signal]

Same-name frontend modules and localization resources added strings and state class names such as `composer.forkSlashCommand.*`, `move-thread-to-local`, `move-thread-to-worktree`, `move-to-local`, and `move-to-worktree`. These point toward UI exposure for forking the current conversation to local state or a new worktree, but the artifact does not allow source-level intent to be reconstructed.

Key evidence includes: `composer.forkSlashCommand.title`, `composer.forkSlashCommand.option.local.title`, `composer.forkSlashCommand.option.worktree.title`, `Button label for moving a local conversation to a new worktree`.

### 3. Model capacity and rate-limit messaging entered the UI surface [Signal]

Several language bundles added `codex.modelCapacityBanner.*`, `composer.mode.rateLimit.modelSectionLabel`, `composer.statusPlain.modelRateLimitSectionLabel`, and `composer.modelChangeDuringConversationWarning.toast`. These same-name frontend-module signals point toward UI messaging for model capacity, rate limits, and model changes during a conversation, but they do not establish complete source-level behavior.

Key evidence includes: `codex.modelCapacityBanner.title`, `codex.modelCapacityBanner.headline.noReset`, `codex.modelCapacityBanner.headline.withReset`, `composer.modelChangeDuringConversationWarning.toast`.

### 4. Permissions, experimental features, plan download, and sync prompts expanded in settings and thread UI [Signal]

Localization resources added strings related to permission dropdowns, experimental features, plan download, sync hints, and follow-up toasts. This points to continued UI expansion around settings and conversation helper actions. Because these come from same-name frontend module diffs, they only establish direction, not full behavior.

Key evidence includes: `settings.general.experimentalFeatures`, `settings.general.experimentalFeatures.empty`, `composer.permissionsDropdown.default.label`, `composer.permissionsDropdown.trigger.tooltip`, `localConversation.planSummary.download`, `localConversation.sync.buttonLabel.tooltip`, `pendingRequestStory.toastFollowupPrompt`.

### 5. Background terminal details and automatic context compaction gained new accessibility and status text [Signal]

The main frontend module added strings for background terminal completion, expanding/collapsing a details panel, and automatic context compaction. These signals point to more visible UI state for background terminals and context compaction, but remain frontend-bundle signals rather than source-level facts.

Key evidence includes: `Background terminal finished`, `Aria label for button that expands background terminals details panel.`, `Aria label for button that collapses background terminals details panel.`, `Automatically compacting context`.

### 6. New runtime paths for opening Xcode and handling notifications [Confirmed]

The main-process bundle added readable strings related to opening paths in Xcode and handling system notifications. This confirms those runtime paths exist in the new artifact. User-visible effects may include fallback behavior when opening a target path in Xcode, plus notification click and action handling.

Key evidence includes: `Opening path in Xcode via xed`, `Opening path in Xcode via LaunchServices fallback`, `show notification`, `notification click open`, `notification action`.

### 7. New cached ESLint script for development workflow [Confirmed]

`package.json` added a `lint:cached` script that runs ESLint with cache enabled and stores the cache at `node_modules/.cache/eslintcache`. This is a development workflow change rather than an end-user feature, but it is visible to anyone building or maintaining the app.

Key diff:

```diff
+    "lint:cached": "eslint src --max-warnings 0 --cache --cache-strategy content --cache-location node_modules/.cache/eslintcache",
```

### 8. Version and build number updated [Confirmed]

Both Info.plist and package.json show the version moving from `26.212.1823` to `26.217.1959`, and the build moving from `661` to `669`. The `app.asar` hash in `ElectronAsarIntegrity` also changed with the app content.

Key evidence: `CFBundleShortVersionString: 26.212.1823 → 26.217.1959`, `CFBundleVersion: 661 → 669`.
