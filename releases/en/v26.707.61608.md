# Codex Desktop 26.707.61608 macOS / Windows Comparison

> Third-party cross-platform changelog · Unofficial

The shared changes center on legacy ChatGPT app migration, bundled browser capability versioning, and build-pipeline updates; the macOS frontend also underwent a much broader module and interface reorganization.

Both packages add legacy ChatGPT app migration paths and refresh shared browser integrations, while the macOS sample carries a substantially broader frontend reorganization.

## About This Report

This report compares two official distribution-package pairs: macOS arm64 from 26.707.51957 (build 5175) to 26.707.61608 (build 5200), and Windows x64 MSIX from 26.707.6957.0 to 26.707.8168.0. The external versions are not aligned across platforms, so every conclusion is derived independently from its platform-specific fact package.

This is a historical reconstruction. The macOS comparison used two official full packages, both of which passed Developer ID notarization verification. The Windows fact package provides SHA-256 hashes for both full packages but does not include corresponding signature-verification output.

Evidence is divided into two levels:

- [Confirmed] covers plist values, plugin manifests, `package.json`, dependency lists, readable configuration, and new literals found in main-process or ordinary bundles. It proves that content entered the package, but a new string does not prove that its interface is enabled for every user.

- [Signal] covers changed strings and style sets in same-name frontend modules, module names, and compressed-size changes. It indicates a development direction but is not equivalent to complete source code or confirmed feature availability.

Platform labels are `[macOS+Windows]`, `[macOS only]`, and `[Windows only]`. Vite hash renames, `.vite/build` hash changes, and pure hash substitutions under `webview/assets` are excluded as noise.

### Legacy ChatGPT App Migration and Failure Recovery

[Confirmed] [macOS+Windows]

The new main-process or frontend bundles on both platforms contain these added literals:

- `ChatGPT Classic.app`
- `Failed to relocate legacy ChatGPT app`
- `Failed to restore legacy ChatGPT app`
- `com.openai.chat`

These strings confirm packaged handling for recognizing a legacy ChatGPT app, relocating or migrating it, and restoring it after a failed migration. They do not reveal the trigger conditions, destination, or user-facing flow, so they do not establish that migration runs automatically on every installation.

### Bundled Browser and Chrome Plugin Version Sync

[Confirmed] [macOS+Windows]

The `browser` and `chrome` plugin manifests on both platforms were updated from version `26.707.51957` to `26.707.61608`.

The `browser` manifest continues to define support for opening, inspecting, navigating, testing, clicking, typing into, and screenshotting local web targets through the in-app browser, including `localhost`, `127.0.0.1`, `::1`, and `file://`. It also directs explicit `@browser` or `@browser-use` requests to the in-app browser instead of system-open commands or generic web browsing.

The `chrome` manifest continues to position the plugin as automation over the user's existing Chrome tabs, authenticated sessions, cookies, extensions, and ChatGPT Chrome Extension setup, with purpose-built connectors, APIs, or CLIs preferred first.

The readable diff shows only version synchronization; it does not show changes to either description.

### Windows Computer Use Plugin Version Sync

[Confirmed] [Windows only]

The Windows `computer-use` plugin manifest was updated from `26.707.51957` to `26.707.61608`. Its description remains “Control desktop apps on Windows from ChatGPT through Computer Use.”

This change appears only in the Windows key-file diff. The Computer Use runtime did not grow on either platform, and neither platform added a `cua_node` dependency.

## ⚖️ Boundaries and Disclaimer

This is an unofficial third-party changelog derived from machine diffs of official distribution packages. It is not an OpenAI release note or product commitment.

The report does not redistribute application binaries. SHA-256 values and signature or notarization results are included only to identify the compared samples and support reproducibility.

[Confirmed] means that the corresponding configuration, manifest, dependency, list entry, or string is present in the package. For compressed-bundle strings, it does not automatically prove that a feature is visible or enabled for every user.

[Signal] means only that frontend modules, sizes, string sets, or style sets changed in a direction consistent with the interpretation. It is not source-code fact and cannot independently prove that a feature launched, was removed, or behaves in a particular way.

The platforms use different external distribution versions and package structures. Except where evidence is explicitly labeled `[macOS+Windows]`, a conclusion from one platform must not be projected onto the other. Corrections based on reproducible samples are welcome.
