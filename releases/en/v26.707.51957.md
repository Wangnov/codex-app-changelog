# Codex Desktop 26.707.51957 macOS / Windows Comparison

> Third-party cross-platform changelog · Unofficial

The shared direction is an expansion of work modes, Quick Chat, browser data controls, and model selection, with platform-specific UI and packaging changes.

本批次的共同主线是首页工作模式、Quick Chat、浏览器数据管理和模型选择能力继续扩展；Windows 与 macOS 的界面构建节奏并不完全一致。

## About This Report

This report compares two pairs of full official packages: macOS arm64 from `26.707.31428` build 5059 to `26.707.51957` build 5175, and Windows x64 MSIX from `26.707.3748.0` to `26.707.6957.0`. Version numbers and release timing may be offset between platforms, so every conclusion below is attributed independently to its own fact pack.

Both macOS samples passed Developer ID notarization checks and were signed by OpenAI OpCo, LLC. The old package SHA-256 is `ffdc351a507105d55d7464e3340302c758b8a54b926711c4b15bf373ccb47d64`, and the new one is `cb9655bb0adfc06287ef369b6d07ed324430d0668e1a9fa473c4e9d26f934f36`. The Windows sample hashes are `181faac13711ed38213c38644be2fc176f78f65a057cd167dc044935d3e81ab9` and `62892fd46d0035f6263a369b58c8c403523d5b15c0f4714712ab782e8aa15dd7`.

Evidence is classified into two levels:

- [Confirmed] covers plist data, manifests, plugin metadata, dependency inventories, human-readable configuration, and package strings outside same-name frontend-module clues. A string being present does not mean the feature is enabled for every user.
- [Signal] covers frontend module names, strings and style sets extracted from changed same-name compressed modules, module sizes, and build-graph direction. These cannot reconstruct source-level behavior.

Platform labels are `[macOS+Windows]`, `[macOS only]`, and `[Windows only]`.

## ✨ Highlights

### Work Modes and Home Positioning Were Rewritten [Signal] [macOS+Windows]

Frontend builds on both platforms add `ChatGPT Work`, `Build, debug, and ship`, `Create, learn, and explore`, and distinct home headings for states with and without a selected project. Older wording such as `For developers`, `For getting work done`, and the generic `What should we get done?` was removed or replaced.

On Windows, the evidence is concentrated in the changed `app-main` module. On macOS, corresponding clues are spread across the main page module and locale resources. Together they indicate restructuring of Codex versus ChatGPT Work positioning, the sidebar mode selector, and project-aware home states, but do not confirm rollout to every account.

### Quick Chat Is Being Connected More Directly to the Current Task [Signal] [macOS+Windows]

Multiple locale modules shared by both platforms add `quickChat.addToTask` and `quickChat.addToTaskTooltip` while removing `quickChat.addToComposer`. Other translations likewise refer to adding the current chat to the current task.

This suggests a transition from simply copying Quick Chat content into the composer toward attaching a chat to a task. macOS additionally includes accessibility strings for opening Quick Chat history, returning to the current chat, starting a conversation, and resizing chat width and height. Those extra interaction clues do not appear in the Windows sample.

### Pull Request Merge Blocking Gains More Detailed States [Signal] [macOS+Windows]

Locale modules on both platforms add merge-disabled state keys for:

- `blocked`
- `draft`
- `failingChecks`
- `pending`
- `pendingChecks`
- `unknown`

Related strings mention fixing failed checks before merging, GitHub still determining mergeability, and another Pull Request update already being in progress. The evidence supports more detailed disabled-button reasons and status feedback, but not any conclusion about the underlying merge implementation.

### Embedded-Browser Data, Download, and Site-Permission Settings Expand [Signal] [macOS+Windows]

The `browser-use-settings` module grows from near-empty to approximately 0.1 MB on both platforms and adds text for:

- Accessing history without asking
- Browser downloads and download history
- Browsing history
- Contact information
- Extensions
- Password management
- Site settings
- Expanding or hiding individual data-clearing options

Windows additionally includes `Add site permission` and an accessibility label for a disabled toggle when a plugin is unavailable. macOS additionally includes back and forward labels for the settings view. Because all of this comes from changed same-name frontend modules, it remains a settings and permission-model signal.

### Ultra Model-Picker Control and a DeviceCheck Error Path Enter Both Packages [Confirmed] [macOS+Windows]

Main-process or frontend bundles on both platforms add:

- `Whether Ultra appears in the model picker slider`
- `DeviceCheck registration returned no cookie`

The first confirms that control text governing whether Ultra appears in the model-picker slider is present in both packages. The second confirms the presence of an error path for DeviceCheck registration returning no cookie. Neither proves that Ultra is available to all users or reveals the registration recovery logic.

### Windows Adds a Multi-Agent Parallel-Delegation Home Banner [Signal] [Windows only]

The Windows `home-announcements` module adds a multi-agent composer banner and related state keys, including:

- `Delegate work to subagents that work in parallel. Note: may increase token usage.`
- `has-seen-multi-agent-composer-banner`
- A dismissal label for the multi-agent banner
- Dismissal labels for Fast mode, rate-limit resets, gift credits, and backend-driven home banners

This shows that the Windows home-banner system contains frontend resources for parallel delegation and several account-related notices. The macOS fact pack contains a new module combination involving `home-announcements`, but not equally explicit multi-agent banner strings, so this item is Windows-only.

### macOS Refines Plan Summaries and File-Diff Interaction [Signal] [macOS only]

The changed macOS onboarding module adds text for:

- Closing the plan side panel
- Collapsing and expanding plan summaries
- Opening a plan summary in a new window
- Toggling a file-diff section
- Opening a large diff in an editor
- Opening the current file from the diff header
- `Binary file not shown`

Related classes include `hunk-actions`, diff-header containers, and hover or focus visibility states. Together these point to layout and accessibility work around plan summaries, file-level diffs, and hunk actions.

### macOS Adds Project Selection, File Input, and Resizable Chat Clues [Signal] [macOS only]

The changed macOS main-page module adds:

- A label for the ChatGPT composer file input
- Project-selector labels for home states with and without a selected project
- `Add local project`
- A Quick Chat history entry
- Chat width and height resizing
- Labels for ChatGPT project selection and project creation

New module combinations also connect `select-workspace-page`, `projects-index-page`, and local and remote conversation pages. This is a mutually reinforcing group of project-context and window-interaction signals, but the fact pack contains no corresponding source or settings manifest.

### macOS Expands Automation Scheduling and Remote-Project State [Signal] [macOS only]

A macOS project and local-environment module grows from near 0 MB to 0.4 MB and adds two explicit errors:

- An automation does not exist in the app, may have been deleted manually, and cannot be updated.
- An automation schedule has no future runs and must contain an occurrence after the current time.

The same module adds `active-remote-project-id`, `active-workspace-roots`, `host-id-remote-control-allowed`, `hook/started`, `hook/completed`, `hotkey-window-projectless-default-enabled`, and `mac-menu-bar-enabled`. Together these signal broader automation validation, remote control, workspace-root, hook-state, and menu-bar settings surfaces.

### macOS Performs Broad Frontend Rechunking [Signal] [macOS only]

After Vite hash-renaming noise is removed, new macOS module combinations cover:

- `codex-micro-settings~codex-micro-bridge`
- `cloud-environments-settings`
- `remote-connections-settings`
- Local and remote conversation pages
- `automations-page`
- Skills, plugins, and settings
- AppGen library, settings, and publication
- Pull Request routes and code review
- Notebook preview and artifact tabs
- `home-ambient-suggestions-content`

Removed chunks include standalone `codex-micro-settings`, `codex-micro-joystick-hud`, `debug-settings`, and multiple older onboarding, Quick Chat, AppGen, and Pull Request combinations. This primarily reflects a reorganization of the frontend dependency graph and loading boundaries. Descriptive module names provide direction, but do not establish new source-level capabilities.

### Bundled Computer Use and Chrome Plugin Versions Advance on macOS [Confirmed] [macOS only]

The `computer-use` plugin manifest moves from `1.0.1000366` to `1.0.1000387`. The `chrome` plugin manifest moves from `26.707.31428` to `26.707.51957`.

The Computer Use description still targets control of macOS desktop applications through ChatGPT while preferring purpose-built connectors, APIs, or CLIs. The Chrome plugin still targets automation that depends on existing tabs, signed-in sessions, cookies, extensions, and ChatGPT Chrome Extension setup.

No plugin-inventory addition, skill-document change, or service-entry change was supplied. The evidence therefore confirms version updates to two existing plugins, not a new plugin or a newly added complete capability surface.

### Computer Use Runtime Size and Dependency Counts Stay Stable [Confirmed] [macOS+Windows]

The macOS `cua_node` total remains 365.7 MB, while Windows remains 287.2 MB. Both changes are `+0 MB` and `+0%`, and both platforms add zero dependency packages.

On macOS, the two packaged Computer Use service and client copies each change by roughly `+0.12 MB` to `+0.13 MB`, while the aggregate runtime total remains unchanged. The Windows fact pack does not list corresponding Computer Use binary-level changes.

### Chromium Base Advances from 150.0.7871.101 to 150.0.7871.115 [Confirmed] [macOS+Windows]

The macOS Info.plist changes `ChromiumBaseVersion` from `150.0.7871.101` to `150.0.7871.115`, accompanied by replacement of the versioned Codex Framework directory.

Windows adds `150.0.7871.115.manifest` and removes `150.0.7871.101.manifest`. Both platforms therefore show the same Chromium patch-base transition, although their binary size movements differ: the new macOS Framework executable is 254.3 MB versus 255.4 MB for the replaced file, while Windows `chrome.dll` drops from 312.4 MB to 311.9 MB.

### Bundled Windows DevTools Updates to Version 150 [Confirmed] [Windows only]

Windows updates its DevTools release material from version 149 to 150. The bundled notes list:

- Experimental memory-debugging tools in DevTools for agents 1.4 and later
- Experimental TOON support for structured-content output
- Improved skill discovery
- AI assistance walkthrough widgets for performance network tracks, timeline events, network requests, syntax-highlighted source, and inspected file lists
- Direct editing of `@container` and `@function` rules in the Styles panel
- A Chrome 148–150 highlights video entry

These are Chromium DevTools package changes and should not be treated as proof that Codex itself added identically named capabilities.

### Native Quit-Menu Wording Is Refined Across Locales [Confirmed] [macOS+Windows]

Thai, Gujarati, Kazakh, and Lithuanian native-menu files receive matching-direction wording changes on both platforms, mainly around quit-confirmation titles, quit buttons, and references to the current device. These are localization refinements and do not change the existing meaning that quitting interrupts local tasks and prevents scheduled tasks from running while the application is closed.
