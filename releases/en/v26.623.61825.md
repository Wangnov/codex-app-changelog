## ✨ Key Changes

### [Confirmed] [macOS+Windows] Bundled Browser and Chrome plugin versions moved with the package

Both macOS and Windows show the bundled `browser` and `chrome` `.codex-plugin/plugin.json` versions changing from `26.623.42026` to `26.623.61825`. The facts do not show semantic description changes for these two plugins. Browser still targets the in-app browser, local targets, localhost, file URLs, screenshots, clicks, and typing; Chrome still targets automation that depends on the user’s existing Chrome state.

The platform difference is that Windows also shows the `computer-use` plugin manifest version changing from `26.623.42026` to `26.623.61825`, with a description for controlling Windows desktop apps from Codex through Computer Use.

### [Confirmed] [Windows only] Computer Use plugin manifest appears in the readable diff

Windows includes `resources/plugins/openai-bundled/plugins/computer-use/.codex-plugin/plugin.json`, with its version updated and the description “Control desktop apps on Windows from Codex through Computer Use.” This is Windows-only manifest evidence.

The macOS fact package has no matching `computer-use` plugin manifest diff. The `cua_node` runtime total is unchanged on both platforms: macOS `377.4 MB → 377.4 MB`, Windows `298.9 MB → 298.9 MB`, and both report `0` new `cua_node` dependencies.
