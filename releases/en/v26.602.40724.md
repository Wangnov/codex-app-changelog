## About This Report

This report is based on reverse diffs of official signed distribution packages: macOS uses full Sparkle packages, and Windows uses MSIX packages. The macOS comparison is `26.602.30954` build `3575` → `26.602.40724` build `3593`; the Windows comparison is `26.602.3474.0` → `26.602.4764.0`. The batch release date is recorded as `2026-06-05` from the provided facts.

Evidence is split into two levels: [Confirmed] means readable evidence from plist, AppxManifest, skill docs, type declarations, package.json, dependency manifests, or similar files; [Signal] means only module names, file counts, or size changes are available, which can indicate direction but cannot reconstruct source intent.

Platform tags mean: `[macOS+Windows]` is supported by both fact packs; `[macOS only]` is proven only by the macOS fact pack; `[Windows only]` is proven only by the Windows fact pack. “Only” here refers to what this fact set can prove, not necessarily what exists in the shared source tree.

## ✨ Highlights

### Small App-Layer Update

[Signal] [macOS+Windows]

The main `app.asar` archive changed slightly on both platforms, with `+0.01 MB` on each side. On macOS, `Contents/Resources/app.asar` changed from `143.1 MB` to `143.1 MB`; on Windows, `resources/app.asar` changed from `146.9 MB` to `146.9 MB`.

The platform difference is important: the macOS fact pack includes readable file diffs, allowing the change to be narrowed down to plugin metadata version sync. The Windows fact pack only provides file-tree and app.asar-level statistics, so the macOS-specific plugin change cannot be directly attributed to Windows.

### Bundled Browser and Chrome Plugin Version Sync

[Confirmed] [macOS only]

In the macOS package, two openai-bundled plugin `plugin.json` files updated their version from `26.602.30954` to `26.602.40724`:

- `plugins/browser/.codex-plugin/plugin.json`
- `plugins/chrome/.codex-plugin/plugin.json`

This confirms that the Browser and Chrome plugin metadata inside the macOS package now tracks the app version. The Windows fact pack does not include corresponding readable `plugin.json` diffs, so this report cannot mark the change as `[macOS+Windows]`.

### No New cua_node Dependencies

[Confirmed] [macOS+Windows]

Both fact packs report `0` newly added `cua_node` packages. There is no evidence in this batch that the native/Node dependency set expanded.

Platform difference: macOS and Windows agree on this point; neither platform added packages.

### Stable Outer File Tree

[Confirmed] [macOS+Windows]

The outer file count stayed unchanged on both platforms: macOS remained `2075 → 2075`, and Windows remained `8724 → 8724`. Both also show `0` added and `0` removed outer files, so there is no evidence of top-level package expansion or contraction.

The difference is in changed-file counts: macOS reports `53` changed outer files, while Windows reports `15`.

### Equal Add/Remove Counts Inside app.asar, Mostly Build Hash Noise

[Signal] [macOS+Windows]

The internal `app.asar` file count stayed unchanged on both platforms, while both report `17` added and `17` removed files. macOS moved `2830 → 2830` with `12` changed files; Windows moved `2778 → 2778` with `11` changed files.

The fact packs explicitly state that many changes under `webview/assets` are Vite build hash renames and should be treated as noise, not as feature additions or removals. This should only be read as a signal that the app layer was rebuilt and changed slightly.
