# Codex Desktop 26.707.62119: macOS / Windows Comparison

> Third-party cross-platform changelog · Unofficial

Both platform samples update the bundled Browser and Chrome plugin version markers; broader interface and interaction changes are concentrated in the macOS sample, while Windows additionally includes a Computer Use plugin update.

Browser and Chrome plugin versions moved forward on both platforms, while broader UI signals are concentrated in the macOS sample and Windows additionally updates its Computer Use plugin.

## About This Report

This report compares two independently obtained official signed distribution package samples:

- macOS arm64: `26.707.61608` build 5200 → `26.707.62119` build 5211.
- Windows x64 MSIX: `26.707.8168.0` → `26.707.8479.0`.

The outer package versions do not align across platforms. This report combines only changes independently supported by both fact packs and does not project frontend or runtime findings from one platform onto the other. The Windows fact pack incorrectly labels its “platform/architecture” field as macOS arm64; following the supplied input definition, this report treats that sample as Windows x64 MSIX.

Evidence is divided into two levels:

- [Confirmed] covers plist data, plugin manifests, service or dependency inventories, signature verification, readable configuration, and package strings outside the “same-name frontend module clues” section.
- [Signal] covers frontend module names, compressed bundle sizes, and string or style-set changes inside same-name frontend modules. These show a direction present in the build, but do not prove general availability.

Platform labels `[macOS+Windows]`, `[macOS only]`, and `[Windows only]` identify the sample that actually supplies the evidence. Vite hash renames are excluded as build noise.

The fact packs do not include separate sections for newly bundled plugin details, new main-process readable strings, newly added components/resources, or complete plugin inventory changes. Neither Bundle tree added or removed files, so this report does not describe version updates to existing plugins as new plugin installations.

## ✨ Highlights

### Browser and Chrome Bundled Plugins Advance Together [Confirmed] [macOS+Windows]

The `browser` and `chrome` plugin manifests on both platforms update their internal versions from `26.707.61608` to `26.707.62119`.

The Browser description continues to cover the ChatGPT in-app browser and aliases such as `@browser` and `@browser-use`, including opening, inspecting, navigating, testing, clicking, typing, and screenshotting local web targets. The Chrome description continues to cover automation that depends on existing tabs, signed-in sessions, cookies, extensions, and ChatGPT Chrome Extension state. The concrete diff changes only the version fields, so it does not establish that these capabilities were first introduced in this batch.

The outer package versions differ: the macOS application and plugin both use `26.707.62119`, while the Windows MSIX outer version is `26.707.8479.0` and its internal plugins advance to `26.707.62119`.

### Windows Computer Use Plugin Advances to the New Internal Version [Confirmed] [Windows only]

The Windows `computer-use` manifest advances from `26.707.61608` to `26.707.62119`. Its description remains control of Windows desktop applications from ChatGPT through Computer Use.

The diff confirms only an update to an existing plugin. It adds no plugin files and no Computer Use dependencies. Windows `cua_node` content remains `287.2 MB`, with zero newly added packages. The macOS sample has no corresponding manifest change, so this is not classified as shared.

### Auto-Review and Permission-Request States Enter the Frontend Change Surface [Signal] [macOS only]

The macOS onboarding module adds groups of strings for:

- a reviewer agent evaluating a request;
- review completion, early termination, and timeout;
- auto-review of network access;
- permission requests with or without reasons;
- MCP tool calls;
- single-file edits.

All of these strings come from the same-name frontend module clues, so they only establish that related statuses and action summaries entered this frontend build direction. They do not prove that auto-review is enabled by default.

In the same batch, the Quick Chat/ChatGPT conversation module removes `Allow once`, `Allow this conversation`, `Always allow`, approval-scope options, denial-button strings, and the `data-codex-approval-surface` style marker. This suggests approval UI restructuring, but does not establish that the approval capabilities were removed.

The Windows fact pack contains no corresponding frontend module or string evidence.

Together, these signals point to Quick Chat window resizing, history navigation, and layout restructuring, but remain frontend signals rather than confirmed availability.

This is a relatively strong signal for organizing browser, review, terminal, file-browser, and task-detail views into side or bottom panels. The fact pack does not prove that every entry point is visible.
