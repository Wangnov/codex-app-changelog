# Codex Desktop 26.707.31428 macOS / Windows Comparison

> Third-party cross-platform changelog · Unofficial

This batch shifts the desktop experience from chats toward tasks, adds complete Deep Research and interactive visualization capability surfaces, and begins a system-level migration toward the ChatGPT brand.

This batch shifts the desktop experience from chats toward tasks, adds full Deep Research and interactive visualization capability surfaces, and begins a system-level migration toward the ChatGPT brand.

## About This Report

This report compares two independent sets of official signed full packages:

- macOS arm64: `26.623.141536` build 4753 → `26.707.31428` build 5059.
- Windows x64 MSIX: `26.623.19656.0` → `26.707.3748.0`.

Version numbers and package contents are cited independently and are not interchangeable. The macOS comparison used two official full packages because the incremental packages had aged out of the Sparkle window. Both macOS builds passed Notarized Developer ID verification under OpenAI OpCo, LLC. The Windows facts provide SHA-256 values for both full packages.

Evidence levels:

- [Confirmed] covers plist entries, AppxManifest data, plugin manifests, skill documentation, type declarations, dependency lists, service entry points, human-readable configuration, and new strings demonstrably present in bundles.
- [Signal] covers frontend module names, string or style changes inside same-name compressed modules, size changes, and packaging layout. These indicate development direction but do not prove general availability.

Platform markers are `[macOS+Windows]`, `[macOS only]`, and `[Windows only]`. Hash-renaming noise under locations such as `webview/assets` and `.vite/build` has been ignored.

Neither fact packet contains a content-security-policy change section, so this report makes no conclusion about whether CSP changed.

## ✨ Highlights

### Deep Research becomes a complete bundled plugin [Confirmed][macOS+Windows]

Both platforms add `deep-research` 0.1.1. This is more than a frontend entry: the plugin contains a manifest, agent configuration, skill instructions, research methodology, report contract, and assets. Its default prompts explicitly cover complex investigations, option comparisons, and reconciliation of conflicting evidence.

The skill contract requires:

- A dedicated research subagent and at least two investigative passes.
- Preference for original research, official records, standards, and first-party material.
- Explicit separation of fact, inference, uncertainty, and conflicting evidence.
- A cited DOCX as the required final artifact, with rendered pages inspected.
- Stopping when the Documents capability is unavailable instead of silently substituting Markdown, PDF, or chat prose.

The package therefore contains an end-to-end research-delivery workflow, not merely a web-search prompt template.

### Visualize adds an interactive visualization capability surface [Confirmed][macOS+Windows]

Both platforms add `visualize` 1.0.11, including a plugin manifest, full skill instructions, HTML/CSS templates, a rendering script, agent configuration, and light/dark assets.

The documented scope includes interactive charts, maps, diagrams, simulations, 3D models, data explorers, and UI previews. The skill distinguishes static Mermaid output from dynamic HTML and defines file placement, size limits, resource loading, script validation, and browser-preview procedures. Both bundles also contain error paths for empty, invalid, and sandbox-unavailable `codex-inline-vis-screenshot` requests.

This confirms that visualization creation, rendering, and screenshot paths entered both packages; it does not establish that every account can see the entry point.

### The workflow systematically moves from “chats” to “tasks” [Confirmed][macOS+Windows]

Both main-process bundles add `New Task`, `New Projectless Task`, `Rename task`, `Attach image to task`, `Recent Codex tasks`, `Opened desktop files in projectless task`, and a command for creating a projectless task.

Corresponding older strings such as `New Chat`, `Rename chat`, `Attach image to thread`, and `Recent Codex threads` are removed. Automation paths also add `Migrated automations to project targets`, missing-project handling, and a one-folder project requirement for worktree automations.

Native menu resources now include localized quit confirmation: active local tasks will be interrupted, and scheduled tasks will not run while the app is closed. This is a direct native-resource change, not only frontend terminology.

### Task sidebar, scheduling, and project organization are being reorganized [Signal][macOS+Windows]

Changes in matching modules point to:

- `automations-page` adopting Scheduled Tasks terminology, status filters, paused states, immediate addition of suggested tasks, creation actions, and list navigation.
- `local-conversation-page` adding a ten-minute PR heartbeat, PR auto-fix, and auto-merge controls.
- `worktree-environment-dropdown` adding local working-tree selection, remote starting branches, branch search, and loading errors.
- `worktrees-settings-page` adding automatic deletion of old worktrees, a deletion limit, and save failures.
- `home-ambient-suggestions-content` shifting toward refreshable and dismissible suggestions, with “Learn my writing style,” `Common patterns`, `Do and avoid`, and `Overall voice`.

Windows additionally contains denser signals in `sidebar-flat-sections`, `app-main`, and `open-project-setup-dialog`: task filters, cloud/local task groups, project source folders, bulk task archiving, and ChatGPT project operations.

### ChatGPT branding reaches system and runtime layers [Confirmed][macOS+Windows]

Shared application configuration changes `codexAppBrand` from `codex` to `chatgpt` and moves the entry point from `bootstrap.js` to `early-bootstrap.js`. Both bundles add extensive ChatGPT for Chrome, ChatGPT webview, ChatGPT window, sign-in, setup-config, and workspace-session wording while removing corresponding Codex wording.

The platform details differ:

- `[macOS only]` `CFBundleDisplayName`, `CFBundleExecutable`, `CFBundleName`, and the URL type name change from Codex to ChatGPT. ChatGPT icon and DMG resources are added, and camera, microphone, Desktop-folder, and Apple Events permission text is renamed.
- `[Windows only]` the package adds `ChatGPT.exe` plus ChatGPT tray and application icons, but `package.json` still uses the Windows package identity `OpenAI.Codex`. The evidence therefore supports an in-progress migration, not a completed Windows package-identity rename.

### `.skill` becomes a system-level file type on both platforms [Confirmed][macOS+Windows]

The Windows AppxManifest adds a `.skill` file association.

macOS provides a more complete definition: a `Codex Skill` document type, `.skill` extension, `application/vnd.openai.codex.skill` MIME type, `com.openai.codex.skill` UTI, and Owner/Viewer registration. The old Chromium `.crx` exported type is replaced.

Skills are therefore entering OS integration as standalone files the desktop app can claim and open.

### Browser-operation APIs and failure handling become more explicit [Confirmed][macOS only]

The Browser and Chrome plugin documentation in the macOS package adds:

- `locator.evaluate(...)` for read-only element scope.
- Direct `locator.selectOption(...)` use for native HTML `<select>` controls.
- A rule that empty or stale tab lists and missing Playwright helpers are not automatically browser disconnections.
- Browser reselection only when a disconnection is explicitly reported.
- Stricter bot-detection classification that excludes bare 403 responses, administrator blocks, proxy failures, DNS/TLS errors, timeouts, and connection resets.
- JSON serialization of bot-detection results.

These are direct plugin API declaration and operating-document changes. Windows also contains shared runtime strings such as `locator.evaluate requires a pageFunction`, but its fact packet does not include equivalent complete plugin-document diffs, so the full rule set is not projected onto Windows.

### Browser sidebar, authentication, and tab-budget paths enter both bundles [Confirmed][macOS+Windows]

New strings cover browser Back, Forward, address-bar focus, queued tab routes, tabs closed by Browser Use, windows closing while webviews attach, authenticated-target checks, and tab selection or suspension caused by budget limits.

Both platforms also contain:

- The ChatGPT Chrome native messaging host.
- Missing webview auth-token providers and cookie-clear failures.
- Rejection paths preventing DeviceCheck tokens from being attached to unexpected URLs.
- The `x-i-am-a-browser: true` marker.

These strings confirm that the commands, validation, and error paths exist in the packages, not that every browser entry point is enabled by default.

### Git snapshots, commit generation, and PR review expand [Confirmed][macOS+Windows]

Both bundles add strings covering:

- Reading, staging, and writing tracked working-tree and turn-diff snapshots.
- Stale Git snapshots, object directories, HEAD trees, untracked files, and binary patches.
- Missing PR head revisions, repository requirements, and failed PR review submission.
- Starting `/review` in the current task or in a separate review task.
- Choosing whether Git Review uses all sources or only recorded Last Turn changes.
- Commit-message bounds, fallback rules, and formatting constraints.

Matching `local-conversation-page` modules also add `Auto-fix PR`, failing-check repair, comment handling, reviewer selection, and a PR-status dropdown. Their specific UI presentation remains a [Signal][macOS+Windows].

### Plugin, Skill, and MCP management continue to converge [Signal][macOS+Windows]

`plugins-page` adds signals for switching managed extension types, plugin suggestions, MCP HTTP headers, environment variables, Bearer tokens, server authentication, and navigation back to MCP settings. The macOS `skills-page` module also adds New Skill, reload, search, recommended, and installed sections.

The shared sandbox preload adds an optional `notifyMcpAppsMcpNotification` port while retaining validation of required ports. [Confirmed][macOS+Windows]

The Windows MCP capability module additionally references subscribable resources, subscription cleanup, non-writable resources, and routing follow-up prompts to new tasks or worktrees. [Signal][Windows only]

### Sites gains a design service, interactive capabilities, and more reliable previews [Confirmed][macOS only]

The macOS Sites plugin moves from 0.1.21 to 0.1.27 and adds:

- A `.mcp.json` service entry point.
- The `sites-design-picker` MCP capability.
- `Interactive` and `Write` capability declarations.

The vinext starter adds a rendered-HTML test and `react-loading-skeleton` 3.5.0. Wrangler and Miniflare state is kept inside the project. Under the macOS Seatbelt sandbox, the preview server switches to polling because FSEvents is blocked, preserving HMR.

Both platforms contain frontend signals for custom domains, DNS records, status refresh, and domain removal, but only macOS provides the direct plugin-manifest and template-configuration evidence above. [Signal][macOS+Windows]

### File drag and theme initialization move into preload [Confirmed][macOS+Windows]

The shared preload adds `startFileDrag` IPC, preload-start timing, and immediate application of `electron-dark` or `electron-light` once the document element exists. This creates new bridge support for native file dragging, startup timing, and consistent initial theming.

### Computer Use runtime is slimmed down and its browser backend replaced [Confirmed][macOS+Windows]

Both platforms remove `@oai/cdp-browser-backend@0.2.0` and update `@oai/sky` from 0.4.19 to 0.4.20.

Runtime totals change independently:

- macOS: 377.4 MB → 365.7 MB, approximately -12 MB and -3%.
- Windows: 298.9 MB → 287.2 MB, approximately -12 MB and -4%.

The macOS `cua_node` manifest further shows a runtime update from `0.0.2/20260620010411-120c650ffb83` to `0.0.3/20260708050802-66a5641b5362-pr-1103797`, with the `node_repl` archive moving from `20260619.1` to `20260707.2`.

### Computer Use and Record & Replay adjust capability boundaries [Confirmed][macOS only]

Computer Use moves from 1.0.857 to 1.0.1000366. Its description changes from controlling Mac apps through Codex to controlling them through ChatGPT and explicitly prefers purpose-built connectors, APIs, or CLIs.

Record & Replay adds two constraints:

- Before creating a skill, it must establish the reusable workflow, intended outcome, and which demonstrated values should become inputs; material ambiguity requires questions first.
- Skill Creator structural validation does not prove that a recorded Computer Use workflow can be reproduced. After creation, the user must receive a summary of steps, inputs, and assumptions.

Neither the previous nor current Windows plugin list contains `record-and-replay`, so this change cannot be projected onto Windows.

### Windows has unique model, sidebar, and DevTools signals [Signal][Windows only]

Windows module changes show:

- Fast mode, “1.5x speed,” higher usage consumption, and advanced Model, Effort, and Speed controls in `model-and-reasoning-dropdown`.
- New modules including `activity-grouping-version`, `task-row-status-indicator`, `sidebar-unified-layout`, `thread-search-results`, `review-file-tree-pane`, `generated-image-preview`, and `wsl-remote-connections-feature`.
- New `hatch-pet` Skill files and preparation script, plus pet egg, spritesheet, detection-history, and battery-status modules.
- Significant growth in `review-file-source-tab`, with richer document/code viewing and accessibility states.

Windows DevTools also directly adds object-tree expansion tracking in `ObjectPropertiesSection.js`, recording properties, internal properties, array ranges, and accessor paths and restoring expansion state when a tree is rebuilt. [Confirmed][Windows only]

These findings apply only to the supplied Windows version interval and do not establish permanent absence on macOS.

### macOS system permissions and runtime framework update [Confirmed][macOS only]

macOS adds a location permission message: “ChatGPT uses your location to provide location-based answers.”

The Chromium base moves from 149.0.7827.197 to 150.0.7871.101, with the main Framework switching accordingly. A new `codex-code-mode-host` binary enters the resources. Windows also adds `.exe` and extensionless code-mode-host binaries, but their presence alone remains a packaging-direction signal. [Signal][macOS+Windows]

## ⚖️ Boundaries and Disclaimer

This is an unofficial third-party analysis based on machine-extracted differences from official distribution packages. It is not an OpenAI release note.

No application binaries are redistributed. A bundled string proves only that wording, a command, a setting, or an error path entered the distribution; it does not prove general availability. [Signal] findings are not source-code facts.

Frontend hash renames, generic icon sets, and dependency-splitting noise have been excluded where possible. Platform release cadence, version numbers, and packaging contents may be offset, so every platform-specific conclusion is limited to the supplied samples. Reproducible corrections are welcome.
