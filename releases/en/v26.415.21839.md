# Codex Desktop `26.415.21839` build 1763

> Third-party changelog · Unofficial · Based on reverse diffing official distribution packages

This release moves from `26.415.20818` to `26.415.21839`, with the clearest changes around the workspace file tree, file opening flows, and a new native helper.  
English one-liner: this build adds a native Launch Services helper and refines workspace file-tree/open-with flows.

## About This Report

This is not an official changelog. It is a **reverse diff report based on official distribution artifacts**. This entry is a historical backfill: the official appcast did not provide a delta package for `26.415.20818` → `26.415.21839`, so both official full packages were downloaded and compared directly.

Evidence is classified into two levels:

- **[Confirmed]**: human-readable evidence such as Info.plist values, package.json diffs, signing/notarization results, readable strings in bundles, and actual added files.
- **[Signal]**: indirect evidence from same-name frontend modules, class names, or size changes. These point in a direction, but cannot be reconstructed into source-level intent.

The comparison covers macOS arm64, from `26.415.20818` build `1727` to `26.415.21839` build `1763`.

## ✨ Highlights

### 1. New macOS Launch Services Native Helper [Confirmed]

`Contents/Resources/native/launch-services-helper` is a new file in this build, about `0.1 MB`. `package.json` also adds `pnpm exec node ./scripts/build-launch-services-helper.mjs` to both the `dev` and `owl` scripts.

This means the Codex desktop app now ships a native helper aimed at system Launch Services. Combined with same-name frontend-module strings for opening workspace files in a specific app, opening them in the preferred app, and loading app options [Signal], this points to an improved workspace file opening experience, though the full source-level intent cannot be reconstructed from the artifact.

Key evidence:

```diff
- "dev": "pnpm run rebuild:sqlite && pnpm --dir .. run generate:third-party-notices && cross-env NODE_ENV=development electron-forge start",
+ "dev": "pnpm run rebuild:sqlite && pnpm exec node ./scripts/build-launch-services-helper.mjs && pnpm --dir .. run generate:third-party-notices && cross-env NODE_ENV=development electron-forge start",
```

### 2. Workspace File Tree Is More Explicitly Treated as a Side-Panel Capability [Signal]

The same-name frontend module `index` adds strings such as `Toggle File Tree`, `Select file tree section`, `Label for showing all files in the thread file tree panel`, and `Label for showing changed files in the thread file tree panel`. It also removes several older strings centered on `thread summary right panel`.

This points to a reorganization of the right-side panel around file tree and summary sections, with clearer file-tree entry points, filters, and empty-folder states. Because these clues come from same-name frontend module diffs, they are only [Signal] for UI, state, layout, or interaction direction.

Additional evidence: the main/frontend bundle adds the readable string `Toggle File Tree` [Confirmed].

### 3. Workspace Browser Shows More Detailed Open-With Menu Signals [Signal]

The `index` module adds workspace browser context-menu strings for adding a file to the current chat, opening a file in a specific app, opening it in the preferred app, and choosing an app from a submenu.

This suggests the workspace file browser is gaining a more complete context-menu and external-app opening flow. Since these strings come from a same-name frontend module, they remain [Signal] and do not prove implementation details at the source level.

### 4. Main Bundle Now Contains Windows Store Update-Check Paths [Confirmed]

The new bundle adds the readable strings `Checking Windows Store for package updates` and `Windows Store package update check completed`.

Although the analyzed package is macOS arm64, these strings definitely exist in the new artifact. They show that shared code or packaging logic now contains a Windows Store package update-check path. This does not imply a new visible UI in the macOS build.

### 5. Settings Frontend Module Shrinks Substantially, Pointing to Migration or Splitting [Signal]

The `general-settings` module drops from about `0.1 MB` to about `0.0 MB`, a reduction of `93277 bytes`. It also removes multiple settings-related strings, including `Agent environment`, `Allow Codex to control the in-app browser for browser tasks. Restart Codex after changing this setting.`, `Allows other signed-in Codex clients to connect to this computer`, and text for code/UI font-size adjustments.

This points to changes in the settings page’s internal structure, loading boundaries, or presentation of related settings. Because the evidence comes from same-name frontend module diffs, it cannot confirm whether those settings were removed, moved, lazy-loaded, or reorganized.

### 6. Version, Build, and app.asar Integrity Hash Updated [Confirmed]

In `Info.plist`, `CFBundleShortVersionString` changes from `26.415.20818` to `26.415.21839`, and `CFBundleVersion` changes from `1727` to `1763`. `ElectronAsarIntegrity/Resources/app.asar/hash` also changes.

This confirms a real build update rather than only an outer signing or metadata replacement.
