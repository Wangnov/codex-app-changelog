# Codex Desktop 26.715.21316 macOS / Windows Comparison

> Third-party cross-platform changelog · Unofficial

The shared direction in this batch is a further shift from local tasks toward a unified Chat model, alongside broader project, pull-request, automation, browser, remote-connection, and inline-visualization surfaces.

Across both packages, this batch shifts local work toward a chat-centered model while expanding projects, pull requests, automations, browser workflows, remote connections, and inline visualization.

## About This Report

This report compares the following official full-package ranges:

- macOS arm64: `26.707.91948` build 5440 → `26.715.21316` build 5484.
- Windows x64 MSIX: `26.707.12708.0` → `26.715.2236.0`.

Both macOS packages passed notarization validation and identify OpenAI OpCo, LLC as their origin. Version numbers, file statistics, and sizes are quoted independently for Windows and macOS and are not interchangeable.

Evidence uses two levels:

- [Confirmed] covers plist and AppxManifest entries, skill and API documentation, configuration, dependency files, service entry points, signature results, and newly embedded readable bundle strings. A string’s presence is confirmed, but does not prove availability to every user.
- [Signal] covers real frontend module names, readable-string and CSS-class changes within same-name modules, and compressed-module size direction. These indicate evolving UI, state, or interaction paths.

Platform labels are `[macOS+Windows]`, `[macOS only]`, and `[Windows only]`. Pure hash renames under Vite, webview, and `.vite/build` are excluded. The inputs do not list newly added bundled plugins; observable plugin changes are concentrated in the existing Browser, Chrome, Computer Use, Sites, Visualize, and Record and Replay packages.

## ✨ Highlights

### Task terminology converges on Chat

[Confirmed] [macOS+Windows]

Both main-process and frontend bundles add `New Chat`, `New Chat in ChatGPT Sidebar`, `New standalone chat`, `Rename chat`, `Attach image to chat`, `Active local chats on this machine will be interrupted`, and `Archiving inactive chats is only supported locally`. Corresponding `New Task`, `Rename task`, and `Attach image to task` strings are removed.

The `/review` wording likewise changes from running in the current task or opening another review task to running in the current chat or opening another review chat. This is a coordinated migration across copy, command paths, and error paths, not merely a renamed module.

[Signal] [macOS+Windows]

Same-name modules for projects, conversation threads, archiving, scheduled tasks, and Codex Micro consistently shift from task to chat. Project search begins describing both ChatGPT chats and local Codex tasks; expired side tasks become side chats; sidebar archive, pin, and recovery states also move toward the chat model.

### Deeper integration between the projects index and ChatGPT Projects

[Signal] [macOS+Windows]

The `projects-index-page` module gains the same clues on both platforms: expanding and collapsing ChatGPT projects, loading more project chats, pinning projects, opening chats, and starting a local Codex task from a ChatGPT project. Search results begin distinguishing or combining matching tasks and chats, with new `partial-error` and `aria-live` state clues.

The home suggestions module also adds “Create a file or build a site,” new documents, presentations, spreadsheets and templates, new websites, plus `Google Calendar` and `Google Drive`. These are same-name frontend-module signals and do not prove that connectors or entry points are enabled for every user.

[Signal] [Windows only]

Windows adds `chatgpt-local-project`, `local-projects`, `composer-project-picker`, `composer-project-selector`, `project-default-icon`, `project-display-icon`, `select-project`, and sidebar project-group modules. Its sidebar module also adds project file upload, `Add source`, `Advanced settings`, and `Archive all chats` clues.

[Signal] [macOS only]

The macOS AppGen settings module gains strings for custom domains, DNS records, site titles, site-deletion confirmation, environment variables, and secrets. These establish an evolving settings UI only.

### Pull-request workflow moves from auto-fix to prepare-to-merge

[Signal] [macOS+Windows]

On both platforms, `local-conversation-page` removes `Auto-fix PR` and adds `Prepare to merge`, `Preparation paused`, `Ready to merge`, automatic merging once requirements are met, and menu copy for pausing, resuming, and performing a merge. The accompanying description explicitly mentions addressing merge conflicts and failing checks caused by the pull request.

`local-conversation-thread` also gains optional guidance for attached PR comments, saving or removing those comments, continuing from an older message in the current workspace or a new worktree, and asking Codex to inspect the project and prepare its environment.

[Signal] [macOS only]

The macOS `pull-request-route` adds PR-detail comment and reply composers, `Leave a comment`, `Post comment`, and `Post reply`.

[Signal] [Windows only]

The Windows thread side panel adds clues for an existing PR on the branch, staging accepted edits, canceling an active Git action, and more complete review, terminal, browser, and workspace-file-browser tabs.

### Plugin scheduled-task templates and automation details are reworked

[Signal] [macOS+Windows]

The `automations-page` module adds the following on both platforms:

- Opening a plugin-provided scheduled-task template.
- Opening the plugin that contributed the task.
- Resetting a task to its plugin template.
- `Already using plugin defaults`.
- Closing the scheduled-task detail panel.
- Opening the chat associated with a heartbeat automation.
- `Create with Codex`.

The former “personalize with ChatGPT” workflow is removed. The new `use-plugin-scheduled-tasks` module corroborates this direction, but does not prove that every plugin template is publicly available.

[Signal] [Windows only]

Windows also adds chat-archive coupling for heartbeat automations: archiving a chat with an active heartbeat can remove its scheduled task and stop future runs.

### Chunked IPC and streaming parsing enter the desktop bridge

[Confirmed] [macOS+Windows]

Both `.vite/build/preload.js` files add the `codex-host-chunked-message-v1` protocol, transfer IDs, sequence numbers, start/chunk/end messages, acknowledgements, and an assembler for arrays, objects, keys, values, and chunked strings.

The packages also add errors for unmatched container endings, multiple root values, incomplete messages, object values without keys, and transfers without an end part. The renderer-to-main bridge therefore contains a structured path for chunking, validating, and reconstructing larger messages.

### Browser and Chrome routing now discovers purpose-built tools per operation

[Confirmed] [macOS+Windows]

The unified Browser and Chrome skill now requires querying available and deferred connectors, APIs, or CLIs before each semantic operation on a linked resource. Earlier browser use does not make subsequent operations browser-first. Browser is used when no applicable tool exists, the tool cannot access the resource, lacks a required capability, or UI work remains.

Plugin mentions now select a browser explicitly: a Browser mention identifies the in-app browser, while Chrome, chrome-internal, and chrome-dev mentions identify Chrome. If the runtime-selected browser demonstrably lacks authentication, another available browser may be tried before asking the user to sign in.

The Browser and Chrome API documents also remove the marker saying file chooser operations and their wait event are unsupported by default in `iab`. This confirms only that the documented restriction marker was removed.

### Sites publishing approval and preview copy change

[Confirmed] [macOS+Windows]

The Sites Hosting skill now requires `request_user_input` before shared or public deployment. The choices must name the resolved access level, such as `Publish publicly` or `Publish to existing shared access`, alongside `Not now`; `deploy_site_version` may run only after approval.

The Sites Building template test changes `Codex is working` to `Building your site`, says the first version will appear automatically when ready, and explicitly asserts that the preview should not contain the word `Codex`.

### Visualize gains a standard table component system

[Confirmed] [macOS+Windows]

The Visualize skill and stylesheet add:

- `.table` for quiet, unframed semantic tables with horizontal dividers and wrapping cells.
- `.table-responsive` for controlled horizontal scrolling when columns cannot fit.
- `.table-sm` for denser rows.
- `.text-end`, `.text-center`, and `.text-nowrap` for table alignment and wrapping control.

The skill continues to prefer reflow and wrapping, reserving the responsive wrapper for tables whose columns genuinely cannot fit.

### Windows terminal shutdown and ConPTY cleanup change

[Confirmed] [macOS+Windows]

The bundled Windows implementation of `node-pty` changes identically in both packages. `WindowsTerminal.kill()` now waits until the terminal is ready and returns the agent kill result. `WindowsPtyAgent` accepts an injected `getConsoleProcessList`, separates ConPTY termination from console-child cleanup, and falls back to the inner PID if the override fails.

This is a substantive Windows terminal-path change. Its presence in the macOS package reflects a shared application dependency bundled on both platforms.

### Local environments gain inheritance and revised script presentation

[Signal] [macOS+Windows]

`local-environments-settings-page` adds inherited environments from parent folders, show/hide controls, source folder and file descriptions, editing, empty platform-script states, project-loading failures, and `Environment needs attention`.

Former cleanup-script summaries, platform overrides, and environment-detail strings are removed. CSS changes add monospace rendering, full wrapping, a taller command area, and screen-reader-only text.

### Remote connections are generalized beyond SSH-specific copy

[Signal] [macOS+Windows]

Remote connection settings replace several `SSH connection` references with the more general `connection` or `remote host connection`. They add `Add a remote project`, a list of devices allowed to control the connection, and a confirmation surface for remote control by other authorized devices.

An SSH row remains one entry point for the remote-project dialog, so this is a generalization signal rather than evidence that SSH support was removed.

### Codex Micro gains knob modes and lighting controls

[Confirmed] [macOS+Windows]

The main/frontend bundles add `Brightness of Codex Micro lighting`.

[Signal] [macOS+Windows]

Codex Micro settings add composer-navigation and reasoning-only knob modes, clockwise and counterclockwise actions, global lighting brightness, and assignment of the six agent keys to active chats. Older task mappings and several fixed keycap actions are removed.

### New Chat, temporary-chat, permission, and run-location controls expand

[Signal] [macOS+Windows]

Both platforms add `chat-home-hero`, `chatgpt-temporary-chat-ui`, `composer-action-bar-run-location-dropdown`, `home-composer-mode-toggle`, `permissions-mode-dropdown`, `review-slash-command-submenu-registration`, `settings-command-menu-section-items`, and `shiki-highlight-provider`.

Together they point to reworked home composer modes, temporary chats, run locations, permission modes, review-command entry points, settings commands, and syntax highlighting. Without independent configuration or copy, they remain signals.

### Slack links, thread catalogs, and download failure paths enter the packages

[Confirmed] [macOS+Windows]

Both bundles add `api.slack.com`, `app.slack.com`, `slack.com`, `Failed to resolve Slack workspace`, and `Failed to open Slack link in Slack`. This confirms embedded Slack domains and error paths, not a fully released Slack integration.

The thread catalog gains validation strings for host IDs, CWD prefixes, included and excluded thread IDs, page limits, cursor host ownership, and mismatched filters or sort orders. Download paths add `File download returned no response body`, `Temporary file is too large`, and native download-history failures.

### Pet, Picture-in-Picture, and native composition recovery expand

[Confirmed] [macOS+Windows]

Both packages add `Send Picture-in-Picture to Pet`, remote-hosted PiP tooltips for close and send-to-Pet controls, and native Pet surface/composition paths for creation, attachment, detachment, updates, disconnection, unavailability, and scheduled recovery.

These strings confirm packaged controls and recovery states, but not default visibility.

### macOS adds Calendar and Reminders permission declarations

[Confirmed] [macOS only]

`Info.plist` adds ordinary and Full Access usage descriptions for Calendar and Reminders. All state that ChatGPT may read and update calendar events or reminders when the user asks. The Windows AppxManifest has no corresponding change.

### Windows adds the application-licensing capability

[Confirmed] [Windows only]

The AppxManifest adds the `appLicensing` capability. The fact package provides no associated call site or UI, so only the package-level declaration is confirmed.

### Computer Use documentation and safety rules evolve by platform

[Confirmed] [macOS only]

The macOS Computer Use REPL type adds `target: "mac"`. Its documentation requires a full accessibility tree on the next inspection when previous accessibility text was disregarded; it directs the agent to screenshots, coordinates, and key presses when AX text or actions are unavailable or unreliable, and states that `press_key` and `type_text` cannot invoke global shortcuts.

The confirmation policy is rewritten into Hand-off, Action-time confirmation, Pre-approval, and Not required modes, with more detailed rules for credentials, persistent access, legal agreements, software sources, sensitive information, high-impact communications, ordinary transactions, and recoverable deletion.

[Confirmed] [Windows only]

The Windows Computer Use `SKILL.md` and guidance are extensively rewritten. The required entry point is the plugin’s `scripts/computer-use-client.mjs`; the full skill must be read; initialization must select exactly one returned window from `list_apps()` rather than guessing app or window fields; and interaction follows an observe-one-turn, perform-one-action, refresh-immediately loop. Element indexes, screenshot IDs, and coordinates are valid only for the observation that produced them.

### Skysight memory output gains structured metadata

[Confirmed] [macOS only]

The Skysight summarizers inside Computer Use and Record and Replay now require fixed YAML frontmatter before the Markdown body:

- `title`
- A single-line, two-to-three-sentence `description`
- A deduplicated `applications` array of bundle identifiers

Ten-minute summaries must use exact identifiers from event records; six-hour summaries aggregate exact identifiers from child summaries and must not guess. Later memory consolidation must ignore resource frontmatter and use only the Markdown body as evidence.

The same update relaxes earlier absolute exclusions for URLs, domains, and Slack content, while still requiring generic descriptions of webpage content, prohibiting Markdown links, and retaining protections for sensitive and privileged material.

### Platform runtime and packaging differences

[Confirmed] [macOS+Windows]

The Chromium baseline moves from `150.0.7871.115` to `150.0.7871.124`. On macOS this is confirmed by Info.plist, Framework-directory, and SCM revision changes; on Windows it is confirmed by the version-manifest replacement and `chrome.dll` change.

The CSP text in `webview/index.html` is unchanged on both platforms. Its existing allowances remain limited to self-hosted resources, blob/data content, Web Sandbox frames, Mapbox/OpenAI connections, and ChatGPT WebSockets. There is no evidence of CSP expansion or tightening in this batch.

[Confirmed] [Windows only]

The Windows native serial-port binding changes from `serialport.node` to `bindings.node`, while the nested `HID.node` grows from 0.6 MB to 1.9 MB. The input provides no corresponding UI change.
