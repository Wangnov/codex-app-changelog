# Codex Desktop 26.608.12217 macOS / Windows Comparison

> Third-party cross-platform changelog · unofficial

The main story in this batch is not one feature, but shared desktop groundwork across runtime packaging, voice, remote control, plugins, and project workflows.

Cross-platform desktop groundwork for voice, remote control, plugins, Sites, and runtime packaging, with Windows-specific Computer Use wiring.

## About This Report

This report only uses the machine-extracted facts in the input packages: macOS arm64 from 26.602.71036 build 3685 to 26.608.12217 build 3722; Windows x64 MSIX from 26.602.9276.0 to 26.608.1337.0. The two packages come from the same codebase but different packaging flows, so versions and package numbers are cited separately.

Evidence levels are split in two: [Confirmed] means plist, manifest, CSP, plugin manifest, skill docs, package.json, dependency list, service entrypoint, signing/package metadata, or readable strings outside the “same-name frontend module clues” section; [Signal] means same-name frontend module strings/classes, module names, and size direction. Platform scope is marked as `[macOS+Windows]`, `[macOS only]`, or `[Windows only]`.

Reviewed evidence groups include version/channel, bundle tree, bundled plugins, Computer Use runtime, cua_node dependencies, app.asar, real frontend module additions/removals, same-name module clues, main/frontend readable strings, Info.plist/CSP, resources, and key file diffs. The Windows fact package did not provide an AppxManifest change section, so no manifest inference is added.

## ✨ Highlights

### [Confirmed] [macOS+Windows] Bundled Node runtime is renamed to `cua_node` and now centralizes Computer Use dependencies

Both platforms remove the previous top-level Node / node_repl files and add a `cua_node` directory. macOS adds `Contents/Resources/cua_node/bin/node` at 119.1 MB and `node_repl` at 15.2 MB, while removing `Contents/Resources/node` at 119.1 MB and `node_repl` at 15.2 MB. Windows adds `resources/cua_node/bin/node.exe` at 91.4 MB and `node_repl.exe` at 13.5 MB, while removing `resources/node.exe` at 91.4 MB and `node_repl.exe` at 13.6 MB.

Both platforms add `@oai/sky@0.4.8-202606020558-pr-981870-f570e0604110`, `corepack@0.34.6`, and `npm@11.9.0`. macOS additionally adds `node-addon-api@8.8.0`, `node-gyp-build@4.8.4`, and `objc-js@1.5.0`, including darwin arm64/x64 native prebuilds for `objc-js`.

### [Confirmed] [Windows only] Computer Use now uses the bundled `cua_node` runtime and adds tool-surface metadata

The Windows `computer-use` plugin manifest moves from `26.602.71036` to `26.608.12217`. Its skill doc changes “plugin vendored runtime” to the bundled `cua_node` `@oai/sky` runtime, and removes the old recovery path that added `<plugin root>/node_modules` after `Module not found: @oai/sky`.

`computer-use-client.mjs` changes its import from `../node_modules/@oai/sky/...` to `@oai/sky/...`, and adds `codex/toolSurface` response metadata. It identifies the target app as either appId or displayName, and only uses native pipe / approval UI inside a trusted environment with `nodeRepl.config` and `nativePipe`. The file tree also shows many old `resources/plugins/openai-bundled/plugins/computer-use/node_modules/@oai/sky/...` files removed and the runtime moved under `resources/cua_node/bin/node_modules/@oai/sky/...`.

The macOS fact package has no matching `computer-use` plugin file diff, so this item is Windows-only.

### [Confirmed] [macOS+Windows] Browser and Chrome plugins update, and DOM snapshot now documents iframe body expansion

Both `browser` and `chrome` plugin manifests move from `26.602.71036` to `26.608.12217`. `browser/docs/api.md`, `browser/.codex-plugin/api-multi-tab.md`, and `chrome/docs/api.md` all update `domSnapshot()` from returning the current DOM string to returning the DOM string “including expanded iframe body content when available.”

The Browser / Chrome skill instructions are tightened as well: the first `browser.documentation()` read must use the exact direct call `nodeRepl.write(await browser.documentation());` in one go. The skill now says not to assign, inspect length, slice, truncate, summarize, or proactively page the documentation unless the tool output explicitly reports truncation.

### [Confirmed] [macOS+Windows] Sites plugin moves from 0.1.12 to 0.1.14, with renamed deployment APIs and revised access-control wording

The `sites` plugin manifest moves from `0.1.12` to `0.1.14` on both platforms. `sites-hosting/SKILL.md` renames environment variable operations to `get-environment-variables` / `update-environment-variables`, and changes create/save/deploy/status operations to `create-site`, `save-site-version`, `deploy-site-version`, and `get-deployment-status`.

The same skill doc adds that `project_id` is the persisted identifier for a Site; existing projects should use `create-source-repository-write-credential` before source fetch/push; access controls should only be used when the user asks to change who can access the deployed site; `workspace_all` should be described as “Anyone in this workspace can visit,” and `custom` as “Private.”

### [Confirmed] [macOS+Windows] CSP adds WebSocket targets for realtime or streaming connections

Both `webview/index.html` files add `wss://chatgpt.com`, `wss://ws.chatgpt-staging.com`, and `wss://ws.chatgpt.com` to `connect-src`. The same file also adds `data-build="<!-- PROD_BUILD_TAG_HERE -->"` and preloads `modulepreload-polyfill` and `isEqual`.

This confirms that those WebSocket targets and the build marker entered the frontend package. It does not by itself prove that every related UI is enabled for all users.

### [Confirmed] [macOS+Windows] Realtime voice and global dictation enter the desktop package

New main/frontend strings appear on both platforms: `Failed to prepare dictation stream connection.` and `Sign in to ChatGPT in Codex Desktop to use dictation.` Composer clues add `Microphone access is required to start realtime voice`, `Realtime voice connection failed`, and `Realtime WebRTC offer did not include SDP`, while the sidebar thread adds an accessible label for an active realtime voice session.

The `global-dictation-page` same-name module adds `Dictation ready`, `Global dictation ready`, `hold-to-dictate`, and tooltip strings for hold/toggle dictation shortcuts on both platforms. The dictation connection and sign-in strings are [Confirmed]; the UI shape from same-name frontend modules remains [Signal].

### [Confirmed] [macOS only] Camera permission wording now says Codex needs camera access for video input

macOS `Info.plist` changes `NSCameraUsageDescription` from `This app needs access to the camera` to `Codex needs access to your camera for video input.` The Windows fact package has no corresponding permission manifest section.

### [Confirmed] [macOS+Windows] Remote control, SSH, and Browser session routing error paths enter the main package

The `app-main` frontend module also adds signals for `Change connection color`, remote connection color picker close/reset buttons, and project hover-card actions. Error paths and status strings are confirmed in the package; the exact UI exposure remains a signal.

### [Confirmed] [macOS+Windows] Avatar overlay / native pet composition surfaces enter the package; macOS also adds a native addon file

Both platforms add real frontend modules including `avatar-overlay-featuregate-page`, `avatar-overlay-native-frame`, `avatar-overlay-native-page`, `avatar-overlay-notifications`, `avatar-overlay-pill-dismiss-button`, `avatarOverlayCompositionSurface`, and `custom-avatars-query`. macOS additionally lists a new `Contents/Resources/native/avatar-overlay.node` file at 0.3 MB. The Windows package does not list a corresponding native addon addition, so the native file is macOS-only, while the strings and frontend modules are cross-platform.

### [Signal] [macOS+Windows] Plugin and skill browsing shifts from “install” toward “add / connect / create”

Both `plugins-page` modules add `Browse plugins or skills`, `Add plugin marketplace`, `Button label for connecting a plugin`, `Button label for adding a recommended skill`, `Clear search`, and `Create options`, while removing `All plugins`, `Choose a category`, `Choose a plugin marketplace`, and `Button label to install a recommended skill`.

### [Signal] [macOS+Windows] Project, archived chat, and local task status UI is being reorganized

Both `data-controls` modules add archived chat filtering/grouping, project archived chat actions, `Delete all in project?`, and cloud/local archived chat filters. `profile-dropdown` adds `Archive chat`, `Archive task`, `Automation chat`, `Awaiting approval`, `Awaiting response`, a local-task waiting-for-user-approval chip, and branch-origin copy.

### [Signal] [macOS+Windows] Google Drive and Slack resource labels enter conversation threads

Both `local-conversation-thread` modules add `Google Docs`, `Google Sheets`, `Google Slides`, `Open in browser`, hover subtitles for opening Google Drive conversation resources, and Slack search target nouns for channels/users. On both platforms, `runtime.worker` grows from 3.0 MB to 3.5 MB, a 481283-byte increase. It adds `Aptos Display`, `Document worker has not been bootstrapped.`, `Document worker received a message before bootstrap`, `Grid Table 1 Light Accent 1` through `Accent 6`, plus `text-body`, `text-display`, `text-section`, and `text-subtitle`. This points to document worker font/style and bootstrap handling changes, but not recoverable source semantics.

### [Confirmed] [macOS+Windows] Build scripts and owl runtime update

Both `package.json` files move from `26.602.71036` to `26.608.12217`, and `codexBuildNumber` moves from `3685` to `3722`. `compile`, `tsc`, `lint`, and `lint:fix` now run `node ./scripts/ensure-owl-electron-types.mjs` first, and `build:ci` changes to `node ./scripts/build-ci.mjs`.

`owl-electron-app.json` updates the runtime archive SHA on both platforms: macOS from `c6a2...06fe` to `df06...a4f`, Windows from `76a8...c86f` to `6a9d...6aa4`. Windows also updates `.installed-owl-shell.json` cacheKey, installedAt, and archiveSha.

### [Confirmed] [macOS only] DMG detach cleanup gains retries

macOS `.vite/build/bootstrap.js` changes source DMG detach cleanup from a single attempt to up to 25 attempts with 250 ms spacing, and adds `Failed to detach app source DMG after retries`. The Windows package contains the same bootstrap code fragment, but that logic returns early when `platform !== darwin`; therefore the actual installer-flow change is macOS-only.

### [Signal] [macOS+Windows] Settings copy adds font, theme, default destination, terminal location, and permissions items

Both `src` modules add `Base UI font size`, `Code font size`, `Chrome theme used in dark mode`, and `Chrome theme used in light mode`. `gpu-tearing-debug-settings` adds `Agent environment`, `Default open destination`, `Default permissions`, `Default terminal location`, `Bottom panel`, `Code review`, `Default to projectless chat`, and `Dictation dictionary`, while removing external agent config import / migrate-to-codex strings.
