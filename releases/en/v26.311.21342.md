# Codex Desktop `26.311.21342` build 993

> Third-party changelog · Unofficial · Based on reverse diffs of official update artifacts

At the artifact level, this release looks less like a single feature drop and more like concentrated groundwork across remote connections, settings, automations, and code-reading surfaces.

## About This Report

This is not an official changelog. It is a **reverse diff report based on official distribution artifacts**. The comparison covers macOS arm64 from `26.309.31024` build 962 to `26.311.21342` build 993.

This entry is a historical backfill: the official appcast did not provide a Sparkle delta for this version pair, so the fact pack compares two official full packages directly. This report only uses reproducible bundle diffs, configuration diffs, dependency changes, readable strings, and size statistics from the fact pack.

Evidence is split into two levels. **[Confirmed]** means there is direct human-readable evidence, such as `package.json`, `Info.plist`, preload bridges, CSP, type/config text, or new string literals. **[Signal]** means the evidence only comes from frontend module names, same-name frontend module string diffs, or size changes; it points in a direction, but cannot reconstruct source-level intent from the artifact.

## ✨ Highlights

### 1. Remote connection, SSH, and Git/GitHub workflow error paths expanded [Confirmed]

The app layer adds an `ssh-config` dependency, while the main/frontend bundle now contains multiple string literals related to remote SSH, worktrees, upstream branches, GitHub CLI handling, and PR generation.

The visible implication is that remote connection failures, synced-branch status, GitHub CLI response failures, and PR title/body generation now have clearer handling traces in the desktop artifact. Key evidence includes `package.json` adding `ssh-config` `^5.0.3`, plus `Failed to connect remote SSH connection.`, `No synced branch config found for the current worktree.`, `Failed to parse GitHub CLI response`, `Unexpected GitHub CLI response`, and `You are a helpful assistant. Generate a pull request title and body.`.

### 2. Automation run and recurring schedule parsing paths appeared [Confirmed]

The new bundle adds string literals for automation runs, model-loading failures, and recurrence-rule parsing.

The likely user-facing impact is clearer internal error paths when automation runs fail or schedules are invalid. Direct evidence includes `Failed to load models for automation run`, `Invalid UNTIL value: `, `Invalid frequency: `, `Invalid weekday string: `, `Unsupported RFC prop `, `unsupported EXRULE parm: `, and `unsupported RRULE parm: `.

### 3. System theme variant entered the Electron preload bridge [Confirmed]

`preload.js` now includes synchronous read and subscription hooks for the system theme variant: `codex_desktop:get-system-theme-variant`, `codex_desktop:system-theme-variant-updated`, `getSystemThemeVariant`, and `subscribeToSystemThemeVariant`.

This means the renderer can now read and subscribe to the system theme variant, giving runtime theme switching and appearance settings a more direct bridge.

### 4. Appearance, connections, and Apps information architecture is being reshaped in Settings [Signal]

New module names include `appearance-settings`, `remote-connections-settings`, `settings-row`, `app-server-connection-state`, and `use-is-apps-enabled`. Same-name frontend module diffs also add strings such as `settings.nav.appearance`, `settings.nav.connections`, `All Apps`, `All skills`, `App enabled`, `App disabled`, and `Choose Plugins, Skills, Apps, or a plugin marketplace`.

These clues point toward Settings being split into appearance and connection areas, while Skills/Plugins/Apps entry points are being consolidated. They are still frontend module and same-name module signals only, so they cannot reconstruct source-level intent or confirm the full rollout scope.

### 5. Markdown/content rendering gained image preview, Mermaid, and accessibility signals [Signal]

The same-name `markdown` frontend module grew from 0.0 MB to 0.5 MB and added strings including `ARIA label for rendered mermaid diagrams`, `Accessible label for the image preview dialog`, and `Aria label for closing the image preview dialog`. The `webview/index.html` preload list also includes resources such as `math`, `katex`, and `markdown`.

These signals point to expanded Markdown rendering around image previews, Mermaid, math content, or accessibility, but source-level intent cannot be recovered from the artifact.

### 6. Diff and multi-language syntax highlighting changed substantially [Signal]

The `diff` module adds syntax tokens such as `markup.changed.diff`, `markup.deleted.diff`, `markup.inserted.diff`, and `meta.diff.header.git`. At the same time, `javascript`, `csharp`, `fortran-free-form`, `haxe`, `scss`, `ruby`, `hlsl`, `graphql`, and `css` show many added grammar strings, while `tsx`, `less`, `python`, and `wasm` shrink or lose strings.

This points to repackaging or upgrading diff/code-highlighting grammars, but it cannot be treated as source-level intent.

### 7. The local `dev` script no longer rebuilds SQLite first [Confirmed]

In `package.json`, the `dev` script changed from running `pnpm run rebuild:sqlite && ... electron-forge start` to generating third-party notices and then starting `electron-forge start`.

For developers, the local development startup path no longer runs `rebuild:sqlite` before `dev`; `forge:package` and `forge:make` still retain `pnpm run rebuild:sqlite && pnpm run rebuild:forge-natives`.
