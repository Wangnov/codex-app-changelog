# Codex Desktop 26.527.60818(macOS / Windows comparison)

> Third-party cross-platform changelog · Unofficial

This batch shows OpenAI Codex Desktop receiving core binary growth and app-layer asset rebuilds on both macOS arm64 and Windows x64, while the clearest readable configuration evidence appears only in the macOS package.  
One-liner: this cross-platform batch looks like a binary/runtime refresh, with macOS exposing plugin version bumps that the Windows fact package does not show.

## About This Report

This report is based on reverse diffs of two official signed distribution packages: the macOS side uses full Sparkle packages, and the Windows side uses full MSIX packages, each compared against its previous version. The macOS comparison is 26.527.31326 build 3390 → 26.527.60818 build 3437. The Windows comparison is 26.527.3686.0 → 26.527.7698.0. The release date is recorded from the supplied fact package as 2026-05-30.

Evidence is split into two levels. [Confirmed] means readable evidence from plist, AppxManifest, skill docs, type declarations, package.json, dependency manifests, or similar files. [Signal] means evidence only from frontend module names, size changes, or build artifacts; it indicates direction but cannot reconstruct source intent.

Platform tags mean the following: [macOS+Windows] appears in both platform fact packages; [macOS only] appears only in the macOS fact package; [Windows only] appears only in the Windows fact package. Hash renames under webview/assets and .vite/build are treated as noise, per the fact packages, and are not counted as feature additions or removals.

## ✨ Highlights

### Codex Core Binaries Grew Again [Signal] [macOS+Windows]

The core Codex executable components grew on both platforms. On macOS, `Contents/Resources/codex` increased from 198.6 MB to 201.4 MB, up 2.81 MB. On Windows, `resources/codex.exe` increased from 253.5 MB to 258.0 MB, up 4.51 MB, while `resources/codex` increased from 223.2 MB to 226.7 MB, up 3.48 MB.

This is a shared cross-platform signal that the core binary contents changed in this batch. The fact packages do not include readable source or symbol-level explanations, so this does not justify claims about specific features, model logic, or runtime behavior.

### app.asar Was Rebuilt, But Feature Meaning Is Unclear [Signal] [macOS+Windows]

The app.asar internal file counts stayed unchanged on both platforms while many add/remove pairs appeared. macOS went from 2774 to 2774 internal files, with 216 added, 216 removed, and 12 changed. Windows went from 2722 to 2722 internal files, with 216 added, 216 removed, and 11 changed.

The fact packages explicitly say that many changes under webview/assets are Vite build hash-renaming noise. This is therefore best read as a signal of app-layer rebuild or repackaging, not as a confirmed list of new user-facing features.

### macOS Bundled Browser/Chrome Plugin Versions Were Synced to the New Build [Confirmed] [macOS only]

The macOS package contains readable diffs for bundled `openai-bundled` plugin configuration. The Browser plugin’s `plugin.json` version changed from `26.527.31326` to `26.527.60818`. The Chrome automation plugin’s `plugin.json` version changed from `26.527.31326` to `26.527.60818` as well.

This is the clearest confirmed app-layer change on macOS in this batch. The Windows fact package does not include corresponding plugin configuration diffs, so the correct cross-platform reading is: macOS has readable evidence of plugin version synchronization; the supplied Windows evidence does not prove the same plugin metadata update.

### Platform Metadata Updated [Confirmed] [macOS only]

The macOS `Info.plist` shows `CFBundleShortVersionString` changing from `26.527.31326` to `26.527.60818`, `CFBundleVersion` changing from `3390` to `3437`, and `ElectronAsarIntegrity/Resources/app.asar/hash` changing from `7b24b4099e5ac419d3b02d8928ab1422f8d1b650cb51a1af754e8bc2f4ff6e85` to `63aa85903e8cd29e881fc2d098bc169fd9e7dce32532dee952b45e776338c0dc`.

The Windows fact package gives the version comparison 26.527.3686.0 → 26.527.7698.0, but it does not list AppxManifest or equivalent manifest-field diffs. This item is therefore confirmed only on macOS at the readable-manifest level; the Windows version change is recorded in the raw version data.

### No New cua_node Packages [Confirmed] [macOS+Windows]

Both platform fact packages report 0 newly added `cua_node` packages. In other words, the machine-extracted results do not show any added `cua_node` dependency packages in this batch.

This is a shared negative confirmation: the visible changes in this batch do not take the form of newly added `cua_node` packages.

### Windows Helper Executables Show Small Size Changes [Signal] [Windows only]

In the Windows package, beyond the core `codex.exe` and `codex` components, `resources/codex-command-runner.exe` stayed at 1.2 MB with a +0.01 MB change, and `resources/codex-windows-sandbox-setup.exe` stayed at 8.4 MB with a +0.00 MB change.

This only shows machine-visible content changes in Windows helper executables. The fact package does not provide readable diffs, so no specific behavioral change can be confirmed for the command runner or Windows sandbox setup.
