## ✨ Key Changes

### 1. New Spreadsheet File Type Associations: TSV / XLS / XLSM / XLSX [Confirmed]

Codex Desktop added document declarations for multiple spreadsheet file types in `Info.plist`, including `TSV`, `XLS`, `XLSM`, and `XLSX`. Their role is `Viewer`, and `LSHandlerRank` is `Alternate`.

This means macOS now knows that Codex can act as an alternate viewer for these files; users may see more complete spreadsheet file support signals in “Open With” or file drag-and-drop related flows.

Key evidence:

```diff
+ CFBundleTypeExtensions = tsv / xls / xlsm / xlsx
+ CFBundleTypeRole = Viewer
+ LSHandlerRank = Alternate
```

### 2. Browser Plugin Resources Changed from `browser-use` to `browser` [Confirmed]

A new `Contents/Resources/plugins/openai-bundled/plugins/browser/` path appears in the file tree, while many corresponding resources under the `browser-use` path were removed, including `browser-client.mjs` and prebuilt `classic-level` modules.

This looks more like a naming and packaging path change for the bundled browser plugin than the addition of an entirely new browser capability. User-visible effects may appear in the plugin name, loading path, or internal invocation method, but the artifact does not provide UI copy.

Key evidence:

```text
Added: Contents/Resources/plugins/openai-bundled/plugins/browser/scripts/browser-client.mjs (0.9 MB)
Removed: Contents/Resources/plugins/openai-bundled/plugins/browser-use/scripts/browser-client.mjs (0.7 MB)
```
