## ✨ Highlights

### [Confirmed] [macOS only] Bundled plugins move from old names to new names while preserving browser automation

The macOS bundled plugin list changes from `browser-use`, `chrome`, `computer-use`, `latex-tectonic` to `browser`, `chrome`, `computer-use`, `latex`. This is not just package-size churn: the new `browser` plugin has manifest metadata, skill documentation, default prompts, and service scripts. Its version is `0.1.0-alpha2`, display name is `Browser`, and short description says it controls the in-app browser with Codex.

The `browser` description explicitly covers opening, navigating, inspecting, testing, clicking, typing, screenshots, localhost, 127.0.0.1, ::1, file://, and the current in-app browser tab. It also keeps aliases such as `@browser`, `@browser-use`, `browser-use`, `Browser`, and `in-app browser`, so the evidence points to capability migration and naming cleanup rather than removal. Its largest files include `scripts/browser-client.mjs` at 0.9 MB and multi-platform classic-level native prebuilds.

The Windows sample has no plugin-list change. This does not prove Windows received or did not receive the rename; it only means this input sample has no Windows content change.

### [Confirmed] [macOS only] The LaTeX plugin becomes a fuller TeX workflow

macOS adds a `latex` plugin with 18 files, 50.4 MB total size, version `0.2.0`, and display name `LaTeX`. The older `latex-tectonic` plugin is removed. The new plugin description expands from bundled Tectonic compilation into a workflow that tries bundled Tectonic first, falls back to detected TeX Live or MacTeX, and can install a Codex-managed full TeX Live runtime when no usable system installation exists.

The skill docs show three concrete entry points: `latex-compile` runs `scripts/compile_latex.py`; `latex-doctor` runs `scripts/latex_doctor.py` to detect Tectonic, TeX Live, MacTeX, latexmk, pdflatex, xelatex, lualatex, biber, and kpsewhich; `texlive-runtime-installer` is detect-only by default and only runs `--install-managed-full` after user confirmation. The managed install path is `~/.cache/codex-runtimes/codex-texlive/full`, and the docs state that it does not use sudo, write `/Library/TeX`, write `/usr/local/texlive`, or modify shell startup files.

The Windows sample has no corresponding change.

This confirms package-level file open/view associations for spreadsheet-like files on macOS. It aligns with signals from `runtime.worker`, `rate-limit-rows`, and the new `google-drive` module, but the file association itself is plist-level confirmed evidence.
