# Codex Desktop 26.513.20950(macOS / Windows comparison)

> Third-party cross-platform changelog · Unofficial

The main conclusion is straightforward: the macOS package moved to 26.513.20950 build 2816, while the compared Windows MSIX remained at 26.506.3741.0 and was identical across both extractions.

One-liner: macOS picked up the observable app, runtime, plugin, and document-handling changes; the compared Windows MSIX did not change.

## About This Report

This report is based on reverse diffs of officially signed distribution artifacts: macOS was compared through Sparkle full packages, and Windows through MSIX packages. The macOS comparison covers 26.506.31421 build 2620 → 26.513.20950 build 2816. The Windows comparison covers 26.506.3741.0 → 26.506.3741.0. The batch release date in the supplied facts is 2026-05-08.

Evidence is split into two levels. [Confirmed] means the change comes from readable evidence such as plist, AppxManifest, CSP, skill documents, type declarations, package.json, or dependency lists. [Signal] means it only comes from frontend module names or size changes, which can indicate direction but cannot reconstruct source intent.

Platform tags mean: [macOS+Windows] for facts seen on both platforms, [macOS only] for facts seen only in the macOS package, and [Windows only] for facts seen only in the Windows package. This batch contains no observed Windows-only feature changes.

## ✨ Highlights

### App Version And Runtime Bump [Confirmed] [macOS only]

The macOS package moved from 26.506.31421 build 2620 to 26.513.20950 build 2816. package.json updated the version, build number, and Sparkle public key. Info.plist also updated `CFBundleShortVersionString`, `CFBundleVersion`, and the app.asar integrity hash.

In the same batch, the Windows package remained at 26.506.3741.0. Its before and after SHA-256 values are identical, with zero file count changes, zero content changes, and zero app.asar internal file changes. These version and runtime changes were not observed in the Windows MSIX.

### Electron And Native Dependency Updates [Confirmed] [macOS only]

The macOS package.json shows Electron moving from 41.2.0 to 42.0.1, and better-sqlite3 moving from ^12.8.0 to ^12.9.0. The Electron Framework binary grew from 177.9 MB to 181.5 MB, a +3.64 MB change.

The compared Windows package had no file-level or visible dependency changes, so this Electron or better-sqlite3 update cannot be attributed to Windows in this batch.

### Development And Packaging Script Cleanup [Confirmed] [macOS only]

In macOS package.json, the `dev` and `owl` scripts now prepare the development environment through `scripts/prepare-dev.mjs`. The package also adds `forge:make:owl-shell` and `e2e:computer-use-native-pipe`. The previous `computer-use:windows:bundle` script is removed in this diff.

These are readable configuration changes inside the packaged artifact, indicating adjustments around development, packaging, and native pipe end-to-end testing. The Windows package itself did not change.

### Expanded Spreadsheet File Type Support [Confirmed] [macOS only]

macOS Info.plist adds document type declarations for TSV, XLS, XLSM, and XLSX. Each is marked with the Viewer role, with LSHandlerRank set to Alternate. At the macOS system level, this makes these file types recognizable as document types Codex can view.

No AppxManifest change was provided for Windows, and the Windows package had no file-level changes, so equivalent file association support cannot be confirmed for Windows in this batch.

### Browser Plugin Directory Migrated From browser-use To browser [Confirmed] [macOS only]

The macOS package adds `plugins/openai-bundled/plugins/browser/...` files, including browser-client.mjs and classic-level prebuilds for multiple platforms. It also removes corresponding files and skill documentation under `plugins/openai-bundled/plugins/browser-use/...`. The newly added browser-client.mjs is 0.9 MB, while the removed browser-use browser-client.mjs was 0.7 MB.

The evidence points more strongly to a plugin naming or directory layout migration than to a simple addition of a separate browser capability. The Windows comparison package did not change, so this migration was only observed on macOS.

### LaTeX Plugin Directory Rename [Confirmed] [macOS only]

The macOS package adds `plugins/openai-bundled/plugins/latex/bin/tectonic` at 50.3 MB and removes `plugins/openai-bundled/plugins/latex-tectonic/bin/tectonic`, also 50.3 MB. The observable facts support a directory rename from latex-tectonic to latex, rather than proving a change in the underlying tectonic binary capability.

No corresponding change was observed in the Windows package.

### Computer Use Helper Components Expanded [Confirmed] [macOS only]

The macOS package adds files under `CUALockScreenGuardian.app`, including a 14.7 MB main binary and a 3.0 MB Assets.car resource. `SkyComputerUseService` also grows from 14.7 MB to 15.0 MB, a +0.27 MB change. Two embedded.provisionprofile files were removed.

The filenames point toward lock-screen guard or helper installer components related to Computer Use, but the fact pack does not include source code, so user-visible behavior cannot be inferred further. The Windows package did not change.

### MCP Sandbox Message Channel Adds navigate [Confirmed] [macOS only]

In macOS `.vite/build/sandbox-preload.js`, the allowed MCP app sandbox port name list adds `navigate`. This is a readable code-level change and clearly shows that the sandbox message bridge gained a channel named `navigate`.

Windows app.asar did not change, so this new channel was only observed in the macOS package.

### Frontend Modules Show Multiple Product-Direction Signals [Signal] [macOS only]

Inside macOS app.asar, internal files increased from 1382 to 1419. After excluding webview/assets and .vite/build hash noise, newly observed module names include `codex-mobile-page`, `codex-mobile-setup-flow`, `device-connections`, `remote-projects`, `google-drive`, `plugin-uninstall-dialog`, `pull-request-code-review-comments`, `mermaid-diagram`, `inline-mentions`, and `keyboard-shortcuts-search-input`.

These names point toward areas such as mobile connection, device connection, remote projects, Google Drive, plugin uninstall, PR code review comments, Mermaid diagrams, inline mentions, and keyboard shortcut search. This is module-level signal only and does not prove that these capabilities are available to users. Windows app.asar stayed at 1336 → 1336, with 0 added, 0 removed, and 0 changed files.

### New Icon Resource Added To The Bundle [Confirmed] [macOS only]

macOS adds `Contents/Resources/icon.icns`, sized 0.9 MB. The Windows package had no file changes, so no corresponding observation exists there.
