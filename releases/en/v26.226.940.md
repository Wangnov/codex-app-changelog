## ✨ Key Changes

### 1. Added Bluetooth Usage Description [Confirmed]

The latest `Info.plist` adds `NSBluetoothAlwaysUsageDescription` with the value:

```text
This app needs access to Bluetooth
```

This means Codex now declares a Bluetooth access description at the system metadata level. It does not mean that a user-facing Bluetooth feature can already be confirmed as shipped; the fact package only proves that this permission usage copy was added to the application metadata.

### 2. Application Version and Build Number Bumped [Confirmed]

### 3. Development-Side Lint Toolchain Consolidated Around oxlint [Confirmed]

In `package.json`, the previous lint script, which ran both `oxlint` and `eslint`, was simplified to run `oxlint` directly. Related dependency entries for `eslint`, `eslint-plugin-import`, and `oxfmt` were removed.

Key evidence:

```diff
-    "lint": "pnpm run lint:oxlint && pnpm run lint:eslint",
+    "lint": "pnpm exec oxlint --tsconfig ./tsconfig.json --max-warnings 0 --type-aware --type-check",
```

This looks more like internal cleanup of the build and development workflow. The fact package does not provide direct evidence of whether ordinary users would notice any change; for developers, it indicates that this version's application source configuration leans more toward a single lint toolchain.

### 4. cross-env Added to Development Command [Confirmed]

The `dev` script changed from setting `NODE_ENV=development` directly to setting the environment variable through `cross-env`, while adding the `cross-env` `^7.0.3` dependency.

Key evidence:

```diff
-    "dev": "pnpm --dir .. run generate:third-party-notices && NODE_ENV=development electron-forge start",
+    "dev": "pnpm --dir .. run generate:third-party-notices && cross-env NODE_ENV=development electron-forge start",
```

This points to a cross-platform compatibility adjustment for the development startup script, but the fact package does not show that it changes end-user functionality.

### 5. usage-settings and useForm Frontend Module Names Appear [Signal]

After removing Vite hash noise, the fact package shows real newly added frontend module names:

```text

```

This points toward usage settings or form handling, but the artifact cannot reconstruct source-level intent, nor can it prove that the corresponding entry points are already open to users.
