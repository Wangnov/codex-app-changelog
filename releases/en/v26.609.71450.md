# Codex Desktop 26.609.71450 macOS / Windows Comparison

> Third-party cross-platform changelog · unofficial

This batch is not synchronized across platforms: macOS 26.609.71450 shows a very small fact surface, while Windows 26.611.7849.0 carries many app-layer and runtime signals.  
One-liner: macOS is mostly a package/version refresh, while Windows carries broader app, browser, remote-control, Sites, voice, and Computer Use changes.

## About This Report

This report is based only on machine-extracted diffs from reverse-reconstructed official signed packages: macOS arm64 from 26.609.71144 build 3952 to 26.609.71450 build 3965; Windows x64 MSIX from 26.609.4994.0 to 26.611.7849.0.

Evidence is split into two levels. [Confirmed] means the claim comes from plist, manifest, CSP, skill documentation, type declarations, package.json, dependency lists, plugin manifests, service entries, or readable strings confirmed inside the package. [Signal] means it comes only from frontend module names, size changes, compressed-bundle direction, or changed same-name frontend modules. Platform scope is marked as `[macOS+Windows]`, `[macOS only]`, or `[Windows only]`.

The two platform versions and release timings may be offset. This report cites each platform’s own fact package and does not apply Windows conclusions to macOS. Hash-only webview/assets and .vite/build renames are treated as build noise.

## ✨ Highlights

### [Confirmed] [macOS+Windows] The package batch advanced on both platforms, but the change surface is much larger on Windows

macOS moved from 26.609.71144 build 3952 to 26.609.71450 build 3965. Info.plist changed `CFBundleShortVersionString`, `CFBundleVersion`, and `ElectronAsarIntegrity/Resources/app.asar/hash`. Its bundle file count stayed at 8298 → 8298, with 0 added, 0 removed, and 61 content changes. app.asar stayed at 2870 → 2870 internal files, with 1 added, 1 removed, and 11 changed.

Windows moved from 26.609.4994.0 to 26.611.7849.0. package.json changed the app version from 26.609.41114 to 26.611.61049, and `codexBuildNumber` from 3888 to 3996. Its file count changed from 11972 → 11943, with 7 added, 36 removed, and 92 content changes. app.asar changed from 2815 → 3024 internal files, with 1819 added, 1610 removed, and 93 changed. The Windows change surface is substantially broader.

### [Confirmed] [macOS only] The macOS sample mainly confirms Browser/Chrome plugin metadata version bumps

The readable plugin diffs in the macOS package are concentrated in two bundled plugin manifests: `openai-bundled/plugins/chrome/.codex-plugin/plugin.json` and `openai-bundled/plugins/browser/.codex-plugin/plugin.json` changed `version` from 26.609.71144 to 26.609.71450. The plugin description text did not show semantic changes.

So the macOS sample confirms that the bundled Browser and Chrome plugin metadata followed the app version. The fact package does not provide new plugin manifests, service entries, skill instructions, default prompts, or largest-file evidence for a new capability surface.

### [Confirmed] [Windows only] Browser/Chrome plugin docs add a JavaScript dialog API and tighten behavior when login blocks browsing

On Windows, the Browser and Chrome plugin skill docs both add the same rule: when the user explicitly asks to navigate to a page in the browser and authentication or sign-in blocks the task, the agent should not switch to web search, a search engine, another site, or another source to work around login. If secure browser authentication is available, use that flow; otherwise stop and ask the user to log in.

The same API docs also extend Browser API: `Tab` adds `getJsDialog(): Promise<undefined | Dialog>`, plus `AlertDialog`, `BeforeUnloadDialog`, `ConfirmDialog`, `PromptDialog`, and the `Dialog` union type. The `clipboard` description changed from “this tab” to “browser session's clipboard.” This is confirmed public packaged documentation, pointing to session-level clipboard wording and JavaScript dialog handling.

### [Confirmed] [Windows only] The browser context menu gains “Copy link” across native menu localizations

Windows main/frontend readable strings add `Copy link`, `browserSidebar.contextMenu.copyLink`, `Open in browser`, `Open in external browser`, `codex.commandDescription.closeTab`, and `codex.commandMenuTitle.closeTab`. Many `native-menu-locales/*.json` files add `browserSidebar.contextMenu.copyLink`, for example Simplified Chinese “复制链接地址,” Japanese “リンクのアドレスをコピー,” and Korean “링크 주소 복사.”

This is a high-visibility UI text change: the browser sidebar context menu has a packaged “copy link address” action. The `Close Tab` and close tab/window string changes also show command naming split or renaming, but the full interaction still needs runtime verification.

### [Confirmed] [Windows only] Computer Use @oai/sky moves from 0.4.10 to 0.4.13 and wraps mac target actions in a policy layer

Windows `@oai/sky/package.json` changed from 0.4.10 to 0.4.13 and removed the `objc-js` 1.5.0 optional dependency. Multiple mac target action files changed from calling `getClient()` directly to wrapping calls through `withComputerUsePolicy`: `press_key`, `type_text`, `drag`, `set_value`, `scroll`, `click`, `perform_secondary_action`, `get_app_state`, and `select_text`. `list_apps` also adds `setComputerUseResponseMeta(null)`.

Type declarations show the mac client moving from `apple-event` to `native-pipe`; `RequestOptions` drops service app path, bundle identifier, and process identifier fields, while adding internal transport-related structure. Added files include `native-pipe.js`, `computer-use-policy.js`, and matching `.d.ts` files. Removed files include `service-app.js`, `apple-event.js`, and declarations. Main-process strings also add `computer-use native pipe helper paths unavailable` and remove `computer-use native pipe transport module missing`.

### [Confirmed] [Windows only] Computer Use total size is almost unchanged, but runtime and dependency layout shifted

Windows `cua_node` total size changed from 285.5 MB to 285.6 MB, shown in the fact package as +0 MB and +0%. No new dependency packages were added, but `@oai/sky` changed version as above. The Windows package also adds `resources/codex` at 271.2 MB and `resources/rg` at 5.4 MB, while `resources/codex.exe` grows by 27.01 MB, `codex-windows-sandbox-setup.exe` by 0.45 MB, and `codex-command-runner.exe` by 0.07 MB.

These are file-level facts: CLI, runner, and sandbox-related binaries changed noticeably in the Windows distribution, but the fact package does not expose their source semantics.

The file tree also removes many files under `resources/plugins/openai-bundled/plugins/sites/skills/sites-building/templates/auth-siwc-enabled/`, including package.json, README, db/schema.ts, auth routes, worker files, D1 examples, and `.openai/hosting.json`.

This is confirmed by both skill documentation and file-tree changes: in the Windows package, the Sites building flow moves away from a bundled SIWC/OAuth starter and toward confirming the platform auth path before implementation.
