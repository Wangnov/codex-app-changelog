### 3. Development tooling shifted from Prettier to oxfmt, with layered oxlint checks introduced [Confirmed]

`package.json` shows that the formatting command changed from Prettier to `oxfmt`, while lint commands were split into two layers: `oxlint` and ESLint.

This is mainly a developer and build-maintenance change, so users typically will not perceive it directly. However, it shows that the desktop app project’s code quality check workflow has been adjusted.

Key evidence:

```diff

+ "format:fix": "oxfmt --write"
```

### 4. Playwright-related development entry points were added [Confirmed]

This release adds the `playwright:agent:repl` script and introduces `playwright` as a development dependency. This points to a newly added or strengthened internal Electron debugging / automation entry point based on Playwright.

This is a development and testing capability change, not equivalent to a new user-facing browser automation feature.
