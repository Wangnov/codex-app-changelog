# Codex Desktop 26.513.31313(macOS / Windows comparison)

> Third-party cross-platform changelog · Unofficial

This batch mainly shows app-layer, browser-related plugin, and lower-level runtime component updates across the OpenAI Codex desktop packages for macOS and Windows, while the Windows package carries stronger signals of plugin layout restructuring.

One-liner: this cross-platform batch updates app-layer and browser-automation-related components, with Windows showing larger packaging and plugin layout changes.

## About this report

This report is based on reverse diffs of official signed distribution packages: the macOS side compares full Sparkle packages, while the Windows side compares MSIX-related distribution packages. The macOS comparison is 26.513.20950(build 2816) → 26.513.31313(build 2867). The Windows comparison is 26.506.3741.0 → 26.519.2736.0. The release date is taken from the supplied fact package: 2026-05-16.

Evidence is split into two levels: [Confirmed] means the claim comes from directly inspectable material such as plist metadata, plugin configuration, readable file diffs, dependency lists, file trees, or package metadata. [Signal] means the claim comes only from frontend module names, size changes, directory movement, or binary size changes; it indicates direction but does not reconstruct source intent.

Platform markers mean: `[macOS+Windows]` appears in both platform fact packages; `[macOS only]` appears only in the macOS package; `[Windows only]` appears only in the Windows package.

## ✨ Highlights

### App-layer resources were updated [Confirmed] [macOS+Windows]

Both platforms changed their app.asar package. On macOS, `app.asar` stayed effectively flat at 147.7 MB → 147.7 MB, about +0.01 MB; internal files moved from 1419 to 1421, with 972 added, 970 removed, and 11 modified. On Windows, `resources/app.asar` grew from 136.7 MB to 159.0 MB, a +22.25 MB increase; internal files moved from 1336 to 1530, with 1188 added, 994 removed, and 15 modified.

The platform difference is substantial: macOS barely changed in size, while Windows grew sharply. Both packages include WebView/assets and Vite build-hash rename noise, so asset filename churn should not be interpreted as new functionality.

### Browser-automation-related plugins kept moving [Confirmed] [macOS+Windows]

On macOS, `browser-client.mjs` changed slightly under both the `chrome` and `browser` plugin paths. On Windows, `resources/plugins/openai-bundled/plugins/chrome/scripts/browser-client.mjs` grew from 0.7 MB to 0.9 MB, and `resources/plugins/openai-bundled/plugins/browser/scripts/browser-client.mjs` was newly added.

The Windows side shows a stronger directory-level change: many scripts, dependencies, and a skill document were removed from the older `browser-use` plugin path, while new content appears under the `browser` plugin path. This looks like a plugin naming or layout migration, but the fact package does not provide source semantics, so it should not be treated as proof of a user-visible capability change.

### Windows added Chrome plugin runtime and extension-host content [Confirmed] [Windows only]

The Windows package added large files under `resources/plugins/openai-bundled/plugins/chrome/app-server-runtime/`, including `codex` at 247.7 MB, `node` at 91.4 MB, and `node_repl` at 13.5 MB. The Chrome extension host binary also grew: `chrome/extension-host/windows/x64/extension-host.exe` moved from 0.6 MB to 0.8 MB, a +0.23 MB increase.

The macOS fact package does not list corresponding large newly added files. What can be confirmed is that the Windows distribution package changed significantly around Chrome plugin runtime and extension-host packaging; whether this exposes a feature entry point is not established by the supplied facts.

### LaTeX component path was renamed [Confirmed] [Windows only]

The Windows package added `resources/plugins/openai-bundled/plugins/latex/bin/tectonic.exe` at 49.6 MB and removed `resources/plugins/openai-bundled/plugins/latex-tectonic/bin/tectonic.exe`, also at 49.6 MB.

This indicates a Windows-side plugin path move from `latex-tectonic` to `latex`. The macOS fact package does not show a corresponding change.

### Computer Use plugin received a macOS version bump [Confirmed] [macOS only]

In the macOS package, the `computer-use` plugin configuration version moved from `1.0.791` to `1.0.793`. The largest macOS size changes are also concentrated in Computer Use-related binaries: `CUALockScreenGuardian` +0.17 MB, `SkyComputerUseService` +0.14 MB, and `SkyComputerUseClient` +0.08 MB.

The Windows fact package does not provide a `computer-use` plugin version change. Therefore, this can only be confirmed as a macOS-side plugin version bump, not as evidence that Windows received the same internal plugin version.

### Frontend state and icon modules changed slightly [Signal] [macOS only]

After filtering hash noise, macOS added frontend modules named `persisted-signal`, `pin`, and `pin-filled`, and removed `settings-route-state`. The module names are not obfuscated, so they can serve as directional signals around persisted state, pin icons, and settings route-state cleanup.

The Windows fact package does not list the same filtered frontend module additions/removals. This should therefore be treated only as a macOS signal, not as a cross-platform functional fact.

### Core executables and runtime components changed size [Signal] [macOS+Windows]

On macOS, `node_repl` moved from 15.0 MB to 15.1 MB, a +0.01 MB increase, and `better_sqlite3.node` moved from 1.9 MB to 2.0 MB, a +0.03 MB increase. On Windows, `Codex.exe` moved from 222.7 MB to 226.6 MB, a +3.92 MB increase; `resources/codex.exe` moved from 245.8 MB to 247.7 MB, a +1.90 MB increase; `resources/codex` moved from 218.7 MB to 219.5 MB, a +0.79 MB increase; `resources/codex-command-runner.exe` moved from 0.8 MB to 0.9 MB, a +0.10 MB increase; and `resources/node_repl.exe` remained 13.5 MB while increasing by +0.04 MB.

These are binary size signals only. They show package or build-output changes, but they do not independently identify specific features.
