# Codex Desktop 26.707.71524 macOS / Windows Comparison

> Third-party cross-platform changelog · Unofficial

The shared core of this batch is the Computer Use runtime upgrade from 0.0.3 to 0.0.4 and cleanup logic for the legacy ChatGPT Sparkle updater; other UI signals differ substantially between platforms.

Computer Use and legacy updater maintenance landed on both platforms, while macOS and Windows carried substantially different UI signals.

## About This Report

This report compares two pairs of officially signed distribution packages:

- macOS arm64: 26.707.62119 (build 5211) → 26.707.71524 (build 5263).
- Windows x64 MSIX: 26.707.8479.0 → 26.707.9564.0.

The platform version sequences and release timing are not aligned. An item is therefore classified as cross-platform only when each fact package independently supports it; evidence from one platform is not projected onto the other.

Two evidence levels are used:

- [Confirmed] covers plist data, manifests, plugin inventories, runtime manifests, dependency metadata, signature results, configuration files, and newly added bundle strings. A bundled string is confirmed to exist, but this does not mean the associated feature is enabled for every user.
- [Signal] covers size changes, readable strings and style classes in same-named frontend modules, and module-name changes after hash noise is removed. These indicate UI, state, or interaction direction rather than source-level facts.

Platform markers are `[macOS+Windows]`, `[macOS only]`, and `[Windows only]`. Pure Vite, `webview/assets`, and `.vite/build` hash renames are excluded.

Both macOS full-package samples passed notarization verification with `accepted`, sourced from `Notarized Developer ID` and signed by OpenAI OpCo, LLC. The Windows fact package includes full-package SHA-256 values but no separate MSIX signature-verification result.

## ✨ Highlights

### Computer Use Runtime Upgraded to 0.0.4 [Confirmed][macOS+Windows]

On both platforms, `cua_node/manifest.json` advances the runtime from `0.0.3/20260708050802-66a5641b5362-pr-1103797` to `0.0.4/20260713191736-ad8b1874340f-pr-1124580`. The `node_repl` archive moves from `20260707.2` to `20260713.2`. The bundled Node version remains 24.14.0.

Platform-specific packaging differs:

- macOS uses the `darwin-arm64` runtime. `node_repl` grows by about 0.02 MB, while total Computer Use content remains 365.7 MB.
- Windows uses the `windows-x64` runtime. `node_repl.exe` grows by about 0.23 MB; total content moves from 287.2 MB to 287.5 MB, although the fact package reports the rounded delta as `+0 MB, +0%`.
- Neither platform adds a `cua_node` dependency package. pnpm remains at 11.0.7; only workspace validation and `prunedAt` timestamps change.

### Bundled Browser and Chrome Plugins Aligned with the App Build [Confirmed][macOS+Windows]

The bundled `browser` and `chrome` plugin manifests advance from 26.707.62119 to 26.707.71524 on both platforms.

The `browser` plugin continues to describe ChatGPT’s in-app browser and local web targets such as `localhost`, `127.0.0.1`, `::1`, `file://`, and the current in-app browser tab, including opening, inspecting, navigating, testing, clicking, typing, and taking screenshots. The `chrome` plugin continues to cover automation that depends on existing Chrome state, including tabs, authenticated sessions, cookies, extensions, and ChatGPT Chrome Extension setup.

These are plugin-manifest versions. They intentionally differ from the outer Windows MSIX version, 26.707.9564.0.

### Windows Computer Use Plugin Version Updated [Confirmed][Windows only]

The Windows bundled `computer-use` plugin manifest advances from 26.707.62119 to 26.707.71524. Its description still states that it controls Windows desktop apps from ChatGPT through Computer Use. The macOS fact package contains no corresponding manifest change, so this item cannot be extended to macOS.

### Chrome Capability Documents and Host Binary Changed [Confirmed][Windows only]

The Windows package removes six Chrome plugin capability documents:

- Tab capabilities: `browserAuth.md`, `botDetection.md`, `cdp.md`, and `pageAssets.md`.
- Browser capabilities: `viewport.md` and `visibility.md`.

At the same time, the Windows x64 `extension-host.exe` grows from 0.8 MB to 0.9 MB, an increase of about 0.10 MB. The evidence confirms the document removals and a host-binary change; it does not establish that the underlying capabilities themselves were removed.

### Legacy ChatGPT Sparkle Updater Cleanup Paths Added [Confirmed][macOS+Windows]

Both new packages contain the same four newly added strings:

- `Failed to boot out legacy ChatGPT Sparkle job`
- `Failed to clear legacy ChatGPT Sparkle cache`
- `Failed to disarm legacy ChatGPT Sparkle updater`
- `Finished disarming legacy ChatGPT Sparkle updater`

These strings confirm bundled success and failure paths for unloading a legacy ChatGPT Sparkle job, clearing its cache, and disarming the old updater. The strings also appear in the Windows package, but the fact package does not expose their trigger conditions; they do not prove that every Windows installation runs this process.

### Task Sidebar, Pinning, and Archiving Expanded [Signal][macOS only]

Changed macOS frontend modules show a reorganization of task navigation:

- New `All tasks` and `Codex Cloud` labels, plus sections for cloud tasks, connection groups, project groups, recent tasks, and a chronological all-tasks list.
- New pin and unpin task actions.
- Archive confirmation flows that distinguish ordinary tasks, tasks with active heartbeat automation, and tasks associated with named or unnamed schedules.
- Conversation environment tooltips for local execution, a local git worktree, and Codex Cloud.
- Project-selector, remote-connection group, initialization status, connection color picker, and `Add local project` accessibility strings.
- Newly emitted module names involving `remote-connections-settings`, `remote-conversation`, `projects-index-page`, `automations-page`, and `thread-app-shell-chrome`.

Together these point to changes in task navigation, remote-connection grouping, and task lifecycle interactions. Because all supporting evidence comes from frontend-module signals, the exact UI and rollout state remain unconfirmed.

`home-announcements` also removes dismissal and content strings for gift credits, rate-limit resets, Fast mode, app promotions, and the multi-agent composer banner. This confirms only that those strings left the corresponding modules; they may have been moved, refactored, or become unused.

### Code Highlighting, Math, and Presentation Handling Expanded [Signal][macOS only]

Several same-named frontend modules grow substantially and add grammar-scope strings:

- PHP grows by about 111,110 bytes, C# by 87,762 bytes, and Python by 69,989 bytes.

These are strong signals for editor grammar resources, formula rendering, and presentation-content handling. No separate package manifest or type declaration supports promotion to source-level confirmed evidence.
