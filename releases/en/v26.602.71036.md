# Codex Desktop 26.602.71036(macOS / Windows comparison)

> Third-party cross-platform changelog · Unofficial

This batch shows Codex Desktop moving through the same app-layer update cycle on macOS and Windows, with different platform version numbers and different levels of readable evidence.  
One-liner: this cross-platform batch looks like a coordinated app-layer refresh with no new files or new cua_node dependencies on either platform.

## About This Report

This report is based on reverse diffs of two officially signed distribution packages: a full Sparkle package for macOS and an MSIX package for Windows. Each platform’s previous and latest package was compared across the file tree, manifests, app.asar, and bundled plugin directories.

Evidence is labeled in two levels: [Confirmed] means readable evidence from plist, AppxManifest, skill documents, type declarations, package.json, dependency manifests, or similar sources; [Signal] means only frontend module names, file sizes, or build artifacts changed, which can indicate direction but cannot reconstruct source intent.

Platform labels mean: `[macOS+Windows]` appears on both platforms; `[macOS only]` appears only in the macOS fact package; `[Windows only]` appears only in the Windows fact package.

Comparison scope: macOS moves from 26.602.40724 build 3593 to 26.602.71036 build 3685; Windows moves from 26.602.4764.0 to 26.602.9276.0. The release date for this batch is taken from the input facts: 2026-06-07.

## ✨ Highlights

### Version And Build Advance [Confirmed] [macOS+Windows]

On macOS, Info.plist shows `CFBundleShortVersionString` changing from `26.602.40724` to `26.602.71036`, and `CFBundleVersion` changing from `3593` to `3685`. On Windows, the version moves from `26.602.4764.0` to `26.602.9276.0`.

The platform difference is naming: macOS uses a short version plus build number, while Windows uses a four-part version. They belong to the same batch, but the numbers are not interchangeable.

### app.asar Was Rebuilt [Confirmed] [macOS+Windows]

The `app.asar` changed on both platforms. On macOS, the `ElectronAsarIntegrity` hash changed from `0d60982859a8e93d9df7bd629e63cf70b2d83c482420bee78c3af36fb0bf5e67` to `1eaacb8bd09c1647a501ac76f4718bc83badec23ad7160e1691ec161790790ab`, which is manifest-level readable evidence. The Windows fact package also lists `resources/app.asar` in the size comparison for this batch.

The platform difference is that macOS provides an explicit asar integrity hash change, while the Windows fact package provides file-level statistics and size entries but no equivalent manifest hash.

### Frontend Build Artifacts Rotated, But This Is Not A Feature Claim [Signal] [macOS+Windows]

Inside app.asar, macOS stays at `2830 → 2830` files, with `285` added, `285` removed, and `12` changed. Windows stays at `2778 → 2778` files, with `285` added, `285` removed, and `11` changed. Both fact packages explicitly state that the large churn under `webview/assets` is Vite build hash renaming and should be treated as noise.

The platform difference is minor: both platforms have the same add/remove counts, while macOS has one more changed internal file. This looks more like the same frontend build refresh than a directly nameable feature.

### Browser And Chrome Plugin Metadata Tracks The App Version [Confirmed] [macOS only]

The macOS readable diff shows the `.codex-plugin/plugin.json` version for both the `browser` and `chrome` plugins changing from `26.602.40724` to `26.602.71036`. This confirms bundled Browser and Chrome plugin metadata moving with the app version.

The platform difference matters: the Windows fact package lists the corresponding script files in the size comparison, but does not include a readable `plugin.json` diff. Therefore this is confirmed only on macOS and should not be stated as a Windows metadata fact.

### No New cua_node Dependencies [Confirmed] [macOS+Windows]

Both platforms report `0` newly added `cua_node` packages.

There is no platform difference here. At the dependency-manifest level, this batch did not introduce new `cua_node` packages.

### Top-Level File Trees Stayed Stable [Confirmed] [macOS+Windows]

The macOS bundle file count remains `2075 → 2075`, with `0` added, `0` removed, and `53` changed. The Windows file count remains `8724 → 8724`, with `0` added, `0` removed, and `15` changed.
