# Codex Desktop `26.417.40842`(build 1851)

> Third-party changelog · Unofficial · Based on reverse-engineered diffs of official update packages

The most visible changes in this release center on consolidation of Browser capabilities, a macOS folder entry point, browser sidebar interactions, and new component signals related to Chronicle.  
One-liner: browser automation is being consolidated while the desktop shell gains small but visible macOS and browser-sidebar affordances.

## ✨ Key changes

### 1. The Browser plugin now takes over Chrome backend-related documentation, while the former standalone chrome plugin was removed [Confirmed]

The Browser plugin’s `plugin.json` added keywords such as `automation`, `chrome`, `iab`, `node-repl`, and `browser-client`, along with a new long description: it teaches Codex to use the `browser-client` runtime, initialize `agent.browser.*`, and execute browser commands through either the IAB backend or the Chrome backend.

This means the browser automation entry point is moving away from the separate `chrome` plugin and toward unified ownership by the Browser plugin. The fact package also shows that `browser-client.mjs`, `SKILL.md`, installation scripts, and `plugin.json` under `plugins/chrome/` were removed.

Key evidence: the `Browser` plugin long description added “IAB backend for local/file targets or the Chrome backend for remote/profile-dependent browser work”.

### 2. Browser automation skill documentation was rewritten around the supported `agent.browser.*` surface [Confirmed]

`browser-use/skills/browser/SKILL.md` substantially rewrote its API reference section, changing it from “Atlas Agent Library API” to “Use this as the supported `agent.browser.*` surface”. At the same time, an earlier extra restriction on `tab.content.export` was removed.

For developers, this is not a small change to a single method. It is a convergence in the documentation model for browser automation: future understanding and use of browser capabilities should be centered around the supported `agent.browser.*` interface set.

Key evidence: `SKILL.md` added “supported `agent.browser.*` surface” and removed the restriction line “Only ever use `tab.content.export` ...”.

### 3. macOS now declares itself as a folder viewer [Confirmed]

The latest `Info.plist` adds `CFBundleDocumentTypes`, with type name `Folder`, role `Viewer`, and content type `public.folder`.

This means Codex is now associated at the system level with the “folder” document type. The user-visible direction is that a foundational declaration now exists for handing folders to Codex from Finder or system open entry points. However, the fact package does not provide the full interaction flow, so the exact entry-point style cannot be asserted further.

Key evidence: `LSItemContentTypes[0] = public.folder`.

### 4. The startup loading page can drag the window [Confirmed]

`webview/index.html` added `-webkit-app-region: drag` to `.startup-loader`.

This is a small but direct desktop experience change: while the app is still on the startup loading screen, the loading area itself can be used as a window drag region, reducing the friction of moving the window during startup.

Key evidence: `.startup-loader { ... -webkit-app-region: drag; }`.

### 5. The browser sidebar context menu now includes Back, Forward, and Reload strings [Confirmed]

A large number of `native-menu-locales` localization files added `browserSidebar.contextMenu.back`, `browserSidebar.contextMenu.forward`, and `browserSidebar.contextMenu.reload`. In Simplified Chinese, these correspond to “返回”, “前进”, and “重新加载”.

This indicates that the browser sidebar context menu is filling in common browser operations. Compared with keeping only “Inspect”, this is closer to what users expect from an embedded browser context menu.

Key evidence: `native-menu-locales/zh-CN.json` added the three browser sidebar menu items `back`, `forward`, and `reload`.

### 6. The Computer Use plugin received a minor version update, and an iPhone Mirroring instruction file appeared [Confirmed]

The Computer Use plugin version was updated from `1.0.750` to `1.0.755`. At the same time, Computer Use-related resources added `AppInstructions/iPhone Mirroring.md`, with one copy appearing under the main app resources path and another under the `SkyComputerUseClient.app` resources path.

This points to expanded application-instruction coverage for Computer Use into iPhone Mirroring. However, the fact package only provides the file name and plugin version change, and does not prove that the capability has been opened to all users.

Key evidence: version number `1.0.750 → 1.0.755` in `computer-use/.codex-plugin/plugin.json`, plus the new `iPhone Mirroring.md`.

### 7. A new `codex_chronicle` binary and `chronicle-setup-state` frontend module were added [Signal]

The bundle added `Contents/Resources/codex_chronicle`, with a size of `4.4 MB`; after removing Vite hash noise, the real newly added frontend modules include `chronicle-setup-state`.

These names point toward Chronicle-related setup or local component directions, but the artifact cannot reconstruct source-level intent, nor can it prove that a corresponding feature has already been exposed in the user interface.

Key evidence: new file `Contents/Resources/codex_chronicle`; new module `chronicle-setup-state`.

### 8. Frontend modules related to local environments, permission mode, and GPU debugging were adjusted [Signal]

After noise removal, newly added frontend modules include `local-environments-utils`, `use-permissions-mode`, `send-app-server-request`, and `gpu-tearing-debug-settings`; removed modules include `app-server-manager-signals`, `local-environment-action-icon-options`, `memory-feature-gate`, `telepathy-setup-state`, `use-auth`, and `use-personality`.

These module names point toward local environments, permission mode, app service requests, and GPU tearing debug directions, but the artifact cannot reconstruct source-level intent; nor can these names determine whether the settings have already been exposed in the interface.

Key evidence: the fact package’s list of real frontend code module additions and removals after hash noise was removed.
