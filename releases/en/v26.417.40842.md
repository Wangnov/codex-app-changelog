# Codex Desktop `26.417.40842` build 1851

> Third-party changelog · Unofficial · Based on reverse diffs of official update artifacts

The most visible movement in this build is around Chronicle, browser automation plugin consolidation, macOS folder handling, and several local-environment and automation UI signals.  
One-liner: Chronicle appears as a bundled sidecar, while browser automation is consolidated under `browser-use`.

## About This Report

This is not an official changelog. It is a **reverse diff report based on official distribution artifacts**. This entry is a historical backfill: the official appcast did not provide a delta package for `26.415.40636` → `26.417.40842`, so the comparison uses two official full packages.

Evidence is split into two levels:

- **[Confirmed]**: Human-readable evidence such as Info.plist keys, package.json, plugin manifests, skill docs, service entries, readable strings, and signing/notarization results.
- **[Signal]**: Indirect evidence from frontend module names, same-name frontend module string/style changes, or binary size changes. These point in a direction but cannot reconstruct source-level intent.

This report intentionally ignores hashed file churn under `webview/assets` and `.vite/build`.

## ✨ Highlights

### 1. Chronicle sidecar and research-preview entry points appear [Confirmed]

The new bundle adds `Contents/Resources/codex_chronicle`, a 4.4 MB file. The main/frontend bundle also contains readable strings such as `Chronicle feature enabled but bundled sidecar binary was not found`, `Pause Chronicle`, `Resume Chronicle`, `Starting Chronicle...`, and `Stopping Chronicle...`.

This means Chronicle is no longer just a frontend naming signal: the distributed app now contains a standalone sidecar and user-visible state text. Settings strings also show `Chronicle research preview`, permission states, and running/not-running status.

Key evidence:

```text
Contents/Resources/codex_chronicle (4.4 MB)
Chronicle feature enabled but bundled sidecar binary was not found
Pause Chronicle / Resume Chronicle
```

### 2. The Chrome automation plugin is removed, with browser capabilities consolidated into `browser-use` [Confirmed]

The bundled plugin list changes from `browser-use`, `chrome`, `computer-use` to `browser-use`, `computer-use`; the separate `chrome` plugin is removed. At the same time, the `browser-use` manifest adds keywords including `automation`, `chrome`, `iab`, `node-repl`, and `browser-client`, plus a new long description.

This points to a packaging boundary change for browser automation: it is no longer shipped as a separate `chrome` plugin, while `browser-use` now describes the IAB backend, Chrome backend, and `agent.browser.*` initialization path.

Key evidence:

```diff
- "browser"
+ "browser", "automation", "chrome", "iab", "node-repl", "browser-client"
```

```text
Removed bundled plugin: chrome — Chrome automation and Codex Chrome Extension installation support.
```

### 3. macOS now declares folder handling [Confirmed]

The new Info.plist adds a `CFBundleDocumentTypes` entry with type name `Folder`, role `Viewer`, and content type `public.folder`.

This means Codex is now declared at the system level as a handler target for folders. Together with the `workspace-root-drop-handler` name appearing in bootstrap, folder/workspace entry handling is now part of the startup path, though the exact interaction flow cannot be fully reconstructed from the artifact.

Key evidence:

```text
CFBundleTypeName = Folder
CFBundleTypeRole = Viewer
LSItemContentTypes[0] = public.folder
```

### 4. Computer Use gets a minor plugin update and iPhone Mirroring instructions [Confirmed]

The `computer-use` plugin version moves from `1.0.750` to `1.0.755`, and two `AppInstructions/iPhone Mirroring.md` files are added. Related Computer Use service binaries also show small size changes.

This shows a material update to the Computer Use capability bundle, now carrying app-instruction material for iPhone Mirroring.

Key evidence:

```diff
- "version": "1.0.750"
+ "version": "1.0.755"
```

```text
AppInstructions/iPhone Mirroring.md
```

### 5. Chat entry points and command-menu copy expand [Confirmed]

The new main/frontend bundle adds multilingual strings for creating chats, pinned chats, recent chats, and Codex version text. It also contains message IDs such as `codex.commandMenu.createChat`, `codex.commandMenu.noChatsEmptyState`, and `codex.commandMenu.pinnedThreadsGroup`.

This points to changes in the chat list, command menu, or empty-state UI. The strings are confirmed to exist in the bundle, but they do not reveal the complete UI flow.

Key evidence:

```text
codex.commandMenu.createChat
codex.commandMenu.noChatsEmptyState
codex.commandMenu.pinnedThreadsGroup
```

### 6. Browser-sidebar context menus gain back, forward, and reload [Confirmed]

Multiple `native-menu-locales/*.json` files add `browserSidebar.contextMenu.back`, `browserSidebar.contextMenu.forward`, and `browserSidebar.contextMenu.reload`. Simplified Chinese expands from only “Inspect” to “Back / Forward / Reload / Inspect”.

This means the browser sidebar’s native menu surface is more complete. The user-visible effect is likely added navigation actions in a context menu or system menu.

Key evidence:

```diff
+ "browserSidebar.contextMenu.back":"返回"
+ "browserSidebar.contextMenu.forward":"前进"
+ "browserSidebar.contextMenu.reload":"重新加载"
```

### 7. Permissions, local environments, app-server requests, and debug settings are being reshaped [Signal]

After removing hash noise, new frontend modules include `local-environments-utils`, `send-app-server-request`, `use-permissions-mode`, `gpu-tearing-debug-settings`, and `chronicle-setup-state`; removed modules include `app-server-manager-signals`, `local-environment-action-icon-options`, `memory-feature-gate`, `use-auth`, and `use-personality`.

These module names point toward reshaping around local environments, permission modes, app-server request paths, GPU tearing debug settings, and personalization/memory-related surfaces. They cannot reconstruct source-level intent, and they do not prove availability for all users.

### 8. Automation error paths and personalization settings show new UI signals [Signal]

The same-name frontend module `index` adds automation error strings, including storage unavailable, invalid id, scheduling state could not be updated, and only supported for local threads. The same-name frontend module `personalization-settings` adds Chronicle permission states, a local custom avatar folder button, and refresh-local-manifests copy.

These clues point to expanded automation state handling and personalization settings, but they come from same-name frontend module diffs and should only be treated as UI, state, or interaction signals.
