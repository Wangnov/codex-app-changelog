# Codex Desktop `26.609.30741` build 3808

> Third-party changelog · Unofficial · Based on reverse diffing official update artifacts

The clearest changes in this build are the larger Computer Use runtime, new macOS Dock/icon hooks, Sites plugin guidance updates, and several workspace UI direction signals.  
One-liner: Computer Use gets a much larger local runtime, while Dock, Mapbox, Sites, and workspace UI surfaces show new capability hooks.

## About This Report

This is not an official changelog. It is a **reverse diff report based on official distribution artifacts**. The comparison covers macOS arm64 from `26.608.12217` build `3722`, released on `2026-06-09`, to `26.609.30741` build `3808`, released on `2026-06-11`.

This entry is a historical backfill. The official appcast did not provide a delta package for this version pair, so the fact package compares two official full ZIP packages directly and provides SHA-256 hashes, notarization results, and structured bundle diffs.

Evidence levels are split into two tiers: **[Confirmed]** means there is direct readable evidence such as configuration, dependencies, plugin manifests, skill documents, CSP, Info.plist, signatures, or newly added strings. **[Signal]** means the evidence comes only from frontend module names, same-name frontend module diffs, or size changes; it points in a direction, but cannot be reconstructed into source-level intent.

## ✨ Highlights

### 1. The Computer Use runtime grew substantially, adding PDF, OCR, image-processing, and browser-automation dependencies [Confirmed]

`cua_node` grew from `213.5 MB` to `365.8 MB`, an increase of `152 MB` or `71%`. New dependencies include `pdfjs-dist@5.4.624`, `tesseract.js@7.0.0`, `tesseract.js-core@7.0.0`, `sharp@0.34.5`, `@napi-rs/canvas@0.1.91`, `playwright@1.57.0`, `playwright-core@1.57.0`, `pixelmatch@7.1.0`, `pngjs@7.0.0`, `jpeg-js@0.4.4`, and `bmp-js@0.1.0`.

This gives the local Computer Use runtime a much broader dependency base for PDF reading, OCR, screenshot/image processing, pixel comparison, and browser automation. The dependencies are confirmed in the artifact, but the dependency list alone does not prove that every product entry point exposes these capabilities.

### 2. `@oai/sky` was upgraded, and Linux Computer Use actions now pass options consistently [Confirmed]

`@oai/sky` moved from `0.4.8-202606020558-pr-981870-f570e0604110` to `0.4.10`. `Options.d.ts` adds `mouse_size_px?: number`, with documentation saying it resizes the mouse pointer in screenshots to a specified pixel size. `sky_linux.js` now passes `--mouse-size-px`, and `click`, `drag`, `move`, `scroll`, `press_key`, `type_text`, and `get_screenshot` now pass `options` into `sky_linux`.

For developers, Linux Computer Use action execution and screenshots now share the same option object, and screenshot mouse pointer size has an explicit configuration path. Another interface tightening is that `to_data_url(bytes, mime_type)` now requires `mime_type`.

### 3. A macOS Dock tile plugin and icon resources were added [Confirmed]

`Info.plist` adds `NSDockTilePlugIn = CodexDockTilePlugin.plugin`, and the bundle adds `Contents/PlugIns/CodexDockTilePlugin.plugin`. Its signing identifier is `com.openai.codex.dock-tile-plugin`, with authority `Developer ID Application: OpenAI OpCo, LLC (2DC432GLL2)`. The build also adds `CodexAppIconBaseName = icon`, `package.json` adds `codexAppBrand = codex`, and resources now include `icon.png`, `icon-dev-dark.png`, `icon-chatgpt.png`, and `icon-dev-light.png`.

These artifacts confirm a Dock tile plugin and multiple icon resources, along with the string `Preferred macOS Dock icon`. They do not, by themselves, prove exactly how Dock icon selection or Dock state is exposed in the UI.

### 4. The Sites plugin moved to `0.1.15`, with stricter D1 and publishing guidance [Confirmed]

The `sites` plugin moved from `0.1.14` to `0.1.15`, and its long description now includes `games` among supported website types. `sites-building/SKILL.md` adds D1 constraints: application queries and runtime initialization should use prepared statements on the raw D1 binding, each `prepare()` call should receive exactly one SQL statement, and multi-statement work should use `batch([...])`.

`sites-hosting/SKILL.md` also changes its role. Instead of directly describing project creation, version saving, deployment, and access control flows, it now emphasizes local preparation of deployable Sites source and build artifacts, while treating the Sites connector and tool descriptions as the source of truth for site creation, environment variables, versions, deployments, and access control. For developers, this changes the default execution path for Sites tasks and the expected preparation before database migration or deployment work.

### 5. WebView CSP now allows Mapbox connections [Confirmed]

Both `webview/index.html` and `webview/avatar-overlay-composition-surface.html` add `https://api.mapbox.com` and `https://events.mapbox.com` to `connect-src`.

This means the main WebView and the avatar overlay composition surface are now allowed to connect to Mapbox API and event endpoints. The fact package does not include a concrete UI, so it does not prove that a map feature is user-facing.

### 6. Main/frontend bundle strings point to browser profile import, terminal detection, and Git prompt paths [Confirmed]

New readable strings in the bundle include `Browser profile is no longer importable`, `Owl browser profile importer is unavailable`, `Cookie domain selection requires cookie import`, `Select cookies, passwords, or both to import`, `fluent terminal`, `git bash`, `tera term`, `windows powershell`, `windows terminal`, and `Using the supplied git context below, generate a single-line git commit message.`.

These strings are confirmed to exist in the minified artifact. They indicate new or changed paths around browser profile and cookie import, terminal type identification, and one-line Git commit message generation, but the compressed bundle does not allow full source-level semantics to be reconstructed.

### 7. Thread bottom panels, side panels, and tab actions appear to be expanding [Signal]

New real frontend modules include `thread-side-panel-active-signals`, `windows-tabs-open`, `windows-tabs-open-schema`, and `run-app-action-in-primary-window`. The same-name module `thread-app-shell-chrome` adds strings for opening side chat, browser tab, review tab, terminal, timeline tab, workspace file browser, and bottom panel. `thread-page-bottom-panel-state` grows by `293231 bytes` and adds strings around terminal crash reload, xterm buffers, mouse events, and accessibility.

These clues point toward expanded bottom/side panel behavior, tab actions, and embedded terminal interactions inside Thread pages, but they are frontend module and string signals only and cannot be converted into source-level intent.

### 8. Plugin detail, sharing, and MCP mention scopes show expansion signals [Signal]

The new module list includes `plugin-detail-queries`. `plugin-detail-page` adds strings such as `Add people or groups`, `Anyone in this workspace with the link`, `Accessible label for the MCP server enablement toggle on the plugin detail page`, and skill preview modal and skill card actions menu labels. `at-mention-list` adds `Computer use`, `Custom agents`, `Desktop apps`, `Live agents`, `MCP servers`, `Mac apps`, and class names related to `mcp-capability-mentions`.

These clues point toward richer plugin detail pages, workspace sharing, MCP server enablement, and mention scopes, but they remain UI and interaction signals.

### 9. Voice, dictation, and settings controls show new surface area [Signal]

New modules include `transcribe-audio`, `use-realtime-waveform`, and `phone`. `general-settings` adds strings for hold-to-dictate hotkey capture, dictation dictionary entries, base font size for code across chats and diffs, base font size for the Codex UI, work mode settings, ambient suggestions, and the Popout Window projectless default setting.

These changes point toward work on voice/dictation, realtime waveform UI, base font size controls, work mode, ambient suggestions, and Popout Window defaults. They do not prove the exact release state of those features.

### 10. Feedback, reporting, tool activity summaries, and workspace notification copy changed [Signal]

`plan-summary-item-content` adds strings for Codex thread report reasons, external report forms, report submission failures, and artifact feedback around incorrect content, topic coverage, and templates. `local-conversation-thread` adds collapsed tool activity summaries for code searches, file reads, loaded skill definitions, named MCP tool calls, and running integrations, while removing auto-review approved/denied copy. `app-main` adds `Workspace announcement`, `Unread chats`, and `Notify owner`, while removing `Introducing Codex mobile` announcement strings.

These clues point to changes in feedback/reporting, tool activity summaries, and workspace notification UX, but same-name frontend module diffs must be treated as signals only.

### 11. macOS permission prompt text received punctuation cleanup [Confirmed]

`NSAppleEventsUsageDescription`, `NSCameraUsageDescription`, and `NSMicrophoneUsageDescription` all removed their trailing period while keeping the same meaning: Apple Events are used to control Mac apps on the user's behalf, Camera is used for video input, and Microphone is used for voice input.

This is a small user-visible copy change in macOS permission prompts. The fact package does not show any expanded permission scope.
