### 2. New shared-object snapshot channel between the frontend and desktop shell [Confirmed]

`.vite/build/preload.js` adds `codex_desktop:get-shared-object-snapshot`, and exposes `getSharedObjectSnapshotValue` through `electronBridge`. At the same time, the local shared-object cache is updated both when the frontend sends `shared-object-set` to the desktop shell and when the desktop shell sends back `shared-object-updated`.

This indicates that the desktop shell and Webview now maintain a synchronizable shared state. User-visible effects may appear as improved consistency across windows, pages, or between the native desktop layer and frontend state, but the fact pack cannot prove which specific UI feature has been exposed.

Key evidence:

```diff

+ t.type===`shared-object-set`&&y(t.key,t.value)
```

This means the embedded frontend can load media resources from the application protocol or inline data sources. It is a verifiable security-policy change, but the fact pack does not identify the specific media feature scenario.

This is friendlier for debugging, isolated runs, automated testing, or multi-instance environments. The fact pack only shows a change in path-selection logic and does not include any user-facing settings entry point.

### 8. Build and runtime packaging flow adjusted [Confirmed]

`package.json` adds a `build:node-runtime` script. `rebuild:forge-natives` changed from directly running `pnpm rebuild macos-alias` to executing `scripts/rebuild-forge-natives.mjs`. Meanwhile, `lint` and `lint:fix` now include `--threads=1`.

This indicates that the internal build flow is moving toward scripted wrappers and limiting lint concurrency. It may not be visible to ordinary users, but it is directly noticeable to people debugging, packaging, or reproducing builds.

Key evidence:

```diff
+ "build:node-runtime": "node ./scripts/bundled-runtimes/bundle.mjs"
- "rebuild:forge-natives": "pnpm rebuild macos-alias"
+ "rebuild:forge-natives": "node ./scripts/rebuild-forge-natives.mjs"
+ oxlint --threads=1
```
