# Codex Desktop 26.519.22136 (macOS / Windows comparison)

> Third-party cross-platform changelog · Unofficial

The main cross-platform takeaway: macOS advanced to 26.519.22136 (build 3003) with Chrome, Browser, and app-shell changes, while the Windows 26.519.2736.0 MSIX package in this comparison has no file-level change at all.

One-liner: macOS moved forward with Chrome/profile and app-shell changes, while the Windows package in this comparison stayed byte-for-byte unchanged.

## About This Report

This report is based on reverse diffs of two official signed distribution packages: macOS via Sparkle-related full-package comparison, and Windows via MSIX package comparison. The macOS comparison range is 26.513.31313 (build 2867) → 26.519.22136 (build 3003). The Windows comparison range is 26.519.2736.0 → 26.519.2736.0.

Evidence is split into two levels: [Confirmed] means the change is backed by readable plist, AppxManifest, CSP, skill documentation, type declarations, package.json, dependency listings, or readable script diffs. [Signal] means the evidence comes only from frontend module names or size changes, which can indicate direction but cannot reconstruct source intent or confirm that a feature is user-facing.

Platform tags mean: [macOS+Windows] appears in both platform fact packs; [macOS only] appears only in the macOS diff; [Windows only] appears only in the Windows diff. The key caveat for this batch is that the Windows comparison package has identical SHA-256 hashes, unchanged file counts, and an unchanged app.asar.

## ✨ Highlights

### Chrome Extension Install Checks Now Scan Multiple Profiles

[Confirmed] [macOS only] The Chrome plugin’s `check-extension-installed.js` no longer checks only one selected Chrome profile. It now scans every usable `Default` or `Profile X` directory and reports each profile’s installed, registered, enabled, disabled-reason, and version state. The JSON output now includes a `profiles` array and a `selectedProfileDirectory` hint.

Platform difference: the macOS package contains both script and skill-documentation updates. The Windows MSIX comparison package has no content changes, so there is no evidence that Windows has received this batch of behavior.

### Chrome Native Messaging Host Installation Now Writes App-Server Configuration

[Confirmed] [macOS only] The Chrome plugin’s `installManifest.mjs` adds `channel:"prod"`, writes `extension-host-config.json`, and introduces app-server-runtime configuration fields including `codexCliPath`, `browserClientPath`, `nodePath`, `nodeReplPath`, `proxyHost`, and `proxyPort`. The new-file list also shows `app-server-runtime/codex`, `app-server-runtime/node`, and `app-server-runtime/node_repl` under the Chrome plugin.

Platform difference: macOS has expanded Chrome native messaging host installation from a manifest-only setup to a runtime-configured setup. The Windows fact pack shows 0 new files and 0 content changes.

### Chrome Plugin Version Now Matches the Desktop Version

[Confirmed] [macOS only] In `plugins/chrome/.codex-plugin/plugin.json`, the Chrome plugin version changed from `0.1.7` to `26.519.22136`, matching the macOS desktop short version.

Platform difference: this appears only in the macOS diff. The Windows comparison package has no plugin-file changes.

### Browser Tab Claiming and Retention Semantics Were Revised

[Confirmed] [macOS only] The Chrome skill documentation updates the semantics for claiming an already-open Chrome tab. Instead of saying the tab is moved into an agent tab group, it now says the current browser session gains control of that tab. The `finalize` cleanup semantics were also revised: deliverable and handoff tabs are released from session control and left open, rather than being uniformly moved into the `✅ Codex` tab group.

Platform difference: the Chrome skill documentation changed on macOS. There is no corresponding Windows file-change evidence.

### Browser and Chrome Skill Docs Relax waitForTimeout Guidance and Clarify keep Fields

[Confirmed] [macOS only] Both Browser and Chrome skill documents remove the rule saying not to use `tab.playwright.waitForTimeout(...)`. They also change `FinalizeTabsOptions.keep` from “Tabs to keep open” to an explicit post-cleanup preservation disposition, and clarify `FinalizeTabsKeep.tab` as the tab object to keep.

Platform difference: these documentation changes appear only in the macOS package. The Windows comparison has no changes.

### Large Frontend App-Shell, Workspace, Git, and Plugin-Page Reorganization Signals

[Signal] [macOS only] After excluding Vite hash noise, the macOS app.asar adds many unobfuscated module names, including `app-shell-state`, `app-shell-tab-controller`, `app-window`, `apps-availability`, `browser-sidebar-manager`, `git-branch-switcher`, `plugin-auth`, `plugin-config-edits`, `thread-side-panel-tabs`, and `worktree-environment-dropdown`. It also removes modules such as `global-settings`, `plugins-page-section`, `plugins-page-selectors`, `remote-control-connections-visibility`, and `vscode-api`.

Platform difference: these frontend reorganization signals appear only in the macOS app.asar. The Windows app.asar remains 1530 → 1530, with 0 additions, 0 removals, and 0 modifications.

### macOS Bootstrap Adds an Extra Startup Gate Signal

[Signal] [macOS only] The readable diff in `.vite/build/bootstrap.js` shows an additional condition before main app startup: Sparkle initialization and main-module import now occur only after `await t.g()` passes. Because this is packaged startup code, the fact pack does not reveal the business meaning of that condition.

Platform difference: this startup-path change appears only in the macOS package. The Windows package has no content changes.

### The Windows Comparison Package Has No Material Update

[Confirmed] [Windows only] On Windows, both the previous and latest packages are `26.519.2736.0`, and both full packages have the same SHA-256: `bf284a70f0c62da1911a62700465c31ce21f849278cd9a8290428c9b43accda1`. The file tree remains 832 → 832, with 0 additions, 0 removals, and 0 content changes. The app.asar remains 1530 → 1530, with 0 additions, 0 removals, and 0 modifications.

Platform difference: in this cross-platform comparison, the Windows package does not show the Chrome, Browser, frontend app-shell, or startup-path changes seen on macOS.
