# Codex Desktop 26.519.31651 macOS / Windows Comparison

> Third-party cross-platform changelog · unofficial

## About This Report

This report is based on machine-extracted diffs from reverse-reconstructed official signed distribution packages: macOS arm64 from `26.519.22136` build `3003` to `26.519.31651` build `3017`, and Windows x64 from `26.519.2736.0` to `26.519.3891.0`. The two platforms may have mismatched versions and publication timing, so every conclusion is tied to its own fact package rather than carried across platforms.

Evidence has two levels: [Confirmed] covers plist, manifest, plugin `plugin.json`, signature verification, package structure statistics, and readable configuration outside the “same-name front-end module change hints” section; [Signal] covers same-name front-end module diffs, module names, size deltas, style class names, and readable string deltas. Platform scope is marked as `[macOS+Windows]`, `[macOS only]`, or `[Windows only]`.

This input sample does not provide expandable facts for newly bundled plugin details, Computer Use runtime totals, exact added/removed front-end module lists, newly added readable strings in the main process or front-end bundles, content security policy changes, newly added components/resources, or Windows AppxManifest changes. Those areas are treated as “not provided” in the raw data section.

## ✨ Highlights

### [Confirmed] [macOS+Windows] The bundled browser plugin version is synced to `26.519.31651`

On both platforms, `browser/.codex-plugin/plugin.json` only shows a version-field change: `26.519.22136` → `26.519.31651`. The plugin description remains focused on Codex’s in-app browser automation: opening, inspecting, navigating, testing, clicking, typing, and screenshotting local Web targets, including `localhost`, `127.0.0.1`, `::1`, `file://`, and the current in-app browser tab.

The platform difference is path-only: macOS uses `Contents/Resources/plugins/openai-bundled/plugins/browser/.codex-plugin/plugin.json`, while Windows uses `resources/plugins/openai-bundled/plugins/browser/.codex-plugin/plugin.json`. The fact package does not show newly added browser plugin skills, service entrypoints, default prompts, or permission changes, so this confirms only plugin metadata version sync, not new capability surface.

### [Confirmed] [macOS+Windows] The bundled chrome plugin version is synced to `26.519.31651`

Its description still points to Chrome automation for remote URLs, authenticated or profile-dependent pages, existing Chrome tabs, cookies, extensions, and Codex Chrome Extension setup.

Again, the platform difference is only the file path: macOS uses `Contents/Resources/plugins/openai-bundled/plugins/chrome/.codex-plugin/plugin.json`, while Windows uses `resources/plugins/openai-bundled/plugins/chrome/.codex-plugin/plugin.json`.
