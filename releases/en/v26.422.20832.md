# Codex Desktop `26.422.20832` build 2025

> Third-party changelog · Unofficial · Based on reverse diffs of official full packages

The most visible change in this build is bundled local LaTeX/TeX compilation, alongside a Browser Use workflow that is more explicitly tuned for local in-app browser targets.  
One-liner: Codex gains bundled LaTeX compilation and a more local-dev-oriented Browser Use workflow.

## About This Report

This is not an official changelog. It is a reverse diff report based on official signed distribution artifacts. The comparison range is macOS arm64 `26.417.41555` build `1858` to `26.422.20832` build `2025`.

This run is a historical backfill: the official appcast did not provide a Sparkle delta package for this version pair, so the comparison was performed directly between two official full ZIP packages. The report summarizes facts visible in bundle files, plugins, configuration, strings, and component sizes.

Evidence levels use two labels: **[Confirmed]** means readable direct evidence such as plugin manifests, skill docs, Info.plist, package.json diffs, signing output, or newly observed readable strings; **[Signal]** means only module names, same-name frontend module string/style diffs, or binary size changes are available. A signal points in a direction, but the artifact does not recover source-level intent.

Hashed filename churn under `webview/assets` and `.vite/build` is not treated as feature addition or removal. Internal `app.asar` added/removed file counts are also excluded from the raw data tables to avoid misreading Vite hash renames as real product changes.

## ✨ Highlights

### 1. New Bundled `latex-tectonic` Plugin [Confirmed]

The bundled plugin list changed from `browser-use`, `computer-use` to `browser-use`, `computer-use`, `latex-tectonic`. The new plugin is version `0.1.0`, displays as `LaTeX Tectonic`, and its description explicitly says it compiles LaTeX and TeX documents with a bundled Tectonic executable.

This gives Codex a local PDF build path that does not rely on a system TeX installation. Key evidence includes the new local plugin entry `./plugins/latex-tectonic` in `marketplace.json`, the skill doc command `bin/tectonic --outdir <output-directory> <tex-file>`, and the largest newly added file, `bin/tectonic`, at `50.3 MB`.

### 2. `browser-use` Is Repositioned as Browser Use for Local IAB Verification [Confirmed]

The `browser-use` plugin interface name changed from `Browser` to `Browser Use`. Its short and long descriptions now emphasize the Codex in-app browser, local targets, and `localhost`, `127.0.0.1`, `::1`, and `file://`. Default prompts changed from browser-client bootstrapping instructions to `Open localhost:3000` and `Test my latest changes in the browser`.

For frontend developers, the plugin now reads more like a local development verification surface for the in-app browser, not a generic browser automation description. The plugin also added `composerIcon` and `logo` pointing to `./assets/browser.png`, with `browser.png` included in the bundle.

### 3. Browser Sidebar Gains a Comment Action, with Expanded Site Access and Permission Copy [Confirmed]

Multiple native menu locale files added `browserSidebar.contextMenu.commentWithCodex`, including Simplified Chinese `评论`, Traditional Chinese `評論`, Korean `댓글 달기`, and Japanese `コメント`. The same batch also substantially revised browser load-error, DNS, proxy, and network-access wording.

The new bundle also contains readable strings in the site-access and permission area, including `Abrir sites sem perguntar`, `Abrir sitios web sin preguntar`, `A Codex soha nem nyitja meg ezeket a webhelyeket`, `Acceso a la red`, `Acceso de lectura`, and `Acceso de escritura`. This confirms an expanded user-facing copy surface around browser sidebar actions and site/permission policy.

### 4. Chronicle, Memories, Custom Avatars, and Personalized Suggestions Copy Enters the Artifact [Confirmed]

New readable strings include `A configurar o Chronicle`, `Activa Memories para usar Chronicle`, `Activez Memories pour utiliser Chronicle`, `Aangepaste avatars`, `A carregar avatares personalizados`, and `Activar sugerencias personalizadas`.

These are direct evidence of new user-facing copy around Chronicle, Memories, custom avatars, and personalized suggestions. Whether each surface is enabled for every user cannot be determined from bundle strings alone.

### 5. `computer-use` Plugin and Desktop Control Service Updated [Confirmed]

The `computer-use` plugin version increased from `1.0.755` to `1.0.758`. Its service binary `SkyComputerUseService` grew from `11.9 MB` to `14.1 MB`, a `+2.23 MB` change.

This confirms an update to desktop-control-related components. The size delta alone does not establish source-level intent, but the version bump and service entry path are direct evidence.

### 6. PDF / Artifact Preview and Presentation Runtime Show Enhancement Signals [Signal]

After hash-noise removal, the newly observed frontend modules include `pdf` and `artifact-preview-page-navigation`. The same-name `pdf-preview-panel` module added strings such as `Zoom in`, `Zoom out`, `Zoom to fit`, and `artifactTab.pdfPreview.zoomIn`. The `runtime.worker` diff added clues such as `Presentation help plugin is not installed`, `presentation-backed`, `move-slide`, and `text-box`.

These signals point toward PDF preview zoom, artifact page navigation, and presentation runtime work, but the artifact does not recover source-level intent and does not confirm which entry points are enabled.

### 7. Custom Avatar Settings, Experimental Settings, and Browser Agent Settings Surface Expanded [Signal]

The de-noised new frontend modules include `codex-avatar`, `browser-use-settings`, `experimental-features-settings`, `external-agent-config-utils`, and `use-browser-agent-availability`. The same-name `appearance-settings` module also contains clues around custom avatars, a local folder, manifest refresh, and a floating avatar overlay.

These signals point toward personalization, experimental features, browser agent availability, and external agent configuration, but the artifact does not recover source-level intent.

### 8. New Native Helper and Notification Sound Resource Changes [Confirmed]

The bundle added `Contents/Resources/native/bare-modifier-monitor`, and `package.json` added `pnpm exec node ./scripts/build-bare-modifier-monitor.mjs` to both the `dev` and `owl` scripts. The bundle also added `Contents/Resources/codex-notification.wav` and removed the older `Contents/Resources/notification.wav`.

This confirms a new native helper related to modifier monitoring and a notification sound asset replacement or migration. The fact package does not provide higher-level behavior details, so no further interaction change is inferred.

### 9. Frontend Engineering Adds Analytics, Routing, Auth, and Remote Middleware Modules [Signal]

The de-noised new module list also includes `analytics-logger-provider`, `auto-track`, `product-event-signal`, `middleware`, `routing-middleware`, `remote-middleware`, `use-auth`, `use-configuration`, and `app-server-manager-signals`.

These module names point toward analytics, routing, middleware, auth, configuration, and app-server management signals, but the artifact does not recover source-level intent. They are lower-level architecture clues rather than directly claimable shipped features.
