In `webview/index.html`, the previous version mainly loaded one entry script and one stylesheet. The new version adds a large number of `modulepreload` links and splits out multiple CSS resources.

This means the frontend loading structure during app startup changed significantly: more feature modules are explicitly preloaded, possibly to reduce waiting time during later view switches or modal openings. However, the specific performance benefit cannot be determined from the artifacts alone.
