# Codex 桌面版 26.707.72221 macOS / Windows 对照

> 第三方跨平台变更日志 · 非官方

本批次共同更新了 Chrome 与内置浏览器插件，而 Windows 和 macOS 分别显露出自动化文档扩充与多组应用层界面重组。

Both platforms refreshed the Chrome and in-app browser plugins, while Windows added automation capability documentation and macOS showed broad UI restructuring signals.

## 关于这份报告

本报告比较两组官方签名全量分发包：

- macOS arm64：`26.707.71524` build 5263 → `26.707.72221` build 5307。
- Windows x64 MSIX：`26.707.9564.0` → `26.707.9981.0`。Windows 事实包中的平台字段误写为 macOS arm64，本报告依照输入所定义的 Windows x64 MSIX 样本归类。
- 两个平台的版本号与发布时间可能错位，因此所有结论均按各自样本独立比较，不以一端推断另一端。
- macOS 两个全量包均通过 Apple 公证验证，来源为 OpenAI OpCo, LLC 的 Developer ID。
- Windows 输入给出了两个全量包的 SHA-256，但没有提供 MSIX 签名验证输出。

证据分为两档：

- 【实证】来自 manifest、plist、插件描述、技能或能力文档、依赖清单、服务入口、签名结果，以及不属于同名前端模块线索的包内可读内容。
- 【信号】来自前端模块名、压缩 bundle 体积、同名模块字符串差异或样式类名。它表示代码进入或离开某个构建分块，不证明功能已开放，也不等同于源码级增删。

平台标记为 `[macOS+Windows]`、`[仅 macOS]` 和 `[仅 Windows]`。

证据盘点结果：

- 版本、渠道、Bundle 文件树、Computer Use 运行时、`cua_node` 依赖及 `app.asar` 统计均已分别核对。
- 两端均提供 Chrome 与 Browser 插件 manifest；Windows还提供 Computer Use manifest。输入未列出完整的内置插件清单增减。
- Windows 新增六份 Chrome 能力文档；macOS 没有新增 Bundle 文件。
- macOS 提供真实前端模块增删和同名模块改动线索；Windows 未提供对应模块明细。
- macOS 提供 Info.plist 与 HTML 入口差异；Windows 未提供 AppxManifest、CSP 或新增可读字符串区块。
- macOS 的 CSP 行仅作为 `index.html` diff 上下文出现，没有显示策略内容变化；不能据此宣称策略发生调整。

## ✨ 重点变化

### Chrome 与内置浏览器插件同步升版

【实证】[macOS+Windows]

两端的 `chrome` 和 `browser` 插件 manifest 都从 `26.707.71524` 更新为 `26.707.72221`。其包内描述继续明确区分两条自动化路径：

- Chrome 插件面向现有 Chrome 状态，包括标签页、登录会话、Cookie、扩展和 ChatGPT Chrome Extension 设置；在专用连接器、API 或 CLI 失效时，需要用户重新认证或明确批准 Chrome 作为回退。
- Browser 插件面向应用内浏览器，覆盖打开、检查、导航、测试、点击、输入及截图，并明确提及 localhost、回环地址、`file://` 与当前应用内浏览器标签页。
- 两端的 `browser-client.mjs` 均有极小内容变化，但四舍五入后仍为 1.0 MB，事实包未提供足以解释其行为差异的可读 diff。

这证明插件构建随本批次更新，但 manifest 版本变化本身不能证明自动化能力对所有账户或任务都已启用。

### Windows 扩充 Chrome 自动化能力文档

【实证】[仅 Windows]

Windows Bundle 新增六份 Chrome 插件能力文档，分别涉及：

- 标签页浏览器认证：`browserAuth.md`
- Bot 检测：`botDetection.md`
- Chrome DevTools Protocol：`cdp.md`
- 页面资源：`pageAssets.md`
- 浏览器视口：`viewport.md`
- 浏览器可见性：`visibility.md`

这是完整落盘的插件文档扩充，不只是包体信号。事实包没有给出文档正文，因此可以确认这些能力主题进入分发包，但不能进一步断言其具体命令、默认策略或用户开放范围。

macOS Bundle 没有新增文件，也未列出这六份文档的内容变化。

### Windows Computer Use 插件随批次升版，运行时规模不变

【实证】[仅 Windows]

Windows 的 `computer-use` 插件 manifest 从 `26.707.71524` 更新为 `26.707.72221`，描述仍为通过 Computer Use 从 ChatGPT 控制 Windows 桌面应用。

与此同时：

- Windows `cua_node` 内容总量保持 287.5 MB。
- Windows 没有新增 `cua_node` 依赖包。
- 这组证据确认插件版本更新，但不支持声称底层 Computer Use 运行时扩容或新增依赖。

macOS 的 `cua_node` 同样没有数量变化，保持 365.7 MB，且没有新增依赖；macOS 事实包没有列出对应的 Computer Use 插件 manifest 改动。

### 站点配置、域名、密钥与发布分享界面正在重组

【信号】[仅 macOS]

`appgen-settings-page` 同名模块新增一组高度相关的字符串和状态类名，指向站点管理界面：

- Site title、站点 slug 与删除确认。
- 环境变量或 secret 的键和值输入、删除行以及 `Add secret`。
- 自定义域名的添加、移除、状态刷新和 DNS 设置记录。
- 输入校验、禁用状态与状态色样式。

`appgen-share-dialog` 同时出现：

- 添加人员或群组。
- 工作区内持链接访问与面向公网访问。
- 发布到公共互联网。
- 复制和外部打开在线站点 URL。
- 保存中状态及访问权限文案。

真实新增模块还包括 `appgen-publication-terms-route`，并重组了 Appgen library、page 与 settings 相关分块。多组相互印证的模块和文案说明站点配置、发布及分享流程进入新的前端构建形态，但这些证据仍来自压缩前端分块，不能确认已向所有用户开放。

Windows 输入未提供对应的前端模块或字符串证据。

### 插件安装接入与浏览器扩展流程增强

【信号】[仅 macOS]

插件安装相关同名模块新增了成组文案：

- `App enabled`、`App disabled`、`App unavailable`。
- 必需应用尚未连接、已经连接或已打开连接流程的按钮状态。
- Google 插件需要付费计划的徽标。
- 浏览器扩展及打开扩展设置 URL。
- 完成插件引导后尝试插件或查看插件详情。
- 插件安装弹窗中的 personalization 开关。

真实模块重组还把 `plugin-detail-page` 与 skills、plugins、MCP settings、automations 及多个 onboarding/new-thread 分块重新组合。Skills/Plugins 设置共享模块新增搜索框清除按钮与 `skills.pageSearchInput.clear`。

这些是插件发现、连接、安装和试用流程调整的强信号；不能把分块变化解释为新插件已经安装，也不能因旧 `skills-page` 分块中的创建、刷新和空状态文案消失，就断言相关功能被删除。

Windows 输入没有对应界面线索。

### 外部代理、项目、聊天与工具设置导入

【信号】[仅 macOS]

导入设置模块新增了一组连续的可访问性文案，覆盖：

- 外部代理 provider 列表与开关。
- 外部代理导入选项和自定义对话框。
- 正在导入与完成导入的行状态。
- 项目、近期聊天、工具与设置的导入说明。
- 找到的应用数量及导入提示中的代理数量。

同一模块移出了 `Agent environment`、自动检测语言、默认文件打开位置、默认权限和默认终端位置等字符串。证据支持导入界面和设置分块发生重组，但不支持断言这些旧设置本身已从产品删除。

Windows 输入没有对应证据。

### 云任务目标与侧栏状态

【信号】[仅 macOS]

项目、热键线程窗口和线程外壳共享模块新增：

- `Cloud task`
- 设置、更新和清除目标失败的错误路径
- 目标不能为空
- 环境 hover card 标签
- 分钟、小时、天、周、月、年的紧凑时间格式
- `pending-worktree-create`、`pending-worktree-update-metadata`
- `write-thread-goal-attachment-for-host`

这些字符串与状态类名共同指向任务目标写入、云任务标识、worktree 等待状态和侧栏时间展示的调整。旧的 Codex Micro bridge 分块则移出了 `All tasks`、`Codex Cloud` 以及多个侧栏任务分组标签，说明相关代码可能被重新分配到其他分块；不能据此判断云任务入口被移除。

Windows 输入没有对应前端线索。

### macOS 权限引导增加 Finder 定位路径

【信号】[仅 macOS]

personalization settings 模块新增：

- 在 Finder 中显示当前应用的按钮。
- 当应用未出现在 macOS 权限设置页时，将应用从 Finder 拖入该页面的说明。
- `permissions.showAppInFinder`。

这是与 macOS 系统权限修复直接相关的界面信号，但没有 plist 权限键或独立配置变化佐证，因此不提升为【实证】。

### 调试面板覆盖浏览器、任务与 worktree 状态

【信号】[仅 macOS]

`debug-modal` 从接近空分块增长到约 0.2 MB，并新增 `Active browser tab`、`Active conversation`、`Ambient suggestion tasks`、`App Actions`、`Account ID`、`Activity grouping version`、`Allow host`、`Allow host starting`、自动清理 worktree 以及命令不可用等字符串。

真实模块列表也新增了 debug window 与 agent settings、ambient suggestions content、workspace selection 的组合分块。这表明内部诊断面板的覆盖面正在扩大，但调试入口是否面向普通用户不可由该证据确定。

### 编辑器与代码预览语法资源扩充

【信号】[仅 macOS]

多个同名语法模块明显增长，并出现完整的语法 scope 字符串：

- C++：注释、数值、模板、运算符重载、lambda 返回类型和 range-based 语法。
- Less 与 PostCSS：媒体查询、容器查询、选择器、属性值和 at-rule。
- Java：类、内部类、方法调用、数组初始化、注解参数。
- YAML：块与流式 mapping、sequence、anchor、tag 和时间戳。
- Markdown：标题、frontmatter、HTML、链接、Git commit 与 rebase。
- 另有 GDScript、正则表达式、Haml、CMake。

这些资源进入了新的前端 bundle 形态，支持编辑器高亮或解析资源发生扩充的判断方向；由于证据来自压缩模块字符串，仍按【信号】处理。

### 前端入口与大分块发生广泛重排

【信号】[仅 macOS]

macOS `app.asar` 内部文件总数保持 6,017，但记录了 4,337 个新增路径、4,337 个移除路径和 11 个内容改动。绝大多数增删来自 Vite hash 改名，不能当作功能增减。

去除 hash 噪音后，真实模块组合仍显示以下方向的重新切分：

- first run、新线程面板、onboarding 与 workspace selection。
- quick chat、ChatGPT conversation、remote/local conversation。
- settings、keyboard shortcuts、local/cloud environments 和 remote connections。
- pull request、code review、artifact tab 与 notebook preview。
- Codex Micro 设置、joystick HUD、debug window 和 agent settings。
- Appgen library、publication terms、site settings 与分享。
- plugins、skills、MCP 和 automations。

`index.html` 与 avatar overlay 页面切换到新的入口和预加载分块，但 CSP 文本没有出现在 diff 的修改行中。这里能确认构建图和前端加载边界变化，不能逐项等同于新增或删除功能。

Windows `app.asar` 仅记录 2 个新增路径、2 个移除路径和 10 个内容改动，输入没有提供可用于同级归纳的模块名。
