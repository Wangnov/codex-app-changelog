# Codex 桌面版 26.611.61753(macOS / Windows 对照)

> 第三方跨平台变更日志 · 非官方

本批次更像一次跨平台同步的小步更新：应用层 `app.asar` 有少量内容变化，但文件总量、Computer Use 运行时和 `cua_node` 依赖都保持不变。  
One-liner: a small cross-platform app-layer refresh, with confirmed bundled plugin version bumps visible on macOS only.

## 关于这份报告

本报告基于官方签名分发包的逆向 diff：macOS 使用 Sparkle 分发的 arm64 全量包对比，Windows 使用 x64 MSIX 全量包对比。macOS 对比范围为 `26.611.61049` build `3996` → `26.611.61753` build `4008`；Windows 对比范围为 `26.611.7849.0` → `26.611.8273.0`。发布日期按本批次输入记录为 `2026-06-15`。

证据分两档：

- 【信号】：仅来自前端模块名、构建产物体积或文件变化方向，能提示变化范围，但不能还原源码意图。

- `[macOS+Windows]`：两份事实包均出现相同方向的事实。

## ✨ 重点变化

### 【实证】[macOS+Windows] 版本批次推进

macOS 从 `26.611.61049` build `3996` 更新到 `26.611.61753` build `4008`，`Info.plist` 中 `CFBundleShortVersionString` 与 `CFBundleVersion` 均发生对应变化。Windows 从 `26.611.7849.0` 更新到 `26.611.8273.0`。

平台差异在于版本命名口径不同：macOS 使用 short version + build，Windows 使用 MSIX 风格的四段版本号。本文索引用 macOS short version `26.611.61753`。

### 【实证】[macOS+Windows] 应用层有少量内容更新，但文件总量不变

两平台的外层文件树数量均未变化：macOS 为 `8267 → 8267`，Windows 为 `11943 → 11943`。外层均无新增或移除文件，只有内容改动：macOS 内容改动 `62` 个文件，Windows 内容改动 `16` 个文件。

应用层 `app.asar` 也保持内部文件总量不变，但有少量增删改：macOS 内部文件 `3074 → 3074`，新增 `7`、移除 `7`、改动 `11`；Windows 内部文件 `3024 → 3024`，新增 `7`、移除 `7`、改动 `10`。

这说明两平台都发生了应用层刷新，但从事实包看不到大规模文件结构调整。

### 【实证】[macOS+Windows] Computer Use 运行时未见体积或依赖变化

Computer Use 运行时中的 `cua_node` 总量保持不变：macOS 为 `365.9 MB → 365.9 MB (+0 MB, +0%)`，Windows 为 `285.6 MB → 285.6 MB (+0 MB, +0%)`。两平台 `cua_node` 新增依赖包数量均为 `0`。

平台差异主要是基线体积不同：macOS 为 `365.9 MB`，Windows 为 `285.6 MB`。但本批次内，两者都没有新增运行时依赖。

### 【实证】[仅 macOS] bundled browser/chrome 插件元数据版本更新

macOS 包内两个 bundled 插件的 `.codex-plugin/plugin.json` 版本字段随主版本更新：

- `browser` 插件：`26.611.61049` → `26.611.61753`
- `chrome` 插件：`26.611.61049` → `26.611.61753`

`browser` 插件描述仍指向 Codex in-app browser，用于打开、检查、导航、测试、点击、输入或截图本地 Web 目标。`chrome` 插件描述仍指向依赖用户既有 Chrome 状态的自动化场景，例如标签页、登录会话、cookies、扩展与 Codex Chrome Extension 设置。

Windows 事实包未提供对应 plugin.json diff，因此这里不能断言 Windows 已同步这两个插件元数据版本更新，只能标为 `[仅 macOS]`。

### 【实证】[仅 macOS] app.asar 完整性哈希更新

macOS `Info.plist` 中 `ElectronAsarIntegrity/Resources/app.asar/hash` 从 `bdc881f00dd574cabfab0e56ca06975555f85ca31ebf8a9415f8a2e3d491d607` 变为 `b1c35ee9911fd72ce5229f4f106275361e39b104bf4a659a0d914b8b5beef33b`。

这与 macOS `app.asar` 内容发生变化一致。Windows 事实包未列出等价的完整性哈希字段，因此不做跨平台外推。
