# Codex 桌面版 26.611.61753 macOS / Windows 对照

本批次没有显示新增文件或新增依赖，变化集中在内置插件版本标记、平台元数据和少量 app.asar 内容差异。

This batch mainly aligns bundled plugin metadata across platforms, with Windows-specific Computer Use metadata and macOS-specific bundle metadata changes.

## 关于这份报告

这份报告基于两份官方签名分发包的机器提取 diff：macOS arm64 比较 `26.611.61049 build 3996` → `26.611.61753 build 4008`，Windows x64 比较 `26.611.7849.0` → `26.611.8273.0`。两平台版本号与发布时间窗口不完全相同，因此下文只按各自事实包归纳，不把一个平台的结论套到另一个平台。

证据等级分两档：【实证】来自 plist、manifest、依赖清单、插件清单、服务入口、签名信息、可读配置或事实包中明确给出的文件级 diff；【信号】来自模块名、体积变化或压缩 bundle 方向线索。本批次事实包没有提供同名前端模块改动线索、新增可读字符串、CSP diff、AppxManifest diff 或新增内置插件明细，因此这些方向只在边界处说明，不扩展推断。

平台标记使用 `[macOS+Windows]`、`[仅 macOS]`、`[仅 Windows]`。webview/assets 与 .vite/build 的 hash 改名按输入说明视为构建噪音，不作为功能增减依据。

## ✨ 重点变化

### 【实证】[macOS+Windows] Browser 插件版本随本批次对齐到 26.611.61753

两平台的内置 `browser` 插件 manifest 都只显示版本字段变化：`26.611.61049` → `26.611.61753`。

插件描述本身没有在事实包中显示内容变化，仍指向 Codex 内置浏览器、本地目标打开、点击、输入、截图，以及 localhost、127.0.0.1、file:// 等本地 Web 目标处理规则。因此这条变化能确认的是“插件包元数据版本进入新批次”，不能据此断言浏览器自动化能力本身新增了功能。

平台差异：macOS 路径为 `Contents/Resources/plugins/openai-bundled/plugins/browser/.codex-plugin/plugin.json`；Windows 路径为 `resources/plugins/openai-bundled/plugins/browser/.codex-plugin/plugin.json`。两边字段变化一致。

### 【实证】[macOS+Windows] Chrome 插件版本随本批次对齐到 26.611.61753

两平台的内置 `chrome` 插件 manifest 都显示版本字段变化：`26.611.61049` → `26.611.61753`。

事实包中的描述仍是依赖用户现有 Chrome 状态的自动化场景，包括标签页、登录态、cookie、扩展以及 Codex Chrome Extension 设置；当专用连接器、API 或 CLI 因认证问题失败时，需要请求用户重新认证或批准 Chrome 作为 fallback。这里能确认的是插件 manifest 版本更新，不确认 Chrome 能力面新增。

平台差异：两平台变化一致，只是 bundle 路径前缀不同。

### 【实证】[仅 Windows] Computer Use 插件版本标记更新

Windows 包内 `resources/plugins/openai-bundled/plugins/computer-use/.codex-plugin/plugin.json` 显示 `computer-use` 插件版本从 `26.611.61049` 更新到 `26.611.61753`。

插件描述为 “Control desktop apps on Windows from Codex through Computer Use.”。这说明 Windows 包中的 Computer Use 插件 manifest 已进入与 macOS short version 相同的插件版本标记。事实包没有给出对应服务入口、技能说明或默认提示 diff，因此不能进一步断言 Computer Use 行为变化。

平台差异：macOS 本批次事实包没有出现 `computer-use` 插件 manifest diff；Windows 独有这条可读配置变化。

### 【实证】[仅 Windows] owl shell 安装时间戳更新，但 runtimeInfo 未显示内容字段变化

Windows 根部 `.installed-owl-shell.json` 中 `installedAt` 从 `2026-06-16T20:33:01.480Z` 更新为 `2026-06-17T02:33:52.524Z`。
