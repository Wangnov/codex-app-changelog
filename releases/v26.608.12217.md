# Codex 桌面版 26.608.12217 macOS / Windows 对照

> 第三方跨平台变更日志 · 非官方

这一批的主线不是单点功能，而是桌面端运行时、语音/远程控制、插件与项目工作流的一次同步铺底。

Cross-platform desktop groundwork for voice, remote control, plugins, Sites, and runtime packaging, with Windows-specific Computer Use wiring.

## 关于这份报告

本文只使用输入事实包中的机器提取事实：macOS arm64 从 26.602.71036 build 3685 到 26.608.12217 build 3722；Windows x64 MSIX 从 26.602.9276.0 到 26.608.1337.0。两端来自同一套代码的不同打包，版本号与包版本分别引用，不互相套用。

证据等级分两档：【实证】来自 plist、manifest、CSP、插件 manifest、技能文档、package.json、依赖清单、服务入口、签名/包元数据，以及非“同名前端模块改动线索”的新增可读字符串；【信号】来自同名前端模块的新增/移除字符串、样式类、模块名和体积方向。平台归属用 `[macOS+Windows]`、`[仅 macOS]`、`[仅 Windows]` 标出。

已盘点的输入区块包括版本与渠道、Bundle 文件树、内置插件、Computer Use 运行时、cua_node 依赖、app.asar、前端模块真实增删、同名前端模块线索、主进程/前端新增字符串、Info.plist/CSP、资源与关键文件 diff。Windows 事实包未提供 AppxManifest 变化区块，因此不扩展推断。

## ✨ 重点变化

### 【实证】[macOS+Windows] 内置 Node 运行时改名为 `cua_node`，并集中承载 Computer Use 相关依赖

两端都把原来的顶层 Node / node_repl 文件移除，并新增 `cua_node` 目录：macOS 新增 `Contents/Resources/cua_node/bin/node` 119.1 MB、`node_repl` 15.2 MB，同时移除 `Contents/Resources/node` 119.1 MB 和 `node_repl` 15.2 MB；Windows 新增 `resources/cua_node/bin/node.exe` 91.4 MB、`node_repl.exe` 13.5 MB，同时移除 `resources/node.exe` 91.4 MB 和 `node_repl.exe` 13.6 MB。

依赖层也同步变化：两端都新增 `@oai/sky@0.4.8-202606020558-pr-981870-f570e0604110`、`corepack@0.34.6`、`npm@11.9.0`。macOS 额外新增 `node-addon-api@8.8.0`、`node-gyp-build@4.8.4`、`objc-js@1.5.0`，并出现 `objc-js` 的 darwin arm64/x64 native 预构建文件。

### 【实证】[仅 Windows] Computer Use 插件改为使用 bundled `cua_node` runtime，并增加工具面元数据

Windows 的 `computer-use` 插件 manifest 版本从 `26.602.71036` 升到 `26.608.12217`。技能文档明确把“plugin vendored runtime”改为“bundled `cua_node` `@oai/sky` runtime”，并取消在 `Module not found: @oai/sky` 时添加 `<plugin root>/node_modules` 的恢复路径。

`computer-use-client.mjs` 的入口从相对路径 `../node_modules/@oai/sky/...` 改为包导入 `@oai/sky/...`，并新增 `codex/toolSurface` 响应元数据。它会把调用目标识别为 appId 或 displayName，且只在具有 `nodeRepl.config` 与 `nativePipe` 的可信环境里使用 native pipe 和 approval UI。包文件树也显示大量 `resources/plugins/openai-bundled/plugins/computer-use/node_modules/@oai/sky/...` 文件被移除，运行时转移到 `resources/cua_node/bin/node_modules/@oai/sky/...`。

macOS 事实包没有对应的 `computer-use` 插件文件 diff，因此这条只归属 Windows。

### 【实证】[macOS+Windows] Browser 与 Chrome 插件升级，DOM snapshot 开始声明可展开 iframe body

两端 `browser` 与 `chrome` 插件 manifest 都从 `26.602.71036` 升到 `26.608.12217`。`browser/docs/api.md`、`browser/.codex-plugin/api-multi-tab.md`、`chrome/docs/api.md` 都把 `domSnapshot()` 说明从“返回当前 DOM 字符串”改成“在可用时包含展开后的 iframe body 内容”。

Browser / Chrome 技能说明也同步收紧：首次读取 `browser.documentation()` 必须用 `nodeRepl.write(await browser.documentation());` 一次性直接输出，不应提前赋值、切片、查长度、分页或摘要；只有工具明确报告截断时，才允许分块读取。

### 【实证】[macOS+Windows] Sites 插件从 0.1.12 升到 0.1.14，部署 API 名称与访问控制语义重写

两端 `sites` 插件 manifest 都从 `0.1.12` 升到 `0.1.14`。`sites-hosting/SKILL.md` 明确把环境变量接口改名为 `get-environment-variables` / `update-environment-variables`，把创建、保存版本、部署与查询部署状态分别改为 `create-site`、`save-site-version`、`deploy-site-version`、`get-deployment-status`。

同一技能文档还新增说明：`project_id` 是 Site 的持久标识；现有项目推送前应使用 `create-source-repository-write-credential`；访问控制只在用户要求改变访问范围时使用，并把 `workspace_all` 面向用户称为 “Anyone in this workspace can visit”，把 `custom` 称为 “Private”。

### 【实证】[macOS+Windows] CSP 新增 WebSocket 放行，指向实时/流式连接能力

两端 `webview/index.html` 的 `connect-src` 都新增 `wss://chatgpt.com`、`wss://ws.chatgpt-staging.com`、`wss://ws.chatgpt.com`。同一文件还新增 `data-build="<!-- PROD_BUILD_TAG_HERE -->"`，并引入 `modulepreload-polyfill` 与 `isEqual` preload。

这只能证明前端包允许这些 WebSocket 目标与构建标记进入页面，不能单独证明所有用户界面已经开放。

### 【实证】[macOS+Windows] 实时语音与全局听写进入桌面包

主进程/前端新增字符串同时出现在两端：`Failed to prepare dictation stream connection.`、`Sign in to ChatGPT in Codex Desktop to use dictation.`。Composer 线索新增 `Microphone access is required to start realtime voice`、`Realtime voice connection failed`、`Realtime WebRTC offer did not include SDP`，侧栏线程还新增“active realtime voice session”的无障碍说明。

`global-dictation-page` 同名模块在两端都新增 `Dictation ready`、`Global dictation ready`、`hold-to-dictate`、toggle/hold 快捷键 tooltip 等字符串与样式类。由于这些来自同名前端模块线索，界面形态属于【信号】；但 dictation 连接错误与登录要求字符串属于【实证】。

### 【实证】[仅 macOS] Camera 权限说明从通用访问改成 Codex 视频输入

macOS `Info.plist` 中 `NSCameraUsageDescription` 从 `This app needs access to the camera` 改为 `Codex needs access to your camera for video input.`。Windows 事实包没有对应权限 manifest 变化区块。

### 【实证】[macOS+Windows] 远程控制、SSH 连接与 Browser session 路由错误路径进入主包

两端新增主进程/前端字符串包括 `Remote control connections are unavailable`、`SSH websocket closed before opening`、`SSH websocket open timed out`、`SSH: local ssh config probe timed out`，以及 Browser Use 路由相关的 `Ignored Browser Use session route rebind with existing page`、`Reassigned duplicate Browser Use session route capture`、`All over-budget Browser tabs are protected`。

前端 `app-main` 同步出现“Change connection color”、远程连接颜色选择器关闭/重置按钮、项目 hover card 操作等信号。这里可以确认错误路径和状态文案进入包内；具体 UI 是否对所有用户开放仍应按【信号】看待。

### 【实证】[macOS+Windows] Avatar overlay / native pet 组成面进入包，macOS 还有 native addon 文件

两端前端真实新增模块包括 `avatar-overlay-featuregate-page`、`avatar-overlay-native-frame`、`avatar-overlay-native-page`、`avatar-overlay-notifications`、`avatar-overlay-pill-dismiss-button`、`avatarOverlayCompositionSurface`、`custom-avatars-query`。主进程/前端新增字符串还包含 `Avatar overlay native addon not found`、`Native pet composition lifecycle`、`Native pet composition surface ready received`、`Native pet material attachment completed` 等一组 native composition / material 状态与错误。

macOS 文件树额外列出新增 `Contents/Resources/native/avatar-overlay.node` 0.3 MB。Windows 事实包没有列出对应 native addon 新增文件，因此 native 文件只标 macOS；跨平台可以确认相关字符串与前端模块进入包内。

### 【信号】[macOS+Windows] 插件/技能浏览页从“安装”转向“添加/连接/创建”，并加强搜索和空状态

两端 `plugins-page` 同步新增 `Browse plugins or skills`、`Add plugin marketplace`、`Button label for connecting a plugin`、`Button label for adding a recommended skill`、`Clear search`、`Create options`，并移除 `All plugins`、`Choose a category`、`Choose a plugin marketplace`、`Button label to install a recommended skill` 等旧文案。

`skills-page` 在 macOS 线索中新增创建技能、刷新技能列表、安装后需要刷新、已安装/推荐技能标题和空状态；Windows 事实包没有列出同名 `skills-page` 改动线索。跨平台插件浏览方向一致，技能页细节只按 macOS 输入样本呈现。

### 【信号】[仅 Windows] 插件详情页出现分享、MCP server enablement、技能卡启用与可用性状态

Windows `plugin-detail-page` 同名模块体积增加 116597 bytes，新增 `Anyone in this workspace with the link`、plugin share link copy、workspace share target removal、`MCP server enablement toggle`、`App unavailable`、skill card actions menu、disabled skill enablement 等字符串与状态类。macOS 事实包没有对应同名模块改动线索，因此只标 Windows 信号。

### 【信号】[macOS+Windows] 项目、归档聊天与本地任务状态的 UI 结构调整

两端 `data-controls` 同步新增 archived chats 的过滤、分组、项目归档动作菜单、`Delete all in project?`、cloud/local archived chats filter 等字符串。`profile-dropdown` 新增 `Archive chat`、`Archive task`、`Automation chat`、`Awaiting approval`、`Awaiting response`、本地任务等待用户批准 chip，以及分支来源说明。

`app-main` 还出现 `Add new project`、`Collapse project`、项目 hover card 里打开 source、编辑 modal、pin project、inline project name editor 等信号。整体看，项目与任务列表正在从旧的 recent/local task 模块组合迁到新的项目 hover、归档筛选和任务状态表达。

### 【信号】[macOS+Windows] Composer 调整粘贴文本、Chrome 选中文本附件和模型 reasoning effort

两端 `composer` 新增“attached large text paste”移除 aria、active Chrome tab 选中文本附件 pill、`Pasted text is unavailable for this target`、`Codex capabilities available this session:`、`Error forking local task`、`Reasoning effort label for a given model: maximum`。同时移除 best-of-N / attempts selector 相关文案。

这表明输入附件与目标能力提示在调整，但具体交互入口仍只能作为前端信号。

### 【信号】[macOS+Windows] PR / review 面板增加评论跳转与 CI checks 状态

两端 `use-diff-annotations` 新增 `Open comment in review`、`Open comment on GitHub`、`View comment on GitHub`、comment author avatar alt、unknown author 等。`review-header-toolbar` 新增 `Checks failing`、`Checks pending`、`Checks successful`、`No CI checks`、单个失败 check 的 fix action label，以及 passing/failing/pending/no-checks 状态行。

主进程字符串还新增 `Failed to determine pull request summary`、`Head branch is required to check pull request summary`、`No git repository found for pull request summary`。评论与 CI checks UI 属【信号】，PR summary 错误路径属于【实证】。

### 【信号】[macOS+Windows] Google Drive 与 Slack 资源展示进入对话线程

两端 `local-conversation-thread` 新增 `Google Docs`、`Google Sheets`、`Google Slides`、`Open in browser`、打开 Google Drive conversation resource 的 hover subtitle，以及 Slack search target nouns for channels/users。它同时移除了多条 Computer Use 工具 active label 文案，这与 Windows 将 Computer Use 工具面元数据移入 client 的方向相互呼应，但跨平台前端层仍按信号处理。

### 【信号】[macOS+Windows] 文档/表格 worker 体积增加，并出现 Aptos 与表格样式字符串

两端 `runtime.worker` 从 3.0 MB 到 3.5 MB，增加 481283 bytes，新增 `Aptos Display`、`Document worker has not been bootstrapped.`、`Document worker received a message before bootstrap`、`Grid Table 1 Light Accent 1` 到 `Accent 6`，并新增 `text-body`、`text-display`、`text-section`、`text-subtitle`。这说明文档 worker 的字体/样式和启动状态处理有变化，但不能还原完整源码语义。

### 【实证】[macOS+Windows] 构建脚本与 owl runtime 更新

两端 `package.json` 版本从 `26.602.71036` 到 `26.608.12217`，`codexBuildNumber` 从 `3685` 到 `3722`。`compile`、`tsc`、`lint`、`lint:fix` 前都加入 `node ./scripts/ensure-owl-electron-types.mjs`，`build:ci` 改为 `node ./scripts/build-ci.mjs`。

`owl-electron-app.json` 的 runtime archive SHA 两端均更新：macOS 从 `c6a2...06fe` 到 `df06...a4f`，Windows 从 `76a8...c86f` 到 `6a9d...6aa4`。Windows 还更新 `.installed-owl-shell.json` 的 cacheKey、installedAt 和 archiveSha。

### 【实证】[仅 macOS] DMG detach 清理加入重试

macOS `.vite/build/bootstrap.js` 显示 source DMG detach 清理由一次尝试变成最多 25 次、每次间隔 250 ms，并新增 `Failed to detach app source DMG after retries`。Windows 包里也有相同 bootstrap 代码片段，但该逻辑条件仍绑定 `platform !== darwin` 直接返回；因此作为实际安装流程变化只归属 macOS。

### 【信号】[macOS+Windows] 设置项新增字体、主题、默认打开位置、终端位置与权限相关文案

两端 `src` 新增 `Base UI font size`、`Code font size`、`Chrome theme used in dark mode`、`Chrome theme used in light mode`。`gpu-tearing-debug-settings` 新增 `Agent environment`、`Default open destination`、`Default permissions`、`Default terminal location`、`Bottom panel`、`Code review`、`Default to projectless chat`、`Dictation dictionary`，并移除外部 agent config import / migrate-to-codex 相关文案。

### 【实证】[macOS+Windows] about dialog 多语言文案从“构建信息”扩展到版本/构建信息

两端 `native-menu-locales/zh-CN.json` 把 `codex.aboutDialog.buildInfoLabel` 从“构建信息”改为“版本和构建信息”。多个语言文件也同步调整 about dialog 的 build/version 表达。菜单本地化属于实证，但不代表对应菜单布局发生了可见结构变化。
