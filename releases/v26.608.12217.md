# Codex 桌面版 26.608.12217(macOS / Windows 对照)

> 第三方跨平台变更日志 · 非官方

这一批的核心是把 Computer Use 相关 Node 运行时迁入 `cua_node`，并同步更新浏览器自动化与 Sites 工作流文档。  
One-liner: this batch reorganizes the packaged computer-use runtime and tightens browser/Sites automation contracts, with macOS carrying a few extra app-surface signals.

## 关于这份报告

本报告基于两份官方签名分发产物的逆向重建 diff：macOS arm64 对比 `26.602.71036` build `3685` → `26.608.12217` build `3722`，Windows x64 MSIX 对比 `26.602.9276.0` → `26.608.1337.0`。发布日期按事实包给定为 `2026-06-09`。

证据分两档：【实证】表示来自 plist、package.json、插件 manifest、技能文档、API 文档、CSP、依赖清单或文件树等可读证据；【信号】表示仅来自前端模块名或体积变化，只能说明方向，不能还原源码意图。平台标记含义为 `[macOS+Windows]` 两端都有，`[仅 macOS]` 或 `[仅 Windows]` 表示当前事实包只在该端出现。

## ✨ 重点变化

### 【实证】[macOS+Windows] Computer Use / Node 运行时迁入 `cua_node`

两端都新增了 `cua_node` 目录，并把原本位于资源根部的 Node 可执行文件迁入其中：macOS 新增 `Contents/Resources/cua_node/bin/node` 与 `node_repl`，同时移除 `Contents/Resources/node` 与 `node_repl`；Windows 新增 `resources/cua_node/bin/node.exe` 与 `node_repl.exe`，同时移除 `resources/node.exe` 与 `node_repl.exe`。

平台差异在于依赖数量：macOS 的 `cua_node` 新增 6 个包，Windows 新增 3 个包。两端共有 `@oai/sky@0.4.8-202606020558-pr-981870-f570e0604110`、`corepack@0.34.6`、`npm@11.9.0`；macOS 额外新增 `node-addon-api@8.8.0`、`node-gyp-build@4.8.4`、`objc-js@1.5.0`，这说明本批次 macOS 包比 Windows 多带了与 Node-API / 原生预构建加载 / Objective-C 桥接相关的依赖。

### 【实证】[macOS+Windows] Computer Use 的 `@oai/sky` 分发位置被重排

Windows 事实包显示，旧版 `resources/plugins/openai-bundled/plugins/computer-use/node_modules/@oai/sky/...` 下大量文件被移除，同时新版在 `resources/cua_node/bin/node_modules/@oai/sky/...` 下出现对应运行时与多平台二进制。macOS 新版也在 `Contents/Resources/cua_node/lib/node_modules/@oai/sky/...` 下新增 Windows、Linux 与 macOS 相关文件。

这更像是打包布局调整，而不是单纯新增一个用户可见功能：两端都把 Computer Use 相关依赖集中进 `cua_node`，Windows 的旧插件内 `node_modules/@oai/sky` 被明显清理，macOS 则没有列出同等旧路径移除清单。

### 【实证】[macOS+Windows] 浏览器与 Chrome 自动化文档更严格

两端的 in-app browser 与 Chrome 技能文档都要求首次读取 `browser.documentation()` 时使用直接调用 `nodeRepl.write(await browser.documentation());`，并明确不要先赋值、检查长度、切片、分页或只输出摘录。只有工具输出明确提示被截断时，才允许分块读取。

同一批次还更新了 browser / Chrome API 文档：`domSnapshot()` 的说明从“返回当前 DOM 字符串”扩展为包含“可用时展开 iframe body 内容”。这属于自动化接口契约变化，对依赖 DOM 快照的浏览器任务更重要。

### 【实证】[macOS+Windows] Sites 技能文档改用新的 connector 操作名

Sites 技能文档把多处 connector 操作名从旧的 project 命名改为 site 命名，例如 `create-project` → `create-site`，`create-project-version` → `save-site-version`，`deploy-project-version` → `deploy-site-version`，`deployment-status` → `get-deployment-status`。环境变量操作也从 `get-environment` / `update-environment` 改为 `get-environment-variables` / `update-environment-variables`。

同一文档还补充 `project_id` 是 Site 的持久标识，并调整访问控制表述：只在用户要求变更访问权限时使用 access controls，且将 `workspace_all` 面向用户称为“Anyone in this workspace can visit”，`custom` 称为“Private”。

### 【实证】[仅 macOS] macOS 增加摄像头权限说明与 WebSocket CSP 放行

macOS 的 `Info.plist` 将 `NSCameraUsageDescription` 从通用描述改为“Codex needs access to your camera for video input.”，这是明确的权限提示文本变化。

macOS 的 `webview/index.html` 还在 CSP（Content Security Policy，内容安全策略）的 `connect-src` 中新增 `wss://chatgpt.com`、`wss://ws.chatgpt-staging.com`、`wss://ws.chatgpt.com`。Windows 事实包没有给出对应 CSP diff，因此本文只能标为 macOS 已见，Windows 未在本包中实证。

### 【信号】[仅 macOS] avatar overlay 与若干前端入口出现模块名信号

macOS 去除 Vite hash 噪音后，新增模块中出现 `avatar-overlay-featuregate-page`、`avatar-overlay-native-frame`、`avatar-overlay-native-page`、`avatar-overlay-notifications`、`avatar-overlay-pill-dismiss-button`、`avatarOverlayCompositionSurface`，同时新增 `custom-avatars-query`、`experimental-feature-visibility` 等模块名。

这些只说明 macOS 包内前端构建出现 avatar overlay / custom avatars / feature visibility 方向的模块名，不能证明功能已开放。Windows 事实包只给出 app.asar 文件统计，没有列出同等模块名清单，因此不能判断 Windows 是否也包含这些前端信号。

### 【实证】[仅 macOS] 构建脚本引入 owl 类型准备与 CI 构建脚本

macOS 的 `package.json` 显示 `compile`、`tsc`、`lint`、`lint:fix` 前增加 `node ./scripts/ensure-owl-electron-types.mjs`；`build:ci` 从内联清理并运行 `forge:package` 改为 `node ./scripts/build-ci.mjs`。`owl-electron-app.json` 的 `runtimeArchiveSha` 也发生变化。

这些是打包/开发流程层面的实证变化。Windows 事实包未提供对应 `package.json` diff，故只标 macOS。

### 【实证】[仅 macOS] 多语言原生菜单文案小幅修订

macOS 多个 `native-menu-locales/*.json` 文件更新了 About 对话框中的“版本 / 构建信息 / 发布日期”等译文，包括 `zh-CN`、`hi-IN`、`el-GR`、`fr-CA`、`ru-RU`、`de-DE` 等。它们集中在文案润色，不体现新的交互入口。

Windows 事实包没有列出对应 locale diff。

### 【信号】[仅 macOS] 前端模块结构继续调整

macOS 新增 `thread-browser-panel-tabs`、`thread-user-message-navigation-rail`、`thread-scroll-controller-context-value`、`sidebar-thread-row-signals`、`home-artifact-templates`、`rich-link`、`inline-mention-content`、`web-search-favicon-icon` 等模块名，同时移除 `pull-request-status`、`pull-request-readonly-comment`、`workspace-dependencies-feature`、`automation-queries`、多类 settings search 模块等。

这些模块名能提示线程导航、浏览器面板、artifact 模板、富链接、提及内容等方向的前端重组，但不能证明用户可见功能变化。Windows 缺少模块名列表，不能做对等判断。
