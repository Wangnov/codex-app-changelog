# Codex 桌面版 26.608.12217(macOS / Windows 对照)

> 第三方跨平台变更日志 · 非官方

这一批次的核心变化是运行时与插件层重组：两端都引入 `cua_node`，并更新浏览器自动化与 Sites 托管相关说明；macOS 与 Windows 的打包细节仍有明显不同步。  
One-liner: Codex desktop moves more computer-use runtime into `cua_node`, refreshes automation/Sites plugin docs, and shows platform-specific packaging drift.

## 关于这份报告

本文基于两份官方签名分发包的逆向 diff：macOS 使用 Sparkle 全量包对比，Windows 使用 x64 MSIX 包对比。macOS 对比范围是 `26.602.71036` build `3685` 到 `26.608.12217` build `3722`；Windows 对比范围是 `26.602.9276.0` 到 `26.608.1337.0`。批次发布日期按输入事实包为 `2026-06-09`。

证据等级分两档：【实证】表示来自 plist、package.json、依赖清单、CSP、技能文档、插件 manifest 等可读证据；【信号】表示只看到模块名、文件移动或体积变化，能指向方向，但不能还原完整源码意图。

平台归属标记含义：`[macOS+Windows]` 表示两端事实包都出现相同方向的变化；`[仅 macOS]` 或 `[仅 Windows]` 表示目前只在该平台事实包中看到。本文忽略 `webview/assets`、`.vite/build` 等 Vite 构建 hash 重命名噪音。

## ✨ 重点变化

### 运行时 Node 迁入 `cua_node`

【实证】[macOS+Windows]  
两端都新增了 `cua_node` 目录，并把原来的顶层 Node/REPL 二进制移走：macOS 移除 `Contents/Resources/node` 与 `node_repl`，新增 `Contents/Resources/cua_node/bin/node` 与 `node_repl`；Windows 移除 `resources/node.exe` 与 `node_repl.exe`，新增 `resources/cua_node/bin/node.exe` 与 `node_repl.exe`。

平台差异在依赖数量上很清楚：macOS 的 `cua_node` 新增 6 个包，Windows 新增 3 个包。两端共有 `@oai/sky@0.4.8-202606020558-pr-981870-f570e0604110`、`corepack@0.34.6`、`npm@11.9.0`；macOS 额外有 `node-addon-api@8.8.0`、`node-gyp-build@4.8.4`、`objc-js@1.5.0`。

### computer-use 依赖打包位置调整

【实证】[仅 Windows]  
Windows 包明确显示，`resources/plugins/openai-bundled/plugins/computer-use/node_modules/@oai/sky/...` 下大量文件被移除，同时 `@oai/sky` 进入 `resources/cua_node/bin/node_modules/`，其中包括 `codex-computer-use.exe`。这说明 Windows 的 computer-use 相关依赖从插件内 node_modules 迁到了 `cua_node` 运行时树。

macOS 也新增了 `Contents/Resources/cua_node/lib/node_modules/@oai/sky/...`，但事实包没有对应列出旧插件目录中的 computer-use node_modules 大规模移除，因此这里不把“从插件目录迁出”扩展到 macOS。

### 浏览器与 Chrome 自动化文档收紧

【实证】[macOS+Windows]  
macOS 的可读 diff 显示，内置浏览器与 Chrome 控制技能要求首次读取 `browser.documentation()` 时直接执行 `nodeRepl.write(await browser.documentation());`，并明确不要先赋值、量长度、截断、摘要或主动分页；只有工具输出明确截断时才分块读取。Windows 事实包也列出同名浏览器/Chrome 技能文档文件发生变化。

这属于代理工具使用约束的加强，目标不是新增 UI，而是让自动化技能更稳定地读取完整 API 文档。

### `domSnapshot()` 覆盖 iframe 内容

【实证】[macOS+Windows]  
macOS 的 `browser/docs/api.md`、`chrome/docs/api.md` 与 multi-tab API 文档显示，`PlaywrightAPI.domSnapshot()` 的说明从“返回当前 DOM 快照”扩展为在可用时包含展开后的 iframe body 内容。Windows 事实包同样列出对应 API 文档发生变化。

这意味着浏览器自动化看到的页面结构可能更完整，尤其是嵌入 iframe 的页面；但事实包只证明 API 文档声明变化，不证明具体实现细节。

### Sites 插件 API 名称与访问控制说明更新

【实证】[仅 macOS]  
macOS 的 Sites 技能文档把多处 connector 操作名更新为更具体的 Sites 术语：例如 `get-environment` → `get-environment-variables`，`update-environment` → `update-environment-variables`，`create-project` → `create-site`，`create-project-version` → `save-site-version`，`deploy-project-version` → `deploy-site-version`，`deployment-status` → `get-deployment-status`。同时文档明确 `project_id` 是 Site 的持久标识符。

访问控制说明也改了：现在强调只有用户要求改变访问范围时才使用 access controls，并把 `workspace_all` 面向用户称为 “Anyone in this workspace can visit”，把 `custom` 称为 “Private”。macOS 的 Sites 插件版本从 `0.1.12` 升到 `0.1.14`。Windows 事实包没有列出 Sites 文档或插件版本变化，因此不能确认 Windows 已同步。

### WebSocket 连接白名单扩展

【实证】[仅 macOS]  
macOS 的 `webview/index.html` 内容安全策略(CSP，浏览器限制页面可连接来源的策略)在 `connect-src` 中新增 `wss://chatgpt.com`、`wss://ws.chatgpt-staging.com`、`wss://ws.chatgpt.com`。

Windows 事实包没有提供 CSP 差异，因此无法确认 Windows 是否同批加入这些 WebSocket 来源。

### macOS 相机权限文案变具体

【实证】[仅 macOS]  
macOS `Info.plist` 的 `/NSCameraUsageDescription` 从通用的 “This app needs access to the camera” 改为 “Codex needs access to your camera for video input.”。这只在 macOS 事实包中出现，属于系统权限提示文案变化。

### 头像叠层与个性化相关前端信号

【信号】[仅 macOS]  
macOS 的非 hash 前端模块新增了 `avatar-overlay-featuregate-page`、`avatar-overlay-native-frame`、`avatar-overlay-native-page`、`avatar-overlay-notifications`、`avatar-overlay-pill-dismiss-button`、`avatarOverlayCompositionSurface`、`custom-avatars-query` 等模块，并新增 `Contents/Resources/native/avatar-overlay.node`。

这些名称强烈指向头像叠层或自定义头像相关能力，但事实包没有源码逻辑或产品入口，不能判断是否对用户开放。Windows 事实包未提供对应模块名列表，也未列出 `avatar-overlay.node`，因此按当前证据记为仅 macOS 信号。

### 线程、提及与链接渲染相关前端重排

【信号】[仅 macOS]  
macOS 新增模块包括 `inline-mention-content`、`mention-icons`、`rich-link`、`external-link-favicon`、`web-search-favicon-icon`、`thread-browser-panel-tabs`、`thread-scroll-controller-context-value`、`thread-user-message-navigation-rail`、`sidebar-thread-row-signals`、`threads-create` 等；同时移除 `home-project-recent-chats`、`local-task-row`、`pull-request-status`、`pull-request-visual-state`、`sidebar-task-pr-chip-signals` 等模块。

这些变化指向线程列表、提及、链接预览或 PR 状态展示的前端结构调整，但只能视为模块层信号。Windows 事实包只给出 app.asar 文件数变化，没有模块名清单，不能做同等判断。

### 构建脚本与运行时归档更新

【实证】[仅 macOS]  
macOS 的 `package.json` 版本从 `26.602.71036` 到 `26.608.12217`，`codexBuildNumber` 从 `3685` 到 `3722`。构建脚本中 `compile`、`tsc`、`lint`、`lint:fix` 增加 `node ./scripts/ensure-owl-electron-types.mjs` 前置步骤，`build:ci` 改为 `node ./scripts/build-ci.mjs`。`owl-electron-app.json` 的 `runtimeArchiveSha` 也发生变化。

Windows 事实包没有 package.json 具体 diff，因此不确认同样脚本变化。

### 多语言原生菜单文案微调

【实证】[仅 macOS]  
macOS 多个 `native-menu-locales/*.json` 更新了关于版本、构建信息、发布日期等文案，例如 `zh-CN` 的 `codex.aboutDialog.buildInfoLabel` 从“构建信息”改为“版本和构建信息”。事实包列出 `hi-IN`、`el-GR`、`fr-CA`、`ru-RU`、`mn`、`zh-CN`、`bs-BA`、`it-IT`、`sw-TZ`、`is-IS`、`kk`、`de-DE`、`lt`、`nl-NL`、`nb-NO`、`sv-SE`、`vi-VN` 等文件变化。

Windows 事实包没有对应 locale diff，不能确认同步。
