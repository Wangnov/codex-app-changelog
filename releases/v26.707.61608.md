# Codex 桌面版 26.707.61608 macOS / Windows 对照

> 第三方跨平台变更日志 · 非官方

本批次的共同变化集中在旧版 ChatGPT 应用迁移、内置浏览器能力版本同步和构建链调整；macOS 前端同时经历了明显更大规模的界面与模块重组。

Both packages add legacy ChatGPT app migration paths and refresh shared browser integrations, while the macOS sample carries a substantially broader frontend reorganization.

## 关于这份报告

本报告比较两组官方分发产物：macOS arm64 从 26.707.51957（build 5175）到 26.707.61608（build 5200），Windows x64 MSIX 从 26.707.6957.0 到 26.707.8168.0。两个平台的外部分发版本并不对齐，因此所有结论均按各自事实包独立建立。

这是一次历史回填。macOS 使用两个官方全量包重建差异，两个版本均通过 Developer ID 公证验证；Windows 事实包提供了两个全量包的 SHA-256，但没有提供对应的签名验证输出。

证据分为两档：

- 【实证】来自 plist、插件 manifest、`package.json`、依赖清单、可读配置，以及主进程或普通 bundle 中新增的字面量。它能证明内容已进入包内，但新增文案不等于相关界面已经对所有用户开放。

- 【信号】来自同名前端模块的字符串或样式集合变化、模块名及压缩体积变化。它适合判断开发方向，不能当作完整源码或功能已开放的证明。

平台标记分别为 `[macOS+Windows]`、`[仅 macOS]` 和 `[仅 Windows]`。Vite hash 文件改名、`.vite/build` hash 变化及 `webview/assets` 中单纯的 hash 置换均已排除为噪音。

## ✨ 重点变化

### 旧版 ChatGPT 应用迁移与失败恢复路径

【实证】[macOS+Windows]

两平台的新版主进程或前端 bundle 都新增了以下字面量：

- `ChatGPT Classic.app`
- `Failed to relocate legacy ChatGPT app`
- `Failed to restore legacy ChatGPT app`
- `com.openai.chat`

这些字符串证明包内加入了识别旧 ChatGPT 应用、迁移或重定位，以及迁移失败后恢复的处理路径。它们没有提供触发条件、迁移目的地或用户可见流程，因此不能进一步断言迁移会在所有安装上自动执行。

### Browser 与 Chrome 内置插件同步升级

【实证】[macOS+Windows]

两平台的 `browser` 和 `chrome` 插件 manifest 版本均由 `26.707.51957` 更新为 `26.707.61608`。

`browser` manifest 继续明确支持通过内置浏览器打开、检查、导航、测试、点击、输入和截图本地 Web 目标，包括 `localhost`、`127.0.0.1`、`::1` 与 `file://`；它还要求显式的 `@browser` 或 `@browser-use` 请求优先使用内置浏览器，而不是系统打开命令或通用网页浏览。

`chrome` manifest 继续将能力定位为利用用户已有的 Chrome 标签页、登录会话、Cookie、扩展以及 ChatGPT Chrome Extension 设置完成自动化，并要求优先考虑专用连接器、API 或 CLI。

本次可读 diff 只显示插件版本同步，没有显示上述描述文本发生变化。

### Windows Computer Use 插件版本同步

【实证】[仅 Windows]

Windows 包中的 `computer-use` 插件 manifest 从 `26.707.51957` 更新到 `26.707.61608`，描述仍为通过 Computer Use 从 ChatGPT 控制 Windows 桌面应用。

该项只出现在 Windows 关键文件差异中。两平台的 Computer Use 运行时本身均未增长，也没有新增 `cua_node` 依赖。
