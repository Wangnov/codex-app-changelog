# Codex 桌面版 `26.429.61741`(build 2429)

> 第三方变更日志 · 非官方 · 基于官方更新包逆向差异

本版从 `26.429.30905` 推进到 `26.429.61741`，整体更像一次低噪声维护更新，而不是功能面大幅扩张。  
English one-liner: a small maintenance-style desktop release with minimal bundle-level movement.

## 关于这份报告

这不是官方变更日志，而是一份**基于官方分发产物的逆向差异报告**。本次属于历史回填：两个版本均已超出 Sparkle(应用自动更新框架) 增量窗口，因此采用两个官方全量包直接比较，而不是增量包重建路径。

对比方法为：下载上一版与最新版官方全量包 → 以 Apple(苹果公司) 公证和 OpenAI Developer ID(开发者身份签名) 作为信任根 → 通过 codesign(代码签名验证工具) / spctl(公证与安全策略验证工具) 验证 → 做结构化 bundle(应用包目录) 差异。

可信度方面，上一版与最新版 `Codex.app` 均通过公证验证，结果为 `accepted`，来源均为 `Notarized Developer ID`，签名来源均为 `Developer ID Application: OpenAI OpCo, LLC (2DC432GLL2)`。

证据等级分为两档：

- **【实证】**：来自 Info.plist(应用元信息文件)、签名验证结果、文件树统计、依赖清单等人类可读的直接证据。
- **【信号】**：来自前端模块名或二进制体积变化等间接证据，只能说明“指向某方向，但无法从产物还原到源码级意图”。

本报告对比范围：`26.429.30905` build `2345`，发布于 `2026-05-01`；到 `26.429.61741` build `2429`，发布于 `2026-05-05`。

## ✨ 重点变化

### 1. 版本与构建号推进到 `26.429.61741` / build `2429` 【实证】

最新版在 `Info.plist` 中将短版本号从 `26.429.30905` 更新为 `26.429.61741`，构建号从 `2345` 更新为 `2429`。

这意味着桌面应用确实进入了一个新的签名发布版本，但仅从本次事实包看，变化主要集中在少量应用内容与元信息更新。

关键证据：

```diff
CFBundleShortVersionString: 26.429.30905 → 26.429.61741
CFBundleVersion: 2345 → 2429
```

### 2. 应用包文件树保持稳定，未出现新增或移除文件 【实证】

Bundle 文件数保持 `508 → 508`，新增 `0`、移除 `0`、内容改动 `46`。

这说明本版没有在 macOS(苹果桌面操作系统) 应用包层面引入新的外置组件、插件或资源目录；变化更可能是既有文件内容替换，而不是结构扩张。

关键证据：

```text
文件数: 508 → 508
新增 0 / 移除 0 / 内容改动 46
```

### 3. 应用主体 `app.asar` 有少量内容变化，但体积几乎不变 【实证】

`Contents/Resources/app.asar` 从 `130.0 MB` 到 `130.0 MB`，记录的体积变化为 `+0.00 MB`。同时，应用层内部文件为 `1208 → 1208`，新增 `2`、移除 `2`、改动 `6`。

这指向一次非常小的应用层调整，但事实包没有提供新增 / 移除文件名，也没有提供可读源码级差异，因此不能进一步写成某个明确功能变化。尤其是 `webview/assets` 与 `.vite/build` 下带 hash(内容指纹) 的文件名变化属于构建产物重命名噪音，不能据此判断新增或删除功能。

关键证据：

```text
Contents/Resources/app.asar: 130.0 MB → 130.0 MB (+0.00 MB)
app.asar 内部文件: 1208 → 1208 (新增 2 / 移除 2 / 改动 6)
```

### 4. 未发现新增 `cua_node` 依赖 【实证】

本次 `cua_node` 依赖变化中，新增包数量为 `0`。

这意味着从事实包可见范围内，本版没有通过新增依赖来扩展底层能力；至少不能归纳出新增 PDF(便携式文档格式) 读取、OCR(光学字符识别)、图像处理或浏览器自动化等能力。

关键证据：

