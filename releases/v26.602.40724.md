# Codex 桌面版 `26.602.40724`(build 3593)

> 第三方变更日志 · 非官方 · 基于官方更新包逆向差异

这是一个非常克制的小版本:从产物看,主要变化集中在版本号、`app.asar`(应用资源包)完整性哈希以及内置 Browser(内置浏览器自动化插件)/Chrome(谷歌浏览器自动化插件)的版本同步。

## 关于这份报告

这不是官方变更日志,而是一份**基于官方分发产物的逆向差异报告**。本次为历史回填:两个版本均超出官方 Sparkle(应用自动更新框架)增量窗口,因此采用两个官方全量包直接比较,再做结构化 bundle(应用包)差异分析。

可信度方面,两个对比产物均来自官方下载地址。上一版与最新版都通过 Apple(苹果)公证验证,来源为 `Notarized Developer ID`,签名主体为 `Developer ID Application: OpenAI OpCo, LLC (2DC432GLL2)`。

本文使用两档证据等级:

- **【实证】**: 来自人类可读的直接证据,例如 `Info.plist`(macOS 应用元信息文件)、插件配置、哈希、签名与公证结果。
- **【信号】**: 仅来自间接证据,例如前端模块名或二进制体积变化;这类证据只能说明“指向某方向,但无法从产物还原到源码级意图”。

对比范围:从 `26.602.30954`(build 3575,2026-06-04)到 `26.602.40724`(build 3593,2026-06-05),平台为 macOS arm64(苹果芯片架构)。

## ✨ 重点变化

### 1. 应用版本推进到 `26.602.40724` 【实证】

`Info.plist` 显示,应用短版本号从 `26.602.30954` 更新到 `26.602.40724`,build 从 `3575` 更新到 `3593`。

这意味着这是一个正式打包的新桌面版构建,但从可读配置看,没有伴随新的权限、入口组件或文件树规模变化。

关键证据:

```diff
/CFBundleShortVersionString: '26.602.30954' → '26.602.40724'
/CFBundleVersion: '3575' → '3593'
```

### 2. 内置 Browser 与 Chrome 插件版本同步 【实证】

内置 `browser` 与 `chrome` 插件的 `plugin.json` 版本号都从 `26.602.30954` 更新到 `26.602.40724`。

可感知点是:这两个自动化插件随桌面应用一起重新发布,但事实包中没有显示它们的描述、能力声明或配置内容发生变化。

关键证据:

```diff
"version": "26.602.30954" → "26.602.40724"
```

涉及文件:

- `Contents/Resources/plugins/openai-bundled/plugins/browser/.codex-plugin/plugin.json`
- `Contents/Resources/plugins/openai-bundled/plugins/chrome/.codex-plugin/plugin.json`

### 3. 应用资源包内容发生小幅变化,但未暴露明确新功能 【信号】

`Contents/Resources/app.asar` 体积从 `143.1 MB` 到 `143.1 MB`,变化为 `+0.01 MB`。内部文件统计为 `2830 → 2830`,其中新增 `17`、移除 `17`、改动 `12`。

这指向应用层资源有重新构建或小幅内容调整,但无法从产物还原到源码级意图。事实包也明确提示,`webview/assets` 与 `.vite/build` 下带 hash(内容指纹)的文件名变化属于构建产物重命名噪音,不能据此判断新增或删除功能。

关键证据:

```text
Contents/Resources/app.asar: 143.1 MB → 143.1 MB (+0.01 MB)
内部文件: 2830 → 2830 (新增 17 / 移除 17 / 改动 12)
```

### 4. 没有新增 `cua_node` 依赖 【实证】

事实包显示 `cua_node` 依赖新增数量为 `0`。

这意味着本次差异里没有看到通过新增依赖来扩展底层能力的证据。结合 bundle 文件总数 `2075 → 2075`,更像是一次小范围维护构建,而不是能力面扩张版本。

关键证据:

```text
新增 0 个包

```
