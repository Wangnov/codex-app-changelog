# Codex 桌面版 26.602.40724 macOS / Windows 对照

> 第三方跨平台变更日志 · 非官方

这一批次显示，macOS 与 Windows 的共同变化集中在内置 Browser/Chrome 插件版本对齐和 profile 分享卡宠物预览 UI 信号；Windows 还单独出现线程面板入口扩展与 Computer Use 相关包内元数据更新。

One-liner: this batch mainly aligns bundled plugin metadata across platforms, adds profile share-card pet-preview signals on both platforms, and shows Windows-only thread panel and Computer Use metadata changes.

## 关于这份报告

本报告只依据两份官方签名分发包逆向重建后的机器提取事实：macOS arm64 从 `26.602.30954` build `3575` 到 `26.602.40724` build `3593`，Windows x64 从 `26.602.3474.0` 到 `26.602.4764.0`。两平台版本号与发布时间可能错位，因此所有结论按平台分别引用，不把一端变化直接套到另一端。

证据等级分两档：`【实证】` 表示来自 plist、manifest、插件清单、服务入口、依赖清单、可读字符串或关键文件 diff 等明确进入包内的事实；`【信号】` 表示仅来自前端模块名、样式类、体积变化或压缩 bundle 方向线索。平台归属使用 `[macOS+Windows]`、`[仅 macOS]`、`[仅 Windows]` 标记。

对比范围覆盖版本与渠道、Bundle 文件树、cua_node 依赖、app.asar 概览、同名前端模块改动线索、主进程/前端新增可读字符串、Info.plist 或 AppxManifest 变化、内容安全策略、新增组件与资源、关键文件具体改动。事实包未提供新增内置插件明细、Computer Use 运行时总量、AppxManifest 变化、内容安全策略变化或新增组件资源的独立明细；本报告不会补写不存在的数字。

## ✨ 重点变化

### 【实证】[macOS+Windows] Browser 与 Chrome 内置插件版本对齐到 26.602.40724

两平台的 `browser` 与 `chrome` 内置插件清单都把版本号从 `26.602.30954` 更新到 `26.602.40724`。插件描述本身在事实包 diff 中未显示内容变化，因此可确认的是版本元数据对齐，而不是能力描述变化。

macOS 路径为 `Contents/Resources/plugins/openai-bundled/plugins/browser/.codex-plugin/plugin.json` 与 `Contents/Resources/plugins/openai-bundled/plugins/chrome/.codex-plugin/plugin.json`。Windows 路径为 `resources/plugins/openai-bundled/plugins/browser/.codex-plugin/plugin.json` 与 `resources/plugins/openai-bundled/plugins/chrome/.codex-plugin/plugin.json`。

### 【信号】[macOS+Windows] profile 分享卡出现自定义宠物预览控制

两平台同名 `profile` 前端模块都有相同的新增可读字符串与样式变化，指向个人资料分享卡预览中的 custom pet 控制：包括 `Custom pet`、`Show pet`、`Hide pet`、`Next custom pet`、`profile.shareCard.preview.hidePetLabel`、`profile.shareCard.preview.nextPet`，以及面向辅助功能的宠物切换说明。

这属于前端模块改动线索，不能断言所有用户已经可见；但 macOS 与 Windows 同时出现同一组字符串，说明相关 UI、状态或交互已进入两平台前端 bundle。

### 【信号】[仅 Windows] 线程侧面板/底部面板新增入口线索

Windows 的 `thread-app-shell-chrome` 模块新增了大量面板入口相关字符串，包括 `Bottom panel`、`Browse project files`、打开 side chat、browser tab、review tab、terminal、timeline tab、workspace file browser，以及关闭 thread bottom panel、从底部面板或侧面板打开另一个 tab 的按钮标签。

这些是高可见 UI 信号，但来源仍是同名前端模块改动线索，因此只能说明 Windows 包内出现了线程侧/底部面板入口与新标签页动作方向，不能当作源码级功能完成事实。macOS 事实包没有对应 `thread-app-shell-chrome` 改动线索。

### 【实证】[仅 Windows] Computer Use 插件版本元数据更新，运行时安装时间刷新

Windows 的 `resources/plugins/openai-bundled/plugins/computer-use/.codex-plugin/plugin.json` 把 `computer-use` 插件版本从 `26.602.30954` 更新为 `26.602.40724`，描述仍为通过 Computer Use 控制 Windows 桌面应用。

Windows 还更新了 `.installed-owl-shell.json` 中的 `installedAt`：从 `2026-06-04T20:33:05.799Z` 到 `2026-06-05T16:55:50.166Z`。同一片段显示 runtimeInfo 架构为 `x64`，archiveSha 为 `76a82b584f412bc890db1bfffdc24456a772f47cae7324a4e22eff7bb838c86f`。事实包没有给出 Computer Use 运行时总量，因此不能写体积或文件数结论。

### 【实证】[仅 macOS] Info.plist 版本与 app.asar 完整性哈希更新

macOS 的 `Info.plist` 显示 `CFBundleShortVersionString` 从 `26.602.30954` 到 `26.602.40724`，`CFBundleVersion` 从 `3575` 到 `3593`。`ElectronAsarIntegrity/Resources/app.asar/hash` 也从 `cd8bc6773c8ff92b1abcfe217382916a4bfc43d96d78932b4fe6b823d20cbcdd` 更新为 `0d60982859a8e93d9df7bd629e63cf70b2d83c482420bee78c3af36fb0bf5e67`。

Windows 事实包没有提供 AppxManifest 变化明细，因此本条仅适用于 macOS。

### 【信号】[仅 macOS] header 模块移除近期任务菜单与 Chrome 线程入口相关文案线索

macOS 的 `header` 模块体积减少 `28840 bytes`，并移除多条可读字符串：近期任务菜单、未读任务更新状态、环境筛选、`All tasks`、返回按钮、打开 active Chrome extension thread、清空搜索等相关标签。

这仍属于同名前端模块线索，不能直接判定功能删除；更稳妥的读法是 header 层的任务菜单、搜索、Chrome 线程入口或相关可访问性文案发生了重排、迁移或压缩产物变化。Windows 事实包没有对应 `header` 模块线索。

【信号】[macOS+Windows] `profile` 模块新增 `leading-3`、`left-1/2`、`mt-3`、`mt-[60px]`、`object-contain`、`px-0`、`py-1`、`rotate-180`、`size-7`、`top-full`、`w-[4.25rem]`，移除 `h-auto`、`w-auto`。这些样式类与宠物预览字符串互相印证，指向分享卡预览控件、定位和图像适配调整；证据边界是前端 bundle 信号。

### macOS header 可访问性与任务入口文案

【信号】[仅 macOS] `header` 移除 `aria-atomic`、`aria-hidden`、`aria-label`、`aria-live`、`flex-1`、`flex-col`、`font-medium`、`gap-*` 等类名/属性名，同时移除任务菜单、筛选和搜索相关字符串。它提示 header 内部结构或状态呈现发生变化；证据边界是移除线索，不能单独证明用户入口消失。

【信号】[macOS+Windows] 两平台 `app.asar` 都只增加约 `+0.01 MB`，且 app.asar 内部新增/移除均为 `17 / 17`。事实包明确指出 webview/assets 下大量增减属于 Vite 构建 hash 重命名，不能据此判断功能增减。
