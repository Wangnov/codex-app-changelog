# Codex 桌面版 26.602.40724(macOS / Windows 对照)

> 第三方跨平台变更日志 · 非官方

本批次主要呈现为应用层小幅更新，macOS 有可读配置证据指向内置 Browser(浏览器自动化插件) 与 Chrome(浏览器自动化插件) 插件版本同步，Windows 侧只看到同级别打包变化信号。

One-liner: A small cross-platform app-layer update, with confirmed bundled Browser/Chrome plugin metadata bumps on macOS and matching-but-not-source-readable Windows packaging movement.

## 关于这份报告

本报告基于官方签名分发包的逆向 diff(差异比对)：macOS 取 Sparkle(自动更新框架) 全量包，Windows 取 MSIX(Windows 应用安装包) 包。macOS 对比 `26.602.30954` build `3575` → `26.602.40724` build `3593`；Windows 对比 `26.602.3474.0` → `26.602.4764.0`。发布日期按本批次事实包标记为 `2026-06-05`。

证据等级分两档：【实证】表示来自 plist(应用属性清单)、AppxManifest(Windows 应用清单)、技能文档、类型声明、package.json(包配置文件)、依赖清单等可读证据；【信号】表示仅来自前端模块名、文件数量或体积变化，只能指出方向，不能还原源码意图。

平台归属标记含义：`[macOS+Windows]` 表示两份事实包都支持该变化；`[仅 macOS]` 表示只有 macOS 事实包能证明；`[仅 Windows]` 表示只有 Windows 事实包能证明。这里的“仅”指本次事实包可证范围，不等同于源码一定没有在另一平台存在。

## ✨ 重点变化

### 应用层小幅更新

【信号】[macOS+Windows]

两平台的主应用包 `app.asar`(Electron 应用归档) 都发生了小幅体积变化，且变化量均为 `+0.01 MB`。macOS 路径为 `Contents/Resources/app.asar`，体积 `143.1 MB` → `143.1 MB`；Windows 路径为 `resources/app.asar`，体积 `146.9 MB` → `146.9 MB`。

平台差异在于：macOS 事实包给出了可读文件 diff，可进一步落到插件元数据版本同步；Windows 事实包只提供文件树和 app.asar 级别统计，没有给出可读源码或配置 diff，因此不能把 macOS 侧的具体插件变化直接归因到 Windows。

### 内置 Browser 与 Chrome 插件版本同步

【实证】[仅 macOS]

macOS 包内两个 openai-bundled(内置 OpenAI 插件集合) 插件的 `plugin.json` 版本号从 `26.602.30954` 更新到 `26.602.40724`：

- `plugins/browser/.codex-plugin/plugin.json`
- `plugins/chrome/.codex-plugin/plugin.json`

这说明 macOS 包内的 Browser 与 Chrome 插件元数据已跟随应用版本同步。Windows 事实包没有提供对应 `plugin.json` 的可读 diff，因此本报告不能将该变化标为 `[macOS+Windows]`。

### 未见 cua_node 依赖新增

【实证】[macOS+Windows]

两平台的事实包都显示 `cua_node` 依赖新增数量为 `0`。这意味着本批次没有证据显示 native/Node(本地运行时与 JavaScript 运行时) 依赖集合扩张。

平台差异：macOS 与 Windows 在该项上结论一致，均未新增包。

### 文件树结构整体稳定

【实证】[macOS+Windows]

两平台外层文件数均保持不变：macOS 为 `2075 → 2075`，Windows 为 `8724 → 8724`。两者也都显示外层新增 `0`、移除 `0`，说明本批次没有证据显示安装包顶层结构扩张或收缩。

平台差异主要体现在内容改动数量：macOS 外层内容改动 `53` 个文件，Windows 外层内容改动 `15` 个文件。

### app.asar 内部存在等量增删，主要应排除构建 hash 噪音

【信号】[macOS+Windows]

两平台的 `app.asar` 内部文件数量都保持不变，但均出现 `新增 17 / 移除 17`。macOS 内部文件为 `2830 → 2830`，改动 `12`；Windows 内部文件为 `2778 → 2778`，改动 `11`。

事实包明确提示 `webview/assets` 下大量增减是 Vite(前端构建工具) 构建 hash(内容指纹) 重命名，属于噪音，不应据此判断功能增减。因此这里仅能作为“应用层被重新构建并有少量内容变化”的信号。

### 版本与完整性元数据更新

【实证】[仅 macOS]

macOS `Info.plist` 显示版本元数据更新：

- `CFBundleShortVersionString`: `26.602.30954` → `26.602.40724`
- `CFBundleVersion`: `3575` → `3593`
- `ElectronAsarIntegrity/Resources/app.asar/hash`: `cd8bc6773c8ff92b1abcfe217382916a4bfc43d96d78932b4fe6b823d20cbcdd` → `0d60982859a8e93d9df7bd629e63cf70b2d83c482420bee78c3af36fb0bf5e67`

Windows 事实包未提供 AppxManifest 或同类可读清单 diff，因此该元数据变化只标为 macOS 可实证。
