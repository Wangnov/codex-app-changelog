## ✨ 重点变化

### 1. 远程 Codex 认证失败路径出现新文案 【实证】

主进程 / 前端 bundle 新增了可读字符串：

```text

```

这表明新版产物中出现了针对远程 Codex 未认证状态的明确错误路径。对用户或开发者而言，可感知点可能是远程相关流程在未认证时给出更具体的失败原因；但压缩产物无法还原完整源码语义，也不能仅凭字符串判断触发条件或 UI 呈现位置。

相关对照中，旧版移除了：

```text

```

这显示远程失败处理的文案焦点发生变化，但不能进一步推断实现细节。

### 2. 底层 `codex` 组件明显增大 【信号】
