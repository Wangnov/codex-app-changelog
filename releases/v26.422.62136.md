# Codex 桌面版 `26.422.62136`(build 2180)

这是一次从 `26.422.30944` 到 `26.422.62136` 的 macOS arm64(苹果芯片架构)桌面版差异回填报告。  
One-liner: stable bundle shape, updated signed build, and a larger core `codex` component.

这不是 OpenAI(开放人工智能公司)官方变更日志，而是一份**基于官方分发产物的逆向差异报告**。

本版属于历史回填：两个版本均已超出官方 Sparkle(应用自动更新框架)增量窗口，因此方法不是增量包重建，而是下载两个官方全量包后直接比较，并生成结构化 bundle(应用包)差异。

- 最新版：`accepted`，`source=Notarized Developer ID`，`origin=Developer ID Application: OpenAI OpCo, LLC (2DC432GLL2)`

本文使用两档证据等级：

- **【实证】**：来自 Info.plist(应用元信息文件)、依赖清单、签名验证、文件级统计等人类可读直接证据。
- **【信号】**：来自组件体积变化等间接证据；这类信息只指向某方向，但无法从产物还原到源码级意图。

## ✨ 重点变化

### 1. 版本与 build 正式推进到 `26.422.62136` / `2180` 【实证】

应用元信息显示，`CFBundleShortVersionString` 从 `26.422.30944` 更新到 `26.422.62136`，`CFBundleVersion` 从 `2080` 更新到 `2180`。

这意味着这不是单纯重打包或签名变化，而是一个带明确版本号推进的 macOS arm64 发布版本。

关键证据：

```diff
CFBundleShortVersionString: 26.422.30944 → 26.422.62136
CFBundleVersion: 2080 → 2180
```

### 2. bundle 文件树保持稳定，没有新增或移除顶层文件 【实证】

两版 bundle 文件数均为 `507`，新增 `0`、移除 `0`、内容改动 `47`。

这说明从应用包外层结构看，本版没有引入新的独立组件、插件或资源目录；变化集中在已有文件内容中。对开发者来说，这更像一次已有组件的更新，而不是应用形态或打包结构的扩展。

关键证据：

```text
文件数: 507 → 507
新增 0 / 移除 0 / 内容改动 47
```

### 3. 核心 `codex` 组件体积增加约 3.48 MB 【信号】

`Contents/Resources/codex` 从 `180.9 MB` 增至 `184.4 MB`，变化为 `+3.48 MB`。

这指向核心运行组件存在实质内容更新，但仅凭二进制体积变化，无法从产物还原到源码级意图，也不能判断具体新增了哪些能力或修复了哪些问题。

关键证据：

```text
Contents/Resources/codex: 180.9 MB → 184.4 MB (+3.48 MB)
```

### 4. 应用层内部文件数量未变，hash 文件名变化不应解读为新功能 【实证】

`app.asar`(Electron 应用资源归档)内部文件数为 `1031 → 1031`，机器差异中有新增 `73`、移除 `73`、改动 `7`。事实包明确说明，`webview/assets` 下大量增减是 Vite(前端构建工具)构建 hash(内容指纹)重命名，属于噪音。

因此，这部分不能写成“新增了 73 个功能文件”或“移除了 73 个功能文件”。可读结论只是：应用层归档内容有更新，但文件名级增减主要受构建产物重命名影响。

关键证据：

```text
内部文件: 1031 → 1031
新增 73 / 移除 73 / 改动 7
注: webview/assets 下大量增减是 Vite 构建 hash 重命名,属噪音
```

### 5. `cua_node` 依赖没有新增包 【实证】

`cua_node` 依赖变化中，新增包数量为 `0`。

这意味着从该依赖清单看，本版没有通过新增 `cua_node` 包来扩展相关能力。可感知层面，本报告没有证据支持“新增了一组依赖驱动的新功能”。

关键证据：

