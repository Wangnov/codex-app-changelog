# Codex 桌面版 26.527.30818(macOS / Windows 对照)

> 第三方跨平台变更日志 · 非官方

本批次的主线是桌面外壳、浏览器自动化宿主与应用层模块结构的同步重排，但 macOS(苹果桌面系统)与 Windows(微软桌面系统)落点并不完全一致。

One-liner: this batch reshapes the desktop runtime and browser-host packaging across platforms, with macOS exposing more Chromium/Owl metadata while Windows shifts to a much fuller Chromium-style layout.

## 关于这份报告

本报告基于两份官方签名分发包的逆向 diff(差异对比):macOS 取 Sparkle(应用更新框架)全量包，Windows 取 MSIX(Windows 应用安装包格式)包。macOS 对比 `26.519.81530` build `3178` → `26.527.30818` build `3370`; Windows 对比 `26.519.11010.0` → `26.527.3378.0`。统一发布日期按事实包给出的 `2026-05-27` 记录。

证据等级分两档:【实证】表示来自 Info.plist(macOS 应用元数据文件)、package.json(包配置文件)、插件配置、脚本文档或文件清单等可读证据;【信号】表示只来自前端模块名、体积或文件布局变化，只能说明方向，不能直接还原源码意图。平台标记含义为:[macOS+Windows] 两平台都有证据、[仅 macOS] 只在 macOS 事实包出现、[仅 Windows] 只在 Windows 事实包出现。

## ✨ 重点变化

### 桌面外壳从 Electron 命名转向 Chromium 命名

【实证】[macOS+Windows] 两个平台都出现了明显的运行时布局重排。macOS 新增 `Contents/Frameworks/Codex Framework.framework/Versions/148.0.7778.179/` 下的大量 Chromium(浏览器内核项目)框架文件，同时移除 `Electron Framework.framework/Versions/A/` 下的大量对应文件;Windows 新增 `chrome.dll`、`VkLayer_khronos_validation.dll`、`chrome_pwa_launcher.exe`、`chrome_proxy.exe` 等文件，并且 `Codex.exe` 从 `226.6 MB` 降到 `4.6 MB`。

【实证】[仅 macOS] macOS 的 `Info.plist` 新增 `ChromiumBaseVersion = 148.0.7778.179`、`ChromiumBaseBundleVersion = 7778.179`、`SCMRevision = 4687a88676f95bfbce9ec62314791faae363fea6-refs/branch-heads/7778@{#3197}`，且 `NSPrincipalClass` 从 `AtomApplication` 改为 `BrowserCrApplication`。这比 Windows 事实包提供了更明确的 Chromium 版本证据。

### 浏览器自动化宿主不再把 app-server-runtime 固定在插件目录里

【实证】[macOS+Windows] 两个平台都移除了 Chrome(谷歌浏览器)插件目录下的 `app-server-runtime` 大文件:macOS 移除 `plugins/chrome/app-server-runtime/codex`、`node`、`node_repl`;Windows 移除 `resources/plugins/openai-bundled/plugins/chrome/app-server-runtime/codex`、`node`、`node_repl`。与此同时，macOS 根资源下的 `Contents/Resources/codex`、`Contents/Resources/node`、`Contents/Resources/node_repl` 仍在变化，Windows 根资源下的 `resources/codex.exe`、`resources/codex` 也在变化，说明运行时被重新安置到更靠近应用资源根的位置。

【实证】[仅 macOS] macOS 的 `installManifest.mjs` 进一步证实了这次路径解耦:旧逻辑从 `app-server-runtime` 拼出 `codex`、`node`、`node_repl`;新逻辑改为接收 `appServerRuntimePaths`，并只在安装配置里拼接 `scripts/browser-client.mjs`。这说明 macOS 包里至少已经把 Chrome Native Messaging(浏览器原生消息机制)宿主安装逻辑改成了可注入运行时路径。

### 应用层体积下降，但内部模块大幅重排

【实证】[macOS+Windows] 两个平台的 `app.asar`(Electron 应用归档格式)都变小。macOS `Contents/Resources/app.asar` 从 `159.0 MB` 降到 `140.3 MB`，Windows `resources/app.asar` 从 `159.0 MB` 降到 `144.1 MB`。两边内部文件数都大幅增加:macOS `1577 → 2774`，Windows `1531 → 2722`。

【信号】[仅 macOS] macOS 事实包列出了去除 Vite(前端构建工具)hash 噪音后的前端模块名，方向集中在 appgen、automation、Mermaid(图表语法库)图表、workspace、MCP(Model Context Protocol，模型上下文协议)能力视图、pull request(代码合并请求)状态、browser-sidebar、annotation 等区域。但这些只是模块名信号，不能证明对应能力已对用户开放。

### 插件版本同步升级，LaTeX 插件单独前进

【实证】[仅 macOS] macOS 的插件配置显示 `chrome` 与 `browser` 插件版本从 `26.519.81530` 升到 `26.527.30818`，`latex` 插件从 `0.2.0` 升到 `0.2.2`。Windows 事实包没有给出对应插件配置 diff，因此不能确认同样的插件版本元数据变化。

【实证】[macOS+Windows] 两个平台都移除了 `plugins/chrome/skills/chrome/SKILL.md` 与 `plugins/browser/skills/browser/SKILL.md` 对应路径的技能文档文件。Windows 还移除了 `resources/plugins/openai-bundled/plugins/browser/skills/browser/agents/openai.yaml`。

### macOS 增加认证、文件类型与脚本能力元数据

【实证】[仅 macOS] `Info.plist` 新增 `ASWebAuthenticationSessionWebBrowserSupportCapabilities` 下的三项能力标记，包含 `CallbackURLMatchingIsSupported`、`EphemeralBrowserSessionIsSupported`、`IsSupported`。这指向 macOS 系统级网页登录/回调能力的声明变化。

【实证】[仅 macOS] `Info.plist` 新增多组 `UTImportedTypeDeclarations` 与 `UTExportedTypeDeclarations`，覆盖 `crx`、`crwebloc`、GIF、HTML、XHTML、JavaScript、JPEG、MHTML、Ogg、PNG、SVG、text、WebM、WebP、PDF、AVIF 等类型。同时新增 `NSAppleScriptEnabled = True`、`OSAScriptingDefinition = scripting.sdef`、`NSBonjourServices[0] = _googlecast._tcp` 等平台元数据。Windows 事实包没有对应 manifest 细节。

### Owl 开发/外壳路径在 macOS 可读配置中更明显

【实证】[仅 macOS] `package.json` 中 `owl` 脚本从直接运行 `owl-shell.mjs run` 改为先执行 `build:dev-vite`，并新增 `build:dev-vite = node ./scripts/build-dev-vite.mjs`。`preload.js` 也新增 `get-uses-owl-app-shell` 与 `usesOwlAppShell()` 暴露。事实包只能证明这些开发/外壳路径存在，不能证明用户界面默认切换。

### Windows 包体结构更接近完整 Chromium 分发

【实证】[仅 Windows] Windows 文件树从 `831 → 8648`，新增 `7827`、移除 `10`、内容改动 `95`，新增文件包含 `chrome.dll` `306.2 MB`、DevTools(开发者工具)相关 `gen/third_party/devtools-frontend/...` 文件、`elevated_tracing_service.exe`、`elevation_service.exe`、`notification_helper.exe` 等。这个变化在 macOS 中表现为 Framework(框架)目录重命名与版本化，在 Windows 中则表现为大量 Chromium 文件直接进入包内。

【实证】[仅 Windows] Windows 的 `resources/native/windows-updater.node` 从 `0.2 MB` 增至 `1.1 MB`，`resources/codex-windows-sandbox-setup.exe` 从 `6.1 MB` 增至 `8.4 MB`，且多个 `node-pty`(Node 终端伪终端库)原生组件显著增大。macOS 事实包没有对应 Windows 原生更新器或沙箱安装器变化。
