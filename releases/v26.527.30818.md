# Codex 桌面版 26.527.30818 macOS / Windows 对照

> 第三方跨平台变更日志 · 非官方

这一批次的核心不是单点功能，而是桌面壳、浏览器自动化、插件运行时与工作区交接能力的一次成体系迁移。

One-liner: Codex Desktop 26.527 moves both platforms toward the Owl/Chromium shell and richer browser/runtime orchestration, while Windows gains a full bundled Computer Use plugin.

## 关于这份报告

本文只使用输入事实包中的机器提取结果。macOS 对比范围是 26.519.81530 build 3178 → 26.527.30818 build 3370；Windows 对比范围是 26.519.11010.0 → 26.527.3378.0。两平台版本号和包结构不同，因此所有结论按各自事实引用。

证据等级分两档：【实证】表示来自 plist、manifest、package.json、插件清单、技能文档、服务入口、依赖清单、可读字符串或具体配置 diff；【信号】表示仅来自前端模块名、体积变化、样式类名或压缩 bundle 的方向线索。

平台归属使用 [macOS+Windows]、[仅 macOS]、[仅 Windows]。同名前端模块改动只作为 UI、状态或交互方向信号，不等同于源码事实或用户已可见功能。

本批次没有提供 Windows AppxManifest 逐项变化，也没有提供内容安全策略变化；因此本文不推断这两类配置。

## ✨ 重点变化

### 【实证】[macOS+Windows] 桌面壳迁移到 Owl/Chromium 方向

两平台 `package.json` 都把版本从 `26.519.81530` 提到 `26.527.30818`，build 从 `3178` 提到 `3370`，并新增 `build:dev-vite`，同时 `owl` 脚本改为先执行 dev vite build 再运行 `owl-shell.mjs`。`preload.js` 新增 `get-uses-owl-app-shell`、`usesOwlAppShell()` 与 `connect-app-host` 消息通道。

macOS 的证据更强：`Info.plist` 中 `NSPrincipalClass` 从 `AtomApplication` 变为 `BrowserCrApplication`，新增 `ChromiumBaseVersion = 148.0.7778.179`、`ChromiumBaseBundleVersion = 7778.179`、大量 Chromium 类型声明，并且文件树显示 `Electron Framework.framework` 被移除、`Codex Framework.framework/Versions/148.0.7778.179` 新增。Windows 侧也出现 `chrome.dll`、`chrome_pwa_launcher.exe`、`elevation_service.exe`、`notification_helper.exe` 等大量 Chromium 相关新增文件，`Codex.exe` 从 226.6 MB 降到 4.6 MB，说明打包形态发生大幅重排。

### 【实证】[仅 Windows] 新增完整 Computer Use 内置插件

Windows 内置插件清单从 `browser`, `chrome`, `latex` 变为 `browser`, `chrome`, `computer-use`, `latex`。新增 `computer-use` 插件包含 187 个文件、总大小 2.9 MB，版本为 `26.527.30818`，显示名为 `Computer Use`，短描述为 “Control Windows apps from Codex”。

该插件不是单纯体积变化：manifest 给出分类 `Productivity`、关键词 `computer-use`, `desktop-control`, `windows`, `automation`, `node-repl`, `accessibility`，默认提示包含播放 playlist、在 Visual Studio 中构建运行项目、打开 Notepad 整理草稿。技能文档说明它通过 SendInput 与 UI Automation 自动化 Windows 应用，并通过 Windows.Graphics.Capture 截取窗口；最大文件包括 `node_modules/@oai/sky/bin/windows/codex-computer-use.exe` 1.7 MB 与 `assets/app-icon.png` 0.9 MB。

macOS 输入样本没有对应的内置插件清单新增事实，因此不能把 Windows 的 Computer Use 插件面套到 macOS。两平台主 bundle 都新增了 `Computer Use helper already has an active request`、`Computer Use native pipe client disconnected` 等字符串，但这只能证明相关错误路径进入包内，不等同于 macOS 打包了同一插件。

### 【实证】[macOS+Windows] Browser Use 路由、侧栏 webview 与 Chrome native host 管线增强

两平台主进程/前端 bundle 新增大量 Browser 相关字符串，包括 `Bound Browser Use session route to exact owner`、`Captured Browser Use session route`、`Registered Browser sidebar thread`、`Requested Browser sidebar webview attachment`、`Runtime attached Browser sidebar webview`、`No in-app Browser host is available for this browser-use session`、`Opening routed Browser page`、`Synced exact Browser owner`。这表明包内新增了围绕 Browser Use 会话路由、所有者绑定、侧栏 webview 附着和恢复的可观测路径。

Chrome native host 也有明确改动。macOS 的 `installManifest.mjs` 不再从 `app-server-runtime` 拼接 `codex/node/node_repl` 路径，而是通过 `appServerRuntimePaths` 注入 runtime 路径；两平台新增字符串还包括 `Chrome native host resource sync requires resourcesPath`、`Invalid Chrome native host install request`、`Failed to stop runtime marketplace Chrome native host`。插件版本方面，macOS `browser` 与 `chrome` 插件版本均从 `26.519.81530` 提到 `26.527.30818`；Windows 包体中旧的 `chrome/app-server-runtime` 大文件也被移除。

### 【实证】[macOS+Windows] RPC、MessagePort 与流式 stub 基础设施进入主包

两平台新增大量 RPC 与 stream 相关错误字符串：`An RPC stub cannot be used as a constructor.`、`Cannot serialize RPC stubs without an RPC session.`、`Cannot create pipes without an RPC session.`、`ReadableStream RPC stub was disposed without being consumed`、`Response body must be of type ReadableStream.`、`Peer closed MessagePort connection.`、`Serialization exceeded maximum allowed depth.` 等。`package.json` 同时新增 `capnweb` 依赖。

### 【实证】[macOS+Windows] 远程 host handoff 与 Git worktree 迁移路径增加

两平台主包新增 `Codex host handoff snapshot`、`Failed to create host handoff snapshot`、`Failed to export git handoff bundle`、`Failed to import git handoff bundle`、`Could not find a shared Git base on the destination host`、`Cross-host handoff artifact transfer between SSH hosts is not supported`、`Remote SSH host is missing a destination`、`Remote localhost port forwarding is unavailable for this host.` 等字符串。
