# Codex 桌面版 26.623.42026 macOS / Windows 对照

> 第三方跨平台变更日志 · 非官方

本批次更像是同一代码线在两种打包节奏下的分流：macOS 包含更大范围的前端路由与界面组合变化，Windows 包的可见变化更少但出现技能页和资料统计相关信号。  
One-liner: this build advances shared settings and preview surfaces, while macOS carries broader AppGen/project/plugin UI movement and Windows shows narrower skills/profile signals.

## 关于这份报告

本报告只使用输入事实包中的机器提取事实，比较 macOS arm64 `26.623.41415` build `4505` → `26.623.42026` build `4514`，以及 Windows x64 MSIX `26.623.5175.0` → `26.623.5546.0`。

证据等级分两档：【实证】来自 plist、manifest、插件清单、类型/依赖清单、服务入口、技能文档、配置、以及事实包中不属于“同名前端模块改动线索”的可读字符串；【信号】来自前端模块名、体积变化、压缩 bundle 中的同名模块字符串和样式类名差异。平台归属使用 `[macOS+Windows]`、`[仅 macOS]`、`[仅 Windows]`。

本批次未提供内容安全策略变化区块，也未提供新增内置插件明细。插件层可见改动主要是现有内置插件 manifest 版本号更新。

## ✨ 重点变化

### 【实证】[macOS+Windows] 内置插件 manifest 跟随版本线更新

两平台都确认 `chrome` 和 `browser` 内置插件的 `.codex-plugin/plugin.json` 版本从 `26.623.41415` 更新到 `26.623.42026`。

`browser` 插件描述仍指向 Codex in-app browser，用于打开、检查、导航、测试、点击、输入或截图本地 Web 目标；`chrome` 插件描述仍定位为依赖用户现有 Chrome 状态的自动化能力。这里的实证是 manifest 版本变更，不代表插件能力文本本身在本批次新增。

Windows 额外确认 `computer-use` 插件 manifest 从 `26.623.41415` 更新到 `26.623.42026`，描述为通过 Computer Use 从 Codex 控制 Windows 桌面应用。

### 【实证】[仅 macOS] macOS 应用版本、build 与 asar 完整性更新

macOS `Info.plist` 显示 `CFBundleShortVersionString` 从 `26.623.41415` 变为 `26.623.42026`，`CFBundleVersion` 从 `4505` 变为 `4514`，`ElectronAsarIntegrity/Resources/app.asar/hash` 也同步变化。

本输入样本中，macOS bundle 文件数保持 `8875 → 8875`，新增 `0`、移除 `0`、内容改动 `67`。最大非 hash 噪音体积变化是 `Contents/Resources/app.asar`：`164.7 MB → 164.7 MB`，增加 `+0.04 MB`。

### 【实证】[仅 Windows] Windows 内置 Computer Use 插件版本更新，但运行时总量未变

Windows 包确认 `resources/plugins/openai-bundled/plugins/computer-use/.codex-plugin/plugin.json` 版本从 `26.623.41415` 到 `26.623.42026`，描述仍是从 Codex 控制 Windows 桌面应用。

同时，Windows `cua_node` 总量保持 `298.9 MB → 298.9 MB`，新增依赖包为 `0`。因此可确认插件 manifest 版本更新，但不能据此断言运行时依赖扩展。

### 【实证】[macOS+Windows] Computer Use 运行时体积和 cua_node 依赖没有扩张

macOS `cua_node` 总量为 `377.4 MB → 377.4 MB`，Windows 为 `298.9 MB → 298.9 MB`，两者均为 `+0 MB, +0%`。两平台的 `cua_node` 新增包数量均为 `0`。

这是运行时包体和依赖层面的稳定信号：本批次重点不在 Computer Use runtime 的依赖扩容。

### 【信号】[macOS+Windows] 设置页继续扩展默认权限、听写、代码显示和工作模式选项

两平台 `general-settings` 同名模块都新增了相同一组可读字符串，包括 disabled default permissions toggle、work mode settings options、code base size adjustment、hold-to-dictate hotkey、hotkey window hotkey、dictation dictionary entry、Popout Window projectless default、ambient suggestions toggle、color diff marker option、chrome theme contrast slider。

这说明设置页正在向默认权限、工作模式、听写字典、弹出窗口默认行为、环境建议、差异颜色标记和 Chrome 主题对比度等方向变化。由于这些来自同名前端模块改动线索，只能视为 UI/交互信号，不能确认所有选项已经对所有用户可见。

### 【信号】[macOS+Windows] Workbook / 文档预览能力明显增重

两平台 `workbook` 模块都从接近空模块增至约 `2.8 MB`，新增字符串包含 unnamed range、workbook.help、valid values、accent/bg/dk 色彩主题片段，以及 `data-granola-*` 版式和数学/段落属性类名。

macOS 还新增了 `settings-page~projects-index-page~notebook-preview-panel~docx-preview-panel` 模块名。综合来看，文档、表格或 notebook/docx 预览相关前端能力有明显进入包内的信号；但除模块名和压缩字符串外，本批次没有提供源文件级实现证据。

### 【信号】[macOS+Windows] 工作树初始化失败路径被收缩或迁移

两平台 `worktree-init-v2-page` 同名模块都移除了 `Continue anyway`、`Edit environment`、`Error creating local task from worktree`，以及打开本地环境设置、开始 repair thread、取消工作树创建并开始本地会话等按钮文案。

这显示工作树创建失败后的 UI 路径在两平台均发生调整。它是前端 bundle 信号，不能直接等同于后端工作树逻辑删除。

### 【信号】[仅 macOS] macOS 前端路由组合大幅重排，AppGen、远程会话、onboarding 和项目页更活跃

macOS 去 hash 后新增大量前端模块，集中在 `appgen-library-page`、`appgen-settings-page`、`appgen-page`、`appgen-publication-terms-route`、`remote-conversation-page`、`new-thread-panel-page`、`onboarding-page`、`projects-index-page`、`pull-requests-page`、`plugin-detail-page`、`remote-connections-settings` 和 `worktree-init-v2-page` 的组合路由。

这不是单纯 hash 噪音：事实包已经剔除 hash 改名后列出真实模块增删。但这些仍是前端代码模块信号，只能说明这些界面组合进入或退出 bundle 拆分结构，不能确认功能发布范围。

### 【信号】[仅 macOS] 插件详情页和 AppGen 分享弹窗出现工作区分享/公开发布文案

macOS `plugin-detail-page` 新增插件分享相关字符串：搜索 workspace users and groups、移除 selected workspace share target、MCP server enablement toggle、section navigation、`Add people or groups`、`Anyone at workspace with the link`、`Anyone in this workspace with the link`、`App unavailable` 等。

macOS `appgen-share-dialog` 新增站点分享和发布文案，包括 `Anyone on the Internet`、copy live site URL、open live site externally、publish a site to the public internet、`Can visit`、`Copied to clipboard`、`Copy link`。这些共同指向插件/AppGen 分享和公开发布界面变化，但证据等级仍是同名前端模块信号。

### 【信号】[仅 Windows] 插件详情页相关分享和 MCP 开关字符串被移除

Windows `plugin-detail-page` 体积从 `0.2 MB` 降到接近空模块，移除了与 macOS 新增方向相同的一组插件分享字符串：workspace share target、workspace users and groups、MCP server enablement toggle、section navigation、`Add people or groups`、workspace link 和 `App unavailable`。

这与 macOS 方向相反，说明两平台在这一批次的插件详情页 bundle 状态并不同步。不能把 macOS 的插件分享界面信号套到 Windows。

### 【信号】[仅 macOS] Header、侧栏和任务入口新增 Cloud tasks / Codex for Chrome / recent tasks 文案

macOS `header` 新增 `Accessible label for opening the recent tasks menu`、unread task updates、`All environments filter option`、`All tasks`、`App settings`、Chrome extension side panel header、`Chrome computer use settings`、`Codex for Chrome`、`Cloud tasks` 等字符串。

macOS 侧栏相关模块还新增 `All chats`、`Codex Cloud`、cloud tasks section、connection thread groups、project thread groups、recent threads list、all chats chronological list、`features.remote_connections` 等。它们共同指向任务菜单、云任务、Chrome 侧栏和远程连接导航的界面重排信号。

### 【信号】[仅 Windows] Header 中 recent tasks / Cloud tasks / Codex for Chrome 相关字符串被移除

Windows `header` 模块体积减少约 `40005 bytes`，移除了 recent tasks menu、unread task updates、All environments、All tasks、App settings、Chrome extension side panel header、clear search 等字符串。

因此，macOS 上的 header 新增不能归纳为跨平台新增；Windows 样本中相同区域反而呈现删除信号。

### 【信号】[仅 Windows] 技能页出现更完整的列表、搜索、推荐和刷新文案

Windows `skills-page` 新增创建新 skill、reload skills list、新装 skills 需要 refresh 后才能在 composer 使用、过滤空态、skills page 标题、installed skills、recommended skills、search input、loading skills 等字符串。

这些是 Windows 独有的高可见 UI 信号。macOS 本批次没有同名 `skills-page` 改动线索，但 macOS 新增模块名中出现 skills settings 和 plugins settings 的组合路由；两者证据层级和位置不同，不能合并成同一结论。

### 【信号】[仅 Windows] Profile stats 文案进入 profile-queries

Windows `profile-queries` 新增 profile streak stats、duration hours、hours and minutes、minutes、minutes and seconds、seconds 等格式化字符串。

这说明资料页或使用统计相关展示在 Windows 包内出现新信号。macOS 对应 `profile-queries` 模块被列为移除模块，同时 `onboarding-page~profile` 仅新增 `Profile photo upload failed` 并大幅减小体积，因此两平台 profile 方向并不同步。

### 【信号】[仅 macOS] 项目 hover card 和运行环境状态文案增强

macOS `remote-conversation-page~projects-index-page` 新增项目 hover card 的打开来源、打开编辑弹窗、pin project、inline project name editor、重命名失败 toast、active chat count / chat count，以及运行位置文案：`This conversation is running in Codex Cloud.`、`This conversation is running in a local git worktree.`、`This conversation is running locally.`。

这是项目列表、hover card 和会话运行环境标识的明确 UI 信号，仅见于 macOS 事实包。

### 【信号】[仅 macOS] 更新恢复路径进入全屏错误页

macOS 相关模块新增 `Checking for updates`、`Downloading update`、`Installing update`、以及 full-screen error page 上安装桌面更新的按钮和禁用状态文案。

这说明 macOS 包内出现应用更新恢复流程的前端文案信号。Windows 事实包没有对应同名线索。

### 【信号】[仅 macOS] 本地 worker / spawn / 权限错误路径进入前端共享模块

macOS 大型共享模块新增 `Electron worker bridge is unavailable`、`Mismatched worker response method`、`Missing cwd`、`Missing git metadata`、`Unknown method: process/spawn`、`operation not permitted`、`permission denied` 等字符串。

这些是前端 bundle 中进入包内的错误路径和进程调用桥接信号，但仍来自同名前端模块改动线索，不能确认具体调用实现。

### 【信号】[macOS+Windows] 代码高亮/语法包发生重排，macOS 范围更广

macOS 多个语言模块新增语法 scope 字符串，包括 `cpp`、`scss`、`yaml`、`postcss`、`tex`、`angular-html`、`json`、`lua`、`glsl`、`csv`、`html-derivative`，同时 `tsx` 和 `wasm` 出现明显体积收缩或内容迁移信号。

Windows 本批次没有这些语言模块的同名线索。跨平台可确认的是 `workbook` 相关预览模块变大；语法高亮重排主要是 macOS 样本中的低层前端信号。

## 🔎 低层信号索引

### 设置、语音和外观

`general-settings` 在两平台同时出现默认权限、work mode、hold-to-dictate、dictation dictionary、projectless popout、ambient suggestions、diff marker、Chrome theme contrast 等字符串。边界：同名前端模块信号，不确认设置项发布状态。

macOS `pets-settings~appearance-settings~general-settings` 还新增 `Agent environment`、`Auto detect`、`Bottom panel`、`Default file open destination`、`Default microphone input option label`、`Default permissions`、`Default terminal location`、`Default to projectless chat`、`Detached code review option label`。边界：macOS 独有的设置页组合信号。

### 任务、云环境和远程连接

macOS `header`、侧栏模块和项目页模块共同出现 `Cloud tasks`、`Codex Cloud`、connection thread groups、remote connections flag、local git worktree / locally / Codex Cloud 运行状态。边界：前端文案信号，不等同于远程执行能力新增。

Windows `header` 对相近字符串呈删除方向，说明该区域不能做跨平台推广。

### 插件、技能和分享

macOS `plugin-detail-page`、`appgen-share-dialog` 进入分享目标、workspace link、public internet 发布、MCP server enablement toggle 等字符串。Windows `plugin-detail-page` 删除相近字符串，但 `skills-page` 新增技能创建、刷新、推荐、搜索和空态文案。边界：两平台插件/技能 UI 节奏明显错位。

### 文件、预览和代码语言

两平台 `workbook` 大幅增重并出现 workbook helper、命名范围、主题色和 `data-granola-*` 属性。macOS 还出现 `notebook-preview-panel~docx-preview-panel` 模块名和多种语言 grammar 字符串。边界：这是预览/高亮相关 bundle 信号，不是文件格式支持矩阵。

### 错误路径和恢复流程

macOS 出现 worker bridge、spawn、cwd、git metadata、权限错误、更新检查/下载/安装恢复文案。两平台同时移除工作树初始化失败后的继续、编辑环境和修复按钮文案。边界：这些是前端错误路径和 UI 文案变化，不能推断底层错误处理完整实现。
