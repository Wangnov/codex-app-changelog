# Codex 桌面版 26.623.42026 macOS / Windows 对照

> 第三方跨平台变更日志 · 非官方

主旨：这批包不是一个简单的同构重打包；macOS 与 Windows 的包内版本、前端模块走向和插件 manifest 变化需要分平台解读。  
One-liner: This batch looks like a shared codebase with platform-specific packaging and UI rollout divergence.

## 关于这份报告

这份报告只基于两份官方签名分发包逆向重建后的机器差异事实。macOS 对比范围是 `26.623.41415 build 4505` 到 `26.623.42026 build 4514`；Windows 对比范围是 `26.623.5175.0` 到 `26.623.5546.0`。

证据分两档：【实证】来自 plist、manifest、插件配置、依赖清单、服务入口、技能文档、签名信息和非同名前端模块线索中的可读字符串；【信号】来自同名前端模块的字符串差异、样式类名差异、模块名和体积变化。平台归属用 `[macOS+Windows]`、`[仅 macOS]`、`[仅 Windows]` 标记。

盘点已覆盖版本与渠道、Bundle 文件树、内置插件 manifest、Computer Use 运行时、`cua_node` 依赖、`app.asar`、前端模块真实增删、同名前端模块改动线索、可读字符串、Info.plist 或 AppxManifest、CSP、新增组件与资源、关键文件 diff。本输入没有给出 CSP 变化，也没有给出 Windows AppxManifest 具体 diff。

## ✨ 重点变化

### 【实证】[macOS+Windows] Browser 与 Chrome 内置插件 manifest 同步到 26.623.42026

两平台的 `browser` 与 `chrome` 插件 manifest 都把 `version` 从 `26.623.41415` 更新到 `26.623.42026`。这不是前端猜测，而是插件 `.codex-plugin/plugin.json` 的直接 diff。

平台差异在于外层包版本并不同步：macOS 外层版本正是 `26.623.42026`，build 从 `4505` 到 `4514`；Windows MSIX 外层版本是 `26.623.5546.0`，但内置插件 manifest 仍更新到 `26.623.42026` 这一套应用版本号。

### 【实证】[仅 Windows] Computer Use 插件 manifest 在 Windows 同步升级

Windows 的 `computer-use` 插件 manifest 把 `version` 从 `26.623.41415` 更新为 `26.623.42026`，描述仍指向 “Control desktop apps on Windows from Codex through Computer Use.” macOS 本输入没有对应的 `computer-use` manifest diff。

这说明 Windows 包里 Computer Use 插件元数据参与了本次版本推进，但不代表运行时体积或依赖扩容。

### 【实证】[macOS+Windows] Computer Use 运行时与 cua_node 依赖没有扩容

macOS 的 Computer Use 运行时总量保持 `377.4 MB → 377.4 MB`，Windows 保持 `298.9 MB → 298.9 MB`。两边 `cua_node` 新增依赖包数都是 `0`。

这是一条重要的负面事实：本批次能看到插件 manifest 与前端界面信号变化，但不能据此说 Computer Use 底层运行时新增了依赖或明显扩容。

### 【信号】[macOS+Windows] 通用设置页出现更多工作模式、权限、语音和显示相关入口

两平台 `general-settings` 同名模块都新增了可读字符串，覆盖默认权限、工作模式设置、代码基础字号、hold-to-dictate 热键、热键窗口热键、听写词典条目、Popout Window projectless 默认开关、ambient suggestions、diff 颜色标记、Chrome 主题对比度滑块。

因为这些来自同名前端模块改动线索，它们属于 UI 与设置方向的【信号】，不能断言所有入口已经面向用户开放。平台上看，这组信号两边一致，是本批次最清楚的跨平台设置页变化。

### 【信号】[仅 macOS] macOS 额外出现 Agent environment、默认终端位置、默认麦克风和 detached code review 设置线索

macOS 的 `pets-settings~appearance-settings~general-settings` 同名模块新增了 `Agent environment`、`Auto detect`、`Auto detect language option`、`Bottom panel`、`Default file open destination`、`Default microphone input option label`、`Default permissions`、`Default terminal location`、`Default to projectless chat`、`Detached code review option label` 等字符串。

这些字符串只在 macOS 事实包中出现，指向本地环境、默认打开目标、语音输入、终端面板位置和 code review 模式相关设置的前端重排或扩展。

### 【信号】[macOS+Windows] Workbook 前端模块显著变大，出现 workbook.help、命名范围和 Granola 类属性

两平台 `workbook` 模块都从接近空模块变为约 `2.8 MB`：macOS 增加 `2,764,364 bytes`，Windows 增加 `2,761,257 bytes`。新增字符串包括 `(unnamed range)`、`workbook.help(` 相关提示，以及大量 `data-granola-*` 样式或结构属性。

这是强烈的前端能力信号，指向 workbook 相关解析、渲染或交互支持进入包内。但它仍来自同名前端模块线索，只能标为【信号】。

### 【信号】[仅 macOS] 插件详情页朝分享、MCP 开关、技能预览和 app tools 菜单方向增强

macOS 的 `plugin-detail-page` 模块新增了插件分享对话框、搜索 workspace 用户和组、MCP server enablement toggle、section navigation、skill card actions、app tools modal、skill preview modal、`App unavailable`、`Anyone in this workspace with the link` 等字符串。

Windows 同名模块方向相反：它移除了插件分享、MCP server enablement toggle、section navigation、`App unavailable` 等字符串。因此这组变化不能写成跨平台一致增强，只能说 macOS 样本新增，Windows 样本移除。

### 【信号】[仅 macOS] AppGen 与站点分享出现 public publish、copy link、workspace share 线索

macOS 的 `appgen-share-dialog` 新增了站点分享相关字符串，包括添加人员或组、workspace link、Internet 公开访问、复制 live site URL、外部打开 live site、发布到 public internet、`Can visit`、`Copied to clipboard`、`Copy link`。

这些是高可见前端信号，说明 AppGen 站点分享与发布流程的文案和交互入口进入 macOS 包内。Windows 事实包没有对应新增。

### 【信号】[仅 macOS] 项目、侧边栏、Cloud tasks 和远程连接导航继续扩展

macOS 的项目索引相关模块新增了项目 hover card 的打开来源、编辑、置顶、内联改名、聊天数量、重命名失败 toast，以及 `This conversation is running in Codex Cloud.`、`This conversation is running in a local git worktree.`、`This conversation is running locally.`

另一个侧边栏相关模块新增 `All chats`、`Codex Cloud`、cloud tasks、connections、project thread groups、recent threads、`features.remote_connections` 等字符串。它们共同指向项目列表、任务分组、远程连接和运行环境标识的界面增强。

### 【信号】[仅 macOS] 更新恢复、下载进度、rate limit 菜单和 workspace 用量入口进入前端字符串

macOS `automations-page` 新增了桌面应用更新下载、安装、进度按钮文案，workspace monthly usage limit increase、rate limit summary、rate limit resets、rate limit 学习入口等字符串。

macOS 另一个错误恢复相关模块新增 `Checking for updates`、`Downloading update`、`Installing update`、`appUpdate.recovery.updateCodex` 等字符串。Windows 没有对应新增，因此不能写成跨平台更新恢复能力。

### 【信号】[仅 Windows] 技能页出现创建、刷新、推荐、搜索、空状态和加载状态文案

Windows 的 `skills-page` 模块新增了创建新 skill、重新加载 skills list、新安装 skill 需要刷新后才能在 composer 使用、过滤后空状态、installed skills、recommended skills、搜索输入、`Loading skills...` 等字符串。

macOS 本输入没有同名 `skills-page` 变化条目。Windows 这里更像是技能管理页的可见信息结构补齐或重排信号。

### 【信号】[仅 Windows] Profile 统计格式化字符串新增，同时 profile-model 模块被移除

Windows 的 `profile-queries` 新增 profile streak 与时长格式化字符串，包括 day streak、hours、hours and minutes、minutes、minutes and seconds、seconds。Windows 前端真实增删里还移除了 `profile-model` 模块。

这组事实说明 Windows 样本里的 profile 统计查询或展示层发生了重组。macOS 没有相同新增；macOS 另有 `onboarding-page~profile` 模块大幅缩小并新增 `Profile photo upload failed`，属于另一条 macOS-only 信号。

### 【信号】[macOS+Windows] Worktree 初始化失败后的继续、修复和编辑环境文案被移除

两平台 `worktree-init-v2-page` 都移除了本地环境 setup 失败后继续启动 thread、打开本地环境设置、启动 repair thread、取消 worktree setup 并启动 local conversation、`Continue anyway`、`Edit environment`、`Error creating local task from worktree` 等字符串。

这不是说相关流程一定被删除，但至少说明这批同名前端模块中的失败恢复文案和状态表达发生了收缩或迁移。

### 【信号】[仅 macOS] Electron worker bridge、process/spawn、cwd、git metadata 与权限错误路径进入包内

macOS 一个大型同名前端模块新增了 `Electron worker bridge is unavailable`、`Mismatched worker response method`、`Missing cwd`、`Missing git metadata`、`Unknown method: process/spawn`、`operation not permitted`、`permission denied` 等字符串。

这些是运行时桥接、进程启动、工作目录、git 元数据和权限错误路径的前端信号。Windows 事实包没有对应新增。

### 【信号】[仅 macOS] 代码语法与文档渲染相关模块出现大规模语法 token 更新

macOS 同名前端模块中，`cpp`、`scss`、`yaml`、`postcss`、`tex`、`angular-html`、`json`、`wasm`、`lua`、`glsl`、`csv`、`html-derivative` 等模块新增大量语法 token 字符串或样式类名。`tsx` 方向相反，移除了一批 TSX 与 regexp token。

这些更像编辑器高亮、预览或语法包重组信号。Windows 本输入没有对应语法模块变化。
