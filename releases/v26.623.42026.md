# Codex 桌面版 26.623.42026 macOS / Windows 对照

> 第三方跨平台变更日志 · 非官方

本批次显示两端共享的设置与 workbook 相关前端信号，同时 macOS 承载更多 AppGen、远程任务、更新恢复和插件详情重排。

One-liner: Cross-platform settings and workbook signals landed, while macOS carries broader AppGen, remote-task, update-recovery, and plugin-detail UI churn.

## 关于这份报告

本报告只使用输入事实包中的机器提取差异。macOS 对比范围是 `26.623.41415 build 4505` → `26.623.42026 build 4514`；Windows 对比范围是 `26.623.5175.0` → `26.623.5546.0`。两端版本号和打包节奏不一致，下面所有结论均分别引用各自平台事实。

证据等级分两档：【实证】来自 plist、manifest、依赖清单、插件 manifest、关键配置、人类可读 diff 和事实包统计；【信号】来自前端模块名、同名前端模块的字符串/样式差、压缩 bundle 体积方向。平台标记使用 `[macOS+Windows]`、`[仅 macOS]`、`[仅 Windows]`。

hash 命名变化、`webview/assets` 和 `.vite/build` 的构建产物重命名不作为功能增减。事实包未给出 CSP 变化区块，也未给出 Windows AppxManifest 变化明细，因此本文不对这些项做额外推断。

## ✨ 重点变化

### 【实证】[macOS+Windows] 内置插件 manifest 版本同步到 26.623.42026

macOS 的 `chrome`、`browser` 内置插件 manifest 版本从 `26.623.41415` 更新到 `26.623.42026`。Windows 的 `computer-use`、`chrome`、`browser` 内置插件 manifest 也出现同样版本更新。

这不是新增插件证据：事实包没有“新增内置插件明细”。可确认的是这些 bundled plugin 的 manifest 版本字段更新；其中 Windows 外层 MSIX 版本仍按自身事实包引用为 `26.623.5546.0`。

### 【实证】[macOS+Windows] Computer Use 运行时总量和 cua_node 依赖没有新增

macOS 的 Computer Use 运行时总量为 `377.4 MB → 377.4 MB (+0 MB, +0%)`；Windows 为 `298.9 MB → 298.9 MB (+0 MB, +0%)`。两端 `cua_node` 新增依赖包数量均为 `0`。

Windows 额外有 `computer-use` 插件 manifest 版本更新；macOS 本事实包只列出 `chrome` 与 `browser` 的插件 manifest 版本更新。

### 【信号】[macOS+Windows] 设置页新增权限、工作模式、语音和显示相关文案

两端 `general-settings` 同名前端模块都新增了默认权限禁用态、工作模式设置、代码基础字号、hold-to-dictate 热键、热键窗口捕获、听写词典条目、Popout Window projectless 默认设置、ambient suggestions、diff 色彩标记、Chrome theme contrast slider 等可读字符串。

这说明设置界面的控制面在前端包里发生了扩展信号，但由于这些证据来自同名前端模块改动线索，不能断言所有入口已经对用户开放。

### 【信号】[仅 macOS] macOS 设置侧还出现环境、终端、文件打开和 code review 默认项

macOS 的 `pets-settings~appearance-settings~general-settings` 模块新增 `Agent environment`、`Auto detect`、`Default file open destination`、`Default microphone input`、`Default permissions`、`Default terminal location`、`Default to projectless chat`、`Detached code review option label` 等字符串。

这些是 macOS 事实包独有信号；Windows 样本没有对应同组字符串。

### 【信号】[仅 Windows] 技能页新增创建、刷新、搜索和推荐/已安装分组文案

Windows 的 `skills-page` 模块新增创建新技能、重新加载技能列表、新安装技能需要刷新、过滤空状态、技能页标题、已安装技能、推荐技能、搜索输入和加载态等字符串。

macOS 事实包有包含 `skills-settings`、`plugins-settings` 的新增模块名组合，但没有同样的 `skills-page` 字符串明细，因此不能把 Windows 技能页文案直接套到 macOS。

### 【信号】[仅 macOS] 插件详情页新增分享、MCP 开关和工具菜单文案

macOS 的 `plugin-detail-page` 模块新增插件分享对话框、workspace 用户/群组搜索、MCP server enablement toggle、section navigation、skill card actions menu、app tools modal、skill preview modal 等字符串。

这是一组高可见插件管理信号，但仍属于同名前端模块线索；它证明文案进入 macOS 前端包，不证明具体权限或分享流已经完全启用。

### 【信号】[仅 Windows] Windows 插件详情页反向移除同类分享/MCP 文案

Windows 的 `plugin-detail-page` 模块移除了插件分享对话框、workspace 链接、MCP server enablement toggle、section navigation、`App unavailable` 等字符串，并且模块体积从约 `0.2 MB` 降到 `0.0 MB`，变化为 `-154755 bytes`。

这与 macOS 同批次的插件详情信号方向相反，应按平台分别阅读。

### 【信号】[仅 macOS] AppGen、站点分享和公开发布链路明显重排

macOS 新增了 `appgen-library-page`、`appgen-settings-page`、`appgen-publication-terms-route` 等相关模块组合。`appgen-share-dialog` 还新增站点分享保存、搜索 workspace 用户/群组、复制 live site URL、外部打开 live site、发布到 public internet、`Anyone on the Internet`、`Can visit`、`Copied to clipboard` 等字符串。

这些信号集中指向 AppGen 站点分享、链接访问和发布流程的前端重排。Windows 事实包没有对应新增模块或字符串。

### 【信号】[仅 macOS] 远程会话、Codex Cloud、最近任务和 Chrome 侧栏入口信号增加

macOS 的 `header` 模块新增 recent tasks menu、`All tasks`、`Cloud tasks`、`Codex for Chrome`、`Chrome computer use settings`、`App settings`、`Clear search` 等字符串。侧栏相关模块还新增 `All chats`、`Codex Cloud`、cloud tasks 分区、connection/project thread groups、recent threads 和 `features.remote_connections`。

项目 hover card 相关字符串也新增了打开项目来源、编辑项目、pin 项目、inline 项目名编辑、rename 失败 toast，以及会话运行在 Codex Cloud、本地 git worktree 或本地的状态文案。

### 【信号】[仅 Windows] Header 模块移除最近任务和应用设置相关文案

Windows 的 `header` 模块移除了 recent tasks menu、unread task updates、`All environments filter option`、`All tasks`、`App settings`、返回上一屏、Chrome extension side panel more actions、清除搜索等字符串。

这同样是 Windows 独有的同名前端模块变化信号，不能与 macOS 新增方向合并解读。

### 【信号】[仅 macOS] 更新恢复和用量限制入口进入前端包

macOS 的 `automations-page` 模块新增桌面应用更新下载、安装、带进度下载、确认/取消更新对话框、请求 workspace 月度用量限制提升、rate limit summary、available rate limit resets、`Learn more about rate limits` 等字符串。

另一个 macOS 模块新增 full-screen error page 上的 `Checking for updates`、`Downloading update`、`Installing update`、`Update Codex` 等恢复路径字符串。Windows 事实包没有同类新增字符串。

### 【信号】[macOS+Windows] Worktree 初始化失败后的本地环境兜底文案被移除

两端 `worktree-init-v2-page` 模块都移除了 `Continue anyway`、`Edit environment`、打开本地环境设置、启动 repair thread、取消 worktree setup 并启动 local conversation、worktree 创建取消、local task 创建错误等字符串。

这说明 worktree 初始化失败场景的前端文案或路径发生收缩/重排信号；由于证据来自同名前端模块线索，不能直接断言功能删除。

### 【信号】[仅 macOS] Electron worker bridge 与 process/spawn 错误路径进入前端包

macOS 的大型前端模块新增 `Electron worker bridge is unavailable`、`Mismatched worker response method`、`Missing cwd`、`Missing git metadata`、`Unknown method: process/spawn`、`operation not permitted`、`permission denied` 等字符串。

这些字符串属于命令执行、worker bridge 和权限错误路径的前端包信号。Windows 事实包没有对应新增组。

### 【信号】[macOS+Windows] Workbook 负载显著增加，macOS 还出现 notebook/docx preview 模块名

两端 `workbook` 模块都从约 `0.0 MB` 增至 `2.8 MB`，macOS 增加 `2764364 bytes`，Windows 增加 `2761257 bytes`。新增字符串包含 unnamed range、workbook help/valid values 提示，以及大量 `data-granola-*` 样式/结构类名。

macOS 还新增 `settings-page~projects-index-page~notebook-preview-panel~docx-preview-panel` 模块名。这些共同构成文档、表格或预览相关前端负载增强信号，但不等同于用户可见功能确认。

### 【信号】[仅 macOS] Profile 上传错误进入包，旧 profile/onboarding 负载大幅下降

macOS 的 `app-initial~app-main~onboarding-page~profile` 模块体积从 `6.1 MB` 降到 `0.0 MB`，变化为 `-6067006 bytes`，同时新增 `Profile photo upload failed` 字符串，并移除一批 diff/editor 相关字符串。

这是 profile/onboarding 区域打包结构变化信号，不应理解为 profile 功能消失。

### 【信号】[仅 Windows] Profile stats 查询字符串增加，同时移除 profile-model 模块

Windows 的 `profile-queries` 模块新增 profile streak day count、duration hours/minutes/seconds 等格式化字符串；真实前端模块增删列表中还列出 `profile-model` 被移除。

这是一组 Windows 独有的 profile 统计与数据模型打包变化信号。
