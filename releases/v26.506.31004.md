# Codex 桌面版 26.506.31004 macOS / Windows 对照

> 第三方跨平台变更日志 · 非官方

本批次在 macOS 侧体现为设置体系、浏览器使用、Hooks、工作树初始化和欢迎页的集中迭代；Windows 侧输入样本前后完全相同。

One-liner: macOS shows a settings-heavy UI and workflow update, while the Windows sample is byte-identical in this diff.

## 关于这份报告

本报告只基于官方签名分发包逆向重建后的机器提取事实。macOS 对比范围是 26.506.21252 build 2575 → 26.506.31004 build 2604；Windows 对比范围是 26.506.3484.0 → 26.506.3484.0。

证据等级分两档：【实证】来自 Info.plist、文件清单、依赖清单、app.asar 统计、签名验证、SHA-256、以及不属于“同名前端模块改动线索”的可读事实；【信号】来自同名前端模块的模块名、体积变化、可读字符串和样式类名差异，只说明包内出现了方向线索，不断言功能已对所有用户可见。

平台归属标记为：[macOS+Windows]、[仅 macOS]、[仅 Windows]。Windows 事实包中上一版和最新版的版本号、build、SHA-256、文件数、app.asar 内部文件数均相同，因此只能说“本输入样本在 Windows 平台无内容变化”。

## ✨ 重点变化

### 【实证】[仅 macOS] 版本与签名验证进入 26.506.31004 / build 2604

macOS 从 26.506.21252 build 2575 更新到 26.506.31004 build 2604。Info.plist 中 `CFBundleShortVersionString`、`CFBundleVersion` 和 `ElectronAsarIntegrity/Resources/app.asar/hash` 均发生变化。

两个 macOS 全量包均通过公证验证，来源为 Notarized Developer ID，签名主体为 OpenAI OpCo, LLC。事实包说明本次是历史回填，未使用 Sparkle 增量包，而是直接比较两个官方全量包。

### 【实证】[仅 Windows] Windows 输入样本没有内容变化

Windows 对比中的上一版与最新版均为 26.506.3484.0，两个全量包 SHA-256 相同，均为 `554ab8b534305bb26338c8b89dd745210625d52f78850ec7996c56772821e4ee`。

文件树统计为 815 → 815，新增 0、移除 0、内容改动 0；app.asar 内部文件为 1336 → 1336，新增 0、移除 0、改动 0。本报告不把 macOS 变化套用到 Windows。

### 【实证】[仅 macOS] app.asar 内容有小幅变化,但外层文件树保持稳定

macOS 外层 bundle 文件数保持 1238 → 1238，新增 0、移除 0、内容改动 54。`Contents/Resources/app.asar` 从 136.7 MB 到 136.8 MB，增加 0.04 MB。

app.asar 内部文件从 1377 到 1382，统计为新增 487、移除 482、改动 10。事实包明确提醒 webview/assets 下大量增减是 Vite 构建 hash 重命名，因此本报告只把去 hash 后仍存在的模块变化作为功能方向证据。

### 【信号】[仅 macOS] 设置体系被明显扩展: general settings、route state、hooks model 和 build flavor

去 hash 后新增前端模块包含 `get-build-flavor`、`hooks-settings-model`、`settings-route-state`、`use-plugins`；同名前端模块 `general-settings` 体积增加 98182 bytes。

`general-settings` 新增字符串覆盖 UI 字体大小、代码字号、Agent environment、默认权限开关、工作模式选项、dictation hotkey capture input、hotkey window hotkey capture input、ambient suggestions toggle、dictionary entry 等。新增样式状态类包含多组 `aria-*`、输入背景、列表 hover、边框、透明背景等，说明设置页在可访问性、输入控件和状态表达上有集中调整。

平台差异：这些设置侧信号只出现在 macOS 事实包；Windows 输入样本没有对应内容变化。

### 【信号】[仅 macOS] Browser Use 设置新增历史、缓存、截图注释和站点来源控制线索

`browser-use-settings` 增加 46103 bytes，新增字符串包括 `Access history without asking`、`Ask before accessing history`、`Always allow`、`Always ask`、`Always include`、`Annotation screenshots`、`Browser cache cleared`、`Aria label for browser use domain input`、移除单个网站 origin 的按钮标签等。

这组线索指向 Browser Use 的权限策略、历史访问策略、缓存清理反馈、截图注释以及按网站来源管理的设置界面变化。证据边界是前端同名模块改动，不能据此断言所有设置项已经稳定开放。

平台差异：仅 macOS 样本出现；Windows 样本无内容变化。

### 【信号】[仅 macOS] Hooks 设置围绕工具执行和对话压缩事件重组

`hooks-settings` 新增字符串包括 `Admin config`、`Before a tool executes`、`After a tool executes`、`Before Codex compacts the conversation`、`After Codex compacts the conversation`，以及 pre/post tool use、pre/post compact、permission request hook event 的描述文案。

按钮文案从 refresh hooks 变为 reload hooks，并新增 trust a hook 的按钮标签。这说明 Hooks 设置页在事件类型、管理员配置、信任动作和重新加载动作上出现了更细粒度的 UI 信号。

平台差异：仅 macOS 样本出现；Windows 样本无内容变化。

### 【信号】[仅 macOS] 工作树初始化流程增加失败恢复、取消与状态反馈文案

`worktree-init-v2-page` 新增字符串覆盖 worktree 创建中、为 fork conversation 创建 worktree、创建成功、创建失败、worktree 已创建但 renderer 未能启动 conversation、创建 local task 失败、Retry、Cancel、Edit environment、失败后打开本地环境设置等。

这组文案显示工作树初始化流程在错误恢复、状态展示、环境设置入口和取消后转为本地对话方面有明显调整信号。

平台差异：仅 macOS 样本出现；Windows 样本无内容变化。

### 【信号】[仅 macOS] 欢迎页出现“编码 / 日常工作”模式分流

`welcome-page` 新增字符串包括 `For coding`、`For everyday work`、`More technical responses and code detail`、`Same powerful agent, less technical detail`，并列出 `Build software`、`Analyze data`、`Design products`、`Manage inbox`、`Manage projects`、`Organize calendar` 等任务导向选项。

这说明欢迎页 v2 至少在包内出现了 coding mode 与 everyday work 选择的文案和任务分类信号。

平台差异：仅 macOS 样本出现；Windows 样本无内容变化。

### 【信号】[仅 macOS] 外部 agent 导入从单体模块转向步骤化模块

去 hash 后新增 `external-agent-import-step`，移除 `external-agent-import`。仅从模块名看，外部 agent 导入流程可能从单一入口拆成步骤化实现。

证据边界：这是模块级信号，没有配套可读字符串或服务入口事实支撑，因此不作为已完成能力面断言。

### 【信号】[仅 macOS] Shield check 与 in-app Browser Use 可用性检测进入前端模块集

去 hash 后新增 `shield-check` 与 `use-in-app-browser-use-availability`。前者暗示安全或拦截检查方向，后者暗示应用内 Browser Use 可用性判断。

证据边界：事实包只给出模块名，未给出相关可读字符串、权限或主进程入口，因此只作为低到中强度的方向信号。

### 【信号】[仅 macOS] Markdown 渲染或语法高亮资产明显收缩并替换

`markdown` 模块从 0.3 MB 降到 0.1 MB，减少 270477 bytes。新增字符串以 Markdown/HTML scope 名称为主，如 `heading.1.markdown` 到 `heading.6.markdown`、`comment.frontmatter`、`entity.name.tag.html`、`constant.other.reference.link.markdown`；移除字符串包含多组旧 HTML/XHTML DTD 片段和部分 DOM 序列化片段。

这更像 Markdown 解析、渲染或语法高亮相关包体替换信号，而不是可直接命名为用户功能的变化。

### 【信号】[仅 macOS] 插件详情页分享 / 启用相关界面线索被移除或重构

`plugin-detail-page` 体积减少 128040 bytes，移除字符串包括 workspace user 搜索与移除、plugin share dialog、MCP server enablement toggle、section navigation、`Add people`、`Add to Codex`、`Adding to Codex`、`Anyone at workspace` 等。

这表明插件详情页中与分享、添加、启用开关或导航相关的前端实现出现较大调整。由于这是移除线索，不能直接断言功能被删除；也可能是迁移到其他模块或重构。

## 🔎 低层信号索引

### 设置与可访问性

`general-settings`、`browser-use-settings`、`hooks-settings` 均出现大量 `aria-*` 标签、按钮说明和输入控件描述。证据边界是前端模块字符串和样式类名，说明包内 UI 可访问性文案和状态类进入或调整。

`src` 新增样式/状态类名 `pt.BROWSER_COMMENT_POPUP:return`，`browser-use-settings` 同时出现历史访问、缓存清理、截图注释和网站 origin 管理文案。证据边界：前者只是样式/状态类名信号，后者是同名前端模块可读字符串信号。

`src` 移除 `active-remote-project-id`、`active-workspace-roots`、`mac-menu-bar-enabled`、`persisted-atom-state`、`pinned-project-ids`、`pinned-thread-ids`、`primary-runtime-update-jitter-ms`、`project-order`、`projectless-thread-ids`、`selected-remote-host-id`、`windows-wsl-reminder-dismissed-date`、`worktree-auto-cleanup-enabled` 等类名或键名。证据边界：这是 bundle 差异中的名称线索，不等同于持久化格式已删除。

### 技能与本地图片内联

`use-skills` 体积减少 9746 bytes，移除 `Failed to inline local image`、`plugin marketplace request requires a marketplace source`、`plugin marketplace request requires one marketplace source`、`react.memo_cache_sentinel`。证据边界：这是同名前端模块移除字符串，说明相关错误路径或实现位置发生变化，不能断言技能功能移除。

### 菜单触发器和图标噪音

`more-menu-trigger` 增加 1215 bytes，新增 `http://www.w3.org/2000/svg`，移除 `aria-label`。这是孤立弱信号，阅读价值有限，可能只是图标或渲染实现变化。
