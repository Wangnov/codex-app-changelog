# Codex 桌面版 26.506.31004(macOS / Windows 对照)

> 第三方跨平台变更日志 · 非官方

本批次的核心结论是:macOS 版本有实质更新,Windows 对照包在本次对比中没有文件或应用层变化。  
One-liner: macOS moved forward with app-layer signals around settings, plugins, browser availability, and import flow, while the Windows package stayed unchanged.

## 关于这份报告

这份报告基于官方签名分发包的逆向 diff(差异比较):macOS 取 Sparkle(应用更新框架)全量包,Windows 取 MSIX(Windows 应用安装包)对照包。macOS 对比的是 `26.506.21252` build `2575` 到 `26.506.31004` build `2604`;Windows 对比的是 `26.506.3484.0` 到 `26.506.3484.0`。

可信度分两档:【实证】表示来自 plist(应用属性清单)、AppxManifest(Windows 应用清单)、依赖清单、类型声明、技能文档或可读包元数据等证据;【信号】表示只看到前端模块名或体积变化,能判断方向,但不能还原源码意图或确认用户可见功能。

平台归属标记含义如下:`[macOS+Windows]` 表示两平台都有对应变化;`[仅 macOS]` 表示只在 macOS 包中观察到;`[仅 Windows]` 表示只在 Windows 包中观察到。本批次最重要的平台差异是:macOS 包已经更新,Windows 包在同一批次对照中没有跟进内容变化。

## ✨ 重点变化

### macOS 升版,Windows 对照包未变化【实证】[仅 macOS]

macOS 的 `CFBundleShortVersionString` 从 `26.506.21252` 变为 `26.506.31004`,`CFBundleVersion` 从 `2575` 变为 `2604`,并且 `app.asar`(Electron 应用代码归档)完整性 hash(哈希值)发生变化。

Windows 对照包的版本保持 `26.506.3484.0`,两次包的 SHA-256(文件指纹)相同,文件数、内容改动数和 `app.asar` 内部文件数都没有变化。这意味着本批次可观察的新内容集中在 macOS 包,Windows 侧没有对应增量。

### 设置、插件与运行环境判断出现新前端模块【信号】[仅 macOS]

macOS 的应用层新增了这些非 hash 噪音模块:`external-agent-import-step`、`get-build-flavor`、`hooks-settings-model`、`settings-route-state`、`shield-check`、`use-in-app-browser-use-availability`、`use-plugins`。

从模块名看,这些信号指向外部 agent(智能体)导入步骤、构建 flavor(构建变体)识别、设置路由状态、插件使用、内置浏览器可用性判断以及 shield 检查。但这些只是模块名层面的信号,不能据此断言具体功能已经对用户开放。Windows 的 `app.asar` 内部文件为 `1336 → 1336`,新增、移除、改动均为 `0`,没有看到对应模块变化。

### 旧导入与聊天相关模块被替换或移除【信号】[仅 macOS]

macOS 去掉 hash 噪音后,移除了 `chats` 与 `external-agent-import` 两个前端模块。结合新增的 `external-agent-import-step`,可以看出外部 agent 导入相关前端结构发生了调整;但由于只有模块名,不能确认具体交互或业务逻辑如何变化。

Windows 侧没有应用层文件变化,因此没有观察到同样的前端模块替换。

### 浏览器插件脚本有极小体积变化【信号】[仅 macOS]

macOS 包内 `plugins/openai-bundled/plugins/chrome/scripts/browser-client.mjs` 与 `plugins/openai-bundled/plugins/browser-use/scripts/browser-client.mjs` 均出现极小体积变化,事实包记录为 `+0.00 MB` 级别。由于只有路径和体积变化,这只能说明浏览器相关脚本有内容差异,不能推断具体功能变化。

Windows 侧文件树内容改动为 `0`,没有对应变化。

### cua_node 依赖没有新增【实证】[macOS+Windows]

两平台事实包都显示 `cua_node` 依赖新增包数量为 `0`。这说明本批次没有观察到 `cua_node` 依赖层面的新增包变化。
