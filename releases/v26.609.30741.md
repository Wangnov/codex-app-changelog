# Codex 桌面版 `26.609.30741`(build 3808)

> 第三方变更日志 · 非官方 · 基于官方更新包逆向差异
> One-line: This release massively expands Codex's Computer Use runtime (PDF / OCR / image processing / browser automation) and lays groundwork for a Dock-tile plugin, multi-brand icons, and map integration.

**对比范围:** `26.608.12217`(build 3722,2026-06-09)→ `26.609.30741`(build 3808,2026-06-11) · macOS arm64

---

## 关于这份报告

这不是官方变更日志,而是一份**基于官方分发产物的逆向差异报告**。官方对桌面版很多小版本并不发布 release notes,即便发布,粒度也远不到文件/依赖/权限这一层。本项目的目标是把官方"沉默"的那部分变化挖出来。

**数据是怎么来的:**

1. 从 OpenAI 官方 Sparkle 更新源取上一版全量包,以及"最新版相对上一版"的官方增量包(delta)。
2. 用官方 Sparkle 公钥做 **EdDSA 签名校验**——证明拿到的是 OpenAI 签名过的原始字节,而不只是 HTTPS 传输可信。
3. 用 `BinaryDelta` 把上一版重建成最新版,再对新旧两个 `.app` 做结构化差异:文件树、`Info.plist`、`app.asar`、原生模块、依赖清单、技能文档、内容安全策略等。
4. 重建产物经 `codesign --deep --strict` 与 `spctl` 验证为 **OpenAI 公证签名**(`accepted / Notarized Developer ID / OpenAI OpCo, LLC, Team 2DC432GLL2`),确保我们 diff 的就是用户实际会装到的那个包。

**证据等级**(每条变化都会标注):

- **【实证】** 有人类可读的直接证据:`Info.plist` 键值、TypeScript 类型声明、技能 Markdown、`package.json`、CSP 文本、包清单。
- **【信号】** 只有间接证据:前端代码块(chunk)文件名、二进制体积变化。指向某个方向,但在没有 source map 的前提下**不能还原到源码级意图**。

## ✨ 重点变化

### 1. Computer Use 运行时大扩容:补齐文档 / OCR / 图像 / 浏览器自动化 【实证】

这是本版分量最重的变化。`Computer Use`(让模型操作桌面/浏览器的能力)所依赖的独立运行时 `cua_node` 体积从 **约 213 MB 暴增到约 377 MB**(+164 MB,+77%),一次性**新增 29 个 npm 包**,无移除。按能力归类:

- **文档解析** — `pdfjs-dist`(Mozilla PDF.js)。
- **OCR 光学字符识别** — `tesseract.js` + `tesseract.js-core`(含多个 WebAssembly 变体:基础 / SIMD / relaxed-SIMD / LSTM)。
- **图像处理与渲染** — `sharp` 及其 darwin-arm64 预编译产物、`@napi-rs/canvas`(Skia 后端 Canvas)、`@img/colour`。
- **图像编解码** — `jpeg-js`、`pngjs`、`bmp-js`(纯 JS 编解码)。
- **像素级对比** — `pixelmatch`(截图比对 / 视觉回归)。
- **浏览器自动化** — `playwright` + `playwright-core`(1.57.0)。
- **支撑库** — `zlibjs`、`semver`、`node-fetch`、`whatwg-url`、`tr46`、`webidl-conversions`、`is-url`、`idb-keyval`、`detect-libc`、`fsevents`、`wasm-feature-detect`、`regenerator-runtime` 等。

这组依赖合在一起,清晰指向同一个方向:让 Computer Use agent 具备**读 PDF、识别图片里的文字、处理与比对图像、驱动真实浏览器**的能力。对用户而言,这通常意味着"看文档截图 / 网页 / 扫描件并据此操作"的可靠性会明显提升。

### 2. Computer Use 截图:鼠标指针尺寸可调 【实证】

`@oai/sky`(Computer Use 的底层桌面控制引擎)从 PR 预发布版 `0.4.8-202606020558-pr-981870` 升到正式版 **`0.4.10`**,full-desktop 接口新增一个选项:

```ts
/** Resize mouse pointer in screenshots to this size in pixels. Set 0 to disable. (default: 12) */
mouse_size_px?: number;
```

即在喂给模型的截图里,把鼠标指针重绘成指定像素大小(默认 12px,设 0 关闭)。目的是让模型更容易看清"鼠标现在在哪",提升点击定位准确率。同时 `@oai/sky` 的 Linux 引擎二进制大幅增重(`sky_linux_x64` 1.25 MB → 2.91 MB,arm64 1.25 MB → 2.77 MB),说明 Linux 全桌面控制能力也在同步增强。

### 3. 前端开始允许连接地图服务(Mapbox)【实证】

两个 webview 页面(`index.html` 与 `avatar-overlay-composition-surface.html`)的内容安全策略(CSP)里,`connect-src` 新增放行:

```
https://api.mapbox.com   https://events.mapbox.com
```

CSP 是前端能联哪些域名的硬性白名单。Mapbox 被加进来,意味着新版前端**已具备调用地图服务的前置条件**。具体哪个界面会用到地图,从产物层面还看不到,但基础设施已经铺好。

### 4. 新增 Dock 磁贴插件 `CodexDockTilePlugin` 【实证】

`Contents/PlugIns/` 下出现一个上一版完全没有的全新组件 **`CodexDockTilePlugin.plugin`**(标识符 `com.openai.codex.dock-tile-plugin`,已由 OpenAI Developer ID 签名),同时 `Info.plist` 新增 `NSDockTilePlugIn` 声明指向它。

`NSDockTilePlugIn` 是 macOS 的标准机制,允许 App **未运行时**也能更新 Dock 图标(画角标、进度、状态)。这通常用于"后台任务跑完了在 Dock 上提示""长任务进度显示"之类的体验。

### 5. 多品牌图标基础设施 【实证】

一组围绕"按品牌切换图标"的改动同时出现:

- `Info.plist` 新增 `CodexAppIconBaseName = icon`。
- `app.asar` 的 `package.json` 新增 `codexAppBrand = "codex"`。
- 资源目录新增图标 `icon.png`、`icon-dev-light.png`、`icon-dev-dark.png`(开发版浅色/深色),前端也新增了 `iconResolver`、`gradient` 等模块。

合起来看,这是在铺设**正式版 / 开发版 / ChatGPT 等不同品牌共用一套代码、运行时切换图标**的能力。对普通用户暂不可见,但解释了"为什么包里突然多了几个图标"。

### 6. Sites 技能重构:职责收缩,以 connector 为准 【实证】

内置的 `sites`(站点托管)技能文档有实质改写:

- **`sites-hosting`** 从"技能内自带一整套创建项目 / 发版 / 部署 / 访问控制流程",**收缩为只负责本地构建产物准备**,并明确"以 Sites connector 的工具描述为唯一真相来源"。原本内嵌的 Project Resolution、Creating a Project、Version Publishing、Production Deployment、Access Control 等整节被删除。`project_id` 的语义也改为"`create_site` 返回的持久标识符,创建前不要填"。
- **`sites-building`** 新增 Cloudflare D1(边缘数据库)的预编译语句(prepared statement)规范:用 `prepare().run()` 而非 `exec()` 跑建表;一次 `prepare()` 只传一条语句;多条语句用 `batch([...])`。

方向上,这是把"部署编排"的权威从写死在技能里挪到 connector 协议中,技能只做本地准备——意味着后续部署行为的调整可以不必跟着客户端版本走。

### 7. 前端新功能模块信号 【信号】

去掉 Vite 构建产生的 hash 文件名噪音后,前端真实**新增 19 个代码模块、移除 4 个**。模块名未经混淆,可以反推方向(但出现 ≠ 已对用户开放,可能仍在功能开关后):

- **语音 / 音频** — `transcribe-audio`、`use-realtime-waveform`(音频转写 + 实时波形,指向语音输入相关能力)。
- **操作审批** — `automatic-approval-review`(agent 操作的自动批准审查流)。
- **标注** — `annotation-mode-button`(标注模式)。
- **多窗口 / 标签** — `windows-tabs-open`、`windows-tabs-open-schema`、`run-app-action-in-primary-window`。
- **应用生成** — `appgen-gating`(appgen 功能门控)。
- **插件 / 扩展** — `plugin-detail-queries`、`extension-info`。
- **其他** — `interrupted-turn-state`(中断回合状态)、`thread-side-panel-active-signals`、`announcement-atoms` / `workspace-message-announcement-state`(工作区公告)。

移除:`browser-sidebar-prompt-context`、`process-manager-target`、`window-app-action-helpers`、`xterm-display-helpers`。

### 8. 底层运行时与权限文案 【实证 + 信号】

- **主二进制** `Contents/Resources/codex` 206.4 MB → 213.4 MB(+7.0 MB);**应用包** `app.asar` 145.8 MB → 150.7 MB(+4.9 MB)。【信号】底层逻辑有更新,但仅凭压缩产物体积无法安全还原具体改动。
- **浏览器自动化栈** — Chrome 扩展宿主 `extension-host` +88 KB,`browser-client.mjs` +4.8 KB(browser 与 chrome 两份插件同步)。【信号】
- **权限说明文案** — Apple Events / 摄像头 / 麦克风三条用途描述去掉了句尾句号,**语义不变**,纯文案微调。【实证】
