# Codex 桌面版 `26.609.30741`(build 3808)

> 第三方变更日志 · 非官方 · 基于官方更新包逆向差异

这一版的主线是扩充 Computer Use(让模型操作桌面 / 浏览器的能力)运行时能力，同时调整 Sites 工作流文档、Dock 集成和若干前端功能方向信号。  
One-liner: Codex Desktop 26.609.30741 expands the local Computer Use runtime and tightens Sites publishing guidance.

## 关于这份报告

这不是官方变更日志，而是一份**基于官方分发产物的逆向差异报告**。方法是取官方 Sparkle 上一版全量包与官方增量包，经 EdDSA 验签后用 BinaryDelta 重建最新版 bundle，再做结构化文件、配置、依赖和可读文本 diff。

可信度方面，上一版全量包与官方增量包均通过官方 Sparkle 公钥 EdDSA 验签；重建出的 `Codex.app` 通过 `codesign --deep --strict`，并由 `spctl` 判定为 `accepted`，来源为 `Notarized Developer ID`，签名主体为 `Developer ID Application: OpenAI OpCo, LLC (2DC432GLL2)`。

证据等级分两档：**【实证】**表示有 Info.plist、CSP、技能文档、类型声明、package.json、依赖清单或签名 identifier 等直接可读证据；**【信号】**表示仅来自前端模块名或二进制体积变化，指向某方向，但无法从产物还原到源码级意图。

对比范围：`26.608.12217` build `3722`，发布于 `2026-06-09`，到 `26.609.30741` build `3808`，发布于 `2026-06-11`，平台为 macOS arm64。

## ✨ 重点变化

### 1. Computer Use 运行时补齐 PDF、OCR、图像处理与浏览器自动化依赖 【实证】

`cua_node` 新增 29 个 npm 包，覆盖 PDF 解析、OCR、图像编解码 / 处理、Canvas、像素级比较和浏览器自动化等方向。最直接的变化是运行时从 `213.5 MB` 增至 `365.8 MB`，增加 `152 MB`，增幅 `71%`。

这意味着 Codex 桌面版随包携带的本地 Computer Use 能力边界明显扩展：`pdfjs-dist` 指向 PDF 读取，`tesseract.js` / `tesseract.js-core` 指向 OCR，`sharp` / `@img/sharp-libvips-darwin-arm64` / `@napi-rs/canvas` 指向图像处理和绘制，`playwright` / `playwright-core` 指向浏览器自动化。

关键证据：新增依赖包含 `pdfjs-dist@5.4.624`、`tesseract.js@7.0.0`、`sharp@0.34.5`、`@napi-rs/canvas@0.1.91`、`playwright@1.57.0`。

### 2. `@oai/sky` 从预发布构建升至 `0.4.10`，截图鼠标尺寸变为可配置 【实证】

Computer Use 里的 `@oai/sky` 从 `0.4.8-202606020558-pr-981870-f570e0604110` 更新到 `0.4.10`。类型声明新增 `mouse_size_px?: number`，说明 Linux 目标截图中的鼠标指针尺寸可以通过像素值调整，`0` 表示关闭，默认值为 `12`。

可感知点在于截图输出更可控，尤其是依赖视觉定位或对比的自动化流程。多个 Linux 操作封装也开始把 `options` 传入底层 `sky_linux` 调用，包括 click、drag、scroll、move、type_text、press_key 和 get_screenshot。

关键证据：`Options.d.ts` 新增 `mouse_size_px?: number`；`sky_linux.js` 新增 `--mouse-size-px` 参数拼接；`@oai/sky/package.json` 版本变为 `0.4.10`。

### 3. macOS Computer Use 服务路径改由环境变量读取 【实证】

macOS 目标的 Computer Use 客户端调整了服务 App 路径解析：从读取 `globalThis.nodeRepl.computerUse.serviceAppPath`，改为读取 `globalThis.nodeRepl.env.SKY_CUA_SERVICE_PATH`，找不到时仍回退到包内 `Codex Computer Use.app` 的路径推断。

这意味着本地或调试环境可以通过 `SKY_CUA_SERVICE_PATH` 指定 Computer Use 服务位置；从产物可确认的是读取入口变化，不能进一步判断外部配置策略。

关键证据：`mac/client.js` 中路径来源变为 `nodeRepl.env.SKY_CUA_SERVICE_PATH`。

### 4. 新增 Dock Tile 插件与多套图标资源 【实证】

`Info.plist` 新增 `NSDockTilePlugIn = CodexDockTilePlugin.plugin`，bundle 内新增 `Contents/PlugIns/CodexDockTilePlugin.plugin`，签名 identifier 为 `com.openai.codex.dock-tile-plugin`，签名主体为 `Developer ID Application: OpenAI OpCo, LLC (2DC432GLL2)`。

这表明应用开始随包提供 macOS Dock Tile 插件机制。新增资源还包括 `icon.png`、`icon-dev-dark.png`、`icon-chatgpt.png`、`icon-dev-light.png`；`package.json` 新增 `codexAppBrand: "codex"`，`Info.plist` 新增 `CodexAppIconBaseName = icon`。

关键证据：`NSDockTilePlugIn`、`CodexDockTilePlugin.plugin`、`codexAppBrand = codex`、`CodexAppIconBaseName = icon`。

### 5. Sites 插件从 `0.1.14` 升至 `0.1.15`，托管流程更依赖 connector 作为事实源 【实证】

Sites 插件版本从 `0.1.14` 升至 `0.1.15`。`sites-hosting` 技能文档大幅改写：定位从“创建或复用 hosting 配置、保存版本、部署、检查状态、管理权限”收敛为“本地准备可部署的 Sites 源码和构建产物”，并明确 Sites connector 和 tool descriptions 是站点创建、环境变量、源码凭据、版本、部署和访问控制的事实源。

这意味着 Codex 在 Sites 工作流里的本地职责被写得更清楚：本地验证、准备 source、准备 artifact；远端项目、版本、部署、环境变量和权限管理则按 connector 描述执行。

关键证据：`sites-hosting/SKILL.md` 新文本写明 “Use this skill for local preparation...” 和 “Use the Sites connector and tool descriptions as the source of truth...”。

### 6. Sites 建站文档新增 Cloudflare D1 prepared statement 约束 【实证】

`sites-building` 技能文档新增 D1 使用建议：应用查询和运行时初始化应使用 raw D1 binding 的 prepared statements；多行单条 SQL 可用 `prepare(...).run()`；每个 `prepare()` 只传入一条 SQL；多语句操作使用 `batch([...])`。

这对开发者可感知的影响是，Codex 在生成或修改 D1 相关 Sites 项目时，会更倾向于可预编译、单语句、可批处理的 SQL 结构，并避免把 newline-separated 输入交给 `exec()`。

关键证据：`sites-building/SKILL.md` 新增 `await env.DB.prepare(schemaSql).run()`、`do not combine multiple semicolon-delimited statements`、`d1.batch([...])` 示例。

### 7. Webview CSP 新增 Mapbox 连接域名 【实证】

两个 webview 页面在 `connect-src` 中新增 `https://api.mapbox.com` 和 `https://events.mapbox.com`：`webview/index.html` 与 `webview/avatar-overlay-composition-surface.html` 均有相同放行。

这说明前端运行环境新增了对 Mapbox API 与事件端点的网络连接许可。仅凭 CSP 不能确认具体 UI 功能已开放，但这是直接的运行时权限变化。

关键证据：`connect-src` 新增 `https://api.mapbox.com`、`https://events.mapbox.com`。

### 8. 前端出现音频、审批、插件详情、窗口标签和公告等方向信号 【信号】

去掉 Vite hash 噪音后，新增前端模块名包括 `transcribe-audio`、`use-realtime-waveform`、`automatic-approval-review`、`plugin-detail-queries`、`windows-tabs-open`、`announcement-atoms`、`workspace-message-announcement-state`、`appgen-gating` 等。

这些模块名指向音频转写 / 实时波形、自动审批复核、插件详情查询、窗口标签状态、公告状态和 app generation gating 等方向，但这只是模块名信号，无法从产物还原到源码级意图，也不代表这些功能已对用户开放。

关键证据：前端真实新增模块列表中出现上述未混淆模块名。
