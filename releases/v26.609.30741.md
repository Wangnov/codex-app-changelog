# Codex 桌面版 `26.609.30741` build 3808

> 第三方变更日志 · 非官方 · 基于官方更新包逆向差异

本版最明显的变化集中在 Computer Use 运行时扩容、macOS Dock/图标能力、Sites 插件指导和若干工作区 UI 方向信号。  
One-liner: Computer Use gets a much larger local runtime, while Dock, Mapbox, Sites, and workspace UI surfaces show new capability hooks.

## 关于这份报告

这不是官方变更日志，而是一份**基于官方分发产物的逆向差异报告**。本次对比范围是 macOS arm64 的 `26.608.12217` build `3722`，发布于 `2026-06-09`，到 `26.609.30741` build `3808`，发布于 `2026-06-11`。

本版属于历史回填：官方 appcast 对这一对版本没有提供增量包，因此事实包采用两个官方全量包直接比较，并给出 SHA-256、公证验证和 bundle 结构化差异。

证据等级分两档：**【实证】**表示有可读配置、依赖、插件清单、技能文档、CSP、Info.plist、签名或新增字符串等直接证据；**【信号】**表示只来自前端模块名、同名前端模块差异或体积变化，指向某方向，但无法从产物还原到源码级意图。

## ✨ 重点变化

### 1. Computer Use 运行时明显扩容，补齐 PDF、OCR、图像处理和浏览器自动化依赖 【实证】

`cua_node` 总量从 `213.5 MB` 增至 `365.8 MB`，增加 `152 MB`，增幅 `71%`。新增依赖包含 `pdfjs-dist@5.4.624`、`tesseract.js@7.0.0`、`tesseract.js-core@7.0.0`、`sharp@0.34.5`、`@napi-rs/canvas@0.1.91`、`playwright@1.57.0`、`playwright-core@1.57.0`、`pixelmatch@7.1.0`、`pngjs@7.0.0`、`jpeg-js@0.4.4` 和 `bmp-js@0.1.0`。

这意味着本地 Computer Use 运行时在读取 PDF、OCR、截图/图像处理、像素比对和浏览器自动化方向具备了更完整的依赖基础；这些依赖确实存在于产物中，但是否对所有产品入口开放不能仅由依赖清单判断。

### 2. `@oai/sky` 升级，并为 Linux Computer Use 动作统一传递 options 【实证】

`@oai/sky` 从 `0.4.8-202606020558-pr-981870-f570e0604110` 升级到 `0.4.10`。`Options.d.ts` 新增 `mouse_size_px?: number`，注释说明可把截图中的鼠标指针缩放到指定像素大小；`sky_linux.js` 新增 `--mouse-size-px` 参数传递，`click`、`drag`、`move`、`scroll`、`press_key`、`type_text` 和 `get_screenshot` 都开始向 `sky_linux` 传入 `options`。

开发者可感知点是 Linux Computer Use 的动作执行和截图路径现在共享同一组选项，截图中的鼠标指针尺寸也有了显式配置入口。另一个接口收紧是 `to_data_url(bytes, mime_type)` 中的 `mime_type` 从可选变为必填。

### 3. 新增 macOS Dock 磁贴插件和图标资源 【实证】

`Info.plist` 新增 `NSDockTilePlugIn = CodexDockTilePlugin.plugin`，bundle 中新增 `Contents/PlugIns/CodexDockTilePlugin.plugin`，签名 identifier 为 `com.openai.codex.dock-tile-plugin`，签名 authority 为 `Developer ID Application: OpenAI OpCo, LLC (2DC432GLL2)`。同时新增 `CodexAppIconBaseName = icon`，`package.json` 新增 `codexAppBrand = codex`，资源中出现 `icon.png`、`icon-dev-dark.png`、`icon-chatgpt.png` 和 `icon-dev-light.png`。

这些证据说明本版加入了 Dock 磁贴插件和多套图标资源，并出现 `Preferred macOS Dock icon` 字符串。实际 UI 如何暴露 Dock 图标选择或 Dock 状态，不能从这些配置单独断言。

### 4. Sites 插件更新到 `0.1.15`，技能文档收紧 D1 与发布流程 【实证】

`sites` 插件从 `0.1.14` 升级到 `0.1.15`，长描述把 `games` 加入可构建网站类型。`sites-building/SKILL.md` 新增 D1 使用约束：应用查询和运行时初始化应使用 raw D1 binding 的 prepared statements，单个 `prepare()` 只传一个 SQL statement，多语句操作使用 `batch([...])`。

`sites-hosting/SKILL.md` 的定位也发生变化：从直接描述创建、保存版本、部署、访问控制等操作，改为强调本地准备 deployable Sites source 和 build artifacts，并把 site creation、environment variables、versions、deployments、access control 的 source of truth 指向 Sites connector 和 tool descriptions。对开发者而言，这会影响 Sites 任务的默认执行路径和数据库迁移/部署前准备方式。

### 5. WebView CSP 新增 Mapbox 连接放行 【实证】

`webview/index.html` 和 `webview/avatar-overlay-composition-surface.html` 的 `connect-src` 新增 `https://api.mapbox.com` 与 `https://events.mapbox.com`。

这表示主 WebView 和 avatar overlay composition surface 现在允许连接 Mapbox API 与事件端点。事实包没有给出具体 UI，因此不能进一步断言地图功能已经对用户开放。

### 6. 主进程/前端 bundle 出现浏览器资料导入、终端识别和 Git 提示相关字符串 【实证】

新版 bundle 中新增可读字符串包括 `Browser profile is no longer importable`、`Owl browser profile importer is unavailable`、`Cookie domain selection requires cookie import`、`Select cookies, passwords, or both to import`、`fluent terminal`、`git bash`、`tera term`、`windows powershell`、`windows terminal`，以及 `Using the supplied git context below, generate a single-line git commit message.`。

这些字符串确凿存在于压缩产物中，可判断新增或调整了浏览器资料/ Cookie 导入、终端类型识别和单行 Git commit message 生成相关路径；但由于仍是压缩 bundle，不能还原完整源码语义。

### 7. Thread 页面底部面板、侧边面板和标签动作在增强 【信号】

新增真实前端模块包括 `thread-side-panel-active-signals`、`windows-tabs-open`、`windows-tabs-open-schema` 和 `run-app-action-in-primary-window`。同名前端模块 `thread-app-shell-chrome` 新增了打开 side chat、browser tab、review tab、terminal、timeline tab、workspace file browser、bottom panel 的可读字符串；`thread-page-bottom-panel-state` 体积增加 `293231 bytes`，并出现 terminal crash reload、xterm buffer、mouse events、accessibility 等字符串。

这些线索指向 Thread 页面内的底部/侧边面板、标签页动作和嵌入终端体验正在扩展，但它们属于前端模块和字符串信号，无法从产物还原到源码级意图。

### 8. 插件详情、分享和 MCP mention 范围出现扩展迹象 【信号】

新增模块 `plugin-detail-queries`，`plugin-detail-page` 新增 `Add people or groups`、`Anyone in this workspace with the link`、`Accessible label for the MCP server enablement toggle on the plugin detail page`、skill preview modal 和 skill card actions menu 相关字符串。`at-mention-list` 新增 `Computer use`、`Custom agents`、`Desktop apps`、`Live agents`、`MCP servers`、`Mac apps` 和 `mcp-capability-mentions` 相关类名。

这些线索指向插件详情页、工作区分享、MCP server 启用和 mention 范围在扩展，但仍只能作为 UI/交互方向信号。

### 9. 语音、听写和设置项出现新入口线索 【信号】

新增模块包含 `transcribe-audio`、`use-realtime-waveform` 和 `phone`。`general-settings` 新增 `Aria label for hold-to-dictate hotkey capture input`、`Aria label for one dictation dictionary entry`、`Adjust the base size used for code across chats and diffs`、`Adjust the base size used for the Codex UI`、`Accessible label for work mode settings options`、`ambient suggestions toggle` 和 `Popout Window projectless default setting toggle` 等字符串。

这些变化指向语音/听写、实时波形、字体基准尺寸、work mode、ambient suggestions 和 Popout Window 默认行为的设置面正在扩展，但无法从这些前端线索断言具体发布状态。

### 10. 反馈、举报、工具活动摘要和工作区通知文案发生变化 【信号】

`plan-summary-item-content` 新增 Codex thread report reason、external report form、report submission failure、artifact incorrect content/topic coverage/template 等字符串，并移除部分旧 artifact feedback 和 legal notice 字符串。`local-conversation-thread` 新增 code search、file read、loaded skill definitions、named MCP tool calls、running integrations 等 collapsed tool activity summary 字符串，同时移除 auto-review approved/denied 相关文案。`app-main` 新增 `Workspace announcement`、`Unread chats`、`Notify owner`，并移除 `Introducing Codex mobile` 等移动端 announcement 文案。

这些线索指向反馈/举报、工具活动摘要和工作区通知体验在调整，但同名前端模块改动只能标为信号。

### 11. macOS 权限提示文案做了标点清理 【实证】

`NSAppleEventsUsageDescription`、`NSCameraUsageDescription` 和 `NSMicrophoneUsageDescription` 的英文说明都移除了句末句点，内容本身保持同义：Apple Events 用于代表用户控制 Mac apps，Camera 用于 video input，Microphone 用于 voice input。

这是用户可能在系统权限弹窗中看到的轻微文案变化；事实包没有显示权限范围扩大。
