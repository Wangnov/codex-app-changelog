# Codex 桌面版 26.623.31921 macOS / Windows 对照

> 第三方跨平台变更日志 · 非官方

这批输入显示，macOS 从 26.623.31443 升至 26.623.31921，Windows 从 26.623.3763.0 升至 26.623.4041.0；两端包内版本、分包与 UI 线索并不同步，不能互相代套。

One-liner: Codex Desktop 26.623.31921 lines up bundled browser plugins while macOS moves toward usage/runtime/model settings, and Windows surfaces plugin sharing, AppGen sharing, debug, RPC and mention-entry signals.

证据等级分两档：【实证】来自 plist、manifest、插件清单、服务入口、依赖清单、技能或配置文档、可读字符串和关键文件 diff；【信号】来自前端模块名、体积变化、样式类名和同名 bundle 的方向性线索。平台标记为 [macOS+Windows]、[仅 macOS]、[仅 Windows]。

对比范围分别是 macOS arm64 的 26.623.31443 build 4441 → 26.623.31921 build 4452，以及 Windows x64 MSIX 的 26.623.3763.0 → 26.623.4041.0。Windows 事实包中的插件版本 diff 使用 26.623.31443 → 26.623.31921 字样，这是包内插件 manifest 的实际差异，不等同于 Windows 外层 MSIX 版本号。

## ✨ 重点变化

### 【实证】[macOS+Windows] Browser 与 Chrome 内置插件版本同步到 26.623.31921

两平台的 `browser` 与 `chrome` 内置插件 manifest 都把版本从 `26.623.31443` 更新到 `26.623.31921`。插件描述本身没有显示语义变化：Browser 仍指向 Codex in-app browser、localhost/file URL、截图和点击输入等场景；Chrome 仍用于依赖用户现有 Chrome 状态的自动化。

平台差异是 Windows 还出现了 `computer-use` 插件 manifest 的版本更新，而 macOS 事实包在关键文件 diff 中只列出 Browser 和 Chrome。

### 【实证】[仅 Windows] Computer Use 插件 manifest 跟随升级

Windows 包内 `resources/plugins/openai-bundled/plugins/computer-use/.codex-plugin/plugin.json` 将 `computer-use` 插件版本从 `26.623.31443` 改为 `26.623.31921`，描述为从 Codex 控制 Windows 桌面应用。

这是一条 Windows 独有的 manifest 证据；macOS 输入没有同等的 `computer-use` manifest diff。运行时总量方面，Windows 的 Computer Use 运行时为 298.9 MB → 298.9 MB，macOS 为 377.4 MB → 377.4 MB，二者都没有体积增长。

### 【实证】[仅 macOS] 应用版本、build 与 app.asar 完整性 hash 更新

macOS `Info.plist` 显示 `CFBundleShortVersionString` 从 `26.623.31443` 到 `26.623.31921`，`CFBundleVersion` 从 `4441` 到 `4452`，`ElectronAsarIntegrity/Resources/app.asar/hash` 也随之变化。

Windows 输入没有提供 AppxManifest 变化区块；只能引用其版本区块里的外层版本 `26.623.3763.0` → `26.623.4041.0`。

### 【信号】[仅 macOS] 配额、充值与用量上限 UI 进入多个前端路径

macOS 的同名前端模块中新增了 `5 hour usage limit`、`Add credits`、`Archived chats`、Codex usage modal、rate limit reset modal、credit reload dialog、minimum balance input、monthly reload spend limit input 等字符串，还在 automations 相关模块出现 `Button to request a workspace monthly usage limit increase` 与 `https://chatgpt.com/gifts/credits`。

这些是前端 bundle 中的可读字符串和样式线索，说明配额、充值、礼品 credits、workspace 月度用量上限申请等 UI 文案进入 macOS 包内。Windows 事实包没有对应新增；不能据此断言 Windows 没有这些能力，只能说本输入样本中未见同类新增证据。

### 【信号】[仅 macOS] 桌面应用更新与 primary runtime 安装状态文案进入自动化页面相关分包

macOS `automations-page` 新增了桌面更新按钮文案：下载中、带进度下载、安装中、确认更新、取消更新，以及 `Codex runtime install canceled`、`Codex runtime installed`、`Error installing primary runtime`、`Latest Codex runtime is already downloaded`。

这些字符串是包内事实，但位于同名前端模块改动线索中，按规则归为【信号】。Windows 本批没有同类新增，相反其 `worktree-init-v2-page` 移除了若干本地环境失败后的继续、编辑环境和修复线程文案。

### 【信号】[仅 macOS] 模型与 reasoning effort 设置开始连接 `config.toml`

macOS `home-ambient-suggestions-content` 新增 `Open config.toml`、`Open config.toml in WSL environment`、`Setting default model and reasoning effort`、`Failed to update model and reasoning effort`、`Model settings host is unavailable`、`composer.modelSettings.errorConfigValidation` 等字符串。

这说明包内出现了默认模型和 reasoning effort 设置、配置校验失败、模型设置 host 不可用、打开 `config.toml` 与 WSL 环境下打开配置文件的 UI 路径。虽然出现了 WSL 文案，但证据来自 macOS 前端 bundle，不能外推为 Windows 当前可见行为。

### 【信号】[仅 macOS] 外部 agent 配置迁移项扩展到 commands、hooks、sessions、skills、subagents 等

macOS `appearance-settings~general-settings` 新增了外部 agent config migration 相关标签，包括 MCP server config、commands、config、hooks、instruction file、plugins、sessions、skills、subagents，以及 `externalAgentConfig.itemType.agentsMd` 等键名。

### 【信号】[仅 macOS] 内嵌浏览器设置页扩展到历史、下载、密码、扩展、站点和数据清理选项

Windows 同名模块本批是反向变化：这些字符串在 Windows 输入中被移除。因此这不是跨平台同步新增，而是两平台样本之间方向相反的浏览器设置线索。

### 【信号】[仅 Windows] 插件详情页获得分享、MCP server 开关与技能菜单相关文案

### 【信号】[仅 Windows] AppGen 站点分享对话框新增公开发布、复制链接和外部打开文案

### 【信号】[macOS+Windows] 站点分享范围文案进入包内，但落点不同

macOS 在 `onboarding-page~profile` 新增了“public internet、workspace、people and groups、groups、people”等站点分享范围的简短标签。Windows 在 `appgen-share-dialog` 新增了 workspace link、public internet、用户/群组搜索、复制链接、外部打开、发布到公网等更直接的对话框文案。

### 【信号】[仅 Windows] RPC stub 与 workers-module 相关错误路径进入包内
