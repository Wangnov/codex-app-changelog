# Codex 桌面版 26.519.81530(macOS / Windows 对照)

> 第三方跨平台变更日志 · 非官方

本批次的核心结论很直接：macOS(苹果桌面操作系统) 端已更新，Windows(微软桌面操作系统) x64 包在本轮对照中没有跟进到可观察内容变化。  
One-liner: this batch is a macOS-side package update, while the compared Windows MSIX stayed byte-identical.

## 关于这份报告

这份报告基于官方签名分发包的逆向 diff(差异比较)：macOS 侧使用 Sparkle(应用更新框架)全量包对比，Windows 侧使用 MSIX(微软应用安装包格式)包对比。macOS 对比范围为 26.519.41501 build 3044 → 26.519.81530 build 3178；Windows 对比范围为 26.519.11010.0 → 26.519.11010.0。

证据等级分两档：【实证】表示来自 plist(苹果属性列表)、AppxManifest(Windows 应用清单)、CSP(内容安全策略)、技能文档、类型声明、package.json(包元数据文件)、依赖清单等可读材料；【信号】表示仅来自前端模块名或体积变化，只能说明方向，不能还原源码意图。

平台归属标记含义为：[macOS+Windows] 表示两平台都能从事实包中观察到；[仅 macOS] 表示只在 macOS 包中观察到；[仅 Windows] 表示只在 Windows 包中观察到。

## ✨ 重点变化

### 【实证】[仅 macOS] macOS 版本与 build 更新

macOS 包从 `26.519.41501` 更新到 `26.519.81530`，`CFBundleVersion` 从 `3044` 更新到 `3178`。`ElectronAsarIntegrity`(Electron 应用 asar 完整性校验)中的 `app.asar` 哈希也随之变化，说明应用层归档内容确实更换。

平台差异很明确：Windows 对照包的版本仍为 `26.519.11010.0`，且上一版与最新版 SHA-256 完全相同，未体现对应内容更新。

### 【实证】[仅 macOS] 内置 Browser 与 Chrome 插件版本号同步到新 macOS 版本

macOS 包内两个内置插件的 `plugin.json` 发生版本号更新：

- `browser` 插件：`26.519.41501` → `26.519.81530`
- `chrome` 插件：`26.519.41501` → `26.519.81530`

这里的 Browser(内置浏览器自动化插件)与 Chrome(谷歌浏览器自动化插件)描述文本未在事实包中显示功能语义变化，能确认的是插件元数据版本已随 macOS 主包同步。

Windows 侧没有对应变化：文件树内容改动为 `0`，`app.asar` 内部改动也为 `0`。

### 【信号】[仅 macOS] 核心可执行组件小幅增大

macOS 包中两个同尺寸核心组件均从 `192.1 MB` 增至 `193.8 MB`，变化为 `+1.70 MB`：

- `Contents/Resources/codex`
- `Contents/Resources/plugins/openai-bundled/plugins/chrome/app-server-runtime/codex`

这属于体积信号：可以确认二进制内容增大，但事实包没有提供源码级解释，因此不能进一步断言新增了哪项能力。

Windows 侧没有类似体积变化记录，文件内容改动为 `0`。

### 【实证】[macOS+Windows] 本轮没有可证实的共同功能变更

跨平台对照后，没有发现一条同时出现在 macOS 与 Windows 的可读配置、依赖或应用层文件变化。macOS 有 61 个 bundle 内容改动，Windows 为 0 个内容改动。

这意味着：本批次可写入日志的实质变化集中在 macOS；Windows 在本轮事实包对应的 MSIX 中尚未体现同批更新。

### 【实证】[macOS+Windows] 依赖层没有新增 cua_node 包

两平台事实包均显示 `cua_node` 新增包数量为 `0`。这只能说明本次对照没有观察到新的 `cua_node` 依赖包加入，不代表其他内部实现没有变化。
