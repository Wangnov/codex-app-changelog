# Codex 桌面版 `26.303.1606` build 806

> 第三方变更日志 · 非官方 · 基于官方更新包逆向差异

这版没有新增 bundle 文件或 cua_node 依赖，但在主进程字符串、设置页文案和前端模块中出现了多组新的用户可见工作流信号。  
One-liner: this build points to deeper Git/worktree automation and deeplink handling, while shrinking the bundled `codex` binary.

## 关于这份报告

这不是官方变更日志，而是一份**基于官方分发产物的逆向差异报告**。本次对比范围是 macOS arm64 的 `26.228.1430` build 760 到 `26.303.1606` build 806。

事实来源是官方全量包解包后的结构化 bundle 差异。由于事实包说明官方 appcast 未对这一对版本提供增量包，本次为历史回填，采用两个官方全量包直接比较。

证据等级分两档：

- **【实证】**：来自 Info.plist、package.json diff、可读字符串、服务入口、签名或验证结果等人类可读直接证据。
- **【信号】**：来自同名前端模块改动线索或二进制体积变化，只能说明指向某方向，但无法从产物还原到源码级意图。

## ✨ 重点变化

### 1. 新增从剪贴板打开 `codex://` deeplink 的路径 【实证】

新版主进程 bundle 新增了 `Open Deeplink from Clipboard`、`Clipboard does not contain a valid codex:// deeplink.`、`Copy a codex:// URL to the clipboard and try again.`、`Invalid Deeplink` 等可读字符串。

这意味着桌面端在这一版至少加入了围绕 `codex://` 链接的剪贴板打开与错误提示路径。能确认的是字符串确实进入产物；完整入口、菜单位置和交互流程无法仅凭压缩 bundle 还原。

### 2. PR 合并相关能力开始进入桌面端 【实证】

新版新增了 `Failed to merge pull request`、`No git repository found for PR merge`、`No workspace available for PR merge`、`Pull request number is required`、`You have uncommitted local changes. Commit or stash them first.` 等主进程字符串。

这组证据指向桌面端新增或强化了 PR merge 流程，并对仓库、workspace、PR 编号和本地未提交变更做了错误路径处理。`git-settings` 前端模块也新增了 `Pull request merge method`、`Choose how Codex merges pull requests`、`Always force push`、`Saved pull request merge method` 等文案，但该模块改动本身属于【信号】，只能说明设置 UI 方向。

### 3. worktree 自动删除取代或重命名了 auto-cleanup 语义 【信号】

`worktrees-settings-page` 同名前端模块新增了 `Automatically delete old worktrees`、`Automatic deletion enabled`、`Automatic deletion disabled`、`Auto-delete limit`、`Disable automatic worktree deletion?` 等字符串，同时移除了 `Auto-cleanup limit`、`Saved auto-cleanup limit` 等字符串。

这指向设置页中 worktree 清理能力的命名或交互从 auto-cleanup 转向 automatic deletion，但无法从前端产物还原到底层策略是否变化。主进程新增 `Failed to clean up archived worktree`，说明归档 worktree 清理失败路径也确实出现在新版产物中。

### 4. 网络请求审批出现允许规则与作用域区分 【信号】

`index` 同名前端模块新增了 `Approve a network request and save a host allowlist rule`、`Approve a network request for the current conversation`、`Approve a network request only for the current attempt`。

这指向网络请求审批 UI 开始区分 host allowlist、当前 conversation 和当前 attempt 等作用域，但这些线索来自同名前端模块 diff，不能作为源码级功能事实。

### 5. 连接与 Agent 环境相关 UI 仍在调整 【信号】

`index` 模块新增了 `Add connection`、`A connection with this display name already exists.`、`Agent environment`，同时去噪后的前端真实移除模块包含 `remote-connections-settings`。

这指向连接管理和 Agent environment 相关界面在调整，但模块名和前端字符串无法证明具体功能被新增、删除或对用户开放。

### 6. WSL 模式检测路径出现在 macOS 产物中 【实证】

新版主进程 bundle 新增了 `WSL mode is enabled but no eligible distro is available` 和 `wsl.exe`。

这说明产物中存在 WSL 模式与发行版可用性检查相关字符串。由于本次平台是 macOS arm64，这更像是跨平台代码进入同一 bundle 的证据，不能据此判断 macOS 端提供 WSL 能力。

### 7. 开发与诊断脚本新增 metadata probe/path 【实证】

`package.json` 新增了两个脚本：

```diff
+    "metadata-path": "pnpm exec tsx ./scripts/dev-metadata.ts path",
+    "metadata-probe": "pnpm exec tsx ./scripts/dev-metadata.ts probe",
```

同时 `playwright:agent:repl` 改为通过 `node --import tsx` 启动。这是面向开发、诊断或测试链路的可读配置变化，对普通用户不一定可见。

### 8. 主二进制显著缩小，app.asar 小幅增长 【信号】

`Contents/Resources/codex` 从 84.4 MB 变为 77.1 MB，减少 7.37 MB；`Contents/Resources/app.asar` 从 36.4 MB 变为 36.5 MB，增加 0.12 MB。

这表明打包后的核心二进制体积明显下降，而应用层 bundle 小幅增长。体积变化只能指向构建、依赖或代码组织变化，无法从产物还原到源码级意图。
