## ✨ 重点变化

### 1. 前端入口从单一主包转向大量模块预加载 【实证】

`webview/index.html` 中，上一版主要加载一个入口脚本与一个样式文件；新版增加了大量 `modulepreload`(提前加载 JavaScript 模块)链接，并拆出多个 CSS(样式表)资源。

这意味着应用启动时的前端装载结构发生了明显变化：更多功能模块被显式预加载，可能用于降低后续界面切换或弹窗打开时的等待，但仅凭产物不能判断具体性能收益。

关键证据：

```diff

+ <script type="module" crossorigin src="./assets/index-CMu6BCpo.js"></script>
```

### 5. 预加载桥新增 Fast Mode 指标接口 【实证】

`.vite/build/preload.js` 中，暴露给前端的 `electronBridge` 新增了 `getFastModeRolloutMetrics` 方法，通过 IPC(进程间通信)调用 `codex_desktop:get-fast-mode-rollout-metrics`。

这说明前端现在可以向主进程请求 Fast Mode rollout metrics(Fast Mode 灰度指标)相关数据；但事实包没有给出该指标的展示位置、含义或用户可见入口。

关键证据：

```diff
+ getFastModeRolloutMetrics:async t=>e.ipcRenderer.invoke(`codex_desktop:get-fast-mode-rollout-metrics`,t)
```
