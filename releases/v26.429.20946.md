# Codex 桌面版 `26.429.20946` build 2312

> 第三方变更日志 · 非官方 · 基于官方更新包逆向差异

这一版从产物层面看，重点集中在桌面自动化权限、Browser Use 插件说明与 API、插件市场管理，以及一批前端功能面的铺垫。  
One-liner: this build expands automation surfaces and starts wiring more app-level workflows into the desktop shell.

## 关于这份报告

这不是官方变更日志，而是一份**基于官方分发产物的逆向差异报告**。本次为历史回填，事实包说明官方 appcast 未对 `26.422.71525` → `26.429.20946` 提供增量包，因此改为下载两个官方全量包直接比较。

证据分为两档：**【实证】**表示来自 Info.plist、package.json、插件配置、技能文档、主进程可读字符串、服务入口等人类可读直接证据；**【信号】**表示只来自前端模块名、同名前端模块字符串差异或二进制体积变化，指向某方向，但无法从产物还原到源码级意图。

对比范围为 macOS arm64，上一版 `26.422.71525` build `2210`，发布于 `2026-04-29`；最新版 `26.429.20946` build `2312`，发布于 `2026-04-30`。

## ✨ 重点变化

### 1. 新增 Apple Events 权限说明，桌面 App 控制边界更明确 【实证】

新版 `Info.plist` 新增 `NSAppleEventsUsageDescription`，文案为：`Codex uses Apple Events to control Mac apps on your behalf.`

这意味着 Codex 在 macOS 上控制其他应用时，已经声明了 Apple Events 的系统权限用途。对用户可感知的变化可能是首次触发相关能力时出现更明确的系统授权说明；对开发者而言，这是 Computer Use 类桌面自动化能力的权限边界变化。

### 2. Computer Use 插件升级到 `1.0.770` 【实证】

内置 `computer-use` 插件的 `.codex-plugin/plugin.json` 版本从 `1.0.758` 升到 `1.0.770`。

同时相关组件体积也有变化：`SkyComputerUseService` 从 `14.1 MB` 增至 `14.5 MB`，`SkyComputerUseClient` 从 `10.5 MB` 降至 `10.0 MB`。版本号变化是直接证据，体积变化只说明实现有所调整，不能单独还原具体行为。

### 3. Browser Use 从“本地目标浏览器”扩展为更完整的内置浏览器自动化描述 【实证】

Browser Use 插件的说明被重写：长描述从强调 `localhost`、`127.0.0.1`、`::1` 和 `file://`，改为说明它可以打开并控制 Codex 内置浏览器，用于导航、检查、点击、输入和截图。

技能文档也同步扩展，新增 `dom_cua: DomCUAAPI`，包含 `click`、`double_click`、`get_visible_dom`、`keypress`、`scroll`、`type` 等 DOM 节点级操作。它意味着 Codex 的内置浏览器自动化不只是坐标或 Playwright 路径，还出现了基于可见 DOM 节点的交互接口。

关键证据：
```diff
+  dom_cua: DomCUAAPI; // API for interacting with the tab via the dom based cua api
```

### 4. Browser Use 对历史访问的描述被收窄 【实证】

Browser 技能文档中 `BrowserUser` 的说明从“tabs and history”改为只提 “tabs”，并移除了 `BrowserHistoryOptions` 与 `BrowserHistoryEntry` 类型。

这表明 Browser Use 的文档接口不再公开历史查询形态。与前端设置里“Ask before accessing history”“Access history without asking”等字符串相比，这里更可靠的结论是技能 API 文档层面对 history 能力的暴露被调整；实际运行时策略不能仅凭产物确认。

### 5. 主进程 preload 新增 MCP app sandbox host 消息通道 【实证】

`.vite/build/preload.js` 新增 `codex_desktop:mcp-app-sandbox-host-message` 监听，并将消息通过 `window.postMessage` 转发到当前 origin。

这是一条跨进程通信能力变化，指向 MCP app sandbox 与 WebView/renderer 之间的消息桥接。由于该文件是压缩后的 preload 产物，能确认通道存在，但不能还原完整调用协议。

关键证据：
```diff
+e.ipcRenderer.on(t,(e,t)=>{let n=window.location.origin;n!==`null`&&window.postMessage(t,n,e.ports)})
```

### 6. 插件市场管理浮出前端，并有市场分类调整 【实证】

内置插件市场配置中有一个条目的 `category` 从 `Engineering` 改为 `Research`。同时前端和主进程新增字符串包含 `Manage-mode tab label for plugin marketplaces`、`Remove marketplace`、`Actualizar todos los marketplaces` 等多语言市场管理文案。

这说明插件市场管理不再只是安装入口，开始出现管理、移除、升级市场的界面与文案。分类调整是配置实证；市场管理 UI 的具体可用性仍取决于运行时入口和实验开关。

### 7. package.json 新增本地浏览器后端、命令与 Markdown AST 依赖 【实证】

新版 `package.json` 新增 `browser-backend-common`、`commands`、`mdast-util-from-markdown`、`mdast-util-to-string` 和 `objc-js`。

这些依赖组合与本版 Browser Use、命令处理、Markdown 内容处理、Objective-C/Apple 平台桥接方向相互印证。它们能证明构建依赖面扩大，但不能单独证明对应用户功能已默认开放。

### 8. 远程连接设置出现更细的 SSH 编辑与在线状态文案 【信号】

新增模块包含 `remote-connections-page`、`remote-connections-onboarding-signals`、`settings-host-dropdown` 等；同名模块 `remote-connections-settings` 新增 `Add SSH connection`、`Edit SSH connection`、`Device connections`、`offline`、`online`、`busy` 等文案。

这些线索指向远程连接管理界面更细化，可能覆盖 SSH 连接编辑、显示名校验、刷新和状态展示。但这些来自前端模块与字符串差异，只能作为 UI/状态方向的信号，无法从产物还原到源码级意图。

### 9. 自动化、Heartbeat 与云端/本地线程桥接相关页面开始铺设 【信号】

新增前端模块包括 `automation-dialog`、`automations-page`、`heartbeat-automation-eligibility`、`heartbeat-automation-permissions`、`heartbeat-automation-thread-bridge`、`send-to-cloud`、`remote-conversation-page` 等。

这组模块名指向自动化、权限判断、线程桥接和云端发送流程，但事实包没有给出可读配置或技能文档来确认具体产品行为。因此只能判断前端层正在出现相关功能面，不能断言该功能已向用户开放。

### 10. PDF 预览从缩放走向标注与演示模式 【信号】

`pdf-preview-panel` 新增 `PDF annotation comment`、`Accessible label for a placed PDF annotation marker`、`Button label that exits PDF presentation mode`、`Next page`、`Previous page` 等文案，移除了部分 `Zoom in`、`Zoom out` 文案。

这指向 PDF artifact 预览正在增加标注、页间导航和演示模式相关交互。由于证据来自同名前端模块字符串差异，只能确认 UI 文案和状态方向发生变化，不能还原底层 PDF 功能实现。

### 11. 个性化从 custom avatars 转向 custom pets 【信号】

`appearance-settings` 中大量 `Custom avatars`、`Loading custom avatars` 等文案被 `Custom pets`、`Loading custom pets`、`floating pet overlay`、`pet picker` 取代；主进程新增字符串也包含多语言的 custom pets 与 mascot 文案。

这说明个性化设置的可见语言从头像转向宠物/mascot 方向。该结论来自前端文案和模块改动线索，指向 UI 命名和交互方向，但不能确认资源格式或完整功能状态。

### 12. 前端外壳被明显拆分，新增首屏、设置、线程、插件、工作区等模块 【信号】

去噪后新增模块包括 `app-shell`、`onboarding-shell`、`settings-page`、`first-run`、`welcome-page`、`plugin-detail-page`、`plugins-page-section`、`local-conversation-page`、`remote-conversation-page`、`worktree-init-v2-page` 等。

这显示 WebView 前端结构在这一版发生较大拆分与重组，覆盖应用外壳、登录/首次使用、设置、插件详情、线程和 worktree 初始化等区域。由于这些是前端模块名信号，能说明方向，不能直接写成“功能已上线”。
