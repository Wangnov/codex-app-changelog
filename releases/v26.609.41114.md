## 关于这份报告

这不是官方变更日志，而是一份**基于官方分发产物的逆向差异报告**。本次事实包采用两个官方全量包直接比较（full-pair），而非从官方增量包重建——全量比较覆盖完整 bundle 内容，代价是不引用官方增量包自带的 EdDSA(一种数字签名算法) 验签。

对比范围是 `26.609.30741`(build 3808，2026-06-11) → `26.609.41114`(build 3888，2026-06-12)，平台为 macOS(苹果桌面操作系统) arm64(Apple Silicon 架构)。

可信度方面，事实包提供了两个官方全量包的 SHA-256(文件哈希)；公证验证结果显示两个 bundle(应用包) 均为 `bundle format unrecognized, invalid, or unsuitable`。事实包没有提供 EdDSA(一种数字签名算法) 增量验签结果，也没有提供 codesign(代码签名工具) 的成功结论，因此本文不额外推断。

证据等级分为两档：

- 【实证】：来自人类可读的直接证据，例如 Info.plist(应用元数据文件)、插件配置 diff(差异文本)、哈希或版本号。
- 【信号】：只有间接证据，例如前端模块名或二进制体积变化；这类证据只能说明“指向某方向，但无法从产物还原到源码级意图”。

## ✨ 重点变化

应用元数据中的短版本号从 `26.609.30741` 更新到 `26.609.41114`，构建号从 `3808` 更新到 `3888`。


### 2. 内置 Browser / Chrome 插件元数据与主应用版本对齐 【实证】

证据：


### 3. `app.asar` 内容有极小幅变化，但没有可确认的新能力 【实证】

`Contents/Resources/app.asar` 体积从 `150.7 MB` 到 `150.7 MB`，差值为 `+0.01 MB`；ElectronAsarIntegrity(asar 完整性校验信息) 中的 `app.asar` 哈希也发生变化。
