# Codex 桌面版 26.609.41114 macOS / Windows 对照

> 第三方跨平台变更日志 · 非官方

这一批次的核心变化集中在 Computer Use 运行时扩容、Sites 技能规则重写、数据库恢复路径、浏览器/终端/插件管理 UI 信号，以及平台特有的外壳资源变化。  
One-liner: This build expands the local Computer Use stack, tightens Sites deployment guidance, and surfaces new recovery, browser, terminal, plugin, and workspace UI paths across platforms.

## 关于这份报告

本报告只基于两份官方签名分发包的逆向 diff 事实包：macOS arm64 从 `26.608.12217` build `3722` 到 `26.609.41114` build `3888`，Windows x64 MSIX 从 `26.608.1337.0` 到 `26.609.4994.0`。两平台来自同一套 Codex 代码的不同打包，但版本号和打包号不完全一致，因此所有结论分别按平台标注。

证据等级分两档：

- 【实证】：来自 plist、manifest、CSP、技能文档、类型声明、package.json、依赖清单、插件清单、服务入口、签名信息，或事实包中不属于“同名前端模块改动线索”的新增可读字符串。
- 【信号】：来自前端模块名、体积变化、同名前端模块可读字符串/样式类变化等压缩 bundle 方向线索。它说明代码包内出现了相关 UI 或状态方向，但不等于确认所有用户已经可见。

平台归属标记为 `[macOS+Windows]`、`[仅 macOS]`、`[仅 Windows]`。webview/assets 和 `.vite/build` 的 hash 改名已按噪音处理。

## ✨ 重点变化

### 【实证】[macOS+Windows] Computer Use 运行时显著扩容，新增图像、PDF、OCR 与浏览器自动化依赖

两平台的 `cua_node` 都大幅增大。macOS 从 `213.5 MB` 到 `365.8 MB`，增加 `152 MB`、`+71%`；Windows 从 `120.6 MB` 到 `285.5 MB`，增加 `165 MB`、`+137%`。

依赖新增方向高度一致：`sharp@0.34.5`、平台预编译 sharp/libvips、`@napi-rs/canvas@0.1.91`、平台 canvas 后端、`pdfjs-dist@5.4.624`、`tesseract.js@7.0.0`、`tesseract.js-core@7.0.0`、`playwright@1.57.0`、`playwright-core@1.57.0`、`pixelmatch@7.1.0`、`pngjs@7.0.0`、`jpeg-js@0.4.4`、`bmp-js@0.1.0` 等进入包内。可确认的能力面是：运行时开始随包携带图像处理、PDF 解析、OCR、像素比较和浏览器自动化相关库。

平台差异主要是原生预编译包不同：macOS 新增 `@img/sharp-darwin-arm64`、`@img/sharp-libvips-darwin-arm64`、`@napi-rs/canvas-darwin-arm64`、`fsevents`；Windows 新增 `@img/sharp-win32-x64`、`@napi-rs/canvas-win32-x64-msvc`。Windows 新增包数为 `27`，macOS 为 `29`。

### 【实证】[macOS+Windows] `@oai/sky` 升级到 `0.4.10`，Linux full-desktop 调用开始透传 options

两平台的 `@oai/sky` 都从 `0.4.8-202606020558-pr-981870-f570e0604110` 升到 `0.4.10`。`Options.d.ts` 新增 `mouse_size_px?: number`，说明截图中的鼠标指针大小可配置。

Linux target 的 `click`、`drag`、`move`、`scroll`、`press_key`、`type_text`、`get_screenshot` 都从只传 `input` 改为传 `{ options, input }` 或 `{ options }`。`sky_linux.js` 还新增 `--mouse-size-px` 参数拼接。`create_client.js` 也从 `get_screenshot:t()` 改为 `get_screenshot:t(i)`，使截图路径也拿到同一组选项。

### 【实证】[macOS+Windows] 本地数据库恢复/回填流程进入主进程字符串

两平台新增同一组主进程/前端 bundle 可读字符串：`Back Up and Rebuild`、`Codex rebuilt its local database.`、`Rebuilding conversation history`、`Database path: `、`Backup folder: `、`state db backfill`、`timed out waiting for state db backfill`、`Failed to show SQLite backfill progress window`、`Failed to show app-server SQLite recovery confirmation`、`Queued app-server SQLite recovery confirmation`、`Showing app-server SQLite recovery confirmation`。

这能确认包内出现了 SQLite 回填、恢复确认、备份重建和进度窗口相关路径。不能据此断言恢复流程一定对所有用户开放，但它已经是明确的运行时代码/文案事实。

### 【实证】[macOS+Windows] 浏览器资料导入、cookie 选择与不可导入状态进入包内

新增字符串包括 `Browser profile is no longer importable`、`Owl browser profile importer is unavailable`、`Cookie domain selection requires cookie import`、`Select cookies, passwords, or both to import`。这表明浏览器 profile 导入、cookie/password 选择和不可导入错误状态已经进入包内。

Windows 另有【信号】级的 `browser-use-settings` 模块大幅变化，出现 `Access history without asking`、`Add allowed domain`、`Add allowed download domain`、`Add allowed upload domain`、embedded browser/site/password/extension settings 等设置文案。macOS 事实包没有列出对应同名模块改动，因此这里只能标为 Windows 侧额外 UI 信号。

### 【实证】[macOS+Windows] Sites 插件升级到 `0.1.15`，发布职责从本地文档转向 connector 描述

两平台 `sites` 插件从 `0.1.14` 到 `0.1.15`。插件 long description 从列举“landing pages、portfolios、dashboards、portals、trackers、hubs、internal tools”扩展为包含 `games`，并调整措辞为 “such as ... with Sites”。

`sites-building/SKILL.md` 新增 D1 prepared statement 规则：应用查询和运行时初始化应使用 `env.DB.prepare(...).run()`，不要把多个分号分隔语句塞进一个 `prepare()`，多语句操作应分别 prepare 后 `batch([...])`。这属于明确的技能行为指导变更。

`sites-hosting/SKILL.md` 大幅改写：从“创建/复用 hosting 配置、保存版本、部署生产、检查状态、管理访问控制”的完整本地流程，改为“本地准备 deployable Sites source and build artifacts”，并明确 Sites connector 和 tool descriptions 是 site creation、environment variables、source credentials、versions、deployments、access control 的事实来源。也就是说，技能文档本身把远端发布和访问控制职责更明确地交给 connector schema。

### 【实证】[macOS+Windows] CSP 新增 Mapbox 连接白名单

两平台 `webview/index.html` 与 `webview/avatar-overlay-composition-surface.html` 的 `connect-src` 都新增 `https://api.mapbox.com` 和 `https://events.mapbox.com`。这确认 webview 和 avatar overlay composition surface 都开始允许连接 Mapbox API 与事件域名。

这只能证明网络策略放行，不能单独证明某个地图 UI 已经可见。

### 【信号】[macOS+Windows] 多面板线程 UI：底部面板、侧边面板、终端/浏览器/Review tab 打开路径增强

两平台新增前端模块一致包含 `thread-side-panel-active-signals`、`windows-tabs-open`、`windows-tabs-open-schema`、`run-app-action-in-primary-window`、`interrupted-turn-state`。`thread-page-bottom-panel-state` 同名模块从近似空模块增至 `0.3 MB`，新增字符串包括终端崩溃后的 reload 按钮、terminal panel crash 描述、`Opening link blocked as opener could not be cleared`、xterm buffer/accessibility 相关错误。

`app-main` 新增 `Browser tab could not be opened`、`Review tab could not be opened`、`Terminal tab could not be opened`、`Terminal tab is unavailable for this thread`。macOS 事实包还在 `thread-app-shell-chrome` 中出现 `Bottom panel`、打开 side chat/browser/review/terminal/timeline/file browser 的 action label，以及 New tab 空状态。这些共同指向线程内多 tab、多面板、终端面板与侧栏入口的 UI 重排或增强。

平台差异：两平台都有底部面板和打开失败文案；macOS 事实包额外给出 `thread-app-shell-chrome` 的侧栏/底栏 action 文案，Windows 事实包未列出该同名模块改动。

### 【信号】[macOS+Windows] 反馈与举报流程从简单评分扩展为更细的报告原因选择

两平台 `plan-summary-item-content` 同名模块新增 `Accessible label for Codex thread report reason options`、`Back button for Codex thread report reason selection`、`Bad response`、`Body shown after a Codex thread safety or legal report is submitted`、`Body shown when a Codex thread report reason needs an external form`、`Button label for opening an external Codex report form`、`Error text shown when Codex thread report submission fails`，并把 artifact feedback 拆成 missed template、incorrect content、incorrect topic coverage。

这说明线程反馈/报告 UI 的原因选择、提交后状态、外部表单和错误状态都有新 bundle 信号。它仍是前端模块信号，不等于确认每个入口已经对所有用户开放。

### 【信号】[macOS+Windows] 工具活动摘要更偏向“技能、集成、MCP、代码搜索、文件读取”

两平台 `local-conversation-thread` 新增折叠工具活动摘要文案：loaded skill definitions、named integrations、named MCP tool call sources、running named MCP tool calls、completed/in-progress code searches、completed/in-progress file reads。移除的文案集中在 auto-review reviewer agent 状态与 `Auto-review approved/denied`。

这表明对话线程中的工具活动展示，正在从自动审查统计文案转向更通用的技能、集成、MCP 工具调用、代码搜索和文件读取摘要。

### 【信号】[macOS+Windows] 插件详情与分享 UI 扩展，但两平台暴露的细节不同

两平台 `plugin-detail-page` 都出现插件分享/安装相关信号，如 `Add plugin`、`Add people or groups`、搜索 workspace users/groups、分享对话框空状态或可访问性标签。

平台差异明显：macOS 事实包出现 MCP server enablement toggle、section navigation、skill preview modal more actions、skill card actions menu、remove selected workspace share target、`App unavailable` 等；Windows 事实包出现上传新版本地插件到已有分享、`Shared version is out of date`、`Shared version updated`、本地版本比共享版本新的状态文案，以及 developer/version 标签。可合理归纳为同一插件管理面在两平台样本中都在变化，但 Windows 侧更突出本地插件分享版本更新，macOS 侧更突出详情页导航、MCP 开关和技能预览操作。

### 【信号】[macOS+Windows] Sites live site 分享对话框加入 workspace 与互联网访问选项

两平台 `appgen-share-dialog` 同名模块新增 `Anyone in this workspace with the link`、`Anyone on the internet`、`Can visit`、`Copy link`、`Copied to clipboard`、`Button label for copying a live site URL`、`Button label for opening a live site externally`，以及 workspace user/group 搜索和保存中状态。

这说明 Sites 或 appgen 的 live site 分享 UI 新增访问范围、复制链接、外部打开和协作者选择相关信号。

### 【信号】[macOS+Windows] 语音、实时波形与 avatar overlay 状态进入前端模块和本地化键

两平台新增模块包含 `transcribe-audio`、`use-realtime-waveform`。多语言资源新增键包括 `avatarOverlay.startRealtimeVoice`、`avatarOverlay.showActivity`、`avatarOverlay.hideActivity`、`avatarOverlay.realtime.createThreadError`。这说明语音转写、实时波形和 avatar overlay 的实时语音状态在 bundle 中有新增信号。

### 【实证】[仅 macOS] 新增 Dock tile 插件和多套图标资源

macOS `Info.plist` 新增 `/CodexAppIconBaseName = icon` 和 `/NSDockTilePlugIn = CodexDockTilePlugin.plugin`。包内新增 `Contents/PlugIns/CodexDockTilePlugin.plugin`，签名 identifier 为 `com.openai.codex.dock-tile-plugin`，签名 authority 为 `Developer ID Application: OpenAI OpCo, LLC (2DC432GLL2)`。

同时新增 `Contents/Resources/icon.png`、`icon-dev-dark.png`、`icon-chatgpt.png`、`icon-dev-light.png`。结合主进程新增字符串 `Preferred macOS Dock icon`，可确认 macOS 侧加入 Dock tile 插件和 Dock 图标偏好相关资源/文案。

### 【实证】[仅 macOS] macOS 权限说明去掉句末句点

`Info.plist` 中 `NSAppleEventsUsageDescription`、`NSCameraUsageDescription`、`NSMicrophoneUsageDescription` 的英文说明去掉了句末句点，语义未变：Apple Events 用于代表用户控制 Mac apps，camera 用于 video input，microphone 用于 voice input。

### 【实证】[仅 Windows] Computer Use 插件版本随应用升级，并保留 Windows 描述

Windows 包内 `resources/plugins/openai-bundled/plugins/computer-use/.codex-plugin/plugin.json` 从 `26.608.12217` 升到 `26.609.41114`，描述仍为 “Control desktop apps on Windows from Codex through Computer Use.”。macOS 事实包没有对应 computer-use 插件清单 diff，因此此条仅按 Windows 事实列出。

### 【信号】[仅 Windows] 技能页和 @mention 列表暴露更多入口分类

Windows `skills-page` 新增 `Button label for creating a new skill`、`Button label for reloading skills list`、`Header title for the skills page`、installed/recommended skills section、搜索输入、空状态和需要刷新才能在 composer 使用的提示。

Windows `at-mention-list` 新增 `Computer use`、`Custom agents`、`Desktop apps`、`Live agents`、`MCP servers`、`Mac apps`、当前 MCP server mention scope、文件 fuzzy search loading/empty 等文案。它指向 @mention 列表正在覆盖 Computer Use、agents、desktop apps、MCP servers 和文件搜索等对象类别。macOS 事实包未列出这两个同名模块变化。
