# Codex 桌面版 26.707.31428 macOS / Windows 对照

> 第三方跨平台变更日志 · 非官方

本批次的主线是将桌面端从“聊天”进一步重组为“任务”，同时引入完整的深度研究与交互式可视化能力，并开始把系统品牌面迁向 ChatGPT。

This batch shifts the desktop experience from chats toward tasks, adds full Deep Research and interactive visualization capability surfaces, and begins a system-level migration toward the ChatGPT brand.

## 关于这份报告

本报告比较两组独立的官方签名全量包：

- macOS arm64：`26.623.141536` build 4753 → `26.707.31428` build 5059。
- Windows x64 MSIX：`26.623.19656.0` → `26.707.3748.0`。

两端版本号和打包内容分别引用，不能相互代替。macOS 因增量包已超出 Sparkle 窗口，改用两个官方全量包重建比较；两版均通过 Notarized Developer ID 验证，签发主体为 OpenAI OpCo, LLC。Windows 样本提供了两个全量包的 SHA-256。

证据等级：

- 【实证】来自 plist、AppxManifest、插件 manifest、技能文档、类型声明、依赖清单、服务入口、人类可读配置，以及确凿进入 bundle 的新增字符串。
- 【信号】来自前端模块名、同名压缩模块的字符串或样式差、体积变化和打包布局。它们能说明开发方向，不能证明功能已向所有用户开放。

平台标记分别为 `[macOS+Windows]`、`[仅 macOS]` 和 `[仅 Windows]`。分析已忽略 `webview/assets`、`.vite/build` 等目录中的 hash 改名噪音。

两份事实包均未给出“内容安全策略变化”区块，因此本报告不对 CSP 是否变化作结论。

## ✨ 重点变化

### Deep Research 成为完整内置插件 【实证】[macOS+Windows]

两端都新增 `deep-research` 0.1.1，而不只是增加一个前端入口。插件包含 manifest、代理配置、技能说明、研究方法、报告契约和资源文件，默认提示明确覆盖复杂问题研究、方案比较和冲突证据协调。

技能契约要求：

- 使用专门的研究子代理执行至少两轮调查；
- 优先核对原始研究、官方记录、标准和第一方资料；
- 区分事实、推断、不确定性和相互冲突的证据；
- 最终必须交付带引用且经过页面检查的 DOCX；
- 缺少 Documents 能力时停止，而不是用 Markdown、PDF 或聊天文本替代。

这说明进入安装包的是一套端到端研究交付工作流，而非普通的“联网搜索”提示模板。

### Visualize 提供交互式可视化能力面 【实证】[macOS+Windows]

两端都新增 `visualize` 1.0.11，包含插件 manifest、完整技能说明、HTML/CSS 模板、渲染脚本、代理配置和明暗主题资源。

其能力范围明确包括交互式图表、地图、关系图、模拟器、3D 模型、数据探索器和 UI 预览。技能文档还区分了静态 Mermaid 与动态 HTML，规定内联可视化文件、大小上限、资源加载、脚本检查和浏览器预览流程。bundle 中同时出现 `codex-inline-vis-screenshot` 的空结果、非法请求和沙箱不可用错误路径。

这证明可视化的创建、渲染与截图链路已经进入两端包内；不代表所有账户已经获得入口。

### “聊天”工作流系统性迁向“任务” 【实证】[macOS+Windows]

两端主进程 bundle 同步加入 `New Task`、`New Projectless Task`、`Rename task`、`Attach image to task`、`Recent Codex tasks`、`Opened desktop files in projectless task` 和新建无项目任务命令。

同时，旧版对应的 `New Chat`、`Rename chat`、`Attach image to thread`、`Recent Codex threads` 等字符串被移除。自动化也出现 `Migrated automations to project targets`、项目不存在、单文件夹 worktree 项目约束等路径。

退出流程新增完整的多语言确认文案：活动本地任务会被中断，应用关闭期间已安排任务不会运行。这是原生菜单资源中的直接变化，不只是前端措辞。

### 任务侧栏、计划任务与项目组织同步重构 【信号】[macOS+Windows]

两端相同模块的变化共同指向：

- `automations-page` 改用 Scheduled Tasks 语言，加入状态筛选、暂停状态、立即采用建议任务、创建任务和返回任务列表；
- `local-conversation-page` 出现任务完成或阻塞前持续检查 PR 的 10 分钟 heartbeat、PR 自动修复和自动合并控制；
- `worktree-environment-dropdown` 增加本地工作树、远程起始分支、分支搜索和加载失败状态；
- `worktrees-settings-page` 增加旧 worktree 自动删除开关、自动删除上限和保存失败提示；
- `home-ambient-suggestions-content` 转向刷新、展开和关闭建议，并加入“学习我的写作风格”、`Common patterns`、`Do and avoid` 和 `Overall voice`。

Windows 的 `sidebar-flat-sections`、`app-main`、`open-project-setup-dialog` 还额外出现任务筛选、云端与本地任务分组、项目源文件夹、批量归档任务和 ChatGPT 项目操作，说明该平台样本包含更密集的侧栏重组信号。

### ChatGPT 品牌迁移进入系统与运行时层 【实证】[macOS+Windows]

共享应用配置把 `codexAppBrand` 从 `codex` 改为 `chatgpt`，入口从 `bootstrap.js` 改为 `early-bootstrap.js`。两端 bundle 新增大量 `ChatGPT for Chrome`、ChatGPT webview、ChatGPT 窗口、ChatGPT 登录、ChatGPT setup config 和 ChatGPT workspace session 文案，并移除对应的 Codex 表述。

平台差异较大：

- `[仅 macOS]` `CFBundleDisplayName`、`CFBundleExecutable`、`CFBundleName` 和 URL 类型名均从 Codex 改为 ChatGPT，新增 ChatGPT 图标与 DMG 资源；相机、麦克风、桌面文件夹和 Apple Events 权限文案也统一改称 ChatGPT。
- `[仅 Windows]` 包中新增 `ChatGPT.exe` 和 ChatGPT 托盘、应用图标资源，但 `package.json` 仍保留 Windows package identity `OpenAI.Codex`。因此该样本支持“迁移进行中”，不支持 Windows 包身份已完整改名。

### `.skill` 成为两端系统级文件类型 【实证】[macOS+Windows]

Windows AppxManifest 新增 `.skill` 文件关联。

macOS 的定义更完整：新增 `Codex Skill` 文档类型、`.skill` 扩展名、`application/vnd.openai.codex.skill` MIME、`com.openai.codex.skill` UTI，并将应用登记为 Owner/Viewer。旧 Chromium 扩展 `.crx` 类型声明被替换。

这表明技能开始以可由桌面应用接管的独立文件形式进入系统集成层。

### 浏览器操作 API 与失败处理更明确 【实证】[仅 macOS]

macOS 包内的 Browser 和 Chrome 插件文档同步加入：

- `locator.evaluate(...)`，限定为只读元素作用域；
- 原生 `<select>` 推荐直接使用 `locator.selectOption(...)`；
- 空标签列表、过期标签和缺失 Playwright helper 不再自动视为浏览器断连；
- 只有明确报告 disconnected 时才重新选择浏览器并读取新文档；
- bot-detection 上报不得把裸 403、管理员拦截、代理、DNS、TLS、超时或连接重置误报为反机器人；
- bot-detection 结果改为 JSON 序列化输出。

这些是插件 API 类型声明和操作文档的直接修改。Windows bundle 也包含 `locator.evaluate requires a pageFunction` 等共享运行时字符串，但事实包没有提供同等完整的 Windows 插件文档 diff，因此不把完整规则外推到 Windows。

### 浏览器侧栏、认证与标签预算路径进入两端 bundle 【实证】[macOS+Windows]

新增字符串覆盖浏览器前进、后退、聚焦地址栏、标签路由排队、Browser Use 关闭标签、等待 webview 附加时窗口关闭、认证目标校验，以及标签预算导致的选择、暂停状态。

此外，两端都包含：

- ChatGPT Chrome native messaging host；
- webview auth token provider 缺失和 cookie 清理失败；
- 仅向预期 OpenAI URL 附加 DeviceCheck token 的拒绝路径；
- `x-i-am-a-browser: true` 标记。

这些字符串证明相应命令、校验和错误路径已进入安装包，但不能证明每个浏览器入口默认可见。

### Git 快照、提交信息与 PR 审查链路扩展 【实证】[macOS+Windows]

两端新增可读字符串覆盖：

- tracked working-tree snapshot 和 turn diff snapshot 的读取、暂存与写入；
- Git snapshot 过期、Git object directory、HEAD tree、未跟踪文件和二进制 patch；
- PR head revision 缺失、仓库要求和提交 PR review 失败；
- 在当前任务内启动 `/review`，或另开审查任务；
- Git Review 使用全部来源还是仅 Last Turn 记录的设置；
- 生成提交信息时的长度边界、回退规则和格式限制。

同名 `local-conversation-page` 模块还出现 `Auto-fix PR`、修复失败检查、处理评论、审查者选择和 PR 状态下拉框，因此具体 UI 呈现仍应视为【信号】[macOS+Windows]。

### 插件、Skills 与 MCP 管理界面继续合流 【信号】[macOS+Windows]

`plugins-page` 同步出现管理型扩展类型切换、插件建议字段、MCP HTTP header、环境变量、Bearer token、服务器认证和返回 MCP 设置等线索；`skills-page` 在 macOS 样本中出现新建 Skill、重新加载、搜索、推荐与已安装分区。

共享 sandbox preload 还新增可选的 `notifyMcpAppsMcpNotification` 端口，并保留既有必需端口校验。【实证】[macOS+Windows]

Windows 的 MCP capability 模块另出现可订阅资源、订阅清理、不可写资源和将后续提示发往新任务或新 worktree 的文案。【信号】[仅 Windows]

### Sites 增加设计服务、交互能力与更可靠的本地预览 【实证】[仅 macOS]

macOS 的 Sites 插件从 0.1.21 升至 0.1.27，并在 manifest 中新增：

- `.mcp.json` 服务入口；
- `sites-design-picker` MCP capability；
- `Interactive` 和 `Write` 能力声明。

vinext starter 新增构建后 HTML 测试和 `react-loading-skeleton` 3.5.0；Wrangler 与 Miniflare 状态被固定在项目目录。针对 macOS Seatbelt 禁止 FSEvents 的情况，预览服务器改用 polling 维持 HMR。

两端前端模块都出现站点自定义域名、DNS 记录、状态刷新和移除域名的 UI 线索，但只有 macOS 提供了上述插件 manifest 与模板配置的直接证据。【信号】[macOS+Windows]

### 文件拖拽与主题初始化下沉到 preload 【实证】[macOS+Windows]

共享 preload 新增 `startFileDrag` IPC、preload 启动时间读取，并在文档节点出现时立即写入 `electron-dark` 或 `electron-light`。这为原生文件拖拽、启动性能计时和首屏主题一致性提供了新的桥接层。

### Computer Use 运行时瘦身并替换浏览器后端 【实证】[macOS+Windows]

两端都移除 `@oai/cdp-browser-backend@0.2.0`，并将 `@oai/sky` 从 0.4.19 升至 0.4.20。

运行时总量分别变化：

- macOS：377.4 MB → 365.7 MB，约减少 12 MB、3%；
- Windows：298.9 MB → 287.2 MB，约减少 12 MB、4%。

macOS `cua_node` manifest 进一步显示 runtime 从 `0.0.2/20260620010411-120c650ffb83` 更新到 `0.0.3/20260708050802-66a5641b5362-pr-1103797`，`node_repl` 归档从 `20260619.1` 更新到 `20260707.2`。

### Computer Use 和 Record & Replay 调整能力边界 【实证】[仅 macOS]

Computer Use 插件从 1.0.857 升至 1.0.1000366，描述从“由 Codex 控制 Mac 应用”改为“由 ChatGPT 控制”，并明确优先采用专用 connector、API 或 CLI。

Record & Replay 的技能流程新增两项约束：

- 创建技能前必须确认目标、预期结果和哪些演示值应成为输入；实质性歧义需要先询问用户。
- Skill Creator 的结构校验不能证明录制的 Computer Use 工作流可成功复现；创建后还必须向用户总结步骤、输入和关键假设。

Windows 上一版和新版插件清单均不含 `record-and-replay`，因此这项变化不能外推到 Windows。

### Windows 出现独有的模型、侧栏和开发者工具信号 【信号】[仅 Windows]

Windows 同名模块和新增模块显示：

- `model-and-reasoning-dropdown` 增加 Fast mode、1.5x speed、更多用量消耗提示，以及 Model、Effort、Speed 高级选项；
- 新增 `activity-grouping-version`、`task-row-status-indicator`、`sidebar-unified-layout`、`thread-search-results`、`review-file-tree-pane`、`generated-image-preview`、`wsl-remote-connections-feature` 等模块；
- 新增 `hatch-pet` Skill 文件和准备脚本，以及 pet egg、spritesheet、检测历史、电池状态等模块；
- `review-file-source-tab` 大幅增长，并引入更丰富的文档/代码查看样式与可访问状态。

Windows DevTools 的 `ObjectPropertiesSection.js` 还直接新增对象树展开状态跟踪，可记录属性、内部属性、数组范围与 getter 展开路径，并在重建树时恢复状态。【实证】[仅 Windows]

这些 Windows 独有项来自该 Windows 版本区间，不能据此断言 macOS 永远缺少对应能力。

### macOS 系统权限与运行框架更新 【实证】[仅 macOS]

macOS 新增定位权限说明：“ChatGPT uses your location to provide location-based answers”。

Chromium 基础版本从 149.0.7827.197 升至 150.0.7871.101，主 Framework 也切换到对应版本。新的 `codex-code-mode-host` 二进制进入资源目录；Windows 同样新增 Windows 与无扩展名的 code-mode-host，但仅能作为打包方向信号。【信号】[macOS+Windows]
