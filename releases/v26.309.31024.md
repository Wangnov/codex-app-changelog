# Codex 桌面版 `26.309.31024` build 962

> 第三方变更日志 · 非官方 · 基于官方更新包逆向差异

这一版的重点不是单一功能点，而是桌面端启动链路、插件/MCP 通道、设置权限和诊断体系的一次集中扩展。

## 关于这份报告

这不是官方变更日志，而是一份**基于官方分发产物的逆向差异报告**。本次为历史回填：官方 appcast 未对 `26.305.950` → `26.309.31024` 提供 Sparkle 增量包，因此改为下载两个官方全量包直接比较。

对比范围为 macOS arm64：`26.305.950` build `863`，发布于 `2026-03-05`；`26.309.31024` build `962`，发布于 `2026-03-12`。

证据分两档：

- **【实证】**：来自人类可读的直接证据，例如 `package.json`、`preload.js`、`webview/index.html`、`Info.plist`、主进程 bundle 新增字符串、文件树和签名验证结果。
- **【信号】**：来自前端模块名、同名前端模块字符串/样式差异或体积变化。它们指向某方向，但无法从产物还原到源码级意图。

`webview/assets` 和 `.vite/build` 下带 hash 的文件名增减已按构建产物重命名处理，不把这类增减写成新增或删除功能。

## ✨ 重点变化

### 1. 主进程入口切到 bootstrap，启动链路更明确 【实证】

`package.json` 的 Electron 入口从 `.vite/build/main.js` 改为 `.vite/build/bootstrap.js`，同时主进程 bundle 出现 `Desktop bootstrap failed to start the main app`、`bootstrap handoff complete`、`main app.whenReady resolved`、`Session Crashed` 等新增字符串。

这意味着桌面端启动被拆成了更明确的 bootstrap 阶段，并新增了启动失败、会话崩溃和 handoff 相关路径。关键证据：`"main": ".vite/build/main.js"` → `"main": ".vite/build/bootstrap.js"`。

### 2. 应用菜单和更新入口暴露给前端桥 【实证】

`preload.js` 新增 `showApplicationMenu`，通过 `codex_desktop:show-application-menu` IPC 调用主进程；主进程 bundle 同时新增 `Check for Updates`、`Install Update`、`application menu refreshed`。

可感知点是桌面应用菜单和更新操作更可能由前端界面触发或刷新，而不是只依赖原生菜单静态路径。

### 3. 插件 marketplace 与 MCP 运行时信号明显增多 【实证】

主进程 bundle 新增 `Failed to fetch plugins from app server`、`plugin.json`、`marketplace.json`，并新增大量 `mcp.*` 字段，例如 `mcp.tool.name`、`mcp.resource.uri`、`mcp.prompt.result`、`mcp.progress.percentage`、`mcp.session.id`。

这表明插件市场、插件描述文件和 MCP 生命周期进入了更核心的运行时路径。事实包没有提供新增内置插件清单，因此这里不能写成具体内置插件新增。

### 4. Agent 配置页出现管理员配置、网络访问和审批策略线索 【信号】

同名 `agent-settings` 前端模块新增 `Admin config`、`Allow network access`、`Approval policy`、`Approval policy is restricted by this installation.`、`Can edit files outside this workspace` 等字符串。

这些线索指向设置页正在呈现更细的权限、网络访问和管理员管控状态，但由于来源是同名前端模块改动线索，无法从产物还原到源码级意图。

### 5. 用量设置指向自动充值与支付管理流程调整 【信号】

同名 `usage-settings` 前端模块新增 `Automatically add credit when you reach your minimum balance.`、`Unable to open payment settings right now. Please try again.`、`manage-payment` 等字符串，同时移除了多条旧的 minimum balance 和 auto top-up 文案。

这指向自动充值配置和支付管理入口的 UI/错误处理调整，但仍属于前端信号，不能断言完整产品流程或计费策略已经改变。

### 6. 远程 SSH 连接文案从 GitHub CLI 发现路径转向 Codex-managed SSH 【实证】

主进程 bundle 新增 `Codex-managed SSH block is malformed`、`failed to read Codex-managed SSH connections`、`failed to refresh remote SSH connections`；同时移除 `Failed to parse GitHub CLI response`、`Unexpected GitHub CLI response`、`gh auth login`、`github.com`。

这说明远程连接相关错误路径的可读文案发生了方向性变化：从 GitHub CLI 登录/解析提示，转向 Codex-managed SSH 连接读写与刷新提示。

### 7. 线程 steer 与应用终端状态出现新的错误路径 【信号】

同名 `app-server-manager-hooks` 前端模块新增 `Failed to read the app terminal for this thread.`、`No app terminal session is attached to this thread yet.`、`note: output is truncated to the latest terminal buffer kept by the app.` 等字符串。主进程 bundle 也新增 `Missing thread follower steer-turn response`、`Unknown thread follower steer-turn response requestId`。

这些线索指向线程终端快照、输出截断和 steer 接受流程的错误处理更细，但前端模块部分只能作为 UI/状态方向信号。

### 8. 可观测性、调试和自动 instrumentation 扩展 【实证】

主进程 bundle 新增 `Desktop Sentry test error (debug button)`、`No Sentry client available, profiling is not started`、`Automatic instrumentation of `、`auto.http.otel.connect`、`auto.db.supabase`、`auto.function.mcp_server`、`The statsigIntegration() can only be used in the browser.` 等字符串。

这说明诊断、profiling、feature flag 集成和自动 instrumentation 的运行时覆盖面扩大。仅凭这些字符串不能推断实际采集策略或开关默认状态。

### 9. Webview 首屏加载结构被重排 【实证】

`webview/index.html` 的入口仍指向新版 `index` chunk，但大量旧的业务 chunk 预载被移除，新的预载集合包含 `trace`、`sentry-init.electron`、`react-dom`、`message-bus`、`loading-page` 等启动相关模块。

可感知点是前端加载路径更像被拆成启动、消息总线、loading 页面和主应用阶段；但不能仅凭产物断言性能收益。

### 10. 代码高亮、数学和浏览器渲染相关前端包发生重切分 【信号】

去噪后的新增模块包含 `shiki-default-theme`，同名前端模块中 `tsx`、`python`、`shellscript`、`go`、`html`、`postcss`、`angular-html`、`json` 等出现大量语法 scope 字符串变化，`katex` 增加 `263700 bytes`，`browser` 增加 `651436 bytes`。

这些信号指向代码高亮、数学渲染和浏览器渲染相关资源被重新组织或扩展，但无法从产物还原到源码级意图。

### 11. 新增 `fa` 本地化资源 【实证】

Bundle 文件树新增 `Contents/Frameworks/Electron Framework.framework/Versions/A/Resources/fa.lproj/locale.pak`，并出现 `fa_FEMININE.lproj`、`fa_MASCULINE.lproj`、`fa_NEUTER.lproj` 资源；Sparkle 也新增 `fa.lproj/Sparkle.strings`。

这说明分发包内新增了 `fa` 相关本地化资源，但不等于完整产品语言切换一定已经对用户开放。
