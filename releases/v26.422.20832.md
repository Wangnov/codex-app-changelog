# Codex 桌面版 `26.422.20832`(build 2025)

> 第三方变更日志 · 非官方 · 基于官方更新包逆向差异

这一版把浏览器自动化、本地开发页面验证、LaTeX 编译能力放到了更明确的位置，同时前端模块里出现了若干指向设置、认证、PDF 和事件上报方向的新信号。  
One-liner: Browser Use becomes more explicit, LaTeX/Tectonic lands as a local plugin, and browser-sidebar localization gets a broad pass.

## 关于这份报告

这不是官方变更日志，而是一份**基于官方分发产物的逆向差异报告**。本次对比范围是 Codex（OpenAI 的编程代理桌面应用）桌面版 `26.417.41555` build（构建号）`1858`，发布于 `2026-04-21`，到 `26.422.20832` build `2025`，发布于 `2026-04-23`，平台为 macOS（苹果桌面操作系统）arm64（Apple Silicon 架构）。

事实包说明本版属于历史回填：两个版本均超出官方 Sparkle（macOS 应用更新框架）增量窗口，因此实际采用两个官方全量包直接比较。信任根是 Apple（苹果）公证与 OpenAI（开发方）Developer ID（开发者身份）代码签名，上一版和最新版均通过 `spctl`（苹果公证检查工具）验证，结果为 `accepted`，来源为 `Notarized Developer ID`，签名来源为 `Developer ID Application: OpenAI OpCo, LLC (2DC432GLL2)`。

证据等级分两档：**【实证】**表示有可读的直接证据，例如 `Info.plist`（应用元数据文件）、`package.json`（包配置文件）、插件配置或技能文档 diff（差异文本）；**【信号】**表示只有模块名、体积变化等间接证据，只能说明指向某方向，但无法从产物还原到源码级意图。

## ✨ 重点变化

### 1. Browser Use 被重新定位为本地开发浏览器自动化入口 【实证】

`browser-use` 插件的展示名从 `Browser` 改为 `Browser Use`，描述也从泛泛的“Control browsers from Codex”改为面向 Codex in-app browser（应用内浏览器）的本地目标操作，特别点名 `localhost`、`127.0.0.1`、`::1` 和 `file://`。

这意味着开发者在让 Codex 打开、检查、测试、点击、输入或截图本地页面时，产品内的推荐入口更明确地指向应用内浏览器，而不是 macOS `open`、外部 Playwright（浏览器自动化工具）或通用网页浏览。关键证据是插件配置新增默认提示：`Open localhost:3000`、`Test my latest changes in the browser`，并加入 `composerIcon` 与 `logo` 指向 `./assets/browser.png`。

### 2. 浏览器技能文档强化了执行路径和 API 边界 【实证】

`browser/skill` 文档新增要求：首次浏览器动作前必须完整读取技能文档；浏览器控制只能通过 Node REPL（Node 交互式运行环境）`js` 工具完成；文中提到的 Playwright 仅指完成 browser-client 初始化后的 `tab.playwright` 接口。

这说明 Browser Use 的调用方式在产物层面被收紧：它更像一条专用的应用内浏览器控制通道，而不是多个浏览器自动化后端的自由切换。接口层面也有变化：删除了 `waitForEvent("filechooser")`、`downloadMedia` 和 `PlaywrightFileChooser` 相关声明，新增 `FinalizeTabsOptions`、`FinalizeTabsKeep`、`FinalizeTabStatus`，并给 `BrowserUserTabInfo` 增加 `id` 字段。

### 3. 新增 LaTeX/Tectonic 本地插件 【实证】

插件市场清单新增 `latex-tectonic`，来源为本地路径 `./plugins/latex-tectonic`，类别为 `Engineering`，安装策略为 `AVAILABLE`，认证策略为 `ON_INSTALL`。文件树里也出现了 `Contents/Resources/plugins/openai-bundled/plugins/latex-tectonic/bin/tectonic`，大小为 `50.3 MB`。

这表示桌面包内新增了一个面向 LaTeX（排版系统）/ Tectonic（LaTeX 编译引擎）的本地能力插件。可感知点可能是 Codex 后续可以更直接地处理或编译 LaTeX 项目；但本报告只确认插件与二进制进入分发包，不推断具体 UI 是否已经开放。

### 4. Computer Use 插件小版本更新，底层服务体积增加 【实证】

`computer-use` 插件版本从 `1.0.755` 升到 `1.0.758`。同时 `SkyComputerUseService` 从 `11.9 MB` 增至 `14.1 MB`，增加 `2.23 MB`。

这说明 Computer Use（让模型操作桌面应用的能力）随包更新了独立插件版本，并伴随服务二进制体积增加。这里能确认的是插件版本与打包体积变化，不能进一步还原具体功能点。

### 5. 浏览器侧“用 Codex 评论”文案进入大量本地化文件 【实证】

多语言 `native-menu-locales` 文件新增或调整了 `browserSidebar.contextMenu.commentWithCodex`。例如简体中文从没有该项变为 `"browserSidebar.contextMenu.commentWithCodex":"评论"`；繁体中文、日文、韩文、西班牙语、法语、德语等大量语言文件也出现对应翻译。

这指向浏览器侧上下文菜单新增了“评论 / Comment with Codex”这一用户可见文案入口。因为证据来自本地化资源，可以确认菜单文案进入包内；但是否在所有场景默认展示，还需要运行时条件配合，事实包没有提供。

### 6. 前端新增模块指向设置、认证、PDF 与事件管线方向 【信号】

去除 Vite（前端构建工具）hash 文件名噪音后，真实新增模块包括 `browser-use-settings`、`experimental-features-settings`、`use-auth`、`pdf`、`product-event-signal`、`analytics-logger-provider`、`app-server-manager-signals`、`artifact-preview-page-navigation` 等。

这些模块名指向浏览器使用设置、实验功能设置、认证状态、PDF 处理、产品事件信号与日志上报等方向，但无法从产物还原到源码级意图，也不能说明这些功能已经对用户开放。这里仅作为间接信号记录。

### 7. 开发启动流程新增 bare modifier monitor 构建步骤 【实证】

`package.json` 中 `dev` 和 `owl` 脚本新增 `pnpm exec node ./scripts/build-bare-modifier-monitor.mjs`。文件树中也新增 `Contents/Resources/native/bare-modifier-monitor`，大小 `0.1 MB`。

这说明桌面包引入了一个名为 `bare-modifier-monitor` 的原生组件，并在开发启动流程里加入构建步骤。仅凭事实包无法确认它对应的具体用户功能，但它不是单纯 hash 重命名。

### 8. 版本元数据与打包完整性更新 【实证】

`Info.plist` 中 `CFBundleShortVersionString` 从 `26.417.41555` 更新到 `26.422.20832`，`CFBundleVersion` 从 `1858` 更新到 `2025`。`ElectronAsarIntegrity/Resources/app.asar/hash` 也从 `eda8b73ad668dd304d3a68504c3c2882429418e6f9af71c4191bcffaf18bdbc6` 更新为 `2b5ed98dcfa233d7ed11349b82999a6ae8e045a51e83aaf02bcd7ecbbbc6b1f7`。

这是标准版本与 asar（Electron 应用资源包）完整性元数据变化，本身不构成用户功能变化，但为本次对比范围提供直接证据。
