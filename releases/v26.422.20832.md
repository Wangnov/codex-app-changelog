# Codex 桌面版 `26.422.20832` build 2025

> 第三方变更日志 · 非官方 · 基于官方全量包逆向差异

本版最可感知的变化是新增本地 LaTeX/TeX 编译能力，并强化 Codex 对 in-app browser 本地目标的检查与测试路径。  
One-liner: Codex gains bundled LaTeX compilation and a more local-dev-oriented Browser Use workflow.

## 关于这份报告

这不是官方变更日志，而是一份基于官方签名分发产物的逆向差异报告。对比范围是 macOS arm64 的 `26.417.41555` build `1858` 到 `26.422.20832` build `2025`。

本次是历史回填：官方 appcast 未对这一对版本提供 Sparkle 增量包，因此方法改为下载两个官方全量包直接比较。报告只归纳 bundle、插件、配置、文案和体积层面的事实。

证据等级分两档：**【实证】**表示来自可读配置、插件清单、技能文档、Info.plist、package.json diff、签名验证或新增可读字符串；**【信号】**表示只来自前端模块名、同名前端模块字符串/样式差异或二进制体积变化，指向某方向，但无法从产物还原到源码级意图。

`webview/assets` 和 `.vite/build` 中带 hash 的文件名增减不作为功能增删依据；`app.asar` 内部新增/移除文件计数也不列入原始数据表，避免把 Vite hash 重命名误读为真实功能变化。

## ✨ 重点变化

### 1. 新增内置插件 `latex-tectonic` 【实证】

最新版内置插件清单从 `browser-use`, `computer-use` 变为 `browser-use`, `computer-use`, `latex-tectonic`。新增插件版本为 `0.1.0`，显示名为 `LaTeX Tectonic`，描述明确写明可用随包的 Tectonic executable 编译 LaTeX 和 TeX 文档。

这意味着 Codex 获得了不依赖系统 TeX 安装的本地 PDF 构建路径。关键证据包括 `marketplace.json` 新增本地插件 `./plugins/latex-tectonic`、技能文档中的 `bin/tectonic --outdir <output-directory> <tex-file>`，以及最大新增文件 `bin/tectonic` 为 `50.3 MB`。

### 2. `browser-use` 重新定位为 Browser Use 和本地 IAB 验证工具 【实证】

`browser-use` 插件的界面名从 `Browser` 改为 `Browser Use`，长短描述都改为强调 Codex in-app browser、本地目标和 `localhost`, `127.0.0.1`, `::1`, `file://`。默认提示也从初始化 browser-client 一类指令，改为 `Open localhost:3000` 和 `Test my latest changes in the browser`。

这对前端开发者更可感知：插件更像是 Codex 内置浏览器的本地开发验证入口，而不是泛化浏览器自动化说明。插件还新增 `composerIcon` 和 `logo` 指向 `./assets/browser.png`，并在 bundle 中加入 `browser.png`。

### 3. 浏览器侧边栏出现评论动作，站点访问与权限文案扩展 【实证】

多语言 native menu 文件新增 `browserSidebar.contextMenu.commentWithCodex`，例如简体中文为 `评论`，繁体中文为 `評論`，韩文为 `댓글 달기`，日文为 `コメント`。同批文件还大面积调整了浏览器加载错误、DNS、代理、网络访问等文案。

同时，新版 bundle 新增了站点访问与权限方向的可读字符串，例如 `Abrir sites sem perguntar`、`Abrir sitios web sin preguntar`、`A Codex soha nem nyitja meg ezeket a webhelyeket`、`Acceso a la red`、`Acceso de lectura`、`Acceso de escritura`。这说明浏览器侧边栏和权限/站点访问策略的用户文案面在扩展。

### 4. Chronicle、Memories、自定义头像和个性化建议文案进入产物 【实证】

新增可读字符串中出现了 `A configurar o Chronicle`、`Activa Memories para usar Chronicle`、`Activez Memories pour utiliser Chronicle`、`Aangepaste avatars`、`A carregar avatares personalizados`、`Activar sugerencias personalizadas` 等文案。

这些是用户可见方向的直接证据：新版至少在产物中加入了 Chronicle、Memories、自定义头像和个性化建议相关文案。是否已对所有用户开放，无法仅凭 bundle 字符串判断。

### 5. `computer-use` 插件与桌面控制服务更新 【实证】

`computer-use` 插件版本从 `1.0.755` 升到 `1.0.758`。其服务二进制 `SkyComputerUseService` 从 `11.9 MB` 增至 `14.1 MB`，变化为 `+2.23 MB`。

这说明桌面控制能力相关组件确实更新。体积变化本身不能说明源码级意图，但版本号和服务入口路径是直接证据。

### 6. PDF / artifact preview 与 presentation runtime 有增强迹象 【信号】

去噪后的前端新增模块包含 `pdf`、`artifact-preview-page-navigation`；同名 `pdf-preview-panel` 模块新增 `Zoom in`、`Zoom out`、`Zoom to fit`、`artifactTab.pdfPreview.zoomIn` 等字符串；`runtime.worker` 新增 `Presentation help plugin is not installed`、`presentation-backed`、`move-slide`、`text-box` 等线索。

这些线索指向 PDF 预览缩放、artifact 页面导航和 presentation runtime 方向，但无法从产物还原到源码级意图，也不能确认具体入口是否已开放。

### 7. 自定义 avatar 设置、实验设置和浏览器 agent 设置面扩大 【信号】

去噪后的真实新增模块包含 `codex-avatar`、`browser-use-settings`、`experimental-features-settings`、`external-agent-config-utils`、`use-browser-agent-availability`。同名 `appearance-settings` 模块还出现了自定义 avatar、本地 folder、manifest 刷新、floating avatar overlay 等字符串线索。

这些线索指向个性化、实验功能、浏览器 agent 可用性和外部 agent 配置方向，但无法从产物还原到源码级意图。

### 8. 新增 native helper 与通知音资源调整 【实证】

bundle 中新增 `Contents/Resources/native/bare-modifier-monitor`，同时 `package.json` 的 `dev` 和 `owl` 脚本都新增 `pnpm exec node ./scripts/build-bare-modifier-monitor.mjs`。此外，新增 `Contents/Resources/codex-notification.wav`，并移除旧的 `Contents/Resources/notification.wav`。

这说明桌面层新增了一个 native modifier 监测相关 helper，并替换或迁移了通知音资源。事实包没有给出更高层行为说明，因此不进一步推断具体交互变化。

### 9. 前端工程层出现 analytics、routing、auth 和 remote middleware 新模块 【信号】

去噪后的新增模块还包括 `analytics-logger-provider`、`auto-track`、`product-event-signal`、`middleware`、`routing-middleware`、`remote-middleware`、`use-auth`、`use-configuration`、`app-server-manager-signals`。

这些模块名指向埋点、路由、中间件、认证、配置和 app-server 管理信号方向，但无法从产物还原到源码级意图；它们更像底层架构线索，而不是可直接声明的功能发布。
