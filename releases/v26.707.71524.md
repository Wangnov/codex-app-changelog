# Codex 桌面版 26.707.71524 macOS / Windows 对照

> 第三方跨平台变更日志 · 非官方

本批次共同变化的核心是 Computer Use 运行时从 0.0.3 升至 0.0.4，以及旧版 ChatGPT Sparkle 更新器的清理逻辑；其余界面线索在两平台间明显错位。

Computer Use and legacy updater maintenance landed on both platforms, while macOS and Windows carried substantially different UI signals.

## 关于这份报告

本报告比较两个官方签名分发包样本：

- macOS arm64：26.707.62119（build 5211）→ 26.707.71524（build 5263）。
- Windows x64 MSIX：26.707.8479.0 → 26.707.9564.0。

两组版本号和发布时间序列并不对齐，因此跨平台共同项只在两份事实包分别提供证据时成立，不能把某个平台的模块或文案变化外推到另一平台。

证据分为两档：

- 【实证】来自 plist、manifest、插件清单、运行时清单、依赖元数据、签名结果、配置文件或新增 bundle 字符串。字符串进入包内是确定事实，但不代表对应功能已向所有用户开放。
- 【信号】来自同名前端模块的体积、可读字符串、样式类名变化，或去除 hash 噪音后的模块名变化。它们反映 UI、状态或交互方向，不等同于源码事实。

平台标记使用 `[macOS+Windows]`、`[仅 macOS]` 和 `[仅 Windows]`。Vite、webview/assets 与 `.vite/build` 的纯 hash 改名均已排除。

macOS 两个全量包均通过公证验证，结果为 `accepted`，来源为 `Notarized Developer ID`，签发主体为 OpenAI OpCo, LLC。Windows 事实包提供了全量包 SHA-256，但未列出单独的 MSIX 签名验证结果。

## ✨ 重点变化

### Computer Use 运行时升级至 0.0.4 【实证】[macOS+Windows]

两平台的 `cua_node/manifest.json` 都把运行时从 `0.0.3/20260708050802-66a5641b5362-pr-1103797` 更新为 `0.0.4/20260713191736-ad8b1874340f-pr-1124580`，`node_repl` 归档也从 `20260707.2` 更新到 `20260713.2`。Node 本体仍为 24.14.0。

平台差异主要体现在目标产物与体积：

- macOS 使用 `darwin-arm64` 运行时，`node_repl` 增加约 0.02 MB；Computer Use 文件内容总量保持 365.7 MB。
- Windows 使用 `windows-x64` 运行时，`node_repl.exe` 增加约 0.23 MB；总量由 287.2 MB 增至 287.5 MB，但按事实包的 MB 取整口径记为 `+0 MB, +0%`。
- 两平台都没有新增 `cua_node` 依赖包；pnpm 仍为 11.0.7，仅工作区验证时间和 `prunedAt` 时间更新。

### 内置浏览器与 Chrome 插件随应用版本对齐 【实证】[macOS+Windows]

两平台的内置 `browser` 和 `chrome` 插件 manifest 版本都从 26.707.62119 更新到 26.707.71524。

`browser` 插件仍明确面向 ChatGPT 内置浏览器和本地 Web 目标，包括 `localhost`、`127.0.0.1`、`::1`、`file://`、当前内置浏览器标签页，以及打开、检查、导航、测试、点击、输入和截图等操作。`chrome` 插件仍面向依赖现有 Chrome 状态的自动化，包括标签页、登录会话、Cookie、扩展和 ChatGPT Chrome Extension 配置。

这里的版本值来自插件自身 manifest；它与 Windows 外层 MSIX 版本 26.707.9564.0 不同。

### Windows Computer Use 插件同步更新 【实证】[仅 Windows]

Windows 内置 `computer-use` 插件 manifest 从 26.707.62119 更新到 26.707.71524，描述仍为通过 Computer Use 从 ChatGPT 控制 Windows 桌面应用。macOS 事实包没有提供对应 manifest 改动，不能据此认定 macOS 同步发生了同项变化。

### Chrome 能力文档与宿主二进制调整 【实证】[仅 Windows]

Windows 包移除了 Chrome 插件的六份能力文档：

- 标签页相关：`browserAuth.md`、`botDetection.md`、`cdp.md`、`pageAssets.md`。
- 浏览器相关：`viewport.md`、`visibility.md`。

与此同时，Windows x64 的 `extension-host.exe` 从 0.8 MB 增至 0.9 MB，约增加 0.10 MB。事实只能确认文档从包内移除以及宿主二进制发生变化，不能据此判断这些底层能力本身被删除。

### 旧版 ChatGPT Sparkle 更新器清理路径进入两平台包内 【实证】[macOS+Windows]

两平台新版 bundle 都新增了相同的四条字符串：

- `Failed to boot out legacy ChatGPT Sparkle job`
- `Failed to clear legacy ChatGPT Sparkle cache`
- `Failed to disarm legacy ChatGPT Sparkle updater`
- `Finished disarming legacy ChatGPT Sparkle updater`

这些字符串确认包内加入了停用旧版 ChatGPT Sparkle job、清理缓存和解除旧更新器的成功及失败路径。Windows 包同样包含这些字符串，但事实包没有给出其触发条件，不能推断所有 Windows 安装都会执行相关流程。

- 主页面出现项目选择器、远程连接组、初始化状态、连接颜色选择器和“Add local project”的可访问性文案。
