# Codex 桌面版 `26.422.71525`(build 2210)

> 第三方变更日志 · 非官方 · 基于官方更新包逆向差异

本版从 `26.422.62136` 推进到 `26.422.71525`，从产物看属于小幅更新：外层 bundle 文件数量没有变化，核心应用包 `app.asar` 内容发生更新但体积几乎持平。  
One-liner: a small notarized desktop update with app-layer content changes and no new top-level bundle files.

## 关于这份报告

这不是官方变更日志，而是一份**基于官方分发产物的逆向差异报告**。本次是历史回填：两个版本均已超出官方 Sparkle(应用自动更新框架)增量窗口，因此没有使用增量包重建流程，而是下载两个官方全量包后直接做结构化 bundle(应用包)差异比较。

可信度方面，两个版本的 `Codex.app` 均通过 Apple(苹果)公证校验，来源显示为 `Notarized Developer ID`，签名来源为 `Developer ID Application: OpenAI OpCo, LLC (2DC432GLL2)`。这说明本次比较对象是通过 Apple 公证、由 OpenAI Developer ID(苹果开发者身份签名)签名的官方分发产物。

证据等级分为两档：

- **【实证】**：来自人类可读的直接证据，例如 `Info.plist`(macOS 应用元数据文件)键值、文件树统计、签名与公证校验结果。
- **【信号】**：来自间接证据，例如前端模块名或二进制体积变化。它只能指向某方向，但无法从产物还原到源码级意图。

对比范围：`26.422.62136` build `2180`，发布于 `2026-04-28`；对比到 `26.422.71525` build `2210`，发布于 `2026-04-29`。

## ✨ 重点变化

### 1. 版本与构建号推进到 `26.422.71525` / build `2210` 【实证】

这是一次正式版本推进：`CFBundleShortVersionString` 从 `26.422.62136` 变为 `26.422.71525`，`CFBundleVersion` 从 `2180` 变为 `2210`。

这意味着 macOS 系统、更新器和外部诊断工具都会把它识别为一个新构建，而不是同一版本的重复打包。

关键证据：

```diff
CFBundleShortVersionString: 26.422.62136 → 26.422.71525
CFBundleVersion: 2180 → 2210
```

### 2. 外层应用包结构基本稳定，未新增或移除顶层文件 【实证】

外层 bundle 文件数保持 `507 → 507`，新增 `0`、移除 `0`、内容改动 `46`。

这说明本次更新没有引入新的顶层组件、插件或独立资源包；变化主要发生在已有文件内容中，对安装目录结构和外层打包形态的影响很小。

关键证据：

```text
文件数: 507 → 507
新增 0 / 移除 0 / 内容改动 46
```

### 3. 核心应用包 `app.asar` 有内容更新，但体积几乎持平 【实证】

`Contents/Resources/app.asar` 从 `119.9 MB` 变为 `119.8 MB`，减少 `0.04 MB`。同时 `Info.plist` 中记录的 `ElectronAsarIntegrity` 哈希发生变化。

这意味着 Electron(跨平台桌面应用框架)应用层内容确实更新了，但从体积看不是一次大规模能力扩张。需要注意，`app.asar` 内部文件的新增与移除大量来自 `webview/assets` 下的 Vite(前端构建工具)哈希文件重命名，不能据此判断新增或删除了具体功能。

关键证据：

```diff
Contents/Resources/app.asar: 119.9 MB → 119.8 MB (-0.04 MB)
ElectronAsarIntegrity hash: 4b1503...c6bb → fe9c0e...062f
```

### 4. 未看到 `cua_node` 新增依赖 【实证】

`cua_node` 依赖变化中新增包数量为 `0`。

这意味着至少从该依赖清单层面看，本版没有新增新的 `cua_node` 包级能力。只能说明依赖清单没有新增，不能反推出相关功能完全没有内部调整。

关键证据：

```text
cua_node 新增 0 个包
```
