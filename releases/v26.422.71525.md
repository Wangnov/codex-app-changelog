# Codex 桌面版 `26.422.71525` build 2210

> 第三方变更日志 · 非官方 · 基于官方全量包逆向差异

这是一次体量很小的 macOS arm64 更新，bundle 文件数保持不变，主要可见线索集中在 onboarding 与权限模式相关前端模块。  
A small desktop update with limited bundle-level movement and a few UI-facing frontend signals.

## 关于这份报告

这不是官方变更日志，而是一份**基于官方签名分发产物的逆向差异报告**。本次为历史回填：官方 appcast 未对 `26.422.62136` → `26.422.71525` 提供增量包，因此对两个官方全量包直接比较。

证据等级分两档：

- **【实证】**：来自 Info.plist、签名/公证验证、bundle 文件树、asar 完整性哈希、可复现下载制品等人类可读直接证据。
- **【信号】**：来自同名前端模块的 hash 产物差异、可读字符串或样式类名变化，只能指向 UI、状态或交互方向，无法从产物还原到源码级意图。

对比范围为 macOS arm64 的 `26.422.62136` build `2180`，发布于 `2026-04-28`，到 `26.422.71525` build `2210`，发布于 `2026-04-29`。

## ✨ 重点变化

### 1. 版本号与 build 更新到 `26.422.71525` / `2210` 【实证】

应用元数据从 `26.422.62136` build `2180` 更新为 `26.422.71525` build `2210`。

这确认本次是 2026-04-29 发布的 macOS arm64 桌面版更新，而不是仅由外部文件名变化造成的误判。

关键证据：

```diff
CFBundleShortVersionString: 26.422.62136 → 26.422.71525
CFBundleVersion: 2180 → 2210
```

### 2. Onboarding 出现“Build software”意图选项线索 【信号】

`index` 前端模块新增了 `Build software`、`Welcome v2 intent option for building software`、`electron.onboarding.welcomeV2.intent.buildSoftware` 等可读字符串。

这指向 welcome/onboarding 流程里可能出现与“构建软件”相关的意图选项，但该证据来自同名前端模块改动线索，无法从产物还原到源码级意图。

关键证据：

```text
webview/assets/index-BSTEB47c.js → webview/assets/index-D-3V455n.js
新增字符串: Build software
新增字符串: Welcome v2 intent option for building software
新增字符串: electron.onboarding.welcomeV2.intent.buildSoftware
```

### 3. 权限模式 UI/状态线索新增 `workspace-write` 【信号】

`use-permissions-mode` 前端模块新增样式/状态类名 `workspace-write`。

这指向权限模式呈现或状态命名中出现了工作区写入相关线索，但该证据仅来自同名前端模块的构建产物差异，不能证明权限模型或运行时行为发生源码级变更。

关键证据：

```text
webview/assets/use-permissions-mode-BKwvpT92.js → webview/assets/use-permissions-mode-BuZhkYOo.js
新增样式/状态类名: workspace-write
```

### 4. 前端设置模块有小幅产物变化，但缺少明确用户语义 【信号】

`use-model-settings` 与 `font-settings` 模块均发生内容变化；其中 `font-settings` 体积增加 `2888 bytes`，并出现 `http://www.w3.org/2000/svg`，同时移除 `data-utility-button`。

这些线索指向设置相关 UI 或内部渲染结构有细微调整，但可读信息不足，无法进一步确认具体功能变化。

关键证据：

```text
use-model-settings: 2.7 MB → 2.7 MB (-2357 bytes)
font-settings: 0.2 MB → 0.2 MB (+2888 bytes)
```

### 5. Bundle 层面几乎没有结构变化 【实证】

顶层 bundle 文件数保持 `507 → 507`，新增 `0`、移除 `0`，内容改动 `46`。唯一列出的关键组件体积变化是 `Contents/Resources/app.asar` 从 `119.9 MB` 到 `119.8 MB`，变化 `-0.04 MB`。

这说明本版不是一次大规模组件增删更新；app.asar 内部的新增/移除计数主要受 Vite hash 重命名影响，不能解读为真实功能增删。

关键证据：

```text

新增 0 / 移除 0 / 内容改动 46
Contents/Resources/app.asar: 119.9 MB → 119.8 MB (-0.04 MB)
```
