# Codex 桌面版 `26.313.41036` build 1032

> 第三方变更日志 · 非官方 · 基于官方更新包逆向差异

这版从产物看是一个偏工作流完善的小版本：本地环境、远程连接、MCP 交互、PR/Git 设置与实验性输入能力都有可见线索。  
English one-liner: this build appears to tighten environment lifecycle, remote connection state, MCP interactions, and editor/runtime UI surfaces.

## 关于这份报告

这不是官方变更日志，而是一份**基于官方分发产物的逆向差异报告**。本次为历史回填：官方 appcast 未对 `26.311.21342` → `26.313.41036` 提供增量包，因此采用两个官方全量包直接比较，并对 bundle、app.asar、Info.plist、前端模块与可读字符串做结构化归纳。

本文使用两档证据等级：

- **【实证】**：来自 Info.plist、package.json、CSP、可读配置、主进程/前端 bundle 新增字符串、签名与验证结果等直接可见事实。
- **【信号】**：来自同名前端模块变动、模块名或体积变化。它们指向某方向，但无法从产物还原到源码级意图。

对比范围为 macOS arm64：`26.311.21342` build `993`，发布于 `2026-03-13`；`26.313.41036` build `1032`，发布于 `2026-03-16`。

## ✨ 重点变化

### 1. 本地环境生命周期新增清理脚本入口 【实证】

本版出现了与 cleanup script 直接相关的文案和设置项，包括 `Cleanup script`、`Cleanup script failed`、`This script will run before a worktree is deleted.` 与 `No cleanup script configured.`。

这意味着本地环境不再只暴露 setup 阶段，开始出现删除 worktree 前执行清理逻辑的能力面。主进程新增字符串还显示旧的 `Failed to run setup script`、`Setup script canceled` 被移除，新版同时保留 `Setup script` 并新增 cleanup 失败路径。

关键证据：

```text
Cleanup script
Cleanup script failed
This script will run before a worktree is deleted.
Overrides the default cleanup script for specific OSes.
```

### 2. 远程连接状态在会话页被显式化 【信号】

`app-server-connection-state` 模块新增了连接状态标签相关文案：connected、connecting、disconnected、error、unauthed，以及 `Login required`。同时新增绿色、黄色、红色、灰色状态类名与 `animate-pulse`。

这指向远程连接状态在 UI 中变得更可见，但这些线索来自同名前端模块变动，无法还原到源码级意图。可感知点可能是线程页里出现更明确的远程连接状态 badge。

关键证据：

```text
threadPage.remoteConnectionStatusBadge.connected
threadPage.remoteConnectionStatusBadge.connecting
threadPage.remoteConnectionStatusBadge.disconnected
threadPage.remoteConnectionStatusBadge.error
threadPage.remoteConnectionStatusBadge.unauthed
Login required
```

### 3. MCP server elicitation 交互路径出现 【实证】

新版主进程/前端 bundle 新增了 `Missing thread follower submit-mcp-server-elicitation response`、`Unknown thread follower submit-mcp-server-elicitation response requestId`、`Failed to forward MCP server elicitation response` 与 `Unexpected MCP server elicitation request method`。

这表明 MCP server 发起请求并等待用户或线程侧响应的路径已经进入产物。前端模块里也出现 `mcp-server-elicitation` 状态类名，可与主进程字符串互相印证。

关键证据：

```text
Failed to forward MCP server elicitation response
Missing thread follower submit-mcp-server-elicitation response
Unexpected MCP server elicitation request method
```

### 4. Git/PR 工作流新增草稿 PR 与批量处理评论线索 【信号】

`git-settings` 模块新增 `Create draft pull requests`、`Display PR status icons on thread rows in the sidebar`、保存失败提示等文案；`index` 模块新增 `Address all`、`Address every actionable comment without asking the user which ones to handle.` 与 `Address requested changes before merging.`。

这指向 PR 相关工作流更强调草稿 PR、侧边栏状态展示和批量处理 review comments，但这些来自同名前端模块变动，只能作为 UI/交互方向信号。

关键证据：

```text
Create draft pull requests
Display PR status icons on thread rows in the sidebar
Address all
Address every actionable comment without asking the user which ones to handle.
```

### 5. Agent 速度档位与实时语音实验入口出现 【信号】

`general-settings` 模块新增了 `Default speed`、`1.5x speed, 2x plan usage`、`About 1.5x faster, with credits used at 2x`、`Realtime voice` 与 `Talk to Codex in real time from the thread composer. Restart Codex after changing this setting.`。

这指向设置页新增或调整了 agent 速度档位和实时语音实验开关，但证据来自同名前端模块变动，无法证明功能在所有用户中开放。

关键证据：

```text
Default speed
1.5x speed, 2x plan usage
Realtime voice
Talk to Codex in real time from the thread composer. Restart Codex after changing this setting.
```

### 6. 插件与 Apps 页补上加载/卸载错误处理文案 【信号】

`skills-page` 模块新增 `Unable to load apps`、`Retry button label for the apps list error state`、`Failed to uninstall plugin`、`Toast shown after successfully uninstalling a plugin`。

这指向插件或 Apps 列表页补充了加载失败、重试、卸载成功和卸载失败状态。由于来自同名前端模块变动，只能确认 UI 文案方向，不能确认后端行为变化。

关键证据：

```text
Unable to load apps
Failed to uninstall plugin
plugins.card.uninstallSuccess
skills.appsPage.loadError.retry
```

### 7. 代码高亮语法资源发生一轮重排 【信号】

多个语言模块发生显著体积变化：`typescript`、`jsx`、`tsx` 新增约 175 KB 到 181 KB，`javascript` 减少约 175 KB；`java`、`go`、`r`、`gdscript` 等新增语法 scope 字符串，`c`、`haxe`、`stylus`、`scss`、`shellscript` 等减少一批 scope 字符串。

这指向内置代码展示或语法高亮资源被重组，可能影响代码片段的高亮细节。但这些都来自同名前端模块和体积变化，只能作为渲染资源方向信号。

关键证据：

| 模块 | 变化 |
| --- | ---: |
| `typescript` | +181049 bytes |
| `jsx` | +177761 bytes |
| `tsx` | +175505 bytes |
| `javascript` | -174796 bytes |
| `c` | -72082 bytes |
| `shellscript` | -41448 bytes |

### 8. macOS 蓝牙权限键更新 【实证】

`Info.plist` 移除了旧的 `/NSBluetoothPeripheralUsageDescription`，新增 `/NSBluetoothAlwaysUsageDescription`，值仍为 `This app needs access to Bluetooth`。

这是权限声明层面的变化，说明 macOS bundle 使用了新的蓝牙权限描述键。事实包没有给出对应功能入口，因此这里只记录权限声明变化，不推断用途。

关键证据：

```text
新增 /NSBluetoothAlwaysUsageDescription = 'This app needs access to Bluetooth'
移除 /NSBluetoothPeripheralUsageDescription
```
