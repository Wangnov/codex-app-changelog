# Codex 桌面版 `26.417.40842` build 1851

> 第三方变更日志 · 非官方 · 基于官方更新包逆向差异

这一版最明显的变化集中在 Chronicle、浏览器自动化插件整合、macOS 文件夹打开声明,以及若干本地环境和自动化相关 UI 信号。  
One-liner: Chronicle appears as a bundled sidecar, while browser automation is consolidated under `browser-use`.

## 关于这份报告

这不是官方变更日志,而是一份**基于官方分发产物的逆向差异报告**。本次为历史回填:官方 appcast 未对 `26.415.40636` → `26.417.40842` 提供增量包,因此比较对象是两个官方全量包。

证据分两档:

- **【实证】**: 来自 Info.plist、package.json、插件清单、技能文档、服务入口、可读字符串、签名/公证结果等人类可读证据。
- **【信号】**: 仅来自前端模块名、同名前端模块字符串/样式变化或体积变化,只能说明方向,无法从产物还原到源码级意图。

本报告刻意忽略 `webview/assets` 和 `.vite/build` 下 hash 文件名增减带来的构建产物噪音。

## ✨ 重点变化

### 1. Chronicle 侧车与研究预览入口出现 【实证】

新版 bundle 新增 `Contents/Resources/codex_chronicle` 这个 4.4 MB 文件,同时主进程/前端 bundle 中出现 `Chronicle feature enabled but bundled sidecar binary was not found`、`Pause Chronicle`、`Resume Chronicle`、`Starting Chronicle...`、`Stopping Chronicle...` 等可读字符串。

这意味着 Chronicle 已经不只是前端命名信号,而是在分发包里出现了独立侧车与用户可见状态文案。设置相关字符串还显示有 `Chronicle research preview`、权限状态、运行/未运行状态等入口。

关键证据:

```text
Contents/Resources/codex_chronicle (4.4 MB)
Chronicle feature enabled but bundled sidecar binary was not found
Pause Chronicle / Resume Chronicle
```

### 2. Chrome 自动化插件被移除,浏览器能力收敛进 `browser-use` 【实证】

内置插件清单从 `browser-use`, `chrome`, `computer-use` 变为 `browser-use`, `computer-use`; 单独的 `chrome` 插件被移除。与此同时,`browser-use` 的插件清单新增 `automation`, `chrome`, `iab`, `node-repl`, `browser-client` 等关键词,并补上长描述。

这指向浏览器自动化能力的打包边界发生变化:不再以单独 `chrome` 插件分发,而是在 `browser-use` 中描述 IAB 后端、Chrome 后端和 `agent.browser.*` 初始化路径。

关键证据:

```diff
- "browser"
+ "browser", "automation", "chrome", "iab", "node-repl", "browser-client"
```

```text
移除内置插件: chrome — Chrome automation and Codex Chrome Extension installation support.
```

### 3. macOS 开始声明可处理文件夹 【实证】

新版 Info.plist 新增 `CFBundleDocumentTypes` 条目,声明类型名为 `Folder`,角色为 `Viewer`,内容类型为 `public.folder`。

这意味着系统层面已经允许 Codex 作为文件夹类型的处理目标。结合 bootstrap 中出现的 `workspace-root-drop-handler` 命名,可以看到文件夹/工作区入口被纳入启动路径,但实际交互细节仍不能仅凭产物完整还原。

关键证据:

```text
CFBundleTypeName = Folder
CFBundleTypeRole = Viewer
LSItemContentTypes[0] = public.folder
```

### 4. Computer Use 插件小版本更新,新增 iPhone Mirroring 指令文件 【实证】

`computer-use` 插件版本从 `1.0.750` 升到 `1.0.755`,并新增两份 `AppInstructions/iPhone Mirroring.md`。相关 Computer Use 服务二进制也有小幅体积变化。

这说明 Computer Use 能力包在本版有实质更新,并开始携带面向 iPhone Mirroring 的应用指令材料。

关键证据:

```diff
- "version": "1.0.750"
+ "version": "1.0.755"
```

```text
AppInstructions/iPhone Mirroring.md
```

### 5. 聊天入口与命令菜单文案扩展 【实证】

新版主进程/前端 bundle 新增多语言的“创建聊天”“固定聊天”“最近聊天”“Codex 版本”等字符串,并出现 `codex.commandMenu.createChat`、`codex.commandMenu.noChatsEmptyState`、`codex.commandMenu.pinnedThreadsGroup` 等 messageId。

这指向聊天列表、命令菜单或空状态的用户界面正在调整。由于这些是 bundle 中的可读字符串,可以确认文案存在,但不能反推完整 UI 流程。

关键证据:

```text
codex.commandMenu.createChat
codex.commandMenu.noChatsEmptyState
codex.commandMenu.pinnedThreadsGroup
```

### 6. 浏览器侧栏上下文菜单补齐后退、前进、重新加载 【实证】

多份 `native-menu-locales/*.json` 增加 `browserSidebar.contextMenu.back`、`browserSidebar.contextMenu.forward`、`browserSidebar.contextMenu.reload`。中文简体也从仅有“检查”扩展为“返回 / 前进 / 重新加载 / 检查”。

这意味着浏览器侧栏的原生菜单能力更完整,用户可见点很可能是右键菜单或系统菜单里的导航操作增加。

关键证据:

```diff
+ "browserSidebar.contextMenu.back":"返回"
+ "browserSidebar.contextMenu.forward":"前进"
+ "browserSidebar.contextMenu.reload":"重新加载"
```

### 7. 权限、本地环境、应用服务请求与调试设置出现前端重排 【信号】

去掉 hash 噪音后,新增前端模块包括 `local-environments-utils`、`send-app-server-request`、`use-permissions-mode`、`gpu-tearing-debug-settings`、`chronicle-setup-state`;移除模块包括 `app-server-manager-signals`、`local-environment-action-icon-options`、`memory-feature-gate`、`use-auth`、`use-personality`。

这些模块名指向本地环境、权限模式、应用服务请求、GPU tearing 调试设置和个性化/记忆相关入口的重排,但无法从产物还原到源码级意图,也不能确认对所有用户开放。

### 8. 自动化错误路径和个性化设置有新增 UI 信号 【信号】

同名前端模块 `index` 新增了多条 automation 错误字符串,包括 storage unavailable、invalid id、scheduling state could not be updated、only supported for local threads 等。同名前端模块 `personalization-settings` 新增了 Chronicle 权限状态、本地自定义 avatar 文件夹、刷新本地 manifests 等文案。

这些线索指向自动化状态处理和个性化设置页正在扩展,但它们来自同名前端模块改动线索,只能作为 UI、状态或交互方向的信号。
