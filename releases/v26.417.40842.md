# Codex 桌面版 `26.417.40842`(build 1851)

> 第三方变更日志 · 非官方 · 基于官方更新包逆向差异

这一版最明显的变化集中在 Browser(浏览器插件) 能力收束、macOS(苹果桌面操作系统) 文件夹入口、浏览器侧栏交互，以及 Chronicle(产物中的组件名) 相关新组件信号。  
One-liner: browser automation is being consolidated while the desktop shell gains small but visible macOS and browser-sidebar affordances.

可信度来自 Apple(苹果公司) 公证与 OpenAI Developer ID(苹果开发者签名身份) 代码签名链。上一版与最新版的验证结果均为 `accepted`，来源均为 `Notarized Developer ID`，签名来源均为 `Developer ID Application: OpenAI OpCo, LLC (2DC432GLL2)`。

## ✨ 重点变化

### 1. Browser 插件接管 Chrome 后端相关说明，原独立 chrome 插件被移除 【实证】

Browser 插件的 `plugin.json` 新增了 `automation`、`chrome`、`iab`、`node-repl`、`browser-client` 等关键词，并新增长描述：它会教 Codex 使用 `browser-client` 运行时、初始化 `agent.browser.*`，并通过 IAB(应用内浏览器后端) 或 Chrome(谷歌浏览器) 后端执行浏览器命令。

这意味着浏览器自动化入口正在从单独的 `chrome` 插件转向 Browser 插件统一承载。事实包同时显示 `plugins/chrome/` 下的 `browser-client.mjs`、`SKILL.md`、安装脚本与 `plugin.json` 被移除。

关键证据：`Browser` 插件长描述新增 “IAB backend for local/file targets or the Chrome backend for remote/profile-dependent browser work”。

### 2. 浏览器自动化技能文档改写为受支持的 `agent.browser.*` 接口面 【实证】

`browser-use/skills/browser/SKILL.md` 大幅重写 API(应用程序接口) 参考部分，从 “Atlas Agent Library API” 改为 “Use this as the supported `agent.browser.*` surface”。同时，原先对 `tab.content.export` 的额外限制说明被删除。

对开发者来说，这不是单个方法的小改动，而是浏览器自动化文档口径的收敛：后续更应围绕 `agent.browser.*` 这组受支持接口理解和使用浏览器能力。

关键证据：`SKILL.md` 中新增 “supported `agent.browser.*` surface”，并移除 “Only ever use `tab.content.export` ...” 这一限制行。

### 3. macOS 现在声明自己可作为文件夹查看器 【实证】

最新版 `Info.plist` 新增 `CFBundleDocumentTypes`，类型名为 `Folder`，角色为 `Viewer`，内容类型为 `public.folder`。

这意味着系统层面已经把 Codex 和“文件夹”这种文档类型关联起来。用户可感知的方向是：从 Finder(文件管理器) 或系统打开入口把文件夹交给 Codex 处理的基础声明已经出现；但事实包没有给出完整交互流程，所以不能进一步断言具体入口样式。

关键证据：`LSItemContentTypes[0] = public.folder`。

### 4. 启动加载页可拖拽窗口 【实证】

`webview/index.html` 给 `.startup-loader` 增加了 `-webkit-app-region: drag`。

这是一处小但直接的桌面体验变化：应用仍在启动加载界面时，加载区域本身可以作为窗口拖拽区域使用，减少“启动中窗口不好移动”的摩擦。

关键证据：`.startup-loader { ... -webkit-app-region: drag; }`。

### 5. 浏览器侧栏右键菜单补齐后退、前进、重新加载文案 【实证】

大量 `native-menu-locales` 本地化文件新增了 `browserSidebar.contextMenu.back`、`browserSidebar.contextMenu.forward`、`browserSidebar.contextMenu.reload`。中文简体里对应为“返回”“前进”“重新加载”。

这说明浏览器侧栏的上下文菜单正在补齐常见浏览器操作。相比只保留“检查”，这更接近用户对内嵌浏览器右键菜单的预期。

关键证据：`native-menu-locales/zh-CN.json` 新增 `back`、`forward`、`reload` 三个浏览器侧栏菜单项。

### 6. Computer Use 插件小版本更新，并出现 iPhone Mirroring 指令文件 【实证】

Computer Use(让 Codex 操作桌面应用的能力) 插件版本从 `1.0.750` 更新到 `1.0.755`。同时，Computer Use 相关资源中新增 `AppInstructions/iPhone Mirroring.md`，且在主应用与 `SkyComputerUseClient.app` 的资源路径下各出现一份。

这指向 Computer Use 的应用指令覆盖范围扩展到了 iPhone Mirroring(iPhone 镜像)。但事实包只提供文件名与插件版本变化，不能证明该能力已经对所有用户开放。

关键证据：`computer-use/.codex-plugin/plugin.json` 的版本号 `1.0.750 → 1.0.755`，以及新增 `iPhone Mirroring.md`。

### 7. 新增 `codex_chronicle` 二进制与 `chronicle-setup-state` 前端模块 【信号】

bundle 中新增 `Contents/Resources/codex_chronicle`，体积为 `4.4 MB`；去掉 Vite(前端构建工具) hash(哈希命名) 噪音后，前端真实新增模块包含 `chronicle-setup-state`。

这些名称指向 Chronicle 相关设置或本地组件方向，但无法从产物还原到源码级意图，也不能证明用户界面已经开放对应功能。

关键证据：新增文件 `Contents/Resources/codex_chronicle`；新增模块 `chronicle-setup-state`。

### 8. 本地环境、权限模式与 GPU 调试相关前端模块出现调整 【信号】

去噪后的前端模块新增 `local-environments-utils`、`use-permissions-mode`、`send-app-server-request`、`gpu-tearing-debug-settings`，同时移除了 `app-server-manager-signals`、`local-environment-action-icon-options`、`memory-feature-gate`、`telepathy-setup-state`、`use-auth`、`use-personality`。

这些模块名指向本地环境、权限模式、应用服务请求和 GPU(图形处理器) 撕裂调试相关方向，但无法从产物还原到源码级意图；也不能据此判断这些设置是否已在界面中暴露。

关键证据：事实包列出的“去掉 hash 噪音后，前端代码模块的真实增删”。
