# Codex 桌面版 26.519.31651 macOS / Windows 对照

> 第三方跨平台变更日志 · 非官方

本批次可确认的共同变化集中在内置 browser 与 chrome 插件元数据版本同步；macOS 还出现了线程侧边面板相关前端模块的交互与布局线索变化。  
One-liner: Both packages mainly refresh bundled browser/chrome plugin metadata, with an additional macOS-only front-end signal around thread side-panel navigation.

## 关于这份报告

这份报告基于两份官方签名分发包逆向重建后的机器 diff：macOS arm64 为 `26.519.22136` build `3003` 到 `26.519.31651` build `3017`，Windows x64 为 `26.519.2736.0` 到 `26.519.3891.0`。两平台版本号和发布时间存在错位，因此每条结论均分别落到各自事实包，不把一个平台的发现套用到另一个平台。

证据等级分两档：【实证】来自 plist、manifest、插件 `plugin.json`、签名验证、包内结构统计和非“同名前端模块改动线索”的可读配置；【信号】来自同名前端模块的 hash 后内容差、模块名、体积变化、样式类名或可读字符串差异。平台归属使用 `[macOS+Windows]`、`[仅 macOS]`、`[仅 Windows]`。

本输入样本没有提供“新增内置插件明细”、Computer Use 运行时总量、前端代码模块真实增删明细、主进程和前端 bundle 新增可读字符串、内容安全策略变化、新增组件与资源、Windows AppxManifest 变化等可展开事实；这些部分只在原始数据中按“未见提供”处理。

## ✨ 重点变化

### 【实证】[macOS+Windows] browser 内置插件版本同步到 `26.519.31651`

两平台的 `browser/.codex-plugin/plugin.json` 都只显示版本字段变化：`26.519.22136` → `26.519.31651`。插件描述保持为面向 Codex 内置浏览器的自动化能力，包含打开、检查、导航、测试、点击、输入、截图本地 Web 目标，以及对 `localhost`、`127.0.0.1`、`::1`、`file://` 和当前内置浏览器标签页的处理说明。

平台差异是路径不同而语义相同：macOS 位于 `Contents/Resources/plugins/openai-bundled/plugins/browser/.codex-plugin/plugin.json`，Windows 位于 `resources/plugins/openai-bundled/plugins/browser/.codex-plugin/plugin.json`。事实包没有显示 browser 插件新增技能、服务入口、默认提示或权限变化，因此本条只能确认插件元数据版本同步，不能推断能力新增。

### 【实证】[macOS+Windows] chrome 内置插件版本同步到 `26.519.31651`

两平台的 `chrome/.codex-plugin/plugin.json` 同样只显示版本字段变化：`26.519.22136` → `26.519.31651`。描述仍指向 Chrome 自动化，覆盖远程 URL、依赖认证或用户配置文件的页面、已有 Chrome 标签页、cookies、扩展，以及 Codex Chrome Extension 设置。

平台差异同样只体现在路径：macOS 为 `Contents/Resources/plugins/openai-bundled/plugins/chrome/.codex-plugin/plugin.json`，Windows 为 `resources/plugins/openai-bundled/plugins/chrome/.codex-plugin/plugin.json`。事实包未给出 chrome 插件服务入口或技能文档变化，因此这里不扩展为新功能声明。

### 【实证】[macOS+Windows] 应用层 asar 内容有小幅结构变化，但文件总量保持稳定

macOS 的 `app.asar` 内部文件数为 `1576 → 1576`，显示新增 `15`、移除 `15`、改动 `9`；Windows 的 `app.asar` 内部文件数为 `1530 → 1530`，显示新增 `15`、移除 `15`、改动 `11`。两平台都明确提示 `webview/assets` 下大量增减是 Vite 构建 hash 重命名噪音，不能据此判断新功能。

这说明两平台都经历了应用层打包内容刷新，但在本事实包中，能落到人类可读配置的共同变化主要是 browser 和 chrome 插件版本字段。Windows 比 macOS 多 `2` 个 asar 改动文件，但事实包没有列出这些额外改动的具体文件名或内容，不能继续解释其含义。

### 【实证】[macOS] Info.plist 标记更新到 macOS 新版本与新 build

macOS `Info.plist` 显示 `/CFBundleShortVersionString` 从 `26.519.22136` 更新到 `26.519.31651`，`/CFBundleVersion` 从 `3003` 更新到 `3017`。`/ElectronAsarIntegrity/Resources/app.asar/hash` 也从旧 hash 更新为新 hash。

这是 macOS 包级版本与 asar 完整性记录的实证变化。Windows 事实包没有给出 AppxManifest 变化，因此不能在 Windows 侧写对应 manifest 结论。

### 【实证】[macOS+Windows] 外层文件树稳定，主要变化集中在内容改动

macOS 外层文件数为 `1335 → 1335`，新增 `0`、移除 `0`、内容改动 `59`；Windows 外层文件数为 `832 → 832`，新增 `0`、移除 `0`、内容改动 `12`。两平台最大组件均为 `app.asar`，旧新体积均显示 `159.0 MB → 159.0 MB`，变化为 `+0.00 MB`。

这支持“本批次不是通过新增大量外层文件交付”的判断。macOS 的内容改动数量明显多于 Windows，但事实包没有列出可解释这些差异的完整目录级明细，不能把数量差直接归因为某项功能。

### 【信号】[仅 macOS] 线程侧边面板的新标签入口与底部面板交互可能被调整或收敛

macOS 的同名前端模块 `thread-app-shell-chrome` 从 `webview/assets/thread-app-shell-chrome-qFOIuoul.js` 变为 `webview/assets/thread-app-shell-chrome-NUIUr2lx.js`，体积减少 `21417 bytes`。移除的可读字符串包括打开侧聊、浏览器标签、review 标签、terminal、timeline、workspace 文件浏览器等来自 thread side panel 或 panel New tab page 的 action label，以及 `Bottom panel`。移除样式或状态类名包括 `aria-hidden`、`aria-label`、`aria-pressed`、`bg-green-500`、`cursor-interaction`、`data-thread-side-panel-new-tab-action-label`、`flex-1`、`flex-col`、`focus-visible:outline` 等。

由于这些证据来自“同名前端模块改动线索”，只能标为【信号】：它强烈指向线程 shell、侧边面板、新标签页入口、底部面板、可访问性状态和焦点样式的变化，但不能断言最终 UI 删除了某个按钮或所有用户都不可见。Windows 事实包没有提供同名前端模块改动线索，因此本条只归属 macOS 输入样本。

### 【实证】[macOS+Windows] cua_node 依赖未新增

两平台的 `cua_node` 依赖变化均显示新增 `0` 个包。事实包没有给出移除包或版本变更明细，因此只能确认“新增依赖为零”，不能进一步推断运行时能力没有变化。

### 【实证】[macOS] 签名与公证验证均为 accepted

macOS 上一版与最新版全量包均给出公证验证结果：`accepted`，来源为 `Notarized Developer ID`，签名来源为 `Developer ID Application: OpenAI OpCo, LLC (2DC432GLL2)`。事实包还给出两个全量包 SHA-256，分别为 `66c314178d0505ded059f7de4ab64ed1f897858d9127c814d1270fe864a80111` 与 `80c4e7c3073dbfd1107e323d7de11d657837ddb9831b2d99b010b3372bce9e4a`。

Windows 事实包提供两个全量包 SHA-256，但没有提供等价签名验证输出。因此签名与公证状态只在 macOS 侧成条。

`aria-hidden`、`aria-label`、`aria-pressed`、`focus-visible:outline`、`focus-visible:outline-2` 等类名或属性线索被列为移除集合。证据边界是前端构建产物的同名模块差异，适合提示可访问性状态或焦点处理有改动，但不足以判断交互结果。

### 【信号】[仅 macOS] 底部面板布局线索

`Bottom panel`、`flex-1`、`flex-col`、`bg-token-bg-secondary`、`bg-token-main-surface-primary` 等线索与面板布局和表面样式有关。证据边界仍是模块级差异，不能上升为确定 UI 设计变更。

### 【实证】[macOS+Windows] hash 噪音被排除

两份事实包都说明 `webview/assets` 下大量增减属于 Vite 构建 hash 重命名噪音。因此本报告没有把 hash 文件名变化计为功能增减，只保留具备可读字符串、配置或统计支撑的变化。
