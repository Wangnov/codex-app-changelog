# Codex 桌面版 `26.519.31651`(build 3017)

> 第三方变更日志 · 非官方 · 基于官方更新包逆向差异

本版从产物层面看是一次小幅维护更新，主要可见变化集中在版本号与打包完整性信息同步。  
English one-liner: a very small maintenance build with no evidence of user-facing feature additions.

## 关于这份报告

这不是 OpenAI(人工智能公司) 官方变更日志，而是一份**基于官方分发产物的逆向差异报告**。本次是历史回填：两个版本均超出官方 Sparkle(应用自动更新框架) 增量窗口，因此改为下载两个官方全量包后直接比较，并生成结构化 bundle(应用包) 差异。

可信度方面，本次信任根来自 Apple(苹果公司) 公证与 OpenAI Developer ID(开发者身份签名)。上一版与最新版均通过 `spctl`(macOS 安全评估工具) 公证验证，来源均为 `Notarized Developer ID`，签名主体均为 `Developer ID Application: OpenAI OpCo, LLC (2DC432GLL2)`。

证据等级分两档：

- **【实证】**：来自人类可读的直接证据，例如 `Info.plist`(macOS 应用信息配置文件)、插件配置 diff(差异文本)、签名与公证结论。
- **【信号】**：来自间接证据，例如前端模块名或二进制体积变化；只能说明“指向某方向，但无法从产物还原到源码级意图”。

对比范围：`26.519.22136` build `3003`，发布于 `2026-05-21`；到 `26.519.31651` build `3017`，发布于 `2026-05-22`。平台为 `macOS arm64`。

## ✨ 重点变化

### 1. 应用版本与构建号推进到 `26.519.31651` / build `3017` 【实证】

这是一次版本推进：`CFBundleShortVersionString` 从 `26.519.22136` 变为 `26.519.31651`，`CFBundleVersion` 从 `3003` 变为 `3017`。

这意味着 macOS 与应用自身看到的版本元数据已经更新，但事实包没有提供足够证据说明新增了具体用户功能。

关键证据：

```diff
/CFBundleShortVersionString: '26.519.22136' → '26.519.31651'
/CFBundleVersion: '3003' → '3017'
```

### 2. 内置 Chrome 与 Browser 插件版本号随主应用同步 【实证】

内置 `chrome` 与 `browser` 插件的 `.codex-plugin/plugin.json` 版本号都从 `26.519.22136` 更新到 `26.519.31651`。

这说明这两个随应用打包的浏览器相关插件与主应用版本保持同步；事实包中没有显示它们的描述、能力边界或接口文本发生变化，因此不应解读为新增浏览器自动化能力。

关键证据：

```diff
"version": "26.519.22136" → "26.519.31651"
```

涉及插件：

| 插件 | 可见变化 |
| --- | --- |
| `chrome` | 插件版本号同步到 `26.519.31651` |
| `browser` | 插件版本号同步到 `26.519.31651` |

### 3. `app.asar` 完整性哈希变化，但未体现为可描述的新功能 【实证】

`ElectronAsarIntegrity` 中 `app.asar`(Electron 应用资源归档) 的哈希发生变化，说明应用资源包内容确实不同。

不过，文件级统计显示 bundle 总文件数保持 `1335 → 1335`，新增 `0`、移除 `0`、内容改动 `59`；关键组件体积也基本不变。事实包还明确提示 `webview/assets` 与 `.vite/build` 下带 hash 的文件名增减属于构建产物重命名噪音，因此不能据此归纳新功能。

关键证据：

