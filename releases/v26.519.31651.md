# Codex 桌面版 26.519.31651(macOS / Windows 对照)

> 第三方跨平台变更日志 · 非官方

本批次更像一次跨平台应用层同步更新：两端都有 `app.asar` 内容变动，但可读证据很少，macOS 端额外暴露了插件元数据版本更新。  
One-liner: both platforms refreshed the application bundle, while only macOS exposes confirmed Browser and Chrome plugin metadata bumps.

## 关于这份报告

本报告基于官方签名分发包的逆向 diff(差异对比)：macOS 来自 Sparkle(应用更新框架)相关全量包对比，Windows 来自 MSIX(微软应用安装包)对比。macOS 对比版本为 `26.519.22136` build `3003` → `26.519.31651` build `3017`；Windows 对比版本为 `26.519.2736.0` → `26.519.3891.0`。

证据分两档：【实证】表示来自 `Info.plist`、插件 `plugin.json`、依赖清单、清单文件等可读证据；【信号】表示仅能从文件名、文件数量、模块体积或打包产物变化推断方向，不能还原源码意图。

平台归属标记含义：`[macOS+Windows]` 表示两平台事实包均出现；`[仅 macOS]` 表示只在 macOS 事实包可见；`[仅 Windows]` 表示只在 Windows 事实包可见。

## ✨ 重点变化

### 【信号】[macOS+Windows] 应用层打包内容同步刷新

两平台的 `app.asar`(Electron 应用归档) 都发生了内部文件变化，且总体体积仍显示为 `159.0 MB → 159.0 MB`、变化 `+0.00 MB`。macOS 内部文件为 `1576 → 1576`，新增 `15`、移除 `15`、改动 `9`；Windows 内部文件为 `1530 → 1530`，新增 `15`、移除 `15`、改动 `11`。

平台差异在于：两端都有同规模的新增/移除数量，但 Windows 的改动文件数比 macOS 多 `2` 个。事实包同时说明 `webview/assets` 与 `.vite/build` 相关 hash(内容指纹)重命名属于构建噪音，因此这里不把这些变化解读成具体功能增减。

### 【实证】[仅 macOS] Browser 与 Chrome 插件元数据版本同步

macOS 包内两个 OpenAI bundled plugin(内置插件)的 `plugin.json` 出现可读 diff：

`browser` 插件版本从 `26.519.22136` 更新为 `26.519.31651`；`chrome` 插件版本同样从 `26.519.22136` 更新为 `26.519.31651`。

平台差异在于：Windows 事实包没有列出对应插件 `plugin.json` 的可读 diff，因此本报告只能确认 macOS 端有该元数据更新；不能据此断言 Windows 端也做了相同插件元数据变更。

### 【实证】[仅 macOS] macOS 应用版本与完整性哈希更新

macOS `Info.plist`(应用属性清单)显示：`CFBundleShortVersionString` 从 `26.519.22136` 更新到 `26.519.31651`，`CFBundleVersion` 从 `3003` 更新到 `3017`，`ElectronAsarIntegrity` 中 `app.asar` 的哈希也从 `05f4d31d74bb97e851579022d9d33bd852097d21d9753290762572a61c8e39f7` 变为 `72b2210d1bd4dc5d59a99f1c677e304418cbf1ae60db818d75d6225a43c2d6c1`。

平台差异在于：这是 macOS bundle(应用包)层面的清单证据；Windows 事实包仅给出版本号与包级统计，没有提供 `AppxManifest`(Windows 应用清单)字段 diff。

### 【实证】[macOS+Windows] 本批次没有新增 cua_node 依赖

两平台事实包均显示 `cua_node` 新增包数量为 `0`。这说明至少从依赖清单层面看，本批次没有新增该依赖组的包。

平台差异：无。macOS 与 Windows 在这一项上保持一致。
