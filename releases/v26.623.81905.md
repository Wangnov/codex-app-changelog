# Codex 桌面版 26.623.81905 macOS / Windows 对照

> 第三方跨平台变更日志 · 非官方

本批次的共同实证变化集中在内置 browser/chrome 插件版本同步；macOS 包内还出现了更密集的前端 UI 与工作流信号。  
One-liner: shared plugin version bump, with broader macOS-side frontend signals around remote environments, credits, onboarding, shortcuts, announcements, and code rendering.

## 关于这份报告

本报告基于两份官方签名分发包逆向重建后的机器差异事实：macOS arm64 从 `26.623.70822` build `4559` 到 `26.623.81905` build `4598`，Windows x64 从 `26.623.9142.0` 到 `26.623.11225.0`。发布日期按输入批次使用 `2026-06-27`。

证据分为两档：【实证】来自 manifest、plist、CSP、插件配置、服务入口、技能文档、依赖清单、明确可读字符串或关键文件 diff；【信号】来自同名前端模块内容变化、模块名、体积变化和样式类名变化。平台归属以 `[macOS+Windows]`、`[仅 macOS]`、`[仅 Windows]` 标注。webview/assets 与 `.vite/build` 的 hash 改名不作为功能增减依据。

## ✨ 重点变化

### 【实证】[macOS+Windows] browser 与 chrome 内置插件版本同步到 26.623.81905

两平台都把 `browser` 与 `chrome` 两个 openai-bundled 插件的 `.codex-plugin/plugin.json` 版本号从 `26.623.70822` 更新到 `26.623.81905`。插件描述文本在事实包 diff 中未显示功能文案变化。

browser 插件仍指向 Codex in-app browser，用于打开、检查、导航、测试、点击、输入或截图 localhost、127.0.0.1、`file://` 与当前 in-app browser tab 等本地 Web 目标。chrome 插件仍描述为依赖用户现有 Chrome 状态的自动化能力，包括 tabs、登录会话、cookies、extensions 与 Codex Chrome Extension setup。

平台差异：macOS 只显示 browser/chrome 两个插件版本变化；Windows 还显示 computer-use 插件版本变化。

### 【实证】[仅 Windows] computer-use 插件版本同步

Windows 包内 `resources/plugins/openai-bundled/plugins/computer-use/.codex-plugin/plugin.json` 的版本号从 `26.623.70822` 更新到 `26.623.81905`。描述仍为通过 Computer Use 从 Codex 控制 Windows 桌面应用。

macOS 事实包没有对应 computer-use 插件 manifest diff，因此不能把这条扩展到 macOS。

### 【实证】[仅 macOS] 应用版本、build 与 app.asar 完整性更新

macOS `Info.plist` 中 `CFBundleShortVersionString` 从 `26.623.70822` 到 `26.623.81905`，`CFBundleVersion` 从 `4559` 到 `4598`，`ElectronAsarIntegrity/Resources/app.asar/hash` 从 `cc1442eaf30d8034a1fb22ed06f9b62efa79e6d6d629e18b0b1af568c5d11baf` 到 `f385081f252f4f4b18b9b0d0129fbb50e8ca6b2569065235a009c14d94959f01`。

Windows 输入未提供 AppxManifest diff，只能引用其版本范围与文件统计，不能补写 manifest 字段变化。

### 【信号】[仅 macOS] 远程、本地与 worktree 会话环境标识更明确

macOS 同名前端模块 `remote-conversation-page~projects-index-page` 新增字符串：`This conversation is running in Codex Cloud.`、`This conversation is running in a local git worktree.`、`This conversation is running locally.`，以及 cloud/local/worktree environment icon tooltip 相关键名。

这说明包内进入了会话运行环境图标与提示文案，但证据来自前端模块改动线索，应视为 UI 信号，不能断言所有用户已经可见。

### 【信号】[仅 macOS] 额度、加购 credit 与 rate-limit reset UI 信号增强

macOS `remote-conversation-page~hotkey-window-thread-page~keyboard-shortcuts` 相关模块体积从约 `0.0 MB` 到 `0.2 MB`，新增 `5 hour usage limit`、`Add credits`、credit reload dialog、monthly reload spend limit、Codex usage modal、rate limit reset modal、redeem rate limit reset 等字符串。

`home-announcements` 也新增 gift credits home banner、rate limit reset home banner、Fast mode home banner、multi-agent composer banner 与 remote connections banner 的 dismiss 文案。它们是明确进入包内的可读字符串，但因出现在同名前端模块改动线索中，本报告按【信号】处理。

### 【信号】[仅 macOS] 自动化页出现本地 agent setup 导入与环境设置线索

macOS `automations-page` 新增 `Agent environment`、`Agent setup imported`、`Auto detect`、`Bring over your setup, projects, and recent chats`，以及检查、导入另一个 local agent setup、迁移 home-scoped external config 项的按钮文案。

同一模块移除了部分旧的项目选择、技能浏览、归档聊天与 approve request 文案。整体指向自动化页或相关入口正在调整本地 agent 配置导入、环境选择和迁移流程，但仍属于前端模块信号。

### 【信号】[仅 macOS] 新线程、远程连接、onboarding、workspace 与快捷键相关 chunk 重新编排

macOS 去 hash 噪音后的新增模块名中出现 `keyboard-shortcuts-settings`、`remote-connections`、`new-thread-panel-page~home-announcements~upgrade-plan-dialog`、`onboarding-page~appearance-settings~general-settings`、`onboarding-page~select-workspace-page~debug-window-page~debug-modal`、`settings-page~codex-mobile-page~debug-window-page~remote-connections` 等组合。

这些模块名说明相关页面被重新切分或加载路径发生变化。由于没有同等级的配置或路由声明支撑，这里只作为快捷键设置、远程连接、onboarding、workspace 选择、升级弹窗与 debug 相关 UI 的结构信号。

### 【信号】[仅 macOS] appgen、publication terms、plugin detail 与 pull requests 页面继续重组

macOS 新增模块包含多组 `appgen-library-page`、`appgen-settings-page`、`appgen-publication-terms-route`、`plugin-detail-page`、`pull-requests-page`、`worktree-init-v2-page` 组合；同时移除了旧的 `appgen-library-page`、若干 appgen/publication terms/plugin detail/pull requests 组合 chunk。

这更像前端 chunk 拆分、依赖共享或页面加载关系调整，而不是单个新增功能实证。高价值点在于 appgen、插件详情、发布条款、PR 与 worktree init 仍是本批次主要前端改动区域。

### 【信号】[仅 macOS] 代码语法高亮或编辑器语言资源明显调整

macOS 多个语言模块新增或调整 TextMate 类字符串：JSX、LESS、Python、YAML、PostCSS、Go、TeX、GDScript、Lua、SDBL、CSS 等模块增加大量 scope；WASM 模块体积从 `0.6 MB` 降到约 `0.0 MB` 但仍出现 WAT 相关字符串；Java、Turtle、XML、Haml 等模块移除部分 scope。

这表明包内代码展示或编辑器语法资源发生重排。由于证据来自同名前端模块改动线索，应按渲染/高亮能力的弱信号处理。

### 【信号】[仅 macOS] presentation 模块加入 pptx/protobuf 相关字符串

macOS `presentation` 模块体积从约 `0.0 MB` 到 `0.3 MB`，新增 `oaiproto.coworker.pptx`、`invalid varint`、`wire type`、`premature EOF` 等 protobuf 解析相关字符串。

这说明包内出现了 PPTX coworker/protobuf 相关代码路径信号，但不能仅据此断言完整演示文稿功能开放。

### 【实证】[仅 macOS] CSP 文本在 index.html diff 中保持同一策略形态

macOS `webview/index.html` 的关键 diff 显示入口脚本与 preload/style 资源换名或换 chunk，同时保留 CSP：`default-src 'none'`，允许 self/app/blob/data/https 图片与媒体，允许 web-sandbox frame/child，script-src 包含 self、一个 sha256 与 `wasm-unsafe-eval`，connect-src 包含 `ab.chatgpt.com`、Mapbox、`cdn.openai.com` 与 ChatGPT websocket 域。

事实包没有列出 CSP 字段级变化，因此这里只能说本 diff 片段中 CSP 策略文本仍在包内，不能声称 CSP 有新增放宽或收紧。

## 🔎 低层信号索引

`onboarding-page`、`automations-page`、额度相关模块新增大量 `aria-*`、`aria-invalid`、`aria-live`、`aria-controls`、`aria-describedby`、`aria-labelledby`、`aria-valuenow` 等类名。边界：这是 UI 状态与可访问性实现信号，不等于可见功能实证。

`home-announcements` 新增 ambient suggestions connected apps、multi-agent composer、remote connections、Fast mode、gift credits、rate limit reset banner 的 dismiss 或 body 文案。边界：字符串进入包内，但是否展示取决于运行时开关或后端配置。

### [仅 macOS] 插件详情与技能页旧文案收缩

`plugin-detail-page` 移除 workspace share、MCP server enablement toggle、section navigation、`Add people or groups` 等字符串；`skills-page` 移除 create skill、reload skills list、installed skills、filtered empty state 等字符串。边界：仅能说明对应旧前端 bundle 文案减少，不能断言功能被删除。

`general-settings` 移除 default permissions toggle、work mode settings、base code size、hold-to-dictate hotkey capture、hotkey window capture 等字符串。边界：这是模块内字符串移除信号，可能来自重构、迁移或延迟加载变化。

### [仅 macOS] avatar、pets、native overlay 相关加载路径更新

新增模块名包含 `avatar-overlay-page~avatar-overlay-native-page~pets-settings`，`avatar-overlay-composition-surface.html` 也大幅替换 preload 列表。边界：这主要是加载图谱变化，不能独立当作 avatar 或 pets 功能变化实证。

### [仅 macOS] 旧 analytics/middleware 字符串减少

`middleware` 移除 `Loaded a Page`、`Loaded a Screen`、`context.device`、`context.ip`、`context.traits.email` 等字符串。边界：只表示该 bundle 中相关可读字符串减少，不足以判断遥测行为改变。
