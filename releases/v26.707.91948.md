# Codex 桌面版 26.707.91948 macOS / Windows 对照

> 第三方跨平台变更日志 · 非官方

本批次的共同主线是浏览器能力、近期任务入口和 Full Access 权限交互重整，但 macOS 与 Windows 的前端能力面明显错位。

Browser automation, recent-task navigation, and Full Access controls moved forward on both platforms, while macOS emphasized browser settings and rich artifacts and Windows emphasized Computer Use, Office control, and task-side-panel actions.

## 关于这份报告

本报告比较两组官方签名分发产物：macOS arm64 从 26.707.72221（build 5307）到 26.707.91948（build 5440），Windows x64 MSIX 从 26.707.9981.0 到 26.707.12708.0。两个版本序列分别引用，不以其中一端推断另一端的发布时间或版本对应关系。

本次属于历史回填，事实包使用两个官方全量包直接比较。macOS 两个样本均通过公证验证，结果为 `accepted`，签发主体均为 OpenAI OpCo, LLC；输入没有给出 Windows 签名验证结果。

证据分为两档：

- 【实证】来自 plist、manifest、插件描述、依赖清单、文件统计和非“同名前端模块改动线索”区块中的明确差异。
- 【信号】来自前端模块名、同名压缩模块中的字符串或样式变化，以及体积方向。它说明相关代码或文案进入、离开或重组了包，但不证明功能已向所有用户开放。

平台标记为 `[macOS+Windows]`、`[仅 macOS]` 和 `[仅 Windows]`。Vite hash 文件改名已按要求排除，不作为功能增减依据。

事实包没有提供完整的“内置插件清单变化”“主进程 bundle 新增字符串”或“新增组件与资源”明细；Windows 也没有提供 AppxManifest 或 CSP 差异。因此，本报告只归纳实际出现的插件 manifest、前端线索和文件级统计。

## ✨ 重点变化

### 浏览器与 Chrome 插件版本同步【实证】[macOS+Windows]

两端的内置 `browser` 和 `chrome` 插件 manifest 均把插件版本从 26.707.72221 更新到 26.707.91948。Windows 外层 MSIX 版本为 26.707.12708.0，因此不能把插件内部版本直接当作 Windows 包版本。

`browser` manifest 继续明确覆盖应用内浏览器、本地地址、当前浏览器标签页，以及打开、检查、导航、测试、点击、输入和截图等操作；`chrome` manifest 则面向依赖现有 Chrome 标签页、登录会话、Cookie、扩展和 ChatGPT Chrome Extension 配置的自动化，并把 Chrome 定位为专用连接器或 API 不可用时需用户明确批准的后备路径。

事实包未显示这两个插件为本批次新增，只确认其 manifest 版本随代码批次更新。

### 近期任务、云任务与 Chrome 入口重组【信号】[macOS+Windows]

两个平台的 `header` 模块出现相同的新字符串集合，包括近期任务菜单、未读任务更新、任务搜索、`All tasks`、`Cloud tasks`、环境筛选、应用设置、`ChatGPT for Chrome` 和 `Chrome computer use settings`。

这表明头部导航正在容纳近期任务状态、云任务过滤和 Chrome 相关入口。由于证据来自同名前端模块差异，不能断言这些入口已对所有账号可见。

### Full Access 与 Ultra reasoning 的冲突处理【信号】[macOS+Windows]

两端都进入了围绕 Full Access 的新确认文案：

- Full Access 可访问网站、发送数据并使用已启用插件；
- 选择 Ultra reasoning 时，可保留 Full Access 或切换权限模式；
- 没有兼容权限模式时，Ultra reasoning 可能不可选；
- 权限面明确提到互联网和已连接应用。

Windows 的 `composer` 还包含 `Full access is on`，并描述可运行命令、安装软件和更改系统设置；`general-settings` 新增了在 composer 中开放 Full Access 前的确认标题。macOS 对应线索位于新线程、AppGen、快捷窗口等共享模块中，并额外出现 Codex Cloud 因未通过 ChatGPT 登录或工作区未启用 Codex 而不可用的原因文案。

这些都是同名前端模块中的字符串变化，因此整体定级为【信号】，不是权限默认值已变更的实证。

### Computer Use 与 Office 实时控制进入 @ 提及能力面【信号】[仅 Windows]

Windows 的 `at-mention-list` 新增了 `Computer use`、`Desktop apps`、`Custom agents`、`Live agents`、`Files and tasks`、ChatGPT conversations，以及 Excel、PowerPoint 的实时工作簿和演示文稿控制文案。

样式状态同时出现 `mcp-capability`、`mcp-capability-mentions`、`microsoft-excel-plugin` 和 `microsoft-powerpoint-plugin`。这组相互印证的线索指向 @ 提及列表正在组织 Computer Use、MCP 能力和 Microsoft Office 实时控制入口，但仍不能证明所有入口已经开放。

### Windows Computer Use 插件随批次更新【实证】[仅 Windows]

Windows 内置 `computer-use` 插件 manifest 的版本从 26.707.72221 更新到 26.707.91948。其描述明确为通过 Computer Use 从 ChatGPT 控制 Windows 桌面应用。

本批次没有新增 `cua_node` 依赖，Computer Use 运行时总量也保持 287.5 MB，因此这是插件版本和应用层整合更新，不是运行时依赖扩张。

### 浏览器设置、历史记录与下载入口扩展【信号】[仅 macOS]

macOS 的 `browser-use-settings` 模块新增了：

- 无需询问即可访问历史记录的选项文案；
- 内嵌浏览器下载与历史记录入口；
- 联系信息、扩展、密码管理器和站点设置；
- 浏览器数据清理选项的展开与收起；
- 查看下载历史的按钮。

`onboarding-page` 同时出现浏览器 tweak 编辑器的横向、纵向、反向排列、对齐和分布选项，以及 `selected-browser-element`、`selected-browser-region`、`selected-browser-text` 状态类。这是浏览器设置与页面调整工具继续细化的信号。

### 任务侧栏支持浏览器标签操作与延续任务【信号】[仅 Windows]

Windows 的 `thread-side-panel-tabs` 新增了新建标签、在当前标签右侧新建、重新加载、复制浏览器标签，以及从当前任务与标签继续创建新任务的文案。

延续任务可选择同一 worktree、新 worktree 或新的本地任务。字符串还明确出现任务侧栏中的空白浏览器标签和 review 标签标题，说明浏览器标签与任务上下文之间的操作面正在扩展。

### 多代理 composer 提示进入包内【信号】[仅 macOS]

macOS 的 `home-announcements` 新增多代理 composer 横幅、关闭状态和说明文案：

`Delegate work to subagents that work in parallel. Note: may increase token usage.`

同一模块还出现 Fast mode、赠送额度、速率限制重置和后端驱动首页横幅的关闭标签。由于这些字符串来自同名前端模块差异，只能确认相关提示与状态逻辑进入了前端包，不能断言横幅已面向所有用户展示。

### 首页项目、附件、听写与个性化线索【信号】[仅 macOS]

macOS 主页面相关模块新增了：

- `Add local project` 和首页项目选择器的无障碍标签；
- `Add photos`、向当前 composer 添加照片；
- `hold-to-dictate` 状态；
- `Adjust tone and response style`；
- 多组工作风格或形象文案；
- Codex Micro 电池充电状态标签。

同时新增或重组了 `codex-micro-onboarding-host`、Codex Micro 设置与键盘快捷键、个性化设置、宠物设置、头像浮层、快速聊天窗口和本地会话模块。这些共同指向首页输入、语音、个性化和轻量浮层体验的持续重整，但模块增删本身不等于功能开放。

### 图片、PDF 与批注交互细化【信号】[仅 macOS]

macOS 的 onboarding 共享模块加入了用户评论截图缩略图、所选文档或 PDF 内容预览，以及从 composer 移除响应批注的无障碍文案。

这些字符串说明图片、文档、PDF 和批注相关的交互路径进入了前端构建，但没有文件类型上限、上传限制或正式可用性实证。

### 富文档、工作簿、演示文稿与数学渲染扩展【信号】[仅 macOS]

macOS 的同名模块变化显示：

- `workbook` 从接近 0 MB 增至 2.6 MB，并出现工作簿范围、主题颜色和大量版式数据属性；
- `workbook-from-markdown` 增加序列化、路径、插件列表与查找替换相关错误；
- `presentation` 增至 0.3 MB，并出现 `oaiproto.coworker.pptx` 及协议解析错误；
- `katex` 增至 0.3 MB，加入数学标记语言、宏定义、控制序列和 MathML 相关内容；
- 新增 `artifact-tab-content` 相关前端模块，并重组 notebook preview、pull request 和 code review 共享模块。

这是一组较强的富文档与 artifact 渲染方向信号，但事实包没有提供对应服务入口、命令或用户可见发布说明。

### 代码与差异渲染语法覆盖扩大【信号】[仅 macOS]

macOS 的 `diff` 模块加入 changed、deleted、inserted、Git header、range 和文件路径等语法作用域。PHP、Stylus、Shell、Go、TeX、R、XML、GraphQL、SDBL、GLSL 与 WAT/WebAssembly 等模块也出现大量新增语法作用域和 token 名称。

这些变化指向语法高亮或代码展示资源更新。WASM 模块本身体积从 0.6 MB 降至接近 0 MB，因此不能简单解读为所有语言模块都在扩容。

### 对话式 onboarding 执行约束进入前端包【信号】[仅 macOS]

macOS 的 onboarding/debug 共享模块新增了一组任务执行提示，包括立即执行选定任务、不再询问后续问题、不调用交互式 onboarding 工具、以简明方式提供进度，以及最后只给一条简洁结果。

同一模块还包含 onboarding 已在启动、插件未就绪、任务隐藏或启动取消等状态。这些是对话式 onboarding 流程和错误路径进入包内的信号，不证明它们属于稳定公开界面。

### 本地、远程与快速聊天模块重新切分【信号】[仅 macOS]

去除 hash 噪音后，macOS 新增了独立的 `local-conversation-page`、`remote-conversation-page~local-conversation-page`、`projects-index-page~remote-conversation-page`、`quick-chat-window-page` 及 ChatGPT conversation 组合模块。

同时，远程连接设置、使用量设置、代码审查设置、技能、插件和个性化设置出现新的共享 chunk 组合。由于大量旧组合模块也被移除，这更像前端代码切分与路由依赖重排，不能据此断言出现了全新的本地或远程会话产品。

### 插件分享、AppGen 设置与任务归档文案收缩【信号】[仅 macOS]

若干同名模块移除了可读字符串：

- `plugin-detail-page` 移除插件分享对象搜索、链接共享、MCP server 启用开关和页面导航文案；
- `appgen-settings-page` 移除自定义域名、站点标题、站点 slug、环境变量和 secret 输入相关文案；
- 项目与线程共享模块移除任务置顶、取消置顶、归档确认和定时任务归档提示；
- 技能与插件设置共享模块移除搜索框清除文案。

这只证明相关字符串不再位于这些同名模块中。它们可能被删除、迁移或重新切分，不能直接表述为对应功能已下线。

### Windows RPC 与警告呈现基础更新【信号】[仅 Windows]

Windows 的 `rpc` 模块新增了 RPC stub 生命周期、ReadableStream/WritableStream stub、session、promise export ID 和 pipe 创建等错误字符串，并出现 `workers-module` 状态类。

前端还新增独立 `triangle-warning` 模块，composer 增加输入验证错误背景和错误文字样式。它们共同构成远程调用错误处理与警告呈现正在调整的弱至中等信号。
