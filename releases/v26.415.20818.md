# Codex 桌面版 `26.415.20818` build 1727

> 第三方变更日志 · 非官方 · 基于官方更新包逆向差异

这一版最明确的变化，是桌面包内新增了 Computer Use / browser-use 相关组件，同时前端模块和可读字符串显示，文档预览、远程连接、自动化、插件管理与更新状态正在扩展。  
English one-liner: this build bundles new computer/browser-use assets and shows multiple UI signals around richer previews, remote control, automations, and plugin/update flows.

## 关于这份报告

这不是官方变更日志，而是一份**基于官方分发产物的逆向差异报告**。本次对比范围是 macOS arm64 的 `26.409.20454` build `1462` 到 `26.415.20818` build `1727`。

这组版本属于历史回填：官方 appcast 未对这一对版本提供 Sparkle 增量包，因此本报告改为下载两个官方全量包直接比较。两个 App bundle 的公证验证结果均为 `accepted`，来源为 `Notarized Developer ID`，签名主体为 `Developer ID Application: OpenAI OpCo, LLC (2DC432GLL2)`。

证据等级分两档：

- **【实证】**：来自人类可读的直接证据，例如 `package.json` diff、Info.plist、服务入口、插件文件、可读字符串或签名验证结果。
- **【信号】**：来自前端模块名、同名前端模块字符串差异或体积变化。它们指向某方向，但无法从产物还原到源码级意图，也不能证明功能已对所有用户开放。

`webview/assets` 和 `.vite/build` 下带 hash 的文件名变化主要是构建产物重命名。本报告只使用事实包中已去噪的真实新增模块和关键 diff，不把大量 hash 文件增删解释为功能增删。

## ✨ 重点变化

### 1. 内置 Computer Use / browser-use 组件随包出现【实证】

新版 bundle 新增了 `Contents/Resources/plugins/openai-bundled/plugins/computer-use/` 下的 `Codex Computer Use.app`，其中包含 `SkyComputerUseService` 和 `SkyComputerUseClient` 两个可执行组件；同时出现 `browser-use` 与 `chrome` 插件目录中的 `browser-client.mjs` 和技能文档。

这意味着桌面端正在补齐与浏览器任务、计算机控制相关的本地执行栈。是否默认开放给所有用户，不能仅凭产物确认。

关键证据：

```text
Contents/Resources/plugins/openai-bundled/plugins/computer-use/Codex Computer Use.app/Contents/MacOS/SkyComputerUseService  11.9 MB
Contents/Resources/plugins/openai-bundled/plugins/computer-use/Codex Computer Use.app/Contents/SharedSupport/SkyComputerUseClient.app/Contents/MacOS/SkyComputerUseClient  10.5 MB
Contents/Resources/plugins/openai-bundled/plugins/browser-use/scripts/browser-client.mjs  0.6 MB
Contents/Resources/plugins/openai-bundled/plugins/chrome/scripts/browser-client.mjs  0.6 MB
```

`package.json` 也新增了：

```diff
+    "browser-common": "link:../../../lib/browser_use/browser-common",
```

### 2. 文档、表格和演示预览面板开始成形【信号】

去掉 hash 噪音后，前端真实新增模块出现了 `pdf-preview-panel`、`docx-preview-panel`、`docx-preview`、`document`、`presentation`、`spreadsheet`、`workbook`、`workbook-from-csv`、`workbook-from-markdown`、`PopcornElectronDocumentPanel`、`PopcornElectronPresentationPanel` 和 `PopcornElectronWorkbookPanel`。

这些模块名指向更丰富的 artifact / 文档预览方向，包括 PDF、DOCX、表格、演示和工作簿处理，但这仍是模块级信号，无法从产物还原到源码级意图。

相关信号还包括：

```text
artifact-preview-status
artifact-preview-zoom
PopcornPageNumberNavigation
popcorn-electron-crdt-bridge
comments
comments-collab-store
```

### 3. Telepathy 与远程连接相关路径进入桌面端【实证】

新版主进程/前端 bundle 新增可读字符串中出现了 `A configurar o Telepathy`，前端真实新增模块中也出现 `telepathy-setup-state`、`use-connected-remote-connections`、`remote-host-globe-icon` 和 `remote-text-edit-session`。

这指向 Telepathy、远程连接和远程编辑会话相关能力正在进入桌面端。模块名本身仍只构成方向信号，不能证明完整功能已上线。

同名前端 `general-settings` 模块还出现了远程控制相关文案线索，例如：

```text
Enable Remote Control on this Computer
Allows other signed-in Codex clients to connect to this computer
```

这部分属于同名前端模块改动线索，只能按【信号】理解。

### 4. 自动化与计划任务的状态文案增加【实证】

新版新增可读字符串中出现了与周期、消息调度和 Codex 任务启动相关的文案，例如 `A cada hora`、`A cada minuto`、`A agendar mensagem` 和 `A iniciar tarefa do Codex`。

这说明桌面端产物中新增了自动化/计划任务相关状态表达。结合 `app-server-manager-signals` 的同名前端模块线索，错误路径也更细，例如自动化不存在、调度状态无法更新、自动化只支持本地线程等；但这些错误字符串来自同名前端模块改动，只能视为【信号】级辅助证据。

### 5. 插件发现、启用与更新 UI 有明显调整【信号】

`plugins-page` 同名前端模块新增了插件 hero 轮播、插件启停失败 toast、App 更新下载/安装状态、关闭标签页等可读字符串：

```text
Downloading update
Installing update
Failed to update app
Failed to update plugin
Primary call to action shown in the plugin discover hero carousel
Toast message shown when enabling or disabling a plugin fails
Close tab
```

这些线索指向插件发现页、卡片网格、标签页和更新状态 UI 正在调整。由于证据来自同名前端模块改动，不能把它写成已确定的源码级实现。

### 6. 设置面板扩展到浏览器控制、Fast mode、memory 与个性化【信号】

`general-settings` 和 `personalization-settings` 的同名前端模块线索显示，设置项正在覆盖更多桌面能力：浏览器控制、推理速度、详情级别、memory 开关、reset memories、头像叠层和默认语气等。

代表性新增文案包括：

```text
Codex browser control
Allow Codex to control the in-app browser for browser tasks. Restart Codex after changing this setting.
Choose how quickly inference runs across chats, subagents, and compaction. Fast uses 2x plan usage
Detail level
Accessible label for enabling memories
Delete all Codex memories
Choose a default tone for Codex responses
Button that toggles the floating avatar overlay
```

这些内容指向设置面板的能力面扩展，但仍不能确认所有设置在生产环境中可见或可用。

### 7. 启动阶段加入 Intel build on Apple Silicon 警告逻辑【实证】

`.vite/build/bootstrap.js` 新增了对 `darwin`、`x64`、`sysctl.proc_translated` 的检测，并准备在 Apple Silicon Mac 上运行 Intel build 时弹出警告。

关键新增文案：

```text
{appName} is running the Intel build on an Apple Silicon Mac
This build works through Rosetta, but the Apple Silicon build launches faster and performs better.
Quit
Continue Anyway
```

当前对比对象是 macOS arm64 包，这段逻辑在 arm64 运行路径下不会触发；它更像是共享启动代码中的跨架构保护。

### 8. 开发与打包链新增 Owl / MSIX 线索【实证】

`package.json` 新增了 `owl` 相关脚本，并引入 `electron-windows-msix`、`fs-extra` 和 `browser-common`。这说明构建/打包链在本版扩展了新的开发入口和 Windows MSIX 相关依赖。

关键 diff：

```diff
+    "owl": "pnpm --dir .. run generate:third-party-notices && cross-env NODE_ENV=development node ./scripts/owl-shell.mjs run",
+    "build:owl": "cross-env PNPM_YES=true node ./scripts/owl-shell.mjs package",
+    "owl:ensure": "node ./scripts/owl-shell.mjs ensure",
+    "owl:package": "cross-env PNPM_YES=true node ./scripts/owl-shell.mjs package",
+    "electron-windows-msix": "2.0.4",
+    "fs-extra": "11.3.2",
+    "browser-common": "link:../../../lib/browser_use/browser-common",
-    "socks-proxy-agent": "^8.0.5",
```
