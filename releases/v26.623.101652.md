# Codex 桌面版 26.623.101652 macOS / Windows 对照

> 第三方跨平台变更日志 · 非官方

本批次核心是同一套 Codex 内嵌应用版本推进到 26.623.101652 / build 4674，但 macOS 与 Windows 打包层版本号、前端改动暴露程度明显不同。  
One-liner: Codex desktop advances the embedded app and bundled browser automation plugins, with broad macOS UI signals and a much narrower Windows diff.

## 关于这份报告

本报告只基于官方签名分发包逆向重建后的机器提取差异。macOS 样本为 26.623.81905 build 4598 → 26.623.101652 build 4674；Windows MSIX 样本为 26.623.11225.0 → 26.623.13972.0，但其内嵌 `package.json` 与插件 manifest 显示应用版本从 26.623.81905 / build 4598 更新到 26.623.101652 / build 4674。

证据等级分两档：【实证】来自 plist、manifest、package.json、插件清单、依赖清单、服务入口、签名信息、明确配置与非“同名前端模块改动线索”的可读字符串；【信号】来自同名前端模块改动线索、模块名、体积变化、样式类名和压缩 bundle 方向线索。平台归属使用 `[macOS+Windows]`、`[仅 macOS]`、`[仅 Windows]` 标记。

对比范围是 macOS arm64 Sparkle 全量包与 Windows x64 MSIX 全量包。webview/assets 与 `.vite/build` 的 hash 改名按噪音处理，不作为功能增减依据。

## ✨ 重点变化

### 【实证】[macOS+Windows] 内嵌应用版本推进到 26.623.101652 / build 4674

两端 `package.json` 均显示 `version` 从 `26.623.81905` 更新到 `26.623.101652`，`codexBuildNumber` 从 `4598` 更新到 `4674`。

平台差异在打包层：macOS `Info.plist` 同步把 `CFBundleShortVersionString` 更新为 `26.623.101652`、`CFBundleVersion` 更新为 `4674`；Windows 的版本与渠道区块显示 MSIX 包版本为 `26.623.11225.0` → `26.623.13972.0`，与内嵌应用版本不是同一个编号体系。

### 【实证】[macOS+Windows] Browser 与 Chrome 内置插件版本随应用版本刷新

两端内置 `browser` 与 `chrome` 插件的 `.codex-plugin/plugin.json` 都从 `26.623.81905` 更新到 `26.623.101652`。插件描述文本本身未在 diff 中显示能力语义变化，但 manifest 版本更新是明确进入包内的事实。

`browser` 插件仍描述为面向 in-app browser、`@browser` / `@browser-use`、本地目标打开、检查、导航、测试、点击、输入和截图的能力；`chrome` 插件仍描述为依赖用户现有 Chrome 状态的自动化能力，包括标签页、登录会话、cookies、扩展和 Codex Chrome Extension 设置。

### 【实证】[仅 Windows] Computer Use 插件 manifest 版本刷新

Windows 侧额外出现 `resources/plugins/openai-bundled/plugins/computer-use/.codex-plugin/plugin.json` 的版本更新：`26.623.81905` → `26.623.101652`。其描述为通过 Computer Use 从 Codex 控制 Windows 桌面应用。

macOS 事实包没有列出同等 `computer-use` 插件 manifest diff；macOS 只给出 `cua_node` 运行时总量保持不变。

### 【实证】[macOS+Windows] Computer Use 运行时体量与依赖数量未变

macOS `cua_node` 总量为 `377.4 MB → 377.4 MB`，变化 `+0 MB, +0%`；Windows 为 `298.9 MB → 298.9 MB`，变化同为 `+0 MB, +0%`。两端 `cua_node` 依赖变化均为新增 `0` 个包。

这说明在本输入样本里，Computer Use 运行时文件总量和依赖集合没有体现扩张；Windows 的插件 manifest 版本刷新不等于运行时体量变化。

### 【信号】[仅 macOS] 前端路由和页面切分出现大规模重组

macOS `app.asar` 内部文件从 `2652` 变为 `2635`，去除 hash 噪音后有大量真实前端模块增删：新增包括 `settings-page`、`page`、`first-run~new-thread-panel-page`、`debug-window-page~debug-modal~home-ambient-suggestions-content`、多组 `remote-conversation-page`、`new-thread-panel-page`、`projects-index-page`、`worktree-init-v2-page`、`pull-requests-page` 与 `appgen` 组合模块。

这类证据只能说明 bundle 切分、页面路由或共享 chunk 结构发生变化，不能直接断言功能已对所有用户开放。但它与同名模块新增字符串共同指向设置、首启、远程会话、项目、AppGen、插件详情、调试窗口和新线程面板等区域都被触及。

### 【信号】[仅 macOS] Onboarding 与设置入口加入更多开发者文档和浏览器布局编辑信号

`app-initial~app-main~onboarding-page` 体积从 `5.5 MB` 降到 `3.5 MB`，新增字符串出现 browser tweak editor 的 flex direction、alignment、distribution 下拉标签；新增类名中包含 `https://developers.openai.com/apps-sdk`、`/codex/app/automations`、`/codex/app/chrome-extension`、`/codex/app/computer-use#locked-use`、`/codex/app/local-environments`、`/codex/concepts/sandboxing#how-you-control-it`、`/codex/config-basic`、`/codex/guides/agents-md/#create-global-guidance`、`/codex/hooks` 等链接。

这些是进入同名前端模块的信号，边界是：它们不能证明对应文档链接或设置项已经在稳定 UI 中可见，但说明 onboarding 或相关共享 chunk 正在携带更多开发者文档、自动化、Chrome extension、Computer Use、local environments、sandboxing、hooks 和全局指导相关入口。

### 【信号】[仅 macOS] Automations 页出现外部 agent 导入与自定义对话框信号

`app-initial~app-main~automations-page` 体积从 `0.7 MB` 增到 `0.9 MB`，新增字符串包括 external agent onboarding customize dialog、external agent import row、external agent provider toggle、external agent import options、external agent provider list 等可访问性标签，并出现多条 agent 形象描述文案。

同时移除字符串包括 `Agent environment`、`Auto detect`、默认终端位置、桌面应用更新下载与安装按钮标签等。这是 Automations 页交互与内容重排的信号，不构成外部 agent 功能全面上线的实证。

### 【信号】[仅 macOS] 远程连接状态在项目索引相关模块中更具体

`remote-conversation-page~projects-index-page` 新增字符串覆盖 SSH 远程连接状态：需要登录、远程 Codex 版本过旧、远程 Codex 可重启到更新版本、SSH 远程缺少 Codex CLI、正在安装或更新 Codex，以及 `Codex is not installed on this remote machine` / `Codex not installed`。

同一模块还新增项目 hover card 的打开来源、编辑、置顶、内联项目名编辑等可访问性标签。证据等级仍为【信号】，但主题很集中：项目列表、远程连接健康状态和远程 Codex 安装路径获得了更细的 UI 文案。

### 【信号】[macOS+Windows] Thread shell chrome 移除底部面板与侧栏新标签入口相关字符串

macOS 与 Windows 均在 `thread-app-shell-chrome` 同名模块中移除 `Bottom panel`、`(max-width: 920px)`，以及 side panel New tab page 中打开 History、side chat、browser tab、review tab、terminal、workspace file browser 的 action label。两端还移除一组 aria 与布局样式类名。

这是两平台共同的高可见 UI 收缩或重组信号。它不是源码级事实，不能断言底部面板功能被删除，但可以确认相关字符串和样式从该 chunk 中消失。

### 【信号】[仅 macOS] 插件详情页加入分享、启用和技能预览相关界面文案

`plugin-detail-page` 从近零体积增至 `0.2 MB`，新增字符串包括插件分享对话框中搜索 workspace users and groups、移除 share target、`Add people or groups`、`Anyone at workspace with the link`、`Anyone in this workspace with the link`，以及 MCP server enablement toggle、section navigation、skill preview modal、skill card actions menu trigger 等可访问性标签。

这表明插件详情、共享范围、MCP server 开关和技能预览相关 UI 代码被打入 macOS 前端 chunk，但仍属于同名模块信号。
