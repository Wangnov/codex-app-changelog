# Codex 桌面版 26.715.31925 macOS / Windows 对照

> 第三方跨平台变更日志 · 非官方

本批次的共同实证集中在内置 Browser 与 Chrome 插件版本同步；更明显的产品面变化主要来自 Windows 的任务引导、插件共享、临时聊天和桌面应用入口信号，以及 macOS 的调试、设置与工作簿模块调整。

Browser plugin manifests moved together, while Windows shows broader task, sharing, temporary-chat, and desktop-control signals and macOS concentrates on debug, settings, and workbook changes.

## 关于这份报告

本报告比较两个官方全量分发包之间的机器提取差异。macOS 比较 `26.715.31251` build 5538 与 `26.715.31925` build 5551；Windows 比较 MSIX `26.715.3651.0` 与 `26.715.4045.0`。两个平台的版本号并不对齐，所有结论均分别取自对应平台事实包。

证据分为两档：

- 【实证】来自 plist、插件 manifest、依赖清单、文件树、签名验证和其他可直接读取的配置。
- 【信号】来自前端模块名、同名压缩模块的字符串或样式集合差，以及体积变化；它说明代码方向进入了包内，但不证明功能已开放，也不能等同于源码事实。

平台标记为 `[macOS+Windows]`、`[仅 macOS]` 或 `[仅 Windows]`。

## ✨ 重点变化

### Browser 与 Chrome 插件同步刷新

【实证】[macOS+Windows]

两个平台的 `browser` 与 `chrome` 内置插件 manifest 都把插件版本从 `26.715.31251` 更新到 `26.715.31925`。

`browser` 的既有说明仍明确面向 ChatGPT 内置浏览器、本地 Web 目标、`localhost`、回环地址、`file://`、点击、输入和截图；`chrome` 仍面向依赖用户现有 Chrome 标签页、登录会话、Cookie、扩展和 ChatGPT Chrome Extension 状态的自动化。该批次可确认的是版本同步，所给 diff 没有显示描述内容改变。

Windows 外层 MSIX 已从 `26.715.3651.0` 升至 `26.715.4045.0`，但包内这两个插件采用 `26.715.31925` 版本号；不能用 MSIX 版本替代插件 manifest 版本。

### Windows Computer Use 插件版本更新

【实证】[仅 Windows]

Windows 的 `computer-use` 插件 manifest 同样从 `26.715.31251` 更新到 `26.715.31925`。其描述明确写入“通过 Computer Use 从 ChatGPT 控制 Windows 桌面应用”。

这不是运行时扩容：Windows `cua_node` 总量保持 `287.5 MB`，新增依赖为 0。macOS 对应运行时也保持 `365.7 MB` 且新增依赖为 0，但 macOS 事实包没有给出 `computer-use` manifest 改动。
