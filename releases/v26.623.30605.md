# Codex 桌面版 26.623.30605 macOS / Windows 对照

> 第三方跨平台变更日志 · 非官方

这一批更像是一次“能力面补齐”: Sites、浏览器、计划任务、PR 与 worktree 相关路径在两端同时变密,Windows 包则额外露出更多本地系统接入和设置界面信号。  
One-liner: Sites, browser automation, scheduled tasks, PR review, remote devices, and worktree flows moved forward cross-platform, with Windows showing additional local-system and settings surfaces.

## 关于这份报告

对比范围为 macOS arm64 `26.616.81150 build 4306` → `26.623.30605 build 4415`,Windows x64 `26.616.10790.0` → `26.623.3245.0`。两端来自同一批事实包,但版本号和 build 不同,下文分别引用。

证据等级分两档:【实证】来自 plist、manifest、技能文档、插件清单、服务入口、类型/依赖清单、可读字符串字面量和人类可读 diff；【信号】来自同名前端模块改动线索、模块名、样式类名和体积方向。平台标记为 `[macOS+Windows]`、`[仅 macOS]`、`[仅 Windows]`。

hash 命名的 `webview/assets` 和 `.vite/build` 改名已按噪音处理。事实包没有提供 CSP 差异区块；Windows 也没有提供 AppxManifest 差异区块,因此不扩展推断。

## ✨ 重点变化

### 【实证】[macOS+Windows] Sites 插件升级,条款和 ChatGPT 登录模板进入包内

Sites 插件从 `0.1.16` 升到 `0.1.21`。插件描述、长描述和 `termsOfServiceURL` 都改为 ChatGPT Sites Terms,并加入“你对网站内容负责”的说明。

`vinext-starter` README 新增 Optional Dispatch-Owned ChatGPT Sign-In: `getChatGPTUser()`、`requireChatGPTUser(returnTo)`、`chatGPTSignInPath(returnTo)`、`chatGPTSignOutPath(returnTo)` 进入模板文档；同时明确 `/signin-with-chatgpt`、`/signout-with-chatgpt`、`/callback` 由 Dispatch 管理,SIWC 只建立身份,不证明 workspace membership。

Sites hosting 技能新增计划连续性、预览 handoff 和部署缩略图规则: hosting 阶段应延续 `sites-building` 的 plan；部署前要尝试 `open_in_codex` 预览 handoff；`public/screenshot.jpeg` 被定义为可选部署缩略图,默认不创建、不刷新、不删除。

### 【实证】[macOS+Windows] Browser / Chrome 自动化说明和浏览器右键菜单继续扩展

Browser 插件 manifest 版本在两端从 `26.616.81150` 升到 `26.623.30605`。Playwright 文档在 Browser 和 Chrome 插件内都做了结构调整,多个小节从 `###` 提升为 `##`,内容仍强调 snapshot discipline、locator 唯一性和受限 Playwright API。

多语言 native menu diff 显示内置浏览器右键菜单新增“在外部浏览器中打开”“在浏览器中打开”“在新标签页中打开链接”；更新确认文案也从硬编码 Codex 改成 `{appName}`。主进程新增可读字符串还包括 Browser 页面快照、导航历史恢复、PiP metadata、sidebar storage ownership、`Quick annotate`、`Selected text`、`Tab persistence is only available with Chrome or multi-tab IAB.` 和 WebMCP JSON 序列化约束。

### 【实证】[macOS+Windows] 内置插件清单和展示资源有一组小但明确的变更

LaTeX 插件从 `0.2.3` 升到 `0.2.4`,展示 logo 从 `latex-logo.svg` 切到 `assets/latex.png`,并新增 `logoDark`。Sites、Browser 的 manifest 变化两端一致可见。

Windows 事实包额外显示 Chrome 插件和 Computer Use 插件 manifest 版本也从 `26.616.81150` 升到 `26.623.30605`。Windows 还新增 Browser / Chrome 的 `api.json`、`unified-skill.md`、`browserAuth.md`、`installed-browsers.js`、`open-chrome-window.js`、`check-extension-installed.js`、`check-native-host-manifest.js` 等文件,同时移除旧的 `api.md`、multi-tab 分拆文档和部分 troubleshooting 文档。

### 【信号】[macOS+Windows] Automations 正在向 Scheduled tasks 迁移

`automations-page` 同名模块新增搜索 scheduled task/templates、任务详情右侧栏、任务与模板切换、暂停/恢复/删除 scheduled task、新 scheduled task 下拉、从侧栏创建任务、通过聊天个性化模板等文案；旧的 automation overview、automation not found、automation started 等文案减少。

`local-conversation-page` 和 `local-conversation-thread` 也把“heartbeat automation”入口替换为“scheduled task”入口。由于这些来自同名前端模块线索,只能说明 UI/状态方向,不能断言所有入口已经对用户开放。

### 【实证】[macOS+Windows] PR / GitHub 校验路径变厚,前端 reviewer 交互有明显信号

主进程新增 `At least one reviewer is required`、GitHub user / pull request 搜索失败、PR head commit 校验失败、PR state/URL/head revision 缺失或无效、account mismatch、head changed during diff acquisition 等字符串。这些说明包内新增了更多 PR 和 reviewer 相关错误路径与校验分支。

同名前端模块还给出【信号】: PR 侧栏出现 reviewer picker、搜索 reviewer、approved / pending / requested changes 状态、修复所有 failing checks、修复或移除所有 PR comments 等可访问性文案。两平台都有这些线索。

### 【实证】[macOS+Windows] Thread catalog、worktree 迁移和 app-server 文件读写进入错误路径

新增字符串包含本地 thread catalog 扫描状态、active scan 时禁止 prune、host 不能绑定 disposed catalog、修复 missing thread cwd metadata、移动 thread 到 worktree 失败、app-server file read/write 失败、ExecutionHost 要求、跨 worker/app-server exclusive copy 不支持等。

Windows 的 `split-items-into-render-groups` 额外给出【信号】: `move-thread-to-local`、`move-thread-to-worktree`、`worktree-create-managed`、`prepare-host-transfer`、`worktree-init`、`Branch is already checked out in another worktree` 等,说明 Windows 前端包里 worktree/thread handoff UI 更具体。

### 【信号】[macOS+Windows] Windows 沙箱和有限访问 onboarding 文案被跨平台打包

两端 `onboarding-page` 都新增 Windows UAC 示意、无管理员权限继续设置 Windows、未完成 Windows sandbox setup 时以 read-only mode 或 limited access 继续、以及“ChatGPT can suggest what to do next by searching project files and connected apps”等文案。功能对象明确是 Windows,但这些字符串同时存在于 macOS 和 Windows 包内,所以只能按包内容标为 `[macOS+Windows]` 的前端信号。

### 【实证】[仅 Windows] Windows 本地系统接入新增原生组件

Windows 新增 `resources/native/windows-account.node` 约 `0.8 MB`,并新增 `resources/icon.ico`。`resources/owl-electron-app.json` 的 `packagedFrom` 从 GitHub Actions 路径变为 `D:\codex-apps-workspace\...`,`runtimeArchiveSha` 也更新。`reading_mode_gdocs_helper_manifest.json` 版本从 `149.0.7827.115` 升到 `149.0.7827.157`。

### 【信号】[仅 Windows] Composer、项目创建、PDF、浏览器设置和用量界面信号集中出现

Windows 独有同名模块线索显示多组高可见 UI 方向: Composer 新增 Plan mode placeholder、`/init` 创建 `AGENTS.md`、已存在时跳过、Windows sandbox readiness retry、安全 buffering 帮助链接、选中文本评论/编辑；本地项目创建新增 source folder picker、项目颜色和图标 marker；PDF 模块体积从 `0.0 MB` 到 `0.4 MB`,出现 PDF password prompt 和 load failure 字符串；browser-use settings 出现 history、downloads、password manager、site settings、blocked/allowed domain 和 “Access history without asking”；rate limit / referral 相关模块出现 credits、usage reset、invite tracking、all-time/current-month 统计；home announcements 出现 gift credits、Fast mode、multi-agent composer、remote connections banner。

这些都是同名前端模块线索,应视为 Windows UI/状态信号,不是源码级实证。

### 【信号】[macOS+Windows] Remote connections 从 pairing 文案转向 signed-in device / secure transport 状态

`remote-connections-settings` 新增 remote Codex app-server version tooltip、online device requires newer version、waiting for device、initializing secure connection、confirming connection、sign in to ChatGPT then refresh to add a device 等文案；旧的 manual pairing dialog 描述和 pairing code 文案减少。两端都有,但来自同名前端模块,证据等级为【信号】。

### 【信号】[macOS+Windows] Skills / Plugins 页面从 Add 语义转向 Install / imported setup

`plugins-page` 新增 Skills page sections、install plugin、install recommended skill、try installed plugin in chat、Finish setup、Imported plugins、official app connectors suggested from imported external agent setup 等文案；旧的 Add skill / Adding plugin / connected recommended skills 文案减少。这指向插件和 skill 安装语言的统一,但仍属于前端模块信号。

### 【信号】[macOS+Windows] Cloud Library 上传和 AppGen Library 类型筛选增强

`appgen-library-page` 新增 cloud Library 文件预览、隐藏上传 input、`Add anything`、上传文件、拖拽任意文件加入 library、MIME fallback、generated images tab。Windows 额外出现 `Library tab that shows output files`; macOS 则出现 `Cloud Library files are unavailable` 和 thumbnails unavailable。两端共同方向是 Library 从“继续聊天/站点设置”转向文件上传、预览和类型过滤。

### 【实证】[仅 macOS] macOS 包装、Chromium 基线和本地资源调整

macOS `Info.plist` 将 `CFBundleShortVersionString` 从 `26.616.81150` 升到 `26.623.30605`,`CFBundleVersion` 从 `4306` 升到 `4415`,`ChromiumBaseVersion` 从 `149.0.7827.115` 升到 `149.0.7827.157`,`ElectronAsarIntegrity` hash 和 `SCMRevision` 也更新。新增 `CodexDmgBackgroundFileName = codex-dmg-background.png` 和 `Contents/Resources/icon-codex-dark-color.png`。

macOS 文件树显示 Chromium Framework 版本目录从 `149.0.7827.115` 换到 `149.0.7827.157`; `app.asar` 增加 `+8.88 MB`。Computer Use 相关 app 内的 `SkyComputerUseService`、`CUALockScreenGuardian`、`SkyComputerUseClient` 二进制各有约 `+1.33 MB` 到 `+1.40 MB` 的增长,但 cua_node 总量仍为 `377.4 MB → 377.4 MB`。macOS 还移除了 `realtime-voice-wake-word` 和 `WakeSingleHead.mlpackage` 相关文件；这只说明该批 macOS 包内这些原生文件消失,不等于可断言所有语音能力被移除。
