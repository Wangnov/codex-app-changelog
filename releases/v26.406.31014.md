# Codex 桌面版 `26.406.31014` build 1395

> 第三方变更日志 · 非官方 · 基于官方更新包逆向差异

这一版不像单纯的小修补：产物中同时出现了后台建议、heartbeat automation、浏览器侧栏评论、托盘菜单和项目外新聊天位置等多条可感知功能线索。  
English one-liner: this build expands background suggestions, automation, browser-side comments, tray actions, and projectless chat routing.

## 关于这份报告

这不是官方变更日志，而是一份**基于官方分发产物的逆向差异报告**。本次为历史回填，官方 appcast 未对 `26.325.31654` → `26.406.31014` 提供增量包，因此对两个官方全量包做直接比较。

证据分两档：**【实证】**表示产物中存在人类可读的直接证据，例如 `package.json`、`Info.plist`、CSP、可读字符串或配置 diff；**【信号】**表示只来自前端模块名、同名前端模块差异或体积变化，指向某方向，但无法从产物还原到源码级意图。

对比范围为 macOS arm64，上一版 `26.325.31654` build `1272`，发布于 `2026-03-28`；最新版 `26.406.31014` build `1395`，发布于 `2026-04-08`。

## ✨ 重点变化

### 1. 新增后台建议与项目感知建议设置 【实证】

新版出现了 `Ambient suggestions`、`Enable ambient suggestions`、`Generate project-aware suggestions on window focus and show the latest 3 pending rows above the home composer` 等设置文案，并伴随 `Failed to refresh ambient suggestions`、`Failed to update ambient suggestions enabled state` 等错误路径。

这意味着 Codex 正在把“窗口聚焦时生成项目感知建议，并在首页输入框上方展示最近 3 条待处理建议”做成可配置能力。关键证据来自 `agent-settings` 与主进程 bundle 新增字符串。

### 2. Heartbeat automation 从提示文案扩展为线程级自动化能力 【实证】

新版新增 `Accessible label for opening the heartbeat automation attached to the current thread`、`Created automation in the app.`、`Updated automation in the app.`、`Failed to delete heartbeat automation for archived thread`、`post a heartbeat message`、`send a heartbeat` 等字符串。

这指向线程可附加 heartbeat automation，并包含创建、更新、归档线程清理失败等路径。和上一版移除的 `Heartbeat found something worth your attention.` 相比，新版更像把 heartbeat 从提醒结果推进到可管理的自动化对象。

### 3. 浏览器侧栏评论与浏览器桥接能力增强 【实证】

主进程新增 `Button label to cancel creating an inline browser comment`、`Button label to revisit the annotated browser location`、`browserSidebarCommentRuntime.*`、`Timed out waiting for browser bridge response`、`Unknown browser bridge error`、`chrome bridge install complete`、`chrome bridge install failed` 等字符串。

用户可感知点是：浏览器侧栏里可能出现可取消、关闭、删除、回访位置的 inline comment 流程；底层还出现 browser bridge 超时、未知错误、Chrome bridge 安装结果等路径。`app-server-manager-hooks` 中也新增 `Browser bridge request failed` 与 `features.browser_use`。

### 4. 托盘菜单和快速聊天入口成型 【实证】

新版新增 `Failed to set up tray`、`trayMenu.newChat`、`trayMenu.openApp`、`trayMenu.openHotkeyWindow`、`trayMenu.pinnedThreads`、`trayMenu.projectlessThreads`、`trayMenu.recentThreads`、`Quick Chat`、`Open Quick Composer` 等字符串，并新增 `codexTemplate.png` 与 `codexTemplate@2x.png` 资源。

这说明桌面托盘菜单已有明确入口集合：打开 App、新聊天、热键窗口、置顶线程、项目外线程与最近线程。新增模板图片资源也与 macOS 托盘图标资源形态一致，但产物未给出完整 UI 行为。

### 5. 项目外新聊天可选择启动位置 【实证】

设置页新增 `New chat location`、`Where should new chats without a project start?`、`Use Codex threads folder`、`Use home folder`、`Label for choosing where non-project chats run` 等文案。

这意味着非项目聊天不再只是隐式落点，而是出现了用户可见设置：项目外新聊天可以在 Codex threads folder 或 home folder 之间选择。

### 6. Pull request 看板与 PR 更新路径继续扩展 【实证】

新版新增 `Accessible label for the pull request auto-merge toggle`、`Accessible label for the pull request board saved view toggle`、`Accessible label for the pull request board search field`、`Failed to load pull request diff`、`Failed to update pull request`、`No git repository found for PR updates`、`No git repository found for pull request diff`。

这表明 PR 相关界面不只停留在展示层，还出现了 diff 加载、PR 更新、自动合并开关、保存视图和搜索框等路径。由于证据来自 bundle 字符串，能确认这些路径存在，但不能确认所有入口已对用户开放。

### 7. 远程连接加入版本过旧拦截提示 【信号】

`app-server-connection-state` 新增 `Update required`、`appServer.error.unsupportedVersion`、`Error shown when an app-server connection is rejected because the remote Codex version is too old`。

这是同名前端模块改动线索，指向远程 Codex 版本过旧时会显示“需要更新”的状态，但无法从产物还原到源码级意图或完整握手机制。

### 8. 前端架构迁移到 React 19，并加入共享对象快照通道 【实证】

`package.json` 新增 `react`、`react-dom`，版本为 `^19.2.0`，同时新增 `@types/react` 与 `@types/react-dom` 的 `catalog:react-19`。`preload.js` 新增 `codex_desktop:get-shared-object-snapshot`、`shared-object-set`、`shared-object-updated` 和 `getSharedObjectSnapshotValue`。

这意味着新版前端运行时已显式带入 React 19，并在 Electron preload 桥中增加共享对象快照读写/同步通道。对用户而言这属于底层变化，但可能支撑跨窗口或主渲染进程状态同步。

### 9. 用户数据目录支持环境变量覆盖 【实证】

`bootstrap.js` 新增 `CODEX_ELECTRON_USER_DATA_PATH` 读取逻辑：如果该环境变量存在，会用 `resolve` 后的路径作为 `userData`，否则继续使用 appData 下按 build flavor 生成的目录。

这对测试、隔离运行或多实例环境有实际价值：可以通过环境变量指定 Codex 的用户数据目录。该能力来自可读 diff，属于直接证据。

### 10. 代码高亮/语法资产重新分布 【信号】

同名前端模块里，`markdown` 大幅减小 `0.7 MB → 0.1 MB`，同时 `typescript`、`c`、`haxe`、`scss`、`vue`、`angular-html`、`gdscript`、`regexp`、`glsl` 等模块新增大量语法 scope 字符串；`javascript`、`php`、`csharp`、`java`、`yaml` 则有对应字符串减少。

这些线索指向语法高亮资产的拆分或重排，但只能作为构建产物层面的信号，无法从产物还原到源码级意图，也不能解读为某些语言支持被真实删除。
