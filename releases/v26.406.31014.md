# Codex 桌面版 `26.406.31014`(build 1395)

> 第三方变更日志 · 非官方 · 基于官方更新包逆向差异

这版从产物层面看，重点不是大规模新增二进制能力，而是桌面壳、前端状态同步、远程环境与设置页结构继续向更细分的应用形态演进。  
English one-liner: this build points to richer shared state plumbing, remote-host awareness, and settings/UI restructuring.

## 关于这份报告

这不是官方变更日志，而是一份**基于官方分发产物的逆向差异报告**。本次为历史回填：两个版本均超出官方 Sparkle(桌面应用自动更新框架)增量窗口，因此事实包采用两个官方全量包直接比较，而不是增量包重建。

对比范围为 Codex 桌面版 `26.325.31654`(build 1272，2026-03-28) 到 `26.406.31014`(build 1395，2026-04-08)，平台为 macOS arm64(苹果芯片架构)。

可信度方面，两个官方全量包均给出了 SHA-256(文件指纹算法)；公证验证结果均为 `CSSMERR_TP_CERT_REVOKED`。事实包说明这意味着部分早期版本签名证书后来被吊销，会影响系统安全安装判断，但不影响对 bundle(应用包目录)内容做差异分析。

本文使用两档证据等级：

- **【实证】**：来自人类可读的直接证据，例如 `Info.plist`、`package.json`、`index.html`、CSP(内容安全策略)文本、构建脚本差异。
- **【信号】**：来自模块名或体积变化等间接证据，指向某方向，但无法从产物还原到源码级意图，也不代表功能一定已对用户开放。

这说明桌面壳和 Webview(内嵌网页视图)之间开始维护一份可同步的共享状态。可感知点可能体现在跨窗口、跨页面或桌面原生层与前端状态的一致性增强，但事实包不能证明具体用户界面已经开放了哪项功能。

关键证据：


这意味着内嵌前端可以从应用协议或内联数据源加载媒体资源。它是可验证的安全策略变化，但事实包没有说明具体媒体功能场景。

### 8. 构建与运行时打包流程调整 【实证】

`package.json` 新增 `build:node-runtime` 脚本，`rebuild:forge-natives` 从直接 `pnpm rebuild macos-alias` 改为执行 `scripts/rebuild-forge-natives.mjs`；同时 `lint` 和 `lint:fix` 增加 `--threads=1`。

这说明项目内部构建流程更偏向脚本化封装，并限制 lint(代码检查)并发。对普通用户未必可见，但对调试、打包和复现构建的人是直接可感知变化。

关键证据：

```diff
+ "build:node-runtime": "node ./scripts/bundled-runtimes/bundle.mjs"
- "rebuild:forge-natives": "pnpm rebuild macos-alias"
+ "rebuild:forge-natives": "node ./scripts/rebuild-forge-natives.mjs"
+ oxlint --threads=1
```
