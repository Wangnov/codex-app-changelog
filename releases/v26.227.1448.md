# Codex 桌面版 `26.227.1448`(build 747)

> 第三方变更日志 · 非官方 · 基于官方更新包逆向差异

这一版从 `26.226.940` 升级到 `26.227.1448`，可读证据主要集中在构建脚本、版本元数据和 macOS(苹果桌面操作系统)权限声明。  
One-liner: a small but concrete build metadata and packaging-script update.

## 关于这份报告

这不是 OpenAI(人工智能公司) 官方变更日志，而是一份**基于官方分发产物的逆向差异报告**。本次为历史回填：两个版本均超出官方 Sparkle(应用自动更新框架)增量窗口，因此事实包显示采用的是两个官方全量包直接比较，而不是增量包重建。

对比范围：`26.226.940`(build `720`，发布于 `2026-02-26`) → `26.227.1448`(build `747`，发布于 `2026-02-28`)，平台为 `macOS arm64`。

可信度方面，两个官方全量包均给出了 SHA-256(安全哈希值)。公证验证结果均为 `CSSMERR_TP_CERT_REVOKED`，事实包说明这是部分早期版本签名证书后来被吊销导致；它会影响系统安全安装判断，但不影响对 bundle(应用包)内容做差异分析。

证据等级说明：

- **【实证】**：来自人类可读的直接证据，例如 `Info.plist`(应用元数据配置)、`package.json`(项目配置文件)diff、版本号、构建号、权限键值。
- **【信号】**：来自间接证据，例如前端模块名或二进制体积变化；这类证据只能说明“指向某方向，但无法从产物还原到源码级意图”。

## ✨ 重点变化

### 1. 构建与测试流程开始显式重建 SQLite 【实证】

`package.json` 中多条脚本被改为先执行 `pnpm run rebuild:sqlite`，覆盖 `dev`、`forge:package`、`forge:make`、`forge:publish`、`test`、`test:quiet` 和 `playwright:agent:repl`。

这意味着本版更强调在开发、测试、打包和发布前确保 SQLite(嵌入式数据库)原生模块处于可用状态。对普通用户未必直接可见，但对开发构建稳定性和自动化测试链路是明确变化。

关键证据：

```diff
-    "test": "node ./scripts/ensure-electron-binary.mjs && vitest run",
+    "test": "pnpm run rebuild:sqlite && node ./scripts/ensure-electron-binary.mjs && vitest run",
```

### 2. TypeScript 编译命令从 `tsc` 切到 `tsgo` 【实证】

`compile` 与 `tsc` 脚本从直接调用 `tsc` 改为通过 `pnpm exec tsgo` 执行。

这说明项目内部的类型检查 / 编译入口发生了调整。事实包没有提供 `tsgo` 的更多上下文，因此这里只能确认命令层面的变化，不展开推断其性能或行为差异。

关键证据：

```diff
-    "compile": "tsc -b",
-    "tsc": "tsc -noEmit",
+    "compile": "pnpm exec tsgo -b",
+    "tsc": "pnpm exec tsgo --noEmit",
```

### 3. 蓝牙权限声明键发生替换 【实证】

`Info.plist` 新增 `NSBluetoothPeripheralUsageDescription`，同时移除旧的 `NSBluetoothAlwaysUsageDescription`，两者文案均指向蓝牙访问需求。

这意味着 macOS 权限声明的键名发生了调整。事实包没有显示新增蓝牙功能，只能确认权限元数据从一个蓝牙说明键迁移到另一个蓝牙说明键。

关键证据：

```text
新增 /NSBluetoothPeripheralUsageDescription = 'This app needs access to Bluetooth'
移除 /NSBluetoothAlwaysUsageDescription = 'This app needs access to Bluetooth'
```

### 4. 主二进制体积增加较明显 【信号】

`Contents/Resources/codex` 从 `75.8 MB` 增至 `84.4 MB`，增加 `8.60 MB`。这是本次 bundle 文件级差异中体积变化最大的组件。

这指向底层可执行组件有实质变化，但仅凭体积无法判断具体功能、性能或行为意图；它指向某方向，但无法从产物还原到源码级意图。

### 5. 应用层资源小幅增大，但内部文件增减主要是构建噪音 【信号】

`Contents/Resources/app.asar` 从 `35.5 MB` 增至 `36.4 MB`，增加 `0.89 MB`。事实包同时说明 `app.asar` 内部 `webview/assets` 下大量增减是 Vite(前端构建工具)hash(内容指纹)重命名噪音。

因此，这里只能说应用层资源包小幅变大；不能把内部 `新增 123 / 移除 123` 解释成新增或删除了 123 个真实功能模块。该信号指向应用层产物有变化，但无法从产物还原到源码级意图。

### 6. 版本号与构建号更新 【实证】

应用短版本号从 `26.226.940` 更新到 `26.227.1448`，构建号从 `720` 更新到 `747`。`package.json` 和 `Info.plist` 均反映了这一点。

关键证据：

```diff
-  "version": "26.226.940",
+  "version": "26.227.1448",
```

```diff
-  "codexBuildNumber": "720",
+  "codexBuildNumber": "747",
```
