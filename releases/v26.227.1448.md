# Codex 桌面版 `26.227.1448` build 747

> 第三方变更日志 · 非官方 · 基于官方更新包逆向差异

本版最清晰的变化集中在热键窗口、app-server 连接启动、沙箱权限提示和 macOS 蓝牙权限声明。  
One-liner: this build appears to refine hotkey-window setup, app-server transport handling, and permission messaging.

## 关于这份报告

这不是官方变更日志，而是一份**基于官方分发产物的逆向差异报告**。本次对比范围为 macOS arm64 的 `26.226.940` build 720 到 `26.227.1448` build 747。

由于官方 appcast 未对这一对历史版本提供增量包，本报告基于两个官方全量包直接比较。事实来源包括 bundle 文件树、Info.plist、package.json diff、主进程/前端 bundle 新增可读字符串，以及同名前端模块的字符串与样式差异。

证据等级分两档：**【实证】**表示有 Info.plist、package.json、可读字符串或其他人类可读产物作为直接证据；**【信号】**表示只有前端模块名、同名模块字符串/样式变化或体积变化等间接证据，指向某方向，但无法从产物还原到源码级意图。

## ✨ 重点变化

### 1. 新增热键窗口运行态与快捷键校验文案 【实证】

新版主进程 bundle 出现了多条与 hotkey window 直接相关的新增字符串，包括 `Failed to apply hotkey window runtime state`、`Hotkey Window hotkey is not configured.`、`Shortcut must include Cmd/Ctrl or Alt.`、`Shortcut must include a non-modifier key.` 和 `Shortcut must include exactly one non-modifier key.`。

这意味着本版至少在产物层面新增或强化了热键窗口配置、运行态应用失败处理，以及快捷键合法性校验相关路径。前端同名模块也出现 `Configured hotkey`、`hotkey-window`、`data-hotkey-window-curtain`、`data-hotkey-window-floating-ui` 等线索，但这些只能作为 UI/交互方向的【信号】补充。

### 2. app-server 连接与 transport 释放路径有新增处理 【实证】

主进程 bundle 新增 `Starting app-server connection` 与 `Error while disposing app-server transport`。

这表明新版在 app-server 连接启动和 transport 释放错误路径上有可见的新增日志或错误处理。对用户或开发者而言，可感知点更可能体现在连接启动状态、失败诊断或后台服务稳定性上，但产物无法还原完整源码级意图。

### 3. Windows sandbox / WSL 入口继续调整，前端提示更偏权限状态说明 【信号】

同名前端模块中，多语言字符串持续移除 `Windows Sandbox`、`runInWsl`、`windowsDocumentation`、`windowsInstallWslBanner` 等入口，同时新增 `Codex automatically runs commands in a sandbox`、`Codex has full access over your computer (elevated risk)`、`Codex is in read-only mode and requires approval to edit files and run commands.` 以及 `Button text to continue using read-only mode instead of enabling Windows sandbox`。

这些线索指向沙箱/只读/完整访问权限说明在 UI 中被重排或改写，但无法从产物还原到源码级意图。特别是 macOS arm64 产物中出现 Windows 相关文案，不能直接推断 macOS 端新增 Windows 功能。

### 4. macOS 蓝牙权限声明从 Always 改为 Peripheral 【实证】

Info.plist 中新增 `/NSBluetoothPeripheralUsageDescription = This app needs access to Bluetooth`，同时移除了旧的 `/NSBluetoothAlwaysUsageDescription`，旧值同样为 `This app needs access to Bluetooth`。

这是一处 macOS 权限声明层面的直接变化。可感知点是系统权限提示所引用的 Info.plist key 发生了调整，但事实包没有给出触发蓝牙访问的具体功能路径。

### 5. 构建与测试流程显式加入 sqlite rebuild，TypeScript 命令切换到 tsgo 【实证】

`package.json` 中 `compile` 从 `tsc -b` 改为 `pnpm exec tsgo -b`，`tsc` 从 `tsc -noEmit` 改为 `pnpm exec tsgo --noEmit`。同时 `dev`、`forge:package`、`forge:make`、`forge:publish`、`test`、`test:quiet` 和 `playwright:agent:repl` 都新增了 `pnpm run rebuild:sqlite` 前置步骤，`postinstall` 中的 `pnpm run rebuild:sqlite` 被移除。

这表明本版把 sqlite native rebuild 从安装后动作转移到开发、打包、测试和 Playwright 入口前显式执行。对开发者而言，这是更明确的构建前置条件；对最终用户而言，这更多是打包产物背后的工程流程变化。

### 6. 多语言资源补齐 MCP 工具输出、patch 失败和计划建议文案 【信号】

多个语言模块新增 `codex.mcpTool.contentBlock.annotationsLine`、`codex.mcpTool.rawOutputHeading`、`codex.mcpTool.rawOutputTriggerTooltip`、`codex.mcpTool.resourceLink.reading`、`composer.aboveSuggestion.plan.*`、`codex.applyResultsDialog.notGitRepo`、`codex.unifiedDiff.patchFailureDetailsIntroReapply` 等字符串。

这些线索指向 MCP 工具输出展示、资源读取状态、计划建议、非 Git 仓库处理和 patch 失败详情的国际化覆盖在扩展，但仍属于同名前端模块改动线索，不能作为源码级事实。
