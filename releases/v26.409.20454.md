# Codex 桌面版 `26.409.20454` build 1462

> 第三方变更日志 · 非官方 · 基于官方更新包逆向差异

这个版本的主线是运行时升级，以及本地 sidecar、远程连接、浏览器面板和 Projectless 会话能力的继续成形。  
One-liner: Electron moved to 41.2.0 while app-server, remote control, browser panel, and projectless-thread surfaces gained visible wiring.

## 关于这份报告

这不是官方变更日志，而是一份**基于官方分发产物的逆向差异报告**。本次对比范围是 macOS arm64 的 `26.406.40811` build 1457 到 `26.409.20454` build 1462，两个版本发布日期均为 `2026-04-11`。

本版为历史回填：事实包说明官方 appcast 未对这一对版本提供增量包，因此改为下载两个官方全量包直接比较。bundle 差异、`Info.plist`、`package.json`、前端入口 HTML、签名与公证验证结果都来自机器提取事实。

证据等级分两档：**【实证】**表示有可读的直接证据，例如配置 diff、权限键、依赖版本、主进程新增字符串或入口文件；**【信号】**表示只有前端模块名、同名 bundle 字符串差异或体积变化等间接证据，指向某方向，但无法从产物还原到源码级意图。

## ✨ 重点变化

### 1. Electron 升级到 `41.2.0` 【实证】

`package.json` 中 Electron 从 `40.0.0` 升级到 `41.2.0`，同时 `better-sqlite3` 从 `^12.4.6` 升级到 `^12.8.0`。

这意味着桌面运行时与本地 SQLite 绑定都有更新；可感知影响可能体现在窗口、WebView、原生模块兼容性和性能边界上。关键证据：

```diff
-    "electron": "40.0.0",
+    "electron": "41.2.0",
-    "better-sqlite3": "^12.4.6",
+    "better-sqlite3": "^12.8.0",
```

### 2. 本地 app-server sidecar 生命周期管理更明确 【实证】

新版主进程 bundle 新增了多条 sidecar 生命周期字符串，包括启动、停止、重启调度、退出、失败恢复和二进制缺失路径。

这表明本地 app-server sidecar 不再只是静态存在，而是有更完整的运行时状态管理和错误路径。关键证据包括 `Starting local app-server sidecar`、`Stopping local app-server sidecar`、`Local app-server sidecar exited`、`Local app-server sidecar restart failed`、`Sidecar binary enabled but command not found`。

### 3. 远程控制入口进入设置面板 【信号】

`remote-connections-settings` 同名前端模块新增了“Enable Remote Control on this Computer”“Allows other signed-in Codex clients to connect to this computer”等文案和设置键。

这指向本机可被其他已登录 Codex 客户端连接的远程控制开关，但无法从产物还原完整启用条件、权限模型或发布状态。关键线索：

```text
Enable Remote Control on this Computer
Allows other signed-in Codex clients to connect to this computer
settings.remoteControlConnections.localRemoteControl.label
```

### 4. 远程连接状态开始处理版本不匹配和重启路径 【信号】

`onboarding-login-content` 模块新增了远程 Codex 版本过旧、需要更新、连接中、连接错误、未认证、可重启等状态文案。

这指向远程连接流程从单纯 SSH 入口转向更细的 Codex-to-Codex 连接状态处理，但无法从前端产物确认完整协议或后端行为。关键线索包括 `Update required`、`Update now`、`Label shown when remote connection needs a newer Codex version`、`Action label shown when remote Codex has a newer installed version and can be restarted`。

### 5. 浏览器面板和浏览器区域能力继续扩展 【实证】

主进程新增字符串中出现 `Toggle Browser Panel`、`Selected browser region`、`browser region`、`Browser sidebar runtime message received without owner window`、`Invalid browser sidebar runtime message received` 等内容；`vscode-api` 模块也新增 `codex.command.toggleBrowserPanel` 和 `codex.command.openAvatarOverlay`。

这说明浏览器侧栏、浏览器区域选择和相关命令入口在本版继续推进。由于主进程字符串和命令名是产物中的可读直接证据，这一方向可标为实证；但具体 UI 完成度仍不能由压缩产物完全还原。

### 6. GitHub PR 评论与 diff 评论路径出现 【实证】

新版主进程 bundle 新增了 `Failed to load GitHub user`、`Failed to post pull request comment`、`Failed to post pull request diff comment`、`Failed to reply to pull request comment`、`Repository is required for PR diff comments` 等字符串。

这表明应用内开始包含 GitHub PR 评论、diff 评论、回复和仓库校验相关路径。可感知点是 Codex 可能在围绕 PR review 工作流增加本地集成能力；但成功路径的完整交互无法仅凭字符串确认。

### 7. Projectless 会话和 ambient suggestions 更成体系 【实证】

主进程新增 `Recent Projectless Codex threads with the same cwd`、`For Projectless chat, ground each suggestion in the user`、`this Projectless chat` 等字符串；前端真实新增模块中也出现 `projectless-thread`。

这说明没有显式项目上下文的会话被单独建模，并与最近线程、建议生成和当前 cwd 关联。`agent-settings` 同名模块还把旧的 project-level 文案替换为更宽泛的 ambient suggestion 描述，这是 UI 和设置方向的信号。

### 8. 更新安装前会提示本地会话会被中断 【信号】

`plugins-page` 模块新增了更新确认对话框文案，明确说明安装更新会退出 Codex，并中断本机活跃的本地会话。

这指向更新流程对 active local sessions 的保护性确认，但由于证据来自同名前端模块改动线索，只能说明 UI 文案与交互方向。关键线索：

```text
Codex will quit to install the update, which will interrupt active local sessions on this machine
Title for the confirmation dialog shown before installing an app update while a local session is active
```

### 9. macOS 权限从蓝牙转向音频采集 【实证】

`Info.plist` 新增 `NSAudioCaptureUsageDescription = This app needs access to audio capture`，同时移除旧的 `NSBluetoothPeripheralUsageDescription`。

这是用户可能在系统权限弹窗中感知到的变化：新版声明音频采集权限，不再声明蓝牙外设权限。关键证据：

```text
新增 /NSAudioCaptureUsageDescription = 'This app needs access to audio capture'
移除 /NSBluetoothPeripheralUsageDescription
```

### 10. 代码高亮语法资产有一轮重排 【信号】

多个语言模块出现明显体积和语法 scope 字符串变化，包括 `tsx`、`less`、`python`、`java`、`yaml`、`shellscript`、`ruby`、`r`、`lua` 等新增 scope，同时 `typescript`、`markdown`、`scss`、`haxe` 等模块减少。

这指向代码高亮或语法资源被重组，但无法从产物还原到源码级意图，也不能断言某种语言支持被新增或移除。可感知影响可能是部分代码块渲染、token 分类或主题样式发生变化。
