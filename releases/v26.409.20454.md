# Codex 桌面版 `26.409.20454`(build 1462)

> 第三方变更日志 · 非官方 · 基于官方更新包逆向差异

这一版更像是一次底座升级与权限调整：Electron（跨平台桌面应用框架）升级，应用体积有所增加，权限从蓝牙转向音频采集，同时前端模块名透露出实验功能、无项目会话与状态管理方向的变化。  
One-liner: runtime upgrade, permission reshaping, and early UI signals around experimental/projectless flows.

## 关于这份报告

这不是官方变更日志，而是一份**基于官方分发产物的逆向差异报告**。本次为历史回填：两个版本均超出官方 Sparkle（macOS 应用更新框架）增量窗口，因此不是通过增量包重建，而是下载两个官方全量包后直接比较，再生成结构化 diff（差异对比）。

可信度方面，两个 Codex.app（应用程序包）均通过系统公证验证：上一版与最新版的 `spctl`（macOS 安全评估工具）结果均为 `accepted`，来源为 `Notarized Developer ID`，签名主体为 `Developer ID Application: OpenAI OpCo, LLC (2DC432GLL2)`。

本文使用两档证据等级：

- **【实证】**：来自人类可读的直接证据，例如 `Info.plist`（macOS 应用元数据文件）、`package.json`（Node/Electron 项目配置文件）、签名与公证结果。
- **【信号】**：来自前端模块名或二进制体积变化，只能说明“指向某方向，但无法从产物还原到源码级意图”。

## ✨ 重点变化

### 1. 桌面运行底座升级到 Electron 41.2.0 【实证】

`package.json` 显示，应用使用的 Electron 从 `40.0.0` 升级到 `41.2.0`。

这意味着桌面外壳、WebView（内嵌网页视图）、系统集成与 Chromium（浏览器内核）相关能力都进入了新一代运行底座；但事实包没有给出具体用户功能变化，因此这里只能确认依赖版本升级本身。

关键证据：

```diff
-    "electron": "40.0.0",
+    "electron": "41.2.0",
```

### 2. 权限声明从蓝牙转向音频采集 【实证】

`Info.plist` 新增了 `NSAudioCaptureUsageDescription`，值为 `This app needs access to audio capture`；同时移除了旧的 `NSBluetoothPeripheralUsageDescription`，旧值为 `This app needs access to Bluetooth`。

这表示新版应用开始声明音频采集权限，同时不再声明蓝牙外设权限。用户可感知层面，这可能影响 macOS 弹出的系统权限提示类型；但事实包没有显示对应功能入口，因此不能进一步判断音频采集会用于哪项具体功能。

关键证据：

```text
新增 /NSAudioCaptureUsageDescription = 'This app needs access to audio capture'
移除 /NSBluetoothPeripheralUsageDescription = 'This app needs access to Bluetooth'
```

### 3. 应用内状态与服务连接模块出现替换迹象 【信号】

去除 Vite（前端构建工具）hash（构建指纹）噪音后，前端真实模块新增 `app-server-manager-signals`，同时移除 `app-server-connection-state`。

这指向应用内“服务连接状态 / 管理器状态”相关实现可能在重组：从连接状态模块转向 signals（响应式信号）式管理。但这只是模块名信号，无法从产物还原到源码级意图，也不能确认用户界面一定发生变化。

关键证据：

```text
新增模块: app-server-manager-signals
移除模块: app-server-connection-state
```

### 4. 实验功能查询模块进入前端包 【信号】

前端真实新增模块中出现 `experimental-features-queries`。

这指向新版前端可能开始集中处理实验功能查询或开关读取。但事实包没有包含接口、文案或 UI（用户界面）入口，因此不能写成“新增实验功能面板”或“开放某个实验功能”。

关键证据：

```text

```

### 5. “无项目会话”相关入口开始出现 【信号】

这指向 Codex 可能在处理不绑定具体项目的对话 / 线程体验；但模块名只能说明方向，无法确认该能力是否已对用户开放，也无法确认具体交互形态。

### 6. UI 组件层补入 banner、checkbox、dropdown 与 tooltip-dismiss 【信号】

去噪后的前端新增模块包含 `banner`、`checkbox`、`dropdown`、`tooltip-dismiss`、`large-empty-state`，同时移除了 `sectioned-page`、`check-circle-filled` 等模块。

这指向界面组件库或页面结构有调整，可能涉及提示条、复选框、下拉菜单、可关闭提示与空状态页面。但这些都是前端模块名信号，无法从产物还原到源码级意图，也不能直接等同于新增了某个可见页面。

移除模块: sectioned-page, check-circle-filled

这指向差异展示相关 CSS（层叠样式表）被独立组织或引入新版构建中。它可能影响代码 diff 的样式组织方式，但事实包没有提供视觉截图或样式内容，因此不能判断具体显示效果。

关键证据：
