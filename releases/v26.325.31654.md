## ✨ 重点变化

### 1. 版本从 `26.325.21211` 推进到 `26.325.31654` 【实证】

应用的 short version 与 build 均发生变化：`CFBundleShortVersionString` 从 `26.325.21211` 变为 `26.325.31654`，`CFBundleVersion` 从 `1255` 变为 `1272`。

这意味着这是一个正式打包出的新 macOS arm64 桌面版本，而不是单纯的重新签名或外部元数据变化。

```diff
CFBundleShortVersionString: 26.325.21211 -> 26.325.31654
CFBundleVersion: 1255 -> 1272
```

### 2. bundle 文件树基本稳定，仅有少量内容改动 【实证】

顶层 bundle 文件数量保持 `414 → 414`，新增 `0`、移除 `0`、内容改动 `39`。应用层 `app.asar` 内部文件同样保持 `806 → 806`，新增 `0`、移除 `0`、改动 `6`。

这说明本次更新从文件结构上看非常克制，没有观察到新组件、新资源目录或大规模应用层重排。`webview/assets` 与 `.vite/build` 下的 hash 文件名变化被视为构建噪音，不作为功能增删依据。

### 3. 主 `codex` 组件仅小幅增大 【信号】

体积变化最大的非噪音组件是 `Contents/Resources/codex`，从 `148.2 MB` 增至 `148.3 MB`，变化为 `+0.05 MB`。

这指向底层可执行或资源内容有轻微调整，但无法从产物还原到源码级意图；仅凭该体积变化不能判断具体功能、性能或行为改变。

### 4. 未观察到新增 cua_node 依赖 【实证】

事实包中的 cua_node 依赖变化显示新增包数量为 `0`。

这意味着本次更新没有通过新增 cua_node 包来扩展可直接观察到的运行时依赖面；可感知变化更可能来自既有文件内容调整，而不是新依赖引入。

### 5. `app.asar` 完整性 hash 更新 【实证】

Info.plist 中 `ElectronAsarIntegrity/Resources/app.asar/hash` 从 `eb71a16a54eef07f5c843b16915673e18c0cde132335bc8758e30c17db3ad46a` 变为 `6744a67925299872f539233eab9a361abcccfd25a26a4d662e34e648efa0f91f`。

这与 `app.asar` 内部 `6` 个文件内容改动相匹配，说明 Electron 应用层产物确实发生了内容变化；但事实包没有提供这些改动的可读语义，因此不进一步推断功能意图。
