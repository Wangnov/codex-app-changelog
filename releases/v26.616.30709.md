# Codex 桌面版 26.616.30709 macOS / Windows 对照

> 第三方跨平台变更日志 · 非官方

这一批更像是“自动化能力面”扩张：浏览器、Sites、实时语音、下载、插件目录、项目/云聊天和 PR 工作流都在移动。  
One-liner: automation, browser downloads, deployment safety, realtime voice, and plugin/workflow surfaces all moved forward, with macOS adding Record & Replay.

## 关于这份报告

本文只基于两份官方签名分发包逆向重建后的机器差异事实：macOS arm64 从 `26.611.62324` build `4028` 到 `26.616.30709` build `4108`，Windows x64 从 `26.611.8604.0` 到 `26.616.3309.0`。两个平台版本号不完全对齐，所有结论分别引用各自事实，不把一端变化套到另一端。

证据分两档：

- 【实证】：来自 plist、manifest、CSP、技能文档、类型声明、package.json、依赖清单、插件清单、服务入口、签名信息，以及事实包中不属于“同名前端模块改动线索”的可读字符串。
- 【信号】：来自前端模块名、模块体积、同名前端模块字符串/样式差异等压缩 bundle 方向线索。它能说明包内出现了相关 UI/状态/交互线索，但不等同于源码级结论，也不保证所有用户可见。

平台归属使用 `[macOS+Windows]`、`[仅 macOS]`、`[仅 Windows]`。哈希文件名变动、Vite 构建产物重命名、孤立资源改名不作为功能增减。

## ✨ 重点变化

### 【实证】[仅 macOS] Record & Replay 成为新的内置插件

macOS 新增内置插件 `record-and-replay`，内置插件列表从 `browser`, `chrome`, `computer-use`, `latex`, `sites` 扩展为再加 `record-and-replay`。插件文件数 `168`，总大小 `56.8 MB`，版本 `1.0.829`，显示名为 `Record & Replay`。

这不是单纯包体增大。插件 manifest、marketplace 条目、服务入口和技能说明都指向一个完整能力面：记录用户在 Mac 上的鼠标点击、输入文本和交互窗口内容，最长 `30` 分钟，并将事件流转化为可复用 skill。默认提示包括“Record my workflow and turn it into a reusable skill”等。MCP 服务为 `event-stream`，入口指向 `SkyComputerUseClient`。

平台差异很清楚：macOS 包新增了 `plugins/record-and-replay` 目录和一套 Computer Use app 运行件；Windows 事实包没有对应新增插件明细，Windows marketplace diff 只显示 LaTeX 分类调整。

### 【实证】[macOS+Windows] Browser / Chrome 安全确认规则更明确

两端的 `browser` 和 `chrome` 确认文档都把第一类规则从“Hand-Off Required”重构为“Unsupported Actions”，并明确要求对绕过浏览器或 Web 安全边界的行为“Find an Alternative”。例如 HTTPS 安全插页绕过、paywall bypass 被归入不应绕过的安全边界；提交改密码的最终步骤仍归为需要用户接管。

两端的浏览器技能说明也同步强调：当用户明确要求导航到某页面而登录阻塞任务时，不要切换到 Web 搜索、搜索引擎、其他网站或其他来源绕过登录；如果环境支持安全浏览器认证则使用认证流，否则请用户登录。

Chrome 技能还新增 session naming 要求：每个 Chrome 浏览器任务 setup 后、打开或 claim tab 前，要调用 `browser.nameSession("...")`，使用短任务名。

### 【实证】[macOS+Windows] Browser API 新增媒体下载能力，下载设置进入主进程和 UI 文案

两端 `browser/docs/api.md` 与 `.codex-plugin/api-multi-tab.md` 都新增 `downloadMedia(options: LocatorDownloadMediaOptions)`，说明可对第一个匹配元素触发媒体下载，并新增 `LocatorDownloadMediaOptions` 类型。

主进程新增字符串同时显示下载设置进入包内：`Choose Downloads Folder`、`Folder where files downloaded by the in-app browser are saved`、`Whether manual browser downloads ask where to save each file`、`Failed to publish downloads state to renderer`。前端设置线索也同步出现 `Download history`、`Ask where to save downloads`、`System Downloads folder`、`Unable to change the downloads folder`、`Browser download history cleared` 等。

平台差异主要在调试面：macOS `debug-modal` 明确出现 `Cancel download`、`Open download`、`Pause download`、`Resume download` 等下载调试动作；Windows 的 `debug-modal` 变化更大，除下载相关方向外还包含大量 App Server、实验、host 许可和环境状态字符串。

### 【实证】[macOS+Windows] Sites 部署策略改为优先私有部署、共享/公开部署需显式批准

两端 `sites-hosting/SKILL.md` 都新增 `Deployment Selection` 和 `User-Facing Handoff`。保存 site version 后，除非用户明确要求保留本地，否则应部署；如果工具目录中有 `deploy_private_site_version`，正常 Sites 流程优先调用它，并由服务端验证 owner-only 访问。若私有部署不可用，或站点已共享、公开、无法验证为仅所有者可见，则不得自动调用 `deploy_site_version`，需要用户批准。

交付方式也更明确：成功部署后返回部署 URL 和简短结果，不在聊天里附本地源码、构建输出、上传包或单个代码文件，除非用户明确要求。Windows 还显示 `sites` 插件版本从 `0.1.15` 升到 `0.1.16`；macOS 事实包没有对应 sites plugin.json 版本差异，但技能文档改动一致。

### 【实证】[macOS+Windows] LaTeX 插件升级并重新归类

两端 `latex` 插件版本都从 `0.2.2` 升到 `0.2.3`，分类从 `Research` 改为 `Education & Research`。macOS marketplace 也同步新增 `record-and-replay` 条目，并把 LaTeX 的 marketplace 分类改为 `Education & Research`；Windows marketplace diff 只体现 LaTeX 分类调整。

### 【实证】[macOS+Windows] 实时语音新增唤醒词、热键和头像悬浮控制路径

两端主进程新增字符串包括 `codex.command.realtimeVoice`、`Failed to apply realtime voice hotkey`、`Failed to start realtime voice wake word`、`Realtime voice wake word detected`、`Realtime voice wake word needs microphone access`、`Realtime voice wake word helper listening/ready/reported an error`。这说明实时语音热键与唤醒词 helper 路径已进入包内。

前端同名模块线索进一步显示头像悬浮控件加入麦克风和输出控制：`Mute microphone`、`Unmute microphone`、`Mute realtime voice`、`Unmute realtime voice`。macOS 包还新增 `Contents/Resources/native/realtime-voice-wake-word` 文件。Windows 额外在 `use-avatar-overlay-selection` 中出现实时语音向其他线程发送消息、创建线程、委派请求等文案，是 Windows 样本里更强的头像/线程委派线索。

### 【实证】[macOS+Windows] 插件目录与连接器权限文案更细

两端 `plugins-page` 出现 `By OpenAI`、`By your workspace`、`Created by you`、`OpenAI internal`、workspace-provided 等目录归属文案，替换了部分 imported plugins / marketplace 相关文案。`plugins-availability` 出现连接器连接弹窗文案：管理员批准、个性化开关、隐私说明、站点可能尝试窃取数据、连接器可能引入风险、管理员禁用等。

这组变化是【信号】与【实证】混合：文案本身在前端模块线索中，属于 UI 方向信号；但插件 manifest 和 marketplace 的 LaTeX/record-and-replay 条目是配置级实证。

### 【信号】[macOS+Windows] Composer 附件入口转向多文件、照片、远程文件和 Appshot

两端 `composer` 同名模块都新增 `Add files`、`Add photos`、`Add remote files`、`Attach appshot`，并新增 `pick-files`、`pick-local-files`、`pick-remote-files`、`multi-file` 等样式/状态类。旧的截图/注释附件文案被迁出到 `cube` 模块线索中。

这说明 composer 的附件入口可能被拆分或重组：本次不是简单新增一个按钮，而是本地文件、远程文件、照片、Appshot 捕获状态共同进入同一片交互区域。但因为证据来自同名前端模块差异，只能标为信号。

### 【信号】[macOS+Windows] 侧栏、任务、云聊天和项目入口在统一

两端 `app-main` 新增 `Cloud chats`、`ChatGPT project`、`Menu item that creates a ChatGPT project from the unified sidebar`、`List label for cloud chats in the sidebar`、`Manual order`、`Get to know Codex`。`header` 在 macOS 样本中还出现 `All tasks`、`Cloud tasks`、`All environments filter option`、recent tasks menu、settings 入口和 Chrome extension thread 入口等。

这组变化指向 sidebar/header 的任务与云聊天整合，但没有源码语义，仍是 UI 信号。Windows 事实包没有列出 `header` 同名模块改动块，因此这里只把核心 `app-main` 归为双平台，把 `header` 细项放到低层索引。

### 【信号】[macOS+Windows] PR / CI 工作流更细：评论处理、检查修复、缺失信息提示

两端 `local-conversation-thread` 出现 `Address all actionable attached PR feedback.`、`Address every actionable comment without asking the user which ones to handle.`、冲突解决后运行最窄验证、提交并推送等指令文案，同时有 `missing-branch-info`、`missing-pr-info`、`pull-request` 等状态类。前端新增模块里也有 `pull-request`、`pull-request-checks-summary`、`pull-request-status`、`pull-request-visual-state`。

Windows 额外在 `pull-requests-page` 出现 reviewer 分组、`Awaiting approval`、`Changes requested`、单个失败检查修复动作 `Per-check action label for fixing a single failing pull request check`。因此 PR 基础方向是双平台信号，Windows 这份样本的 PR 页面细节更丰富。

### 【信号】[macOS+Windows] 自动化历史从暂停/恢复转向归档、批量归档、标记已读

两端 `automations-page` 新增 `Archive all`、`Context menu item to archive an automation history row`、`Dropdown menu item to mark all automation runs as read`、`Action label to unarchive an automation history thread`、`Custom schedule`，移除了暂停/恢复/删除自动化的一组菜单文案。这指向 automation history 的信息架构调整：运行历史处理更偏归档和阅读状态，而不是直接管理自动化本体。

### 【信号】[macOS+Windows] 远程连接加入 WSL 错误与离线删除规则

两端 `remote-connections-settings` 新增 `WSL connection failed`、`settings.remoteConnections.details.wslDistro`、`settings.remoteConnections.deviceConnections.wslSubtitle`、`Only offline environments can be deleted`、`SSH config entry not found`。主进程也新增 `Only offline remote control environments can be deleted.`。这是远程环境管理规则和 WSL 行为进入 UI/错误路径的信号与字符串实证组合。

### 【实证】[仅 macOS] macOS 新增 Desktop 权限说明

macOS `Info.plist` 新增 `NSDesktopFolderUsageDescription = Codex needs access to your Desktop for the task you selected`。结合 onboarding 模块中 `Button label while conversational onboarding loads Desktop access configuration`、`requests Desktop access` 等字符串，macOS 侧有明确的系统权限声明变化。Windows 没有对应 AppxManifest 变化块。

### 【实证】[macOS+Windows] native/runtime 与依赖装载路径更新

两端 `package.json` 都把产品版本更新为 `26.616.30709`，`codexBuildNumber` 从 `4028` 到 `4108`，`e2e:computer-use-native-pipe` 从直接 `node` 改为 `node --import tsx`。两端 `owl-electron-app.json` 的 `runtimeArchiveSha` 均变化，说明 owl runtime archive 发生替换。

macOS 额外显示 `node-gyp-build` 从 `4.6.0` 到 `4.8.4`，新增 imports 映射，并调整 native addon 解析、tag 匹配与 Windows shell spawn 行为。Windows 事实包没有列出这组 node-gyp-build 具体文件 diff，但 Windows 包体中 `resources/codex.exe`、`resources/codex` 和 `resources/app.asar` 都明显增大。
