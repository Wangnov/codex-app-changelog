# Codex 桌面版 26.616.30709(macOS / Windows 对照)

> 第三方跨平台变更日志 · 非官方

本批次最重要的结论是：macOS 侧对比结果完全相同，Windows 侧则出现了明确的版本推进、应用层变更和打包清理。

One-liner: macOS shows an identical-package diff, while Windows carries the observable update in this batch.

## 关于这份报告

本报告基于两份官方分发产物的逆向 diff：macOS 使用 Sparkle 体系下的官方全量包对比，Windows 使用 MSIX 分发包对比。macOS 本批次是历史回填，事实包显示上一版与最新版下载地址、SHA-256、版本号和 build 均相同；Windows 则从 `26.611.8604.0` 对比到 `26.616.3309.0`。

证据等级分两档：【实证】表示来自 plist、manifest、技能文档、类型声明、依赖清单、可读字符串、文件清单或签名校验等可直接核验的信息；【信号】表示来自模块名、体积变化或压缩 bundle 中的方向性线索，能提示变化方向但不能还原完整源码意图。

平台归属标记含义为：[macOS+Windows] 表示两平台事实包均支持；[仅 macOS] 表示只在 macOS 事实包中出现；[仅 Windows] 表示只在 Windows 事实包中出现。本批次未观察到可归为 [macOS+Windows] 的共同功能变化。

对比范围：macOS arm64 为 `26.616.30709` build `4108` 对 `26.616.30709` build `4108`；Windows x64 为 `26.611.8604.0` 对 `26.616.3309.0`。发布日期按本批次输入为 `2026-06-18`。

## ✨ 重点变化

### 【实证】[仅 Windows] Windows 包发生版本推进，macOS 包未发生内容变化

Windows 从 `26.611.8604.0` 更新到 `26.616.3309.0`，文件树从 `11943` 个文件降至 `9216` 个文件，新增 `2` 个、移除 `2729` 个、内容改动 `36` 个。macOS 侧上一版与最新版均为 `26.616.30709` build `4108`，文件数 `8416 → 8416`，新增、移除、内容改动均为 `0`。

这意味着本批次的可观察变化集中在 Windows 包；macOS 事实包只能证明该对比样本没有变化，不能据此推断 macOS 没有在其他批次获得同类改动。

### 【实证】[仅 Windows] 应用层 app.asar 有实质改动

Windows 的 `resources/app.asar` 从 `157.6 MB` 增至 `159.6 MB`，增加 `1.94 MB`；内部文件从 `3024` 增至 `3078`，新增 `1682`、移除 `1628`、改动 `27`。事实包同时提示 `webview/assets` 下大量增减属于 Vite 构建 hash 重命名噪音，不应按功能增减解读。

macOS 的 `app.asar` 内部文件保持 `3115 → 3115`，新增、移除、改动均为 `0`。因此，应用层变化仅能在 Windows 侧确认。

### 【实证】[仅 Windows] 浏览器控制与多标签能力继续扩展

Windows 新包中出现或改动了多项浏览器相关文件与字符串，包括 `resources/plugins/openai-bundled/plugins/browser/.codex-plugin/api-multi-tab.md`、`resources/plugins/openai-bundled/plugins/browser/docs/api.md`、`resources/plugins/openai-bundled/plugins/chrome/docs/capabilities/browser/viewport.md`，以及 `Open link in new tab`、`Copy link address`、`Locators must belong to the same tab`、`Target.attachToTarget did not return a sessionId` 等可读字符串。

这些证据指向内置浏览器、Chrome 插件、多标签 API、viewport 能力和 Playwright locator（页面元素定位器）校验的调整。macOS 侧未出现对应文件或内容变化。

### 【实证】[仅 Windows] 下载目录与浏览器下载设置出现新增文案

Windows 新增可读字符串包括 `Choose Downloads Folder`、`Folder where files downloaded by the in-app browser are saved`、`Whether manual browser downloads ask where to save each file`，并出现 `Failed to publish downloads state to renderer`、`allowDownload requires a non-empty url`、`allowDownload requires an integer tabId`。

这组证据表明 Windows 包新增或调整了内置浏览器下载目录、手动下载保存行为和下载状态发布相关逻辑。macOS 事实包没有对应变化。

### 【实证】[仅 Windows] 实时语音与唤醒词相关路径出现

Windows 新增字符串包含 `codex.command.realtimeVoice`、`codex.commandDescription.realtimeVoice`、`Failed to apply realtime voice hotkey`、`Failed to start realtime voice wake word`、`Realtime voice wake word detected`、`Realtime voice wake word needs microphone access`、`Realtime voice wake word helper ready` 等。

这些字符串确证 Windows 包中存在实时语音命令、快捷键、唤醒词 helper 和麦克风权限相关路径。事实包没有提供完整源码语义，也没有显示 macOS 同步变化。

### 【实证】[仅 Windows] 自定义文件处理与外部会话导入相关逻辑出现

Windows 新增字符串包括 `Custom file handlers for opening files from Codex App`、`Dropping invalid custom file handler`、`Dropping invalid custom file handlers config`、`External agent session import failed`、`Invalid Library file`、`Library file is unavailable`、`Selected CSV file is unavailable`。

这表明 Windows 包内出现了自定义文件处理、外部 agent session 导入、Library 文件和 CSV 文件可用性校验相关路径。macOS 侧没有文件或字符串层面的同步证据。

### 【实证】[仅 Windows] Native pet / avatar overlay 相关错误与恢复路径增加

Windows 新增字符串包括 `Failed to detach native pet material`、`Failed to dump native pet hierarchy`、`Failed to prewarm avatar overlay`、`Native pet composition recovery attempt`、`Native pet composition recovery failed`、`Native pet composition recovery succeeded` 等。

这些是可读错误和恢复状态文案，说明 Windows 包内存在 native pet 组合、材质、层级 dump 和 avatar overlay 预热相关代码路径。macOS 无对应 diff。

### 【实证】[仅 Windows] 包体清理：大量 node-headers / OpenSSL 头文件被移除

Windows 文件数减少 `2729` 个，移除清单中大量条目位于 `node-headers/include/node/openssl/archs/...`，还包括 `node-headers/Release/node.lib` 和旧的 `resources/icon.ico`。同时新增 `resources/codex-tray.ico`。

这更像是 Windows 打包内容清理和图标资源调整，而不是运行时能力本身的直接变化。macOS 包没有对应文件树变化。

### 【实证】[仅 Windows] 主要可执行组件体积上升

Windows 中 `resources/codex.exe` 从 `307.7 MB` 增至 `318.0 MB`，增加 `10.29 MB`；`resources/codex` 从 `271.2 MB` 增至 `278.4 MB`，增加 `7.17 MB`。`resources/app.asar` 同时增加 `1.94 MB`。

macOS 的 Computer Use 运行时总量保持 `365.9 MB → 365.9 MB`，Windows 的 Computer Use 运行时总量保持 `285.6 MB → 285.6 MB`，两平台 `cua_node` 均显示新增依赖包 `0` 个。因此，本批次不能把 Windows 可执行体积上升归因于 `cua_node` 新增依赖。
