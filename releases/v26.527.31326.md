# Codex 桌面版 `26.527.31326`(build 3390)

> 第三方变更日志 · 非官方 · 基于官方更新包逆向差异

这次从 `26.527.30818` 到 `26.527.31326` 的变化非常克制：文件数量不变，依赖不变，可读差异主要集中在版本号与打包完整性哈希更新。  
English one-liner: a minimal maintenance-style build with no directly evidenced user-facing feature additions.

## 关于这份报告

这不是 OpenAI(人工智能公司) 官方变更日志，而是一份**基于官方分发产物的逆向差异报告**。本次是历史回填：两个版本均已超出 Sparkle(应用自动更新框架) 增量窗口，因此不是通过增量包重建，而是下载两个官方全量包后直接比较。

可复现链路为：获取上一版与最新版官方全量包 → 以 Apple(苹果公司) 公证和 OpenAI Developer ID(开发者身份签名) 作为信任根 → 对 `Codex.app` 做结构化 bundle(应用包) 差异。事实包显示，两个版本均通过 `codesign`(代码签名检查) / `spctl`(系统安全评估工具) 公证验证，来源均为 `Notarized Developer ID`，签名主体为 `Developer ID Application: OpenAI OpCo, LLC (2DC432GLL2)`。

本文使用两档证据等级：

- **【实证】**：来自人类可读的直接证据，例如 `Info.plist` 键值、插件配置、依赖清单、签名与公证结论。
- **【信号】**：只来自间接证据，例如前端模块名或二进制体积变化；这类信息只能说明“指向某方向，但无法从产物还原到源码级意图”。

对比范围：`26.527.30818` build `3370`，发布于 `2026-05-29`；到 `26.527.31326` build `3390`，发布于 `2026-05-30`。平台为 macOS(苹果桌面操作系统) arm64(Apple Silicon 架构)。

## ✨ 重点变化

### 1. 应用版本与 build 号推进 【实证】

这是一个版本推进构建：`CFBundleShortVersionString` 从 `26.527.30818` 更新到 `26.527.31326`，`CFBundleVersion` 从 `3370` 更新到 `3390`。

可感知点是：用户和开发者在系统信息、应用包元数据、插件版本声明里会看到新的版本号，但事实包没有给出新增功能、权限或依赖能力的直接证据。

关键证据：

```diff
/CFBundleShortVersionString: '26.527.30818' → '26.527.31326'
/CFBundleVersion: '3370' → '3390'
```

### 2. Browser 与 Chrome 插件跟随主应用版本更新 【实证】

内置的 Browser(内置浏览器自动化插件) 与 Chrome(谷歌浏览器自动化插件) 配置文件中，插件版本号同步从 `26.527.30818` 更新到 `26.527.31326`。

这意味着插件包随主应用一起重新打包发布；但除版本字段外，事实包未显示插件描述、能力说明或接口行为发生变化。

关键证据：

```diff
"version": "26.527.30818"
→
"version": "26.527.31326"
```

### 3. app.asar 内容发生少量变化，但没有可归纳的新能力证据 【实证】

`app.asar`(Electron 应用代码归档) 的完整性哈希发生变化，说明应用层归档内容确实更新过；同时 bundle 文件树显示总文件数保持 `1982 → 1982`，新增 `0`、移除 `0`、内容改动 `51`。

可感知点是：这是一次真实重新打包或小幅内容更新，而不是只改外层版本号；但事实包中可读 diff 仅能支撑“版本与打包内容更新”，不能支撑“新增功能”结论。

关键证据：

```diff
/ElectronAsarIntegrity/Resources/app.asar/hash:
'efd27b4e1ae7bb61d8d058c92edaaf3b3a809cb5dbdc1cfd321bbd192dec50ee'
→
'7b24b4099e5ac419d3b02d8928ab1422f8d1b650cb51a1af754e8bc2f4ff6e85'
```

### 4. 没有新增 cua_node 依赖 【实证】

事实包显示 `cua_node` 依赖新增数量为 `0`。因此，本版没有直接证据表明底层能力包在依赖层面扩张。

这对开发者的含义是：至少从依赖清单看，本次更新不像是引入了新的 PDF、OCR、图像处理或浏览器自动化能力栈；如果存在行为变化，也没有在新增依赖层面体现出来。

关键证据：

```text

```
