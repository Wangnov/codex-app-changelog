## ✨ 重点变化

### 版本与构建号推进

【实证】[macOS+Windows]

macOS 从 `26.527.30818` 更新到 `26.527.31326`，`CFBundleVersion` 从 `3370` 更新到 `3390`。Windows 从 `26.527.3378.0` 更新到 `26.527.3686.0`。

平台差异在于版本体系不同:macOS 使用 short version(短版本号)+ build(构建号)，Windows 使用完整包版本号。两者都属于同一批次，但不能把 Windows 的版本号直接换算成 macOS build。

### 应用层 app.asar 重打包，但未见可确认功能增减

【信号】[macOS+Windows]

两平台的 `app.asar` 都发生了内部文件变动。macOS 内部文件数保持 `2774 → 2774`，新增 `289`、移除 `289`、改动 `12`；Windows 内部文件数保持 `2722 → 2722`，新增 `289`、移除 `289`、改动 `11`。

平台差异很小:两边新增/移除数量完全一致，但改动文件数 macOS 多 1 个。事实包明确提示 `webview/assets` 与 `.vite/build` 下的大量变化属于 Vite(前端构建工具)hash(哈希)重命名噪音，因此不能据此判断新增功能。

### macOS bundled plugin 元数据版本同步

【实证】[仅 macOS]

macOS 包内 `openai-bundled` 插件的 `browser` 与 `chrome` 配置文件版本号从 `26.527.30818` 更新到 `26.527.31326`。这说明 macOS 分发包里的 Browser(内置浏览器自动化插件)与 Chrome(谷歌浏览器自动化插件)元数据跟随应用版本同步。

平台差异很明确:Windows 事实包没有给出对应插件配置 diff，因此不能声称 Windows 同步了同样的插件元数据；只能说 macOS 有可读证据，Windows 在本批次事实包中缺少同等级证据。

### 依赖层没有新增 cua_node 包

【实证】[macOS+Windows]

两平台的 `cua_node` 依赖变化均显示“新增 0 个包”。

平台差异:无。至少从事实包列出的依赖层看，本批次没有出现新增 `cua_node` 包。

### 外层文件树稳定，主要变化集中在内容更新

【实证】[macOS+Windows]

macOS 外层文件数保持 `1982 → 1982`，新增 `0`、移除 `0`、内容改动 `51`。Windows 外层文件数保持 `8648 → 8648`，新增 `0`、移除 `0`、内容改动 `13`。

平台差异:macOS 的内容改动文件数明显多于 Windows，但两者都没有外层文件新增或删除。结合 `app.asar` 体积显示 `+0.00 MB`，这更像内容替换与重构建，而不是引入大体积新组件。
