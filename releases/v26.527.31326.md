# Codex 桌面版 26.527.31326 macOS / Windows 对照

> 第三方跨平台变更日志 · 非官方

这一批次的核心是内置插件版本同步，同时两个平台的前端 bundle 呈现出明显不同的 UI 方向信号。  
One-liner: shared bundled plugin version bumps, with macOS surfacing more task/browser/Computer Use UI signals while Windows removes several matching UI traces in this sample.

## 关于这份报告

本报告只基于输入事实包：macOS 来自 26.527.30818 build 3370 → 26.527.31326 build 3390，Windows 来自 26.527.3378.0 → 26.527.3686.0。两个平台发布时间和版本号不完全对齐，因此所有结论分别引用各自事实，不把某一平台的变化外推到另一平台。

证据分两档：【实证】表示来自 plist、manifest、插件清单、依赖清单、服务入口、可读字符串或关键文件 diff；【信号】表示仅来自同名前端模块、体积变化、样式类名或压缩 bundle 方向线索。

平台归属标记为：[macOS+Windows]、[仅 macOS]、[仅 Windows]。webview/assets 与 .vite/build 的 hash 改名按事实包说明视为构建噪音，不当成功能增减。

## ✨ 重点变化

### 【实证】[macOS+Windows] 内置 Browser 与 Chrome 插件版本同步到 26.527.31326

两平台的 `browser` 插件 manifest 都从 `26.527.30818` 更新到 `26.527.31326`。插件描述继续强调 in-app browser、localhost / 127.0.0.1 / file:// 目标、截图、点击、输入、测试，以及显式 @browser / @browser-use 请求不能被 macOS `open` 或通用 web browsing 静默替代。

两平台的 `chrome` 插件 manifest 也从 `26.527.30818` 更新到 `26.527.31326`。描述保持为面向远程 URL、认证或 profile 依赖页面、现有 Chrome 标签页、cookie、扩展和 Codex Chrome Extension setup 的自动化能力。

平台差异：macOS 只给出了 `browser` 与 `chrome` 两个插件 manifest 的版本变化；Windows 除这两个之外，还给出了 `computer-use` 插件版本变化和 owl shell 安装记录变化。

### 【实证】[仅 Windows] Windows 的 Computer Use 插件版本同步，并刷新 owl shell 安装记录

Windows 样本中 `resources/plugins/openai-bundled/plugins/computer-use/.codex-plugin/plugin.json` 的版本从 `26.527.30818` 更新到 `26.527.31326`，描述为通过 Computer Use 从 Codex 控制 Windows 桌面应用。

同一 Windows 样本的 `.installed-owl-shell.json` 中 `installedAt` 从 `2026-05-29T16:39:46.972Z` 变为 `2026-05-29T21:44:41.484Z`，`cacheKey` 指向 `owl-b7c292defc716f761f309794f6e96fd278525d4e-codex-win32-x64.zip`，`archiveSha` 为 `dd57bb72e98b5fbfa35820db4c05bcaa67c0f6a17372696485d741fc0993d85b`。

平台差异：macOS 输入没有给出 Computer Use 插件 manifest 或 owl shell 记录变化，因此不能断言 macOS 有相同变化。

### 【信号】[仅 macOS] 任务菜单、云任务与环境筛选入口增强

macOS 的 `header` 模块体积增加 27,874 bytes，新增可读字符串包括 recent tasks menu、`All tasks`、`Cloud tasks`、`All environments filter option`、清除搜索、环境搜索失败和环境列表失败等。样式类名新增 `aria-live`、`aria-atomic`、多组 gap 与布局类，指向一个可搜索、可筛选、带错误态的任务/环境入口。

平台差异：Windows 的 `header` 模块方向相反，移除了相同的一批 recent tasks、cloud tasks、environment filter、clear search 等字符串，因此该 UI 线索在本 Windows 样本中是移除信号。

### 【信号】[仅 macOS] @ mention 菜单出现 Computer Use、Custom agents 与桌面应用能力分组

macOS 的 `at-mention-list` 模块新增 `Computer use`、`Custom agents`、`Desktop apps`、`Live agents`、`Mac apps`、`No results`，以及 app capability mention server / scope 的 section header 文案。它们共同指向 composer 里的能力提及列表，可能开始按当前应用能力、外部 agent 或桌面应用能力组织入口。

平台差异：Windows 的同名模块移除了 `Computer use`、`Custom agents`、`Desktop apps`、`Live agents`、`Mac apps` 等字符串，所以不能说 Windows 在这个样本里同步出现该入口；只能说 Windows 包内对应信号被移除。

### 【信号】[仅 macOS] Browser Use 设置扩展到历史访问、域名允许/阻止和上传下载域

macOS 的 `browser-use-settings` 模块增加 61,356 bytes，新增字符串包括 `Access history without asking`、embedded extension / password manager / site settings 的可访问标签，以及 `Add allowed domain`、`Add blocked domain`、`Add allowed download domain`、`Add blocked download domain`、`Add allowed upload domain` 等。

这说明 macOS 包内出现了更细的浏览器权限或站点策略设置线索，尤其是历史访问、扩展设置、密码管理器设置、站点设置，以及上传/下载域名 allow/block 规则。

平台差异：Windows 输入没有给出同名设置模块变化，不能推断 Windows 同步获得这些设置。

### 【信号】[仅 macOS] Markdown 渲染和引用交互增强：图片、视频、文件引用与选中文本加入聊天

macOS 的 `markdown` 模块增加 63,388 bytes，新增字符串覆盖 markdown 图片按钮、图片加载占位、图片失败 fallback、视频 fallback、无 alt text 的可访问标签、extensionless file citation chip、带行号的 file citation chip，以及 `Add to chat` 和“把 overlay 中当前选中文本加入聊天”的按钮标签。

这些信号指向 markdown 内容中的媒体加载状态、引用 chip 可访问性，以及从选中文本直接添加到聊天的交互增强。由于这些来自同名前端模块改动线索，属于 UI/交互信号，不等同于完整功能已对所有用户可见。

平台差异：Windows 输入没有给出 `markdown` 模块变化。

### 【信号】[仅 macOS] 线程侧栏出现 CI 诊断工作流提示和 PR / staging 文案

macOS 的 `thread-side-panel-tabs` 模块增加 291,755 bytes，新增字符串包括 `A pull request already exists for this branch`、`Accept edits to stage them`，以及一组 CI 处理指令：先检查安装的 skills 或 tools 是否能访问该 CI 系统，再检查 credentials、permissions、tokens、MCP 或 tool access，缺失时明确告诉用户缺什么，否则获取 external logs、诊断并做最小安全修复。

这些字符串进入包内是【实证】级可读文本，但由于它们出现在同名前端模块改动线索中，本条按模块变化整体列为【信号】：它强烈指向线程侧栏和自动化工作流开始承载 PR、暂存编辑和 CI 修复上下文。

平台差异：Windows 输入没有给出 `thread-side-panel-tabs` 变化。

### 【信号】[macOS+Windows] 线程侧栏 bottom panel / 新标签页动作线索被移除

macOS 与 Windows 的 `thread-app-shell-chrome` 模块都移除了 `Bottom panel`、打开 side chat、browser tab、review tab、timeline tab、terminal、workspace file browser 的 action label，以及 `(max-width: 920px)`。样式类名也都移除了 `aria-pressed`、`data-thread-side-panel-new-tab-action-label` 和一批面板布局类。

平台差异：两平台方向一致，都是移除信号；但这只是前端 bundle 线索，不能断言对应功能被产品层面下线。

### 【信号】[macOS+Windows] App generation 分享对话框线索被移除

两平台的 `appgen-share-dialog` 都移除了站点分享对话框里的用户/群组搜索、保存按钮 loading 标签、`Add people or groups`、workspace link、internet link、复制 live site URL、外部打开 live site 的可读字符串，同时移除若干布局类。

平台差异：两平台方向一致，都是分享对话框相关 UI 文案被移除的信号。

### 【信号】[仅 macOS] 插件详情页的添加、启用、分享相关文案被移除

macOS 的 `plugin-detail-page` 模块减少 130,713 bytes，移除了 `Add to Codex`、`Adding to Codex`、`App enabled`、`App disabled`、MCP server enablement toggle、section navigation，以及 workspace link 分享相关可访问标签。

平台差异：Windows 输入没有给出 `plugin-detail-page` 变化。macOS 这里是插件详情页 UI 文案移除信号，不代表插件系统能力被移除。

### 【信号】[仅 Windows] Debug modal 中调试、账号、host 与 worktree cleanup 文案被移除

Windows 的 `debug-modal` 模块减少 171,302 bytes，移除了 `(command unavailable)`、`Account ID`、`Active browser tab`、`Active conversation`、`Allow automatic worktree cleanup in this unpackaged build.`、`Allow host`、`Allow host starting`、`Already current` 等字符串，并移除一批 modal、状态色和 backdrop 样式类。

平台差异：macOS 输入没有给出 `debug-modal` 变化。

### 【信号】[仅 Windows] Worktree 初始化失败后的继续、设置和修复入口文案被移除

Windows 的 `worktree-init-v2-page` 模块减少 5,023 bytes，移除了本地环境 setup 失败后继续启动 thread、打开 local environment settings、启动 repair thread、取消 worktree setup 并开始 local conversation、`Continue anyway`，以及创建 worktree / fork conversation 的进度文案。

平台差异：macOS 输入没有给出 `worktree-init-v2-page` 变化。

## 🔎 低层信号索引

### 可访问性与 aria 状态

macOS 多个模块新增 `aria-label`、`aria-expanded`、`aria-haspopup`、`aria-pressed`、`aria-live`、`aria-atomic` 等类名或属性线索，集中出现在 thread side panel、markdown、browser settings、mention list、header。边界：这些是样式/状态信号，说明 UI 结构可能调整，不构成源码实现事实。

Windows 多个模块则移除相同类别的 aria 与布局类，尤其在 debug modal、mention list、header、thread shell、share dialog 中出现。边界同上。

### 浏览器设备工具栏

macOS `thread-side-panel-tabs` 新增 browser device toolbar 的 width、height、zoom select、close mode 可访问标签。边界：这是包内 UI 字符串信号，不能单独证明设备工具栏已对所有用户开放。

### 文件引用与媒体预览

macOS `markdown` 新增 line information citation chip、extensionless file citation chip、markdown image/video fallback、placeholder、preview trigger 和 selected text overlay 相关线索。边界：可读字符串能证明这些文案进入包内；模块上下文只提供 UI 方向信号。

### 插件与分享入口收缩

macOS `plugin-detail-page` 移除添加到 Codex、应用启用状态、MCP enablement toggle、workspace link 分享文案；两平台 `appgen-share-dialog` 同时移除站点分享文案。边界：这说明对应 bundle 文案变化，不等于权限模型或后端能力变化。

### Windows 本地环境与调试面板收缩

Windows `debug-modal` 与 `worktree-init-v2-page` 同时移除调试、host、worktree cleanup、本地环境修复和继续入口文案。边界：这是一组 Windows 前端模块移除信号，不能扩展成所有本地环境流程变化。
