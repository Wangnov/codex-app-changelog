English one-liner: a small maintenance release with frontend messaging and permission metadata changes.

## 关于这份报告

这不是官方变更日志，而是一份**基于官方分发产物的逆向差异报告**。本版属于历史回填：两个版本均已超出 Sparkle（macOS 应用更新框架）增量窗口，因此本次没有使用增量包重建，而是下载两个官方全量包后直接比较。

对比流程为：下载上一版官方全量包与最新版官方全量包 → 校验 SHA-256（安全散列值）→ 展开 Codex.app → 结构化 diff（差异比较）。事实包同时给出了 codesign（代码签名验证工具）/ spctl（macOS 安全评估工具）公证验证结果：两个版本均显示 `CSSMERR_TP_CERT_REVOKED`，即部分早期版本证书后续被吊销。这会影响系统安全安装判断，但不影响对 bundle 内容做差异分析。

本文使用两档证据等级：

- **【实证】**：来自人类可读的直接证据，例如 `Info.plist`、HTML（网页入口文件）diff、签名 / 公证验证结论。
- **【信号】**：来自间接证据，例如未混淆的前端模块名或二进制体积变化。它们只指向某方向，但无法从产物还原到源码级意图。

对比范围：`26.323.20928` build `1173`（2026-03-24）→ `26.324.21329` build `1214`（2026-03-25），平台为 macOS arm64。

## ✨ 重点变化

### 1. 前端通信层出现 `message-bus`，替代入口预加载中的 `logger` 【信号】

最新版 `webview/index.html` 中，入口预加载模块从 `logger-DGKjeFMx.js` 变为 `message-bus-DcBNa_PD.js`；去噪后的真实前端模块增删也显示新增 `message-bus`、移除 `logger`。

这指向前端内部通信或事件分发方式有调整，但无法从产物还原到源码级意图，也不能证明对应功能已经对用户开放。

关键证据：

```diff
- <link rel="modulepreload" crossorigin href="./assets/logger-DGKjeFMx.js">
+ <link rel="modulepreload" crossorigin href="./assets/message-bus-DcBNa_PD.js">
```

### 2. 蓝牙权限声明从旧键迁移到新键 【实证】

`Info.plist` 新增 `NSBluetoothAlwaysUsageDescription`，旧的 `NSBluetoothPeripheralUsageDescription` 被移除，文案保持为 `This app needs access to Bluetooth`。

这意味着应用的蓝牙权限声明方式发生了元数据层面的更新。可感知点是：如果系统触发蓝牙权限提示，提示依据会来自新的权限键；但事实包没有说明新增了任何蓝牙相关功能。

关键证据：

```text
新增 /NSBluetoothAlwaysUsageDescription = 'This app needs access to Bluetooth'
移除 /NSBluetoothPeripheralUsageDescription，旧值为 'This app needs access to Bluetooth'
```

### 3. 前端主题资源出现 `raycast-dark` / `raycast-light` 【信号】

去掉 Vite（前端构建工具）hash 噪音后，前端真实新增模块包含 `raycast-dark` 与 `raycast-light`。

这指向界面主题或配色资源方向的变化，但无法从产物还原到源码级意图，也不能证明用户界面中已经新增可见主题选项。

关键证据：

```text
新增模块: message-bus, raycast-dark, raycast-light
移除模块: logger
```

### 4. 启动页不再显式声明 `color-scheme: light dark` 【实证】

`webview/index.html` 的内联样式中移除了 `color-scheme: light dark;`。

这可能影响浏览器内建控件或系统主题适配的默认行为；从当前证据只能确认这行样式被删除，不能进一步推断具体视觉效果。

关键证据：

```diff
-        color-scheme: light dark;
```

### 5. 应用主体只有小幅体积变化 【信号】

顶层 bundle 文件数保持 `415 → 415`，没有新增或移除文件，内容改动 `40` 个。体积变化最大的组件是 `Contents/Resources/codex`，从 `146.5 MB` 到 `146.9 MB`，增加 `0.37 MB`；`app.asar` 从 `42.2 MB` 到 `42.2 MB`，增加 `0.07 MB`。

这指向一次小规模维护更新，而不是大范围组件替换；但二进制体积变化只能作为间接信号，无法从产物还原到源码级意图。
