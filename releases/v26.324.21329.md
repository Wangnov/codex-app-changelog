## ✨ 重点变化

### 1. 插件 / 技能页面走向统一浏览与创建入口 【信号】

`plugins-page` 出现一组与统一浏览、分类选择、创建入口和详情页返回路径相关的新文案，指向插件与技能管理体验正在合流，但无法从产物还原到源码级意图。

可感知点可能包括：浏览插件和技能的入口更统一，安装弹窗、移除确认、详情页返回、更多操作菜单和创建流程更完整。关键线索包括 `Choose whether to browse plugins or skills`、`Choose a category`、`Create plugin`、`Create skill`、`Adding to Codex`，以及状态类名 `has-opened-plugin-creator-prefill-v1`、`has-opened-skill-creator-prefill-v1`。

### 2. 应用主入口从 `logger` 预加载切到 `message-bus` 预加载 【实证】

`webview/index.html` 中的模块预加载列表发生明确替换：`logger-DGKjeFMx.js` 被 `message-bus-DcBNa_PD.js` 取代，同时去噪后的真实新增模块也包含 `message-bus`，移除模块为 `logger`。

这意味着前端应用的跨模块通信或事件分发基础设施可能发生调整。可确认的是入口 HTML 已预加载 `message-bus`，但具体通信语义不能从压缩产物直接还原。

```diff
- <link rel="modulepreload" crossorigin href="./assets/logger-DGKjeFMx.js">
+ <link rel="modulepreload" crossorigin href="./assets/message-bus-DcBNa_PD.js">
```

### 3. Automation thread 与远程连接相关体验出现新入口信号 【实证】

新版主进程 / 前端 bundle 新增可读字符串 `Failed to set automation thread title`，前端 `index` 模块新增 `Automation thread`、`Accessible label for the automation thread icon`，同时新增远程连接 banner 相关文案。

这说明产物中已经出现 automation thread 和远程连接提示的用户可见文案。可感知点可能包括线程图标、标题设置失败路径、首页 composer 上方远程连接横幅，以及 `Add Connections` 入口。

### 4. 新增窗口缩放菜单文案 【实证】

新版 bundle 新增 `Zoom In`、`Zoom Out`、`Actual Size` 三个可读字符串。

这通常对应桌面应用菜单或窗口视图缩放能力的用户可见文本。事实包只能确认这些字符串已进入新版产物，不能确认具体菜单位置或快捷键。

### 5. 新增 Raycast 主题变体 【信号】

去噪后的新增前端模块包含 `raycast-dark`、`raycast-light`，同名前端模块 `use-resolved-theme-variant` 也新增 `Raycast Dark` 与 `Raycast Light` 字符串。

这指向新增或准备接入 Raycast 风格的深色 / 浅色主题变体，但无法从产物还原到源码级意图，也无法确认是否已对所有用户开放。

### 6. macOS 蓝牙权限声明切换到新版键名 【实证】

`Info.plist` 中蓝牙权限描述从旧键 `/NSBluetoothPeripheralUsageDescription` 移除，改为新增 `/NSBluetoothAlwaysUsageDescription`，描述文本仍为 `This app needs access to Bluetooth`。

这属于 macOS 权限元数据调整。对用户可感知的部分是系统权限提示可能走新的 plist 键；事实包没有显示蓝牙相关功能本身发生变化。

### 7. 设置与 App Connect 文案出现个性化开关调整 【信号】

`control-group` 新增 `Accessible label for the personalization toggle in the app connect modal` 和 `settings.mcp.appConnectModal.personalizationToggle`，同时移除 `Continuous sync`、同步说明标题与描述相关字符串。

这指向 App Connect 弹窗从强调持续同步，转向包含个性化开关的配置项，但无法从产物还原到源码级意图。

### 8. 使用量 / 计费提示文案有替换 【信号】

`index` 模块新增 `Banner message for non-owners on usage-based seat plans when spend cap blocks usage`，并移除多条 credits upsell banner 文案，包括 Plus 计划、CBP 计划和 credits 用尽提示。

这指向使用量限制提示的表达方式发生调整，尤其是面向 usage-based seat plans 与 spend cap 的场景；但这些只是前端文案级信号，不能推断计费规则本身改变。
